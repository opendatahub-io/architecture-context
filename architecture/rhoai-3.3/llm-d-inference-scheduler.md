## Purpose
**Short**: Endpoint Picker (EPP) and P/D routing sidecar that makes optimized routing decisions for LLM inference requests via Kubernetes Gateway API.

**Detailed**: The llm-d Inference Scheduler provides two main binaries: the Endpoint Picker (EPP) and the P/D (Prefill/Decode) routing sidecar. The EPP extends the upstream Gateway API Inference Extension (GIE) project and acts as an external processor (ext-proc) for Envoy-based Gateway API implementations. It receives inference requests from the gateway, evaluates candidate model-serving pods through a pluggable pipeline of filters and scorers (based on KV cache locality, load, session affinity, etc.), and returns the optimal pod endpoint for request routing.

The P/D routing sidecar runs alongside vLLM decode workers and orchestrates disaggregated prefill/decode inference. When the EPP determines that a request should use separate prefill and decode workers, the sidecar intercepts the request on the decode pod, forwards the prefill stage to the selected prefill worker, and coordinates the decode stage locally. The sidecar supports multiple P/D connector protocols including NIXL v2, LMCache (deprecated), and SGLang, and includes SSRF protection via InferencePool-based allowlisting.

Both components are built with controller-runtime for Kubernetes integration and use Gateway API (InferencePool, HTTPRoute) as the core scheduling abstraction. The architecture is designed to be pluggable, with filters, scorers, and profile handlers that can be configured via YAML without code changes.

### SSRF Protection

The P/D routing sidecar implements SSRF protection by maintaining a dynamic allowlist of valid prefill targets:
- Watches InferencePool resources to discover pod selectors
- Watches matching Pods to build allowlist of pod IPs and hostnames
- Validates `x-prefiller-host-port` header against allowlist before forwarding
- Returns HTTP 403 if prefill target is not in the allowlist
- Enabled via `--enable-ssrf-protection` flag with `--inference-pool-namespace` and `--inference-pool-name`
