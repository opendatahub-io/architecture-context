## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that implements Ragas (Retrieval Augmented Generation Assessment) metrics for LLM evaluation.

**Detailed**: This component provides Ragas evaluation capabilities as a pluggable provider for the Llama Stack evaluation API. It enables users to evaluate LLM outputs using metrics such as answer relevancy, context precision, faithfulness, and context recall. The provider wraps Ragas' evaluation framework and integrates it with Llama Stack's inference, dataset, and benchmark APIs.

The provider ships in two execution modes: an **inline** provider that runs Ragas evaluation in the same process as the Llama Stack server, and a **remote** provider that offloads evaluation to Kubeflow Pipelines for scalable, distributed execution. The remote provider submits KFP pipelines that retrieve data from Llama Stack, run Ragas evaluations, and store results in S3-compatible storage.

The component is built as a container image (`odh-trustyai-ragas-lls-provider-dsp-rhel9`) using UBI9 Python 3.12 base for RHOAI distribution, and is used as the base image for Kubeflow Pipeline components in remote evaluation mode.

### Llama Stack Provider APIs

This component implements the Llama Stack `Eval` API interface:

| Method | Purpose | Supported Modes |
|--------|---------|-----------------|
| `run_eval` | Start an evaluation job for a registered benchmark | Inline, Remote |
| `job_status` | Check status of an evaluation job | Inline, Remote |
| `job_result` | Retrieve results of a completed evaluation job | Inline, Remote |
| `job_cancel` | Cancel a running evaluation job | Remote only |
| `register_benchmark` | Register a benchmark with dataset and scoring functions | Inline, Remote |
| `evaluate_rows` | Evaluate specific rows (not implemented) | Neither |

### Evaluation Metrics

| Metric Name | Ragas Function | Purpose |
|-------------|----------------|---------|
| answer_relevancy | `ragas.metrics.answer_relevancy` | Measures how relevant the answer is to the question |
| context_precision | `ragas.metrics.context_precision` | Measures precision of retrieved context |
| faithfulness | `ragas.metrics.faithfulness` | Measures factual consistency of answer with context |
| context_recall | `ragas.metrics.context_recall` | Measures recall of relevant context |

### Token Handling

The remote provider obtains Kubeflow Pipelines authentication tokens through a two-tier strategy:
1. **Explicit token**: `KUBEFLOW_PIPELINES_TOKEN` environment variable / config field (SecretStr type)
2. **Kubeconfig fallback**: Loads token from in-cluster config or local kubeconfig file via `kubernetes.config`
