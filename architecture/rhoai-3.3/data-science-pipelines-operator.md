## Purpose

**Short**: Kubernetes operator that manages the lifecycle of Data Science Pipelines (DSP) infrastructure, deploying and reconciling all components needed for ML pipeline execution on OpenShift.

**Detailed**: The Data Science Pipelines Operator (DSPO) is a Kubernetes operator built on the controller-runtime framework that watches `DataSciencePipelinesApplication` (DSPA) custom resources and deploys a complete ML pipeline execution stack per-namespace. When a user creates a DSPA CR, the operator reconciles the desired state by deploying an API server, persistence agent, scheduled workflow controller, ML Metadata (MLMD) services, database (MariaDB or external), and object storage (Minio or external S3-compatible), along with optional Argo Workflow controllers.

The operator uses Go template-based manifest rendering via the `manifestival` library to dynamically generate Kubernetes resources from templates stored in `config/internal/`. Each DSPA CR instance results in a namespaced set of pipeline infrastructure components, with pod-to-pod TLS enabled by default. The operator performs health checks against both the database and object storage before deploying application-level components, and exposes Prometheus metrics for monitoring component readiness. Authentication for the API server is enforced via a `kube-rbac-proxy` sidecar that performs SubjectAccessReview authorization against the DSPA resource.

The operator is deployed by the `rhods-operator` using kustomize overlays. The RHOAI overlay at `config/overlays/rhoai/` targets the `redhat-ods-applications` namespace and includes both the base operator manifests and Argo Workflows CRDs/RBAC.

| Component | Type | Purpose |
|-----------|------|---------|
| DSPO Controller Manager | Go Operator (controller-runtime) | Watches DSPA CRs and reconciles all pipeline infrastructure components |
| DS Pipelines API Server | Deployment (per-DSPA) | REST/gRPC API for pipeline management, created dynamically from templates |
| kube-rbac-proxy | Sidecar container | RBAC-based authentication proxy for API Server and MLMD Envoy (port 8443) |
| Persistence Agent | Deployment (per-DSPA) | Syncs pipeline run status from Argo Workflows to the pipeline database |
| Scheduled Workflow | Deployment (per-DSPA) | Manages cron-scheduled pipeline runs |
| Workflow Controller | Deployment (per-DSPA, optional) | Argo Workflow controller for pipeline orchestration, managed via global config |
| MLMD gRPC Server | Deployment (per-DSPA, optional) | ML Metadata gRPC service for artifact/execution tracking |
| MLMD Envoy Proxy | Deployment (per-DSPA, optional) | Envoy proxy fronting the MLMD gRPC server |
| MariaDB | Deployment (per-DSPA, optional) | Default database for pipeline metadata; can be replaced with external DB |
| Minio | Deployment (per-DSPA, optional) | Default object store for pipeline artifacts; can be replaced with external S3 |
| DS Pipelines Webhook | Deployment (operator namespace) | Validating/mutating webhook for PipelineVersion CRs (when pipelineStore=kubernetes) |
