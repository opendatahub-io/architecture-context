## Purpose

**Short**: A lightweight Rust-based HTTP microservice that detects sensitive content (PII) in text using predefined and custom regular expressions.

**Detailed**: The Guardrails Regex Detector is a stateless HTTP service built in Rust using the Axum web framework. It provides a REST API endpoint that accepts text content and applies regex-based pattern matching to detect personally identifiable information (PII) such as email addresses, Social Security numbers, and credit card numbers. It also supports arbitrary custom regex patterns provided in the request payload.

This service is designed to integrate with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator) as a downstream detector. In the RHOAI guardrails pipeline, the orchestrator routes text content to this detector for PII scanning before or after LLM inference. The service is intentionally minimal — a single stateless binary with no database or external dependencies — optimized for low-latency regex evaluation at scale.

The container image is based on Red Hat UBI 9 Minimal for production deployments, ensuring compatibility with Red Hat's supported container ecosystem and FIPS compliance requirements.
