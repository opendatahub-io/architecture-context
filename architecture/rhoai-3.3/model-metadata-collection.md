## Purpose
**Short**: A Go-based build-time tool that extracts, enriches, and catalogs AI model metadata from Red Hat container images and HuggingFace, producing a static data container image consumed by the RHOAI Model Catalog.

**Detailed**: Model Metadata Collection is a Go application that runs as a build-time/CI pipeline tool (not a runtime operator or service). It discovers AI models from HuggingFace validated model collections, processes OCI container images (ModelCar format) from `registry.redhat.io` to extract model cards, enriches the extracted metadata with data from the HuggingFace API, and generates comprehensive YAML catalogs of model metadata.

The tool produces two key data files: `models-catalog.yaml` and `validated-models-catalog.yaml`. These catalogs are packaged into a minimal container image (`ubi9-minimal` base for Konflux builds, `ubi9-micro` for regular builds) that runs `sleep infinity` and serves as a data volume. Other RHOAI components (such as the Model Registry or Dashboard) mount this container to access the pre-generated model catalog data. The tool also includes a metadata reporting CLI (`metadata-report`) that generates completeness and data source analysis reports.

The repository also ships sample benchmark data (`sample-data/`) with model evaluation metrics in NDJSON format, organized by provider/model structure, which is included in the container image for benchmark consumption.
