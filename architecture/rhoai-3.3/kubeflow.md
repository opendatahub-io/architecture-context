## Purpose
**Short**: Manages the lifecycle of Kubeflow Notebook custom resources by reconciling StatefulSets, Services, and optionally Istio VirtualServices for each notebook instance.

**Detailed**: The Kubeflow Notebook Controller (kf-notebook-controller) is a Kubernetes operator that watches for `Notebook` custom resources (CRD group `kubeflow.org`) and reconciles the underlying infrastructure needed to run notebook servers. For each Notebook CR, the controller creates and manages a StatefulSet (with a single replica) to run the notebook pod, a ClusterIP Service for network access, and optionally an Istio VirtualService for ingress routing when Istio integration is enabled.

The controller also includes an optional idle-culling feature (CullingReconciler) that monitors notebook activity by querying the Jupyter `/api/kernels` and `/api/terminals` endpoints. When a notebook has been idle beyond a configurable threshold (default 1440 minutes / 24 hours), the controller sets a stop annotation on the Notebook CR, causing the StatefulSet replicas to scale to zero. The controller re-emits Pod and StatefulSet events onto the Notebook CR for better observability, and exposes Prometheus metrics for notebook creation, failure, and culling counts.

In the RHOAI OpenShift overlay deployment, Istio integration is disabled (`USE_ISTIO=false`) and `ADD_FSGROUP=false` is set. The upstream Kubeflow Istio VirtualService pattern is not used; instead, the companion odh-notebook-controller (deployed separately) handles OpenShift-specific ingress via Gateway API HTTPRoutes and kube-rbac-proxy sidecars in RHOAI 3.x.
