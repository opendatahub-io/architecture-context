## Purpose

**Short**: MLServer is a Python-based inference server implementing the V2 Inference Protocol (KFServing dataplane API) for serving machine learning models over REST and gRPC.

**Detailed**: MLServer is an open-source inference server originally developed by Seldon Technologies, forked and maintained by Red Hat for inclusion in Red Hat OpenShift AI (RHOAI). It provides a standardized way to serve machine learning models using the V2/Open Inference Protocol, which defines both REST (HTTP) and gRPC interfaces for model inference, health checking, model metadata, and model repository management.

The server supports multiple ML frameworks through a pluggable runtime architecture. The RHOAI distribution ships with three runtimes bundled in the container image: Scikit-learn (`mlserver-sklearn`), XGBoost (`mlserver-xgboost`), and LightGBM (`mlserver-lightgbm`). Additional runtimes exist in the codebase for MLflow, HuggingFace Transformers, Alibi Explain/Detect, CatBoost, and Apache Spark MLlib, though these are not included in the default Konflux build.

MLServer is deployed as a containerized workload within KServe `InferenceService` pods on OpenShift. It supports features like adaptive batching, parallel inference workers, response caching, Prometheus metrics export, OpenTelemetry distributed tracing, Kafka-based async inference, CORS configuration, GZip compression, and custom model environments via conda-pack tarballs. The server runs as non-root user (UID 1000) and is compatible with OpenShift's random UID assignment.

MLServer does not manage its own ingress. Ingress is managed by KServe (via Gateway API / HTTPRoute in RHOAI 3.x or Istio Gateway in RHOAI 2.x) which creates external-facing routes to the MLServer container.
