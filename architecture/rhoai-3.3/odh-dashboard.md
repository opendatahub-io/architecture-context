## Purpose
**Short**: The ODH Dashboard is the primary web UI for Red Hat OpenShift AI, providing a management console for data science workbenches, model serving, model registry, pipelines, and platform configuration.

**Detailed**: The ODH Dashboard serves as the centralized web interface for the OpenShift AI platform. It consists of a React/PatternFly 6 frontend and a Node.js/Fastify backend-for-frontend (BFF) that proxies Kubernetes API calls on behalf of authenticated users. The application runs as a Deployment with multiple containers: the main dashboard container (Node.js serving both the React SPA and the BFF API), a kube-rbac-proxy sidecar for authentication, and three federated UI module containers (model-registry-ui, gen-ai-ui, maas-ui) that provide modular architecture extensions.

The dashboard uses the RHOAI 3.x ingress pattern: traffic enters through a Gateway API HTTPRoute referencing the platform's `data-science-gateway` Gateway, is authenticated by the kube-rbac-proxy sidecar (which checks that the user can list OpenShift projects), and then proxied to the backend Node.js application. The backend proxies Kubernetes API requests using the user's forwarded access token, ensuring that all actions respect Kubernetes RBAC. The dashboard also manages several CRDs for configuration (OdhDashboardConfig), application catalog (OdhApplication), documentation (OdhDocument), and quickstarts (OdhQuickStart).

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| rhods-dashboard | HTTPRoute (Gateway API) | PathPrefix "/" | 8443/TCP | HTTPS | TLS (Gateway-terminated) | Passthrough to kube-rbac-proxy | External |
| odh-dashboard (legacy) | Route (OpenShift) | auto-generated | 8443/TCP (dashboard-ui) | HTTPS | TLS reencrypt | HSTS enabled | External (legacy/redirect) |

**HTTPRoute details**: The HTTPRoute references parent Gateway `data-science-gateway` in namespace `openshift-ingress`. All traffic matching path prefix `/` is routed to the `rhods-dashboard` Service on port 8443. The gateway-name is injected as a kustomize variable from params.env.

**Authentication flow (RHOAI 3.x)**:
1. External request arrives at Gateway API (`data-science-gateway`)
2. Gateway terminates TLS and routes to `rhods-dashboard:8443`
3. kube-rbac-proxy sidecar on port 8443 validates the user can list `projects` in `project.openshift.io`
4. If authorized, request is proxied to `localhost:8080` (dashboard backend) with `X-Auth-Request-User` and `X-Auth-Request-Groups` headers
5. Dashboard backend uses `x-forwarded-access-token` to make Kubernetes API calls on behalf of the user
