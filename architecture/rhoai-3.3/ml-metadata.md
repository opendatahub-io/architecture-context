## Purpose
**Short**: ML Metadata (MLMD) is a library and gRPC server for recording and retrieving metadata associated with ML workflows, including artifacts, executions, contexts, and their lineage relationships.

**Detailed**: ML Metadata (MLMD) provides a metadata store that tracks the lifecycle of ML artifacts (datasets, models, metrics), executions (training runs, transformations), and contexts (experiments, pipelines). It maintains a lineage graph connecting these entities through events and associations, enabling reproducibility, auditing, and provenance tracking of ML workflows.

In the RHOAI platform, MLMD serves as the backend metadata store for Kubeflow Pipelines. It provides a gRPC API that pipeline components use to record inputs, outputs, and execution state. The server connects to an external database (MySQL or PostgreSQL) for persistent storage, supporting schema migrations and SSL connections. The Red Hat distribution builds the server binary from source using Bazel on UBI9, producing a minimal container image (`odh-mlmd-grpc-server`) that exposes the gRPC service on port 8080.

The component is a pure data service with no Kubernetes operator, CRDs, or controller logic. It does not manage any Kubernetes resources dynamically. Its deployment is managed externally by the RHOAI operator (Data Science Pipelines controller).

## Configuration

### Server Configuration Modes

The server supports four configuration modes via the `--metadata_source_config_type` flag:

| Mode | Flag Value | Description |
|------|-----------|-------------|
| Default | `default` | Tries config file first, then MySQL flags, falls back to in-memory fake DB |
| Config File | `config_file` | Reads MetadataStoreServerConfig protobuf from file |
| MySQL | `mysql` | Uses `--mysql_config_*` flags for MySQL connection |
| PostgreSQL | `postgresql` | Uses `--postgres_config_*` flags for PostgreSQL connection |
