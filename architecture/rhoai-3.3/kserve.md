## Purpose
**Short**: KServe is a Kubernetes-native model serving platform that provides serverless inference, model management, and LLM inference orchestration for RHOAI.

**Detailed**: KServe is the core model inference serving component of Red Hat OpenShift AI. It provides a standardized, Kubernetes-native API for deploying and managing machine learning models at scale. The operator manages the full lifecycle of InferenceService custom resources, supporting multiple deployment modes including RawDeployment (default in RHOAI) and Serverless (Knative-based).

The component includes a dedicated LLMInferenceService controller (`llmisvc`) for large language model serving, which integrates with the Gateway API Inference Extension (GIE) for intelligent request routing via InferencePools and InferenceModels. The LLM controller supports single-node, multi-node (via LeaderWorkerSet), and disaggregated prefill/decode deployment topologies, using vLLM as the inference engine with llm-d for intelligent scheduling and routing.

KServe also manages InferenceGraphs for model pipeline orchestration, TrainedModels for multi-model serving, and a LocalModel system for caching models on nodes. The operator creates Istio VirtualServices or Gateway API HTTPRoutes for ingress routing, and injects storage-initializer init containers and kube-rbac-proxy sidecars for model downloading and authentication respectively.

**Controller-Managed Ingress Details**:
- **InferenceService (RawDeployment mode)**: Creates Istio VirtualService CRs for ingress routing by default. When `enableGatewayApi` is true, creates HTTPRoute CRs instead.
- **LLMInferenceService**: Always uses Gateway API. Creates HTTPRoute CRs referencing configured parent Gateways. Also creates Istio DestinationRule CRs for traffic policies. Integrates InferencePool/InferenceModel CRs from the Gateway API Inference Extension.
- **InferenceGraph**: Creates Knative Services (Serverless mode) or raw Deployments + OpenShift Routes for ingress.
