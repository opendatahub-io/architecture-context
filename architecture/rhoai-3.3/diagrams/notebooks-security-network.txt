RBAC Summary:
  This component (Notebooks/Workbench Images) does not define any RBAC
  resources. RBAC is managed by:
  - ODH Notebook Controller: Manages workbench pod lifecycle, injects auth
    proxy sidecars, creates Services and HTTPRoutes
  - rhods-operator: Deploys ImageStream manifests via kustomize overlay

Authentication Architecture:
  - RHOAI 3.x: kube-rbac-proxy sidecar injected by ODH Notebook Controller
    - Terminates TLS on 8443/TCP
    - Validates Bearer Token via Kubernetes SubjectAccessReview
    - Proxies to workbench container on localhost (plaintext)
  - RHOAI 2.x: oauth-proxy sidecar injected by ODH Notebook Controller
    - Terminates TLS on 8443/TCP
    - Validates OAuth token via OpenShift OAuth server
    - Proxies to workbench container on localhost (plaintext)

  IMPORTANT: All workbench images intentionally disable built-in
  authentication. Security is enforced entirely at the pod boundary
  by the injected auth proxy sidecar.

Secrets:
  - rhel-subscription-secret (Opaque) - RHEL subscription for RStudio
    BuildConfig builds, provisioned manually by admin, no auto-rotation

Container Image Registries:
  - registry.redhat.io/rhoai/ : N-1 shipped RHOAI images (10 images)
  - quay.io/modh/ : N-2 through N-6 historical images (35 images)
  - quay.io/opendatahub/ : N (latest) images (18 images)
