═══════════════════════════════════════════════════════════════════════════
TRUST BOUNDARIES
═══════════════════════════════════════════════════════════════════════════

1. External → Ingress
   - Trust level: Untrusted → DMZ
   - Enforcement: OpenShift Router, TLS termination
   - Protocol transition: HTTPS (external) → HTTPS (OAuth proxy)

2. Ingress → Authentication Layer
   - Trust level: DMZ → Authentication
   - Enforcement: OAuth Proxy validates OpenShift OAuth tokens
   - Protocol transition: HTTPS → HTTP (pod-local, OAuth validated)

3. Authentication Layer → Application Layer
   - Trust level: Authenticated → Trusted
   - Enforcement: OAuth proxy already validated user, Jupyter has no auth
   - Security: Pod network isolation, non-root UID, SCC restrictions

4. Application Layer → External Services
   - Trust level: Trusted → Untrusted (Internet)
   - Enforcement: TLS for all external connections, user-provided credentials
   - Protocols: HTTPS/TLS 1.2+ for all egress

═══════════════════════════════════════════════════════════════════════════
SECURITY CONSIDERATIONS
═══════════════════════════════════════════════════════════════════════════

Authentication:
- External access requires OpenShift OAuth authentication
- OAuth proxy enforces authentication before allowing access to workbench
- Jupyter Server authentication disabled (relies on OAuth proxy)
- Users must have namespace access (enforced by OpenShift RBAC)

Encryption:
- External traffic: TLS 1.2+ (HTTPS)
- Internal pod-to-pod: HTTP (no encryption, pod-local only)
- OAuth proxy to OAuth server: HTTPS/TLS 1.2+
- All egress: HTTPS/TLS 1.2+

Network Isolation:
- Workbenches run in user namespaces (multi-tenant isolation)
- ClusterIP services (internal only, no external exposure without route)
- OpenShift network policies enforce namespace boundaries

Container Security:
- Non-root user (UID 1001)
- Restricted SCC (no privileged operations)
- No host network/IPC/PID access
- Images scanned for vulnerabilities (Konflux CI/CD)
- Images signed (Konflux)

Data Protection:
- User data stored in PVCs (persistent, namespace-scoped)
- Secrets for credentials (Git, S3, API tokens)
- No secrets in environment variables (mounted as files)

Compliance:
- Images built from UBI9 (Red Hat Universal Base Image)
- FIPS mode available for UBI9 images
- Vulnerability scanning in CI/CD pipeline
- Multi-architecture support (x86_64, aarch64, ppc64le, s390x)
