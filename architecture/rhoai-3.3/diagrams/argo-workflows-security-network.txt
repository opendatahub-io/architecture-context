- argo-workflows-agent-ca-certificates (Opaque) - CA certs for agent comms, admin-provisioned, no auto-rotate
- SSO client-id/client-secret (Opaque) - OIDC SSO credentials, admin-provisioned, no auto-rotate

Authentication Modes:
- Bearer Token: ServiceAccount token validated via Kubernetes RBAC impersonation
- SSO (OIDC JWT): Optional, group-based RBAC via claim mapping
- Server mode: Uses argo-server's own ServiceAccount permissions
- /healthz, /metrics: No authentication required

Network Policies:
- No NetworkPolicy resources defined in base manifests
- Ingress managed by Data Science Pipelines Operator (not Argo directly)
