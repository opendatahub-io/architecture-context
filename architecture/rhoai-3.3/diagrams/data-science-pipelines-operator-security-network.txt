Network Policies (Per-DSPA):
- ds-pipelines-{name}:
  - Target: app=ds-pipeline-{name}
  - Allow port 8443 from all (authenticated proxy access)
  - Allow ports 8888/8887 from monitoring namespaces, DSP components,
    pipeline pods, workbenches
- ds-pipelines-envoy-{name}:
  - Target: app=ds-pipeline-metadata-envoy-{name}
  - Allow port 8443 from all (authenticated proxy access)
  - Allow port 9090 from DSP components
- mariadb-{name}:
  - Target: app=mariadb-{name}
  - Allow port 3306 from DSPO namespace operator, API Server, MLMD gRPC

RBAC Summary:
- ClusterRole: manager-role
  - ServiceAccount: controller-manager (redhat-ods-applications)
  - Key permissions:
    - datasciencepipelinesapplications.opendatahub.io: DSPA resources (full CRUD)
    - core: pods, secrets, configmaps, services, PVCs (full CRUD)
    - apps: deployments (full CRUD)
    - argoproj.io: workflows, workflowtasksets (full CRUD)
    - route.openshift.io: routes (full CRUD)
    - networking.k8s.io: networkpolicies (full CRUD)
    - rbac.authorization.k8s.io: roles, rolebindings, clusterroles (full CRUD)
    - authentication.k8s.io: tokenreviews (create)
    - authorization.k8s.io: subjectaccessreviews (create)
    - serving.kserve.io: inferenceservices (CRUD)
    - ray.io: rayclusters, rayjobs (CRUD)

- ClusterRole: argo-cluster-role
  - Bound per-DSPA for workflow execution
  - core: pods, pods/exec (full CRUD)
  - argoproj.io: workflows, workflowtasksets (full CRUD)

- ClusterRoleBinding: system:auth-delegator
  - Bound per-DSPA for API server and MLMD Envoy SAs
  - Enables SubjectAccessReview delegation
