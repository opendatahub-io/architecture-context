Service Mesh Configuration (Optional)
═══════════════════════════════════════════════════════════════════════════

mTLS Support:
  - Optional mTLS can be enabled for pod-to-pod communication
  - Requires service mesh integration (Istio/Maistra)
  - TLS certificates provisioned via OpenShift service-ca
  - Benefits: Encrypted communication between API Server, MariaDB, MLMD, Workflow Pods

Authentication Flow:
  1. External: OAuth Proxy validates OpenShift OAuth tokens
  2. Internal: Kube RBAC Proxy validates Bearer tokens via TokenReview API
  3. Database: Username/Password from secrets
  4. Storage: S3 Access/Secret Keys from secrets
  5. Kubernetes API: ServiceAccount tokens for controllers
