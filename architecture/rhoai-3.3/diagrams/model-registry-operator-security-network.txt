  - Internal TLS: TLS 1.2+ (kube-rbac-proxy)
  - Certificates: OpenShift Service CA or user-provided

kube-rbac-proxy:
  - TLS Version: TLS 1.2+
  - Certificate Source: OpenShift Service CA annotation or user-provided secret
  - Client Auth: Bearer token (validated via SubjectAccessReview)

Webhook Service:
  - TLS Version: TLS 1.2+
  - Certificate Source: cert-manager (webhook-server-cert)
  - Client Auth: mTLS (Kubernetes API Server client certificate)

PostgreSQL/MySQL (optional):
  - TLS Version: TLS 1.2+
  - Client Certificate: Optional (configured in ModelRegistry CR)
  - Server Certificate: Database server certificate

Security Hardening:
══════════════════════════════════════════════════════════════════════════

- Operator runs as non-root with restricted SCC (restricted-v2)
- Database passwords stored in Kubernetes Secrets (recommend external secret management)
- NetworkPolicies automatically created for database isolation
- gRPC endpoint deprecated and removed in v1beta1 (reduced attack surface)
- Admission webhooks enforce CR validation and apply secure defaults
- kube-rbac-proxy requires valid Kubernetes user/serviceaccount bearer tokens
- SubjectAccessReview ensures only authorized users can access REST API
