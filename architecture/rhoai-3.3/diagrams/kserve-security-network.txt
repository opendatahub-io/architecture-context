  - API Groups: route.openshift.io (Routes)

  - API Groups: keda.sh (ScaledObjects)
  - API Groups: leaderworkerset.x-k8s.io (LeaderWorkerSets)
  - Verbs: create, delete, get, list, patch, update, watch

- ClusterRoleBinding: kserve-manager-rolebinding
  - Role: kserve-manager-role
  - ServiceAccount: kserve/kserve-controller-manager

Service Mesh Configuration:
- PeerAuthentication: mTLS mode optional (can be set to STRICT per-namespace)
- AuthorizationPolicy: Per-InferenceService via annotations
  - Can enforce Bearer Token validation
  - Can integrate with Authorino for external authorization

Secrets:
- kserve-webhook-server-cert (kubernetes.io/tls)
  - TLS cert for webhook server
  - Provisioned by: cert-manager (selfsigned-issuer)
  - Auto-rotate: No (self-signed)

- storage-config (Opaque)
  - Default S3/GCS credentials for model download
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, etc.
  - Provisioned by: User/Admin
  - Auto-rotate: No

- Custom storage secrets (Opaque)
  - Per-InferenceService storage credentials
  - Referenced via serving.kserve.io/s3-secret-name annotation
  - Provisioned by: User
  - Auto-rotate: No

Network Policies:
- Default: Allow all (relies on Istio for network segmentation)
- Can be configured per-namespace for additional isolation
- Egress to S3/GCS/Azure typically requires NetworkPolicy allowing HTTPS/443

Observability:
- Metrics endpoint: 8443/TCP (controller)
  - Protected by: OAuth Proxy (kube-rbac-proxy sidecar)
  - Auth: Bearer Token (ServiceAccount)
  - Encryption: TLS 1.2+

- Inference metrics: 8000/TCP (runtime pods)
  - Scraped by: Prometheus via ServiceMonitor/PodMonitor
  - Protocol: HTTP
  - Auth: None (internal)

- Logging:
  - Request/Response logs → Knative Eventing Broker (8080/TCP HTTP)
  - Container logs → OpenShift logging stack
