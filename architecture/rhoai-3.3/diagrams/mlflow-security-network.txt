Authentication & Authorization Summary
================================================================================

Authorization Modes:
  1. SelfSubjectAccessReview (default)
     - Caller's bearer token used to evaluate access via K8s API
  2. SubjectAccessReview
     - Proxy provides x-remote-user / x-remote-groups headers
     - MLflow server SA performs the review on behalf of the user

Authorization Cache:
  - TTL: 300 seconds (configurable via MLFLOW_K8S_AUTH_CACHE_TTL_SECONDS)

Unprotected Paths (no auth required):
  - /health, /version, /metrics
  - /static/*, /favicon.ico
