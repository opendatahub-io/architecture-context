RBAC Summary:
- Dynamic test RBAC (created per test run):
  - kubeflow.org: pytorchjobs [get, list, watch, create, delete]
  - ray.io: rayclusters, rayjobs [get, list, watch, create, delete]
  - kueue.x-k8s.io: clusterqueues, localqueues, resourceflavors, workloads [get, list, watch, create, delete]
  - trainer.kubeflow.org: trainjobs, clustertrainingruntimes [get, list, watch, create, delete]
  - core: pods, services, configmaps, secrets, persistentvolumeclaims [get, list, watch, create, delete]
  - jobset.x-k8s.io: jobsets [get, list, watch, create, delete]
  - security.openshift.io: securitycontextconstraints [get, watch, list, create, update, patch, delete]

Authentication & Authorization:
- Kubernetes API: ServiceAccount Bearer Token → RBAC enforcement
- Ray Dashboard API: No auth (cluster-internal, network isolation)
- S3 Storage: AWS access key / secret key → IAM
- HuggingFace Hub: Bearer Token (HF_TOKEN) → Token auth

Secrets:
- AWS credentials (Opaque) - S3 access for dataset/model upload/download, manual provisioning, no auto-rotate
- HuggingFace token (Opaque) - Access to gated HuggingFace models, manual provisioning, no auto-rotate

Network Notes:
- No Service Mesh configuration (not an operator deployment)
- No Network Policies defined
- Inter-pod communication (DDP 29500/TCP, NCCL) is plaintext within cluster
- Ray cluster communication (6379, 8265, 10001) is plaintext within cluster
- All egress to external services uses TLS 1.2+
