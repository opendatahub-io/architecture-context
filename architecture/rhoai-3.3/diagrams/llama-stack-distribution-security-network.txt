RBAC Summary:
================================================================================

  No RBAC roles defined. This is a container image, not a Kubernetes operator.
  RBAC is managed by the deploying entity (Llama Stack K8s Operator or user).

================================================================================
Authentication & Authorization:
================================================================================

  Endpoint: /v1/* (all APIs)
  Auth Mechanism: None (server-level)
  Enforcement Point: None
  Policy: Server has no built-in auth; relies on platform-level auth
          (kube-rbac-proxy sidecar in RHOAI 3.x, OAuth proxy in RHOAI 2.x)

  IMPORTANT: The Llama Stack server MUST be deployed behind an authentication
  proxy. Direct exposure without auth is a security risk.

================================================================================
Security Notes:
================================================================================

  1. No TLS on server port (8321): TLS must be provided by platform ingress
  2. No built-in auth: Authentication must be enforced by proxy/sidecar
  3. PostgreSQL connection defaults to plaintext - configure TLS in production
  4. All cloud provider API keys stored as plaintext environment variables
  5. AWS Bedrock token has 12-hour expiry requiring manual rotation
  6. No automatic secret rotation for any credentials
