RBAC Summary:
============
ClusterRoles:
- odh-model-controller-role
  - Manages: InferenceServices, InferenceGraphs, ServingRuntimes, LLMInferenceServices
  - Creates: Routes, NetworkPolicies, ServiceMonitors, PodMonitors, RBAC, TriggerAuthentications,
    AuthPolicies, EnvoyFilters, Templates
  - Watches: DataScienceClusters, DSCInitializations, Gateways, HTTPRoutes, Kuadrants
  - Full CRUD on NIM Accounts (nim.opendatahub.io/v1)

- odh-model-controller-leader-election-role
  - ConfigMaps, Leases: full CRUD for leader election

- odh-model-controller-proxy-role
  - TokenReviews, SubjectAccessReviews: create (for kube-rbac-proxy)

- odh-model-controller-metrics-reader
  - Services/proxy: get (for metrics access)

ClusterRoleBindings:
- odh-model-controller-role-binding → odh-model-controller SA
- odh-model-controller-leader-election-role-binding → odh-model-controller SA
- odh-model-controller-proxy-rolebinding → odh-model-controller SA

ServiceAccount:
- Name: odh-model-controller
- Namespace: redhat-ods-applications (or deployment namespace)

Network Policies Created by Controller:
========================================
Pattern: InferenceService Network Isolation
- Selector: serving.kserve.io/inferenceservice label
- Ingress: Allow from Istio ingress gateway, Allow from same namespace
- Egress: Allow to Kubernetes API, Allow to model storage (S3/PVC)

Secrets:
========
- odh-model-controller-webhook-cert (kubernetes.io/tls)
  - Purpose: TLS cert for webhook server (9443/TCP)
  - Provisioned by: OpenShift service-ca-operator
  - Rotation: Automatic (90 days)
  - Annotation: service.beta.openshift.io/serving-cert-secret-name

- [NGC API Key Secrets] (Opaque)
  - Purpose: NVIDIA NGC API keys for NIM account validation
  - Provisioned by: User/Administrator
  - Referenced by: NIM Account CRs
  - Rotation: Manual

- [Model Pull Secrets] (kubernetes.io/dockerconfigjson)
  - Purpose: Container image pull secrets for NIM models
  - Provisioned by: NIM Account Controller (generated from NGC keys)
  - Rotation: Manual

Security Notes:
===============
1. Webhook server uses mTLS - Kubernetes API server presents client cert
2. Metrics endpoint currently HTTP (TODO: migrate to HTTPS on 8443)
3. Controller runs as non-root (UID 2000) in UBI9 minimal image
4. FIPS compliance: Built with strictfipsruntime tags
5. HTTP/2 disabled by default (CVE-2023-44487 mitigation)
6. Leader election enabled for HA (default: 1 replica)
7. OpenShift-specific: Requires Routes, Templates, service-ca for certs
8. Network Policies created per InferenceService for traffic control
