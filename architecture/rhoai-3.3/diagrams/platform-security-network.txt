================================================================================
  AUTHENTICATION PATTERNS
================================================================================

| Pattern                                    | Components                          | Mechanism                              | Port          |
|--------------------------------------------|-------------------------------------|----------------------------------------|---------------|
| Gateway API + EnvoyFilter + kube-auth-proxy| All Gateway-fronted services        | OAuth2/OIDC via ext_authz              | 443 -> 8443   |
| kube-rbac-proxy sidecar                    | TrustyAI, guardrails, dashboard     | SubjectAccessReview, FIPS TLS          | 8443          |
| K8s admission webhooks                     | rhods-operator, kserve, trainer     | API server TLS client cert             | 9443          |
| Bearer Token (app-level)                   | vllm-cpu, OVMS                     | SHA-256 token via --api-key            | 8000, 8080    |
| Istio mTLS                                 | model-registry, KServe              | DestinationRule MUTUAL                 | various       |
| API Key (MaaS)                             | models-as-a-service                 | Kuadrant Authorino validation          | via Gateway   |
