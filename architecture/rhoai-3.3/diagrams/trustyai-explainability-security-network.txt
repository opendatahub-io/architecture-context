RBAC Summary:
================================================================================
- ClusterRole: trustyai-role
  - API Groups: "" (core)
  - Resources: configmaps
  - Verbs: create, delete, get, list, patch, update, watch

- ClusterRoleBinding: trustyai-clusterrolebinding
  - Scope: Cluster-wide
  - Role: trustyai-role (ClusterRole)
  - ServiceAccount: trustyai-serviceaccount

Authentication & Authorization:
================================================================================
- Base manifests: No authentication on REST endpoints
- Production (RHOAI): kube-rbac-proxy sidecar injected by TrustyAI Operator
- Prometheus scrape: Bearer Token via ServiceMonitor (bearerTokenFile from SA)
- All internal service-to-service calls: No auth (plaintext HTTP)

Secrets:
================================================================================
- TLS certificate (kubernetes.io/tls)
  - Files: tls.crt, tls.key at /etc/tls/internal/
  - Purpose: HTTPS on port 4443
  - Provisioned by: External (cert-manager or operator)
  - Auto-Rotate: Unknown

- Database credentials (Opaque)
  - Keys: QUARKUS_DATASOURCE_USERNAME, QUARKUS_DATASOURCE_PASSWORD
  - Purpose: MariaDB/MySQL authentication
  - Provisioned by: External (operator/manual)
  - Auto-Rotate: No

Network Security Notes:
================================================================================
- Base manifests expose service WITHOUT TLS or authentication
- Production deployments rely on TrustyAI Operator for:
  - TLS termination (cert injection)
  - Authentication (kube-rbac-proxy sidecar)
  - Network policy (operator-managed)
- gRPC outbound calls to model endpoints are unencrypted (plaintext)
- Database connections are unencrypted (plaintext JDBC)
- MinIO connections encryption is configurable
