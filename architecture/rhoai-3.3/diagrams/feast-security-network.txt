Authentication Methods:
1. Bearer Token (JWT):
   - Mechanism: OIDC token validation
   - Endpoints: All feature server endpoints
   - Config: Secret {name}-oidc
   - Validation: OIDC Provider at 443/TCP HTTPS

2. mTLS (Mutual TLS):
   - Mechanism: Client certificate validation
   - Endpoints: All HTTPS endpoints (6567, 6571, 6573, 8816, 8443)
   - Certificates: Secret {name}-client-tls, {name}-server-tls
   - Provisioning: User or cert-manager

3. Kubernetes RBAC:
   - Mechanism: SubjectAccessReview API
   - Resource: feast.dev/featurestores
   - Enforcement: Feature servers check permissions

4. Token Review:
   - Mechanism: Kubernetes TokenReview API
   - Endpoint: /metrics (operator)
   - Port: 8443/TCP HTTPS

Secrets:
- {name}-client-tls (kubernetes.io/tls) - Client mTLS certificates
- {name}-server-tls (kubernetes.io/tls) - Server TLS certificates
- {name}-oidc (Opaque) - OIDC credentials (client_id, client_secret, etc.)
- {name}-db-secret (Opaque) - PostgreSQL credentials
- {name}-redis-secret (Opaque) - Redis connection credentials
- {name}-s3-secret (Opaque) - S3 access credentials
- {name}-git-token (Opaque) - Git repository access token
