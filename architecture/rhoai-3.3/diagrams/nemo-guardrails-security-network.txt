RBAC Summary:
- No Kubernetes RBAC resources defined
- NeMo Guardrails runs as a stateless application pod
- Does not interact with the Kubernetes API server

Authentication & Authorization:
- Server does NOT implement its own authentication
- Platform-level auth expected:
  - kube-rbac-proxy sidecar
  - Istio mTLS (PeerAuthentication)
  - Network Policies
- CORS middleware disabled by default
  - Enable via: NEMO_GUARDRAILS_SERVER_ENABLE_CORS env var
- Config ID path traversal protection:
  - Regex validation on config_id parameter
  - os.path.commonprefix check against config directory

Security Hardening:
- Container runs as non-root user (UID 1001)
- pam_namespace.so and namespace.conf removed (prevent namespace escaping)
- Chat UI disabled in RHOAI container (--disable-chat-ui)
- Closed-source guardrail providers stripped at build time
- protobuf >= 5.29.5 pinned (CVE-2024-7254)
- Build profile: "opensource" only

Secrets:
- LLM API Key (Opaque) - API key for backend LLM endpoint
  - Provisioned by: User/Platform
  - Auto-Rotate: No
- git-auth (kubernetes.io/basic-auth) - Build-time only
  - Provisioned by: Konflux/PaC
  - Auto-Rotate: Yes
