════════════════════════════════════════════════════════════════════════════
AUTHENTICATION & AUTHORIZATION
════════════════════════════════════════════════════════════════════════════

MLflow Server API:
- Endpoint: /api/2.0/*
- Auth Mechanism: kubernetes-auth (Bearer Token)
- Enforcement: MLflow server performs self_subject_access_review via K8s API
- Policy: Users must have valid Kubernetes ServiceAccount token
- Authorization Mode: self_subject_access_review

Operator Metrics:
- Endpoint: /metrics (port 8443)
- Auth Mechanism: Bearer Token (ServiceAccount)
- Enforcement: kube-rbac-proxy
- Policy: ServiceAccount must have metrics reader role

Gateway Traffic:
- External TLS termination at data-science-gateway (TLS 1.3)
- Bearer token passed through to MLflow server
- End-to-end HTTPS (gateway → mlflow service → mlflow pod)
