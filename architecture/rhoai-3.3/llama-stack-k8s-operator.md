## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of Llama Stack server deployments, providing declarative management of AI inference distribution services on OpenShift/Kubernetes.

**Detailed**: The llama-stack-k8s-operator is a controller-runtime based Kubernetes operator that watches `LlamaStackDistribution` custom resources and reconciles the desired state by creating and managing Deployments, Services, NetworkPolicies, Ingresses, PersistentVolumeClaims, HorizontalPodAutoscalers, PodDisruptionBudgets, and ServiceAccounts. It uses kustomize-based manifest rendering with Go-based transformer plugins to dynamically configure resources per-instance.

The operator supports multiple distribution types (named distributions mapped to container images, or custom images), persistent storage with PVC management, user-provided configuration via ConfigMaps, CA bundle management for TLS (including auto-detection of ODH/RHOAI trusted CA bundles), network access controls via NetworkPolicies, optional external access via Kubernetes Ingress resources, horizontal pod autoscaling, and pod disruption budgets for high availability. It also performs health checks against the deployed Llama Stack server's `/v1/health` endpoint and queries provider and version information from the server's API.

For RHOAI deployments, the operator is deployed into the `redhat-ods-applications` namespace by the rhods-operator via kustomize overlays. The RHOAI overlay injects the `RELATED_IMAGE_ODH_LLAMASTACK_OPERATOR` and `RELATED_IMAGE_RH_DISTRIBUTION` environment variables, mapping the operator image and the default Red Hat distribution image respectively.

The operator creates a Kubernetes `Ingress` resource (not an OpenShift Route or Gateway API HTTPRoute) when the CR's `spec.network.exposeRoute` is set to `true`. The Ingress routes traffic with `PathType: Prefix` at path `/` to the backing `{instance-name}-service` on the configured service port. External URL is reported in `status.routeURL`.

### Network Policies

| Name | Type | Direction | Ports | Default Peers | Purpose |
|------|------|-----------|-------|---------------|---------|
| {instance-name}-network-policy | NetworkPolicy | Ingress | {service-port}/TCP | Same-namespace pods with `app.kubernetes.io/part-of: llama-stack`, operator namespace, OpenShift router namespaces | Restricts inbound access to Llama Stack server pods |

NetworkPolicy creation is controlled by the `enableNetworkPolicy` feature flag (default: `true`). Ingress rules are dynamically configured via the NetworkPolicy transformer plugin based on:
- **Default**: Same-namespace pods with `app.kubernetes.io/part-of=llama-stack` label, all pods from the operator namespace
- **spec.network.allowedFrom.namespaces**: Explicit namespace names (or `*` for all)
- **spec.network.allowedFrom.labels**: Namespace label keys (OR semantics)
- **OpenShift router**: Namespaces with `network.openshift.io/policy-group: ingress` label (when `spec.network` is configured)

- **Namespace**: `redhat-ods-applications`
