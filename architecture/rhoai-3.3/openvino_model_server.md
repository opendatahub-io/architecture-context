## Purpose
**Short**: OpenVINO Model Server is a high-performance C++ inference serving runtime that hosts AI/ML models and exposes them over gRPC and REST APIs using KServe, TensorFlow Serving, and OpenAI-compatible protocols.

**Detailed**: OpenVINO Model Server (OVMS) is a scalable, high-performance model serving system developed by Intel, optimized for deployment on Intel architectures using the OpenVINO inference toolkit. It runs as a standalone container-based service that loads AI/ML models from local storage, cloud object stores (S3, GCS, Azure Blob), or HuggingFace Hub, and serves inference requests over both gRPC and HTTP/REST APIs.

In the RHOAI platform, OVMS functions as one of the available inference runtimes for KServe-based model serving. It supports multiple API protocols including KServe Predict V2, TensorFlow Serving, and OpenAI-compatible generative APIs (chat completions, embeddings, reranking, image generation, text-to-speech, and speech-to-text). The server supports advanced features such as model versioning, DAG-based pipeline scheduling, MediaPipe graph execution, continuous batching for LLMs, streaming inference, and Prometheus metrics export.

OVMS is built entirely in C++ for performance, with optional Python support for custom processing nodes. It uses Bazel as its build system and packages the OpenVINO runtime, OpenVINO GenAI, and OpenVINO Tokenizers as dependencies. The RHOAI build uses Konflux for CI/CD with Link-Time Optimization (LTO) enabled and GPU driver support included.

### Ingress

OVMS does not manage its own ingress. When deployed via KServe in RHOAI 3.x, ingress is managed by:
- KServe creates HTTPRoute CRs referencing the platform Gateway (data-science-gateway)
- kube-rbac-proxy sidecars provide authentication
- The OVMS container itself handles only plaintext traffic; TLS is terminated at the gateway/proxy layer
