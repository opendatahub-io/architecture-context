## Purpose
**Short**: MaaS API is a Go-based API gateway service that provides OpenAI-compatible model discovery, tiered token/API key management, and policy-driven rate limiting for LLM inference services on OpenShift.

**Detailed**: Models as a Service (MaaS) API is a comprehensive platform for providing LLM model access with real-time policy management. It serves as a management plane that sits in front of KServe LLMInferenceService endpoints, providing user authentication via Kubernetes TokenReview, tier-based access control (free/premium/enterprise), and token issuance through Kubernetes ServiceAccount tokens with JWT claims.

The service integrates deeply with the Gateway API ecosystem and Kuadrant policy engine (Authorino for authentication/authorization, Limitador for rate limiting). MaaS API issues ephemeral tokens and persistent API keys that carry tier information, enabling downstream gateway policies (AuthPolicy, TokenRateLimitPolicy) to enforce per-user, per-tier rate limits on LLM token consumption. The architecture uses Gateway API HTTPRoutes for traffic routing and Kuadrant policies attached to a dedicated `maas-default-gateway` Gateway resource.

MaaS API watches KServe InferenceService and LLMInferenceService CRs via shared informers and dynamically determines model availability and user authorization. It supports three storage backends for API key metadata: in-memory (ephemeral), disk (SQLite file), and external (PostgreSQL) for high-availability production deployments.

**Routing Rules** (from maas-api-route HTTPRoute):
- `/v1/models` → maas-api:8080 (OpenAI-compatible model list endpoint)
- `/maas-api/*` → maas-api:8080 with URL rewrite (strip `/maas-api` prefix)

**Authentication Flow** (via Kuadrant AuthPolicy):
1. `maas-api-auth-policy` (on HTTPRoute): Authenticates OpenShift identities via KubernetesTokenReview (audiences: `https://kubernetes.default.svc`, `maas-default-gateway-sa`); sets `X-MaaS-Username` and `X-MaaS-Group` response headers
2. `gateway-auth-policy` (on Gateway): Authenticates SA tokens via KubernetesTokenReview (audience: `maas-default-gateway-sa`); performs SubjectAccessReview authorization on `serving.kserve.io/llminferenceservices` resources; enriches identity with tier via callback to `POST /v1/tiers/lookup`

**Rate Limiting** (via Kuadrant TokenRateLimitPolicy on Gateway):
- Free tier: 100 tokens/minute per user
- Premium tier: 50,000 tokens/minute per user
- Enterprise tier: 100,000 tokens/minute per user
- Excludes `/v1/models` endpoint from rate limiting
