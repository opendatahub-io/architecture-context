## Purpose
**Short**: A secure, batch-oriented container runtime for evaluating language models against standardized benchmarks using the EleutherAI lm-evaluation-harness framework.

**Detailed**: The lm-evaluation-harness (LMEval) component provides a containerized execution environment for running language model evaluations within the Red Hat OpenShift AI (RHOAI) platform. It is a fork of the upstream EleutherAI lm-evaluation-harness (v0.4.8), customized for Red Hat's distribution requirements including multi-architecture support (amd64, arm64, ppc64le, s390x), offline metric evaluation, S3-compatible model/data download, OCI artifact push for evaluation results, and Unitxt framework integration.

This component is not a long-running service or operator. It is designed as a one-shot batch job container image that is launched by the TrustyAI LMEval operator (trustyai-service-operator) as a Kubernetes Job. Each job instance evaluates a specific language model against configured benchmark tasks, produces evaluation metrics, and optionally pushes results to an OCI registry or S3-compatible storage. The container supports both local model evaluation (via HuggingFace Transformers, vLLM) and remote API-based evaluation (via OpenAI-compatible endpoints, IBM watsonx.ai).

The container ships with 50+ bundled offline metrics (accuracy, BLEU, ROUGE, F1, etc.) and 150+ evaluation task families comprising 1200+ individual benchmark configurations, covering areas such as MMLU, HellaSwag, ARC, GSM8K, TruthfulQA, and many more multilingual and domain-specific benchmarks.
