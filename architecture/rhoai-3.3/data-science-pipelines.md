## Purpose
**Short**: Kubeflow Pipelines (KFP) provides end-to-end ML workflow orchestration on Kubernetes, enabling pipeline authoring, scheduling, execution, and artifact tracking.

**Detailed**: Data Science Pipelines is Red Hat's downstream distribution of Kubeflow Pipelines, a platform for building and deploying portable, scalable machine learning workflows based on Kubernetes. It provides a complete pipeline management system including a REST/gRPC API server for pipeline CRUD operations, an Argo Workflows-based execution engine, pipeline caching for performance optimization, scheduled workflow management, metadata tracking via ML Metadata (MLMD), and a web-based frontend UI for visualization and management.

The system follows a microservices architecture with several cooperating components: the API server handles all client-facing operations and manages pipeline definitions, runs, experiments, and recurring runs; the persistence agent monitors Argo Workflow execution and reports status back to the API server; the scheduled workflow controller manages cron-based and periodic pipeline execution; the cache server implements a mutating admission webhook to enable step-level caching; the metadata writer records execution metadata to MLMD; and the frontend provides a React-based dashboard. All components are built using Red Hat UBI9 base images with FIPS-compliant Go builds (`GOEXPERIMENT=strictfipsruntime`) for the RHOAI distribution.

The system supports both MySQL and PostgreSQL as backing databases, and S3-compatible object storage (MinIO/SeaweedFS) for artifact persistence. In multi-user mode, Istio service mesh provides mTLS and authorization policies between components.
