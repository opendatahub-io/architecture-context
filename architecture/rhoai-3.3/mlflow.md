## Purpose
**Short**: MLflow is an open-source AI/ML lifecycle platform providing experiment tracking, model registry, LLM observability/tracing, and evaluation capabilities, extended for RHOAI with Kubernetes-native workspace isolation and RBAC.

**Detailed**: MLflow serves as the experiment tracking and model registry backend for Red Hat OpenShift AI. It provides REST and GraphQL APIs for managing ML experiments, runs, model versions, datasets, traces, and prompts. The server exposes a React-based web UI for interactive exploration and a comprehensive REST API consumed by Python/CLI clients.

The RHOAI distribution adds a Kubernetes workspace provider that maps MLflow workspaces 1:1 to Kubernetes namespaces, enabling multi-tenant isolation. An accompanying authorization plugin enforces Kubernetes RBAC via `SelfSubjectAccessReview` or `SubjectAccessReview` against a custom `mlflow.kubeflow.org` API group. This allows workspace-scoped access control using standard Kubernetes service account tokens, compatible with kube-rbac-proxy sidecar authentication patterns.

The server runs on UBI9 (Python 3.11) and is built via Konflux pipelines. It uses gunicorn/uvicorn as the WSGI/ASGI server, backed by SQLAlchemy for metadata storage (supporting PostgreSQL and SQLite) with a pluggable artifact storage layer (S3, GCS, Azure Blob, local filesystem).

The Kubernetes authorization plugin is designed to work behind `kube-rbac-proxy` or similar reverse proxies. When `MLFLOW_K8S_AUTH_AUTHORIZATION_MODE=subject_access_review`, MLflow trusts `x-remote-user` and `x-remote-groups` headers from the proxy and performs SubjectAccessReview on behalf of the reported identity.

**Authorization Modes:**
1. **SelfSubjectAccessReview** (default): Uses caller's bearer token to evaluate access against K8s API
2. **SubjectAccessReview**: Uses proxy-provided `x-remote-user`/`x-remote-groups` headers; server's own SA performs the review

**Authorization cache**: Decisions are cached for 300 seconds (configurable via `MLFLOW_K8S_AUTH_CACHE_TTL_SECONDS`).
