## Purpose
**Short**: Container-native workflow engine for orchestrating parallel jobs on Kubernetes, used as the workflow execution backend for Data Science Pipelines in RHOAI.

**Detailed**: Argo Workflows is an open source container-native workflow engine implemented as a Kubernetes CRD. It enables defining workflows where each step is a container, supporting both sequential (steps) and parallel (DAG) execution patterns. In the RHOAI distribution, Argo Workflows provides the execution engine for Data Science Pipelines, managing the lifecycle of ML pipeline runs.

The component ships two container images built via Konflux: the workflow-controller (which watches Workflow CRDs and orchestrates pod execution) and argoexec (the executor sidecar injected into workflow pods to manage container lifecycle, artifact collection, and output handling). The RHOAI build produces FIPS-compliant binaries using `GOEXPERIMENT=strictfipsruntime` on UBI9 base images. The argo-server (API server with UI) is also part of the codebase but the RHOAI distribution focuses on the controller and executor components as they are consumed by the Data Science Pipelines Operator.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| workflow-controller | Go Controller | Watches Workflow/CronWorkflow CRDs, creates and manages pods for each workflow step, handles leader election, artifact GC, and workflow archival |
| argoexec | Go Sidecar Executor | Injected into workflow pods as the emissary executor; manages step container lifecycle, collects artifacts, reports task results |
| argo-server | Go API Server | Provides REST/gRPC API and Web UI for workflow management; supports SSO, Bearer token, and server auth modes (not directly shipped as RHOAI container) |
