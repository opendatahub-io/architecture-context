## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of Model Registry instances and the Model Catalog service on OpenShift/RHOAI.

**Detailed**: The Model Registry Operator automates the deployment and management of Model Registry instances via the `ModelRegistry` custom resource (CRD). Each ModelRegistry CR triggers the operator to create and maintain a complete set of Kubernetes resources including Deployments, Services, ServiceAccounts, RBAC resources, NetworkPolicies, OpenShift Routes, and kube-rbac-proxy authentication sidecars. The operator supports PostgreSQL and MySQL as backend databases, with optional auto-provisioning of PostgreSQL deployments.

In addition to per-instance ModelRegistry management, the operator includes a ModelCatalogReconciler that manages a singleton Model Catalog service. This catalog provides a centralized model metadata discovery endpoint backed by its own PostgreSQL database, with data populated from configurable catalog sources. The operator handles CRD version migration (v1alpha1 to v1beta1), supports BYOIDC (Bring Your Own Identity Provider) mode for OpenShift 4.20+, and includes admission webhooks for validation and defaulting.

The operator is deployed by the rhods-operator using kustomize overlays (specifically `config/overlays/odh`), with container image references injected at deploy time via params.env replacements.
