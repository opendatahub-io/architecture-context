## Purpose
**Short**: Pre-built container image providing a Llama Stack API server with multiple inference, evaluation, safety, and vector store providers for the Red Hat OpenShift AI platform.

**Detailed**: The Llama Stack Distribution is a containerized deployment of the [Llama Stack](https://github.com/llamastack/llama-stack) framework, packaged as an OCI container image for use in Red Hat OpenShift AI (RHOAI). It provides a unified REST API server (OpenAI-compatible) that exposes inference, agents, evaluation, safety, vector I/O, scoring, tool runtime, file management, batch processing, and dataset I/O capabilities through a plugin-based provider architecture.

The distribution ships with pre-configured providers for multiple inference backends (vLLM, AWS Bedrock, WatsonX, Azure, Vertex AI, OpenAI, Sentence Transformers), vector databases (Milvus, Faiss, pgvector, Qdrant), safety (TrustyAI FMS), evaluation (TrustyAI LMEval, RAGAS), and tool runtimes (Brave Search, Tavily Search, RAG, MCP). Providers are conditionally enabled via environment variables at runtime. Persistent state is stored in PostgreSQL using both key-value and SQL storage backends.

The image is built through the Konflux CI/CD pipeline for RHOAI releases and via GitHub Actions for upstream/development builds. It uses a UBI 9 (Red Hat Universal Base Image) with Python 3.12 and includes a pre-downloaded embedding model (ibm-granite/granite-embedding-125m-english).
