## Purpose
**Short**: An HTTP gateway that provides OpenAI-compatible chat completion endpoints with configurable detector-based content filtering, routing requests through the FMS Guardrails Orchestrator.

**Detailed**: The vLLM Orchestrator Gateway serves as a routing and filtering layer between clients and the FMS Guardrails Orchestrator. It dynamically creates per-route `/v1/chat/completions` endpoints based on YAML configuration, each bound to a specific set of content detectors (e.g., PII detection via regex). When a client sends a chat completion request to a named route (e.g., `/{route_name}/v1/chat/completions`), the gateway injects the configured detector specifications into the request payload and forwards it to the orchestrator's `/api/v2/chat/completions-detection` endpoint.

The gateway supports both streaming (SSE) and non-streaming chat completion modes. When detections are found in the orchestrator's response (e.g., PII in generated output), the gateway can replace the response content with a configurable fallback message. This allows operators to create guardrailed inference endpoints as drop-in replacements for standard OpenAI-compatible chat completion endpoints, enforcing content safety policies transparently to clients.

The gateway supports mTLS communication with the orchestrator backend using OpenShift service-serving certificates (`/etc/tls/private/tls.crt`, `/etc/tls/private/tls.key`) and custom CA certificates (`/etc/tls/ca/service-ca.crt`). It also forwards `Authorization` and `X-Forwarded-*` headers from clients to the orchestrator.
