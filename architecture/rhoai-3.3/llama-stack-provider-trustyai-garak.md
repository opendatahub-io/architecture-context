## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that performs automated LLM vulnerability scanning and red teaming using NVIDIA's Garak framework.

**Detailed**: This component implements the Llama Stack `Eval` API as an external provider, enabling automated vulnerability assessment of Large Language Models. It integrates NVIDIA's Garak red-teaming framework to scan LLMs for prompt injection, jailbreaks, toxicity, bias, and other security vulnerabilities against compliance benchmarks such as OWASP LLM Top 10 and AVID (AI Vulnerability Database) taxonomies.

The provider supports two execution modes: **inline** (local process-based scanning for development/testing) and **remote** (Kubeflow Pipelines-based execution for production workloads on OpenShift AI). In remote mode, scans are submitted as KFP pipeline runs that execute within dedicated pods, with results uploaded to the Llama Stack Files API for retrieval. The provider also supports shield testing, where guardrails (input/output shields) can be evaluated alongside LLM responses to measure their effectiveness against adversarial attacks.

The container image (`odh-trustyai-garak-lls-provider-dsp-rhel9`) is built by Konflux and serves as the base image for Kubeflow Pipeline task components, containing all necessary dependencies including Garak and the provider code pre-installed.
