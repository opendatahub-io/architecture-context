## Purpose
**Short**: A Kubernetes operator that manages TrustyAI AI/ML model explainability services, LM evaluation jobs, guardrails orchestration, and NVIDIA NeMo Guardrails deployments.

**Detailed**: The TrustyAI Service Operator is a multi-controller Go operator built on controller-runtime that manages four distinct AI trustworthiness and safety services within the OpenShift AI (RHOAI) platform. It provides:

1. **TrustyAI Service (TAS)**: Deploys and manages TrustyAI explainability service instances that provide bias detection, fairness metrics, and model monitoring for AI/ML inference services (ModelMesh and KServe). Each TrustyAIService CR triggers the creation of a Deployment with kube-rbac-proxy sidecar, internal and TLS Services, OpenShift Routes, Istio VirtualService/DestinationRule (when available), ServiceMonitors, and PVC/database storage management.

2. **LM Evaluation Service (LMES)**: Manages LMEvalJob CRs that run language model evaluation benchmarks using the lm-evaluation-harness framework. It creates evaluation Pods with a driver sidecar for job lifecycle management, supports Kueue integration for resource queuing, and handles batch processing of evaluation tasks.

3. **Guardrails Orchestrator (GORCH)**: Deploys the FMS Guardrails Orchestrator that provides content moderation and safety guardrails for large language model inference. It manages multi-container deployments including the orchestrator, built-in regex detectors, and a sidecar gateway for vLLM integration, each optionally fronted by kube-rbac-proxy for authentication.

4. **NeMo Guardrails (NEMO)**: Deploys NVIDIA NeMo Guardrails server instances for programmable guardrails on LLM applications, with optional kube-rbac-proxy authentication and CA bundle management.

The operator is deployed by the rhods-operator using kustomize overlays. Image references are injected at deploy time via RELATED_IMAGE environment variables, enabling disconnected/air-gapped installations.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| TrustyAIService Controller (TAS) | Go Controller | Reconciles TrustyAIService CRs; creates Deployments with kube-rbac-proxy sidecar, Services, Routes, Istio VirtualService/DestinationRule, ServiceMonitors, PVCs, ConfigMaps |
| LMEvalJob Controller (LMES) | Go Controller | Reconciles LMEvalJob CRs; creates evaluation Pods with driver init/sidecar containers for LM benchmark execution |
| GuardrailsOrchestrator Controller (GORCH) | Go Controller | Reconciles GuardrailsOrchestrator CRs; creates multi-container Deployments (orchestrator + detector + gateway), Services, Routes, kube-rbac-proxy ConfigMaps, ServiceMonitors |
| NemoGuardrails Controller (NEMO) | Go Controller | Reconciles NemoGuardrails CRs; creates Deployments with optional kube-rbac-proxy sidecar, Services, Routes, CA bundle ConfigMaps |
| Job Manager (JOB_MGR) | Go Controller | Kueue integration for LMEvalJob workload scheduling; creates/manages Kueue Workload resources |
| Operator Manager | Go Binary | Main entrypoint (/manager); runs health/readiness probes on port 8081, metrics on port 8080; enables leader election |
