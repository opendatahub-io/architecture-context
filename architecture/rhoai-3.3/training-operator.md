## Purpose
**Short**: Kubernetes-native operator for managing distributed training jobs across multiple ML frameworks (PyTorch, TensorFlow, XGBoost, MPI, PaddlePaddle, JAX).

**Detailed**: The Kubeflow Training Operator (KFTO) is a Kubernetes operator that enables distributed machine learning model training on Kubernetes clusters. It provides CRDs for defining training jobs across six ML frameworks: PyTorchJob, TFJob, MPIJob, XGBoostJob, PaddleJob, and JAXJob. Each CRD allows users to specify replica sets (masters, workers, parameter servers) with framework-specific configurations.

The operator manages the complete lifecycle of training jobs including pod creation, headless service management for inter-pod communication, environment variable injection for distributed training coordination (e.g., MASTER_ADDR, WORLD_SIZE, RANK), elastic training support with HPA for PyTorch, gang scheduling integration (Volcano and scheduler-plugins), and NetworkPolicy enforcement for PyTorchJob workloads. The operator also manages webhook certificates for validating webhook configurations that enforce CRD schema validation.

In RHOAI, the operator is deployed via kustomize overlays applied by the rhods-operator. The RHOAI overlay configures the operator in the `opendatahub` namespace with the `kubeflow-` name prefix, injects the RHOAI-specific container image, and adds RBAC roles that aggregate training job permissions to standard Kubernetes edit/view cluster roles.

### Dynamically Created Services (per training job)

The operator dynamically creates headless ClusterIP Services for each replica in a training job to enable inter-pod DNS resolution for distributed training communication.

| Service Pattern | Type | Protocol | Purpose |
|-----------------|------|----------|---------|
| `{job-name}-{replica-type}-{index}` | ClusterIP (headless) | TCP | DNS-based service discovery for distributed training pods |

### Network Policies

The PyTorchJob controller dynamically creates a NetworkPolicy per PyTorchJob that:
1. Allows all ingress traffic between pods belonging to the same PyTorchJob (matched by `training.kubeflow.org/job-name` label)
2. Allows ingress from the `openshift-monitoring` namespace on port 8080/TCP for Prometheus metrics scraping
