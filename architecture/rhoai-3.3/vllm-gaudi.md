## Purpose
**Short**: A vLLM hardware plugin providing optimized large language model inference on Intel Gaudi (Habana) AI accelerators.

**Detailed**: vllm-gaudi is a hardware plugin for the vLLM inference engine that enables high-performance LLM serving on Intel Gaudi accelerators. It implements the vLLM pluggable hardware architecture (RFC #11162 and #19161), providing a modular interface that extends vLLM to use Intel Gaudi HPU (Habana Processing Unit) devices for model execution.

The plugin provides HPU-specific implementations of attention backends, model runners, workers, custom operations (FP8 quantization, fused MoE, rotary embeddings, layer normalization, etc.), distributed communication via HCCL, KV-cache transfer via NIXL, and speculative decoding (EAGLE). It registers custom model implementations for architectures like Gemma3, Qwen2.5-VL, Qwen3-VL, Pixtral, DeepSeek, HunYuan, MiniMax-M2, and others that require HPU-specific optimizations.

In the RHOAI platform, this container image is deployed as a KServe/ServingRuntime container for model inference workloads running on Intel Gaudi hardware. The container entrypoint is the vLLM OpenAI-compatible API server (`vllm.entrypoints.openai.api_server`), which serves models over HTTP with an OpenAI-compatible REST API.
