## Purpose
**Short**: Kubeflow Trainer is a Kubernetes operator that manages distributed machine learning training jobs using the TrainJob/TrainingRuntime CRD API.

**Detailed**: The Kubeflow Trainer operator (formerly known as the Kubeflow Training Operator) provides a declarative API for running distributed ML training workloads on Kubernetes. Users create `TrainJob` custom resources that reference `TrainingRuntime` or `ClusterTrainingRuntime` templates, and the controller reconciles these into underlying `JobSet` resources that manage the actual training pods.

The RHOAI distribution adds several enhancements over upstream: progression tracking (real-time training metrics polling via HTTP from training pods), NetworkPolicy-based pod isolation for training jobs, and pre-configured `ClusterTrainingRuntime` resources for PyTorch distributed training with CUDA and ROCm GPU support. The operator manages its own webhook certificates via an internal cert controller, validates CRDs through admission webhooks, and supports integration with scheduling frameworks like Volcano and Kueue (via scheduler-plugins).

The controller creates `JobSet` resources that orchestrate multi-node training using PyTorch's torchrun, DeepSpeed, MPI, or TorchTune. Training pods communicate peer-to-peer via NCCL/MPI/gRPC on configurable ports (default 29500/TCP). The RHOAI-specific progression tracking feature polls training pods' HTTP metrics endpoints (default port 28080/TCP) to report real-time training progress (steps, epochs, ETA) back to the `TrainJob` status annotations.

### Dynamic NetworkPolicies (Created per TrainJob)

The controller dynamically creates a `NetworkPolicy` for each `TrainJob` when reconciling. The policy:

1. **Pod isolation (always)**: Allows ingress from pods in the same JobSet (matched by `jobset.sigs.k8s.io/jobset-name` label) on all ports. This permits NCCL/MPI/gRPC inter-node training communication.
2. **Metrics access (when progression tracking enabled)**: Allows ingress from the controller namespace pods (matched by `app.kubernetes.io/name=trainer` and `app.kubernetes.io/component=controller`) on the configured metrics port (default 28080/TCP).

The NetworkPolicy uses `OwnerReference` pointing to the TrainJob for automatic cleanup.
