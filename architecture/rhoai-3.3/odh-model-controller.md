## Purpose
**Short**: A Kubernetes controller that manages model serving infrastructure for KServe InferenceService and LLMInferenceService resources on OpenShift AI, handling routes, monitoring, authentication, certificate management, and NVIDIA NIM integration.

**Detailed**: The odh-model-controller is a controller-runtime based Go operator that extends KServe's model serving capabilities with Red Hat OpenShift AI platform integrations. It reconciles KServe InferenceService, LLMInferenceService, InferenceGraph, and ServingRuntime resources to create and manage supporting infrastructure including OpenShift Routes for model endpoint exposure, Prometheus ServiceMonitors and PodMonitors for observability, RBAC resources for authentication, NetworkPolicies for isolation, and CA certificate bundles for TLS trust chains.

The controller also manages LLMInferenceService resources (RHOAI 3.x pattern) with Gateway API integration, creating Kuadrant AuthPolicy resources for authentication and Istio EnvoyFilter resources for TLS bootstrap between gateways and Authorino. It provides webhook validation and mutation for InferenceService, InferenceGraph, LLMInferenceService, Pod, and ConfigMap resources. Additionally, it supports NVIDIA NIM integration through the Account CRD, managing API key validation, pull secret generation, and runtime template creation for NIM model deployments. It manages multi-node (Ray) TLS certificate generation for distributed inference workloads, and reconciles data connection secrets into KServe-compatible storage-config secrets.

**RHOAI 3.x Gateway API Pattern**: The LLMInferenceService controller watches Gateway API resources (Gateway, HTTPRoute) and creates Kuadrant AuthPolicy CRs targeting those resources for authentication. It also creates Istio EnvoyFilter resources to bootstrap TLS between the gateway and Authorino. The default gateway name is `openshift-ai-inference` in namespace `openshift-ingress`.

**Legacy Route Pattern**: The InferenceService controller creates OpenShift Routes for KServe InferenceService resources when the `networking.kserve.io/visibility=exposed` label is set. Routes use Edge TLS termination by default, or Reencrypt when `security.opendatahub.io/enable-auth=true` annotation is present.
