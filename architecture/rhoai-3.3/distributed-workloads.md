## Purpose
**Short**: Provides GPU-accelerated training runtime container images and comprehensive end-to-end integration tests for the RHOAI Distributed Workloads stack (Ray, Kueue, Training Operator/Trainer v2, and Kubeflow).

**Detailed**: The distributed-workloads repository serves two primary functions within the Red Hat OpenShift AI (RHOAI) platform. First, it is the source repository for building training runtime container images that ship with the product. These images provide pre-configured environments for distributed machine learning training, supporting NVIDIA CUDA (12.1, 12.4, 12.8), AMD ROCm (6.2, 6.4), and CPU-only configurations with PyTorch versions ranging from 2.4.1 through 2.9.0, all built on Red Hat UBI9 base images. The images include essential ML training dependencies such as Flash Attention, NCCL for multi-GPU communication, InfiniBand/RDMA networking support, and the fms-hf-tuning library for fine-tuning large language models.

Second, the repository contains a comprehensive Go-based integration test suite that validates the entire distributed workloads stack on OpenShift. The tests exercise Ray clusters (via KubeRay operator), Kueue job queuing and resource management, the Kubeflow Training Operator (PyTorchJob), and the newer Kubeflow Trainer v2 (TrainJob). Test scenarios include MNIST training, LLM supervised fine-tuning, hyperparameter optimization with Ray Tune, model training with DeepSpeed, and upgrade validation. The tests interact with DataScienceCluster and DSCInitialization custom resources to manage component lifecycle.

Additionally, the repository includes example notebooks and scripts demonstrating distributed training patterns: fine-tuning LLMs with Ray and DeepSpeed, hyperparameter optimization with Ray Tune, Stable Diffusion fine-tuning with DreamBooth, and Retrieval-Augmented Generation (RAG) workflows.
