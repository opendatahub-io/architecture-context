## Purpose
**Short**: TrustyAI is a Java-based REST service providing AI fairness metrics, bias tracking, data drift detection, and explainability algorithms for ML models served via ModelMesh and KServe.

**Detailed**: TrustyAI Explainability is a Quarkus-based microservice that integrates with the Open Data Hub / RHOAI platform to provide responsible AI capabilities. It consists of four modules: a core library (`explainability-core`) with fairness metrics and XAI algorithms, a connectors module (`explainability-connectors`) implementing KServe v2 gRPC protocol for model inference, an Arrow module (`explainability-arrow`) for Python interoperability, and the main service (`explainability-service`) exposing REST APIs for metric calculation, drift detection, and model explanations.

The service consumes model inference payloads (inputs/outputs) either via a REST consumer endpoint (`/consumer/kserve/v2`) from ModelMesh or via CloudEvents from KServe inference loggers. It stores this data in PVC-based flat files, MinIO object storage, or a relational database (MariaDB/MySQL via Hibernate ORM). Scheduled metrics are computed periodically and exposed as Prometheus metrics (`trustyai_spd`, `trustyai_dir`). For explanations, the service performs outbound gRPC calls to ModelMesh/KServe-served models using the KServe v2 inference protocol.

In the RHOAI distribution, the service is managed by the separate [TrustyAI Operator](https://github.com/trustyai-explainability/trustyai-service-operator). Certain explainer endpoints (SHAP, LIME, Counterfactual, TSSaliency, Global explainers) are disabled in the RHOAI profile, while fairness and drift metrics remain enabled.
