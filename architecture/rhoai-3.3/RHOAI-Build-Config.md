## Purpose
**Short**: Central build configuration repository that defines the OLM operator bundle, FBC (File-Based Catalog) fragments, image references, and CI/CD pipelines for shipping Red Hat OpenShift AI (RHOAI) 3.3.

**Detailed**: RHOAI-Build-Config is the authoritative source for the RHOAI product's build and release metadata. It does not contain application source code or runtime controllers. Instead, it houses the Operator Lifecycle Manager (OLM) bundle manifests for the `rhods-operator`, including the ClusterServiceVersion (CSV), Custom Resource Definition (CRD) schemas, RBAC resources, and related image references. The bundle defines the operator deployment specification (3 replicas, webhook server, health probes) and declares all 82+ container images shipped as part of the product via `RELATED_IMAGE_*` environment variables, enabling disconnected/air-gapped installation.

The repository also manages the File-Based Catalog (FBC) fragments for OCP versions v4.19, v4.20, and v4.21, which define the operator's upgrade graph across releases. Konflux (Tekton-based) CI/CD pipelines build the operator bundle and FBC fragment images, while GitHub Actions workflows automate bundle processing, image patching, and promotion to staging/production. Build configuration files map image registries between quay.io (development) and registry.redhat.io (production), and an ImageDigestMirrorSet enables mirroring for disconnected environments.
