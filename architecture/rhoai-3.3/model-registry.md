## Purpose
**Short**: A central repository service for recording, storing, and managing ML model metadata including registered models, model versions, artifacts, inference services, and experiments.

**Detailed**: The Model Registry is a Go-based REST API server that provides a central metadata store for ML workflows. It replaces the legacy C++ gRPC ml-metadata service with a higher-level OpenAPI-based REST interface. The server connects to a relational database backend (MySQL or PostgreSQL) using an embedded metadata datastore ("embedmd") and exposes a comprehensive REST API (v1alpha3) for managing registered models, model versions, model artifacts, serving environments, inference services, experiments, and experiment runs.

The component ships multiple sub-components: (1) the core REST API proxy server that stores and retrieves ML metadata, (2) a Kubernetes controller that watches KServe InferenceService CRs and synchronizes their state with the model registry, (3) a CSI storage initializer that resolves `model-registry://` URIs to actual model artifact locations for KServe, (4) a Model Catalog API server that aggregates model metadata from external sources like HuggingFace into a searchable catalog backed by PostgreSQL, (5) a UI backend-for-frontend (BFF) Go service, and (6) an async-upload Python job for transferring models between storage backends. The build uses FIPS-compliant Go compilation (`strictfipsruntime`) for Red Hat compliance.
