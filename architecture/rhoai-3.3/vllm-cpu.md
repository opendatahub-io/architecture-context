## Purpose

**Short**: CPU-only build of vLLM providing high-throughput, memory-efficient LLM inference serving with an OpenAI-compatible API.

**Detailed**: vLLM CPU is the CPU-optimized variant of the vLLM inference engine, packaged as a Red Hat container image for deployment within the Red Hat OpenShift AI (RHOAI) platform. It enables organizations to serve large language models (LLMs) on CPU-only infrastructure without requiring GPU hardware, which is valuable for cost-sensitive deployments, edge scenarios, and architectures where GPU resources are unavailable.

The server exposes an OpenAI-compatible REST API built on FastAPI/Uvicorn, supporting chat completions, text completions, embeddings, audio transcription/translation, and the newer Responses API. It also provides Anthropic Messages API compatibility. The runtime supports advanced features including LoRA adapter hot-loading, speculative decoding, continuous batching, and Prometheus metrics export.

This repository is a downstream fork of the upstream vllm-project/vllm, maintained by Red Hat with specific build scripts and Dockerfiles optimized for ppc64le, s390x, and x86_64 CPU architectures on UBI 9 base images. The Konflux build system (`Dockerfile.konflux.cpu`) is the primary build mechanism for RHOAI shipment.
