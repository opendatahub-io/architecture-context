## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of MLflow tracking server instances on OpenShift/Kubernetes clusters.

**Detailed**: The MLflow Operator watches for `MLflow` custom resources (cluster-scoped) and deploys fully configured MLflow tracking server instances using an embedded Helm chart renderer. The operator reconciles the desired state declared in the MLflow CR into a set of Kubernetes resources including Deployments, Services, ServiceAccounts, PersistentVolumeClaims, NetworkPolicies, and RBAC resources.

The operator integrates with the RHOAI 3.x platform ingress pattern by dynamically creating Gateway API `HTTPRoute` resources that reference the platform's `data-science-gateway` Gateway in the `openshift-ingress` namespace. It also creates OpenShift `ConsoleLink` resources for integration with the OpenShift web console application menu. The MLflow server itself runs with TLS enabled (using OpenShift service-ca generated certificates), Kubernetes-native authentication (`kubernetes-auth` app), and supports workspaces backed by Kubernetes namespaces.

The operator supports both local storage (SQLite + PVC) and remote storage backends (PostgreSQL, MySQL, S3, GCS) for metadata and artifacts. Configuration is driven by the MLflow CR spec with environment-variable-based defaults injected by the rhods-operator at deploy time via kustomize replacements from `params.env`.

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| mlflow (per MLflow CR) | HTTPRoute (Gateway API) | Inherited from data-science-gateway | 8443/TCP | HTTPS | TLS (passthrough to service-ca cert) | Backend TLS | External (via Gateway) |

The operator dynamically creates `HTTPRoute` resources (Gateway API v1) for each MLflow CR instance. The HTTPRoute references the `data-science-gateway` Gateway in the `openshift-ingress` namespace as its parent. Three routing rules are configured:
1. `/mlflow/api/*` -> rewrites to `/api/*` on the mlflow service (port 8443)
2. `/mlflow/v1/*` -> rewrites to `/v1/*` on the mlflow service (port 8443)
3. `/mlflow/*` -> forwards directly to the mlflow service (port 8443)

The HTTPRoute creation is conditional - the operator checks for HTTPRoute CRD availability at startup using the discovery API. If Gateway API is not installed, HTTPRoute creation is skipped gracefully.
