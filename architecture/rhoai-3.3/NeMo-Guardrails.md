## Purpose
**Short**: NeMo Guardrails is an open-source toolkit for adding programmable guardrails to LLM-based conversational systems, controlling and securing model outputs.

**Detailed**: NeMo Guardrails, originally developed by NVIDIA, provides a programmable guardrails layer between application code and Large Language Models (LLMs). It enables developers to define safety rails that control LLM behavior including topic restriction, dialog path enforcement, output moderation, jailbreak/prompt-injection detection, sensitive data detection, and fact-checking. The toolkit uses a domain-specific language called Colang (versions 1.0 and 2.x) to define conversational flows and guardrail rules.

In the RHOAI distribution, NeMo Guardrails is shipped as a containerized server (`odh-trustyai-nemo-guardrails-server-rhel9`) built on UBI 9 with Python 3.12. The container exposes an OpenAI-compatible chat completions API on port 8000. At build time, closed-source guardrail providers (ActiveFence, Cleanlab, Clavata, PrivateAI, Fiddler, PatronusAI, Prompt Security, GCP) are stripped, keeping only open-source guardrails. Pre-trained NLP models (spaCy, sentence-transformers, FastEmbed, NLTK) are pre-downloaded into the container image for air-gapped/disconnected deployments.

The server accepts guardrail configuration via mounted ConfigMaps (at `/app/config/`), where each configuration specifies LLM model endpoints, Colang flow definitions, and guardrail rules. This makes it suitable for RHOAI's TrustyAI integration, providing safety guardrails for AI model serving endpoints.

**Note**: The NeMo Guardrails server itself does not implement authentication. In RHOAI deployments, authentication is expected to be provided by the platform (e.g., kube-rbac-proxy sidecar, Istio mTLS, or network policies). The server includes CORS middleware that is disabled by default and can be enabled via `NEMO_GUARDRAILS_SERVER_ENABLE_CORS` environment variable.

**Security Hardening**:
- Container runs as non-root user (UID 1001)
- `pam_namespace.so` and `namespace.conf` are removed to prevent container namespace escaping
- Chat UI is disabled in the RHOAI container (`--disable-chat-ui`)
- Config ID path traversal protection via regex validation and `os.path.commonprefix` check
- Closed-source guardrail providers stripped at build time (only open-source guardrails included)
- `protobuf >= 5.29.5` pinned to address CVE-2024-7254
