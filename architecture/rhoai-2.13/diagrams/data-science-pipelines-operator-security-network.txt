  - Protocol: MySQL (3306/TCP)
  - Auth Mechanism: Username/Password
  - Enforcement Point: MariaDB server

  - Protocol: S3 API (HTTP/HTTPS)
  - Auth Mechanism: Access Key + Secret Key
  - Enforcement Point: Minio/S3 server
  - Policy: S3 signature validation (AWS Signature v4)
  - Encryption: TLS 1.2+ for external S3, None for Minio (dev only)

Configuration Notes:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Development vs Production:
  * Dev: Minio (no TLS) + embedded MariaDB (no TLS)
  * Prod: External S3 (TLS 1.2+) + managed database (TLS optional)

- Pod-to-Pod TLS:
  * Enabled via podToPodTLS: true (v2 only)
  * Requires mTLS between pipeline components

- Route Exposure:
  * Controlled via apiServer.enableRoute (default: true)
  * Uses OpenShift Route with reencrypt TLS mode

- Health Check Bypass:
  * Can disable via disableHealthCheck for dev/testing
  * Skips connectivity checks to external services

- Artifact URL Expiry:
  * Default: 60 seconds
  * Configurable via artifactSignedURLExpirySeconds

- CA Bundle Support:
  * Custom CA certificates injectable for external DB/S3
  * Mounted via ConfigMap to pods
