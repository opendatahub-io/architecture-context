┌─────────────────────────────────────────────────────────────────────────┐
│  DEPLOYMENT MODES                                                        │
└─────────────────────────────────────────────────────────────────────────┘

1. ModelMesh Mode:
   - No service mesh integration
   - OpenShift Route → Service → ModelMesh Pods
   - ServiceAccount + ClusterRoleBinding for storage access
   - Minimal networking resources

2. KServe Serverless Mode (Knative):
   - Full service mesh integration (Istio)
   - OpenShift Route → Istio Gateway → VirtualService → Knative Service → Predictor Pod
   - STRICT mTLS enforced (PeerAuthentication)
   - Autoscaling via Knative Serving
   - Optional authentication (Authorino AuthConfig)
   - Comprehensive monitoring (ServiceMonitor, PodMonitor, Grafana dashboards)

3. KServe Raw Mode (Kubernetes Deployment):
   - OpenShift Route → Service → Deployment → Predictor Pod
   - No service mesh integration
   - Minimal additional resources
   - Standard Kubernetes networking

┌─────────────────────────────────────────────────────────────────────────┐
│  TRUST BOUNDARIES                                                        │
└─────────────────────────────────────────────────────────────────────────┘

1. External → DMZ (OpenShift Router)
   - TLS 1.2+ edge termination
   - Optional Bearer Token authentication (if AuthConfig enabled)
   - Rate limiting (OpenShift Router capabilities)

2. DMZ → Service Mesh Ingress (Istio Gateway)
   - mTLS STRICT (service-to-service)
   - Service identity verification
   - VirtualService routing policies

3. Service Mesh Internal (Trusted)
   - mTLS STRICT enforced by PeerAuthentication
   - Service identity propagation
   - AuthorizationPolicy enforcement (optional)
   - Telemetry collection

4. Service Mesh → External Services (S3)
   - TLS 1.2+
   - AWS IAM credentials
   - Custom CA bundle support

5. Control Plane (Controller → API Server)
   - TLS 1.2+
   - ServiceAccount Token authentication
   - RBAC enforcement
   - Webhook mutual TLS (API server → webhook server)
