Network Policies
═══════════════════════════════════════════════════════════════════════════

No explicit NetworkPolicies defined by default.
Network isolation relies on:
  1. OpenShift SDN / Kubernetes Network Policies (platform-level)
  2. Istio Service Mesh policies (when USE_ISTIO=true)
  3. Namespace isolation

Recommended NetworkPolicy (not included by default):
  - Allow controller → Kubernetes API (443/TCP)
  - Allow controller → Notebook Services (80/TCP) for culling
  - Allow Prometheus → Controller metrics (8080/TCP)
  - Allow Istio Gateway → Notebook Services (80/TCP)
  - Deny all other ingress to notebook pods

═══════════════════════════════════════════════════════════════════════════
Known Security Considerations
═══════════════════════════════════════════════════════════════════════════

1. Webhook Server Disabled
   - Conversion webhook code exists but is commented out
   - No validation/mutation of Notebook CRs at admission time
   - Relies on CRD schema validation only

2. No Default Network Encryption
   - Controller → Notebook Service communication is plaintext HTTP
   - Culling queries sent over unencrypted HTTP/80
   - Mitigation: Enable Istio mTLS (USE_ISTIO=true)

3. Metrics Endpoint Unauthenticated
   - /metrics on port 8080 is publicly accessible within cluster
   - No authentication or authorization
   - Exposes notebook usage statistics

4. Culling Feature Network Access
   - Requires controller to access all notebook service endpoints
   - Network path must allow HTTP/80 from controller to notebooks
   - Consider disabling if not needed (ENABLE_CULLING=false)

5. fsGroup Disabled by Default (OpenShift)
   - ADD_FSGROUP=false for OpenShift SCC compatibility
   - May affect volume permissions in some configurations

6. Single Controller Instance
   - No HA by default (leader election available but disabled)
   - Single point of failure for notebook management
   - Enable with --enable-leader-election flag for production
