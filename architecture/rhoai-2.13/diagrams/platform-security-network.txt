TRUST BOUNDARIES
═══════════════════════════════════════════════════════════════════════════════════════════════

1. External → Ingress
   • Trust: UNTRUSTED → DMZ
   • Enforcement: OpenShift OAuth, TLS 1.2+ termination, Route policies

2. Ingress → Platform Control Plane
   • Trust: DMZ → TRUSTED (platform operators)
   • Enforcement: mTLS ServiceAccount authentication, RBAC

3. Platform Control Plane → User Workloads
   • Trust: TRUSTED (operators) → SEMI-TRUSTED (user pods)
   • Enforcement: RBAC (operators can manage user resources), Pod Security Standards

4. User Workloads → External Services
   • Trust: SEMI-TRUSTED → UNTRUSTED
   • Enforcement: Network Policies (optional), Secret-based auth (S3 keys, OAuth tokens)

5. Service Mesh Internal
   • Trust: TRUSTED (mTLS authenticated pods)
   • Enforcement: PeerAuthentication STRICT, AuthorizationPolicy, mTLS cert validation
