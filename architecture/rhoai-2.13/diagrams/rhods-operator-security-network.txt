TRUST BOUNDARIES
═══════════════════════════════════════════════════════════════════════════

1. EXTERNAL → CLUSTER: TLS 1.2+, User OAuth/kubeconfig
2. API SERVER → OPERATOR: TLS 1.2+, ServiceAccount token, RBAC
3. OPERATOR → MANAGED COMPONENTS: TLS 1.2+, ServiceAccount token, manifest deployment
4. OPERATOR → EXTERNAL SERVICES: TLS 1.2+, public/pull secret auth
5. INTERNAL (metrics): HTTP plaintext (cluster-internal only, ServiceAccount token required)
6. MONITORING: HTTPS with service-ca TLS, mTLS
7. ROUTES: TLS termination/passthrough, OpenShift OAuth
