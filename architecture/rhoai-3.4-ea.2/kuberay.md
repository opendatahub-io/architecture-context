## Purpose
**Short**: KubeRay operator manages the lifecycle of Ray clusters, Ray jobs, and Ray services on Kubernetes/OpenShift.

**Detailed**: The KubeRay operator is a Kubernetes controller-runtime based operator that manages three primary custom resources: RayCluster, RayJob, and RayService. It automates the provisioning and lifecycle management of Ray distributed computing clusters, including head and worker pod orchestration, autoscaler configuration, service creation, and ingress management.

In the RHOAI distribution, the operator includes significant OpenShift-specific enhancements: a mutating webhook enforces Gateway API-only access by disabling basic Route/Ingress creation and enabling secure trusted networking annotations on all RayClusters. The authentication controller creates HTTPRoute resources in the platform namespace (referencing the data-science-gateway) with kube-rbac-proxy sidecar injection for RBAC-based authentication. A dedicated NetworkPolicy controller creates per-cluster network policies when the secure trusted network annotation is present. An mTLS controller manages cert-manager Certificate and Issuer resources for intra-cluster mutual TLS between Ray head and worker nodes.

The operator is built with FIPS-compliant Go compilation (`GOEXPERIMENT=strictfipsruntime`) for RHOAI and is deployed into the `opendatahub` namespace via kustomize overlays managed by the rhods-operator.

**RHOAI 3.x Ingress Pattern (Gateway API)**:
- The mutating webhook sets `enableIngress=false` and `odh.ray.io/secure-trusted-network=true` on all RayClusters on OpenShift
- The Authentication Controller creates HTTPRoute CRs in the platform namespace (`opendatahub`) referencing the `data-science-gateway` Gateway in `openshift-ingress` namespace
- HTTPRoute path: `/ray/{namespace}/{cluster}` with URL rewrite to `/` on the backend (port 8265)
- Cross-namespace service references are enabled via ReferenceGrant (`kuberay-gateway-access`) created in the RayCluster's namespace
- kube-rbac-proxy sidecar (port 8443/TCP) fronts the Ray dashboard for RBAC-based authentication
- Direct OpenShift Route creation is suppressed by the webhook to enforce centralized Gateway API access
