## Purpose
**Short**: Provides container runtime images and integration tests for distributed AI/ML training workloads on Red Hat OpenShift AI (RHOAI).

**Detailed**: The Distributed Workloads repository is a container image build and integration testing repository for the RHOAI distributed workloads stack. It does not contain an operator or controller itself, but rather provides:

1. **Runtime container images** for distributed training across multiple GPU accelerator backends (NVIDIA CUDA, AMD ROCm) and Python/PyTorch versions. These images are used as the execution environment for PyTorchJob, TrainJob, and RayCluster/RayJob workloads. Images are built via Konflux (Tekton-based CI/CD) pipelines and include CUDA/ROCm libraries, PyTorch, DeepSpeed, FlashAttention, NCCL, InfiniBand/RDMA drivers, and ML training libraries.

2. **Universal training images** (Training Hub 0.6 / "th06") that combine Jupyter workbench base images with runtime ML stacks (PyTorch, FlashAttention, Mamba) for both interactive and batch training use cases. These are FIPS-friendly with multi-stage builds that isolate build tools from runtime.

3. **Comprehensive end-to-end integration tests** (Go-based) that validate the distributed workloads stack including Kubeflow Training Operator (PyTorchJob), Kubeflow Trainer v2 (TrainJob), KubeRay (RayCluster/RayJob), Kueue (job queuing), JobSet, and their integration with RHOAI's DataScienceCluster and Kueue Operator CRDs. Tests cover MNIST training, LLM fine-tuning (SFT), hyperparameter optimization (Ray Tune), upgrade scenarios, and multi-node/multi-GPU configurations.
