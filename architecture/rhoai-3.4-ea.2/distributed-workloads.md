## Purpose
**Short**: Provides container runtime images for distributed ML training workloads and end-to-end integration tests for the RHOAI distributed workloads stack.

**Detailed**: The distributed-workloads repository is not an operator or service itself; rather, it serves two critical roles in the RHOAI platform. First, it provides the container images used as runtime environments for distributed machine learning training jobs. These images include Ray cluster runtimes (with CUDA and ROCm GPU support), PyTorch training runtimes (across multiple CUDA/ROCm and PyTorch version combinations), and "universal" training images that combine Jupyter workbench capabilities with ML runtime stacks (TH06 generation).

Second, the repository houses a comprehensive Go-based integration test suite that validates the distributed workloads stack across multiple components: Kubeflow Training Operator (PyTorchJob), Kubeflow Trainer v2 (TrainJob), KubeRay (RayCluster/RayJob), Kueue (job queuing and resource management), and JobSet. The tests exercise workflows including MNIST training, LLM fine-tuning with SFT/LoRA/OSFT, hyperparameter optimization with Ray Tune, and DeepSpeed-based distributed training. Tests validate both ODH and RHOAI environments, with support for OCP, OSD, Hypershift, and KIND cluster types.

The Tekton CI/CD pipeline definitions (in `.tekton/`) are synchronized from the `konflux-central` repository and drive Konflux-based container image builds for all runtime images.
