## Purpose
**Short**: A lightweight Rust-based HTTP service that detects sensitive content (PII) in text using predefined and custom regular expressions, designed to integrate with the FMS Guardrails Orchestrator.

**Detailed**: The Guardrails Regex Detector is a specialized text analysis microservice built in Rust using the Axum web framework. It provides regex-based content detection capabilities as part of the Red Hat AI (RHOAI) guardrails infrastructure. The service accepts text content via a REST API and scans it against both built-in patterns (email addresses, Social Security numbers, credit card numbers) and user-supplied custom regular expressions.

This component operates as a backend detector service for the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator). The orchestrator routes text content to this detector for PII scanning, and the detector returns structured detection results including match positions, matched text, detection type classification, and confidence scores. This enables AI/ML inference pipelines to filter or flag sensitive content before it reaches end users.

The service is intentionally minimal and stateless, with no external dependencies beyond the Rust standard library and a few well-audited crates (axum, regex, serde, tokio). This makes it suitable for high-performance, low-latency detection in production guardrails pipelines.

### Request/Response Schema

**POST /api/v1/text/contents**

Request body:
```json
{
  "contents": ["string array of text to scan"],
  "detector_params": {
    "regex": ["ssn", "email", "credit-card", "^custom-regex$"]
  }
}
```

Response body:
```json
[
  [
    {
      "start": 0,
      "end": 11,
      "text": "matched text",
      "detection_type": "pii|custom",
      "detection": "EmailAddress|SocialSecurity|CreditCard|custom",
      "score": 1.0
    }
  ]
]
```
