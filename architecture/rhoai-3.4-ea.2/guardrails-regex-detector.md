## Purpose
**Short**: A lightweight Rust-based HTTP server that detects patterns (PII and custom regex) in text content, integrated with the FMS Guardrails Orchestrator.

**Detailed**: The Guardrails Regex Detector is a stateless microservice built in Rust using the Axum web framework. It provides a REST API for scanning text content against both built-in PII detection patterns (social security numbers, credit card numbers, email addresses) and user-supplied custom regular expressions.

The service is designed to be called by the FMS Guardrails Orchestrator as part of a guardrails pipeline for large language model (LLM) inference. When integrated into the RHOAI platform, it enables content filtering by identifying sensitive information in text before or after model inference. Each detection returns the matched text, its position (start/end offsets), a detection type classification (e.g., "pii" or "custom"), and a confidence score.

The application is packaged as a container image based on Red Hat UBI 9 Minimal and listens on a configurable HTTP port (default 8080). It has no external service dependencies, no persistent storage requirements, and no CRDs — it is a pure stateless detection service.
