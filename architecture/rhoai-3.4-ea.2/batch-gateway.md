## Purpose
**Short**: OpenAI-compatible Batch API gateway for asynchronous LLM inference request processing within the llm-d ecosystem.

**Detailed**: The batch-gateway implements the [OpenAI `/v1/file` and `/v1/batch` APIs](https://platform.openai.com/docs/guides/batch), enabling users to submit, track, and manage batch inference jobs asynchronously. Batch inference allows grouping large numbers of inference requests into jobs with relaxed time tolerances (hours to days), as opposed to interactive inference (seconds to minutes). This enables higher throughput, better GPU utilization, and potentially differential billing for batch workloads.

The system consists of two main components: an **API Server** that exposes the OpenAI-compatible REST API for managing files and batch jobs, and a **Batch Processor** that asynchronously pulls jobs from a priority queue, reads the input JSONL files, dispatches individual inference requests to a downstream inference gateway (llm-d or GAIE), and writes result files. The data layer supports pluggable backends, with Redis for metadata storage, priority queues, event channels, and temporary status; and S3-compatible or filesystem-based storage for batch input/output files.

The component is in early access (EA) status. The processor's job execution logic is partially implemented (marked with TODO comments), while the API server's file and batch management endpoints are functional. Multi-tenancy is supported via a configurable tenant ID header (default: `X-MaaS-User`).
