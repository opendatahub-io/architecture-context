## Purpose
**Short**: High-performance C++ model inference server supporting KServe, TensorFlow Serving, and OpenAI-compatible APIs over gRPC and REST.

**Detailed**: OpenVINO Model Server (OVMS) is a high-performance inference serving system developed by Intel and packaged by Red Hat for RHOAI. Implemented in C++ for maximum scalability and optimized for deployment on Intel architectures, OVMS serves machine learning models via gRPC and REST APIs. It supports multiple API protocols including KServe Predict v2, TensorFlow Serving, and OpenAI-compatible generative AI endpoints (chat completions, embeddings, reranking, image generation, speech-to-text, text-to-speech).

The server provides a modular architecture with distinct modules for gRPC serving, HTTP/REST serving (via Drogon web framework), model management, metrics collection (Prometheus-compatible), MediaPipe graph execution for complex pipelines, Python code execution support, and HuggingFace model pulling. Models can be loaded from local filesystem, Amazon S3, Google Cloud Storage, or Azure Blob Storage. OVMS supports advanced features including DAG (Directed Acyclic Graph) pipeline scheduling, model versioning, dynamic input shapes, continuous batching for LLMs, streaming inference, and custom node extensions.

In the RHOAI platform, OVMS functions as a model serving runtime container that is typically deployed by KServe's InferenceService controller or directly as a Kubernetes Deployment. It does not manage any CRDs itself; instead, it is a workload container that receives inference requests and returns predictions.
