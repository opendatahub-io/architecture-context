Network Policies:
- {registry-name}-https-route:
  - Pod: app={registry-name}, component=model-registry
  - Allow from: namespaceSelector network.openshift.io/policy-group=ingress
  - Port: 8443/TCP
  - Purpose: Allow OpenShift router ingress to kube-rbac-proxy

- {registry-name}-postgres:
  - Pod: app={registry-name}, app.kubernetes.io/name={registry-name}-postgres
  - Allow from: podSelector app={registry-name}, component=model-registry
  - Port: 5432/TCP
  - Purpose: Only model registry pods can access PostgreSQL

- model-catalog-https-route:
  - Pod: app=model-catalog, component=model-catalog
  - Allow from: namespaceSelector network.openshift.io/policy-group=ingress
  - Port: 8443/TCP
  - Purpose: Allow OpenShift router ingress to catalog kube-rbac-proxy

- model-catalog-postgres:
  - Pod: app=model-catalog, app.kubernetes.io/name=model-catalog-postgres
  - Allow from: podSelector app=model-catalog, component=model-catalog
  - Port: 5432/TCP
  - Purpose: Only catalog pods can access catalog PostgreSQL

RBAC Summary:
- ClusterRole: manager-role
  - ServiceAccount: controller-manager
  - Key permissions:
    - modelregistry.opendatahub.io/modelregistries: full CRUD
    - core resources (configmaps, secrets, services, sa, pvc): full CRUD
    - apps/deployments: full CRUD
    - rbac.authorization.k8s.io (roles, rolebindings, clusterrolebindings): full CRUD
    - route.openshift.io/routes: full CRUD
    - user.openshift.io/groups: full CRUD
    - networking.k8s.io/networkpolicies: full CRUD
    - authentication.k8s.io/tokenreviews: create
    - authorization.k8s.io/subjectaccessreviews: create
    - migration.k8s.io/storageversionmigrations: full CRUD
    - components.platform.opendatahub.io/modelregistries: read-only
    - services.platform.opendatahub.io/auths: read-only

- Per-Registry Role: registry-user-{name}
  - Resources: services/{name}, endpoints/{name}
  - Verbs: get
  - Purpose: kube-rbac-proxy SAR authorization for registry access

- ClusterRoleBinding: {registry-name}-auth-delegator
  - Role: system:auth-delegator
  - SA: {registry-name} in registry namespace
  - Purpose: Allow kube-rbac-proxy to perform SAR

Secrets:
- {registry-name}-kube-rbac-proxy (kubernetes.io/tls)
  - TLS cert for kube-rbac-proxy sidecar
  - Provisioned by: OpenShift serving cert controller (annotation)
  - Auto-rotate: Yes

- {registry-name}-postgres-credentials (Opaque)
  - Auto-generated PostgreSQL username/password (random 16-byte password)
  - Auto-rotate: No

- model-catalog-postgres-credentials (Opaque)
  - Auto-generated catalog PostgreSQL username/password
  - Auto-rotate: No

- webhook-service TLS (kubernetes.io/tls)
  - TLS cert for webhook server
  - Provisioned by: OpenShift serving cert controller
  - Auto-rotate: Yes

- controller-manager-metrics-service TLS (kubernetes.io/tls)
  - TLS cert for metrics endpoint
  - Provisioned by: OpenShift serving cert controller
  - Auto-rotate: Yes

Authentication & Authorization:
- Model Registry REST API (8443): SubjectAccessReview via kube-rbac-proxy
  - User must have `get` access to services/{registry-name} in registry namespace
- Model Catalog API (8443): SubjectAccessReview via kube-rbac-proxy
  - User must have `get` access to services/model-catalog in catalog namespace
- Operator Metrics (8443): Bearer Token + SubjectAccessReview
- Webhooks (9443): Kubernetes API Server mTLS
