RBAC Summary:
- MLServer does NOT define its own RBAC resources
- Authentication/Authorization is enforced externally by:
  - kube-rbac-proxy sidecar (RHOAI 3.x): RBAC on Kubernetes ServiceAccount
  - oauth-proxy sidecar (RHOAI 2.x, legacy): OpenShift OAuth
- MLServer listens on plaintext ports within the pod

Container Security:
- User ID: 1000 (mlserver) - non-root execution
- Working Directory: /opt/mlserver
- World-writable app dir: chmod 776 /opt/mlserver (OpenShift random UID compat)
- FIPS Compliant: Yes (platform-level)
- Base Image: quay.io/aipcc/base-images/cpu:3.4.0-ea.2

Secrets:
- MLServer does NOT directly consume Kubernetes Secrets
- Model artifacts are mounted via volumes managed by KServe

Network Exposure:
- Port 8080/TCP: REST (V2 Inference Protocol) - internal only
- Port 8081/TCP: gRPC (V2 Inference Protocol) - internal only
- Port 8082/TCP: Prometheus metrics - internal only
- All ports are plaintext; TLS is terminated at the KServe Gateway level
- No direct external exposure; all access via KServe ingress infrastructure
