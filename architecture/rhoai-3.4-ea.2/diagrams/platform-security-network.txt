RBAC Summary

Cluster-Scoped ClusterRoles (Major):

  ClusterRole: opendatahub-operator-manager
  - Component: rhods-operator
  - Permissions: Full CRUD on all ODH CRDs, K8s core resources, OLM,
                 monitoring, gateway, networking

  ClusterRole: rhods-dashboard
  - Component: odh-dashboard
  - Permissions: Read/write notebooks, ISVCs, configmaps, secrets, RBAC, namespaces

  ClusterRole: odh-model-controller-role
  - Component: odh-model-controller
  - Permissions: ISVCs, routes, gateways, authpolicies, envoyfilters, networkpolicies

  ClusterRole: training-operator
  - Component: training-operator
  - Permissions: All training job CRDs, pods, services, networkpolicies, PodGroups

  ClusterRole: kubeflow-trainer-controller-manager
  - Component: trainer
  - Permissions: TrainJob, JobSet, PodGroup, networkpolicies

  ClusterRole: manager-role (trustyai)
  - Component: trustyai-operator
  - Permissions: All TrustyAI CRDs, ISVCs, kueue workloads, routes, RBAC

  ClusterRole: spark-operator-controller
  - Component: spark-operator
  - Permissions: SparkApplication CRDs, pods, services, ingresses

  ClusterRole: manager-role (wva)
  - Component: workload-variant-autoscaler
  - Permissions: VariantAutoscaling, deployments/scale, InferencePool

Aggregated User-Facing Roles:

  Role: training-edit -> aggregates to: edit, admin
  - Resources: PyTorchJob, TFJob, XGBoostJob, MPIJob, PaddleJob, JAXJob, TrainJob

  Role: training-view -> aggregates to: view
  - Resources: All training job CRDs (read-only)

Authentication Mechanisms
================================================================================

  Mechanism                              | Enforcement Point            | Components
  ---------------------------------------|------------------------------|---------------------------------------------
  kube-rbac-proxy (SubjectAccessReview)  | Sidecar container            | Dashboard, Notebooks, KServe ISVCs,
                                         |                              | TrustyAI, GuardrailsOrch, NeMo, KubeRay, DSP
  OAuth2/OIDC (kube-auth-proxy)          | Gateway EnvoyFilter          | All Gateway API-routed services
  Istio mTLS + AuthorizationPolicy       | Istio sidecar proxy          | Model Registry (proxy + catalog)
  Kuadrant AuthPolicy (Authorino)        | Gateway + HTTPRoute          | LLMInferenceService (MaaS)
  K8s RBAC (SelfSubjectAccessReview)     | Application code             | MLflow (kubernetes-auth plugin)
  API Key (Bearer token)                 | Application middleware       | vLLM, OVMS, NeMo Guardrails
  TLS client certificates                | TLS termination              | EvalHub
