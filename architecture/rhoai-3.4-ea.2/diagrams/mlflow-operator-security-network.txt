NetworkPolicy (per MLflow instance):
  Ingress:
  - Allow TCP/8443 from any pod in any namespace
    (cluster-wide: gateway, dashboard, Prometheus)

  Egress:
  - DNS: 53/UDP, 53/TCP
  - HTTPS: 443/TCP
  - Kubernetes API: 6443/TCP
  - PostgreSQL: 5432/TCP
  - MySQL: 3306/TCP
  - MinIO: 9000/TCP
  - SeaweedFS: 8333/TCP
  - Custom rules via spec.networkPolicyAdditionalEgressRules (max 32)

Secrets:
  - mlflow-tls (kubernetes.io/tls)
    Purpose: TLS cert/key for MLflow server HTTPS
    Provisioned by: OpenShift service-ca operator (auto-rotate)
  - mlflow-artifact-connection (Opaque)
    Purpose: S3 credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY,
             AWS_S3_BUCKET, AWS_S3_ENDPOINT)
    Provisioned by: User (per namespace)

Authentication & Authorization:
  - MLflow API: Kubernetes RBAC via SelfSubjectAccessReview
    Checks pseudo-resources: datasets, experiments, registeredmodels,
    gatewayendpoints, gatewaymodeldefinitions in mlflow.kubeflow.org
  - Operator Metrics: Bearer Token validated via TokenReview + SubjectAccessReview
