- ClusterRole: manager-role
  - mlflow.opendatahub.io: mlflows (full CRUD), mlflows/status (get, update, patch), mlflows/finalizers (update)
  - mlflow.kubeflow.org: mlflowconfigs (get, list, watch)
  - core: namespaces (get, list, watch), secrets/mlflow-artifact-connection (get)
  - gateway.networking.k8s.io: httproutes (full CRUD)
  - console.openshift.io: consolelinks (full CRUD)
  - rbac.authorization.k8s.io: clusterroles, clusterrolebindings (full CRUD)
- ClusterRole: metrics-auth-role
  - authentication.k8s.io: tokenreviews (create)
  - authorization.k8s.io: subjectaccessreviews (create)
- Namespace Role: manager-role
  - core: configmaps, pvc, secrets, serviceaccounts, services (full CRUD)
  - apps: deployments (full CRUD)
  - networking.k8s.io: networkpolicies (full CRUD)
  - monitoring.coreos.com: servicemonitors (full CRUD)

MLflow Server ServiceAccount (runtime):
- ClusterRole: mlflow (shared)
  - core: namespaces (get, list, watch), secrets/mlflow-artifact-connection (get)
  - mlflow.kubeflow.org: mlflowconfigs (get, list, watch)

User-Facing Aggregate Roles:
- mlflow-view: read-only access to MLflow and MLflowConfig resources (aggregates to view, edit, admin)
- mlflow-edit: full CRUD on MLflow resources (aggregates to edit, admin)
- mlflow-integration: API access for automated tools (standalone role)

Secrets:
- mlflow-tls (kubernetes.io/tls) - TLS cert/key for MLflow HTTPS, provisioned by OpenShift service-ca, auto-rotated
- mlflow-artifact-connection (Opaque) - S3 credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_S3_BUCKET, AWS_S3_ENDPOINT), user-provisioned, no auto-rotation

Authentication & Authorization:
- MLflow API: Kubernetes ServiceAccount Token → SelfSubjectAccessReview against mlflow.kubeflow.org pseudo-resources
- Security middleware hardcoded ENABLED (MLFLOW_SERVER_DISABLE_SECURITY_MIDDLEWARE="false", CRD validation prevents override)
- Operator metrics: Bearer Token validated via TokenReview + SubjectAccessReview
- MLflow metrics: No auth (Prometheus ServiceMonitor with service-ca TLS)

Network Policies:
- Policy: mlflow (pod selector: app=mlflow)
  - Ingress: Allow 8443/TCP from any namespace (cluster-wide)
  - Egress: DNS(53/UDP,53/TCP), HTTPS(443/TCP), K8s API(6443/TCP),
            PostgreSQL(5432/TCP), MySQL(3306/TCP), MinIO(9000/TCP),
            SeaweedFS(8333/TCP), + user-configured additional rules (max 32)
