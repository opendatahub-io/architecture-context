Authentication Mechanisms:
- External API access: SubjectAccessReview via kube-rbac-proxy (DSPA 'api' subresource)
- External MLMD access: SubjectAccessReview via kube-rbac-proxy (DSPA 'api' subresource)
- Internal API access: NetworkPolicy-restricted (no auth, trusted network)
- Internal MLMD access: NetworkPolicy-restricted (no auth, trusted network)
- MariaDB access: Username/Password + NetworkPolicy
- S3/Minio access: AccessKey/SecretKey
- Webhook: TLS client cert from Kubernetes API server
- K8s API access: ServiceAccount token (controller-manager SA)
