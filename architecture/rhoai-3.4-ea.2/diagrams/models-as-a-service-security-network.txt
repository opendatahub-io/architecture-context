AUTHENTICATION & AUTHORIZATION
================================================================================

Endpoint: /v1/models (via gateway)
  Auth: Bearer Token (API key sk-oai-* or OpenShift token)
  Enforcement: Authorino (AuthPolicy on maas-api-route HTTPRoute)
  Policy: Dual auth - API key via maas-api callback; OpenShift token via
          Kubernetes TokenReview

Endpoint: /maas-api/* (via gateway)
  Auth: Bearer Token (API key sk-oai-* or OpenShift token)
  Enforcement: Authorino (AuthPolicy on maas-api-route HTTPRoute)
  Policy: Same dual auth as /v1/models

Endpoint: Model inference paths (via gateway)
  Auth: Bearer Token (API key sk-oai-*)
  Enforcement: Authorino (per-model AuthPolicy on model HTTPRoutes)
  Policy: API key validation + group/user membership check;
          subscription selection for rate limiting

Endpoint: /internal/v1/api-keys/validate
  Auth: None (NetworkPolicy restricted)
  Enforcement: NetworkPolicy (maas-authorino-allow)
  Policy: Only Authorino pods can reach this endpoint

Endpoint: /v1/subscriptions/select
  Auth: None (NetworkPolicy restricted)
  Enforcement: NetworkPolicy (maas-authorino-allow)
  Policy: Only Authorino pods can reach this endpoint
