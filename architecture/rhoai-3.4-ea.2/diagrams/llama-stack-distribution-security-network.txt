RBAC Summary:
  This component does not define its own RBAC resources.
  - Runs as non-root user (CNB_USER_ID/CNB_GROUP_ID via UBI9 base image)
  - Kubernetes API RBAC is managed by the deploying operator (e.g., rhods-operator)
  - Platform ingress RBAC via kube-rbac-proxy (RHOAI 3.x)

Authentication Summary:
  - Container-level: No authentication (HTTP plaintext on port 8321)
  - Platform-level: kube-rbac-proxy enforces OpenShift token validation at ingress
  - Backend auth: Per-provider API keys/tokens via environment variables

  - git_auth_secret (Tekton workspace) - Git auth for Konflux builds - Konflux - No auto-rotate

Network Security Notes:
  - The Llama Stack server does NOT implement TLS or authentication natively
  - All security enforcement is delegated to the RHOAI platform ingress layer
  - Internal cluster traffic to port 8321 is plaintext HTTP
  - Provider connections support configurable TLS (per-provider env vars)
  - Milvus supports mTLS with client certificates
  - TrustyAI FMS supports custom CA cert path (FMS_SSL_CERT_PATH)
  - vLLM TLS verification can be controlled via VLLM_TLS_VERIFY
