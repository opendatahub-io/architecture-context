INGRESS SUMMARY
================================================================================

  No ingress resources defined in this repository.
  Ingress is managed by the RHOAI platform operator via:
  - HTTPRoute + kube-rbac-proxy (RHOAI 3.x Gateway API pattern)
  - Component only exposes plain HTTP server on port 8321

================================================================================
AUTHENTICATION & AUTHORIZATION SUMMARY
================================================================================

  Application-Level Auth: NONE
  - All /v1/* endpoints have no application-level authentication
  - Relies entirely on platform-level auth (kube-rbac-proxy / Gateway API)

  Egress Auth Mechanisms:
  - vLLM: API Token (VLLM_API_TOKEN)
  - AWS Bedrock: AWS Bearer Token (AWS_BEARER_TOKEN_BEDROCK)
  - Azure OpenAI: API Key (AZURE_API_KEY)
  - Google Vertex AI: Service Account JSON
  - IBM WatsonX: API Key (WATSONX_API_KEY)
  - OpenAI: API Key Bearer (OPENAI_API_KEY)
  - PostgreSQL: Username/Password (POSTGRES_PASSWORD)
  - Milvus: Token + client certs (MILVUS_TOKEN)
  - pgvector: Username/Password (PGVECTOR_PASSWORD)
  - Qdrant: API Key (QDRANT_API_KEY)
  - Brave Search: API Key (BRAVE_SEARCH_API_KEY)
  - Tavily Search: API Key (TAVILY_SEARCH_API_KEY)
  - S3: AWS IAM (AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY)
  - Kubeflow: API Token (KUBEFLOW_PIPELINES_TOKEN)

================================================================================
SECURITY NOTES
================================================================================

  1. NO TLS at application level - all connections to port 8321 are plaintext HTTP
  2. Platform is expected to provide TLS termination and authentication
  3. All secrets are passed as environment variables (no Kubernetes Secret mounts)
  4. No RBAC roles/bindings defined in this repository
  5. No PeerAuthentication or AuthorizationPolicy defined
  6. PostgreSQL connection defaults to plaintext (configurable)
  7. vLLM TLS verification is configurable via VLLM_TLS_VERIFY env var
