RBAC Summary:
- This repository does NOT define RBAC resources
- RBAC is managed by the ODH Notebook Controller and RHOAI Operator
- kube-rbac-proxy sidecar enforces SubjectAccessReview per notebook

Authentication & Authorization:
- Endpoint: Workbench UI (8888/TCP)
  - Methods: GET, POST, WS
  - Auth Mechanism: kube-rbac-proxy (RHOAI 3.x)
  - Enforcement Point: kube-rbac-proxy sidecar (8443/TCP)
  - Policy: Kubernetes RBAC SubjectAccessReview

Container Security:
- Base Image: UBI9 (Red Hat Universal Base Image 9)
- User: 1001 (non-root)
- Group: 0 (root group for OpenShift compatibility)
- Build Mode: Hermetic (Konflux + cachi2 prefetch)
- Dependency Verification: Hash-verified (pylock.toml with --require-hashes)
- FIPS Compliance: Enabled (compat-openssl11 for FIPS scan compliance)
- Supply Chain: Tekton pipelines with cachi2 hermetic builds
- Supported Architectures: amd64, arm64, ppc64le, s390x

Secrets:
- No secrets defined in this repository
- Workbench containers run as non-root user (UID 1001, GID 0)

- Service Name: (notebook-name) - dynamically created by ODH Notebook Controller
  - Type: ClusterIP
  - Port: 8888/TCP → 8888
  - Protocol: HTTP

  - Auth: None (auth at ingress)
