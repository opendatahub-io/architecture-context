RBAC Summary:
=============
This component does not define RBAC resources directly.
RBAC is managed by the ODH Notebook Controller and rhods-operator.

- kube-rbac-proxy sidecar enforces SubjectAccessReview for workbench access
- Users must have RBAC permissions on the Notebook resource in their namespace

Secrets:
========
- rhel-subscription-secret (Opaque)
  - Purpose: RHEL subscription for RStudio BuildConfig
  - Provisioned by: Manual / cluster admin
  - Auto-rotate: No

Container Security:
===================
- Run as non-root: Yes (USER 1001)
- Read-only root filesystem: No (writable for pip installs, user data)
- Capabilities dropped: Standard OpenShift SCC
- Base image: UBI 9 (Red Hat Universal Base Image)
- FIPS-compliant: Yes (enabled via operator feature)

Authentication & Authorization:
===============================
- External access: kube-rbac-proxy sidecar (RHOAI 3.x)
  - Enforcement: RBAC SubjectAccessReview on 8443/TCP
  - Protocol: HTTPS with TLS
- In-pod access (not directly exposed):
  - JupyterLab: Token auth (generated at startup)
  - RStudio: Session-based auth
  - code-server: Password-based auth
