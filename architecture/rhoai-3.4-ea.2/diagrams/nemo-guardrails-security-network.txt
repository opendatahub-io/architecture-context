RBAC Summary:
- No RBAC resources defined (stateless application, not an operator)
- RBAC managed by deploying operator (rhods-operator or TrustyAI operator)
- Container runs as non-root user (UID 1001)

Authentication & Authorization:
- NeMo Guardrails itself does NOT implement authentication
- Security provided by:
  1. Network-level isolation (ClusterIP service, not externally exposed)
  2. Platform ingress authentication (kube-rbac-proxy sidecar if exposed via HTTPRoute)
  3. Authorization headers forwarded transparently to upstream LLM endpoint

Secrets:
- (configurable) Opaque - LLM API keys via environment variables
  - MAIN_MODEL_BASE_URL: Upstream LLM endpoint URL
  - Authorization headers: Forwarded to upstream LLM
  - Provisioned by: User/Operator
  - Auto-Rotate: No

Security-Relevant Environment Variables:
- MAIN_MODEL_ENGINE: LLM provider engine type (openai, nvidia)
- MAIN_MODEL_BASE_URL: Upstream LLM endpoint URL
- DEFAULT_CONFIG_ID: Default guardrails configuration
- NEMO_GUARDRAILS_SERVER_ENABLE_CORS: CORS enable/disable (default: false)
- NEMO_GUARDRAILS_SERVER_ALLOWED_ORIGINS: CORS origins (default: "*")
- GUARDRAILS_PROFILE: Build-time guardrail selection (opensource/all)

- Base Image: UBI9/Python 3.12 (Red Hat Universal Base Image)

- Build System: Konflux (Tekton PipelineRun)
- Guardrails Profile: opensource (closed-source integrations removed at build time)
