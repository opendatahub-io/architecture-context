Configuration:
- Config mounted via ConfigMap at /app/config/<CONFIG_ID>/
- Required files: config.yaml, rails.co
- Optional: challenges.json
- Environment: CONFIG_ID, PORT, HF_HOME, TRANSFORMERS_CACHE, etc.

Secrets:
- LLM API Key (Opaque) - Auth to backend LLM endpoint, provisioned by user/deployer, no auto-rotate
- Redis password (Opaque, optional) - Auth to Redis datastore, provisioned by user/deployer, no auto-rotate

Authentication Notes:
- NeMo Guardrails server does NOT implement authentication itself
- Authentication/authorization enforced at platform ingress layer:
  - kube-rbac-proxy sidecar, or
  - Gateway API auth policies
- Backend LLM auth via Bearer Token / API Key configured in guardrails config.yaml

Container Security:
- Base image: UBI9 Python 3.12
- Non-root user: 1001
- Closed-source guardrails removed at build time (opensource profile)
- FIPS compliant (per operator feature flags)
- Multi-arch: amd64, arm64, ppc64le, s390x
