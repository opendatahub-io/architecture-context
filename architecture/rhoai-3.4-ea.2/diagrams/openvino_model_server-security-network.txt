RBAC Summary:
- No RBAC resources are defined by OVMS.
- OVMS is a workload container; RBAC is managed by the deploying controller
  (KServe InferenceService controller or ModelMesh).

Authentication & Authorization:
- /v3/* endpoints (OpenAI-compatible):
  - Optional API Key authentication via Bearer token or "api-key" header
  - Configured via --api_key_file CLI flag or API_KEY environment variable
  - Enforcement: OVMS HttpRestApiHandler
- /v1/* endpoints (TFS): No built-in authentication
- /v2/* endpoints (KServe): No built-in authentication
- gRPC services: No built-in authentication; relies on network-level security
- /metrics endpoint: No built-in authentication

CORS Configuration:
- Allow Credentials: false (default)
- Allowed Origins: * (default, configurable via --allowed_origins)
- Allowed Methods: * (default, configurable via --allowed_methods)
- Allowed Headers: * (default, configurable via --allowed_headers)

Secrets:
- HF_TOKEN (env var, Opaque) - HuggingFace Hub auth, provisioned by User/Admin, no auto-rotate
- API_KEY (env var, Opaque) - API key for /v3 endpoints, provisioned by User/Admin, no auto-rotate
- AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY (env vars, Opaque) - S3 access, User/Admin, no auto-rotate
- GOOGLE_APPLICATION_CREDENTIALS (env var, Opaque) - GCS access, User/Admin, no auto-rotate
- Azure storage credentials (env vars, Opaque) - Azure access, User/Admin, no auto-rotate

Container Security:
- Runs as non-root user: ovms (UID 5000, GID 5000)
- Base image: UBI9-minimal (registry.access.redhat.com/ubi9-minimal:9.7)
- No privileged capabilities required
