RBAC Summary:
=============

Static ClusterRole: manager-role
  Bound to: SA odh-notebook-controller-manager (opendatahub)
  - core: configmaps, secrets, serviceaccounts, services → create, get, list, patch, update, watch
  - authentication.k8s.io: tokenreviews → create
  - authorization.k8s.io: subjectaccessreviews → create
  - config.openshift.io: proxies → get, list, watch
  - datasciencepipelinesapplications.opendatahub.io: datasciencepipelinesapplications → get, list, watch
  - datasciencepipelinesapplications.opendatahub.io: datasciencepipelinesapplications/api → create, delete, get, patch, update
  - gateway.networking.k8s.io: gateways → get, list, watch
  - gateway.networking.k8s.io: httproutes, referencegrants → create, delete, get, list, patch, update, watch

Authentication & Authorization:
===============================

| Endpoint                               | Auth Mechanism                              | Enforcement Point     |
|----------------------------------------|---------------------------------------------|-----------------------|
| Notebook via HTTPRoute (auth enabled)  | kube-rbac-proxy: TokenReview + SAR          | kube-rbac-proxy sidecar|
| Notebook via HTTPRoute (auth disabled) | None (direct to notebook port 8888)         | Gateway only          |
| Webhook endpoint                       | Kubernetes API server webhook auth          | API server            |

Cross-Namespace Architecture:
============================

- HTTPRoutes: Created in controller namespace (opendatahub), NOT user namespace
- ReferenceGrants: Created in user namespaces to allow cross-namespace backend refs
- Gateway: data-science-gateway in openshift-ingress namespace
- Configurable via: NOTEBOOK_GATEWAY_NAME, NOTEBOOK_GATEWAY_NAMESPACE env vars
