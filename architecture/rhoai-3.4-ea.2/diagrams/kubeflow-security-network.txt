RBAC Summary:
  ClusterRole: manager-role
    ServiceAccount: manager (controller namespace)
    Key Permissions:
    - kubeflow.org: notebooks (get, list, patch, update, watch)
    - kubeflow.org: notebooks/finalizers (patch, update)
    - gateway.networking.k8s.io: httproutes, referencegrants (create, delete, get, list, patch, update, watch)
    - gateway.networking.k8s.io: gateways (get, list, watch)
    - networking.k8s.io: networkpolicies (create, get, list, patch, update, watch)
    - rbac.authorization.k8s.io: clusterrolebindings, rolebindings (create, delete, get, list, patch, update, watch)
    - "": configmaps, secrets, serviceaccounts, services (create, get, list, patch, update, watch)
    - authentication.k8s.io: tokenreviews (create)
    - authorization.k8s.io: subjectaccessreviews (create)
    - image.openshift.io: imagestreams (get, list, watch)
    - oauth.openshift.io: oauthclients (delete, get, list, patch, update, watch)
    - config.openshift.io: proxies (get, list, watch)
    - datasciencepipelinesapplications.opendatahub.io: datasciencepipelinesapplications (get, list, watch)

  Aggregated Roles (user-facing):
    - notebooks-view: get, list, watch on notebooks
    - notebooks-edit: full CRUD on notebooks
    - notebooks-admin: aggregates notebooks-edit

  Dynamic Per-Notebook:
    - ClusterRoleBinding: {notebook}-rbac-{ns}-auth-delegator (system:auth-delegator)
    - RoleBinding: elyra-pipelines-{notebook} (ds-pipeline-user-access-dspa)

Secrets:
  - odh-notebook-controller-webhook-cert (kubernetes.io/tls)
    Provisioned by: OpenShift service-ca, Auto-Rotate: Yes
    Purpose: TLS certificate for mutating webhook server

  - {notebook}-kube-rbac-proxy-tls (kubernetes.io/tls, per notebook)
    Provisioned by: OpenShift service-ca (via annotation), Auto-Rotate: Yes
    Purpose: TLS certificate for per-notebook kube-rbac-proxy service

  - ds-pipeline-config (Opaque, per namespace)
    Provisioned by: Controller (from DSPA CR and S3 credentials), Auto-Rotate: No
    Purpose: Elyra runtime pipeline configuration (S3 creds, DSPA endpoint)

ConfigMaps:
  - workbench-trusted-ca-bundle: Aggregated CA certs (odh-trusted-ca-bundle + kube-root-ca.crt + openshift-service-ca.crt)
  - {notebook}-kube-rbac-proxy-config: kube-rbac-proxy authorization config
  - pipeline-runtime-images: Pipeline runtime image metadata from ImageStreams
  - notebook-controller-setting-config: Controller settings (e.g., INJECT_CLUSTER_PROXY_ENV)
