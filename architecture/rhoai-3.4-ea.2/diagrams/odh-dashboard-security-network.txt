Authentication Flow:
================================================================================

1. User authenticates via OpenShift OAuth (external to dashboard)
2. Browser sends Bearer token with each request
3. kube-rbac-proxy validates token via TokenReview API
4. kube-rbac-proxy checks SubjectAccessReview: can user list projects?
5. If authorized, request forwarded to dashboard backend with identity headers
6. Dashboard backend uses user's token for K8s API calls (user's own permissions)
7. Federated modules receive token via X-Forwarded-Access-Token header
