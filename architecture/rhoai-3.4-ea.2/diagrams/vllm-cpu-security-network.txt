RBAC Summary:
- No RBAC resources defined by this component
- RBAC managed by KServe and RHOAI platform operator
- KServe creates ServiceAccount, Services, and HTTPRoutes for this runtime

Container Security:
- User: 2000 (non-root)
- Group: 0 (root group, OpenShift arbitrary UID pattern)
- Base Image: registry.access.redhat.com/ubi9/ubi-minimal (FIPS compliant)
- Telemetry: Disabled (VLLM_NO_USAGE_STATS=1)

Secrets:
- HF_TOKEN (Opaque) - HuggingFace Hub auth token, user-provisioned, no auto-rotate
- SSL certs (kubernetes.io/tls) - Optional TLS certs, user/cert-manager provisioned
- API key (Opaque) - Optional API key for request auth, user-provisioned

Authentication Summary:
- /v1/* endpoints: Optional API Key (Bearer token via --api-key or VLLM_API_KEY env)
- Health/metrics endpoints: Always unauthenticated
- gRPC endpoints: No built-in auth; relies on network-level controls
- Platform level: mTLS / kube-rbac-proxy enforced by RHOAI Gateway/Istio/KServe

Network Security Notes:
- vLLM serves HTTP on plaintext; TLS termination at platform Gateway
- gRPC port (50051) has no encryption or auth - must be network-isolated
- Optional SSL support via --ssl-keyfile/--ssl-certfile (not typical in RHOAI)
- PyZMQ IPC used for multi-process engine communication (localhost only)
