RBAC Summary:
================================================================================

kube-auth-proxy does not define its own ClusterRoles. It inherits
permissions from the parent component's ServiceAccount.

  Required Permissions (via parent SA):
  - API Group: authentication.k8s.io
    Resource: tokenreviews
    Verbs: create
    Purpose: K8s service account token validation

  - API Group: user.openshift.io
    Resource: users
    Verbs: get
    Purpose: OpenShift user info retrieval (OpenShift provider)

================================================================================
Authentication & Authorization Summary:
================================================================================

  kube-auth-proxy mode (port 4180):
  - Auth Mechanisms: OIDC ID Token / OpenShift OAuth / K8s SA Token / Cookie
  - Policy: Email domain validation, allowed groups
  - ext_authz: /oauth2/auth returns 200/401 for Envoy integration

  kube-rbac-proxy mode (port 8443):
  - Auth Mechanism: Bearer Token (ServiceAccount)
  - Authorization: SubjectAccessReview against K8s API
  - TLS: Required (HTTPS)

  Unauthenticated Endpoints:
  - /ping (liveness)
  - /ready (readiness)
