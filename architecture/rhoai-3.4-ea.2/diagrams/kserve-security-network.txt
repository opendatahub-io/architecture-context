   - Purpose: Self-signed TLS certs for LLMInferenceService workload/scheduler

   - Auto-Rotate: No

5. {isvc-name}-predictor-serving-cert (kubernetes.io/tls) [dynamic]

   - Provisioned By: OpenShift service-ca

Authentication & Authorization Summary
================================================================================

1. InferenceService (RawDeployment, auth enabled)
   - Methods: POST
   - Auth: SubjectAccessReview via kube-rbac-proxy
   - Enforcement: kube-rbac-proxy sidecar (8443/TCP)
   - Policy: SAR config from ConfigMap ({isvc-name}-kserve-proxy-config)

2. Webhook endpoints
   - Methods: POST
   - Auth: TLS client certificate
   - Enforcement: Kubernetes API server
   - Policy: Webhook CA bundle validation

3. Controller metrics
   - Methods: GET
   - Auth: Bearer Token
   - Enforcement: kube-rbac-proxy
   - Policy: TokenReview + SubjectAccessReview

4. LLMInferenceService workload
   - Methods: POST
   - Auth: mTLS (self-signed certs + Istio DestinationRule)
   - Enforcement: Istio gateway / direct TLS
   - Policy: TLS SIMPLE mode with CA verification
