Virtual API Resources (mlflow.kubeflow.org API group - enforced by kubernetes-auth):
  - experiments:              get, list, create, update, delete
  - registeredmodels:         get, list, create, update, delete
  - datasets:                 get, list, create, update, delete
  - assistants:               get, create, update
  - gatewaysecrets:           get, list, create, update, delete
  - gatewaysecrets/use:       create
  - gatewayendpoints:         get, list, create, update, delete
  - gatewayendpoints/use:     create
  - gatewaymodeldefinitions:  get, list, create, update, delete
  - gatewaymodeldefinitions/use: create

Role Bindings:
  - mlflow-k8s-workspace-provider (ClusterRoleBinding) → mlflow-server SA (cluster-wide)
  - mlflow-k8s-namespace-access (RoleBinding) → user SAs (per-workspace namespace)

Secrets:
  - mlflow-artifact-connection (Opaque) - S3 bucket URI per-namespace override, provisioned by namespace admin, no auto-rotation
  - Database credentials (Opaque) - PostgreSQL connection string, provisioned by platform operator, no auto-rotation
  - MLFLOW_CRYPTO_KEK_PASSPHRASE (Opaque) - Key Encryption Key for gateway secret encryption, manual rotation (mlflow crypto rotate-kek)
  - AI Gateway provider API keys (Opaque, encrypted at rest) - LLM provider credentials, provisioned by user via Gateway API, no auto-rotation

Security Middleware:
  - HostValidationMiddleware (FastAPI/ASGI) - DNS rebinding protection via Host header validation
  - CORSBlockingMiddleware (FastAPI/ASGI) - Blocks cross-origin state-changing requests
  - SecurityHeadersMiddleware (FastAPI/ASGI) - X-Content-Type-Options: nosniff, X-Frame-Options
  - CORSMiddleware (Starlette) - CORS preflight (OPTIONS) handling
  - Flask security middleware - Legacy Host and CORS validation

CRD:
  - mlflow.kubeflow.org/v1 MLflowConfig (Namespaced)
    Purpose: Per-namespace artifact storage configuration (artifactRootPath, artifactRootSecret)
    Note: Optional, consumed (watched) by MLflow server, not defined/installed by it
