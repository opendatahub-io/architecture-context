    - Resources: namespaces

  - API Group: "" (core)

    - Verbs: get
  - API Group: mlflow.kubeflow.org
    - Resources: mlflowconfigs
    - Verbs: list, watch

ClusterRoleBinding:
- mlflow-k8s-workspace-provider → ClusterRole: mlflow-k8s-workspace-provider
  - ServiceAccount: mlflow server SA (cluster-scoped)

- experiments:              get, list, create, update, delete

- registeredmodels:         get, list, create, update, delete
- assistants:               get, create, update
- gatewaysecrets:           get, list, create, update, delete
- gatewaysecrets/use:       create
- gatewayendpoints:         get, list, create, update, delete
- gatewayendpoints/use:     create
- gatewaymodeldefinitions:  get, list, create, update, delete
- gatewaymodeldefinitions/use: create

================================================================================
Authentication & Authorization
================================================================================

- External auth: kube-rbac-proxy validates Bearer Token (SA/OIDC)
- Internal auth: kubernetes-auth plugin enforces per-request RBAC
  - Uses SubjectAccessReview / SelfSubjectAccessReview
  - Checks against mlflow.kubeflow.org virtual resources
  - Resource mapped per endpoint (experiments, registeredmodels, etc.)
  - Workspace (namespace) determined from request path/body

Unprotected paths (no auth required):
  /health, /version, /metrics, /static-files/*, /

Network Summary
================================================================================

Ingress:
- 8443/TCP HTTPS (via kube-rbac-proxy) - TLS 1.2+, Bearer Token auth
- 443/TCP HTTPS (via operator-managed Route/HTTPRoute) - TLS 1.2+

Egress:
- 443/TCP HTTPS → Kubernetes API Server (SA Token)
- 5432/TCP TCP  → PostgreSQL (Username/Password, TLS configurable)
- 443/TCP HTTPS → S3 Storage (AWS credentials)
- 443/TCP HTTPS → LLM Provider APIs (API Keys)

Internal:
- 5000/TCP HTTP → MLflow server (plaintext, pod-local only)
