Secrets:
- mysql-secret (Opaque) - Database credentials (username/password), provisioned by DSPO/Admin, no auto-rotation
- mlpipeline-minio-artifact (Opaque) - S3 credentials (accesskey/secretkey), provisioned by DSPO/Admin, no auto-rotation
- webhook-server-tls (kubernetes.io/tls) - Webhook TLS certs, provisioned by DSPO/cert-manager, rotation depends on setup

Authentication Mechanisms:
- gRPC API (8887): Bearer Token → TokenReview against Kubernetes API; projected SA tokens with audience 'pipelines.kubeflow.org'
- REST API (8888): Forwarded via gRPC-Gateway from HTTP headers; header-based identity in multi-user mode
- Webhooks (8443): Kubernetes API server mTLS (automatic)
- Database: Username/Password credentials from Kubernetes Secret
- Object Storage: Access Key/Secret Key from Kubernetes Secret

FIPS Compliance:
- All Go binaries built with GOEXPERIMENT=strictfipsruntime
- Base images: UBI9 (registry.access.redhat.com/ubi9/ubi-minimal)
- Builder images: UBI9 Go Toolset 1.24
