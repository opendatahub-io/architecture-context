TLS Configuration Summary:
- Server TLS Modes:
  - Plaintext: No TLS env vars set
  - TLS: TLS_CERT_PATH + TLS_KEY_PATH
  - mTLS: TLS + TLS_CLIENT_CA_CERT_PATH (client cert verification)
- Client TLS Modes:
  - Plaintext: No tls in config
  - TLS: cert_path in service tls config
  - mTLS: cert_path + key_path + client_ca_cert_path
  - Insecure: insecure: true (skip server cert verification)
- TLS Provider: rustls with ring crypto provider
- Named TLS configs: Defined once, referenced by multiple services

Secrets:
- TLS server certificate (File PEM) - Server TLS encryption, cert-manager or manual
- TLS server key (File PEM) - Server TLS encryption
- TLS client CA cert (File PEM) - mTLS client verification
- Client TLS certificates (File PEM) - mTLS to backend services
- API Token (Environment variable) - OpenAI-compatible backend auth, no auto-rotation

Authentication & Authorization:
- /health, /info: Open access (no auth)
- /api/v1/*, /api/v2/*: Header passthrough to backends
  - X-Forwarded-Access-Token rewritten to Authorization: Bearer
  - Enforcement at upstream (deployment-dependent)
- Backend services: mTLS client certs / API Token (per-service config)
