RBAC Summary:
- ClusterRole: kuberay-operator
  - Core API: pods, services, configmaps, events, serviceaccounts, secrets
    Verbs: get, list, watch, create, update, patch, delete
  - ray.io: rayclusters, rayjobs, rayservices (+status, +finalizers)
    Verbs: get, list, watch, create, update, patch, delete
  - gateway.networking.k8s.io: gateways (get,list,watch), httproutes/referencegrants (full CRUD)
  - networking.k8s.io: networkpolicies, ingresses (full CRUD)
  - cert-manager.io: certificates, issuers (full CRUD)
  - authentication.k8s.io: tokenreviews (create)
  - authorization.k8s.io: subjectaccessreviews (create)
  - rbac.authorization.k8s.io: roles (CRUD), rolebindings (CRUD), clusterroles/clusterrolebindings (read)

- Leader Election Role (namespace-scoped):
  - configmaps, events: full CRUD
  - coordination.k8s.io/leases: create, get, list, update

SecurityContextConstraints:
- SCC: run-as-ray-user
  - UID: 1000 (MustRunAs)
  - Capabilities: DROP ALL, no privilege escalation
  - Purpose: Constrains Ray cluster pods to non-root user

Network Policies (per RayCluster):
- {cluster}-head:
  - Intra-cluster pods (ray.io/cluster={cluster}): all ports
  - Same-namespace pods: 10001/TCP, 8265/TCP
  - Operator namespace: 8265/TCP, 10001/TCP
  - Monitoring namespace (optional): 8080/TCP
  - Gateway namespace (optional): 8265/TCP
  - Anywhere: 8443/TCP (kube-rbac-proxy)
  - RayJob submitter pods (conditional): all ports

- {cluster}-workers:
  - Only pods with ray.io/cluster={cluster} label (intra-cluster)

Secrets:
- kuberay-webhook-server-cert (kubernetes.io/tls) - Webhook TLS cert, OpenShift serving-cert-signer, auto-rotate
- {cluster}-ca-secret-{uid} (kubernetes.io/tls) - Per-cluster CA cert, cert-manager, renew 30d before 10y expiry
- ray-head-secret-{cluster} (kubernetes.io/tls) - Head node TLS cert, cert-manager, renew 15d before 90d expiry
- ray-worker-secret-{cluster} (kubernetes.io/tls) - Worker node TLS cert, cert-manager, renew 15d before 90d expiry

Authentication Flow (kube-rbac-proxy):
1. Client sends request to Gateway (443/TCP, HTTPS)
2. Gateway forwards to kube-rbac-proxy sidecar (8443/TCP, HTTPS)
3. kube-rbac-proxy performs TokenReview against K8s API (validates bearer token)
4. kube-rbac-proxy performs SubjectAccessReview (checks RBAC permissions)
5. If authorized, proxies to Ray Dashboard (127.0.0.1:8265, HTTP loopback)
