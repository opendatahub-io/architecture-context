Service Accounts:
  - <instance>-<namespace>-job (tenant namespace) - Used by evaluation K8s Jobs

Authentication & Authorization:
  - /api/v1/evaluations/*:
    - AuthN: Bearer Token -> K8s TokenReview (DelegatingAuthenticator, 2min cache TTL)
    - AuthZ: SubjectAccessReview (DelegatingAuthorizer, 5min allow / 30s deny cache)
    - Tenant isolation: namespace from X-Tenant header
  - /api/v1/health, /metrics, /docs, /openapi.yaml:
    - AuthN: None (public endpoints)

Secrets:
  - TLS cert/key pair (kubernetes.io/tls) - HTTPS server TLS, provisioned by Operator/cert-manager, auto-rotate
  - MLflow SA token (Projected ServiceAccountToken) - MLflow auth, K8s kubelet provisioned, 1h expiry auto-rotate
  - OCI credentials (kubernetes.io/dockerconfigjson) - Registry pull for OCI export, user-provisioned, no auto-rotate
  - Model auth secret (Opaque) - Bearer token for model inference endpoint, user-provisioned, no auto-rotate
  - S3 test data secret (Opaque) - AWS credentials for test data download, user-provisioned, no auto-rotate
  - <instance>-service-ca (ConfigMap) - OpenShift service-serving CA bundle, auto-rotated

Container Security:
  - Run as non-root: Yes (user: evalhub, UID 1000)
  - FIPS compliance: GOEXPERIMENT=strictfipsruntime
  - Privilege escalation: Disabled (AllowPrivilegeEscalation=false)
  - Capabilities: Drop ALL
  - Seccomp profile: RuntimeDefault
