- RBAC is managed externally by the TrustyAI Service Operator
- Service itself does not define Kubernetes RBAC resources
- External access is controlled via kube-rbac-proxy (operator-deployed)

- TLS certificates (tls.crt/tls.key) - kubernetes.io/tls

- Database credentials (QUARKUS_DATASOURCE_USERNAME/PASSWORD) - Opaque

- MinIO credentials (access-key/secret-key) - Opaque
  - Purpose: MinIO/S3 authentication
  - Provisioned by: TrustyAI Operator / User

ConfigMaps:

    PVC filenames, metrics schedule, batch size)

Authentication & Authorization:
- All REST endpoints: No direct auth (cluster-internal)
- External access: Delegated to operator-managed ingress (kube-rbac-proxy)
- HTTPS (port 4443): TLS client verification (optional)
- TLS certificates provisioned by TrustyAI Operator

RHOAI-Specific Restrictions:
- Explainer endpoints (LIME, SHAP, Counterfactual, TSSaliency, PDP) are DISABLED
- Only fairness metrics (SPD, DIR) and drift metrics (KS, Meanshift, FourierMMD) are enabled
- Data download endpoint is disabled by default
