==============================================================================
RBAC Summary
==============================================================================

No RBAC resources defined in this repository.
RBAC is managed by the deploying component (FMS Guardrails Orchestrator, KServe).

==============================================================================
Authentication & Authorization
==============================================================================

All detector endpoints: No authentication, no authorization.
Detectors are designed as internal-only services.
Auth is enforced at the orchestrator/ingress level, not at detector level.

==============================================================================
Security Controls
==============================================================================

1. Custom Detector Static Code Analysis:
   - AST-based analysis of user-provided Python detector code
   - Forbidden imports: os, subprocess, sys, shutil
   - Forbidden calls: eval, exec, open, compile, input
   - Allowed exceptions: os.environ, os.getenv

2. SSL/TLS Support (Optional):
   - DetectorBaseAPI main() supports ssl_ca_certs configuration
   - ssl.CERT_REQUIRED via config.yaml
   - Enables mTLS when configured
