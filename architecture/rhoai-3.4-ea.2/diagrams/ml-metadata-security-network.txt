================================================================================
RBAC Summary:
================================================================================

No RBAC resources are defined by MLMD. It is a stateless gRPC server binary
that runs as non-root user (UID 65534). RBAC for the pod is managed by the
deploying component (Data Science Pipelines operator).

================================================================================
Security Configuration:
================================================================================

gRPC Server TLS (SSLConfig):
- server_cert: Path to server TLS certificate
- server_key: Path to server TLS private key
- custom_ca: Path to custom CA certificate
- client_verify: When true, requires valid client certificate (mTLS)

MySQL SSL Options:
- sslcert: Client SSL certificate
- sslkey: Client SSL private key
- sslrootcert: CA certificate for server verification
- sslcapath: CA certificate directory
- sslcipher: Allowed cipher list
- verify_server_cert: When true, verifies MySQL server certificate

PostgreSQL SSL Options:
- sslmode: disable | allow | verify-ca | verify-full
- sslcert: Client SSL certificate
- sslkey: Client SSL private key
- sslpassword: SSL key password
- sslrootcert: CA certificate for server verification

================================================================================
Secrets:
================================================================================

- (database credentials) (Opaque) - MySQL/PostgreSQL connection credentials,
  provisioned by DSP Operator / Admin, no auto-rotation
- (TLS certificates) (kubernetes.io/tls) - Optional server TLS cert/key for
  gRPC SSL, optional client cert for mTLS, provisioned by Admin/cert-manager,
  no auto-rotation
- (DB SSL certificates) (Opaque) - Optional MySQL/PostgreSQL SSL certificates,
  provisioned by Admin, no auto-rotation

Note: Secret names are deployment-specific and configured by the Data Science
Pipelines operator that deploys this server.
