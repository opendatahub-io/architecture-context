## Purpose
**Short**: A Go application that extracts, enriches, and catalogs metadata from Red Hat AI container model images, producing structured YAML catalogs shipped as a static data container.

**Detailed**: Model Metadata Collection discovers AI models from HuggingFace collections and OCI container registries (primarily registry.redhat.io), extracts model card metadata from container image layers, enriches that metadata with data from HuggingFace APIs, and generates structured YAML catalogs. The tool supports multiple model index files (validated models, other models, Red Hat models) and merges static supplemental catalogs with dynamically extracted metadata.

The output is a set of YAML catalog files (`models-catalog.yaml`, `validated-models-catalog.yaml`, `other-models-catalog.yaml`) containing standardized model metadata including name, provider, description, license, language, tasks, tags, artifacts with OCI URIs, and custom properties like model type classification and architecture information. These catalogs are pre-generated during CI/CD and shipped as static data inside a minimal UBI9-based container image that runs `sleep infinity` to serve as a data volume for other platform components (such as the RHOAI dashboard) to mount and consume.

The tool also includes a metadata report generator that analyzes field completeness and data source tracking across all cataloged models.

## Build & Deployment

### Container Build

The component is built by Konflux using `Dockerfile.konflux`, which produces a minimal UBI9-based container image containing only pre-generated catalog YAML files. The build pipeline is defined in `.tekton/odh-model-metadata-collection-v3-4-ea-2-push.yaml` and builds for four architectures: `linux/x86_64`, `linux/arm64`, `linux/ppc64le`, `linux/s390x`.

**Output image**: `quay.io/rhoai/odh-model-metadata-collection-rhel9:rhoai-3.4-ea.2`

### Runtime Behavior

The container runs `sleep infinity` as its entrypoint. It does not execute any application logic at runtime. Its sole purpose is to provide the `/app/data` volume containing:
- `models-catalog.yaml` - Red Hat models catalog
- `models-index.yaml` - Red Hat models index
- `validated-models-catalog.yaml` - Validated models catalog (largest, ~1.5MB)
- `validated-models-index.yaml` - Validated models index
- `other-models-catalog.yaml` - Other models catalog
- `other-models-index.yaml` - Other models index

### Model Catalog Contents

The validated models index contains ~88 OCI model entries from `registry.redhat.io/rhelai1/` covering model families including Llama 4, Mistral, Granite, Qwen, Phi, Gemma, and DeepSeek. Models are labeled with categories such as `validated`, `featured`, `lab-teacher`, and `lab-base`.
