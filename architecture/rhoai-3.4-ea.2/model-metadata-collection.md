## Purpose
**Short**: A build-time tool and data container that collects, enriches, and packages AI model metadata from OCI container registries and HuggingFace into structured YAML catalogs for consumption by the RHOAI Model Registry and dashboard.

**Detailed**: The model-metadata-collection component serves two distinct roles. First, it is a Go-based CLI tool (`model-extractor`) that extracts metadata from Red Hat AI model containers (modelcar OCI artifacts) by inspecting container image layers for model cards, parsing YAML frontmatter and markdown content, and enriching the extracted metadata with additional information from the HuggingFace API (model details, README content, tags, license information, and language data). The tool supports concurrent processing of multiple models and generates structured YAML catalog files.

Second, the component produces a minimal data container image (based on `ubi9-minimal`) that ships pre-generated catalog YAML files (`models-catalog.yaml`, `validated-models-catalog.yaml`, `other-models-catalog.yaml`, and their corresponding index files) at `/app/data/`. This container is designed to be used as an init container or volume mount source, providing model metadata catalogs to other RHOAI components such as the Model Registry and the RHOAI Dashboard. The container itself runs no services -- it simply exposes the static data files via a volume mount point.

A secondary CLI tool (`metadata-report`) generates completeness reports analyzing how well model metadata fields are populated across the catalog, tracking data sources (modelcard YAML, regex extraction, HuggingFace API, registry metadata) and providing per-model and aggregate statistics.

## Build System

### Container Build

The component uses **Konflux** (Tekton-based) for CI/CD:
- **Primary Dockerfile**: `Dockerfile.konflux` - builds a minimal `ubi9-minimal` data container
- **Container image**: `quay.io/opendatahub/odh-model-metadata-collection`
- **Build strategy**: Copy pre-generated catalog YAML files into the container (no Go binary in the final image)
- **Tekton pipelines**: `.tekton/` directory contains pull-request and push pipelines
