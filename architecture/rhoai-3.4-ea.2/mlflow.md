## Purpose
**Short**: MLflow is an open-source platform for managing the end-to-end machine learning lifecycle, customized for Red Hat OpenShift AI with Kubernetes-native workspace isolation, RBAC authorization, and federated UI integration.

**Detailed**: This repository contains the Red Hat-adapted version of MLflow, providing experiment tracking, model versioning and registry, LLM observability and tracing, prompt management, and an AI Gateway for proxying requests to LLM providers. The server exposes both Flask and FastAPI endpoints, with a React-based UI that can run standalone or be embedded into the OpenShift AI dashboard via Webpack Module Federation.

The RHOAI customization adds a Kubernetes workspace provider that maps MLflow workspaces 1:1 to Kubernetes namespaces, enabling multi-tenant isolation. An authorization plugin (`kubernetes-auth`) enforces Kubernetes RBAC via `SelfSubjectAccessReview` or `SubjectAccessReview` for every MLflow API call, with resource-level access control for experiments, datasets, registered models, AI gateway endpoints, and more. Per-namespace artifact storage can be overridden via an optional `MLflowConfig` CRD (`mlflowconfigs.mlflow.kubeflow.org`).

The container image is built via Konflux pipelines using `Dockerfile.konflux`, based on UBI 9 with Python 3.12. It ships the MLflow server, the Kubernetes workspace provider plugin, the Kubernetes authorization plugin, the React UI, and Prometheus metrics exporter. The image is designed to run behind a reverse proxy (such as kube-rbac-proxy) for authentication.
