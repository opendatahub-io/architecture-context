## Purpose
**Short**: MLflow is an open-source ML experiment tracking, model registry, and AI gateway platform adapted for Red Hat OpenShift AI with Kubernetes-native workspace and authorization integration.

**Detailed**: MLflow provides a comprehensive platform for the complete machine learning lifecycle. It tracks experiments (parameters, metrics, artifacts), manages model versions through a registry, provides an AI Gateway for proxying LLM requests, and offers a React-based web UI for visualization and management. The server exposes both REST and GraphQL APIs, with an OTLP-compatible endpoint for OpenTelemetry trace ingestion.

The RHOAI distribution extends upstream MLflow with a Kubernetes workspace provider that maps MLflow workspaces 1:1 to Kubernetes namespaces, and a Kubernetes authorization plugin (`kubernetes-auth`) that enforces RBAC via `SelfSubjectAccessReview` or `SubjectAccessReview` against the `mlflow.kubeflow.org` API group. This enables multi-tenant operation where each namespace represents an isolated workspace, with fine-grained access control for experiments, registered models, datasets, and gateway resources. The authorization plugin supports both direct bearer token authentication and delegated authentication via `kube-rbac-proxy` headers (`x-remote-user`, `x-remote-groups`).

The server is built with Flask (legacy routes) wrapped by FastAPI (new routes for OTLP, Gateway, Jobs, Assistant APIs) and served via Gunicorn/Uvicorn. It supports PostgreSQL as a backend store, S3-compatible object storage for artifacts, and per-namespace artifact root overrides via the optional `MLflowConfig` CRD (`mlflowconfigs.mlflow.kubeflow.org`).

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| Platform Gateway (HTTPRoute) | HTTPRoute (Gateway API) | Configured by platform operator | 443/TCP | HTTPS | TLS 1.2+ | SIMPLE | External |

Note: MLflow itself does not create HTTPRoute or Route resources. Ingress is managed by the deploying operator (e.g., rhods-operator) which creates HTTPRoute CRs referencing the platform `data-science-gateway`. When deployed behind `kube-rbac-proxy`, the proxy handles TLS termination and authentication on port 8443/TCP, forwarding authenticated requests to MLflow on port 5000/TCP with `x-remote-user` and `x-remote-groups` headers.

## Container Image

Built via `Dockerfile.konflux` using multi-stage build:

| Stage | Base Image | Purpose |
|-------|-----------|---------|
| ui-builder | registry.access.redhat.com/ubi9/nodejs-20 | Builds React UI with yarn |
| python-builder | registry.access.redhat.com/ubi9/python-312 | Builds Python wheels (mlflow, mlflow-tracing, mlflow-skinny, kubernetes-workspace-provider) |
| runtime | registry.access.redhat.com/ubi9/ubi | Final image with Python 3.12, PostgreSQL client libs, OpenBLAS |

**Runtime Configuration**:
- `MLFLOW_DISABLE_TELEMETRY=true` - Telemetry disabled
- `MLFLOW_SERVER_ENABLE_JOB_EXECUTION=false` - Job execution disabled by default
- `MLFLOW_ENABLE_ASSISTANT=false` - AI assistant disabled
- `MLFLOW_ENABLE_AI_GATEWAY=false` - AI Gateway disabled by default
- Default CMD: `python3.12 -m mlflow server --host 0.0.0.0 --port 5000 --app-name kubernetes-auth --enable-workspaces --workspace-store-uri kubernetes:// --serve-artifacts`
