## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates Ragas (Retrieval Augmented Generation Assessment) metrics into the Llama Stack eval API.

**Detailed**: This component implements [Ragas](https://github.com/explodinggradients/ragas) as an external provider for [Llama Stack](https://github.com/meta-llama/llama-stack), enabling evaluation of LLM-based RAG systems using standardized metrics such as answer relevancy, context precision, faithfulness, and context recall. It exposes these evaluation capabilities through Llama Stack's eval API, leveraging Llama Stack's built-in APIs for inference (LLMs and embeddings), datasets, and benchmarks.

The provider ships in two modes: an **inline** provider that runs Ragas evaluation in the same process as the Llama Stack server, and a **remote** provider that offloads evaluation to Kubeflow Pipelines for distributed, scalable execution. The remote mode submits KFP pipeline runs that retrieve data from a Llama Stack server, execute Ragas evaluations inside pipeline containers, and persist results to S3-compatible storage. The component is built by Konflux and shipped as `odh-trustyai-ragas-lls-provider-dsp-rhel9` on UBI9 with Python 3.12.

## Evaluation Metrics

The provider supports the following Ragas metrics (defined in constants.py):

| Metric | Purpose |
|--------|---------|
| answer_relevancy | Measures how relevant the generated answer is to the question |
| context_precision | Measures precision of retrieved context relative to ground truth |
| faithfulness | Measures factual consistency of the answer with retrieved context |
| context_recall | Measures recall of retrieved context relative to ground truth |
