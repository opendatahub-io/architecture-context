## Purpose
**Short**: A lightweight REST API service for orchestrating LLM evaluations across multiple backends on OpenShift.

**Detailed**: EvalHub is a Go-based REST API service that orchestrates Large Language Model (LLM) evaluation workflows across multiple evaluation frameworks. It provides a unified API for scheduling, tracking, and managing evaluation jobs that run benchmarks such as lm-evaluation-harness, LightEval, and GuideLLM against LLM inference endpoints. The service creates Kubernetes batch Jobs to execute evaluations and tracks results via MLflow experiment tracking.

The service supports a pluggable architecture where evaluation providers are declared in YAML configuration files. Each provider defines its container image, resource requirements, and available benchmarks. When an evaluation job is submitted via the REST API, EvalHub constructs Kubernetes Job and ConfigMap resources in the target namespace, mounts secrets for model authentication and S3 test data, and reports benchmark results back through status event callbacks. Collections provide a way to group benchmarks into predefined suites (e.g., leaderboard-v2) with pass/fail criteria.

EvalHub is managed by the TrustyAI Service Operator via a custom resource (`trustyai.opendatahub.io/v1alpha1 EvalHub`). The operator provisions the deployment, service accounts, and service CA ConfigMaps. Authentication and authorization are handled via Kubernetes TokenReview and SubjectAccessReview APIs, with multi-tenant isolation enforced through namespace-scoped RBAC checks using the `X-Tenant` header. The service is built with FIPS-compliant Go (`GOEXPERIMENT=strictfipsruntime`) and runs on UBI9-minimal.
