## Purpose
**Short**: A lightweight REST API service for orchestrating LLM evaluations across multiple backends on OpenShift.

**Detailed**: EvalHub is a Go-based REST API service that routes evaluation requests to various LLM evaluation frameworks including lm-evaluation-harness, RAGAS, Garak, GuideLLM, LightEval, and MTEB. It provides a unified API for creating, tracking, and managing evaluation jobs while supporting pluggable evaluation providers through the Bring Your Own Framework (BYOF) pattern via a complementary Python SDK (eval-hub-sdk).

The service orchestrates evaluation workloads on Kubernetes by creating batch Jobs for each benchmark evaluation. It integrates with MLflow for experiment tracking and result aggregation, uses a pluggable storage layer (SQLite for development, PostgreSQL for production), and exposes Prometheus metrics for observability. In production, EvalHub is deployed and managed by the TrustyAI Service Operator via the `EvalHub` custom resource. Authentication and authorization are enforced using Kubernetes TokenReview and SubjectAccessReview (SAR) delegation when running in-cluster.

The service is built as a FIPS-compliant container image using `GOEXPERIMENT=strictfipsruntime` on UBI9, and supports disconnected/air-gapped environments. It uses OpenTelemetry for distributed tracing and structured JSON logging with zap.

### Ingress

EvalHub does not define its own ingress resources. When deployed via the TrustyAI Service Operator on RHOAI 3.x, external access is managed by the platform operator's ingress infrastructure (Gateway API / HTTPRoute pattern). The operator is responsible for creating any necessary HTTPRoute or Route resources to expose the service.
