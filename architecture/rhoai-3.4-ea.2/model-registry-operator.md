## Purpose
**Short**: Manages the lifecycle of Model Registry and Model Catalog custom resources and their associated Kubernetes resources on OpenShift/Kubernetes clusters.

**Detailed**: The Model Registry Operator is a Kubernetes operator built with controller-runtime that manages `ModelRegistry` custom resources (CRDs). For each `ModelRegistry` CR, the operator reconciles a full deployment stack including a REST API service (model-registry), an optional PostgreSQL or MySQL database, kube-rbac-proxy sidecars for authentication, OpenShift Routes for external access, RBAC resources (Roles, RoleBindings, ClusterRoleBindings), NetworkPolicies, and ServiceAccounts.

The operator also includes a `ModelCatalogReconciler` that manages a singleton model catalog service. The catalog provides a centralized metadata collection service with its own PostgreSQL database, kube-rbac-proxy authentication, RBAC for admin groups, and configurable data sources. The catalog is deployed as a single shared instance in the registries namespace rather than per-CR.

The operator supports multiple cluster environments: traditional OpenShift (with user.openshift.io Groups), BYOIDC-enabled OpenShift 4.20+ (where user.openshift.io is absent), and plain Kubernetes. It detects cluster capabilities at startup and conditionally creates platform-specific resources. The operator has migrated from oauth-proxy to kube-rbac-proxy for authentication and automatically migrates existing CR specs from the old configuration.
