## Purpose
**Short**: Manages the lifecycle of Model Registry and Model Catalog instances on OpenShift/Kubernetes, deploying and configuring the model registry REST service with optional authentication proxy and database backends.

**Detailed**: The Model Registry Operator is a Kubebuilder-based Go operator that reconciles `ModelRegistry` custom resources to deploy and manage model registry service instances. Each `ModelRegistry` CR triggers the creation of a complete service stack including a REST API deployment (model-registry proxy), Kubernetes Service, ServiceAccount, RBAC roles, and optional kube-rbac-proxy sidecar for authentication. The operator supports PostgreSQL and MySQL database backends, with the ability to auto-provision a PostgreSQL deployment including PVC, Service, and NetworkPolicy.

In addition to per-registry management, the operator includes a `ModelCatalogReconciler` that manages a singleton model catalog service. The catalog provides a curated registry of model metadata sourced from configurable catalog sources (both operator-managed defaults and user-managed sources). The catalog deployment includes init containers for loading model metadata collections and benchmark data, with its own PostgreSQL backend for persistence.

The operator performs runtime cluster capability detection to support multiple platforms: traditional OpenShift, BYOIDC-enabled OpenShift 4.20+ (where `user.openshift.io` API is absent), and plain Kubernetes. On OpenShift, it conditionally creates Routes for external access, OpenShift Groups for user management, and leverages OpenShift Serving Certificates for TLS. The operator is actively migrating from OAuth proxy (legacy RHOAI 2.x pattern) to kube-rbac-proxy (RHOAI 3.x pattern) for authentication, with automatic migration logic that converts existing OAuthProxy configurations to KubeRBACProxy.

**Note on ingress architecture**: This operator creates OpenShift Routes directly (not Gateway API HTTPRoutes). The kube-rbac-proxy sidecar handles TLS termination and RBAC enforcement. Routes use `reencrypt` TLS termination, meaning the OpenShift router terminates the external TLS connection and re-encrypts traffic to the kube-rbac-proxy backend on port 8443. The operator migrates existing OAuthProxy configurations to KubeRBACProxy automatically during reconciliation.
