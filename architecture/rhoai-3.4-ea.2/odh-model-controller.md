## Purpose
**Short**: A Kubernetes controller that manages OpenShift-specific integrations and lifecycle resources for KServe InferenceServices, LLMInferenceServices, and NVIDIA NIM accounts on the RHOAI platform.

**Detailed**: The odh-model-controller is a controller-runtime based Go operator that bridges KServe model serving with the OpenShift AI (RHOAI) platform. It watches KServe InferenceService, ServingRuntime, InferenceGraph, and LLMInferenceService custom resources and dynamically creates platform-specific resources such as OpenShift Routes (for external access in RawDeployment mode), monitoring resources (ServiceMonitors, PodMonitors, metrics Services), RBAC resources (ClusterRoleBindings for Prometheus access), and network policies. For the newer RHOAI 3.x LLMInferenceService pattern, it manages Kuadrant AuthPolicy CRs (targeting both Gateway and HTTPRoute) and Istio EnvoyFilter CRs for authentication/authorization via Authorino.

The controller also manages data connection secrets (aggregating S3-compatible storage configuration into a unified `storage-config` secret), custom CA certificate bundles for KServe (merging ODH global certs and service CA certs), multi-node Ray TLS certificates for distributed inference, NVIDIA NIM account integration (API key validation, model catalog ConfigMaps, pull secrets, and OpenShift Templates), KEDA-based autoscaling resources for Prometheus metrics, and optional Model Registry integration for tracking deployed InferenceServices. It includes mutating and validating webhooks for Pods, InferenceServices, InferenceGraphs, and NIM Accounts.

**Controller-managed ingress for LLMInferenceService (RHOAI 3.x pattern)**:
- The LLMInferenceService controller creates Kuadrant **AuthPolicy** CRs targeting both Gateway and HTTPRoute resources
- Gateway-level AuthPolicy: enforces authentication (UserDefined) on the platform gateway (default: `openshift-ai-inference` in `openshift-ingress` namespace)
- HTTPRoute-level AuthPolicy: creates anonymous access policy when auth annotation is not set to UserDefined
- Creates Istio **EnvoyFilter** CRs for Authorino TLS bootstrap on managed Gateways
- EnvoyFilters are owned by the Gateway and persist across LLMInferenceService lifecycle
- This controller does NOT create HTTPRoutes or Gateways directly; it configures auth/security on existing Gateway API infrastructure

**Controller-managed ingress for InferenceService (legacy pattern)**:
- Creates OpenShift **Route** CRs per InferenceService when label `networking.kserve.io/visibility=exposed` is set
- Routes target the predictor or transformer Service with Edge TLS (or Reencrypt when auth enabled)
- Only applies to RawDeployment mode InferenceServices
