## Purpose
**Short**: A Kubernetes controller that automates model serving infrastructure for KServe InferenceServices on OpenShift AI, managing routes, monitoring, authentication policies, certificate trust chains, and NVIDIA NIM integration.

**Detailed**: The odh-model-controller is a controller-runtime based Go operator that extends KServe InferenceService capabilities on the Red Hat OpenShift AI (RHOAI) platform. It watches InferenceService, ServingRuntime, LLMInferenceService, and NIM Account custom resources, automatically creating and managing the ancillary Kubernetes resources needed for production model serving. These include OpenShift Routes for external access, Prometheus ServiceMonitors/PodMonitors for observability, RBAC ClusterRoleBindings for monitoring access, CA certificate trust bundles, storage-config Secrets from data connections, and Ray TLS certificates for multi-node serving.

For the new LLMInferenceService API (RHOAI 3.x), the controller manages Kuadrant AuthPolicy resources for authentication enforcement on Gateway API HTTPRoutes and Istio EnvoyFilter resources for TLS bootstrap between the ingress gateway and Authorino. It also provides mutating webhooks to inject Ray TLS init-containers into multi-node inference pods and validating webhooks to enforce InferenceService naming and namespace restrictions.

A separate Gateway Discovery HTTP server provides an authenticated REST API (`/api/v1/gateways`) that allows the RHOAI dashboard to discover Gateway API Gateways available to users, performing RBAC checks against the caller's Bearer token to filter results by namespace authorization. This is a TLS-only server deployed as a separate container from the same repository.

**Note on Ingress Patterns**: The InferenceService controller creates OpenShift Routes for RawDeployment-mode ISVCs when the label `networking.kserve.io/visibility=exposed` is present. Routes use Edge TLS termination by default, or Reencrypt when `security.opendatahub.io/enable-auth=true`. The LLMInferenceService controller operates via Gateway API — it manages AuthPolicies and EnvoyFilters that reference existing HTTPRoutes/Gateways created by KServe, rather than creating Routes directly.

### Serving Runtime Templates
The controller deploys OpenShift Template resources for various inference runtime configurations:
- vLLM (CUDA, ROCm, Gaudi, CPU, CPU-x86, Spyre variants for x86/s390x/ppc64le)
- OpenVINO Model Server (OVMS)
- MLServer
- vLLM multi-node (with Ray worker support)
- Guardrails HuggingFace detector

### Environment Variables
- `NIM_STATE`: Controls NIM Account controller (managed/removed)
- `KSERVE_STATE`: Controls KServe integration (managed)
- `MODELREGISTRY_STATE`: Controls Model Registry integration (removed/managed)
- `POD_NAMESPACE`: Controller namespace (injected from downward API)
- `MONITORING_NAMESPACE`: Prometheus monitoring namespace
- `MR_SKIP_TLS_VERIFY`: Skip TLS for Model Registry (default: false)
