## Purpose
**Short**: NeMo Guardrails is a programmable guardrails toolkit for controlling and securing LLM-based conversational systems.

**Detailed**: NeMo Guardrails, originally developed by NVIDIA, provides an open-source framework for adding programmable guardrails to LLM-based conversational applications. Guardrails ("rails") are specific controls over LLM output, including topic restrictions, response formatting, dialog flow enforcement, language style control, structured data extraction, and protection against LLM vulnerabilities such as jailbreaks and prompt injections.

The RHOAI distribution packages NeMo Guardrails as a containerized server built on Red Hat UBI9 with Python 3.12. The server exposes an OpenAI-compatible chat completions API with guardrails extensions, allowing it to act as a guardrails proxy in front of LLM inference endpoints. The build includes a comprehensive library of open-source guardrail modules (content safety, hallucination detection, jailbreak detection, sensitive data detection, topic safety, etc.) with closed-source guardrails filtered out during build. The container ships pre-downloaded NLP models (spaCy en_core_web_lg, fastembed, sentence-transformers) for offline/disconnected operation.

The server uses Colang (versions 1.0 and 2.x), a domain-specific language for defining guardrail flows and conversational patterns. It integrates with LangChain for LLM provider abstraction, supporting NVIDIA NIM endpoints, OpenAI, and other LLM backends.

### Request/Response Schema

The `/v1/chat/completions` endpoint extends the standard OpenAI ChatCompletion schema with a `guardrails` field:

**Request** (`GuardrailsChatCompletionRequest`):
- Standard OpenAI fields: `messages`, `model`, `stream`, `max_tokens`, `temperature`, `top_p`, `stop`
- Extended field: `guardrails.config_id` - guardrails configuration to apply
- Extended field: `guardrails.config_ids` - multiple configurations to combine
- Extended field: `guardrails.config` - inline guardrails configuration
- Extended field: `guardrails.thread_id` - thread ID for conversation continuity
- Extended field: `guardrails.context` - additional context data
- Extended field: `guardrails.options` - generation options (logging, rail selection)
- Extended field: `guardrails.state` - state for continued interaction

**Response** (`GuardrailsChatCompletion`):
- Standard OpenAI `ChatCompletion` fields
- Extended field: `guardrails.config_id`, `guardrails.state`, `guardrails.llm_output`, `guardrails.log`

### Streaming

Server-Sent Events (SSE) streaming is supported via `stream: true` parameter on `/v1/chat/completions`.
