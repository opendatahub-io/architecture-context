## Purpose
**Short**: NeMo Guardrails is an open-source toolkit for adding programmable guardrails to LLM-based conversational systems, providing input/output safety checks, jailbreak detection, content moderation, and topic safety enforcement.

**Detailed**: NeMo Guardrails, developed by NVIDIA, enables developers to add programmable safety guardrails between application code and LLM inference endpoints. It acts as a proxy that intercepts requests to LLMs, applies configurable input rails (checking user messages before sending to the LLM) and output rails (checking LLM responses before returning to the user). The system supports a wide range of safety mechanisms including content safety checks, topic safety enforcement, jailbreak detection, sensitive data detection (PII), hallucination detection, and injection prevention.

The Red Hat distribution (red-hat-data-services fork) packages NeMo Guardrails as a container image built on UBI9/Python 3.12, with pre-downloaded NLP models (spaCy, sentence-transformers, FastEmbed) baked into the image for disconnected/air-gapped deployments. The build process filters out closed-source guardrail integrations (ActiveFence, Cleanlab, PrivateAI, etc.) to ship only open-source guardrails. The server exposes an OpenAI-compatible chat completions API on port 8000, enabling it to be placed in front of any LLM endpoint as a transparent guardrails proxy. Configuration is provided via ConfigMaps mounted at `/app/config/`.
