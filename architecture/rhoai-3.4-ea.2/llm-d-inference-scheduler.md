## Purpose
**Short**: An inference request scheduler that makes optimized routing decisions for LLM inference workloads via a Kubernetes Gateway API Inference Extension (GIE) Endpoint Picker (EPP).

**Detailed**: The llm-d Inference Scheduler provides an Endpoint Picker (EPP) component for the llm-d inference framework. It schedules incoming inference requests arriving through a Kubernetes Gateway API-compatible gateway by evaluating pluggable filters and scorers. The EPP integrates with Envoy via the ext-proc (External Processing) filter callback mechanism to make routing decisions that direct inference requests to optimal model-serving pods.

The scheduler extends the upstream Gateway API Inference Extension (GIE) project with llm-d-specific features, most notably disaggregated Prefill/Decode (P/D) execution. In P/D mode, the prefill stage (prompt processing) and decode stage (token generation) are routed to separate specialized pods. A P/D routing sidecar proxy runs alongside the decode worker, orchestrating communication between prefill and decode pods using one of three supported connector protocols: NIXL v2, Shared Storage (deprecated), or SGLang.

The scheduler supports KV-cache-aware routing via the precise-prefix-cache-scorer, which tracks real-time KV-cache states across vLLM instances using ZMQ event subscriptions for near-real-time scoring. Additional plugins provide load-aware scoring, session affinity, active request tracking, and label-based filtering. The entire plugin pipeline is configurable via YAML (EndpointPickerConfig), supporting multiple scheduling profiles for different request types.

| Component | Type | Purpose |

### SSRF Protection

The P/D sidecar includes SSRF protection via an AllowlistValidator:
- Watches InferencePool resources to discover pod selectors
- Watches Pods matching InferencePool selectors to build an allowlist of valid IP addresses
- Validates prefill target host:port against the allowlist before proxying requests
- Enabled via --enable-ssrf-protection flag
