## Purpose
**Short**: A platform for serving LLM inference models as a managed service with API key management, tiered subscriptions, and token-based rate limiting via Gateway API and Kuadrant policies.

**Detailed**: Models as a Service (MaaS) provides a comprehensive platform for exposing LLM inference models through a centralized Gateway API gateway, with fine-grained access control and rate limiting. The platform consists of two primary components: a Go REST API (`maas-api`) that handles API key lifecycle management, tier lookups, model discovery, and subscription selection; and a Kubernetes controller (`maas-controller`) built on controller-runtime that reconciles three custom resources (MaaSModelRef, MaaSAuthPolicy, MaaSSubscription) into Kuadrant AuthPolicy and TokenRateLimitPolicy resources attached to Gateway API HTTPRoutes.

The system integrates deeply with the Kuadrant policy engine (Authorino for authentication/authorization, Limitador for rate limiting) and KServe's LLMInferenceService for model serving. When a user creates a MaaSModelRef, the controller discovers the corresponding HTTPRoute (created by KServe) and validates it references the MaaS gateway. MaaSAuthPolicy resources define which users/groups can access which models, and the controller aggregates these into per-model Kuadrant AuthPolicies that perform API key validation through callbacks to the maas-api service. MaaSSubscription resources define per-model token rate limits and billing metadata, which the controller translates into per-model Kuadrant TokenRateLimitPolicies with CEL-based group membership predicates.

The architecture uses a "defense in depth" approach: gateway-level default policies deny access to unconfigured models (AuthPolicy returns 401/403; TokenRateLimitPolicy returns 429), while per-route policies created by the controller override these defaults with model-specific rules. The API supports both OpenAI-compatible API keys (sk-oai-* format) and OpenShift token authentication for management endpoints.

**Gateway Architecture**: The MaaS gateway (`maas-default-gateway`) is deployed in the `openshift-ingress` namespace using the `openshift-default` gatewayClassName. It accepts routes from all namespaces (`allowedRoutes.namespaces.from: All`). The maas-api HTTPRoute handles management endpoints (/v1/models, /maas-api/*). Per-model HTTPRoutes are created by KServe for LLMInferenceService resources and validated by the MaaSModelRef controller. The controller verifies each model's HTTPRoute references the MaaS gateway and populates endpoint information in the MaaSModelRef status.

**Policy Stack**: Two default policies are applied at the gateway level:
1. `gateway-default-auth` (AuthPolicy) - denies access to unconfigured models (returns 401/403)
2. `gateway-default-deny` (TokenRateLimitPolicy) - secondary safety net denying tokens to unconfigured models (returns 429)

Per-route policies generated by the controller override these defaults using Kuadrant's atomic strategy.
