## Purpose
**Short**: Central build configuration repository that defines the operator bundle, FBC catalog fragment, and image composition for Red Hat OpenShift AI (RHOAI) 3.4.0-ea.2.

**Detailed**: RHOAI-Build-Config is the authoritative source of truth for how the Red Hat OpenShift AI product is assembled, versioned, and released via Konflux CI/CD. It does not contain application source code or controller logic; instead, it aggregates 92 container image references from across the RHOAI ecosystem into an OLM (Operator Lifecycle Manager) operator bundle and a file-based catalog (FBC) fragment.

The repository manages the full operator bundle lifecycle: the ClusterServiceVersion (CSV) with all CRD ownership declarations, RBAC rules for the rhods-operator, related image pinning for disconnected/air-gapped installs, and OCP version-specific FBC catalog images (v4.19, v4.20, v4.21). Tekton PipelineRun definitions in `.tekton/` automate hermetic builds via Konflux/AppStudio, including multi-architecture (amd64, arm64, ppc64le, s390x) FBC fragment builds. Enterprise Contract verification ensures supply chain integrity before release.

The build configuration also includes scheduling triggers, GitHub Actions workflows, and release artifacts organized by stage/prod promotion. A secondary PNC (Project Newcastle) build config exists for TrustyAI Java artifacts. The operator bundle produced here is the rhods-operator bundle that gets installed on OpenShift clusters, defining the DataScienceCluster and DSCInitialization CRDs as the primary user-facing APIs.
