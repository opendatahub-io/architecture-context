## Purpose
**Short**: Build configuration repository that defines the OLM operator bundle and file-based catalog (FBC) fragments for shipping Red Hat OpenShift AI (RHOAI) 3.4.0-ea.2 via Konflux.

**Detailed**: RHOAI-Build-Config is the central build orchestration repository for the Red Hat OpenShift AI product. It does not contain operator source code or controllers; instead, it houses the OLM (Operator Lifecycle Manager) ClusterServiceVersion (CSV), CRD manifests, FBC catalog definitions, and Konflux/Tekton build pipeline configurations needed to assemble and ship the `rhods-operator` bundle and catalog images.

The repository tracks 92 shipped container images via RELATED_IMAGE environment variables in the CSV, ensuring disconnected installation support. It defines the upgrade path from earlier RHOAI versions (1.20.x through 3.4.0-ea.2) in the FBC catalog entries. Build automation is managed through Tekton pipelines (Konflux) for push and scheduled builds, and GitHub Actions workflows for bundle/FBC processing, nightly builds, and stage push operations.

This repository is the "bill of materials" for RHOAI -- it pins exact image digests (SHA256) for every component shipped in the product, from the operator itself to workbench images, serving runtimes, pipeline runtimes, and training images.
