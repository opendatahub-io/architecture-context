## Purpose
**Short**: A central repository for ML model developers to store, manage, and retrieve model metadata including registered models, model versions, artifacts, inference services, and serving environments.

**Detailed**: Model Registry is a metadata management service for machine learning workflows. It provides a REST API (OpenAPI v1alpha3) that enables data scientists and ML engineers to register models, track model versions and their associated artifacts (model files, documentation), manage inference services, and organize serving environments. The service is built on an embedded metadata datastore backed by either MySQL or PostgreSQL databases.

The component consists of multiple sub-components: (1) a Go REST API server that serves the Model Registry OpenAPI, (2) a Kubernetes controller that watches KServe InferenceService CRs and synchronizes their state with the Model Registry, (3) a CSI storage initializer that enables KServe to fetch model artifacts using `model-registry://` URIs, (4) a Model Catalog server providing a curated catalog API (v1alpha1) for browsing models and MCP servers from external sources like HuggingFace, and (5) a UI consisting of a Go BFF (Backend-for-Frontend) and a React/TypeScript frontend.

The RHOAI build uses `Dockerfile.konflux` with FIPS-compliant Go build flags (`GOEXPERIMENT=strictfipsruntime -tags strictfipsruntime`) on a UBI9 base image. The service integrates with Istio service mesh for mTLS and traffic routing, and with KServe for inference service lifecycle management.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| model-registry (proxy server) | Go REST API Service | Serves Model Registry REST API (v1alpha3) for CRUD operations on models, versions, artifacts, inference services, and serving environments |
| model-registry (catalog server) | Go REST API Service | Serves Model Catalog REST API (v1alpha1) for browsing curated models from external sources (HuggingFace) and MCP servers; uses PostgreSQL with leader election |
| model-registry-controller | Go Operator (controller-runtime) | Watches KServe InferenceService CRs and syncs their state (creation, URL updates, deletion/UNDEPLOYED) with Model Registry via REST API |
| mr-storage-initializer (CSI) | Go CLI / Init Container | KServe ClusterStorageContainer that resolves `model-registry://` URIs and downloads model artifacts for inference |
| model-registry-ui (BFF) | Go HTTP Server | Backend-for-Frontend proxy serving static React assets and proxying API calls to Model Registry; supports internal and user_token auth |
| model-registry-ui (frontend) | React/TypeScript SPA | User interface for managing model registry entities, built with PatternFly |
| async-upload job | Python Job | Kubernetes Job that uploads models from sources (HuggingFace) to destinations (S3/OCI); uses model-registry Python client |
| model-registry Python client | Python Library | Python SDK for interacting with Model Registry REST API; supports boto3, OCI (olot), and model signing |
