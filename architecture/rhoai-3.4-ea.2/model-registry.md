## Purpose
**Short**: A central metadata registry for recording, versioning, and retrieving ML model metadata, serving environments, inference services, experiments, and model catalog entries.

**Detailed**: Model Registry provides a REST API server and persistence layer for managing the lifecycle metadata of machine learning models. It stores information about registered models, model versions, model artifacts, serving environments, inference services, experiments, and experiment runs. The service acts as a centralized repository where ML developers and data scientists can track their models from training through deployment.

The component includes four main subcomponents: (1) a REST API proxy server backed by an embedded metadata store (EmbedMD) supporting MySQL or PostgreSQL databases, (2) a Kubernetes controller that watches KServe InferenceService custom resources and synchronizes their state with the Model Registry, (3) a CSI storage initializer for KServe that enables model loading via `model-registry://` URI scheme, and (4) a Model Catalog server providing curated model discovery with support for HuggingFace sources and MCP (Model Context Protocol) server catalogs.

The system is built with FIPS compliance in mind (using `strictfipsruntime` build tags in Konflux builds) and supports TLS 1.2+ for database connections with configurable cipher suites. In RHOAI deployments, the service integrates with the Istio service mesh for mTLS and can be exposed via Istio VirtualService through the Kubeflow Gateway.

### TLS Configuration

The model registry supports TLS 1.2+ for database connections with the following configurable options:
- Client certificate (`--embedmd-database-ssl-cert`)
- Client key (`--embedmd-database-ssl-key`)
- Root CA certificate (`--embedmd-database-ssl-root-cert`)
- CA certificate (`--embedmd-database-ssl-ca`)
- Cipher suites (`--embedmd-database-ssl-cipher`) - colon-separated, insecure ciphers rejected
- Server certificate verification (`--embedmd-database-ssl-verify-server-cert`)
- Minimum TLS version enforced: TLS 1.2

### FIPS Compliance
