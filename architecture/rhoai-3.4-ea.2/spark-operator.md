## Purpose
**Short**: Kubernetes operator for managing Apache Spark application lifecycle on OpenShift, including submission, scheduling, monitoring, and scaling of Spark workloads.

**Detailed**: The Spark Operator (originally Kubeflow Spark Operator) automates the deployment and management of Apache Spark applications on Kubernetes/OpenShift clusters. It provides Custom Resource Definitions (CRDs) that allow users to declaratively define Spark applications, scheduled Spark applications, and Spark Connect server instances.

The operator consists of two main deployments: a controller that watches SparkApplication, ScheduledSparkApplication, and SparkConnect custom resources and manages their lifecycle (including Spark driver/executor pod creation, service creation, ingress management, and monitoring configuration), and a webhook that performs mutation and validation of Spark pods and SparkApplication resources. The controller uses `spark-submit` internally to launch Spark driver pods, and manages the complete application state machine from submission through completion or failure, with retry and scheduling support.

For RHOAI, this component is deployed via kustomize overlays into the `redhat-ods-applications` namespace by the rhods-operator. The RHOAI overlay adds network policies, RHOAI-specific labels, and configures image references via params.env substitution. The operator supports multiple batch schedulers including Kubernetes default scheduler, Volcano, and YuniKorn.

| MutatingWebhookConfiguration Controller | Controller | Self-manages MutatingWebhookConfiguration CA bundle updates |
| ValidatingWebhookConfiguration Controller | Controller | Self-manages ValidatingWebhookConfiguration CA bundle updates |
