## Purpose
**Short**: Kubernetes operator managing TrustyAI explainability services, LM evaluation jobs, guardrails orchestration, NeMo Guardrails, and evaluation hub services for AI/ML model trustworthiness on OpenShift.

**Detailed**: The TrustyAI Service Operator is a multi-controller Go operator built with controller-runtime that manages five distinct service controllers within a single binary: TrustyAI Service (TAS) for AI model explainability and fairness monitoring, LMEvalJob (LMES) for language model evaluation using the lm-eval-harness framework, GuardrailsOrchestrator (GORCH) for orchestrating AI guardrail detectors around inference services, NeMo Guardrails for NVIDIA NeMo-based guardrail deployments, and EvalHub for centralized evaluation management with multi-tenant support.

The operator integrates deeply with the OpenDataHub/RHOAI ecosystem, watching KServe InferenceService resources to automatically configure payload processors for model monitoring. It supports both PVC and database-backed storage for the TrustyAI service, Kueue-based job scheduling for LMEvalJobs, and features automatic detection of guardrail detector inference services. Each controller manages its own Deployments, Services, Routes, and RBAC resources, with kube-rbac-proxy sidecars providing authentication enforcement on managed service endpoints.

The operator is built with FIPS compliance (strict FIPS runtime) via Konflux CI/CD pipelines and uses OpenShift serving certificates for TLS. It deploys OpenShift Routes with TLS re-encryption for external access to managed services.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| TrustyAI Service Controller (TAS) | Go Controller | Manages TrustyAI explainability service Deployments with kube-rbac-proxy sidecars, Services, Routes, PVCs, ServiceMonitors, and patches InferenceService payload processors |
| LMEvalJob Controller (LMES) | Go Controller | Manages LM evaluation job Pods with driver sidecars, handles job lifecycle (New/Scheduled/Running/Complete/Cancelled/Suspended), supports offline S3/PVC storage and OCI output |
| Job Manager Controller (JOB_MGR) | Go Controller | Kueue integration controller for LMEvalJob workload scheduling via Kueue Workload CRs |
| GuardrailsOrchestrator Controller (GORCH) | Go Controller | Manages guardrails orchestrator Deployments with optional built-in detector and gateway sidecars, kube-rbac-proxy authentication, auto-configuration from InferenceServices |
| NeMo Guardrails Controller (NEMO_GUARDRAILS) | Go Controller | Manages NVIDIA NeMo Guardrails server Deployments with CA bundle aggregation, kube-rbac-proxy auth, and configmap-based configuration mounting |
| EvalHub Controller (EVALHUB) | Go Controller | Manages EvalHub evaluation service Deployments with TLS, provider/collection ConfigMap mounting, multi-tenant namespace RBAC provisioning, PostgreSQL database support, and OpenTelemetry configuration |
| Operator Manager | Go Binary | Single binary running all controllers, with `--enable-services` flag selecting active controllers (default: TAS,LMES,GORCH,NEMO_GUARDRAILS,EVALHUB) |
