## Purpose
**Short**: Multi-controller operator managing TrustyAI model explainability services, LM evaluation jobs, guardrails orchestration, NVIDIA NeMo Guardrails, and EvalHub evaluation hubs on OpenShift.

**Detailed**: The TrustyAI Service Operator is a Go-based Kubernetes operator built with controller-runtime that manages five distinct service controllers within a single binary: TAS (TrustyAI Service), LMES (LM Evaluation Service), GORCH (Guardrails Orchestrator), NEMO_GUARDRAILS (NVIDIA NeMo Guardrails), and EVALHUB (Evaluation Hub). Each controller watches its own CRD and reconciles the necessary Kubernetes resources (Deployments, Services, Routes, ConfigMaps, ServiceAccounts, RBAC) to deploy and manage the respective workloads.

The operator is deployed by the rhods-operator via kustomize overlays. The RHOAI overlay at `config/overlays/rhoai/` merges a `params.env` ConfigMap that provides container image references and configuration parameters. The operator supports multiple storage backends (PVC and database) for TrustyAI Service, integrates with KServe InferenceServices for model monitoring, supports Kueue for job queue management of LMEvalJobs, and provides optional kube-rbac-proxy authentication sidecars for secure access. It also manages Istio VirtualService and DestinationRule resources when service mesh is present.

The operator creates OpenShift Routes for external access to managed services (TrustyAI Service, Guardrails Orchestrator, NeMo Guardrails, EvalHub). Each managed service deployment includes a kube-rbac-proxy sidecar container for RBAC-based authentication, listening on port 8443/TCP and proxying to the application container on port 8080/TCP.

| Job Manager Controller (JOB_MGR) | Go Controller | Integrates LMEvalJob with Kueue workload queue management for resource-aware scheduling |
| Operator Manager | Go Binary | Main entry point; runs all enabled controllers via `--enable-services` flag, health/readiness probes on port 8081 |
