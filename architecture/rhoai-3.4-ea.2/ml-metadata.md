## Purpose
**Short**: ML Metadata is a gRPC server and library for recording and retrieving metadata associated with ML workflows, providing lineage tracking for artifacts, executions, and contexts.

**Detailed**: ML Metadata (MLMD) is a core infrastructure component in RHOAI that provides a metadata store for tracking machine learning workflow artifacts, executions (pipeline steps), and contexts. It is originally developed by Google as part of the TFX (TensorFlow Extended) ecosystem and is forked by Red Hat for use in OpenShift AI.

The component ships as a standalone gRPC server (`metadata_store_server`) built from C++ source code using Bazel. It exposes a comprehensive gRPC API (`MetadataStoreService`) that allows ML pipeline systems (primarily Kubeflow Pipelines / Data Science Pipelines) to store and query metadata about pipeline runs, artifacts produced and consumed, execution lineage graphs, and type registries. The server supports MySQL, PostgreSQL, and SQLite as backend storage, with configurable SSL/TLS for both the gRPC channel and database connections.

In the RHOAI platform, MLMD serves as the metadata backbone for Data Science Pipelines, storing artifact lineage, execution history, and pipeline context relationships. The container image is built via Konflux using a multi-stage build on UBI 9, producing the `odh-mlmd-grpc-server` image.

## Security

### Server TLS Configuration

The gRPC server supports optional SSL/TLS via `MetadataStoreServerConfig.SSLConfig`:

| Setting | Type | Purpose |
|---------|------|---------|
| server_key | PEM string | Private key for server TLS |
| server_cert | PEM string | Server certificate for TLS |
| custom_ca | PEM string | Custom CA certificate(s) |
| client_verify | Boolean | When true, requires and verifies client certificates (mTLS) |

Without SSL configuration, the server uses insecure gRPC credentials (plaintext).

### Database Connection Security

**MySQL SSL Options:**

| Setting | Purpose |
|---------|---------|
