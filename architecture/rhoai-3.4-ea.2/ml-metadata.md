## Purpose
**Short**: ML Metadata (MLMD) is a gRPC server for recording and retrieving metadata associated with ML workflows, artifacts, executions, and contexts.

**Detailed**: ML Metadata (MLMD) is a library and gRPC service originally developed by Google as part of the TFX (TensorFlow Extended) ecosystem. It provides a metadata store that records the provenance, lineage, and properties of ML artifacts, executions, and contexts. The MLMD data model centers around three core concepts: **Artifacts** (data inputs/outputs like datasets and models), **Executions** (computational steps in a pipeline), and **Contexts** (groupings like pipeline runs or experiments). Events link artifacts to executions, forming a lineage graph.

In the RHOAI distribution, the MLMD gRPC server is built as a C++ binary from source using Bazel and packaged into a UBI9-based container image (`odh-mlmd-grpc-server`). It is built via Konflux CI/CD pipelines. The server connects to a backend database (MySQL, PostgreSQL, or SQLite) to persist metadata and exposes a gRPC API on a configurable port (default 8080). This component is consumed by Kubeflow Pipelines and Data Science Pipelines to track pipeline runs, artifacts, and lineage information.

The server supports optional TLS encryption for the gRPC channel and SSL connections to the database backend. It also supports database schema migrations for upgrades and downgrades across versions.
