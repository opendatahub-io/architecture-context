## Purpose
**Short**: A containerized Llama Stack server distribution for Red Hat OpenShift AI, providing a unified API gateway to multiple AI/ML inference, safety, evaluation, and vector storage backends.

**Detailed**: The Llama Stack Distribution is a container image that packages the [Llama Stack](https://github.com/llamastack/llama-stack) server with Red Hat-specific configurations, providers, and dependencies. It provides a single API endpoint (port 8321) that abstracts access to multiple AI inference backends (vLLM, AWS Bedrock, Azure, Vertex AI, OpenAI, WatsonX), vector databases (Milvus, pgvector, Qdrant, FAISS), safety systems (TrustyAI FMS), evaluation frameworks (LM Eval, RAGAS, Garak), and tool runtimes (Brave Search, Tavily, MCP, RAG).

The distribution is built on UBI9 with Python 3.12 and uses a Konflux-based CI/CD pipeline for RHOAI releases. The Containerfile is auto-generated from a template (`Containerfile.in`) by a build script (`build.py`) that uses the `llama stack list-deps` CLI to resolve dependencies. The image ships with an IBM Granite embedding model (`granite-embedding-125m-english`) pre-downloaded. Provider activation is controlled via environment variables, allowing conditional enablement of backends without image rebuilds. PostgreSQL is used as the primary persistent storage backend for key-value stores, SQL stores, inference logging, and agent state.
