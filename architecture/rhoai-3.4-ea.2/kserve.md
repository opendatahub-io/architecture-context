## Purpose
**Short**: KServe provides a Kubernetes-native platform for serving machine learning models and LLM inference workloads on OpenShift AI.

**Detailed**: KServe is the model serving component of Red Hat OpenShift AI (RHOAI). It defines Custom Resource Definitions (CRDs) for deploying, scaling, and managing inference services, serving runtimes, and inference graphs. The component ships two independent controller managers: the **kserve-controller-manager** handles traditional InferenceService, InferenceGraph, TrainedModel, and related CRDs, while the **llmisvc-controller-manager** handles the newer LLMInferenceService CRD optimized for large language model (LLM) serving with features like disaggregated prefill/decode, multi-node inference via LeaderWorkerSets, Gateway API integration with InferencePool-based routing through the llm-d inference scheduler, and Istio DestinationRule management for TLS communication.

KServe supports multiple deployment modes: RawDeployment (default in RHOAI), Knative Serverless, and ModelMesh. The controllers dynamically create Deployments, Services, HTTPRoutes, VirtualServices, InferencePools, Istio DestinationRules, self-signed TLS certificates, RBAC resources, and Prometheus monitoring resources (PodMonitors/ServiceMonitors) based on the desired state expressed in CRs. For RHOAI, the default deployment mode is RawDeployment with Istio integration for ingress traffic routing.

| llmisvc-controller-manager | Go Operator (controller-runtime) | Reconciles LLMInferenceService CRD; creates Deployments, LeaderWorkerSets, Services, HTTPRoutes, InferencePools, InferenceModels, Istio DestinationRules, self-signed TLS certs, PodMonitors/ServiceMonitors, RBAC resources |

| kserve-router | Container (Go) | Routes requests across InferenceGraph nodes with header propagation |
| kube-rbac-proxy | Sidecar Container | Provides metrics endpoint authentication proxy (used by RHOAI for auth, replaces oauth-proxy) |
| llm-d-inference-scheduler | Container (Go) | Gateway API InferencePool endpoint picker (EPP); provides gRPC-based request routing with KV-cache-aware scheduling for LLM workloads |
| llm-d-routing-sidecar | Init Container | Handles routing setup for disaggregated prefill/decode topologies |
| vllm-cuda / vllm-rocm / vllm-spyre | Container (Python) | vLLM inference engine images for NVIDIA CUDA, AMD ROCm, and IBM Spyre accelerators |
