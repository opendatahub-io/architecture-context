## Purpose
**Short**: KServe is the model serving platform for RHOAI, providing a Kubernetes-native way to deploy, manage, and scale machine learning inference services, including LLM inference workloads.

**Detailed**: KServe provides a standardized, cloud-native model serving framework built on Kubernetes. It supports multiple deployment modes: Knative serverless (via Knative Serving), RawDeployment (plain Kubernetes Deployments), and ModelMesh (for high-density multi-model serving). KServe manages the complete lifecycle of inference services including model storage initialization, traffic routing, autoscaling, canary rollouts, and inference graph orchestration.

In RHOAI 3.x, KServe introduces the LLMInferenceService (v1alpha2) CRD specifically designed for large language model inference workloads. This controller manages single-node and multi-node (LeaderWorkerSet) LLM deployments, Gateway API HTTPRoute-based traffic routing, InferencePool-based scheduling (via the gateway-api-inference-extension), and Istio DestinationRules for TLS communication. The LLMInferenceService controller supports disaggregated prefill/decode architectures and integrates with the llm-d inference scheduler for intelligent request routing.

For RawDeployment mode InferenceServices on OpenShift, KServe injects kube-rbac-proxy sidecar containers for authentication, enforcing SubjectAccessReview-based authorization in front of model endpoints. The controller creates HTTPRoute CRs for Gateway API-based ingress and manages Istio VirtualService resources for service mesh-based routing.

**Controller-managed ingress details**:
- **InferenceService (RawDeployment mode)**: Controller creates HTTPRoute CRs referencing a parent Gateway. On OpenShift with auth enabled, it injects a `kube-rbac-proxy` sidecar container (port 8443/TCP) that performs SubjectAccessReview-based authorization before proxying to the application container (port 8080/TCP).
- **LLMInferenceService**: Controller creates HTTPRoute CRs for Gateway API routing, optionally with InferencePool-based intelligent load balancing. On OpenShift with Istio gateways, the controller creates DestinationRule CRs for TLS communication between the gateway and workload/scheduler services (using service-ca certificates or self-signed certificates).
- **InferenceGraph**: Controller creates OpenShift Routes for external access to inference graph endpoints.
