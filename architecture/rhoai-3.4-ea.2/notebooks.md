## Purpose
**Short**: Provides pre-built workbench (IDE) and pipeline runtime container images for data science workflows on Red Hat OpenShift AI.

**Detailed**: The notebooks repository builds and maintains a comprehensive set of container images that serve as interactive development environments (workbenches) and pipeline execution runtimes for data scientists using the Red Hat OpenShift AI (RHOAI) platform. Workbench images include JupyterLab, VS Code (code-server), and RStudio Server, each available in multiple variants with different ML framework stacks (PyTorch, TensorFlow, TrustYAI) and accelerator support (CPU, NVIDIA CUDA, AMD ROCm). Pipeline runtime images provide minimal execution environments for Elyra-based pipeline nodes.

The repository does not contain an operator or controller. It produces container images that are consumed by the ODH Notebook Controller (separate repository), which manages the lifecycle of Notebook CR instances. The deployment manifests in this repository define OpenShift ImageStream resources that register built images in the cluster's internal registry, making them available for selection through the RHOAI Dashboard. Images are built hermetically via Konflux/Tekton pipelines with all dependencies prefetched using cachi2 for supply chain security.

### Ingress

Ingress is managed by the ODH Notebook Controller (RHOAI 3.x pattern):

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| (notebook-name) | HTTPRoute (Gateway API) | Per-notebook dynamic hostname | 443/TCP | HTTPS | TLS (terminated at gateway) | SIMPLE | External via data-science-gateway |

The ODH Notebook Controller creates:
- **HTTPRoute CRs** per notebook, referencing the platform `data-science-gateway` Gateway
- **kube-rbac-proxy sidecar** (8443/TCP) injected into each notebook pod for RBAC-based authentication before proxying to the application container (8888/TCP)

### Image Versioning

Each ImageStream maintains multiple tagged versions for rolling upgrades:
- **N** (3.4): Current release, `opendatahub.io/workbench-image-recommended: 'true'`
- **N-1** (2025.2): Previous release
- **N-2** through **N-7**: Historical versions marked with `opendatahub.io/image-tag-outdated: 'true'`

Current N images are sourced from `registry.redhat.io/rhoai/` with SHA256 digest pinning. Historical images reference `quay.io/modh/` repositories.
