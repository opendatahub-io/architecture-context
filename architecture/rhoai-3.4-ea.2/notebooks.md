## Purpose
**Short**: Provides pre-built container images for data science workbenches (Jupyter, Code Server, RStudio) and Elyra pipeline runtime environments, deployed as OpenShift ImageStreams for use by the ODH Notebook Controller.

**Detailed**: The notebooks repository is a container image build repository that produces workbench and runtime images for Red Hat OpenShift AI (RHOAI). It does not contain an operator or controller — instead, it defines Dockerfiles for building workbench environments (Jupyter notebooks, Code Server, RStudio) and pipeline runtime images, along with kustomize manifests that declare OpenShift ImageStream resources referencing those pre-built images.

The workbench images support multiple accelerator backends (CPU, NVIDIA CUDA, AMD ROCm) and Python versions (3.11, 3.12). Each workbench type maintains a versioned chain of images (from the latest "N" version back to "1.2"), enabling users to select specific versions. The kustomize manifests are consumed by the rhods-operator at deploy time: the operator patches image references from its ClusterServiceVersion's `RELATED_IMAGE_*` environment variables, renders the ImageStreams via kustomize, and applies them to the cluster.

Images are built using Konflux (Tekton) pipelines. The repository contains 94 Tekton pipeline definitions covering pull-request validation and push builds for each image variant. The resulting images are published to `registry.redhat.io/rhoai/` for RHOAI releases and `quay.io/modh/` or `quay.io/opendatahub/` for upstream/development.
