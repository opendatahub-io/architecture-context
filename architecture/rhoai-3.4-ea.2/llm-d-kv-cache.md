## Purpose
**Short**: A pluggable KV-Cache management library and tokenizer sidecar service for KV-Cache aware routing in vLLM-based LLM inference platforms.

**Detailed**: The `llm-d-kv-cache` component is a high-performance library and service suite that enables KV-Cache aware routing within the llm-d distributed LLM inference platform. It maintains a global, near-real-time view of KV-Cache block locality across a fleet of vLLM pods by consuming structured `KVEvents` streamed from vLLM instances via ZeroMQ (ZMQ). These events describe KV-block creation and eviction, allowing the indexer to track which blocks reside on which pods and on which device tier (GPU, CPU, or disk).

The primary use case is to score vLLM pods for a given inference prompt, enabling schedulers (such as `llm-d-inference-scheduler`) to make optimal KV-cache-aware placement decisions. This significantly improves Time To First Token (TTFT) and overall inference throughput by reusing existing KV-cache blocks rather than recomputing them. The component also includes a UDS (Unix Domain Socket) tokenizer sidecar service that provides gRPC-based tokenization using vLLM tokenizers, as well as KV-connector plugins (`llmd_fs_backend`, `pvc_evictor`) for filesystem-backed KV-cache storage and eviction.

For RHOAI, only the UDS tokenizer sidecar (`services/uds_tokenizer`) is shipped as a container image via Konflux. The Go library code is consumed as a dependency by other components (e.g., llm-d-inference-scheduler).
