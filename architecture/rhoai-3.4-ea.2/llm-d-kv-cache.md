## Purpose
**Short**: A pluggable KV-Cache indexing and management library for KV-Cache-aware routing in vLLM-based LLM inference platforms.

**Detailed**: The llm-d-kv-cache project is a core component of the llm-d distributed LLM inference platform. It provides a high-performance library (and optional standalone services) that maintains a global, near-real-time view of KV-Cache block locality across a fleet of vLLM pods. By tracking which KV-cache blocks reside on which pods (and on which device tier - GPU vs CPU), it enables intelligent, cache-aware routing decisions that significantly improve Time To First Token (TTFT) and overall inference throughput.

The project consists of several key pieces: (1) the **KV-Cache Indexer** library - the primary Go library that provides pod scoring based on KV-cache hits, designed to be embedded into schedulers like `llm-d-inference-scheduler`; (2) the **UDS Tokenizer Service** - a Python gRPC sidecar service that provides tokenization capabilities over Unix Domain Sockets; (3) the **llmd-fs-backend** - a CUDA-accelerated vLLM offloading connector for filesystem-backed KV-cache storage; and (4) the **PVC Evictor** - a multi-process Python utility for automatic disk space management on PVC-backed KV-cache storage.

The RHOAI build (via Konflux) ships the UDS Tokenizer Service as the container image `odh-llm-d-kv-cache-rhel9`. The Go library is consumed as a dependency by other llm-d components (e.g., the inference scheduler).

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| KV-Cache Indexer (`pkg/kvcache`) | Go Library | Main orchestrator: tokenizes prompts, generates block keys, queries index, scores pods for cache-aware routing |
| KV-Block Index (`pkg/kvcache/kvblock`) | Go Library | Pluggable data store mapping KV-block hashes to pod locations; backends: in-memory LRU, cost-aware (Ristretto), Redis, Valkey |
| KV-Events Pool (`pkg/kvevents`) | Go Library | Sharded worker pool consuming ZMQ events from vLLM pods to update the block index in near-real-time |
| ZMQ Subscriber Manager (`pkg/kvevents`) | Go Library | Manages per-pod ZMQ subscriber lifecycle with Kubernetes pod reconciler for auto-discovery |
| Tokenization Pool (`pkg/tokenization`) | Go Library | Worker pool for prompt tokenization with composite fallback (local -> HuggingFace -> UDS service) |
| UDS Tokenizer Service (`services/uds_tokenizer`) | Python gRPC Sidecar | gRPC tokenizer service communicating over Unix Domain Sockets; uses vLLM tokenizer for chat template rendering |
| Indexer gRPC Service (`api/indexerpb`) | gRPC API | gRPC service definition for remote KV-cache scoring (GetPodScores RPC) |
| Tokenizer gRPC Service (`api/tokenizerpb`) | gRPC API | gRPC service definition for tokenization (Tokenize, RenderChatTemplate, InitializeTokenizer RPCs) |
| llmd-fs-backend (`kv_connectors/llmd_fs_backend`) | Python/CUDA Extension | vLLM offloading connector for filesystem-backed KV-cache with GPU DMA transfers |
| PVC Evictor (`kv_connectors/pvc_evictor`) | Python Service | Multi-process (N+2 architecture) utility for automatic disk space management on KV-cache PVCs |
| Prometheus Metrics (`pkg/kvcache/metrics`) | Go Library | Prometheus metrics for cache admissions, evictions, lookup hits, latency, and tokenization performance |
| OpenTelemetry Tracing (`pkg/telemetry`) | Go Library | OTLP-based distributed tracing with W3C trace context propagation |
| Helm Chart (`vllm-setup-helm`) | Helm Chart | Reference Helm chart for deploying vLLM fleet with KV-cache manager and optional LMCache/Redis |
