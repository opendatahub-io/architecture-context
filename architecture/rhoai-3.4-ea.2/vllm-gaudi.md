## Purpose
**Short**: Hardware plugin enabling vLLM large language model inference on Intel Gaudi AI accelerators.

**Detailed**: The vLLM Hardware Plugin for Intel Gaudi integrates Intel Gaudi (formerly Habana Labs) AI accelerators with the vLLM inference engine to optimize large language model (LLM) serving. It follows the vLLM hardware plugin architecture, providing a modular platform interface that allows vLLM to leverage Gaudi's HPU (Habana Processing Unit) hardware for high-performance inference.

The plugin provides HPU-specific implementations for critical inference components including attention backends (paged attention, unified attention, MLA), custom operations (fused MoE, rotary embeddings, quantization), model runners, worker processes, and distributed communication. It supports multiple quantization formats (FP8, compressed-tensors, AWQ, GPTQ) and advanced features like LoRA adapters, speculative decoding (EAGLE), KV-cache transfer via NIXL, and data parallelism.

In the RHOAI distribution, this component is shipped as a container image (`odh-vllm-gaudi-rhel9`) built on UBI 9.6 with Habana Synapse AI drivers, PyTorch, and the vLLM OpenAI-compatible API server. It serves as the inference runtime for Intel Gaudi accelerator nodes within Red Hat OpenShift AI, typically deployed via KServe/ModelMesh as a model serving runtime.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| HpuPlatform | Python Plugin (vLLM Platform) | Core platform class registering Gaudi HPU as a vLLM device target; configures attention backends, quantization, compilation, and worker classes |
| HPU Attention Backends | Python Module | HPU-optimized attention implementations: HPUAttentionV1, HPUUnifiedAttention, HPUMLA (Multi-Latent Attention) |
| HPU Custom Ops | Python Module | Hardware-optimized operators: fused MoE, rotary embedding, layer normalization, FP8/AWQ/GPTQ quantization, convolution |
| HPU Model Runner | Python Module | Manages model execution on HPU devices including batch scheduling, graph capture, and memory management (317KB of logic) |
| HPU Worker | Python Module | Worker process managing per-device model loading, KV-cache allocation, and inference execution on HPU devices |
| HPU Communicator | Python Module | Distributed communication (all-reduce, all-gather, reduce-scatter) via Habana Collective Communication Library (HCCL) |
| NIXL KV Connector | Python Module | Disaggregated KV-cache transfer between HPU devices using NIXL protocol for prefill-decode separation |
| EAGLE Speculative Decoding | Python Module | HPU-specific implementation of EAGLE speculative decoding for faster inference |
| Bucketing System | Python Module | Request batching strategies (exponential, linear, unified, vision) for efficient HPU utilization |
| Calibration Tools | Python Scripts | FP8 quantization calibration pipeline (measure scales, postprocess, quantize, unify measurements) |
| Model Overrides | Python Module | HPU-optimized model implementations for Gemma3, Qwen2.5-VL, Qwen3-VL, Qwen3-MoE |
| OpenAI API Server | Container Entrypoint | vLLM OpenAI-compatible API server (`vllm.entrypoints.openai.api_server`) as the container entrypoint |
