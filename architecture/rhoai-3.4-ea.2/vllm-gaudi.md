## Purpose
**Short**: vLLM hardware plugin for Intel Gaudi AI accelerators enabling optimized large language model inference as an OpenAI-compatible API server.

**Detailed**: The vllm-gaudi repository provides the Intel Gaudi hardware plugin for the vLLM inference engine. It implements the vLLM hardware plugin architecture (following the "Hardware Pluggable" RFC) to enable high-performance LLM inference on Intel Gaudi (HPU) accelerators. The plugin registers a custom platform (`HpuPlatform`), custom attention backends, custom ops (FP8 quantization, rotary embeddings, fused MoE, etc.), and custom model implementations optimized for Gaudi hardware.

The container image ships a pre-built vLLM installation with the Gaudi plugin installed, running the `vllm.entrypoints.openai.api_server` entrypoint. This exposes an OpenAI-compatible HTTP API for model inference (chat completions, completions, embeddings). The image is built on Red Hat UBI 9.6 and includes Intel Synapse AI drivers (v1.23.0), PyTorch for Habana (v2.9.0), and all required Habana system libraries. It runs as non-root user (UID 2000) for OpenShift compatibility.

The plugin supports advanced features including tensor parallelism via HCCL (Habana Collective Communications Library), KV-cache transfer via NIXL for disaggregated serving, FP8 quantization (compressed-tensors, inc, awq_hpu, gptq_hpu), speculative decoding (EAGLE), LoRA adapters, and multi-modal models (Gemma3, Qwen2.5-VL, Qwen3-VL).
