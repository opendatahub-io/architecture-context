## Purpose
**Short**: TrustyAI is a Java-based REST service providing AI fairness metrics, model explainability algorithms, and data drift detection for ML models deployed on OpenShift AI.

**Detailed**: TrustyAI Explainability is the core library and service component of the TrustyAI project, providing Explainable AI (XAI) capabilities within Red Hat OpenShift AI. It operates as a Quarkus-based REST service that integrates with ModelMesh and KServe model serving platforms to intercept and store model inference data (inputs and outputs), then compute fairness metrics (e.g., Statistical Parity Difference, Disparate Impact Ratio), data drift metrics (e.g., KS-Test, FourierMMD, Meanshift), and local/global explainability (LIME, SHAP, Counterfactuals, PDP, TSSaliency).

The repository is organized as a multi-module Maven project. The `explainability-core` module provides the underlying Java library with fairness metrics, drift detection, anomaly detection, and language model accuracy metrics. The `explainability-connectors` module provides gRPC client integration with KServe/ModelMesh via the KServe Prediction Protocol v2. The `explainability-service` module wraps these into a containerized Quarkus REST service that can be deployed in Kubernetes/OpenShift, managed by the separate TrustyAI Service Operator. The `explainability-arrow` module provides Apache Arrow-based interoperability between TrustyAI Java and Python.

The service supports multiple storage backends including PVC-based flat file storage, in-memory storage, MinIO object storage, and database storage via Hibernate ORM (MariaDB/MySQL). It exposes Prometheus metrics for scheduled fairness and drift computations, and can consume inference payloads via REST endpoints or Knative CloudEvents.
