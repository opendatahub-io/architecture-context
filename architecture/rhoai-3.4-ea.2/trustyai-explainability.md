## Purpose
**Short**: TrustyAI is a Java library and Quarkus REST service providing fairness metrics, bias tracking, data drift detection, and explainable AI (XAI) algorithms for ML model inference monitoring.

**Detailed**: TrustyAI Explainability is the core Java library and service component of the TrustyAI ecosystem within Red Hat OpenShift AI (RHOAI). It provides a comprehensive set of tools for Responsible AI, including group fairness metrics (Statistical Parity Difference, Disparate Impact Ratio), data drift detection algorithms (KS Test, Meanshift, Fourier MMD), and local/global explainability methods (LIME, SHAP, Counterfactuals, Partial Dependence Plots, Time Series Saliency).

The service operates as a Quarkus-based REST application that integrates with ModelMesh and KServe inference platforms. It consumes inference payloads (input/output pairs) from model serving infrastructure via REST endpoints or Knative CloudEvents, reconciles them into a unified data store, and computes fairness, drift, and explainability metrics on demand or on a scheduled basis. Computed metrics are exported to Prometheus for monitoring and alerting. The service supports multiple storage backends including PVC-based flat file storage, MinIO object storage, and database storage via Hibernate ORM (MariaDB/MySQL).

The repository is structured as a multi-module Maven project: `explainability-core` (algorithms and data structures), `explainability-connectors` (KServe v1/v2 HTTP and gRPC integration), `explainability-arrow` (Apache Arrow interop for Python bridge), `explainability-service` (Quarkus REST service), and `explainability-integrationtests` (Kogito integration tests). RHOAI profile restricts explainer endpoints (SHAP, LIME, Counterfactual, TSSaliency, Global explainers are disabled) while keeping fairness and drift metrics enabled.
