## Purpose

**Short**: CPU-optimized build of vLLM providing high-throughput LLM inference serving with OpenAI-compatible and gRPC APIs.

**Detailed**: vLLM CPU is a Red Hat downstream fork of the vLLM project, optimized for running large language model (LLM) inference on CPU hardware without requiring GPU accelerators. It provides an OpenAI-compatible HTTP API server (FastAPI/Uvicorn) and a gRPC server for efficient binary communication, supporting text generation, chat completions, embeddings, and other inference tasks.

The container image is built using Konflux CI (`Dockerfile.konflux.cpu`) on UBI 9 minimal base, with architecture-specific build scripts for ppc64le and s390x platforms. It includes CPU-optimized attention kernels, DNNL integration, and architecture-specific SIMD optimizations (x86 AVX, ARM NEON, VSX for Power, VXE for s390x). The image ships as part of RHOAI and is deployed as a KServe inference runtime within OpenShift AI.

The server supports PagedAttention for efficient memory management, continuous batching, speculative decoding, structured output generation (JSON schema, regex, grammar), multi-LoRA adapters, and Prometheus metrics. It also includes Model Context Protocol (MCP) tool integration and Anthropic Messages API compatibility.
