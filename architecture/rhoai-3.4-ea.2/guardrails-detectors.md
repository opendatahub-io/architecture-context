## Purpose
**Short**: A collection of detector algorithm microservices for the FMS Guardrails Orchestrator, providing content safety analysis via regex/heuristic, HuggingFace model-based, and LLM-as-a-judge detection patterns.

**Detailed**: The guardrails-detectors repository provides three distinct detector microservices that integrate with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator) to perform content safety and compliance analysis on LLM inputs and outputs. Each detector is packaged as an independent container image and deployed via KServe ServingRuntime/InferenceService on OpenShift.

The **Built-in Detector** provides lightweight, CPU-only detection functions including regex-based PII detection (email, credit card, SSN, phone numbers, IP addresses, UK postcodes), file-type validation (JSON, XML, YAML with optional schema validation), and user-defined custom detectors written in Python with static code analysis for security sandboxing. The **HuggingFace Detector** is a generic runtime that loads any `AutoModelForSequenceClassification` or `GraniteForCausalLM` model to perform ML-based content classification (e.g., toxicity, prompt injection, bias detection) with optional CUDA/GPU acceleration. The **LLM Judge Detector** integrates the [vLLM Judge](https://github.com/trustyai-explainability/vllm_judge) library to use an external OpenAI-compatible LLM server for evaluation across built-in metrics such as safety, accuracy, helpfulness, and custom natural-language criteria.

All three detectors expose the IBM Detector API (`/api/v1/text/contents`) and are instrumented with Prometheus metrics for monitoring detections, requests, errors, and runtime latency. Container images are built via Konflux (Tekton PipelineRuns) from UBI9-minimal base images with multi-architecture support (amd64, arm64, ppc64le, s390x).

### Security Sandboxing (Custom Detectors)

The CustomDetectorRegistry performs static code analysis on user-provided Python detector code before execution. The following are forbidden by default:
- **Forbidden imports**: `os` (except `os.environ`/`os.getenv`), `subprocess`, `sys`, `shutil`
- **Forbidden function calls**: `eval`, `exec`, `open`, `compile`, `input`
