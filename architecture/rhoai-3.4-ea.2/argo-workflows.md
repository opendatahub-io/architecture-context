## Purpose
**Short**: Argo Workflows is a Kubernetes-native workflow engine that orchestrates parallel jobs on Kubernetes, used as the execution backend for Data Science Pipelines in RHOAI.

**Detailed**: Argo Workflows provides a container-native workflow engine for Kubernetes. It defines workflows as Kubernetes Custom Resources (CRDs) where each step in a workflow runs as a container. The workflow controller watches for Workflow CRs and manages their execution lifecycle, creating pods for each workflow step, handling DAG and step-based parallelism, managing artifact passing between steps, and tracking workflow status.

In the RHOAI distribution, Argo Workflows serves as the core execution engine for Data Science Pipelines (DSP). The DSP operator deploys the workflow controller and argoexec (executor) components. The workflow controller reconciles Workflow CRDs and creates Kubernetes Pods for each workflow node. The argoexec binary runs as a sidecar/init container within workflow pods (emissary executor pattern) to manage artifact collection, output parameter capture, and container lifecycle coordination. The Argo Server (API server) component provides a REST/gRPC API and web UI for workflow management, though in the RHOAI context the DSP API server typically handles user-facing API interactions.

The RHOAI build produces two container images via Konflux: `odh-data-science-pipelines-argo-workflowcontroller` and `odh-data-science-pipelines-argo-argoexec`. Both are FIPS-compliant builds using `GOEXPERIMENT=strictfipsruntime` on UBI9 base images.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| Workflow Controller | Go Controller (client-go informers) | Watches Workflow CRDs, reconciles workflow state, creates/manages pods for workflow steps, handles DAG scheduling, cron workflows, and garbage collection |
| Argo Executor (argoexec) | Go Sidecar Binary | Runs within workflow pods using emissary executor pattern; manages artifact upload/download, output parameter capture, container wait/lifecycle |
| Argo Server | Go API Server (gRPC + HTTP) | Provides REST API (gRPC-gateway) and Web UI for workflow CRUD operations, log streaming, artifact serving, and SSO authentication |
| Workflow UI | TypeScript/React (webpack) | Browser-based dashboard for workflow visualization, log viewing, and management |
| CRD Controller | Go (client-go) | Manages CronWorkflow scheduling, WorkflowTaskSet agent execution, and WorkflowArtifactGCTask cleanup |

### FIPS Compliance

The argoexec image contains two binaries:
- `/bin/argoexec` - Standard build (CGO_ENABLED=0, statically linked)
- `/bin/argoexec-fips` - FIPS-compliant build (CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime)

The workflow controller is built exclusively with FIPS compliance (CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime).
