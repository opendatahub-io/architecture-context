## Purpose
**Short**: REST API orchestrator that coordinates AI text generation with safety guardrails by invoking content detectors on LLM input and output.

**Detailed**: The FMS Guardrails Orchestrator (fms-guardrails-orchestr8) serves as middleware between clients and language models to provide content safety detection, filtering, and secure text generation. It orchestrates multiple detector services, text chunking services, and LLM backends through unified HTTP APIs.

The orchestrator supports multiple generation backends including TGIS (Text Generation Inference Server) via gRPC and OpenAI-compatible APIs via HTTP. It routes text through configurable detector pipelines that can analyze content for harmful patterns, hate speech, profanity (HAP), and other safety concerns. Detectors are invoked on both input prompts and generated output, with configurable thresholds for filtering.

The service supports both unary and streaming request patterns, including Server-Sent Events (SSE) and NDJSON streaming. It provides TLS and mTLS capabilities for secure communication, OpenTelemetry integration for observability (traces and metrics), and configurable header passthrough for authentication delegation.

## Security

### TLS Configuration

The orchestrator supports multiple TLS modes:

| Mode | Server Config | Client Config | Description |
|------|---------------|---------------|-------------|
| Plaintext | No TLS env vars | No tls in config | No encryption |
| TLS | `TLS_CERT_PATH` + `TLS_KEY_PATH` | `cert_path` in service tls | Server-side TLS encryption |
| mTLS | TLS + `TLS_CLIENT_CA_CERT_PATH` | `cert_path` + `key_path` + `client_ca_cert_path` | Mutual TLS with client certificate verification |
| Insecure TLS | N/A | `insecure: true` in tls config | Client skips server certificate verification |

Named TLS configurations can be defined once and referenced by multiple service connections.
