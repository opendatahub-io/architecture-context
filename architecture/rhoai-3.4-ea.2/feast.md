## Purpose
**Short**: The Feast Operator manages the lifecycle of Feast feature stores on Kubernetes/OpenShift, deploying and configuring online stores, offline stores, registries, and UI services via the FeatureStore custom resource.

**Detailed**: Feast (Feature Store) is an open-source feature store for machine learning that helps teams manage, serve, and discover features. The Feast Operator is a Kubernetes operator built with controller-runtime that automates the deployment and management of Feast infrastructure components on OpenShift/Kubernetes clusters.

The operator watches `FeatureStore` custom resources and reconciles the desired state by creating and managing Deployments, Services, ConfigMaps, PersistentVolumeClaims, ServiceAccounts, RBAC resources, CronJobs, and OpenShift Routes. It supports multiple persistence backends (file-based, PostgreSQL, Snowflake, Redis, etc.), authorization models (Kubernetes RBAC and OIDC), TLS configuration (including automatic OpenShift service serving certificates), and a namespace registry for cross-namespace feature store discovery.

The operator also includes a Notebook ConfigMap controller that integrates with Kubeflow Notebooks (OpenDataHub), automatically creating ConfigMaps with Feast client configuration when notebooks are annotated with feast project references. This enables seamless feature store access from within notebook environments.

| FeatureStoreReconciler | Controller | Main reconcile loop for FeatureStore CRs; manages all feast service deployments (online, offline, registry, UI) |
| NotebookConfigMapReconciler | Controller | Watches Kubeflow Notebook CRs and creates/updates feast client ConfigMaps for notebook integration |
| FeastServices | Service Layer | Business logic for deploying feast services, setting up TLS, PVC, containers, and service configurations |
| FeastAuthorization | Auth Module | Manages Kubernetes RBAC or OIDC authorization for feast services; creates Roles, RoleBindings, ClusterRoles, ClusterRoleBindings |
| Feature Server | Python Application | Serves features online (HTTP port 6566/6567), offline (HTTP port 8815/8816), and registry (gRPC port 6570/6571, REST port 6572/6573) |
| Feast UI | Python Application | Web UI for browsing feature store metadata (HTTP port 8888/8443) |
| CronJob | Batch Job | Scheduled execution of `feast apply` and `feast materialize-incremental` commands against feature stores |
| Namespace Registry | ConfigMap | Cross-namespace ConfigMap (`feast-configs-registry`) tracking deployed FeatureStore instances across the cluster |
