## Purpose
**Short**: Kubeflow Trainer is a Kubernetes operator that manages distributed machine learning training jobs using the TrainJob/TrainingRuntime CRD abstraction.

**Detailed**: Kubeflow Trainer (v2) provides a Kubernetes-native API for orchestrating distributed training workloads. Users define training configurations through `TrainJob` custom resources that reference reusable `TrainingRuntime` or `ClusterTrainingRuntime` templates. The controller reconciles these CRs into underlying `JobSet` resources (from the Kubernetes SIG), which in turn manage the actual training pods.

The RHOAI distribution extends the upstream Kubeflow Trainer with Red Hat-specific features including progression tracking (real-time training metrics polling via HTTP from training pods), NetworkPolicy-based pod isolation for training jobs, and pre-configured ClusterTrainingRuntime definitions for PyTorch distributed training across CPU, CUDA, and ROCm accelerator targets. The operator also ships OpenShift ImageStreams for Training Hub workbench images used with the RHOAI dashboard.

The operator supports gang-scheduling through integration with Kubernetes scheduler-plugins (coscheduling) and Volcano, enabling coordinated scheduling of all training pods. It uses the JobSet API to manage replicated training nodes with support for PyTorch distributed (torchrun), MPI, and elastic training configurations.

The controller dynamically creates a `NetworkPolicy` per `TrainJob` with the following rules:

| Rule | Source | Destination | Port | Protocol | Purpose |
|------|--------|-------------|------|----------|---------|

| Controller metrics access | Controller pods (namespace: opendatahub, labels: app.kubernetes.io/name=trainer, app.kubernetes.io/component=controller) | Metrics port (default 28080/TCP) | TCP | HTTP | Progression tracking - only when annotation `trainer.opendatahub.io/progression-tracking: "true"` is set |
