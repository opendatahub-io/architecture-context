## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates NVIDIA's Garak red-teaming framework for automated LLM vulnerability scanning, supporting both inline (local) and remote (Kubeflow Pipelines) execution modes.

**Detailed**: The TrustyAI Garak Llama Stack Provider implements the Llama Stack `Eval` API to perform automated security and safety assessments of Large Language Models. It wraps NVIDIA's Garak framework to probe LLMs for vulnerabilities including prompt injection, jailbreaks, toxicity, bias, and other security weaknesses. The provider supports compliance benchmarks against OWASP LLM Top 10, AVID taxonomy, and CWE classifications.

The provider operates in two execution modes: an inline mode that runs Garak scans as local subprocesses within the Llama Stack server process, and a remote mode that orchestrates scans via Kubeflow Pipelines (KFP) on OpenShift AI. The remote mode deploys a 6-step KFP pipeline (validate, resolve_taxonomy, sdg_generate, prepare_prompts, garak_scan, parse_results) that runs scans in dedicated Kubernetes pods using the provider's container image. Results are persisted to the Llama Stack Files API and include JSONL reports, AVID taxonomy mappings, HTML reports, and a TBSA (Tier-Based Security Aggregate) score.

Additionally, the provider integrates with the eval-hub evaluation platform as a framework adapter, supporting both simple subprocess execution and KFP-delegated execution. It also includes shield testing capabilities that orchestrate input/output safety shields around the target LLM during scanning, using the Llama Stack Safety API. Synthetic Data Generation (SDG) via the sdg-hub library enables custom intent-based risk assessments with adversarial prompt generation.

The provider itself does not define ingress resources. It relies on:
- The Llama Stack operator's NetworkPolicy for restricting access to port 8321
- A supplementary NetworkPolicy (`allow-kfp-to-llamastack`) to allow KFP pipeline pods to reach the Llama Stack service on port 8321/TCP
- DSPA's external route for KFP API access (when using remote mode from outside the cluster)
