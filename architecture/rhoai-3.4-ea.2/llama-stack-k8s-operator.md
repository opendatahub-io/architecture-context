## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Llama Stack AI inference server deployments.

**Detailed**: The Llama Stack K8s Operator is a controller-runtime based Kubernetes operator that automates the deployment, scaling, and management of Llama Stack distribution servers on OpenShift/Kubernetes clusters. It introduces the `LlamaStackDistribution` (LLSD) Custom Resource Definition that allows users to declaratively specify a Llama Stack server deployment including distribution type, container configuration, storage, networking, autoscaling, and TLS settings.

The operator watches for LLSD custom resources and reconciles the desired state by dynamically creating and managing Deployments, Services, ServiceAccounts, PersistentVolumeClaims, NetworkPolicies, Ingresses, PodDisruptionBudgets, and HorizontalPodAutoscalers. It also watches referenced ConfigMaps for user configuration and CA bundles, automatically rolling out pod restarts when configuration changes are detected.

In the RHOAI distribution, the operator is deployed into the `redhat-ods-applications` namespace by the rhods-operator using kustomize overlays. The operator image and the default Llama Stack distribution image are injected as `RELATED_IMAGE_*` environment variables, enabling disconnected/air-gapped deployments.

### Network Policies (Created per LLSD instance)

| Name | Type | Selector | Ingress Rules | Purpose |
|------|------|----------|---------------|---------|
| {instance-name}-network-policy | NetworkPolicy | app=llama-stack, app.kubernetes.io/instance={name} | Same namespace pods, operator namespace, configured namespaces/labels, OpenShift router namespaces | Controls access to Llama Stack server |

Default ingress rules allow:
1. All pods in the same namespace as the LLSD instance
2. All pods from the operator namespace
3. Namespaces listed in `spec.network.allowedFrom.namespaces`
4. Namespaces matching labels in `spec.network.allowedFrom.labels`
5. OpenShift router namespaces (when `spec.network` is configured)
6. Wildcard `"*"` in namespaces list allows all namespaces
