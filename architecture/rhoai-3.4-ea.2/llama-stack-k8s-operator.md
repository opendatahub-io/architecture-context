## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Llama Stack server distributions on OpenShift/Kubernetes clusters.

**Detailed**: The Llama Stack K8s Operator is a controller-runtime based Kubernetes operator that automates the deployment, configuration, and management of Meta's Llama Stack server distributions. It provides a declarative API via the `LlamaStackDistribution` custom resource, enabling users to deploy Llama Stack inference servers with configurable distributions (starter, remote-vllm, meta-reference-gpu, postgres-demo, or custom images).

The operator manages the full lifecycle of Llama Stack deployments including: creating Deployments with proper container configurations and health probes, configuring persistent storage via PVCs, exposing services via ClusterIP Services and optional Kubernetes Ingress for external access, managing NetworkPolicies for namespace-level access control, setting up HorizontalPodAutoscalers for auto-scaling, creating PodDisruptionBudgets for availability guarantees, and handling TLS CA bundle injection for disconnected/proxy environments. The operator also supports user-provided configuration via ConfigMaps and automatically restarts pods when referenced ConfigMaps change.

The operator integrates with OpenShift-specific features including Security Context Constraints (SCC) for pod security and auto-detection of ODH/RHOAI trusted CA bundle ConfigMaps for seamless operation in disconnected environments. It uses kustomize internally for manifest rendering with custom Go-based transformer plugins for field mutation, namespace setting, name prefixing, and NetworkPolicy rule generation.

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| {instance-name}-ingress | Kubernetes Ingress | * (no host constraint) | 80/TCP | HTTP | None | N/A | External (when spec.network.exposeRoute=true) |

The operator creates a Kubernetes `Ingress` resource (not OpenShift Route or Gateway API HTTPRoute) when `spec.network.exposeRoute` is set to `true` on the LlamaStackDistribution CR. This is a standard Kubernetes Ingress with `PathType: Prefix` routing all traffic (`/`) to the backend Service. The Ingress does not configure TLS termination by default.

### Network Policies

| Name | Type | Pod Selector | Ingress Rules | Purpose |
|------|------|--------------|---------------|---------|
| {instance-name}-network-policy | NetworkPolicy | app=llama-stack, app.kubernetes.io/instance={name} | Configurable via spec.network.allowedFrom | Restricts ingress traffic to the Llama Stack server pods |

NetworkPolicy ingress rules are dynamically generated by the NetworkPolicy transformer plugin based on `spec.network.allowedFrom` configuration:
- **Default**: Allows traffic from the instance's own namespace and the operator namespace
- **Namespaces**: Explicit list of namespace names (use `"*"` for all namespaces)
- **Labels**: Namespace label selectors for access control (OR semantics)
- **Feature flag**: Can be globally disabled via `enableNetworkPolicy: false` in operator config
