## Purpose
**Short**: A secure, batch-oriented container runtime for executing language model evaluation benchmarks using the lm-evaluation-harness framework, integrated with EvalHub for job orchestration and result reporting.

**Detailed**: LM Evaluation Harness (LMES Job) provides a standardized runtime environment for evaluating language models against a large library of academic benchmarks and evaluation tasks. It is a downstream fork of EleutherAI's lm-evaluation-harness, extended with Red Hat-specific integrations for the EvalHub platform.

The component runs as a one-shot Kubernetes Job (not a long-running service). It receives a job specification (JobSpec) via a mounted ConfigMap at `/meta/job.json`, connects to language model inference endpoints using OpenAI-compatible APIs (via the `local-completions` backend), runs the specified benchmarks, and reports results back to EvalHub through a callback URL. Results can also be persisted as OCI artifacts to container registries using skopeo, and optionally logged to MLflow for experiment tracking.

The container image is built via Konflux CI/CD pipelines for all four supported CPU architectures (amd64, arm64, ppc64le, s390x), with architecture-specific build stages for PyTorch and Apache Arrow on ppc64le and s390x. The image includes the Unitxt evaluation framework, over 150 evaluation task categories, and 54 offline metrics modules for disconnected operation.
