## Purpose
**Short**: MLServer is an open-source Python inference server that serves machine learning models via REST (V2 Inference Protocol) and gRPC interfaces, used as a model serving runtime within KServe on RHOAI.

**Detailed**: MLServer provides a high-performance, standards-compliant inference server for machine learning models. It implements the V2 Inference Protocol (also known as the KFServing V2 Dataplane specification) on both REST and gRPC transports, enabling interoperability across different model serving frameworks. The server supports multi-model serving, allowing multiple models to be loaded and served within a single process, as well as adaptive batching to group inference requests together for improved throughput.

MLServer is designed as a pluggable inference runtime framework. It ships with pre-packaged runtimes for popular ML frameworks including Scikit-Learn, XGBoost, LightGBM, and ONNX (the runtimes included in the RHOAI Konflux build). Additional runtimes exist for MLflow, HuggingFace, Alibi-Detect, Alibi-Explain, CatBoost, and Spark MLlib. The server supports parallel inference workers for vertical scaling, custom Python environments per model, response caching, Prometheus metrics, OpenTelemetry tracing, and optional Kafka-based inference pipelines.

In the RHOAI platform, MLServer serves as one of the inference runtime backends used by KServe. It is deployed as a container sidecar within KServe InferenceService pods, handling the actual model loading and prediction logic. The RHOAI build packages the core MLServer with the lightgbm, onnx, sklearn, and xgboost runtimes into a single container image based on the AIPCC base image.

| Metrics | Container Port | 8082/TCP | 8082 | HTTP | None (plaintext in-pod) | None | Internal (Prometheus scrape) |

### Ingress

MLServer does not manage its own ingress. External access to MLServer is handled by KServe, which creates:
- **RHOAI 3.x**: HTTPRoute CRs (Gateway API) referencing the platform `data-science-gateway`, with kube-rbac-proxy for authentication
- **RHOAI 2.x (legacy)**: OpenShift Routes with oauth-proxy sidecars

MLServer receives requests from KServe's ingress infrastructure on ports 8080/TCP (REST) and 8081/TCP (gRPC).
