## Purpose
**Short**: Kubeflow Pipelines provides an end-to-end ML workflow orchestration platform that enables authoring, scheduling, and monitoring machine learning pipelines on Kubernetes.

**Detailed**: Data Science Pipelines (based on Kubeflow Pipelines v2) is a core component of Red Hat OpenShift AI that enables data scientists and ML engineers to build, deploy, and manage end-to-end machine learning workflows. The system orchestrates pipeline execution using Argo Workflows as the execution engine, storing pipeline definitions and metadata in a relational database (MySQL or PostgreSQL), and pipeline artifacts in S3-compatible object storage (MinIO or external).

The platform consists of several microservices: an API server providing both REST (HTTP) and gRPC interfaces for pipeline management, a persistence agent that synchronizes Argo Workflow state back to the API server, a scheduled workflow controller for cron-based pipeline scheduling, a cache server for step-level execution caching via mutating webhooks, a metadata store (ML Metadata / MLMD) for artifact lineage tracking, and a React-based frontend UI. The RHOAI distribution builds all components with FIPS-compliant cryptography using `GOEXPERIMENT=strictfipsruntime` on UBI9 base images.

| ml-pipeline-viewer-crd | Go Controller (controller-runtime) | CRD controller for Viewer resources; manages TensorBoard/visualization deployments |

| metadata-grpc-server (MLMD) | C++ Service | ML Metadata store server providing gRPC API for artifact and execution lineage |
| metadata-envoy | Envoy Proxy | gRPC-Web to gRPC proxy fronting the MLMD server |
| kfp-driver | Go Binary | V2 pipeline driver injected into Argo Workflow pods to orchestrate DAG execution |
| kfp-launcher | Go Binary | V2 pipeline launcher injected into Argo Workflow pods to execute user containers |
