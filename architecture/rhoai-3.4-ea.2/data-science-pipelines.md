## Purpose
**Short**: Kubeflow Pipelines provides end-to-end ML workflow orchestration on Kubernetes, enabling creation, scheduling, and management of reusable machine learning pipelines.

**Detailed**: Data Science Pipelines (based on upstream Kubeflow Pipelines v2) is the ML pipeline orchestration platform for Red Hat OpenShift AI. It enables data scientists and ML engineers to define, compile, schedule, and monitor multi-step ML workflows as directed acyclic graphs (DAGs). Pipelines are executed via Argo Workflows as the underlying workflow engine, with each step running as a container on Kubernetes.

The system consists of several cooperating components: an API server providing REST and gRPC interfaces for pipeline CRUD operations, a persistence agent that synchronizes workflow state to the database, a scheduled workflow controller for cron-based recurring runs, a cache server for task output caching via a mutating admission webhook, and driver/launcher binaries that execute within pipeline step pods to manage artifact I/O and metadata tracking. A React-based frontend UI is also provided. All Go binaries are built with FIPS-compliant cryptography (`GOEXPERIMENT=strictfipsruntime`) for RHOAI distribution.

The platform stores pipeline definitions and run metadata in a relational database (MySQL or PostgreSQL) and pipeline artifacts in S3-compatible object storage (e.g., MinIO). ML Metadata (MLMD) is used for artifact and execution lineage tracking. In multi-user mode, Istio service mesh provides mTLS, authorization policies, and virtual services for traffic routing.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| ml-pipeline (API Server) | Go Service | Central API server exposing REST (8888/TCP) and gRPC (8887/TCP) endpoints for pipeline, run, experiment, and artifact management. Also serves admission webhooks (8443/TCP) for PipelineVersion validation/mutation. |
| ml-pipeline-persistenceagent | Go Agent | Watches Argo Workflow CRs and ScheduledWorkflow CRs, synchronizes execution state to the API server database via gRPC/REST calls. |
| ml-pipeline-scheduledworkflow | Go Controller | Kubernetes controller that reconciles ScheduledWorkflow CRs to create Argo Workflow instances on cron/periodic schedules. |
| cache-server | Go Webhook Server | Mutating admission webhook (443/TCP -> 8443/TCP) that intercepts pod creation to check task output cache, skipping redundant executions. |
| ml-pipeline-driver | Go Binary | V2 pipeline step driver, runs within Argo Workflow step pods to resolve inputs, manage caching decisions, and create sub-DAGs. |
| ml-pipeline-launcher | Go Binary | V2 pipeline step launcher, runs within Argo Workflow step pods to execute user containers, upload artifacts, and record outputs. |
| ml-pipeline-ui | React/TypeScript Frontend | Web UI (3000/TCP) for visualizing pipelines, runs, experiments, and artifacts. Served behind a Node.js Express server. |
| ml-pipeline-visualizationserver | Python Service | Generates pipeline run visualizations (8888/TCP), used by the UI for rendering experiment results. |
| ml-pipeline-viewer-crd | Go Controller | Reconciles Viewer CRs to manage visualization server deployments for specific pipeline outputs. |
| metadata-grpc-server | Go/gRPC Service | ML Metadata (MLMD) gRPC server (8080/TCP) for artifact and execution lineage tracking. |
| metadata-envoy | Envoy Proxy | Envoy sidecar (9090/TCP) proxying gRPC-Web requests to the MLMD gRPC server. |
| cache-deployer | Go Job | Deployment job that sets up the cache webhook's MutatingWebhookConfiguration and TLS certificates. |
| pipelines-profile-controller | Go Controller | Multi-user mode controller that manages per-namespace pipeline resources (kfp-launcher ConfigMaps, secrets, RBAC) for Kubeflow profiles. |
