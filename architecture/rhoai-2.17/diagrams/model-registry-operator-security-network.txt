═══════════════════════════════════════════════════════════════════════════
NETWORK FLOWS
═══════════════════════════════════════════════════════════════════════════

WITHOUT ISTIO (Cluster-Internal Only):
─────────────────────────────────────────────────────────────────────────
Client → Model Registry Service:8080 (HTTP) or :9090 (gRPC)
  - Encryption: None
  - Auth: None (unprotected cluster-internal access)
  - Exposure: ClusterIP only (no external access)

WITH ISTIO (External + Internal Access):
─────────────────────────────────────────────────────────────────────────
External Client → Istio Gateway:443 (HTTPS) → VirtualService →
  → Authorino (TokenReview + RBAC check) →
  → Envoy Sidecar (mTLS) → Model Registry Container:8080/9090 →
  → Database:5432/3306 (TLS optional)

WITH OPENSHIFT ROUTES (Alternative External Access):
─────────────────────────────────────────────────────────────────────────
External Client → OpenShift Router:443 (HTTPS edge termination) →
  → Model Registry Service:8080/9090 (HTTP) →
  → Model Registry Container → Database (TLS optional)

Note: OpenShift Routes provide TLS edge termination but no built-in
      authentication (unlike Istio+Authorino integration)
