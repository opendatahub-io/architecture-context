Trust Boundaries:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. EXTERNAL → CONTROL PLANE
   - User credentials validated by OpenShift/K8s API Server
   - TLS 1.2+ encryption on all external traffic

2. CONTROL PLANE → OPERATOR
   - Webhook calls use mTLS (service CA)
   - Watch API is internal Kubernetes communication

3. OPERATOR → CONTROL PLANE
   - Service account token authentication
   - TLS 1.2+ encryption

4. OPERATOR → APPLICATION NAMESPACES
   - Resources created via Kubernetes API with service account authorization
   - RBAC enforces least privilege

5. MONITORING → OPERATOR
   - Service account token for metrics scraping
   - NetworkPolicy restricts access to monitoring namespaces

6. USER → MONITORING
   - OAuth authentication via OpenShift
   - Re-encrypt TLS on routes
   - OAuth proxy validates tokens before forwarding to Prometheus/Alertmanager
