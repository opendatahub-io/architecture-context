SECURITY RECOMMENDATIONS
═══════════════════════════════════════════════════════════════════════════════════════

1. Network Policies
   - Implement NetworkPolicies to restrict Ray cluster communication to authorized pods
   - Default deny ingress/egress, allow only necessary connections

2. Ingress Authentication
   - Secure Ray Dashboard with OAuth/OIDC proxy (e.g., oauth2-proxy)
   - Do NOT expose Ray Dashboard publicly without authentication

3. Ray Cluster mTLS
   - Enable mutual TLS for Ray cluster internal communication in production
   - Configure via RayCluster spec: rayStartParams.tls_server_cert, tls_server_key

4. Image Security
   - Verify Ray container image signatures before deployment
   - Use private registry with pull secrets for controlled image distribution
   - Scan images for vulnerabilities (CVE scanning)

5. Resource Limits
   - Set appropriate CPU/memory limits to prevent resource exhaustion
   - Configure pod resource quotas per namespace

6. RBAC
   - Restrict user permissions to create RayCluster CRDs in multi-tenant environments
   - Use namespace-scoped RoleBindings instead of ClusterRoleBindings where possible

7. Secrets Management
   - Rotate image pull secrets regularly
   - Use Kubernetes External Secrets or Vault for sensitive credentials
   - Enable automatic ServiceAccount token rotation

8. Audit Logging
   - Enable Kubernetes audit logging for Ray CRD operations
   - Monitor operator logs for suspicious activity
