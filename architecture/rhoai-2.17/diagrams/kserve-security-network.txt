DEPLOYMENT MODES
═══════════════════════════════════════════════════════════════════════════

1. Serverless Mode (Default)
   ├─ Uses: Knative Serving for scale-to-zero autoscaling
   ├─ Components: Knative Activator, Queue Proxy, Istio
   ├─ Autoscaling: Request-based (KPA - Knative Pod Autoscaler)
   ├─ Cold Start: 2-5 seconds (model loading time)
   └─ Use Case: Variable traffic, cost optimization

2. RawDeployment Mode
   ├─ Uses: Standard Kubernetes Deployment + HPA
   ├─ Components: Kubernetes Deployment, Service, HPA
   ├─ Autoscaling: CPU/Memory-based (HPA)
   ├─ Cold Start: None (always running)
   └─ Use Case: Consistent traffic, low latency requirements

3. ModelMesh Mode
   ├─ Uses: High-density multi-model serving
   ├─ Components: ModelMesh controller, model router
   ├─ Autoscaling: Not supported (fixed pods)
   ├─ Model Loading: Dynamic (LRU cache)
   └─ Use Case: Large number of small models, frequent model updates

═══════════════════════════════════════════════════════════════════════════
THREAT MODEL & MITIGATIONS
═══════════════════════════════════════════════════════════════════════════

Threat: Unauthorized access to inference endpoints
├─ Mitigation: Optional JWT authentication via Istio + Authorino
├─ Mitigation: Istio AuthorizationPolicy per InferenceService
└─ Status: User-configurable (opt-in)

Threat: Model artifact tampering during download
├─ Mitigation: TLS 1.2+ for S3/GCS/Azure connections
├─ Mitigation: Cloud provider IAM authentication
└─ Status: Enforced by default

Threat: Man-in-the-middle attacks on inference requests
├─ Mitigation: TLS 1.3 at Istio Gateway (external traffic)
├─ Mitigation: mTLS STRICT within service mesh (internal traffic)
└─ Status: Enforced when Istio is enabled

Threat: Privilege escalation via controller RBAC
├─ Mitigation: Least-privilege RBAC (kserve-manager-role)
├─ Mitigation: Namespace-scoped permissions where possible
└─ Status: Reviewed and audited

Threat: Webhook certificate expiration
├─ Mitigation: cert-manager auto-rotation (90-day certificates)
├─ Mitigation: Monitoring and alerting on cert expiry
└─ Status: Automated

Threat: Storage credential exposure
├─ Mitigation: Kubernetes Secret encryption at rest
├─ Mitigation: RBAC restricts secret access to init containers only
└─ Status: Best practice (manual rotation required)
