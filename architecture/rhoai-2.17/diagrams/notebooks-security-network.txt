RBAC Summary:
- No ClusterRoles/RoleBindings defined in this component
- RBAC managed by odh-notebook-controller and OpenShift
- Workbench pods run with dynamically created ServiceAccounts
- Access control configured by notebook controller based on user identity

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription for RStudio BuildConfig, Admin, No rotation
- aws-connection-* (Opaque) - S3 credentials for data access, User/Admin, No rotation
- git-credentials (kubernetes.io/basic-auth) - Git repository access, User, No rotation
- notebook-oauth-config (Opaque) - OAuth configuration, Notebook Controller, No rotation

Network Policies:
- Not defined in this component
- Network isolation managed by OpenShift NetworkPolicy (if configured)

Image Security:
- Base Images: UBI9/RHEL9/CentOS Stream 9 (security-maintained)
- Weekly security patch updates (PATCH version)
- Images built with locked dependencies (Pipfile.lock)
- Image provenance tracked via commit.env ConfigMap

Authentication Flow:
1. User authenticates to OpenShift (OAuth/OIDC)
2. OAuth proxy validates token and forwards to workbench
3. JupyterLab/RStudio/Code Server validates token or uses basic auth
4. Token-based auth: Per-workbench token generated on spawn
5. OAuth integration: Configured by notebook controller

Build Security (RStudio BuildConfig):
- Source: GitHub public repository (HTTPS/443)
- Base Image Pull: Authenticated with pull secret
- Output Push: Service account authentication to internal registry
- RHEL Subscription: Required secret for RHEL builds
