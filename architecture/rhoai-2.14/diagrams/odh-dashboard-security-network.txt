AUTHENTICATION & AUTHORIZATION FLOWS
═══════════════════════════════════════════════════════════════════════════

Endpoint: / (Frontend SPA)
├─ Auth: OAuth Bearer Token (via oauth-proxy)
├─ Enforcement: oauth-proxy validates OAuth cookie
└─ Policy: Authenticated users only (OpenShift login required)

Endpoint: /api/* (Backend API)
├─ Auth: OAuth Bearer Token (header: X-Forwarded-Access-Token)
├─ Enforcement: oauth-proxy validates token, backend trusts proxy
└─ Policy: User must have project list permission in OpenShift

Endpoint: /api/dashboardConfig (Admin-only)
├─ Auth: OAuth Bearer Token + Admin Group Membership
├─ Enforcement: Backend (secureAdminRoute middleware)
└─ Policy: User must be in admin groups defined in OdhDashboardConfig CR

Endpoint: /api/k8s/* (Kubernetes API passthrough)
├─ Auth: OAuth Bearer Token + K8s RBAC
├─ Enforcement: Kubernetes API Server
└─ Policy: Per-resource RBAC (user permissions apply)

Endpoint: /wss/k8s/* (WebSocket for K8s events)
├─ Auth: OAuth Bearer Token + K8s RBAC
├─ Enforcement: Kubernetes API Server
└─ Policy: Per-resource RBAC (watch permission required)

Endpoint: /metrics (Prometheus scraping)
├─ Auth: None (skip-auth-regex bypass)
├─ Enforcement: oauth-proxy skips authentication
└─ Policy: Unauthenticated access for Prometheus scraper

Endpoint: /api/health, /oauth/healthz (Health checks)
├─ Auth: None
├─ Enforcement: None
└─ Policy: Unauthenticated for cluster health monitoring
