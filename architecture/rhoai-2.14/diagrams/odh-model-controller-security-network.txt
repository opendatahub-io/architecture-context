SECRETS AND CERTIFICATES
══════════════════════════════════════════════════════════════════════════════════════════

Secret: odh-model-controller-webhook-cert
  - Type: kubernetes.io/tls
  - Purpose: TLS certificate for ValidatingWebhook server (9443/TCP)
  - Provisioned By: OpenShift Service CA Operator
  - Auto-Rotate: Yes (Service CA managed)
  - Rotation Period: 90 days (default Service CA)
  - Mounted to: odh-model-controller pod

Secret: storage-config
  - Type: Opaque
  - Purpose: Aggregated S3 storage credentials for model loading
  - Contents: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, endpoint URLs
  - Provisioned By: StorageSecret Controller (aggregates data connection secrets)
  - Auto-Rotate: On data connection secret change
  - Mounted to: Inference service pods

Secret: kserve-custom-ca-cert
  - Type: Opaque
  - Purpose: Custom CA certificates for external S3/services
  - Provisioned By: KServeCustomCACert Controller
  - Source: User-provided ConfigMap with CA bundle
  - Auto-Rotate: On source ConfigMap change
  - Mounted to: Inference service pods

ConfigMap: auth-refs (Controller Namespace)
  - Keys: AUTH_AUDIENCE, AUTHORINO_LABEL
  - Provisioned By: opendatahub-operator
  - Purpose: Configure Authorino integration

ConfigMap: service-mesh-refs (Controller Namespace)
  - Keys: CONTROL_PLANE_NAME, MESH_NAMESPACE
  - Provisioned By: opendatahub-operator
  - Purpose: Service mesh enrollment configuration

DEPLOYMENT MODES SUPPORTED

   - Resources Created: Knative Service, Route, Gateway, VirtualService

   - Dependencies: Knative Serving v0.39.3, Istio v1.19.4

   - Resources Created: Deployment, Service, Route, Gateway, VirtualService

   - Scale to Zero: No

   - Resources Created: Deployment, Service, Route, ClusterRoleBinding
   - Multi-model Serving: Yes
   - Dependencies: ModelMesh Serving
