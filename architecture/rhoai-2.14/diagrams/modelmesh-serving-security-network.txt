NETWORK POLICIES:

1. modelmesh-controller NetworkPolicy:
   - Selector: control-plane=modelmesh-controller
   - Ingress: Allow 8443/TCP from all (metrics)
   - Ingress: Allow 9443/TCP from Kubernetes API server (webhook)

2. modelmesh-runtimes NetworkPolicy:
   - Selector: modelmesh-service=*
   - Ingress: Allow 8033/TCP, 8080/TCP from modelmesh pods (internal coordination)
   - Ingress: Allow 8033/TCP, 8008/TCP, 2112/TCP from all (inference, metrics)

3. modelmesh-webhook NetworkPolicy:
   - Selector: control-plane=modelmesh-controller
   - Ingress: Allow 9443/TCP from Kubernetes API server

SECRETS:

1. model-serving-etcd (Opaque)
   - Purpose: etcd connection config and credentials
   - Keys: etcd-hosts, etcd-username, etcd-password
   - Provisioned: User/Operator
   - Auto-Rotate: No
   - Mount: Controller and Runtime pods

2. storage-config (Opaque)
   - Purpose: Model storage credentials (S3, MinIO)
   - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, S3_ENDPOINT
   - Provisioned: User/Operator
   - Auto-Rotate: No
   - Mount: Storage Helper init containers

3. modelmesh-webhook-server-cert (kubernetes.io/tls)
   - Purpose: Webhook server TLS certificate
   - Keys: tls.crt, tls.key
   - Provisioned: cert-manager or Controller
   - Auto-Rotate: Yes (if cert-manager)
   - Mount: Controller webhook server

SERVICE MESH CONFIGURATION:

- PeerAuthentication: Varies by deployment (PERMISSIVE or STRICT)
- AuthorizationPolicy: Application-defined (delegates to service mesh)
- Inference endpoints (8033, 8008): No built-in auth (relies on NetworkPolicy,
  service mesh, or application-level auth)

DATA AT REST:

- Model artifacts in S3: Server-side encryption (SSE-S3, SSE-KMS)
- etcd data: Unencrypted by default (can enable encryption-at-rest)
- PersistentVolumes: Depends on storage class encryption settings

DATA IN TRANSIT:

- External → Gateway: TLS 1.3 (HTTPS/gRPC)
- Gateway → Services: Plaintext HTTP/gRPC (cluster-internal)
- Services → etcd: Plaintext HTTP (can enable mTLS)
- Services → S3: TLS 1.2+ HTTPS (production) or HTTP (MinIO dev)
- Controller → Kubernetes API: TLS 1.2+
- Prometheus → Metrics: TLS 1.2+ (8443) or HTTP (2112)
