RBAC Summary:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
No ClusterRoles or RoleBindings defined by this component.

RBAC is managed by the ODH Notebook Controller, which creates:
- ServiceAccount for each workbench instance
- RoleBinding granting workbench SA permissions to access PVC
- RoleBinding for OAuth proxy to validate user identity

Policy Enforcement:
- OAuth Proxy ensures user owns the notebook instance (namespace + username match)
- OpenShift RBAC controls which users can create/delete notebook instances
- Jupyter token validation is optional (disabled by default in ODH deployments)

Secrets Management:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
No secrets are created or managed by this component.

Secrets are injected by ODH Notebook Controller:
- Object storage credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
- OAuth tokens for external service integration
- Git credentials for repository access
- Custom environment variables from ConfigMaps/Secrets

Secrets are mounted as:
- Environment variables: Injected into pod spec
- Files: Mounted from secret volumes (e.g., /opt/app-root/src/.aws/credentials)

Security Features:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓ Non-root containers (UID 1001, GID 0)
✓ OpenShift restricted SCC compatible
✓ No privileged operations required
✓ No host path mounts
✓ No host network access
✓ TLS termination at ingress (edge/re-encrypt routes)
✓ OAuth-based user authentication
✓ Namespace isolation (one workbench per user namespace)
✓ Read-only container filesystem (except /opt/app-root/src from PVC)
✓ Arbitrary UID support for OpenShift (root group GID 0)

Image Variants by Security Posture:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
All images share the same security posture:
- Base: Red Hat Universal Base Image 9 (UBI9) or CentOS Stream 9 (C9S)
- Security updates: Automated via GitHub Actions (digest updater)
- CVE scanning: Quay.io security scanner
- Minimal attack surface: Only necessary tools installed
- Hardware accelerator images: Include vendor drivers but maintain non-root execution
