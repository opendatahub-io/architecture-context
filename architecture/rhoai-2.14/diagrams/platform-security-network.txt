PeerAuthentication:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Mode: PERMISSIVE (namespace-level, allows both mTLS and plaintext)
- mTLS Mode: ISTIO_MUTUAL (default for service-to-service communication)
- Scope: Inference service namespaces
- Reason: Backwards compatibility with non-mesh workloads

Traffic Management:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- VirtualService: Routes for KServe InferenceServices, ModelMesh, Model Registry
- Gateway: External ingress (443/TCP → 8080/TCP internal)
- Created by: odh-model-controller

Authorization:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Provider: Authorino (external authorization)
- AuthConfig: Created by odh-model-controller for InferenceServices
- Enforcement: Bearer token validation (Kubernetes ServiceAccount JWT)
- Optional: Can be disabled for unauthenticated inference

Sidecar Injection:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Method: Selective (annotation-based)
- Components using mesh: KServe InferenceServices, ModelMesh, Model Registry
- Components NOT using mesh: Notebooks, DSP, Training jobs, Ray clusters

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Allow ingress from openshift-monitoring namespace (Prometheus scraping)
- Allow ingress from openshift-user-workload-monitoring namespace (metrics)
- Allow ingress from openshift-ingress namespace (Route traffic)
- Allow ingress from namespaces with label opendatahub.io/generated-namespace=true
- Deny all other ingress by default

Component-Specific Policies:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Notebooks: Allow 8888/TCP from controller namespace, 8443/TCP for OAuth proxy
- Ray Clusters: Allow 8265/TCP (dashboard), 10001/TCP (client) from same namespace
- ModelMesh: Allow 8033/TCP (gRPC), 8008/TCP (REST) from all, 2112/TCP (metrics)
- KServe: Managed by odh-model-controller based on deployment mode

SECURITY POSTURE
════════════════════════════════════════════════════════════════════════════════════════

✅ Non-root containers across all components
✅ RBAC enforced for all API operations
✅ TLS for external ingress (OpenShift Routes, Istio Gateways)
✅ Optional mTLS for service-to-service (Istio Service Mesh)
✅ OpenShift SCC compliance (restricted)
✅ NetworkPolicies created for traffic isolation
⚠️  Secrets rotation varies (auto for service-ca, manual for user secrets)
⚠️  mTLS enforcement: PERMISSIVE mode (not STRICT) for backwards compatibility
⚠️  Some external connections may use plaintext (MinIO HTTP, etcd)
