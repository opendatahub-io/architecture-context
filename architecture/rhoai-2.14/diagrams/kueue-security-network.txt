DATA FLOW SUMMARY
═══════════════════════════════════════════════════════════════════════════

1. Job Submission Flow:
   User → K8s API (6443 HTTPS TLS1.2+ Bearer)
       → Webhook (9443 HTTPS TLS1.2+ K8s mTLS)
       → Controller watches via K8s API
       → Creates Workload, evaluates queue
       → Admits or enqueues job

2. Metrics Collection:
   Prometheus → metrics-service (8080 HTTP plaintext)

3. Visibility API:
   User → K8s API (6443 HTTPS TLS1.2+ Bearer)
       → visibility-server (8082 HTTPS TLS1.2+ K8s mTLS)

4. Cluster Autoscaling:
   Controller → K8s API (6443 HTTPS TLS1.2+ ServiceAccount)
       → ProvisioningRequest creation
       → Cluster Autoscaler watches and provisions nodes
