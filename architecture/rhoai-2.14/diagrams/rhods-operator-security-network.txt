TRUST BOUNDARIES
═══════════════════════════════════════════════════════════════════════════════════════════

1. External → Kubernetes API (6443/TCP):
   - TLS 1.3 encryption
   - Authentication: kubeconfig credentials (user certificates, tokens)
   - Authorization: Kubernetes RBAC

2. API Server → Webhook (9443/TCP):
   - TLS 1.2+ encryption (service-ca certificate)
   - Authentication: API Server client certificate (mTLS)
   - Trust: Only API Server can call webhooks

3. Operator → API Server (6443/TCP):
   - TLS 1.3 encryption
   - Authentication: ServiceAccount JWT token (controller-manager)
   - Authorization: ClusterRole controller-manager-role

4. Operator → External Services (443/TCP):
   - github.com: TLS 1.2+, no authentication (public repos)
   - quay.io: TLS 1.2+, no authentication (public images)
   - Trust: Public internet, verified by TLS certificates

5. Prometheus → Metrics Endpoint (8443/TCP):
   - TLS 1.2+ encryption
   - Authentication: ServiceAccount Bearer Token
   - Authorization: kube-rbac-proxy validates RBAC (get /metrics)
   - NetworkPolicy: Only monitoring namespaces allowed
