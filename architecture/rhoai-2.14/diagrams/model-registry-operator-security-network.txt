Network Policies:
- Not explicitly configured by operator
- Relies on Istio service mesh for mTLS and authorization
- Internal traffic (without Istio) is unencrypted within cluster network
- External access requires Istio Gateway + Authorino OR OpenShift Route

Trust Boundaries:
1. EXTERNAL → INGRESS: TLS 1.2+, Bearer token authentication
2. INGRESS → SERVICE MESH: mTLS (if Istio ISTIO_MUTUAL), Authorino authorization
3. SERVICE MESH → POD: Internal HTTP/gRPC, no encryption (localhost communication)
4. POD → DATABASE: TLS 1.2+ (configurable), username/password authentication
