SECRETS:

1. odh-notebook-controller-webhook-cert (kubernetes.io/tls)
   - Purpose: TLS certificate for webhook server (8443/TCP)
   - Provisioned By: OpenShift Service CA Operator
   - Auto-Rotate: Yes (Service CA handles rotation)
   - Mount: /tmp/k8s-webhook-server/serving-certs/

2. {notebook-name}-oauth-config (Opaque)
   - Purpose: OAuth proxy cookie encryption secret
   - Contents: cookie_secret (base64-encoded random 32 bytes)
   - Provisioned By: ODH Notebook Controller (on Notebook creation)
   - Auto-Rotate: No (static unless regenerated)
   - Mount: /etc/oauth/config/cookie_secret

3. {notebook-name}-tls (kubernetes.io/tls)
   - Purpose: TLS certificate for {notebook}-tls Service
   - Provisioned By: OpenShift Service CA (via annotation trigger)
   - Annotation: service.beta.openshift.io/serving-cert-secret-name={notebook-name}-tls
   - Auto-Rotate: Yes (Service CA handles rotation)
   - Mount: /etc/tls/private/ (tls.crt, tls.key)

SERVICE MESH INTEGRATION (Optional):

When annotation `opendatahub.io/service-mesh=true` is set:
  - Mutually exclusive with OAuth injection
  - Istio sidecars injected for mTLS
  - NetworkPolicy for OAuth not created
  - Service Mesh handles traffic routing and encryption

ERROR: Cannot have both `opendatahub.io/service-mesh=true` AND
       `notebooks.opendatahub.io/inject-oauth=true` - webhook rejects with error

OBSERVABILITY:

- Logging: Structured JSON (zap), RFC3339 timestamps, controlled by --debug-log flag
- Metrics: Prometheus format at /metrics (8080/TCP HTTP)
- Health: /healthz (8081/TCP) - Liveness probe (15s initial delay, 20s period)
- Health: /readyz (8081/TCP) - Readiness probe (5s initial delay, 10s period)

LEADER ELECTION (when --leader-elect=true):

- Leader Election ID: odh-notebook-controller
- Lock Type: ConfigMap or Lease (coordination.k8s.io/v1)
- Namespace: Controller deployment namespace
- Only active leader reconciles; standby replicas wait

DEPENDENCIES:

External (Required):
  - Kubeflow Notebook Controller v1.27.0+ (creates StatefulSets)
  - OpenShift Route API (route.openshift.io/v1)
  - OpenShift OAuth Proxy image (ose-oauth-proxy@sha256:4bef31eb993feb6)
  - OpenShift Service CA Operator (OpenShift 4.x) - for TLS cert generation
  - OpenShift OAuth Server (OpenShift 4.x) - for user authentication

Internal ODH (Optional):
  - ODH Dashboard - links to notebook Routes
  - Service Mesh (Istio) - optional, via opendatahub.io/service-mesh annotation
