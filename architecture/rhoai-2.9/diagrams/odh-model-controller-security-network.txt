Notes
═════════════════════════════════════════════════════════════════════════════════════════

Deployment Modes:
The controller supports three InferenceService deployment modes:
1. ModelMesh: Multi-model serving with shared runtime pools
2. KServe Serverless: Knative-based autoscaling (scale-to-zero)
3. KServe RawDeployment: Direct Kubernetes Deployments (no Knative)

Each mode has different networking and security configurations.

Service Mesh Integration:
- Controlled by DataScienceCluster CR and MESH_DISABLED env var
- When enabled: Creates PeerAuthentication, VirtualServices, Telemetry
- Authorization (Authorino) is optional and detected via capability check

OpenShift-Specific Features:
- Routes (not Ingress) for external access
- Service serving certificates for webhook TLS
- Red Hat Service Mesh (Maistra) integration

Security Posture:
- Controller runs as non-root (UID 65532)
- SecurityContext prevents privilege escalation
- mTLS enforced in service mesh (STRICT mode)
- Optional JWT authentication via Authorino
- Network policies for pod isolation
- RBAC scoped to specific resources and verbs
