Security Considerations:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚠️  Ray Dashboard has NO built-in authentication
    → Must secure via network policies or external auth proxy
    → Consider deploying OAuth2 Proxy or similar

⚠️  Ray internal communication is NOT encrypted by default
    → GCS (6379/TCP): plaintext Redis protocol
    → Ray Client (10001/TCP): unencrypted TCP
    → Consider Service Mesh (Istio) for mTLS

⚠️  GCS Redis password is OPTIONAL but RECOMMENDED for production
    → Configure via {raycluster}-redis-secret

⚠️  Ray Serve has application-level auth only
    → Configure API key authentication in application code
    → Use TLS termination at Route/Ingress level

⚠️  Route/Ingress should use TLS termination
    → Configure TLS certificates for external endpoints
    → Use edge or reencrypt TLS termination mode
