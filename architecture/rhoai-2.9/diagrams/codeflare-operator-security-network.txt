NetworkPolicy Configuration:

  1. NetworkPolicy: {cluster}-head (per RayCluster)
     Pod Selector: ray.io/cluster={cluster}, ray.io/node-type=head
     Ingress Rules:
       - From same Ray cluster pods: All ports, all protocols
       - From any pod in same namespace: 10001/TCP (RayClient), 8265/TCP (Dashboard)
       - From KubeRay operator (app.kubernetes.io/component=kuberay-operator):
         Ports 8265/TCP, 10001/TCP
         Namespaces: opendatahub, redhat-ods-applications, or DSCInitialization.spec.applicationsNamespace
       - From OpenShift monitoring (namespace: openshift-monitoring): Port 8080/TCP
       - OAuth/mTLS secured: 8443/TCP (OAuth), 10001/TCP (mTLS)

  2. NetworkPolicy: {cluster}-workers (per RayCluster)
     Pod Selector: ray.io/cluster={cluster}, ray.io/node-type=worker
     Ingress Rules:
       - From same Ray cluster pods (ray.io/cluster={cluster}): All ports, all protocols

     - Purpose: OAuth proxy service TLS certificate

  4. ca-secret-{cluster} (Opaque) - per RayCluster
     - Purpose: Ray cluster CA private key and certificate for mTLS

Authentication Methods:

  1. Webhook Endpoints (/mutate-ray-io-v1-raycluster, /validate-ray-io-v1-raycluster)
     - Method: K8s API Server mTLS
     - Enforcement: Kubernetes API Server
     - Policy: K8s RBAC

  2. Ray Dashboard (via OAuth proxy)
     - Method: OpenShift OAuth
     - Enforcement: OAuth proxy sidecar
     - Policy: OpenShift RBAC (delegate to pod get verb)

  3. RayClient (mTLS)
     - Method: mTLS client certificates
     - Enforcement: Ray head node
     - Policy: Ray cluster CA certificate validation

  4. Metrics Endpoint (/metrics)
     - Method: Bearer token (when ServiceMonitor enabled)
     - Enforcement: ServiceMonitor configuration
     - Policy: OpenShift monitoring RBAC

  5. Health Endpoints (/healthz, /readyz)
     - Method: None (open for kubelet probes)
     - Enforcement: None
     - Policy: None
