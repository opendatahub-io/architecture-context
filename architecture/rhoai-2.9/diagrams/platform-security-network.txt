RBAC SUMMARY
═════════════════════════════════════════════════════════════════════════════════════

ClusterRoles (selected):
- controller-manager-role (RHODS Operator): Full DSC/DSCI/component CRD management
- odh-dashboard: Read nodes/configmaps, manage namespaces, read component CRs
- kserve-manager-role: Manage InferenceServices, Knative, Istio resources
- odh-model-controller-role: Manage Routes, Istio, Authorino, NetworkPolicies
- modelmesh-controller-role: Manage Predictors, ServingRuntimes, Deployments, HPAs
- manager-role (DSP, CodeFlare, TrustyAI): Component-specific resource management
- kueue-manager-role: Manage ClusterQueues, LocalQueues, Workloads, all job types
- kuberay-operator: Manage RayClusters, RayJobs, RayServices

Service Mesh Configuration:
- PeerAuthentication: STRICT (when Service Mesh enabled)
- Applies to: KServe InferenceServices, ModelMesh serving pods, DSP components
- Certificate Management: Istio Citadel (auto-rotation, workload identity)
- AuthorizationPolicy: Authorino integration (optional) for InferenceService auth

Secrets Rotation:
- OAuth proxy TLS: service-ca operator (auto-rotation)
- Webhook server certs: cert-manager (90-day rotation)
- S3 credentials: Manual rotation required
- Database passwords: Manual rotation required
- Service Mesh certs: Istio Citadel (24-hour default rotation)

NetworkPolicies:
- CodeFlare creates NetworkPolicies for Ray clusters (ingress/egress control)
- ODH Model Controller can create NetworkPolicies for InferenceServices
- Default: Kubernetes CNI allows all traffic (NetworkPolicies opt-in)

Security Context Constraints (OpenShift SCCs):
- Ray pods: anyuid SCC (required for Ray user UID 1000)
- Notebook pods: restricted SCC (default, non-root UID)
- InferenceService pods: restricted SCC
- ModelMesh pods: restricted SCC

Trust Boundaries:
1. External ↔ Ingress: Public internet to OpenShift Routes (TLS termination)
2. Ingress ↔ Platform Services: Route → OAuth proxy → Application (HTTPS/reencrypt)
3. Platform Services ↔ Operators: OAuth/SA tokens → Kubernetes API (mTLS)
4. Operators ↔ Workloads: API calls via ServiceAccount tokens (RBAC enforced)
5. Service Mesh (Internal): mTLS STRICT between mesh members (trusted zone)
6. Workloads ↔ External Services: TLS 1.2+ (S3, databases, registries)

MONITORING & OBSERVABILITY
═════════════════════════════════════════════════════════════════════════════════════

Metrics Collection:
- All operators expose :8080/metrics (Prometheus format)
- ServiceMonitor CRs auto-discovered by Prometheus Operator
- Platform Prometheus: redhat-ods-monitoring namespace
- Cluster Prometheus: openshift-monitoring namespace (aggregation)

Health Probes:
- Liveness: /healthz (all operators)
- Readiness: /readyz (all operators)
- Startup probes: Optional for slow-starting workloads

Logging:
- Structured JSON logging (all operators, controller-runtime)
- Centralized: OpenShift Logging (EFK/ELK stack)
- Log levels: Configurable via operator flags

Distributed Tracing:
- Service Mesh: Jaeger integration (optional)
- Trace context propagation: Istio Envoy sidecars
