SERVICE MESH CONFIGURATION
═══════════════════════════════════════════════════════════════════════════

Note: ModelMesh Serving does NOT use Istio/OpenShift Service Mesh by default.
All communication within runtime pods is unencrypted (internal pod network).
External access is secured via OpenShift Router (TLS edge termination) and OAuth Proxy.

Authentication Points:
  1. External → Router: TLS 1.3 (public certificate)
  2. Router → OAuth Proxy: TLS 1.2+ (internal certificate)
  3. OAuth Proxy → OpenShift OAuth: OAuth 2.0 token validation
  4. OAuth Proxy → REST Proxy: No authentication (same pod)
  5. REST Proxy → ModelMesh: No authentication (same pod)
  6. ModelMesh → Model Server: No authentication (same pod)

Inter-Pod Communication:
  - ModelMesh pods communicate via port 8080/TCP (gRPC, no encryption)
  - Network policies restrict access to modelmesh-service labeled pods

Controller → API Server:
  - Port: 6443/TCP
  - Protocol: HTTPS
  - Encryption: TLS 1.3
  - Auth: ServiceAccount Token (modelmesh-controller)

Webhook Server ← API Server:
  - Port: 9443/TCP
  - Protocol: HTTPS
  - Encryption: TLS 1.2+ (mTLS)
  - Auth: API server validates webhook server certificate
