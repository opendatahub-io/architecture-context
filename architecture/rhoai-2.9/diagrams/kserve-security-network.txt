═══════════════════════════════════════════════════════════════════════════════════
THREAT MODEL & MITIGATIONS
═══════════════════════════════════════════════════════════════════════════════════

Threat: Unauthorized access to inference endpoints
├── Impact: Data exfiltration, model theft
└── Mitigations:
    ├── mTLS (STRICT) enforced by Istio service mesh
    ├── Optional OAuth2/OIDC token validation via Authorino
    ├── Namespace isolation via Kubernetes RBAC
    └── Network policies restricting ingress to Istio Gateway only

Threat: Model poisoning via malicious storage
├── Impact: Serving compromised models, backdoor attacks
└── Mitigations:
    ├── TLS 1.2+ for all storage communication
    ├── Signature verification (AWS SigV4, GCP SA)
    ├── Immutable model storage (S3 versioning, retention policies)
    └── RBAC controls on storage-config secret access

Threat: Privilege escalation via webhook compromise
├── Impact: Cluster-wide resource manipulation
└── Mitigations:
    ├── cert-manager automatic certificate rotation (90 days)
    ├── Webhook server isolated in kserve namespace
    ├── API Server mTLS authentication for webhook calls
    └── Minimal RBAC permissions for controller ServiceAccount

Threat: Container escape from inference pods
├── Impact: Node compromise, data access
└── Mitigations:
    ├── Non-root runtime containers (SecurityContext)
    ├── ReadOnlyRootFilesystem enabled
    ├── AppArmor/SELinux profiles (OpenShift)
    └── Pod Security Standards (Restricted)

Threat: Secret exposure in logs or events
├── Impact: Storage credential theft
└── Mitigations:
    ├── Secrets mounted as volumes (not env vars)
    ├── Structured logging without secret fields
    ├── RBAC limiting secret access to authorized SAs
    └── Secret encryption at rest (etcd encryption)
