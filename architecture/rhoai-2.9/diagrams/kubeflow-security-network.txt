═══════════════════════════════════════════════════════════════════════════════════
Security Concerns & Recommendations
═══════════════════════════════════════════════════════════════════════════════════

⚠️  HIGH PRIORITY:
1. Culling HTTP Security: Culling controller communicates with notebook pods over
   unencrypted HTTP (8888/TCP) without authentication. This exposes:
   - Kernel execution state
   - Terminal activity logs
   - Potential for spoofing/MITM attacks
   Recommendation: Implement mTLS or token-based authentication

2. Metrics Unprotected: /metrics endpoint (8080/TCP) exposed without authentication
   - Leaks notebook creation patterns and usage statistics
   Recommendation: Enable Prometheus auth or restrict via NetworkPolicy

3. RBAC Scope: ClusterRole grants broad permissions (* verbs) on StatefulSets,
   Services, and VirtualServices across ALL namespaces
   Recommendation: Consider namespace-scoped Roles where possible

⚠️  MEDIUM PRIORITY:
4. No mTLS: Internal service-to-service communication does not use mutual TLS
   Recommendation: Enable Istio PeerAuthentication STRICT mode

5. No NetworkPolicy: No NetworkPolicy resources defined - relies on cluster defaults
   Recommendation: Define explicit egress/ingress policies

6. Webhook Disabled: Conversion webhook implemented but disabled in production
   Status: Low risk if not needed, but document reasoning

INBOUND (Controller):
  - Kubernetes API → Controller (Watch events, TLS 1.2+, ServiceAccount Token)
  - Prometheus → Controller:8080 (HTTP, no auth) ⚠️
  - Kubelet → Controller:8081 (HTTP health probes, no auth)

OUTBOUND (Controller):
  - Controller → Kubernetes API:6443 (HTTPS, TLS 1.2+, ServiceAccount Token)
  - Controller → Notebook Pod:8888 (HTTP, no auth) ⚠️

INBOUND (Notebook Pod):
  - Culler → Notebook:8888 (HTTP, no auth) ⚠️
  - External Users → Istio Gateway:443 → VirtualService → Service:80 → Pod:8888
    (TLS terminates at gateway, plaintext internally)

OUTBOUND (Notebook Pod):
  - Pod → Container Registry:443 (HTTPS, TLS 1.2+, ImagePullSecrets)
