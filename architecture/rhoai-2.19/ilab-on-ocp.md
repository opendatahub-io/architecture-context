## Purpose
**Short**: Provides the InstructLab LLM fine-tuning pipeline for Red Hat OpenShift AI, orchestrating synthetic data generation, distributed multi-node GPU training, and model evaluation via Data Science Pipelines.

**Detailed**: The ilab-on-ocp repository is the central location for the InstructLab pipeline that runs on Red Hat OpenShift AI (RHOAI). It implements a complete LLM fine-tuning workflow using Kubeflow Pipelines v2 (KFP), orchestrated by Data Science Pipelines Application (DSPA). The pipeline takes a base LLM model (e.g., Granite), generates synthetic training data using a teacher model (Mixtral) via the InstructLab SDG library, processes the data, runs two-phase distributed training using PyTorchJobs on multi-GPU nodes via the Kubeflow Training Operator, and evaluates the resulting model using MT-Bench and MMLU benchmarks with a judge model (Prometheus).

The pipeline is automatically provisioned by the DataSciencePipelinesApplication (DSPA) resource when InstructLab is enabled. The repository produces a container image (`odh-ml-pipelines-runtime-generic`) that embeds the compiled pipeline YAML and all required Python dependencies. Pipeline tasks run as Kubernetes pods using various container images including RHEL AI (for SDG and training), vLLM (for model serving during evaluation), and toolbox images (for data movement). The trained model can be pushed to an OCI registry as a ModelCar and optionally registered in the RHOAI Model Registry.
