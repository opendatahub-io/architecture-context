## Purpose

**Short**: Kubernetes controller that manages the lifecycle of Jupyter Notebook servers by reconciling Notebook custom resources into StatefulSets, Services, and optionally Istio VirtualServices.

**Detailed**: The Kubeflow Notebook Controller is a controller-runtime based Go operator that watches for `Notebook` custom resources (CRD group `kubeflow.org`) and reconciles them into the underlying Kubernetes primitives needed to run notebook servers. For each Notebook CR, the controller creates a StatefulSet (with a single replica) to run the notebook container and a ClusterIP Service to expose it within the cluster. When Istio service mesh integration is enabled (via the `USE_ISTIO` environment variable), the controller also creates Istio VirtualService resources to route traffic to the notebook through the mesh gateway.

The controller includes an optional idle notebook culling feature (enabled via `ENABLE_CULLING` environment variable) that monitors notebook activity by polling the Jupyter `/api/kernels` and `/api/terminals` endpoints. When a notebook has been idle beyond the configurable `CULL_IDLE_TIME` threshold (default 1440 minutes / 24 hours), the controller sets a `kubeflow-resource-stopped` annotation which causes the StatefulSet replicas to be scaled to zero. The controller also supports notebook restart via the `notebooks.opendatahub.io/notebook-restart` annotation and exposes Prometheus metrics for monitoring notebook creation, failure, and culling events.

This is the upstream Kubeflow notebook controller component. In RHOAI deployments, it works alongside the `odh-notebook-controller` (a separate controller in the same repository) which adds OpenShift-specific functionality such as OAuth proxy injection, Route creation, and NetworkPolicy management.
