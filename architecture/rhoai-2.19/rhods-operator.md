## Purpose
**Short**: The RHOAI Operator (rhods-operator) is the platform operator for Red Hat OpenShift AI, responsible for lifecycle management of all AI/ML platform components on OpenShift.

**Detailed**: The rhods-operator is a Go-based Kubernetes operator built on controller-runtime that serves as the central control plane for Red Hat OpenShift AI (RHOAI). It manages the installation, configuration, upgrade, and lifecycle of all RHOAI components including Dashboard, Workbenches, KServe (model serving), ModelMesh, Data Science Pipelines, CodeFlare, Ray, Kueue, TrustyAI, Model Registry, and Training Operator.

The operator uses two primary custom resources: `DSCInitialization` (DSCI) for platform-level initialization (namespace creation, network policies, Service Mesh configuration, trusted CA bundles, monitoring, and authentication) and `DataScienceCluster` (DSC) for component-level lifecycle management. Each component has a dedicated sub-controller and corresponding component CRD (e.g., `Kserve`, `Dashboard`, `Workbenches`).

For RHOAI 2.x, the operator manages Istio Service Mesh integration for KServe including Istio Gateways, EnvoyFilters, AuthorizationPolicies, and Knative Serving configuration. It uses OpenShift Routes for monitoring and alertmanager ingress, and manages oauth-proxy-based authentication. The operator deploys monitoring infrastructure (Prometheus, Alertmanager, Blackbox Exporter) for managed RHOAI clusters and creates NetworkPolicies to secure cross-namespace communication.
