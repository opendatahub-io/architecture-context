## Purpose
**Short**: Kubeflow Pipelines is an ML workflow orchestration platform that enables building, deploying, and managing reusable end-to-end machine learning pipelines on Kubernetes.

**Detailed**: Data Science Pipelines (based on Kubeflow Pipelines) provides a platform for defining, running, scheduling, and tracking machine learning workflows as directed acyclic graphs (DAGs). The system includes an API server that manages pipeline definitions and runs, a persistence agent that synchronizes workflow state from Kubernetes to the pipeline database, a scheduled workflow controller that handles recurring pipeline executions, and a cache server that enables step-level caching to avoid redundant computations.

The platform supports two workflow execution engines: Argo Workflows and Tekton Pipelines. In RHOAI, the v2 pipeline architecture uses a driver/launcher pattern where the driver container handles DAG orchestration and the launcher container manages individual step execution, artifact I/O, and metadata logging. The API server exposes both gRPC (port 8887) and REST HTTP (port 8888) endpoints with support for v1beta1 and v2beta1 API versions. Pipeline artifacts and metadata are stored in S3-compatible object storage (MinIO or external) and a relational database (MySQL or PostgreSQL).
