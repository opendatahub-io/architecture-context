## Purpose
**Short**: A high-performance text generation inference server (TGIS) for serving large language models (LLMs) with gRPC and HTTP APIs, originally forked from HuggingFace TGI and developed by IBM.

**Detailed**: The Text Generation Inference Server (TGIS) is a production-grade inference runtime for serving large language models. It is an early fork of HuggingFace's text-generation-inference that was developed internally at IBM and has since diverged with independent feature development. The system is designed for high-throughput, low-latency text generation with support for both causal (decoder-only) and sequence-to-sequence (encoder-decoder) model architectures.

The architecture follows a three-tier process model: a **launcher** (Rust) orchestrates the startup of one or more **model server shards** (Python/PyTorch) and a **router** (Rust). The Python server shards load the model weights and perform GPU-accelerated inference via PyTorch, communicating with the router over Unix domain sockets using an internal gRPC protocol. The router handles external client requests via a public gRPC API (the `fmaas.GenerationService` protocol) and an HTTP health/metrics endpoint, performing request validation, tokenization, batching, and response assembly. It supports multiple deployment frameworks including HuggingFace Transformers, TGIS Native (custom tensor parallel), IBM FMS, and ONNX Runtime, along with quantization via GPTQ and bitsandbytes. Key features include continuous batching with dynamic memory scaling, flash attention v2, streaming generation, prompt prefix caching, PyTorch 2 compilation support, and optional TLS with mutual authentication.
