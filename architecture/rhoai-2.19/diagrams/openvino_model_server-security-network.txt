RBAC Summary:
  No RBAC resources defined. OVMS does not interact with the Kubernetes API.
  It runs as a standalone server process.

Authentication & Authorization:
  - gRPC (port 9178): No auth (InsecureServerCredentials)
  - REST (configurable port): No auth
  - Security delegated to platform layer (Istio mTLS, network policies)

Secrets:
  - AWS_ACCESS_KEY_ID (env var) - S3 auth, provisioned by User/Operator, no auto-rotate
  - AWS_SECRET_ACCESS_KEY (env var) - S3 auth, provisioned by User/Operator, no auto-rotate
  - AZURE_STORAGE_CONNECTION_STRING (env var) - Azure auth, provisioned by User/Operator, no auto-rotate
  - GOOGLE_APPLICATION_CREDENTIALS (env var) - GCS auth (file path), provisioned by User/Operator, no auto-rotate

Container Security:
  - Base Image: registry.access.redhat.com/ubi9/ubi:9.4
  - Non-root user: ovms (UID 5000, GID 5000)
  - Video groups: 39, 44 (for GPU/accelerator access)
  - No Kubernetes API access required

Platform Security (when deployed in RHOAI):
  - KServe: Istio service mesh provides mTLS between pods
  - ModelMesh: Internal gRPC communication within mesh
  - Optional: nginx mTLS sidecar for direct TLS termination
  - Network policies restrict pod-to-pod communication
