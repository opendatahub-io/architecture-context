Authentication & Authorization:
- /q/metrics: Bearer Token (ServiceAccount token via ServiceMonitor bearerTokenFile)
- All other endpoints: No auth enforced at service level
- Access control expected at network/ingress layer (NetworkPolicy, Route)

Secrets:
- TLS cert (/etc/tls/internal/tls.crt) - kubernetes.io/tls
  - Purpose: HTTPS on port 4443
  - Provisioned by: TrustyAI Operator / cert-manager
  - Auto-Rotate: Unknown
- Database credentials (QUARKUS_DATASOURCE_USERNAME/PASSWORD) - Opaque (env vars)
  - Purpose: MariaDB/MySQL authentication
  - Provisioned by: User / Operator
  - Auto-Rotate: No
- MinIO credentials (minio.access-key/secret-key) - Opaque (config)
  - Purpose: MinIO object storage authentication
  - Provisioned by: User / Operator
  - Auto-Rotate: No

Security Notes:
- Route has tls:null - no TLS termination at ingress layer by default
- TLS on port 4443 requires operator-provisioned certificates at /etc/tls/internal/
- RHOAI profile disables explainability endpoints (LIME, SHAP, CF, TSSaliency)
- No authentication on most REST endpoints - relies on network-level access control
- gRPC to model servers has configurable TLS (not enforced by default)
