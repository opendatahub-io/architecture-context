RBAC Summary
================================================================================

Cluster Roles:
- ml-pipeline (multi-user)
  - API Groups: "", argoproj.io, kubeflow.org, authorization.k8s.io,
    authentication.k8s.io
  - Resources: pods, pods/log, workflows, scheduledworkflows,
    subjectaccessreviews, tokenreviews
  - Verbs: get, list, watch, create, update, patch, delete

- kubeflow-pipelines-cache-deployer-clusterrole
  - API Groups: certificates.k8s.io, admissionregistration.k8s.io
  - Resources: certificatesigningrequests, certificatesigningrequests/approval,
    mutatingwebhookconfigurations, signers
  - Verbs: create, delete, get, update, list, patch, approve

Namespace Roles:
- ml-pipeline
  - Manages: pods, workflows, scheduledworkflows, pipelineruns, taskruns,
    conditions, runs, tasks, customruns, pipelineloops
- ml-pipeline-persistenceagent-role
  - Watches: workflows, scheduledworkflows, pipelineruns, taskruns
- ml-pipeline-scheduledworkflow-role
  - Manages: workflows, scheduledworkflows, events
- ml-pipeline-viewer-controller-role
  - Manages: deployments, services, viewers
- ml-pipeline-ui
  - Reads: pods, pods/log, events, secrets, configmaps, viewers, workflows
- kubeflow-pipelines-cache-role
  - Manages: pods, configmaps, workflows, pipelineruns, taskruns
- pipeline-runner
  - Full access (*): secrets, configmaps, PVs, PVCs, pods, services,
    deployments, replicasets, kubeflow.org/*, jobs, tekton resources

Ingress Note
================================================================================

No static ingress resources defined in this repository. Ingress is managed by
the data-science-pipelines-operator (DSPO) which creates OpenShift Routes or
other ingress resources to expose ml-pipeline and ml-pipeline-ui externally.
