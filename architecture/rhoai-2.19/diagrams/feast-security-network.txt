RBAC Summary:
- ClusterRole: manager-role
  - API Groups: apps, "" (core), feast.dev, rbac.authorization.k8s.io, route.openshift.io
  - Resources: deployments, configmaps, persistentvolumeclaims, serviceaccounts,
               services, secrets, featurestores, featurestores/status,
               featurestores/finalizers, roles, rolebindings, routes
  - Verbs: create, delete, get, list, update, watch, patch (varies by resource)

- ClusterRole: metrics-auth-role
  - API Groups: authentication.k8s.io, authorization.k8s.io
  - Resources: tokenreviews, subjectaccessreviews
  - Verbs: create

- ClusterRole: metrics-reader
  - Non-resource: /metrics
  - Verbs: get

- Role: leader-election-role (namespace-scoped)
  - API Groups: "" (core), coordination.k8s.io
  - Resources: configmaps, leases, events
  - Verbs: get, list, watch, create, update, patch, delete

- Dynamic Role: feast-{name} (per FeatureStore CR)
  - Purpose: Grants read access to roles/rolebindings for K8s RBAC authorization

Service Accounts:
- controller-manager (operator namespace) - Feast operator service account
- feast-{name} (per FeatureStore CR) - Feast deployment pods service account

Secrets:
- feast-{name}-{service}-tls (kubernetes.io/tls)
  - Purpose: TLS certificates for feast services
  - Provisioned by: OpenShift service serving certificates (annotation-based)
  - Auto-rotate: Yes (OpenShift managed)

- User-provided DB store secret (Opaque)
  - Purpose: Database connection parameters for persistence backends
  - Auto-rotate: No

- User-provided OIDC secret (Opaque)
  - Contents: client_id, auth_discovery_url, client_secret, username, password
  - Auto-rotate: No

Authentication & Authorization:
- Online/Offline/Registry servers:
  - Kubernetes RBAC: TokenReview + SubjectAccessReview via K8s API (in-process)
  - OIDC: JWT validation against OIDC provider (in-process)
  - Both optional/configurable per FeatureStore CR
- Operator metrics (8443/TCP):
  - TokenReview/SubjectAccessReview via controller-runtime metrics filter
