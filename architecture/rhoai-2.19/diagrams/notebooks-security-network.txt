RBAC Summary:
==============================================================================
This component does NOT define any RBAC resources.
RBAC for workbench pods is managed by the ODH Notebook Controller.

- Workbench pods run under user-scoped ServiceAccounts
- Auth proxy sidecar enforces OAuth2 / RBAC authorization
- Access control is per-namespace (data science project)

Authentication & Authorization:
==============================================================================
- Endpoint: Workbench UI (JupyterLab, code-server, RStudio)
- Methods: GET, POST, WebSocket
- Auth Mechanism: OAuth2 / RBAC proxy
- Enforcement Point: ODH Notebook Controller sidecar
  - RHOAI 2.x: oauth-proxy
  - RHOAI 3.x: kube-rbac-proxy
- Policy: User must be authenticated and authorized

Container Security:
==============================================================================
- All images run as non-root user (UID 1001)
- Base image: Red Hat UBI9 (security patching)
- CI/CD scans: Clair, ClamAV, Snyk, Coverity, rpms-signature-scan
- Images are signed and include SBOMs
- Supported architectures: amd64, ppc64le, s390x

Secrets:
==============================================================================
- rhel-subscription-secret (Opaque)
  - Purpose: RHEL subscription for RStudio BuildConfig builds
  - Provisioned by: Manual / Cluster Admin
  - Auto-Rotate: No
