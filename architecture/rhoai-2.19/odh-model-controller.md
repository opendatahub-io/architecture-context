## Purpose
**Short**: A Kubernetes controller that manages the lifecycle of OpenShift resources required for KServe, ModelMesh, and NVIDIA NIM model serving inference services on the RHOAI platform.

**Detailed**: The odh-model-controller is a Go-based Kubernetes operator built with controller-runtime that reconciles supplementary OpenShift/Kubernetes resources for model serving workloads. It watches KServe InferenceService, ServingRuntime, and InferenceGraph custom resources, and dynamically creates and manages OpenShift Routes (for external access), Istio Gateway servers (for TLS termination), ServiceMeshMembers (for mesh enrollment), NetworkPolicies (for network isolation), Authorino AuthConfigs (for authorization), PeerAuthentications (for mTLS), ServiceMonitors/PodMonitors (for metrics), and RoleBindings (for Prometheus access).

The controller supports three deployment modes: **KServe Serverless** (with Knative and Istio service mesh integration), **KServe RawDeployment** (without service mesh, using direct Routes), and **ModelMesh** (for multi-model serving). It also provides a dedicated NIM Account controller that manages NVIDIA NIM integration by validating API keys, creating model data ConfigMaps, image pull Secrets, and OpenShift Templates for NIM-based ServingRuntimes.

Additionally, the controller manages certificate trust chains (CA bundle propagation from ODH global certs to KServe-specific ConfigMaps), storage configuration secrets (aggregating data connection secrets into a unified storage-config), and Ray TLS certificates for multi-node serving runtimes. It includes admission webhooks for mutating and validating Pods, InferenceServices, InferenceGraphs, Knative Services, and NIM Accounts.
