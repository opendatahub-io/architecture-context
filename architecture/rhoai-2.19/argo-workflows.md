## Purpose
**Short**: Argo Workflows is a Kubernetes-native workflow engine used by Data Science Pipelines to orchestrate and execute containerized ML/data pipeline steps.

**Detailed**: Argo Workflows provides a container-native workflow orchestration engine for Kubernetes. In the RHOAI (Red Hat OpenShift AI) context, it serves as the underlying execution engine for Data Science Pipelines (DSP), enabling users to define, schedule, and execute multi-step ML workflows as Kubernetes custom resources.

The system consists of three main binaries: the **workflow-controller** which watches Workflow CRDs and orchestrates pod creation/lifecycle management; the **argo-server** which provides both a gRPC and REST API (via grpc-gateway) along with a web UI for workflow management; and the **argoexec** executor which runs as a sidecar/init container within workflow step pods, handling artifact collection, output capture, and container lifecycle management. The controller supports leader election for high availability, and the server supports multiple authentication modes including SSO (OIDC), Kubernetes service account tokens, and server-side authentication.

This Red Hat fork builds on the upstream Argoproj Argo Workflows project, with RHOAI-specific Dockerfiles (Konflux-based builds using UBI8 base images) and Red Hat container labeling for distribution through the Red Hat ecosystem.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|

| argo-server | Go Binary (Deployment) | Provides gRPC + HTTP/REST API (via grpc-gateway on port 2746), serves web UI, handles SSO/auth, artifact serving, and workflow archive queries |
| argoexec | Go Binary (Sidecar) | Executor injected into workflow step pods; handles artifact upload/download, output parameter capture, and container wait/signal logic |
| argo-workflows-ui | React/TypeScript SPA | Web-based UI for workflow visualization, submission, and management; served by argo-server |
