## Purpose
**Short**: KServe is a Kubernetes-native model serving platform that provides serverless inference with autoscaling, canary rollouts, and multi-framework support for machine learning models.

**Detailed**: KServe (formerly KFServing) is the model serving component of Red Hat OpenShift AI (RHOAI). It provides a standardized Kubernetes Custom Resource Definition (InferenceService) for deploying and managing ML model inference endpoints. The controller manages the full lifecycle of inference services including model storage initialization, predictor/transformer/explainer component orchestration, autoscaling configuration, and ingress routing.

KServe supports two deployment modes: Serverless (using Knative Serving for scale-to-zero and autoscaling) and RawDeployment (using standard Kubernetes Deployments and Services). In the RHOAI distribution, it integrates with Istio service mesh for traffic management via VirtualServices, and with OpenShift Routes for external access. The controller dynamically injects oauth-proxy sidecar containers for authentication when raw deployment mode with auth is enabled.

The component also includes an InferenceGraph controller for composing multiple inference services into DAG-based inference pipelines, a TrainedModel controller for multi-model serving, and a LocalModel controller for pre-caching models on cluster nodes. Python-based model serving runtimes (sklearn, xgboost, pytorch, huggingface, etc.) and a Go-based storage initializer are also part of the repository.
