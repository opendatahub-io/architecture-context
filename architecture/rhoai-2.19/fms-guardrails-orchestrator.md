## Purpose
**Short**: Orchestration server for invoking content safety detectors on text generation model input and output, providing guardrails for LLM-based applications.

**Detailed**: The FMS Guardrails Orchestrator (fms-guardrails-orchestr8) is a Rust-based HTTP/HTTPS server that sits between clients and text generation models, orchestrating content safety detection on both inputs (user prompts) and outputs (generated text). It coordinates calls to multiple backend services including text generation servers (TGIS or caikit-nlp via gRPC), content safety detector servers (via HTTP), and text chunker servers (via gRPC) to provide a unified guardrails API.

The orchestrator supports multiple API versions: a v1 API for classification-with-text-generation workflows, and a v2 API providing finer-grained detection endpoints for text content, chat messages, context documents, generated text, and streaming content. It also supports an OpenAI-compatible chat completions detection endpoint. The server supports TLS and mTLS for both incoming connections and outgoing connections to backend services, and integrates with OpenTelemetry for distributed tracing and metrics.
