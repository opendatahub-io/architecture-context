## Purpose
**Short**: Feast is a feature store platform that manages and serves machine learning features for both training and online inference.

**Detailed**: Feast (Feature Store) provides an operational data system for managing and serving machine learning features. It bridges the gap between data engineering and ML engineering by providing a centralized store for feature definitions, materialization, and serving. The system consists of an offline store (for historical feature retrieval), an online store (for low-latency feature serving), and a registry (for feature metadata management).

The RHOAI distribution packages Feast with a Kubernetes operator (`feast-operator`) that automates deployment and lifecycle management of Feast services on OpenShift. The operator watches `FeatureStore` custom resources and reconciles the desired state by creating Deployments, Services, ConfigMaps, PVCs, ServiceAccounts, RBAC resources, and OpenShift Routes. It supports multiple persistence backends (file-based, PostgreSQL, Redis, Snowflake, etc.), TLS configuration (including automatic OpenShift service serving certificates), and authorization via Kubernetes RBAC or OIDC.

The feature server component provides both HTTP and gRPC APIs for online feature retrieval, while the offline store provides Arrow Flight-based data retrieval for batch/training workloads. A UI server provides a web interface for exploring feature definitions.
