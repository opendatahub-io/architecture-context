## Purpose
**Short**: A reverse-proxy sidecar that translates KServe V2 REST inference requests into gRPC calls for ModelMesh Serving.

**Detailed**: The REST Proxy is a lightweight Go service that leverages the gRPC-Gateway library to create a reverse-proxy server translating RESTful HTTP API calls into gRPC. It enables clients to send inference requests using the KServe V2 REST Predict Protocol to model serving platforms (ModelMesh Serving) that natively expect the gRPC V2 Predict Protocol.

The proxy runs as a sidecar container within ModelMesh Serving deployments. It listens for REST requests on a configurable HTTP port (default 8008) and forwards them as gRPC calls to the local inference service on a configurable gRPC port (default 8033). The proxy handles complex tensor data serialization including multi-dimensional arrays, base64-encoded BYTES tensors, and raw binary tensor contents. It supports optional TLS for both the REST-facing HTTP server and the gRPC backend connection.

The proxy implements the KServe V2 inference protocol defined in a protobuf specification, supporting ModelMetadata and ModelInfer operations with HTTP REST bindings. Custom JSON marshaling/unmarshaling ensures proper translation between the REST and gRPC representations of inference requests and responses.
