## Purpose
**Short**: High-performance model inference server built on OpenVINO, serving AI/ML models via gRPC and REST APIs compatible with TensorFlow Serving, KServe, and OpenAI standards.

**Detailed**: OpenVINO Model Server (OVMS) is a C++ inference serving system optimized for deployment on Intel architectures. It hosts machine learning models and makes them accessible over standard network protocols, allowing clients to send inference requests and receive predictions. The server supports multiple model frameworks including TensorFlow, PaddlePaddle, and ONNX through the OpenVINO runtime.

OVMS provides multiple API interfaces: TensorFlow Serving API (v1), KServe/KFS Inference Protocol (v2), and OpenAI-compatible API (v3) for generative AI use cases including chat completions, text embeddings, and reranking. It features model management with versioning, dynamic input shapes, directed acyclic graph (DAG) pipelines for chaining models, MediaPipe graph execution, and Prometheus-compatible metrics. Models can be loaded from local storage or remote object stores (S3, Azure Blob, Google Cloud Storage).

In the RHOAI distribution, OVMS serves as the core inference runtime backing model serving deployments. It is deployed as a container image built on Red Hat Universal Base Image (UBI 9) and operates as a standalone inference server process within Kubernetes pods, typically managed by higher-level components like KServe or ModelMesh.
