## Purpose

**Short**: A Kubernetes operator that manages the lifecycle of Model Registry instances, deploying and configuring the gRPC metadata store and REST proxy services along with optional Istio service mesh integration.

**Detailed**: The Model Registry Operator automates the deployment and management of Model Registry instances on OpenShift/Kubernetes clusters. Each `ModelRegistry` custom resource triggers the creation of a complete model registry stack consisting of an MLMD (ML Metadata) gRPC server backed by PostgreSQL or MySQL, and a REST API proxy that translates HTTP requests into gRPC calls to the metadata server. The operator supports the Kubeflow Model Registry and Google ML Metadata upstream projects.

The operator provides deep integration with Istio service mesh and OpenShift Service Mesh (Maistra). When Istio is enabled, the operator creates VirtualService, DestinationRule, Gateway, and AuthorizationPolicy resources for each registry instance. Authentication is handled through Authorino AuthConfig resources that enforce Kubernetes RBAC-based authorization via SubjectAccessReview. The operator also manages OpenShift Routes for both direct HTTP access and Istio Gateway ingress.

On OpenShift clusters, the operator additionally creates user Groups and RoleBindings to manage access control, and can create OpenShift Routes for external access to registry services. The operator supports webhook-based validation and defaulting, ensuring unique registry names across the cluster and proper configuration of database and Istio settings.
