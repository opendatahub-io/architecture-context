## Purpose
**Short**: vLLM is a high-throughput, memory-efficient inference and serving engine for Large Language Models (LLMs), providing OpenAI-compatible API and gRPC endpoints.

**Detailed**: vLLM is a production-grade LLM inference server that provides an OpenAI-compatible REST API for serving language models. It is the core serving runtime used in Red Hat OpenShift AI (RHOAI) for model inference workloads. The engine supports advanced features including PagedAttention for efficient memory management, continuous batching for high throughput, tensor parallelism and pipeline parallelism for distributed inference across multiple GPUs, LoRA adapter hot-loading, speculative decoding, and multimodal model support (vision, audio, video).

The Red Hat distribution ships multi-architecture container images (amd64/CUDA, amd64/ROCm, ppc64le, s390x) built on UBI9 (Red Hat Universal Base Image). Each architecture produces two container image variants: an OpenAI-compatible HTTP API server (`vllm-openai`) and a gRPC adapter (`vllm-grpc-adapter`) that wraps vLLM with the TGIS (Text Generation Inference Service) gRPC protocol via the `vllm-tgis-adapter` package. The gRPC adapter enables compatibility with KServe/Caikit serving infrastructure.

vLLM runs as a non-root containerized service (UID 2000) and is typically deployed as a KServe InferenceService or standalone Deployment within OpenShift AI. It loads models from HuggingFace Hub or local storage and exposes inference endpoints for chat completions, text completions, embeddings, scoring, reranking, pooling, and audio transcription.
