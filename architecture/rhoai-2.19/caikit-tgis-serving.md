## Purpose
**Short**: Caikit-TGIS-Serving is a container image that packages the Caikit AI runtime with the TGIS (Text Generation Inference Server) backend to enable Large Language Model (LLM) inference on Red Hat OpenShift AI.

**Detailed**: Caikit-TGIS-Serving provides an inference serving layer for Large Language Models (LLMs) within the Red Hat OpenShift AI (RHOAI) platform. It combines the Caikit AI toolkit — which offers developer-friendly APIs for model management — with the TGIS backend for efficient text generation inference. The container runs the Caikit runtime (`python -m caikit.runtime`) and connects to a co-located TGIS container over gRPC (port 8033) to perform the actual model inference.

This component is deployed as a KServe `ServingRuntime` custom resource, which defines a multi-container pod pattern: one container running Caikit-TGIS-Serving (the "transformer-container") and another running the TGIS inference engine (the "kserve-container"). Users create `InferenceService` CRs referencing this runtime to deploy models. The stack integrates with OpenShift Service Mesh (Istio) for mTLS and traffic management, and OpenShift Serverless (Knative Serving) for autoscaling and serverless deployment patterns. Models are loaded from S3-compatible storage or PersistentVolumeClaims.

The repository itself does not contain operator or controller code — it is a container image definition (Dockerfile + Python dependencies) plus deployment manifests, demo scripts, and test infrastructure for the KServe/Caikit/TGIS serving stack.

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| knative-ingress-gateway | Istio Gateway | * | 443/TCP | HTTPS | TLS 1.2+ | SIMPLE (wildcard-certs) | External |
| knative-local-gateway | Istio Gateway | * | 8081/TCP | HTTP | plaintext | N/A | Internal |

Ingress is managed by KServe + Knative Serving + Istio. When an `InferenceService` CR is created:
1. KServe creates a Knative Service
2. Knative Serving creates a Knative Route, mapping to the Istio ingress gateway
3. External traffic enters via the Istio ingress gateway on port 443/TCP (HTTPS with TLS termination)
4. Internal traffic uses the knative-local-gateway on port 8081/TCP (HTTP)
5. Istio sidecars enforce mTLS between mesh-enrolled pods

Annotations on InferenceService CRs enable Istio sidecar injection:
- `sidecar.istio.io/inject: "true"`
- `sidecar.istio.io/rewriteAppHTTPProbers: "true"`
- `serving.knative.openshift.io/enablePassthrough: "true"`
