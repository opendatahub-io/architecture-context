## Purpose
**Short**: Provides pre-built workbench and runtime container images for data science workflows in Red Hat OpenShift AI.

**Detailed**: The notebooks repository builds and maintains a comprehensive set of container images that serve as interactive development environments (workbenches) and pipeline execution runtimes for data scientists on the RHOAI platform. Workbench images include Jupyter Notebook, JupyterLab, code-server (VS Code in browser), and RStudio Server — each pre-configured with Python and common data science libraries.

The repository ships multiple image variants targeting different accelerator hardware: CPU-only images built on UBI9, NVIDIA CUDA GPU images (CUDA 12.6.3 with cuDNN 9.5), and AMD ROCm GPU images. Framework-specific images bundle PyTorch or TensorFlow. Runtime images provide minimal environments for Elyra pipeline execution. All images are built via Konflux CI/CD pipelines and published to `quay.io/modh/` registries as part of the RHOAI product.

The deployment mechanism uses OpenShift ImageStream resources managed through kustomize overlays. The RHOAI operator (rhods-operator) deploys these ImageStream manifests, which reference pinned container image digests. This allows the RHOAI Dashboard to present available workbench types to users, who can then launch notebook workloads via the ODH Notebook Controller.

### Container Security

- All images run as non-root user (UID 1001) by default
- Images are built on Red Hat UBI9/RHEL9 base images with security updates
- Build pipelines include: Clair vulnerability scanning, ClamAV malware scanning, SAST checks (Snyk, Coverity), RPM signature scanning, deprecated image checks, ecosystem cert preflight checks
- Images are pinned by SHA256 digest in params.env for reproducibility
