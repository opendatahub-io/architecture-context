## Purpose
**Short**: ModelMesh is a distributed LRU cache and intelligent routing layer for high-scale, high-density model serving that runs as a sidecar alongside model runtime containers.

**Detailed**: ModelMesh is a mature, general-purpose model serving management and routing layer designed for high-scale, high-density, and frequently-changing model use cases. It works with existing or custom-built model servers and acts as a distributed LRU cache for serving runtime models. The framework manages model lifecycle operations including registration, loading, unloading, scaling, and inference request routing across a cluster of pods.

ModelMesh operates as a sidecar container within Kubernetes pods, communicating with a co-located model runtime container via a local gRPC interface (either over localhost TCP or Unix Domain Sockets). It uses etcd (or ZooKeeper) as a distributed key-value store for cluster coordination, model registry state, and dynamic configuration. The framework automatically handles model placement, cache eviction (LRU-based), auto-scaling of model copies across replicas, and intelligent request routing with cache-miss handling.

For full Kubernetes-based deployment and management, ModelMesh integrates with ModelMesh Serving (modelmesh-serving), which provides a separate controller for managing ServingRuntimes and InferenceServices via Kubernetes custom resources. In the RHOAI distribution, ModelMesh is shipped as a container image (`odh-modelmesh`) and deployed by the modelmesh-serving operator.
