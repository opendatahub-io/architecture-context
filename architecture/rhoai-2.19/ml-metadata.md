## Purpose
**Short**: A gRPC server for recording and retrieving metadata associated with ML pipeline workflows, used as a sidecar in RHOAI Data Science Pipelines.

**Detailed**: ML Metadata (MLMD) is a library and gRPC server originally developed by Google for recording and retrieving metadata associated with ML developer and data scientist workflows. It is an integral part of TFX (TensorFlow Extended) and Kubeflow Pipelines, and is used in RHOAI as the metadata backend for Data Science Pipelines.

The server stores three core entity types: **Artifacts** (datasets, models, metrics), **Executions** (training runs, pipeline steps), and **Contexts** (pipeline runs, experiments). It also tracks **Events** that link Artifacts to Executions (inputs/outputs) and supports lineage graph traversal. The metadata store supports pluggable database backends including MySQL, PostgreSQL, and SQLite.

In RHOAI, this component is shipped as the `odh-mlmd-grpc-server` container image, which runs as a sidecar alongside the Data Science Pipelines API server, providing persistent metadata storage for pipeline run artifacts and lineage tracking. The container is built via Konflux CI from a Bazel-compiled C++ binary.

### TLS Configuration

The server supports optional TLS via `MetadataStoreServerConfig.SSLConfig`:
- `server_key`: PEM-encoded private server key
- `server_cert`: PEM-encoded public server certificate
- `custom_ca`: PEM-encoded custom certificate authority
- `client_verify`: When `true`, requires and verifies client certificates (mTLS)

When no `ssl_config` is provided, the server uses insecure (plaintext) gRPC credentials.
