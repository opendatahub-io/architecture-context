## Purpose
**Short**: Kueue is a Kubernetes-native job queueing system that manages when workloads should be admitted to start based on quota and fair-sharing policies.

**Detailed**: Kueue provides a set of APIs and controllers for job-level resource management in Kubernetes clusters. It acts as a job-level manager that decides when a job should be admitted to start (allowing pod creation) and when it should stop (triggering active pod deletion), based on quotas, priorities, and fair-sharing policies defined through its CRD hierarchy of ClusterQueues, LocalQueues, and ResourceFlavors.

Kueue integrates with multiple job frameworks including Kubernetes batch/Job, Kubeflow training operators (PyTorchJob, TFJob, MPIJob, MXJob, PaddleJob, XGBoostJob), Ray (RayJob, RayCluster), JobSet, and AppWrapper (via external framework integration). It provides admission webhooks that intercept job creation to enforce queueing policies, and a scheduler component that continuously evaluates pending workloads against available quota to make admission decisions.

In the RHOAI distribution, Kueue is deployed in the `opendatahub` namespace with specific feature gate configurations: VisibilityOnDemand disabled, LocalQueueMetrics enabled, and MultiKueue disabled. The RHOAI deployment includes Prometheus monitoring integration with ServiceMonitor, PrometheusRule alerts, and a dedicated metrics service account for scraping metrics securely.
