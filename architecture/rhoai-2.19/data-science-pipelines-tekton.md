## Purpose
**Short**: Kubeflow Pipelines backend and frontend adapted to use Tekton Pipelines as the workflow execution engine for ML pipeline orchestration in RHOAI.

**Detailed**: Data Science Pipelines Tekton (DSP Tekton) is a fork of Kubeflow Pipelines that replaces the default Argo Workflows engine with Tekton Pipelines for executing ML workflows on Kubernetes. It provides an API server for pipeline CRUD operations, a persistence agent for syncing workflow state to a database, a scheduled workflow controller for cron/periodic pipeline execution, a cache server for task-level caching via a mutating admission webhook, a visualization server for rendering pipeline outputs, and a web frontend (UI) for managing pipelines, experiments, runs, and artifacts.

The system integrates with an object store (MinIO/S3-compatible) for artifact storage and a relational database (MySQL/MariaDB/PostgreSQL) for pipeline metadata. ML Metadata (MLMD) is provided via a separate gRPC metadata store server with an Envoy proxy sidecar. The architecture uses Tekton PipelineRun and TaskRun CRDs as the execution primitives, with the ScheduledWorkflow CRD for recurring pipeline execution. In the RHOAI distribution, the UI is fronted by an OpenShift oauth-proxy sidecar for authentication, exposed via an OpenShift Route with TLS re-encryption.

| ds-pipeline-viewer-crd | Go Controller | Manages Viewer CRD, creates deployments/services for TensorBoard instances |
| metadata-grpc-server | gRPC Service (MLMD) | ML Metadata gRPC store server on port 8080/TCP, backed by MySQL/MariaDB |
