## Purpose
**Short**: A central metadata repository for managing ML model registrations, versions, and artifacts with REST and gRPC APIs.

**Detailed**: Model Registry provides a centralized store for machine learning model metadata, enabling data scientists and ML engineers to register models, track versions, manage model artifacts, and record inference service deployments. It is built on top of Google's ML Metadata (MLMD) project and provides a higher-level REST API (OpenAPI v1alpha3) as a Go proxy that translates requests to the underlying MLMD gRPC backend.

The architecture consists of multiple components: a Go-based REST API proxy server that exposes an OpenAPI-compliant interface on port 8080; an MLMD gRPC metadata store server (ml_metadata_store_server) on port 9090 that handles the actual metadata persistence to MySQL or PostgreSQL databases; a UI Backend-for-Frontend (BFF) Go service that provides Kubernetes-aware API endpoints for the web UI; a React/TypeScript frontend for browsing and managing models; a Python client library for programmatic access; and a CSI storage initializer for KServe integration. The service supports both MySQL and PostgreSQL database backends and can be deployed with Istio service mesh integration for mTLS and traffic management.
