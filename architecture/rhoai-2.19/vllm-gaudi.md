## Purpose
**Short**: A vLLM fork optimized for Intel Gaudi (HPU) AI accelerators, providing OpenAI-compatible and gRPC inference serving for large language models on Red Hat OpenShift AI.

**Detailed**: vllm-gaudi is a downstream fork of the HabanaAI/vllm-fork project, which itself is a fork of the upstream vLLM project. It provides high-throughput, low-latency LLM inference serving specifically optimized for Intel Gaudi 2 and Gaudi 3 AI accelerators (HPUs). The project implements PagedAttention and continuous batching techniques adapted for the Habana software stack.

The container image ships two entrypoints: an OpenAI-compatible REST API server (the primary entrypoint) and a gRPC adapter via `vllm-tgis-adapter` for compatibility with the Text Generation Inference Service (TGIS) protocol used by RHOAI's model serving infrastructure. The image is built on a Red Hat UBI 9.4 base with Habana Labs PyTorch installer, providing enterprise-grade support for deploying LLMs on Intel Gaudi hardware within OpenShift AI environments.

The serving engine supports a wide range of capabilities including chat completions, text completions, embeddings, pooling, reranking, scoring, tokenization, audio transcription, and LoRA adapter hot-loading. It exposes Prometheus metrics for monitoring and supports OpenTelemetry tracing for observability.
