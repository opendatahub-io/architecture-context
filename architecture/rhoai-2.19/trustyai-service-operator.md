## Purpose
**Short**: Kubernetes operator that manages TrustyAI explainability services, LM evaluation jobs, and guardrails orchestration for AI/ML model governance on OpenShift.

**Detailed**: The TrustyAI Service Operator is a multi-controller Go operator that provides AI/ML model trustworthiness and governance capabilities within the Red Hat OpenShift AI (RHOAI) platform. It manages three distinct services through separate controllers that are selectively enabled at startup via the `--enable-services` flag.

The TrustyAI Service (TAS) controller deploys and manages TrustyAI explainability service instances that integrate with KServe InferenceServices (ModelMesh, Raw, and Serverless deployment modes) to provide model bias detection (SPD, DIR metrics), explainability, and fairness monitoring. Each TrustyAI service deployment includes an OAuth proxy sidecar for authentication and creates OpenShift Routes for external access.

The LM Evaluation (LMES) controller manages LMEvalJob custom resources that orchestrate language model evaluation workloads using the lm-evaluation-harness framework. It creates evaluation pods with a driver sidecar for job lifecycle management and integrates with Kueue for workload scheduling and resource management. The Guardrails Orchestrator (GORCH) controller deploys the FMS Guardrails Orchestrator service that provides content safety guardrails (detectors, chunkers, generators) with optional vLLM gateway and OpenTelemetry tracing support.

| Component | Type | Purpose |

| LMEvalJob Controller (LMES) | Go Controller | Reconciles LMEvalJob CRs; creates evaluation Pods with lm-eval container and driver sidecar; manages job lifecycle (New, Scheduled, Running, Complete, Cancelled, Suspended); creates PVCs for output storage |
| Job Manager Controller (JobMgr) | Go Controller | Kueue integration for LMEvalJob workload scheduling; creates Kueue Workload CRs for resource-aware job management |
| Guardrails Orchestrator Controller (GORCH) | Go Controller | Reconciles GuardrailsOrchestrator CRs; creates Deployments (with optional vLLM gateway and regex detector containers), Services, OpenShift Routes (HTTP and health), ServiceAccounts |
| Operator Manager | Go Binary | Main entrypoint (`/manager`); configures controller-runtime manager with leader election, health/readiness probes; selectively enables controllers via `--enable-services` flag |
