## Purpose
**Short**: ModelMesh Serving is a Kubernetes operator that manages the lifecycle of multi-model serving runtimes and model inference services using the KServe API, backed by the ModelMesh distributed model serving framework and etcd for coordination.

**Detailed**: ModelMesh Serving provides a Kubernetes-native control plane for deploying and managing machine learning model inference services at scale. It extends the KServe Serving API (serving.kserve.io) with CRDs for Predictors, ServingRuntimes, ClusterServingRuntimes, and InferenceServices to enable multi-model serving where many models can be efficiently packed into shared serving runtime pods.

The controller operator watches these CRDs and reconciles Kubernetes Deployments for each ServingRuntime, dynamically injecting sidecar containers including the ModelMesh container (for model routing and lifecycle management), a REST proxy (for HTTP/REST inference), an oauth-proxy (for authentication), and a storage puller (for fetching model artifacts from object storage). The controller communicates with ModelMesh via gRPC to manage virtual model (vmodel) registrations in etcd, supporting features like model hot-loading, scale-to-zero, automatic model placement across runtime replicas, and blue-green model transitions.

In the RHOAI distribution, the controller operates in cluster scope mode, watching namespaces annotated with `modelmesh-enabled` labels. It creates headless Kubernetes Services for gRPC inference and manages per-namespace etcd secrets for model-mesh coordination. The built-in serving runtimes support OpenVINO Model Server (OVMS), Triton, MLServer, and TorchServe backends.
