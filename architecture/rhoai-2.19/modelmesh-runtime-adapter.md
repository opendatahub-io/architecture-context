## Purpose
**Short**: Sidecar container that acts as an intermediary between ModelMesh and third-party model-server containers, providing model storage retrieval and runtime-specific protocol adaptation.

**Detailed**: The ModelMesh Runtime Adapter is a multi-purpose sidecar container image that runs alongside model server containers (Triton, MLServer, OpenVINO Model Server, TorchServe) within ModelMesh Serving pods. It contains two categories of binaries: a model-serving puller and multiple runtime-specific adapters.

The **model-serving puller** intercepts ModelRuntime gRPC calls from ModelMesh, downloads model artifacts from various cloud storage backends (S3/IBM COS, Google Cloud Storage, Azure Blob Storage, HTTP/HTTPS endpoints, and PersistentVolumeClaims), and then forwards the load/unload requests to the appropriate runtime adapter. This decouples model storage management from the model serving runtimes.

The **runtime adapters** (Triton, MLServer, OVMS, TorchServe) implement the ModelRuntime gRPC service defined by ModelMesh. They translate the generic ModelMesh model lifecycle operations (load, unload, predict size, runtime status) into runtime-specific protocol calls (gRPC for Triton/TorchServe/MLServer, HTTP REST for OVMS). Each adapter also handles model layout adaptation, converting the pulled model files into the directory structure expected by its target runtime.

### Storage Credential Configuration

Storage credentials are read from a mounted directory (default `/storage-config`). Each file in the directory represents a storage configuration keyed by the filename. The JSON content includes:

- **S3**: `type`, `access_key_id`, `secret_access_key`, `endpoint_url`, `region`, `bucket`, `certificate` (optional custom CA)
- **GCS**: `type`, `private_key`, `client_email`, `token_uri` (optional)
- **Azure**: `type`, `account_name`, `container`, `client_id`, `client_secret`, `tenant_id` (or `connection_string`)
- **HTTP**: `type`, `url`, `headers` (optional), `certificate` (optional CA), `client_certificate`, `client_key` (optional mTLS)
- **PVC**: `type`, `name` (PVC name, mounted at `/pvc_mounts/<name>`)
