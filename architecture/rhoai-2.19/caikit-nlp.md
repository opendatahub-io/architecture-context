## Purpose
**Short**: Caikit-NLP is a Python library providing Natural Language Processing capabilities (text generation, embeddings, reranking, classification) built on the Caikit AI framework with support for local and remote (TGIS) model serving.

**Detailed**: Caikit-NLP implements the Caikit framework's "task" and "module" abstractions to provide a comprehensive set of NLP capabilities. It supports text generation via local HuggingFace Transformers models and remote Text Generation Inference Server (TGIS) backends, text embeddings via sentence-transformers, reranking via both bi-encoder (embedding) and cross-encoder models, sentence similarity, sequence classification, token classification, and tokenization.

The library is designed to be used as a runtime library within the Caikit serving framework, exposing NLP models through both HTTP (port 8080) and gRPC (port 8085) endpoints. It supports parameter-efficient fine-tuning (PEFT) methods including prompt tuning and multi-task prompt tuning (MPT), enabling efficient model customization without modifying base model weights. In the RHOAI platform, caikit-nlp is packaged as a container image and deployed as part of the model serving infrastructure, typically behind KServe/ModelMesh.

## Build & Deployment

### Container Build

The container image is built via Konflux CI/CD pipeline using the standard `Dockerfile` (no Konflux-specific Dockerfile exists). The Tekton PipelineRun (`.tekton/caikit-nlp-v2-19-push.yaml`) triggers on pushes to the `rhoai-2.19` branch and produces images tagged to `quay.io/modh/caikit-nlp`.

- **Base image**: `registry.access.redhat.com/ubi9/ubi-minimal:latest`
- **Build method**: `tox -e build` producing a Python wheel, installed into a venv at `/opt/caikit/`
- **Runtime user**: `caikit` (UID 1001, GID 0)
- **Default command**: `python` (runtime startup configured externally)
- **Volume**: `/caikit/` for configuration and model artifacts

### Runtime Configuration

Runtime is configured via YAML files (pointed to by `CONFIG_FILES` env var) and/or environment variables. Key configuration areas:
- `runtime.library`: Set to `caikit_nlp`
- `model_management.finders`: LOCAL and TGIS-AUTO model finders
- `model_management.initializers`: Backend priority (TGIS backend with connection settings)
- `embedding.*`: Embedding module configuration (trust_remote_code, batch_size, IPEX, autocast)
- `tgis_request_timeout`: Request timeout for TGIS backend (default 60s)
