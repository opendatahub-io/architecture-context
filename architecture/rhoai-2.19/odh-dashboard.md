## Purpose

**Short**: Web-based management dashboard for Red Hat OpenShift AI (RHOAI) that provides a unified UI for data science workflows, notebook management, model serving, and platform administration.

**Detailed**: The ODH Dashboard is the primary user interface for the Red Hat OpenShift AI platform. It consists of a Node.js/Fastify backend that acts as a Backend-for-Frontend (BFF) API server and a React/TypeScript single-page application frontend built with Webpack. The backend proxies authenticated Kubernetes API calls on behalf of the logged-in user and serves the static frontend assets.

The dashboard provides management interfaces for Jupyter notebooks, model serving (KServe, ModelMesh), model registries, accelerator and hardware profiles, data science pipelines, distributed workloads, connection types, and ISV integrations (NVIDIA NIM, OpenVINO, Starburst, etc.). It reads and manages several custom resources including OdhDashboardConfig, AcceleratorProfile, HardwareProfile, OdhApplication, OdhDocument, and OdhQuickStart.

Authentication is handled via an oauth-proxy sidecar container that performs OpenShift OAuth authentication, forwarding the user's access token to the backend via the `x-forwarded-access-token` header. The backend then uses this token to make impersonated calls to the Kubernetes API on the user's behalf. In the RHOAI distribution, the dashboard is deployed as `rhods-dashboard` with 5 replicas and is exposed via an OpenShift Route with TLS re-encryption.
