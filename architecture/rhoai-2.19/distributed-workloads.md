## Purpose

**Short**: Repository for building GPU-accelerated runtime container images (Ray, Training) and integration tests for the RHOAI Distributed Workloads stack.

**Detailed**: The distributed-workloads repository serves two primary purposes within the Red Hat OpenShift AI (RHOAI) ecosystem. First, it provides Dockerfiles and build configurations for constructing GPU-accelerated runtime container images used by the distributed workloads stack. These images include Ray runtime images (with CUDA and ROCm GPU acceleration), PyTorch training runtime images, dataset images (Alpaca), model images (BLOOM-560m), and utility images (MinIO CLI). These images are built via Konflux CI/CD pipelines and published to `quay.io/modh/`.

Second, it contains comprehensive Go-based integration test suites that validate the distributed workloads stack components: Kubeflow Training Operator (KFTO) for PyTorchJob-based model fine-tuning, KubeRay for Ray cluster-based distributed training/HPO, Kueue for job queuing and resource quota management, AppWrapper for workload wrapping, and CodeFlare SDK integration. The tests exercise multi-node, multi-GPU training scenarios across NVIDIA CUDA and AMD ROCm accelerators, validating LLM fine-tuning (SFT, LoRA, QLoRA), MNIST training, hyperparameter optimization with Ray Tune, and LLM fine-tuning with DeepSpeed. This repository does not contain any operator or controller code -- it produces runtime images and validates the platform-level integration of distributed workloads components.
