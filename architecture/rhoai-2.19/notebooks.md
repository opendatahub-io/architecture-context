## Purpose
**Short**: Provides pre-built container images for Jupyter, code-server, and RStudio workbench environments used by data scientists in the RHOAI platform.

**Detailed**: The Notebooks repository builds and maintains a collection of container images that serve as workbench environments for the Red Hat OpenShift AI (RHOAI) platform. These images are launched by the ODH Notebook Controller and provide users with interactive development environments for data science, machine learning, and AI experimentation.

The repository produces three categories of container images: (1) IDE workbench images including JupyterLab notebooks (minimal, data science, GPU-accelerated variants for CUDA and ROCm), code-server (VS Code in browser), and RStudio Server; (2) runtime images used by the Elyra pipeline editor for executing pipeline nodes outside of the interactive notebook session; and (3) CUDA/ROCm base images for GPU-accelerated workloads. Each workbench type ships multiple versioned tags (N through N-5) to allow users to select specific release versions, enabling gradual adoption of new versions while maintaining backward compatibility.

The images are built by Konflux CI/CD via Tekton pipelines and are deployed to OpenShift clusters as ImageStream resources that reference pre-built container images from the `quay.io/modh` registry. The manifests use Kustomize to manage image references via ConfigMaps (`params.env` for image digests, `commit.env` for build commit hashes), enabling automated image updates through GitHub Actions.
