══════════════════════════════════════════════════════════════════════════
SERVICE MESH INTEGRATION (OPTIONAL)
══════════════════════════════════════════════════════════════════════════

If Istio/Service Mesh is enabled:
  - PeerAuthentication: Can be set to STRICT for mTLS between services
  - AuthorizationPolicy: Can be configured for fine-grained access control
  - DestinationRule: Can configure mTLS mode for specific services
  - VirtualService: Can configure traffic routing and timeout policies

Note: ModelMesh Serving supports optional service mesh integration for:
  - mTLS encryption on inference traffic (8033/8008/TCP)
  - Service-to-service authentication
  - Traffic routing and load balancing
  - Request tracing and observability

Default configuration: All internal traffic is plaintext gRPC/HTTP
