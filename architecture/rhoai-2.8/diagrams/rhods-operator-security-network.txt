Service Mesh Configuration:
===========================

When Service Mesh is enabled (required for KServe single-model serving):
  - Operator creates Istio resources: Gateway, VirtualService, DestinationRule, AuthorizationPolicy
  - Namespace configuration via ServiceMeshMember or ServiceMeshMemberRoll
  - mTLS mode: PERMISSIVE or STRICT (configurable via DSCInitialization)
  - AuthorizationPolicy: Controls access to KServe inference endpoints
