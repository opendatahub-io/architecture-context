AUTHENTICATION & AUTHORIZATION FLOW
═══════════════════════════════════════════════════════════════════════════

Endpoint: /api/* (All API endpoints)
├─ Auth Mechanism: Bearer Token (OAuth)
├─ Enforcement: OAuth Proxy (validates session cookie or bearer token)
├─ Flow:
│  1. User requests API endpoint with session cookie
│  2. OAuth proxy validates cookie against OpenShift OAuth server
│  3. If valid, proxy adds Bearer token to request header
│  4. Backend receives authenticated request with user identity
│  5. Backend performs RBAC check via delegated verb check on odh-dashboard service
└─ Policy: Authenticated OpenShift users only

Endpoint: /oauth/* (OAuth callbacks)
├─ Auth Mechanism: OAuth 2.0 Authorization Code Flow
├─ Flow:
│  1. User redirected to OpenShift OAuth server
│  2. User authenticates with OpenShift credentials
│  3. OAuth server redirects to /oauth/callback with code
│  4. OAuth proxy exchanges code for token
│  5. Proxy sets session cookie and redirects to original URL
└─ Policy: OpenShift OAuth server validation

Endpoint: /metrics (Prometheus)
├─ Auth Mechanism: None (skipped via --skip-auth-regex)
├─ Enforcement: OAuth Proxy (regex bypass)
└─ Purpose: Health monitoring by Prometheus scraper
