AUTHENTICATION & AUTHORIZATION
═══════════════════════════════════════════════════════════════════════════

Inference Endpoints (/v1/models/*, /v2/models/*)
  - Default: No authentication (open)
  - Optional: Can be secured with Istio AuthorizationPolicy + Authorino
  - Token validation: Bearer token via RequestAuthentication
  - Enforcement point: Istio ingress gateway

Webhook Endpoints (/mutate-*, /validate-*)
  - Auth: mTLS (Kubernetes API Server client certificate)
  - Enforcement point: Kubernetes API Server
  - Policy: Only K8s API server can call webhooks

Controller Metrics (/metrics on 8080/TCP)
  - Auth: None
  - Exposure: Internal only (ClusterIP service)
  - Scraping: Prometheus with service discovery

K8s API Access (Controller Manager)
  - Auth: Service Account Token (kserve-controller-manager)
  - RBAC: kserve-manager-role (ClusterRole)
  - Permissions: Full CRUD on KServe CRDs, Knative Services, Istio
                 VirtualServices, K8s Deployments/Services/ConfigMaps/Secrets

═══════════════════════════════════════════════════════════════════════════
DATA CLASSIFICATION
═══════════════════════════════════════════════════════════════════════════

In Transit:
  - Client → Istio Gateway: HTTPS/TLS 1.2+ (encrypted)
  - Istio Gateway → Services: HTTP with mTLS (encrypted)
  - Services ↔ Services: HTTP with mTLS (encrypted)
  - Pods → S3/GCS: HTTPS/TLS 1.2+ (encrypted)

At Rest:
  - Model artifacts in S3/GCS: Encrypted by cloud provider (SSE)
  - Model cache in emptyDir: No encryption (ephemeral storage)
  - Secrets in etcd: Encrypted at rest (K8s encryption provider)
  - Model files on PVC: Depends on storage class encryption

Credentials:
  - Storage credentials: Stored in Secrets (encrypted in etcd)
  - Service account tokens: Auto-generated, mounted as volumes
  - TLS certificates: Stored in Secrets, auto-rotated by cert-manager
