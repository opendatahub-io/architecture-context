## Purpose
**Short**: Manages the lifecycle of Model Registry and Model Catalog instances on OpenShift/Kubernetes clusters.

**Detailed**: The Model Registry Operator is a Kubernetes operator built with controller-runtime that manages ModelRegistry custom resources. Each ModelRegistry CR represents a metadata registry service backed by either PostgreSQL or MySQL, providing REST API endpoints for tracking ML model artifacts, versions, and metadata. The operator dynamically creates Deployments, Services, ServiceAccounts, Roles, RoleBindings, Routes, NetworkPolicies, and authentication proxies (kube-rbac-proxy) per registry instance.

The operator also includes a Model Catalog controller that manages a singleton catalog service for browsing curated model metadata. The catalog is backed by its own PostgreSQL database and sources model metadata from configurable catalog data images. Both registry and catalog services use kube-rbac-proxy sidecars for authentication (migrated from the legacy oauth-proxy pattern), exposing HTTPS endpoints via OpenShift Routes with TLS reencrypt termination.

The operator supports multiple cluster environments: traditional OpenShift, BYOIDC-enabled OpenShift 4.20+ (where user.openshift.io APIs are absent), and plain Kubernetes, using runtime capability detection to conditionally create platform-specific resources.

| catalog-data-init | Init Container | Copies curated model metadata YAML files from catalog data image into shared volume |
| benchmark-data-init | Init Container | Copies performance benchmark data from benchmark data image into shared volume |
| PostgreSQL | Database (optional) | Auto-provisioned PostgreSQL 16 database for model registry or catalog storage |
