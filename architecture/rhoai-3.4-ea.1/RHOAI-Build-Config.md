## Purpose
**Short**: Central build configuration repository that defines, assembles, and releases the RHOAI operator bundle and OLM catalog for Red Hat OpenShift AI.

**Detailed**: RHOAI-Build-Config is the build-time orchestration hub for Red Hat OpenShift AI (RHOAI). It does not contain operator source code or controller logic; instead, it aggregates container image references, CRD manifests, RBAC definitions, and CSV (ClusterServiceVersion) metadata from 43 upstream source repositories into a single OLM (Operator Lifecycle Manager) operator bundle. This bundle is the installable artifact that OLM deploys onto OpenShift clusters.

The repository manages the complete build-to-release pipeline via Konflux (Red Hat's CI/CD platform). It contains Tekton PipelineRun definitions that build the operator bundle image and FBC (File-Based Catalog) fragments for each supported OCP version (v4.19, v4.20, v4.21). The catalog fragments are what make the `rhods-operator` discoverable and installable through the OpenShift OperatorHub.

Beyond build orchestration, this repository serves as the version-pinning source of truth for all 84+ container images shipped in RHOAI 3.4.0-ea.1. Each image is pinned by SHA256 digest in the bundle-patch.yaml, additional-images-patch.yaml, and CSV relatedImages sections, ensuring reproducible and auditable builds suitable for disconnected and FIPS-compliant environments.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| bundle/Dockerfile | OLM Bundle Image | Builds the `odh-operator-bundle` image from scratch, embedding manifests, metadata, and git provenance labels |
| bundle/manifests/ | OLM Bundle Manifests | Contains CRDs, CSV, RBAC, and Service manifests that OLM installs on the cluster |
| bundle/bundle-patch.yaml | Image Pin Configuration | Defines version (3.4.0-ea.1) and pins 84 RELATED_IMAGE references by SHA256 digest for the bundle |

| catalog/ | FBC Catalog Fragments | Per-OCP-version (v4.19/v4.20/v4.21) Dockerfiles and catalog data for the `rhods-operator` OLM catalog |
| catalog/catalog-patch.yaml | Catalog Configuration | Defines OLM package name (`rhods-operator`), default channel (`stable`), beta channel entry, and skip range |

| .github/workflows/ | GitHub Actions | Workflow automations for insta-merge, bundle/FBC processing, stage push, and nightly build triggers |
| integration-tests/ | Enterprise Contract | Tekton pipeline for verifying container images against Enterprise Contract policies |

### Ingress

This repository does not define ingress resources. The operator deployed by this bundle (rhods-operator) manages Gateway API infrastructure (RHOAI 3.x pattern) through its controller code, including:
- Gateway API (GatewayClass, Gateway, HTTPRoute) creation via `gateway.networking.k8s.io` RBAC
- OpenShift Route management via `route.openshift.io` RBAC
- Istio EnvoyFilter and DestinationRule via `networking.istio.io` RBAC
