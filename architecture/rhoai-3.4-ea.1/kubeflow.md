## Purpose
**Short**: A Kubernetes controller and mutating webhook that extends Kubeflow Notebook CRs with OpenShift/RHOAI-specific features including Gateway API ingress, kube-rbac-proxy authentication sidecars, certificate management, and pipeline integration.

**Detailed**: The ODH Notebook Controller is a supplementary controller that watches Kubeflow `Notebook` CRs (managed by the upstream Kubeflow notebook-controller) and reconciles additional OpenShift-specific resources for each notebook workbench. In RHOAI 3.x, the controller creates `HTTPRoute` CRs in a central namespace referencing the platform's `data-science-gateway` Gateway for ingress, along with `ReferenceGrant` CRs to allow cross-namespace backend references. It manages `NetworkPolicy` resources to secure notebook pod traffic and handles certificate trust bundles (`workbench-trusted-ca-bundle` ConfigMap) by aggregating certificates from ODH, self-signed, and service CA sources.

The controller includes a mutating admission webhook that intercepts Notebook CR create/update operations. The webhook injects `kube-rbac-proxy` sidecar containers for authentication (when the `notebooks.opendatahub.io/inject-auth` annotation is set), resolves container images from OpenShift ImageStreams, mounts CA certificate bundles, injects Elyra pipeline runtime configurations, mounts Feast feature store configs, and optionally injects cluster-wide proxy environment variables. The webhook also implements a "reconciliation lock" pattern to prevent race conditions during notebook startup and blocks webhook-initiated pod spec changes on running notebooks to avoid unintended restarts.

### Ingress (Created Dynamically Per Notebook)

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| nb-{namespace}-{notebook} | HTTPRoute (Gateway API) | Inherited from data-science-gateway | 8888/TCP (non-auth) or 8443/TCP (auth) | HTTP/HTTPS | TLS terminated at Gateway | SIMPLE (Gateway-level) | External via Gateway |

**HTTPRoute Routing Details**:
- Path match: `/notebook/{namespace}/{notebook-name}` (PathPrefix)
- Parent reference: `data-science-gateway` in `openshift-ingress` namespace (configurable via `NOTEBOOK_GATEWAY_NAME` and `NOTEBOOK_GATEWAY_NAMESPACE` env vars)
- Non-auth mode: Backend refs point to notebook Service on port 8888
- Auth mode: Backend refs point to `{notebook}-kube-rbac-proxy` Service on port 8443
- HTTPRoutes are created in the controller's central namespace (not user namespace) for security
- ReferenceGrant (`notebook-httproute-access`) in user namespace allows cross-namespace backend references
