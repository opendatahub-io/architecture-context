## Purpose
**Short**: A containerized runtime for batch-oriented language model evaluation using the EleutherAI lm-evaluation-harness framework, integrated with Red Hat's EvalHub evaluation orchestration system.

**Detailed**: The lm-evaluation-harness component provides a secure, containerized execution environment for evaluating language models against standardized benchmarks. It is based on the upstream EleutherAI lm-evaluation-harness (v0.4.8) and is packaged as a Kubernetes Job container image (`odh-ta-lmes-job-rhel9`). The container runs one-shot evaluation workloads: given a job specification (via ConfigMap-mounted JSON), it loads benchmark tasks, sends inference requests to an external model endpoint (via OpenAI-compatible completions API), computes evaluation metrics, and persists results as OCI artifacts to a container registry.

The RHOAI fork extends the upstream framework with an EvalHub adapter (`main.py`) that integrates with the `eval-hub-sdk` for job lifecycle management, callback-based status reporting, and OCI artifact persistence via `skopeo` and `olot`. It also includes Unitxt framework integration for structured evaluation tasks, offline metrics support, and multi-architecture container builds (amd64, arm64, ppc64le, s390x) with custom-compiled PyTorch and Apache Arrow for non-x86 platforms.
