## Purpose
**Short**: A Kubernetes controller that manages the lifecycle and platform integration of KServe model serving deployments on OpenShift AI (RHOAI).

**Detailed**: The odh-model-controller bridges the gap between KServe's model serving infrastructure and the OpenShift AI platform. It watches KServe InferenceService, ServingRuntime, InferenceGraph, and LLMInferenceService custom resources and reconciles platform-specific resources needed for model serving in RHOAI.

The controller manages several critical functions: (1) Creating and managing OpenShift Routes for exposing model inference endpoints, (2) Managing Kuadrant AuthPolicy and Istio EnvoyFilter resources for LLMInferenceService authentication/authorization on Gateway API, (3) Aggregating data connection secrets into KServe-compatible storage-config secrets, (4) Managing custom CA certificate bundles for model storage access, (5) Managing Ray TLS certificates for multi-node (distributed) serving runtimes, (6) Creating monitoring infrastructure (ServiceMonitor, PodMonitor, metrics dashboards) for inference services, (7) Managing KEDA autoscaling integration, and (8) Managing NVIDIA NIM integration through the Account CRD.

The controller also deploys ServingRuntime templates for various ML inference runtimes including vLLM (CUDA, ROCm, Gaudi, CPU, Spyre), OpenVINO Model Server (OVMS), MLServer, and Hugging Face guardrails detector. These templates are deployed as kustomize resources and their container images are injected by the rhods-operator at deploy time.

**Controller-Managed Ingress (RHOAI 3.x patterns)**:

- **InferenceService Routes (RawDeployment mode)**: The InferenceService controller creates OpenShift Route CRs per InferenceService when the label `networking.kserve.io/visibility=exposed` is set. Routes use either TLS Edge termination (no auth) or TLS Reencrypt termination (with auth via `security.opendatahub.io/enable-auth=true` annotation). Routes target predictor or transformer Services created by KServe.

- **LLMInferenceService AuthPolicy + EnvoyFilter (Gateway API)**: The LLMInferenceService controller creates Kuadrant AuthPolicy CRs targeting both Gateway and HTTPRoute objects for authentication enforcement via Authorino. It also creates Istio EnvoyFilter CRs on Gateways for Authorino TLS bootstrap when the `security.opendatahub.io/authorino-tls-bootstrap` annotation is present. The default gateway is `openshift-ai-inference` in namespace `openshift-ingress`.

## Kustomize Deployment

The component is deployed by rhods-operator using kustomize overlay at `config/base/`.

**Overlay structure**: `config/base/kustomization.yaml` references `config/default/` which includes:
- `config/crd/` - NIM Account CRD
- `config/rbac/` - ClusterRole, ClusterRoleBinding, ServiceAccount, leader election
- `config/manager/` - Deployment (1 replica, 500m CPU / 2Gi memory limits)
- `config/webhook/` - MutatingWebhookConfiguration, ValidatingWebhookConfiguration, webhook Service
- `config/prometheus/` - ServiceMonitor for controller metrics
- `config/runtimes/` - ServingRuntime templates (vLLM variants, OVMS, MLServer, HF detector)

**Image injection**: The rhods-operator injects 11 image references via params.env → ConfigMap replacements:
- `odh-model-controller` - Controller image
- `ovms-image`, `mlserver-image` - Model server images for runtime templates
- `vllm-cuda-image`, `vllm-cpu-image`, `vllm-cpu-x86-image`, `vllm-gaudi-image`, `vllm-rocm-image`, `vllm-spyre-image` - vLLM variant images
- `guardrails-detector-huggingface-runtime-image` - Guardrails detector image

**Feature flags** (env vars on Deployment):
- `NIM_STATE` (default: managed) - Enables/disables NIM Account controller
- `KSERVE_STATE` (default: managed) - KServe integration state
- `MODELREGISTRY_STATE` (default: removed) - Enables/disables Model Registry reconciliation
