## Purpose
**Short**: A Kubernetes operator that manages the lifecycle and integration of KServe model serving InferenceServices with OpenShift and OpenDataHub platform features.

**Detailed**: The odh-model-controller is a controller-runtime based Go operator that bridges KServe model serving with the Red Hat OpenShift AI (RHOAI) platform. It watches KServe InferenceService, ServingRuntime, InferenceGraph, and LLMInferenceService custom resources and reconciles supplementary platform resources such as OpenShift Routes for external access, NetworkPolicies for network isolation, ServiceMonitors for Prometheus metrics scraping, storage configuration secrets from data connections, and CA certificate bundles for TLS trust chains.

The controller also manages NVIDIA NIM (NVIDIA Inference Microservice) integration through the `Account` CRD, which handles API key validation, model catalog configuration, pull secrets, and OpenShift Template creation for NIM-based model serving. For the newer LLMInferenceService CRD, the controller manages Kuadrant AuthPolicy resources for authentication/authorization via Gateway API, Istio EnvoyFilters for TLS bootstrap between gateways and Authorino, and RBAC resources for KEDA-based autoscaling. Multi-node (Ray-based) serving runtimes are supported with automatic TLS certificate generation and distribution across worker pods.

**Controller-Managed Ingress (RHOAI 3.x LLMInferenceService)**:
- The LLMInferenceService controller creates **Kuadrant AuthPolicy** CRs targeting both Gateway and HTTPRoute resources for authentication/authorization
- AuthPolicies reference the platform Gateway (default: `openshift-ai-inference` in `openshift-ingress` namespace)
- The controller creates **Istio EnvoyFilter** CRs for TLS bootstrap between the gateway Envoy proxy and Authorino authorization service
- Gateway-level AuthPolicy enforces user-defined authentication; HTTPRoute-level AuthPolicy enables anonymous access where configured
- The InferenceService controller creates **OpenShift Route** CRs per InferenceService with `networking.kserve.io/visibility=exposed` label (RHOAI 2.x pattern, still active)
