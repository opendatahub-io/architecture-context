## Purpose
**Short**: Kubeflow Trainer is a Kubernetes-native operator for distributed machine learning training job orchestration, supporting PyTorch, MPI, DeepSpeed, and other ML frameworks.

**Detailed**: Kubeflow Trainer (v2) provides a declarative API for submitting and managing distributed training workloads on Kubernetes. Users define `TrainJob` custom resources that reference `TrainingRuntime` or `ClusterTrainingRuntime` templates, which specify the ML framework configuration (PyTorch torchrun, MPI, etc.), the number of training nodes, and pod group policies for gang-scheduling.

The controller reconciles TrainJob resources by creating `JobSet` objects (from the Kubernetes sigs.k8s.io/jobset project) that manage the lifecycle of training pods. It supports PyTorch distributed training with elastic scaling, MPI-based training with SSH key management, and integration with gang-scheduling systems (Kubernetes scheduler-plugins coscheduling, Volcano). The RHOAI distribution adds progression tracking capabilities that poll training pod metrics endpoints to provide real-time training progress annotations on TrainJob resources, along with NetworkPolicy-based pod isolation for security hardening.

The operator also includes a built-in certificate management system for webhook TLS, validating webhooks for all CRD types, and pre-configured ClusterTrainingRuntime resources with CUDA and ROCm GPU-accelerated training images for RHOAI deployments.

### Ingress

This component does not expose external ingress routes. All communication is cluster-internal:
- Kubernetes API server → webhook endpoint (443→9443/TCP, TLS)
- Prometheus → metrics endpoint (8443/TCP, HTTPS)
- Controller → training pod metrics (28080/TCP, HTTP, restricted by NetworkPolicy)

### Egress

| Destination | Port | Protocol | Encryption | Auth | Purpose |
|-------------|------|----------|------------|------|---------|
| Kubernetes API server | 443/TCP | HTTPS | TLS 1.2+ | ServiceAccount token | CRUD operations on JobSets, PodGroups, NetworkPolicies, Secrets, ConfigMaps |
| Training pod IP (in-cluster) | 28080/TCP (configurable) | HTTP | plaintext | NetworkPolicy | Poll training progress metrics (RHAI progression tracking) |

### Network Policies (Controller-Created)

The controller dynamically creates a `NetworkPolicy` per TrainJob for pod isolation:

| Policy Name | Namespace | Pod Selector | Ingress Rules | Purpose |
|-------------|-----------|-------------|---------------|---------|
| `<trainjob-name>` | TrainJob namespace | `jobset.sigs.k8s.io/jobset-name: <trainjob-name>` | Rule 1: Same-job pods → all ports (NCCL/MPI/gRPC). Rule 2 (if progression enabled): Controller namespace pods → metrics port (TCP) | Training pod isolation and metrics endpoint security |
