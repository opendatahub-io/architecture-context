## Purpose
**Short**: Kubeflow Trainer is a Kubernetes-native operator for managing distributed machine learning training jobs across frameworks like PyTorch, JAX, TensorFlow, DeepSpeed, and Megatron-LM.

**Detailed**: Kubeflow Trainer (v2) provides a unified API for defining, scheduling, and managing distributed training workloads on Kubernetes. Users create `TrainJob` custom resources that reference pre-defined `TrainingRuntime` or `ClusterTrainingRuntime` templates, which define the compute topology, ML framework configuration, and container images needed for training. The controller reconciles these into underlying `JobSet` resources for actual workload execution.

The RHOAI distribution includes Red Hat-specific enhancements: RHAI progression tracking (real-time training metrics polling via HTTP from worker pods), automatic NetworkPolicy creation for pod isolation and metrics endpoint security, FIPS-compliant builds, and pre-configured ClusterTrainingRuntime definitions for PyTorch distributed training with CUDA and ROCm GPU support. The controller manages its own webhook certificates and exposes Prometheus metrics over HTTPS.

Steps: (1) User creates TrainJob CR, (2) Webhook validates the CR, (3) Controller reads TrainJob + referenced runtime, builds JobSet objects, applies them via server-side apply, creates NetworkPolicy, (4) Controller updates TrainJob status based on JobSet status.

### Flow 2: Progression Tracking (RHAI Feature)

| Step | Source | Destination | Port | Protocol | Encryption | Auth |
|------|--------|-------------|------|----------|------------|------|
| 1 | Trainer controller | Kubernetes API server | 443/TCP | HTTPS | TLS 1.2+ | ServiceAccount token |
| 2 | Trainer controller | Training pod (primary worker) | 28080/TCP | HTTP | plaintext | NetworkPolicy |
| 3 | Trainer controller | Kubernetes API server | 443/TCP | HTTPS | TLS 1.2+ | ServiceAccount token |

Steps: (1) Controller lists pods to find primary worker (rank 0) by IP, (2) Controller polls `http://<pod-ip>:28080/metrics` for JSON training status (progress %, steps, epochs, ETA), (3) Controller patches TrainJob annotations with `trainer.opendatahub.io/trainerStatus` JSON. Requeues every 30s (configurable 5s-300s).

## Deployment Configuration

- Sets namespace to `opendatahub`
- Injects container images via `params.env` ConfigMap replacement
- Enables RHAI features (`ENABLE_RHAI_FEATURES=true`)
