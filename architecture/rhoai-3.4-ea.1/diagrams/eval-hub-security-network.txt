RBAC Summary
================================================================================

Service Account: (operator-provisioned, per EvalHub instance)
Provisioned by: TrustyAI Service Operator

  Role (operator-provisioned):
    - API Group: batch
      Resources: jobs
      Verbs: create, delete, get

    - API Group: "" (core)
      Resources: configmaps
      Verbs: create, delete, get, update

Authentication & Authorization:
  - All /api/v1/* endpoints: Bearer Token -> kube-rbac-proxy -> SubjectAccessReview
  - /metrics endpoint: Bearer Token -> kube-rbac-proxy -> SubjectAccessReview
  - Adapter Job callbacks: projected SA token -> kube-rbac-proxy -> SubjectAccessReview

Notes
================================================================================

  - EvalHub does NOT define its own CRDs, ClusterRoles, or NetworkPolicies
  - All RBAC and deployment resources are provisioned by TrustyAI Service Operator
  - The kube-rbac-proxy sidecar provides the HTTPS/TLS termination and auth layer
  - Callback URL format: https://<instance-name>.<namespace>.svc.cluster.local:8443
  - No ingress resources defined; external access managed by platform operator
