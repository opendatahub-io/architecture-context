NetworkPolicy Summary:
When annotation odh.ray.io/secure-trusted-network: "true" is set:

  Head Pod NetworkPolicy ({cluster}-head):
  - Intra-cluster: ray.io/cluster={name} pods → All ports
  - Same-namespace: Any pod → 10001/TCP, 8265/TCP
  - Operator: Operator namespace → 8265/TCP, 10001/TCP
  - Monitoring: MONITORING_NAMESPACE → 8080/TCP
  - Gateway: GATEWAY_NAMESPACE or openshift-ingress → 8265/TCP
  - Auth proxy: Any source → 8443/TCP
  - RayJob submitter: Batch job pods → All ports

  Worker Pod NetworkPolicy ({cluster}-workers):
  - Intra-cluster only: ray.io/cluster={name} pods → All ports

RBAC Summary:
- ClusterRole: kuberay-operator
  - ray.io: rayclusters, rayjobs, rayservices (full CRUD + status + finalizers)
  - core: pods, services, serviceaccounts, configmaps, events, secrets (full CRUD)
  - batch: jobs (full CRUD)
  - networking.k8s.io: ingresses, networkpolicies (full CRUD)
  - route.openshift.io: routes (full CRUD)
  - gateway.networking.k8s.io: gateways (read), httproutes, referencegrants (full CRUD)
  - cert-manager.io: certificates, issuers (full CRUD)
  - rbac.authorization.k8s.io: roles (full CRUD), rolebindings (read + create + delete)
  - authentication.k8s.io: tokenreviews (create)
  - authorization.k8s.io: subjectaccessreviews (create)
  - config.openshift.io: authentications, oauths (read)
  - apps: deployments (read + update + patch)
  - coordination.k8s.io: leases (get, list, create, update)

Secrets:
- {cluster}-ca-secret-{uid} (kubernetes.io/tls) - CA cert, cert-manager, 10yr/30d renewal
- ray-head-secret-{cluster} (kubernetes.io/tls) - Head TLS cert, cert-manager, 90d/15d renewal
- ray-worker-secret-{cluster} (kubernetes.io/tls) - Worker TLS cert, cert-manager, 90d/15d renewal
- {cluster}-oauth-config (Opaque) - OAuth proxy config, Auth Controller, no auto-rotate
- {cluster}-oidc-config (Opaque) - OIDC proxy config, Auth Controller, no auto-rotate
- {cluster}-oauth-tls / {cluster}-oidc-tls (kubernetes.io/tls) - Auth proxy TLS, service-ca, auto-rotate

Authentication Mechanisms:
- Ray Dashboard (HTTPRoute): kube-rbac-proxy → TokenReview + SubjectAccessReview
- Ray Dashboard (Route, legacy): oauth-proxy → OpenShift OAuth
- Ray GCS (intra-cluster): mTLS via cert-manager certificates
- Operator Metrics (8080): No authentication
