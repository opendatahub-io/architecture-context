RBAC Summary:
- No Kubernetes RBAC roles defined
- Component operates as a stateless service
- Does not interact with the Kubernetes API

Authentication & Authorization:
- Tenant isolation via X-MaaS-User HTTP header
- Items scoped by tenant ID; mismatched tenant returns 404
- /health and /metrics endpoints are public (no auth)
- Inference Gateway auth via optional Bearer Token (API key)
- Auth/AuthZ/RateLimit middleware stubs exist but are currently disabled

Secrets:
- {release}-apiserver-tls (kubernetes.io/tls) - API server TLS cert/key, cert-manager, auto-rotate
- {release}-processor-tls (kubernetes.io/tls) - Processor TLS cert/key, manual provisioning
- {database-secret} (Opaque) - Database connection URL for processor, manual provisioning

Container Security:
- runAsNonRoot: true (UID 65532 or 1001)
- allowPrivilegeEscalation: false
- capabilities: drop ALL
- readOnlyRootFilesystem: true
- seccompProfile: RuntimeDefault

TLS Configuration (when enabled):
- Minimum version: TLS 1.2
- Cipher suites:
  - TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
  - TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
  - TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256
  - TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384
  - TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305
  - TLS_ECDHE_ECDSA_WITH_CHACHA20_POLY1305
- mTLS supported for inference gateway communication
