RBAC Summary:
- This component does not define its own RBAC resources.
- Operates within the Llama Stack server's service account context.
- The llama-stack-k8s-operator manages RBAC for the server pod.

Secrets:
- kubeflow-pipelines-token (Opaque) - KFP API auth token, user-provisioned, no auto-rotation
- aws-credentials (Opaque) - AWS credentials for S3 results storage, user-provisioned, no auto-rotation
- llama-stack-inference-model-secret (Opaque) - Inference model URL/token/TLS settings, user-provisioned, no auto-rotation
- S3 credentials secret (configurable name, Opaque) - Mounted as env vars into KFP pods via kubernetes.use_secret_as_env

ConfigMaps:
- trustyai-service-operator-config (ns: redhat-ods-applications / opendatahub)
  - Key: ragas-provider-image → KFP component base image resolution
- kubeflow-ragas-config (ns: user namespace)
  - Keys: KFP endpoint, namespace, base image, S3 prefix

Authentication & Authorization:
- Kubeflow Pipelines API: Bearer Token (OpenShift), enforced by KFP server
- Llama Stack API (from KFP pods): No auth by default
- S3 storage: AWS IAM credentials as env vars from K8s Secret

CRDs Referenced:
- llamastack.io/v1alpha1 LlamaStackDistribution (Namespaced)
  - Owned by: llama-stack-k8s-operator
  - This component is deployed as a provider within the distribution
