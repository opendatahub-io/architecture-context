Authentication Flow:
================================================================================

1. User accesses dashboard via data-science-gateway (HTTPS/443)
2. Gateway terminates external TLS, re-encrypts to kube-rbac-proxy (8443)
3. kube-rbac-proxy validates user token via SubjectAccessReview:
   - Resource: project.openshift.io/projects
   - Verb: list
   - If denied: 403 Forbidden
4. On success, proxies to Fastify on localhost:8080 with:
   - X-Auth-Request-User header
   - X-Auth-Request-Groups header
   - x-forwarded-access-token header (original bearer token)
5. Fastify passes user token to K8s API for all operations (no privilege escalation)
6. Federated modules receive user token via x-forwarded-access-token header
