Security Notes:
- The Llama Stack server itself has NO application-level authentication
- Authentication is delegated to the platform (kube-rbac-proxy sidecar or Gateway API)
- PostgreSQL and PgVector connections use plaintext (no TLS)
- Cloud provider connections all use TLS 1.2+
- vLLM TLS verification is configurable via TLS_VERIFY environment variable
- Milvus supports optional TLS with CA and client certificates
- All secrets are provided as environment variables (no Kubernetes Secret objects defined by this component)
