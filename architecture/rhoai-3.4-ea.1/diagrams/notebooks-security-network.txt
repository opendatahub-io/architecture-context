RBAC Summary:
=============
This component does NOT define any RBAC resources (ClusterRole, Role, RoleBinding).
RBAC for workbench access is managed by:
  - ODH Notebook Controller: Controls access to Notebook CR namespaces
  - rhods-operator: Manages operator-level RBAC

Authentication & Authorization:
===============================
| Endpoint              | Auth Mechanism                         | Enforcement Point                    |
|-----------------------|----------------------------------------|--------------------------------------|
| Workbench UI (all)    | kube-rbac-proxy (RHOAI 3.x)           | Sidecar injected by Notebook Ctrl    |
| Workbench UI (legacy) | oauth-proxy (RHOAI 2.x)               | Sidecar injected by Notebook Ctrl    |
| JupyterLab API        | Token disabled (--ServerApp.token='')  | Delegated to proxy sidecar           |
| code-server           | --auth none                            | Delegated to nginx + proxy sidecar   |

Secrets:
========
This component does NOT define any Secret resources.
Secrets for notebook OAuth credentials are managed by the ODH Notebook Controller.

Container Security:
===================
- Non-root user: UID 1001
- Read-only container layers; writable volumes mounted at runtime
- Hermetic builds via Konflux (no network during build)
- Hash-pinned Python dependencies (pylock.toml with --require-hashes)
- UBI9 base: FIPS compliant
- 95 Tekton PipelineRun definitions for CI/CD

ImageStream Resources:
======================
- 11 Notebook ImageStreams (up to 8 version tags each)
- 7 Runtime ImageStreams (single tag each)
- 2 BuildConfigs (RStudio CPU + CUDA)
- ConfigMaps: notebook-image-params, notebook-image-commithash
