Authentication Mechanisms
================================================================================

1. kube-rbac-proxy (SubjectAccessReview on port 8443)
   Components: odh-dashboard, DSP API Server, model-registry, kuberay,
               eval-hub, MLMD Envoy, WVA metrics
   Enforcement: Sidecar container

2. kube-auth-proxy (OAuth2/OIDC)
   Components: odh-dashboard, notebooks (RHOAI 3.x)
   Enforcement: Sidecar proxy with OpenShift OAuth

3. Kubernetes ServiceAccount Token
   Components: All operators, DSP persistence agent, scheduled workflow,
               pipeline tasks
   Enforcement: Projected volumes / automounted tokens

4. Bearer Token (API key)
   Components: vllm-cpu, vllm-gaudi, batch-gateway, MaaS API keys
   Enforcement: Application-level

5. Kuadrant AuthPolicy
   Components: models-as-a-service, LLMInferenceService
   Enforcement: Gateway API integration

6. Webhook TLS (K8s API server)
   Components: kserve, trainer, training-operator, spark-operator, DSPO,
               model-registry-operator
   Enforcement: Admission webhook certs

7. mTLS Client Certificates (when Istio enabled)
   Components: kserve, ml-metadata
   Enforcement: Service mesh / application
