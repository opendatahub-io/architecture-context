NETWORK POLICIES:
- Policy: maas-authorino-allow
  - Applied To: pods with app.kubernetes.io/part-of: models-as-a-service
  - Ingress From: Authorino pods in kuadrant-system or openshift-operators
  - Purpose: Allow Authorino to reach MaaS API for AuthPolicy evaluation

RBAC Summary:
- ClusterRole: maas-api
  - core: configmaps (get, list, watch, create, update, patch, delete)
  - core: namespaces (get, list, watch, create)
  - core: serviceaccounts (get, list, watch, create, delete)
  - core: serviceaccounts/token (create)
  - authentication.k8s.io: tokenreviews (create)
  - serving.kserve.io: inferenceservices, llminferenceservices (get, list, watch)
  - gateway.networking.k8s.io: httproutes (get, list, watch)
  - core: pods, services, endpoints (get, list, watch)
- ClusterRoleBinding: maas-api -> ServiceAccount: maas-api (cluster-scoped)

Secrets:
- maas-api-serving-cert (kubernetes.io/tls) - TLS cert for HTTPS, OpenShift service-ca, auto-rotate
- ${CERT_NAME} (kubernetes.io/tls) - Gateway TLS cert, cert-manager or manual, depends on provisioner

Authentication & Authorization:
- Gateway: Kubernetes TokenReview (audience: maas-default-gateway-sa), cached 600s
- Gateway: SubjectAccessReview (post on llminferenceservices in target namespace)
- MaaS API routes: TokenReview with X-MaaS-Username/X-MaaS-Group header injection
- /v1/tiers/lookup: No auth (cluster-internal only, Authorino callback)
- /health: No auth (health check)

Observability:
- ServiceMonitor: limitador-metrics (kuadrant-system, /metrics, 30s)
- Istio Telemetry: latency-per-tier (openshift-ingress, tier label on REQUEST_DURATION)
- TelemetryPolicy: user-group (openshift-ingress, model/tier/user labels)
- Grafana Dashboards: dashboard-ai-engineer, dashboard-platform-admin
