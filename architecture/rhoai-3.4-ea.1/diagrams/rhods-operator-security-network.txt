SERVICE MESH CONFIGURATION:
==============================================================================

  EnvoyFilter: data-science-authn-filter
  - Namespace: openshift-ingress
  - Applied to: Gateway Envoy pods
  - Filters: ext_authz (kube-auth-proxy) + Lua (header processing)
  - Purpose: Enforce authentication on all HTTPRoutes through the Gateway

  DestinationRule: data-science-tls-rule
  - TLS Mode: ISTIO_MUTUAL
  - Purpose: mTLS between Gateway and kube-auth-proxy

AUTHENTICATION MECHANISMS:
==============================================================================

  1. Gateway (all HTTPRoutes):
     - OAuth2/OIDC via kube-auth-proxy (EnvoyFilter ext_authz)
     - Kubernetes ServiceAccount tokens (TokenReview validation)
     - Bearer token forwarded to upstream after auth

  2. Webhook endpoints (9443/TCP):
     - mTLS with Kubernetes API server
     - API server validates client cert

  3. Operator metrics (8080/TCP):
     - No authentication (internal ClusterIP only)
