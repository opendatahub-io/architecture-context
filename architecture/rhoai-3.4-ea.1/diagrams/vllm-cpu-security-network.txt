RBAC Summary:
- No RBAC definitions. vLLM-CPU is a user-space application container, not an operator.
- RBAC is managed by KServe and the platform operator (rhods-operator).

Platform Ingress (managed by KServe, NOT by vLLM-CPU):
- KServe controller creates HTTPRoute CRs (RHOAI 3.x pattern)
- Data-science-gateway (Gateway API) handles external routing
- kube-rbac-proxy sidecars injected for authentication

Security Controls:
- Container runs as non-root user (UID 2000)
- Built on UBI 9 (ubi-minimal) base image
- TLS configurable via --ssl-keyfile, --ssl-certfile, --ssl-ca-certs
- Certificate hot-reload via SSLCertRefresher (file system watcher)
- Bearer token auth uses SHA-256 comparison (timing-safe)
- gRPC port has NO authentication - relies on network-level isolation

Secrets:
- ssl-keyfile (File mount) - TLS private key, provisioned by Platform/User, auto-rotates
- ssl-certfile (File mount) - TLS certificate, provisioned by Platform/User, auto-rotates
- ssl-ca-certs (File mount) - CA certificates, provisioned by Platform/User, auto-rotates
- api-key (Environment variable) - Bearer token for /v1/* auth, provisioned by User, no auto-rotation
