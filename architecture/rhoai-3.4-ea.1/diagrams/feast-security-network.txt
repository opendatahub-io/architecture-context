Operator ClusterRoles:
- manager-role
  - feast.dev: featurestores, featurestores/status, featurestores/finalizers
    → get, list, watch, create, update, patch, delete
  - apps: deployments → get, list, create, update, watch, delete
  - "": services, configmaps, persistentvolumeclaims, serviceaccounts
    → get, list, create, update, watch, delete
  - "": secrets, pods, namespaces → get, list, watch
  - "": pods/exec → create
  - rbac.authorization.k8s.io: roles, rolebindings, clusterroles,
    clusterrolebindings, subjectaccessreviews
    → get, list, create, update, watch, delete
  - authentication.k8s.io: tokenreviews → create
  - route.openshift.io: routes → get, list, create, update, watch, delete
  - batch: cronjobs → get, list, watch, create, update, patch, delete
  - kubeflow.org: notebooks → get, list, watch
  - apiextensions.k8s.io: customresourcedefinitions → get, list

- metrics-auth-role
  - authentication.k8s.io: tokenreviews → create
  - authorization.k8s.io: subjectaccessreviews → create

- metrics-reader
  - /metrics (non-resource URL) → get

Dynamically Created RBAC (per FeatureStore):
- feast-token-review-cluster-role (ClusterRole)
  - rbac.authorization.k8s.io: rolebindings → list
  - authentication.k8s.io: tokenreviews → create
  - rbac.authorization.k8s.io: subjectaccessreviews → create
  - "": namespaces → get, list, watch
  - rbac.authorization.k8s.io: clusterroles, clusterrolebindings → get, list

- feast-{name} (Role, per namespace)
  - rbac.authorization.k8s.io: roles, rolebindings → get, list, watch
  - authentication.k8s.io: tokenreviews → create
  - rbac.authorization.k8s.io: subjectaccessreviews → create
  - "": namespaces → get, list, watch

Role Bindings:
- manager-rolebinding (cluster) → manager-role → feast-operator-controller-manager
- feast-{name} (namespace) → feast-{name} Role → feast-{name} SA
- feast-{name}-cluster-binding (cluster) → feast-token-review-cluster-role → feast-{name} SA
- feast-configs-registry-reader (operator ns) → feast-configs-registry-reader → system:authenticated

OpenShift TLS Configuration:
- Services annotated with service.beta.openshift.io/serving-cert-secret-name
- Dual-protocol registry uses SANs via service.beta.openshift.io/serving-cert-sans
- CA ConfigMap annotated with service-ca.crt for TLS verification
- TLS secrets and CA bundles mounted into feature server containers

Secrets:
- feast-{name}-{service}-tls (kubernetes.io/tls) - OpenShift service serving certs, auto-rotated
- User-provided DB secrets (Opaque) - Database connection parameters, no auto-rotation
- User-provided OIDC secret (Opaque) - OIDC client credentials, no auto-rotation
- User-provided TLS secrets (kubernetes.io/tls) - Custom TLS certs, rotation varies
