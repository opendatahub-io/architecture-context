Authentication & Authorization Summary:
- Ray Dashboard API: No auth (cluster-internal, accessed via port-forward/Route)
- S3 Storage: AWS IAM credentials (Access Key + Secret Key, SigV4)
- OpenShift API: Bearer Token / kubeconfig, enforced by Kubernetes RBAC
- HuggingFace Hub: Bearer Token (HF_TOKEN)

Network Security Notes:
- NCCL inter-pod communication uses plaintext TCP (no encryption)
- Ray internal communication (GCS 6379, Client 10001) uses plaintext TCP
- Ray Dashboard (8265) exposed without authentication in test environments
- All external egress uses HTTPS/TLS 1.2+
- No service mesh (mTLS) configuration documented for training pods
- Training pod network isolation relies on namespace-level network policies
