  - Client CA: --tls-client-ca-cert-path (optional, enables mTLS)
  - Client verification: WebPKI
- Client TLS (outbound): Per-service YAML configuration
  - Named TLS configs for reuse across services
  - Supports: cert_path, key_path, client_ca_cert_path, insecure mode
  - gRPC: tonic TLS with native roots + WebPKI roots
  - HTTP: hyper-rustls with HTTP/1.1 and HTTP/2
