  - Auth: Kubernetes API server TLS client cert

  - Endpoint: /webhooks/mutate-pipelineversion

Network Policies:
=================

Policy: default-allow-same-namespace
  - Namespace: kubeflow
  - Pod Selector: All pods ({})
  - Ingress From: Same namespace pods
  - Purpose: Allow all intra-namespace pod communication

Secrets:
========

- mysql-secret (Opaque)
  Purpose: Database credentials (username/password) for MySQL/PostgreSQL
  Provisioned By: Administrator / DSP Operator
  Auto-Rotate: No

- mlpipeline-minio-artifact (Opaque)
  Purpose: Object storage credentials (accesskey/secretkey) for MinIO/S3
  Provisioned By: Administrator / DSP Operator
  Auto-Rotate: No

- webhook-server-tls (kubernetes.io/tls)
  Purpose: TLS certificates for cache server webhook
  Provisioned By: cert-manager / cache-deployer
  Auto-Rotate: No

- persistenceagent-sa-token (Projected ServiceAccount Token)
  Purpose: SA token for persistence agent API calls
  Audience: pipelines.kubeflow.org
  Expires: 3600s
  Auto-Rotate: Yes (Kubernetes managed)

- scheduledworkflow-sa-token (Projected ServiceAccount Token)
  Purpose: SA token for scheduled workflow controller API calls
  Audience: pipelines.kubeflow.org
  Expires: 3600s
  Auto-Rotate: Yes (Kubernetes managed)

Authentication & Authorization:
===============================

- /apis/* (API Server): Bearer Token (ServiceAccount) → SubjectAccessReview / TokenReview via K8s API
- /webhooks/* (Webhook): Kubernetes API server TLS client cert → AdmissionWebhookConfiguration
- cache-server:443: Kubernetes API server TLS client cert → MutatingWebhookConfiguration
