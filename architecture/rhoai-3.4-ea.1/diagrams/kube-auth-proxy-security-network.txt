RBAC Summary:

  kube-auth-proxy (auth mode):
  - API Group: authentication.k8s.io
    Resources: tokenreviews
    Verbs: create
    Purpose: Validate Kubernetes service account tokens

  kube-rbac-proxy (rbac mode):
  - API Group: authentication.k8s.io
    Resources: tokenreviews
    Verbs: create
    Purpose: Authenticate bearer tokens via delegating authenticator

  - API Group: authorization.k8s.io
    Resources: subjectaccessreviews
    Verbs: create
    Purpose: Authorize requests via SubjectAccessReview

  Note: kube-auth-proxy does not install its own ClusterRoles.
  RBAC permissions must be provisioned by the deploying operator.

Authentication & Authorization:

  Auth Mode (kube-auth-proxy):
  - Methods: OAuth2 Cookie Session, Bearer Token (JWT), K8s SA Token (TokenReview)
  - Enforcement: kube-auth-proxy process
  - Policy: Email domain validation, htpasswd, authenticated-emails-file
  - ext_authz: /oauth2/auth returns 200 (authenticated) or 401 (not authenticated)

  RBAC Mode (kube-rbac-proxy):
  - Methods: Delegating Authenticator (Bearer via TokenReview), Client Cert (X.509), OIDC JWT
  - Enforcement: kube-rbac-proxy process
  - Policy: SubjectAccessReview against Kubernetes API, Static Auth Config

Secrets:

  - (configurable) cookie-secret (Opaque)
    Purpose: Cookie encryption key for session cookies
    Provisioned by: Deployer/Operator
    Auto-Rotate: No

  - (configurable) client-secret (Opaque)
    Purpose: OAuth2 client secret for OIDC/OpenShift provider
    Provisioned by: Deployer/Operator
    Auto-Rotate: No

  - (configurable) tls-cert/key (kubernetes.io/tls)
    Purpose: TLS certificate and key for HTTPS listeners
    Provisioned by: cert-manager/Operator
    Auto-Rotate: Depends on provisioner

  - jwt_signing_key.pem (Opaque, baked into image)
    Purpose: JWT signing key placeholder (empty by default)
    Provisioned by: Build process
    Auto-Rotate: No

  - /var/run/secrets/kubernetes.io/serviceaccount/token
    Purpose: Kubernetes API authentication (in-cluster)
    Provisioned by: Kubernetes
    Auto-Rotate: Yes (auto-mounted)

  - /var/run/secrets/kubernetes.io/serviceaccount/ca.crt
    Purpose: TLS verification for Kubernetes API
    Provisioned by: Kubernetes
    Auto-Rotate: No
