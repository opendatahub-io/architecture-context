RBAC Summary:
================================================================================
MLServer does not define any Kubernetes RBAC resources.
RBAC is managed by KServe and the RHOAI platform.

Authentication Summary:
================================================================================
- MLServer runs WITHOUT authentication at the container level
- All auth is enforced externally:
  - RHOAI 3.x: kube-rbac-proxy sidecar or Gateway API-level auth
  - RHOAI 2.x: oauth-proxy sidecar or Istio AuthorizationPolicy

Secrets:
================================================================================
MLServer does not define or require any Kubernetes Secrets directly.
Authentication and TLS are handled at the platform level by KServe and
the RHOAI gateway infrastructure.

Security Configuration:
================================================================================
- CORS: Disabled by default (MLSERVER_CORS_ALLOW_ORIGINS=[])
- GZip: Enabled by default (MLSERVER_GZIP_ENABLED=true)
- Debug logging: Enabled by default (MLSERVER_DEBUG=true)
- All ports listen on 0.0.0.0 (all interfaces within pod)
- No TLS/mTLS configured at MLServer level (handled by platform)
