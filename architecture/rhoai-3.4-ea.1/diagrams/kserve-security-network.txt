- openshift-ai-llminferenceservice-scc (priority: 10)
  - Capabilities: IPC_LOCK, SYS_RAWIO, NET_BIND_SERVICE, NET_RAW
  - Purpose: Standard LLMInferenceService workloads (GPU memory locking)

- openshift-ai-llminferenceservice-multi-node-scc (priority: 11)
  - Capabilities: IPC_LOCK, SYS_RAWIO, NET_BIND_SERVICE, NET_RAW + hostPath volumes
  - Purpose: Multi-node LLM workloads requiring host directory access

- kserve-webhook-server-cert (kubernetes.io/tls) - TLS cert for webhook, OpenShift service-ca, auto-rotate
- kserve-webhook-server-secret (Opaque) - Legacy webhook secret placeholder, no auto-rotate
- storage-config (Opaque) - S3/GCS/Azure storage credentials, user-provisioned, no auto-rotate
- {llmisvc-name}-self-signed-tls (kubernetes.io/tls) - Self-signed TLS for LLM workloads, controller-generated, no auto-rotate

Authentication & Authorization:
- Webhook endpoints: mTLS (API server client cert) enforcement at kube-apiserver
- InferenceService endpoints: kube-rbac-proxy sidecar (when enabled via oauthProxy config) → Token review + SAR
- LLMInferenceService endpoints: Per-service auth annotation (serving.kserve.io/enable-auth), disabled by default
