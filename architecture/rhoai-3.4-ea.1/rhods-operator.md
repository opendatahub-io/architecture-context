## Purpose
**Short**: Platform operator that manages the lifecycle of all Red Hat OpenShift AI (RHOAI) data science components, platform services, and ingress infrastructure.

**Detailed**: The RHODS Operator (also known as the OpenDataHub Operator) is the central orchestrator for the RHOAI platform. It provides two primary Custom Resources -- `DSCInitialization` for platform-level initialization (namespaces, monitoring, authentication, gateway configuration) and `DataScienceCluster` for enabling and managing individual data science components (Dashboard, Workbenches, KServe, Model Registry, Pipelines, Ray, Kueue, TrustyAI, Training Operator, etc.).

In RHOAI 3.x, the operator introduces the Gateway API-based ingress architecture. The Gateway controller deploys a `data-science-gateway` Gateway resource in the `openshift-ingress` namespace, an EnvoyFilter for authentication via ext_authz, a `kube-auth-proxy` deployment for OAuth2/OIDC authentication, and associated HTTPRoutes. Components create HTTPRoute CRs that reference this central gateway, replacing the legacy per-component OpenShift Route + oauth-proxy pattern from RHOAI 2.x.

The operator also manages platform services including Auth (RBAC group management), Monitoring (MonitoringStack, Tempo, Perses, OpenTelemetry), and certificate ConfigMap propagation across namespaces. It deploys component manifests from prefetched kustomize bundles stored in `prefetched-manifests/`.

**Gateway API Architecture (RHOAI 3.x)**:
- The Gateway controller creates a `data-science-gateway` Gateway CR in `openshift-ingress` namespace
- Uses GatewayClass `data-science-gateway-class` with controller `openshift.io/gateway-controller/v1`
- Supports two ingress modes: `OcpRoute` (ClusterIP + OpenShift Route) and `LoadBalancer` (direct LB)
- An EnvoyFilter intercepts all gateway traffic, calling kube-auth-proxy via ext_authz for authentication
- After authentication, a Lua filter strips OAuth2 cookies and forwards bearer tokens to upstream services
- Component controllers (e.g., notebook, dashboard) create HTTPRoute CRs referencing `data-science-gateway` as parent
- Authentication supports OpenShift OAuth (IntegratedOAuth), external OIDC, or no auth (external auth mode)
- Default subdomain is `rh-ai` (legacy `data-science-gateway` subdomain redirected via EnvoyFilter)
