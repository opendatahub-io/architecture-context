## Purpose
**Short**: Provides pre-built workbench container images (Jupyter, Code Server, RStudio) and Elyra pipeline runtime images for data science workflows on OpenShift AI.

**Detailed**: The notebooks repository is the source for all out-of-the-box workbench container images shipped with Red Hat OpenShift AI (RHOAI). These workbench images provide interactive development environments (IDEs) for data scientists, including JupyterLab notebooks, VS Code-based Code Server, and RStudio Server. Each workbench variant is tailored for specific use cases: minimal Python environments, GPU-accelerated computing with CUDA or ROCm, deep learning frameworks (PyTorch, TensorFlow), model explainability (TrustyAI), and LLM compression workflows (LLM Compressor).

In addition to IDE workbench images, the repository produces Elyra pipeline runtime images used for executing pipeline nodes in data science pipelines. These runtime images contain the same library stacks as their corresponding workbench images but without the IDE components, optimized for headless pipeline execution.

The repository does not contain an operator or controller. Instead, it produces OpenShift ImageStream manifests that are deployed by the rhods-operator via kustomize overlays. The ImageStream resources register the workbench images in the OpenShift image registry, making them discoverable by the ODH Dashboard and selectable when creating notebook workbench instances. The ODH Notebook Controller (from a separate repository) is responsible for launching Pod instances using these images.

### Image Version Strategy

Each notebook ImageStream maintains up to 8 version tags following a rolling window:
- **N** (latest, e.g., `3.4`): Current release, Python 3.12, referenced via `params-latest.env` (updated by Konflux nudges)
- **N-1** (e.g., `2025.2`): Previous release, Python 3.12
- **N-2 through N-7** (e.g., `2025.1` through `1.2`): Older releases, Python 3.11/3.9/3.8, progressively marked as `opendatahub.io/image-tag-outdated: "true"`

Current release images are sourced from `registry.redhat.io/rhoai/`, while older versions reference `quay.io/modh/` images.
