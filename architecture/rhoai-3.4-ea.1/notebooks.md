## Purpose
**Short**: Provides pre-built notebook workbench and pipeline runtime container images for the RHOAI data science platform.

**Detailed**: The Notebooks repository builds and maintains a comprehensive set of container images that serve as interactive data science workbench environments and pipeline runtime images for Red Hat OpenShift AI (RHOAI). These images are based on UBI9 (Red Hat Universal Base Image) and CentOS Stream 9, and include popular data science frameworks such as JupyterLab, PyTorch, TensorFlow, and Trustworthy AI tooling.

The repository produces two categories of images: (1) Workbench images that provide interactive development environments (JupyterLab, VS Code Server, RStudio Server) with pre-installed ML/AI libraries, and (2) Runtime images that provide lightweight execution environments for Elyra pipeline nodes. Images are built with support for CPU-only, NVIDIA CUDA GPU, and AMD ROCm GPU acceleration. The build system uses Konflux (Tekton-based CI/CD) for hermetic, reproducible builds with hash-verified Python dependencies.

The deployment manifests consist of OpenShift ImageStream resources that reference specific image digests from `registry.redhat.io`. These ImageStreams are consumed by the RHOAI operator (rhods-operator) at deploy time, which injects the correct image references from its ClusterServiceVersion's `RELATED_IMAGE_*` environment variables via kustomize replacements. The ODH Notebook Controller then uses these ImageStreams to launch user workbenches.

## Network Architecture

### Services

This component produces container images, not Kubernetes Services directly. Services are created dynamically by the ODH Notebook Controller when a user launches a workbench.

| Service Name | Type | Port | Target Port | Protocol | Encryption | Auth | Exposure |
|--------------|------|------|-------------|----------|------------|------|----------|
| notebook-{name} (created by controller) | ClusterIP | 8888/TCP | 8888 | HTTP | None (internal) | None | Internal |
| notebook-{name}-tls (created by controller) | ClusterIP | 8443/TCP | 8443 | HTTPS | TLS (kube-rbac-proxy) | kube-rbac-proxy RBAC | Internal |

### Ingress

Ingress is managed by the ODH Notebook Controller (RHOAI 3.x pattern), not by this repository:

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| notebook-{name} (created by controller) | HTTPRoute (Gateway API) | Per-notebook | 443/TCP | HTTPS | TLS 1.2+ | SIMPLE | External via data-science-gateway |

**Note**: In RHOAI 3.x, the ODH Notebook Controller creates HTTPRoute CRs per notebook that reference the platform's `data-science-gateway` Gateway, with `kube-rbac-proxy` sidecars injected for authentication.

## Image Hierarchy

The notebook images follow a layered build hierarchy:

```
UBI9/C9S Base Image
  └── Base Image (CPU/CUDA/ROCm)
       ├── Jupyter Minimal
       │    ├── Jupyter Data Science
       │    │    └── Jupyter Trustworthy AI
       │    ├── Jupyter PyTorch
       │    │    └── Jupyter PyTorch+LLMCompressor
       │    └── Jupyter TensorFlow
       ├── VS Code Server (Code Server)
       └── RStudio Server
            └── RStudio GPU (CUDA)

  Runtime Images (separate hierarchy):
  UBI9 Base
  └── Runtime Minimal
       ├── Runtime Data Science
       ├── Runtime PyTorch
       ├── Runtime TensorFlow
       └── Runtime PyTorch+LLMCompressor
```
