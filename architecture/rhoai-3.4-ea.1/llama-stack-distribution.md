## Purpose
**Short**: A containerized Llama Stack distribution providing a unified API server for AI inference, agents, evaluation, safety, and tool orchestration within Red Hat OpenShift AI.

**Detailed**: The Llama Stack Distribution packages Meta's Llama Stack framework into a production-ready container image for deployment on Red Hat OpenShift AI (RHOAI). It provides a single HTTP API server (port 8321) that exposes multiple AI-related APIs including inference, agents, evaluation, safety, scoring, vector I/O, file management, dataset I/O, tool runtime, and batch processing.

The distribution is built using the Konflux build system from a RHEL 9 UBI base image with pre-installed Python dependencies. It supports a pluggable provider architecture where backend providers for each API (e.g., vLLM for inference, Milvus for vector storage, TrustyAI for safety/evaluation) can be enabled or disabled through environment variables. The configuration is driven by a YAML file (`config.yaml`) that defines providers, storage backends (PostgreSQL), telemetry, and registered resources.

The distribution integrates with the broader RHOAI ecosystem through providers that connect to external services such as vLLM inference servers, TrustyAI guardrails orchestrator, Kubeflow pipelines, and various cloud AI services (AWS Bedrock, Azure OpenAI, Google Vertex AI, IBM watsonx, OpenAI). It also supports OpenTelemetry-based observability when configured.
