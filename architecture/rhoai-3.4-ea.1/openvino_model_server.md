## Purpose
**Short**: High-performance model inference server built on Intel OpenVINO, serving AI/ML models via gRPC and REST APIs with support for TensorFlow Serving, KServe v2, and OpenAI-compatible generative AI endpoints.

**Detailed**: OpenVINO Model Server (OVMS) is a C++ inference serving system optimized for Intel architectures. It hosts machine learning models and makes them accessible via standard network protocols (gRPC and HTTP/REST). Clients send inference requests to the server, which executes model inference using the OpenVINO runtime and returns predictions. The server supports multiple model frameworks including TensorFlow, PaddlePaddle, and ONNX.

OVMS provides extensive generative AI capabilities through OpenAI-compatible APIs (v3 endpoints), including chat completions, text completions, embeddings, reranking, image generation, text-to-speech, and speech-to-text. It supports continuous batching for LLM inference, DAG (Directed Acyclic Graph) pipelines for multi-step inference, MediaPipe graph execution, and Python code execution within inference pipelines. Models can be loaded from local filesystems, AWS S3, Google Cloud Storage, Azure Blob Storage, or HuggingFace Hub.

In RHOAI, OVMS is deployed as a container image (`odh-openvino-model-server-rhel9`) that runs as an inference runtime, typically managed by KServe's InferenceService CRD. It serves as one of the default model serving runtimes for the RHOAI platform.
