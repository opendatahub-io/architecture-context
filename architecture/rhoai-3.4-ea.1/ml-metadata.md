## Purpose
**Short**: ML Metadata (MLMD) is a gRPC server and library for recording and retrieving metadata associated with ML workflows, used by Data Science Pipelines in RHOAI.

**Detailed**: ML Metadata provides a metadata store service that tracks artifacts, executions, contexts, and their relationships (events, attributions, associations) produced during ML workflows. It is originally a Google/TFX project (google/ml-metadata) and is forked by Red Hat for inclusion in Red Hat OpenShift AI (RHOAI).

The component ships as a single container image (`odh-mlmd-grpc-server-rhel9`) built via Konflux. The server binary (`metadata_store_server`) exposes a gRPC API defined in protobuf and supports MySQL, PostgreSQL, and SQLite as backend metadata storage databases. In the RHOAI context, it is deployed as part of Data Science Pipelines to store pipeline run metadata, artifact lineage, and execution provenance. The server supports SSL/TLS for both client-server communication and database connections, with optional mutual TLS (client certificate verification).

### Configuration Options

The server is configured via command-line flags:
- `--grpc_port`: gRPC listen port (default: 8080)
- `--metadata_source_config_type`: Connection type selection (default, config_file, mysql, postgresql)
- `--metadata_store_server_config_file`: Path to protobuf text config file
- `--metadata_store_connection_retries`: Max connection retries (default: 5)
- `--mysql_config_*`: MySQL connection parameters (host, port, database, user, password, SSL options)
- `--postgres_config_*`: PostgreSQL connection parameters (host, hostaddr, port, user, password, dbname, SSL options)
- `--enable_database_upgrade`: Enable schema migration on startup
- `--downgrade_db_schema_version`: Downgrade schema to specific version
- `--grpc_channel_arguments`: Custom gRPC channel arguments
