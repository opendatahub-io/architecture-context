## Purpose

**Short**: Kubernetes operator that automates the lifecycle management of Apache Spark applications and Spark Connect servers on OpenShift/Kubernetes clusters.

**Detailed**: The Spark Operator (originally Kubeflow Spark Operator) provides a Kubernetes-native way to run Apache Spark applications. It introduces three Custom Resource Definitions (CRDs) - `SparkApplication`, `ScheduledSparkApplication`, and `SparkConnect` - that allow users to declaratively define, submit, schedule, and monitor Spark workloads.

The operator consists of two main deployments: a **controller** that watches CRDs and manages Spark driver/executor pod lifecycles via `spark-submit`, and a **webhook** server that performs admission mutation (injecting Spark configuration into pods) and validation (ensuring SparkApplication specs are valid before creation). The controller handles a complex state machine for SparkApplication lifecycle (New -> Submitted -> Running -> Succeeding/Failing -> Completed/Failed) with support for suspending, resuming, and scheduled reruns.

In the RHOAI distribution, the operator deploys into the `redhat-ods-applications` namespace and is managed by the rhods-operator via kustomize overlays. The image references are injected at deploy time through the `RELATED_IMAGE_ODH_SPARK_OPERATOR_IMAGE` environment variable, supporting disconnected/air-gapped installations.

| ScheduledSparkApplication Controller | Reconciler | Manages cron-scheduled SparkApplication execution with concurrency policies (Allow/Forbid/Replace); creates child SparkApplication CRs on schedule |
| SparkConnect Controller | Reconciler | Manages Spark Connect server lifecycle; creates server Pods, Services (ports 7078, 7079, 4040, 15002), and ConfigMaps for executor pod templates |
| MutatingWebhookConfiguration Controller | Reconciler | Manages the lifecycle of MutatingWebhookConfiguration resources |
| ValidatingWebhookConfiguration Controller | Reconciler | Manages the lifecycle of ValidatingWebhookConfiguration resources |
