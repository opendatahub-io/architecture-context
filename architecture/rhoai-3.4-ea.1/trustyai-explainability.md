## Purpose
**Short**: TrustyAI is a Java-based AI fairness, bias tracking, drift detection, and explainability service integrated with ModelMesh and KServe inference platforms.

**Detailed**: TrustyAI Explainability is a Quarkus-based REST service and Java library that provides Explainable AI (XAI) capabilities for machine learning models deployed on OpenShift AI. The service intercepts inference payloads from ModelMesh and KServe serving platforms, stores them (via PVC, MinIO/S3, or relational database), and computes fairness metrics (Statistical Parity Difference, Disparate Impact Ratio), drift detection metrics (KS Test, Meanshift, Fourier MMD), and explainability algorithms (LIME, SHAP, Counterfactual, Time-Series Saliency) over the collected data.

The repository is organized as a multi-module Maven project: `explainability-core` provides the foundational XAI algorithms and fairness metrics as a Java library; `explainability-connectors` implements KServe V2 gRPC protocol support for communicating with inference servers; `explainability-arrow` provides Arrow-based interoperability with the TrustyAI Python library; and `explainability-service` wraps everything into a containerized Quarkus REST service. The service exposes Prometheus metrics for scheduled fairness/drift computations, enabling integration with OpenShift monitoring dashboards. In RHOAI deployments, certain explainer endpoints (LIME, SHAP, Counterfactual, Time-Series Saliency, global explainers) are disabled by default, with only fairness and drift metrics enabled.
