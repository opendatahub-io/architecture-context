## Purpose
**Short**: A lightweight REST API service for orchestrating LLM evaluations across multiple backends on OpenShift.

**Detailed**: EvalHub is a Go-based REST API service that routes evaluation requests to frameworks such as lm-evaluation-harness, LightEval, GuideLLM, RAGAS, and Garak. It provides a unified API for creating, managing, and tracking LLM evaluation jobs. The service uses Go's standard `net/http` router, structured logging with zap, Prometheus metrics, and a pluggable storage layer supporting SQLite (development) and PostgreSQL (production).

When deployed on Kubernetes/OpenShift, EvalHub creates Kubernetes batch Jobs to execute evaluations. Each evaluation benchmark is dispatched as a separate Job pod running a provider-specific adapter image. The service integrates with MLflow for experiment tracking and supports OCI artifact exports. EvalHub is not an operator itself; it is managed by the TrustyAI Service Operator via an `EvalHub` custom resource. The operator provisions the Deployment, ServiceAccount, Service, and supporting resources including a kube-rbac-proxy sidecar for HTTPS authentication.
