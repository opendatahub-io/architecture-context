## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Data Science Pipelines (DSP) infrastructure components based on DataSciencePipelinesApplication (DSPA) custom resources.

**Detailed**: The Data Science Pipelines Operator (DSPO) is a controller-runtime based Go operator that reconciles `DataSciencePipelinesApplication` (DSPA) custom resources to deploy and manage a complete Kubeflow Pipelines V2 stack on OpenShift. When a user creates a DSPA CR, the operator dynamically provisions and configures all pipeline infrastructure components: the DSP API Server, Persistence Agent, Scheduled Workflow controller, Argo Workflow Controller, ML Metadata (MLMD) gRPC server with Envoy proxy, and optionally a MariaDB database and Minio object storage.

The operator uses Go template-based manifest rendering (via the manifestival library) to create Kubernetes resources dynamically at reconcile time. It performs health checks against the database and object storage backends before deploying pipeline components, publishes Prometheus metrics for component readiness, and manages TLS certificates for pod-to-pod encrypted communication. It also supports FIPS-compliant builds, custom CA bundles, proxy configuration, and a Kubernetes-native pipeline storage mode with validating/mutating webhooks.

The operator is deployed by the `rhods-operator` via kustomize overlays. The RHOAI overlay (`config/overlays/rhoai`) includes the base operator deployment plus Argo Workflows CRDs and RBAC resources. Image references are injected at deploy time via RELATED_IMAGE environment variables from the rhods-operator.

| Minio | Deployment (created dynamically) | Default object storage for pipeline artifacts (optional, can use external S3) |
