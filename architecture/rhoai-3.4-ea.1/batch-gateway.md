## Purpose
**Short**: OpenAI-compatible batch inference gateway for the llm-d platform, implementing the `/v1/files` and `/v1/batches` APIs.

**Detailed**: The batch-gateway is an llm-d component that provides an OpenAI-compatible Batch API for submitting and managing large-scale batch inference jobs. It consists of two microservices: an API server that exposes REST endpoints for file management and batch job lifecycle operations, and a batch processor that dequeues jobs from a priority queue, reads input files (JSONL format), fans out individual inference requests to a downstream inference gateway (e.g., llm-d or GAIE), and writes results back.

The system supports multi-tenancy via a configurable HTTP header (`X-MaaS-User` by default) for tenant isolation. It uses Redis as a backing store for job metadata, priority queuing, event channels, and status tracking, with optional PostgreSQL support. File storage is pluggable with filesystem and S3 backends. The API server and processor are independently scalable and deployable via Helm charts.

| batch-gateway-processor | Go Worker Service | Polls priority queue for batch jobs, fans out inference requests to downstream inference gateway, manages job lifecycle and status |

| S3 / Filesystem | External Dependency | Pluggable file storage backend for batch input/output JSONL files |
| Inference Gateway (llm-d/GAIE) | External Dependency | Downstream service that processes individual inference requests (e.g., `/v1/chat/completions`) |
