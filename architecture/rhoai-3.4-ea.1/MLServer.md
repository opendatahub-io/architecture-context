## Purpose
**Short**: An open source inference server for serving machine learning models via REST and gRPC interfaces, fully compliant with the KFServing V2 Dataplane protocol.

**Detailed**: MLServer is a Python-based model inference server originally developed by Seldon Technologies, packaged by Red Hat for RHOAI. It provides a standardized way to serve machine learning models through both REST (HTTP) and gRPC interfaces, implementing the V2 Inference Protocol (also known as KFServing V2 Dataplane). The server supports multi-model serving, allowing multiple models to run within a single process, and provides adaptive batching to group inference requests together for improved throughput.

MLServer includes a pluggable runtime architecture that supports multiple ML frameworks including scikit-learn, XGBoost, LightGBM, ONNX, MLflow, HuggingFace Transformers, Alibi Explain/Detect, CatBoost, and Apache Spark MLlib. The RHOAI Konflux build ships with LightGBM, sklearn, and XGBoost runtimes by default. The server also supports parallel inference workers for vertical scaling, an optional Kafka consumer/producer interface for asynchronous inference, Prometheus metrics export, and OpenTelemetry distributed tracing.

In the RHOAI platform, MLServer runs as a KServe inference runtime container, deployed within inference service pods managed by KServe. It serves as one of the core Python-based serving runtimes alongside other options like OpenVINO Model Server and vLLM.

### Authentication & Authorization

| Endpoint | Methods | Auth Mechanism | Enforcement Point | Policy |
|----------|---------|----------------|-------------------|--------|
| All V2 endpoints | GET/POST | None (in-container) | KServe / RHOAI Gateway | Auth enforced at platform ingress level, not at MLServer level |

MLServer itself runs without authentication. In RHOAI deployments, authentication is enforced externally:
- **RHOAI 3.x**: kube-rbac-proxy sidecar or Gateway API-level auth
- **RHOAI 2.x**: oauth-proxy sidecar or Istio AuthorizationPolicy
