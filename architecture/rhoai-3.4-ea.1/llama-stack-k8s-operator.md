## Purpose
**Short**: Kubernetes operator that manages the lifecycle of LlamaStack AI server distributions on Kubernetes/OpenShift clusters.

**Detailed**: The LlamaStack K8s Operator automates the deployment and management of LlamaStack server distributions, which provide a unified API for AI/ML inference capabilities including model serving, agents, and tool integration. The operator uses a single Custom Resource Definition (LlamaStackDistribution) to declaratively manage the full stack of Kubernetes resources needed to run a LlamaStack server instance.

The operator supports multiple LlamaStack distributions (starter, remote-vllm, meta-reference-gpu, postgres-demo) and allows custom container images. It manages Deployments, Services, PersistentVolumeClaims, NetworkPolicies, Ingresses, HorizontalPodAutoscalers, PodDisruptionBudgets, and ServiceAccounts. It also handles CA bundle management for TLS trust, user configuration via ConfigMaps, automatic pod restarts on config changes, and health checking via the LlamaStack `/v1/health` endpoint.

For RHOAI deployments, the operator runs in the `redhat-ods-applications` namespace and uses the `RELATED_IMAGE_RH_DISTRIBUTION` environment variable to resolve Red Hat-specific distribution images. The operator includes upgrade cleanup logic for migrating from cluster-scoped to namespace-scoped RBAC bindings.
