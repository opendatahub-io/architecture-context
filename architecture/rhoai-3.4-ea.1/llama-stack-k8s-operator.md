## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of Llama Stack distribution server instances on OpenShift/Kubernetes.

**Detailed**: The llama-stack-k8s-operator is a controller-runtime based Go operator that provides a `LlamaStackDistribution` Custom Resource Definition (CRD) for declaratively deploying and managing Llama Stack AI inference server instances. When a user creates a `LlamaStackDistribution` CR, the operator reconciles the desired state by creating and managing a full set of Kubernetes resources including Deployments, Services, ServiceAccounts, PersistentVolumeClaims, NetworkPolicies, PodDisruptionBudgets, HorizontalPodAutoscalers, Ingresses, and RoleBindings.

The operator supports multiple distribution types (starter, remote-vllm, meta-reference-gpu, postgres-demo) with the RHOAI deployment using a Red Hat-specific distribution image (`quay.io/aipcc/llama-stack/cpu-ubi9:latest`). It uses kustomize internally for manifest rendering with custom Go-based transformer plugins for namespace setting, name prefixing, field mutation, and NetworkPolicy rule injection. The operator also supports CA bundle management (explicit and auto-detected ODH trusted CA bundles), user configuration via ConfigMaps, TLS configuration, horizontal pod autoscaling, topology spread constraints, and external access via Kubernetes Ingress resources.

The operator is deployed into the `redhat-ods-applications` namespace by the RHOAI rhods-operator using kustomize overlays. It watches for changes to referenced ConfigMaps and automatically triggers pod restarts when configuration changes via content-hash annotations.

### Network Policies

The operator creates a `NetworkPolicy` per LlamaStackDistribution instance (when `enableNetworkPolicy` feature flag is enabled, which is the default). The policy:

- **Pod Selector**: Targets pods with `app=llama-stack` and `app.kubernetes.io/instance={instance-name}`
- **Policy Type**: Ingress only
- **Default Peers**: Same-namespace pods with `app.kubernetes.io/part-of=llama-stack`, all pods from operator namespace
- **Configurable Peers**: Explicit namespace list (`spec.network.allowedFrom.namespaces`), label-based namespace selection (`spec.network.allowedFrom.labels`), wildcard `*` for all namespaces
- **Router Peers**: OpenShift ingress controller namespaces (label `network.openshift.io/policy-group=ingress`) when network spec is configured
- **Port**: Configured service port (default 8321/TCP)
