## Purpose
**Short**: Kubernetes operator that manages MLflow tracking server deployments on OpenShift/RHOAI using CRDs and embedded Helm chart rendering.

**Detailed**: The MLflow Operator is a controller-runtime based Kubernetes operator that reconciles `MLflow` custom resources (cluster-scoped) and deploys MLflow tracking server instances. It uses an embedded Helm chart to render Kubernetes manifests (Deployment, Service, ServiceAccount, PVC, NetworkPolicy, ClusterRole/ClusterRoleBinding) and applies them via Server-Side Apply.

The operator supports the RHOAI 3.x ingress pattern by dynamically creating Gateway API `HTTPRoute` resources that reference the platform's `data-science-gateway` Gateway in the `openshift-ingress` namespace. It also creates OpenShift `ConsoleLink` resources for integration with the OpenShift web console application menu. The operator handles multi-instance deployments via resource naming suffixes and supports workspace-aware artifact storage through the namespace-scoped `MLflowConfig` CRD. CA bundle management is built in, supporting both platform-injected (`odh-trusted-ca-bundle`) and user-provided CA certificates through an init container and sidecar watcher pattern.

The MLflow server itself runs with TLS (via OpenShift service-ca certificates), Kubernetes-native authentication (`self_subject_access_review` authorization mode), and supports multiple backend store configurations (SQLite, PostgreSQL, MySQL) and artifact storage backends (file, S3, GCS).

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| mlflow HTTPRoute | HTTPRoute (Gateway API) | Inherited from data-science-gateway | 8443/TCP | HTTPS | TLS (Gateway terminates) | PASSTHROUGH to backend | External (via data-science-gateway in openshift-ingress) |

The operator dynamically creates `HTTPRoute` resources in the target namespace with three routing rules:
1. `/mlflow/api/*` -> rewrites to `/api/*` and forwards to mlflow service:8443
2. `/mlflow/v1/*` -> rewrites to `/v1/*` and forwards to mlflow service:8443
3. `/mlflow/*` -> forwards directly to mlflow service:8443 (static assets and UI)

All HTTPRoutes reference the `data-science-gateway` Gateway in the `openshift-ingress` namespace as their parent.
