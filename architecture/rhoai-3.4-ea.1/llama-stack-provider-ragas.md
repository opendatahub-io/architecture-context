## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates the Ragas framework for LLM evaluation, supporting both inline (in-process) and remote (Kubeflow Pipelines) execution modes.

**Detailed**: This component implements Ragas (Retrieval Augmented Generation Assessment) as an external provider for Meta's Llama Stack evaluation API. It enables users to evaluate LLM outputs using established RAG quality metrics—answer relevancy, context precision, faithfulness, and context recall—through the standardized Llama Stack eval interface.

The provider ships in two modes: an **inline provider** that executes Ragas evaluation directly within the Llama Stack server process, and a **remote provider** that offloads evaluation to Kubeflow Pipelines for scalable, distributed execution on OpenShift. The remote provider submits KFP pipeline runs that retrieve datasets from Llama Stack, run Ragas evaluation, and persist results to S3-compatible storage.

The component is built by Konflux and shipped as a container image (`odh-trustyai-ragas-lls-provider-dsp-rhel9`) based on UBI9/Python 3.12. It wraps Llama Stack's inference and embedding APIs into Ragas-compatible LLM and embeddings interfaces using adapter/wrapper classes, bridging the two frameworks.
