## Purpose
**Short**: A Go-based build-time tool that extracts, enriches, and catalogs metadata from Red Hat AI container model images (ModelCars), producing static YAML catalog files shipped in a data-only container image.

**Detailed**: The model-metadata-collection component is a metadata processing pipeline that runs at build time (CI/CD) to produce a curated catalog of AI model metadata. It discovers AI models from HuggingFace validated model collections, inspects OCI container images (ModelCars) in the Red Hat container registry to extract embedded model cards, enriches the extracted metadata with data from the HuggingFace API (model details, tags, YAML frontmatter, README content), and generates comprehensive YAML catalog files.

The final output is a minimal container image (based on UBI9-minimal) that contains only the pre-generated `models-catalog.yaml`, `models-index.yaml`, `validated-models-catalog.yaml`, and `validated-models-index.yaml` files mounted at `/app/data`. This data container is used as an init container or volume source by other RHOAI components (notably the ODH Dashboard) to provide the AI model catalog displayed to users. The container runs `sleep infinity` as its sole process - it exists purely as a data delivery mechanism.

The tool supports concurrent processing of models, automatic model type classification (generative, predictive, unknown), intelligent tag merging from multiple sources, model deduplication, static catalog merging, and metadata quality reporting. It follows a strict data priority hierarchy: modelcard.md data > HuggingFace YAML frontmatter > HuggingFace API > registry metadata > generated defaults.
