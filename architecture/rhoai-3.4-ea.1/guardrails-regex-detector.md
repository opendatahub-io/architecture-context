## Purpose
**Short**: A lightweight Rust-based HTTP microservice that detects sensitive information (PII) and custom patterns in text using regular expressions.

**Detailed**: The Guardrails Regex Detector is a stateless HTTP service designed to scan text content for predefined PII patterns (email addresses, Social Security numbers, credit card numbers) as well as arbitrary custom regular expressions provided at request time. It is built with the Axum web framework on Tokio async runtime, providing high-performance text scanning capabilities.

This component is primarily integrated with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator), serving as one of the detection backends in the RHOAI guardrails pipeline. The orchestrator routes text content to this detector, which returns matched spans with position offsets, detection types, and confidence scores. The service is part of the broader RHOAI guardrails ecosystem that provides content safety and compliance filtering for AI model inputs and outputs.

The service supports three built-in detection patterns (`email`, `ssn`, `credit-card`) and allows callers to pass arbitrary regex patterns as `detector_params`. All matches return a score of `1.0` since regex matching is binary (match or no match).

**Request Schema** (`POST /api/v1/text/contents`):
```json
{
  "contents": ["string array of text to scan"],
  "detector_params": {
    "regex": ["ssn", "email", "credit-card", "^custom-regex$"]
  }
}
```

**Response Schema**:
```json
[[
```
