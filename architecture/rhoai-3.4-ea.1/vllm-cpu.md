## Purpose
**Short**: CPU-optimized build of vLLM providing high-throughput LLM inference and serving with an OpenAI-compatible API.

**Detailed**: vLLM-CPU is Red Hat's CPU-only build of the vLLM inference engine, designed for running large language model inference without requiring GPU hardware. It provides an OpenAI-compatible REST API server (FastAPI/Uvicorn) and a gRPC server for efficient binary communication with routing infrastructure. The server supports text generation (chat completions, completions), embeddings, classification, scoring, tokenization, audio transcription/translation, and the Anthropic Messages API.

The container image is built on UBI 9 (ubi-minimal) and supports multiple CPU architectures including x86_64 (with Intel OpenMP optimizations), aarch64/arm64, ppc64le (with OpenBLAS), and s390x. It uses PyTorch as the underlying tensor computation framework with CPU-specific optimizations. The image runs as a non-root user (UID 2000) and is designed for deployment within KServe serving runtimes on OpenShift AI.

The build uses Konflux CI/CD pipelines (Dockerfile.konflux.cpu) for production builds. The architecture includes PagedAttention for efficient memory management, continuous batching, speculative decoding, and structured output support (JSON schema, regex, grammar constraints).
