## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates NVIDIA Garak red-teaming framework for automated LLM vulnerability scanning and security assessment.

**Detailed**: This component implements Garak (NVIDIA's LLM vulnerability scanner) as an external evaluation provider for the Llama Stack framework. It enables automated red-teaming of Large Language Models against industry-standard vulnerability taxonomies including OWASP LLM Top 10, AVID (AI Vulnerability and Incident Database), CWE, and quality assessments. The provider supports testing prompt injection, jailbreaks, toxicity, bias, and other LLM vulnerabilities.

The provider operates in two execution modes: **inline** (local subprocess execution of Garak scans within the Llama Stack server process) and **remote** (offloading scans to Kubeflow Pipelines as containerized pipeline steps running on OpenShift AI). The remote mode is the production deployment pattern, where scans run as KFP pipeline components with GPU support, retry logic, and result persistence via the Llama Stack Files API. Both modes support shield testing, allowing users to measure the effectiveness of guardrails (e.g., Prompt-Guard-86M) by running scans with shields applied to model input/output.

The component produces a single container image (`odh-trustyai-garak-lls-provider-dsp-rhel9`) built on UBI9/Python 3.12, which serves as both the Llama Stack provider runtime and the base image for KFP pipeline components. The image is built via Konflux with support for amd64, arm64, ppc64le, and s390x architectures.
