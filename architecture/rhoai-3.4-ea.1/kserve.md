## Purpose
**Short**: KServe is the model serving controller that manages the lifecycle of inference services on Kubernetes, providing serverless and raw deployment modes for ML model serving.

**Detailed**: KServe (formerly KFServing) is a Kubernetes-native platform for serving machine learning models. It provides a standardized API and set of CRDs for deploying, scaling, and managing inference services. The controller manages the full lifecycle of model serving workloads including predictors, transformers, and explainers as components of an InferenceService.

KServe supports multiple deployment modes: Serverless (via Knative Serving) and RawDeployment (via Kubernetes Deployments). For ingress, it supports both Istio VirtualService-based routing and Gateway API (HTTPRoute) based routing. The RHOAI 3.x distribution is transitioning to Gateway API with HTTPRoute CRs for ingress and kube-rbac-proxy for authentication. The controller also manages storage initialization for downloading models from cloud storage (S3, GCS, Azure), local model caching across nodes, and a new LLMInferenceService controller for large language model workloads with disaggregated serving (prefill/decode separation), scheduler, and Gateway API Inference Extension integration.

**Controller-managed ingress behavior**:
- **Serverless mode (Istio)**: InferenceService controller creates Istio VirtualService CRs and ExternalName Services per ISVC. VirtualService routes traffic through Istio ingress/local gateways (knative-serving/knative-ingress-gateway, knative-serving/knative-local-gateway).
- **Raw+GatewayAPI mode (RHOAI 3.x pattern)**: InferenceService controller creates HTTPRoute CRs per ISVC component (predictor, transformer, explainer) + a top-level HTTPRoute. All HTTPRoutes reference a parent Gateway (configurable via `kserveIngressGateway`, default: `kserve/kserve-ingress-gateway`).
- **LLMInferenceService**: Creates HTTPRoutes, InferencePools, InferenceModels, and Istio DestinationRules. Watches Gateway and HTTPRoute CRs for status. Supports Gateway API Inference Extension for intelligent LLM routing.
- **InferenceGraph**: Creates Knative Services for serverless mode; creates Deployments + OpenShift Routes for raw mode.
- **kube-rbac-proxy sidecar** (8443/TCP) fronts controller metrics (8080/TCP) with SubjectAccessReview authorization.
