## Purpose
**Short**: KServe is the model serving controller for RHOAI that manages the lifecycle of ML inference services on Kubernetes, including standard InferenceServices and LLM-optimized LLMInferenceServices.

**Detailed**: KServe provides a Kubernetes-native platform for deploying and managing machine learning models in production. It supports multiple deployment modes including RawDeployment (Kubernetes Deployments + Services) and Serverless (Knative Services), with the RHOAI overlay defaulting to RawDeployment mode.

The controller manages the full lifecycle of inference endpoints: creating Deployments, Services, and ingress resources (HTTPRoute via Gateway API or Istio VirtualService) for each InferenceService. It handles model storage initialization, auto-scaling (HPA, KEDA), canary deployments, and multi-model serving through TrainedModel resources.

A major feature is the LLMInferenceService controller, purpose-built for large language model serving. This controller orchestrates LLM-specific infrastructure including the llm-d inference scheduler (for intelligent request routing), InferencePool/InferenceModel resources from the Gateway API Inference Extension, multi-node deployments via LeaderWorkerSet, disaggregated prefill/decode topologies, and routing sidecars. It integrates with Istio for mTLS and uses self-signed TLS certificates for workload-to-workload communication.
