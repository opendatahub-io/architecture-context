## Purpose
**Short**: The ODH Dashboard is the primary web UI for Red Hat OpenShift AI (RHOAI), providing a unified management console for data science workflows, model serving, notebooks, pipelines, and platform administration.

**Detailed**: The ODH Dashboard serves as the central user-facing web application for the RHOAI platform. It is built as a monorepo using a React (PatternFly 6) frontend with a Fastify-based Node.js backend-for-frontend (BFF) that proxies Kubernetes API requests on behalf of authenticated users. The backend acts as a secure API gateway, making Kubernetes API calls using user-impersonated tokens forwarded through the kube-rbac-proxy authentication sidecar.

In RHOAI 3.x, the dashboard has adopted a modular architecture using Webpack Module Federation. The core dashboard container is augmented with federated UI modules for Model Registry, Gen AI, and Models-as-a-Service (MaaS), each running as separate containers (with Go-based BFF backends) within the same pod. These modules communicate via TLS-encrypted internal ports and are composed at runtime into a unified UI experience. External access is secured through an HTTPRoute (Gateway API) with kube-rbac-proxy providing RBAC-based authentication, replacing the legacy oauth-proxy + OpenShift Route pattern from RHOAI 2.x.

The dashboard manages several custom resource types (OdhApplication, OdhDocument, OdhQuickStart, OdhDashboardConfig) and interacts with numerous platform CRDs including Notebooks (kubeflow.org), InferenceServices (serving.kserve.io), ModelRegistries, HardwareProfiles, DataScienceClusters, and more. It provides UI pages for project management, notebook workbenches, model serving, data science pipelines, distributed workloads, model registry, model catalog, LM evaluation, connection types, and cluster administration settings.

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| odh-dashboard | HTTPRoute (Gateway API) | Auto-assigned by gateway | 8443/TCP | HTTPS | TLS (gateway-terminated + re-encrypt to service) | PASSTHROUGH to kube-rbac-proxy | External |
| odh-dashboard | Route (OpenShift, legacy) | Auto-assigned | 443/TCP | HTTPS | TLS, reencrypt termination | HSTS enabled (max-age=31536000) | External |

**Note**: RHOAI 3.x uses the HTTPRoute (Gateway API) pattern. The HTTPRoute references a parent Gateway named `$(gateway-name)` in the `openshift-ingress` namespace (the data-science-gateway deployed by rhods-operator). The legacy OpenShift Route is retained for compatibility but HTTPRoute is the primary ingress path.

**Authentication flow**: External request -> Gateway (TLS termination) -> HTTPRoute -> kube-rbac-proxy (8443, re-encrypts, validates RBAC via TokenReview/SubjectAccessReview) -> dashboard app (8080, plaintext). User identity is forwarded via `X-Auth-Request-User` and `X-Auth-Request-Groups` headers.
