## Purpose
**Short**: The OpenShift AI Dashboard is the primary web UI for Red Hat OpenShift AI, providing a unified interface for data scientists and administrators to manage notebooks, model serving, pipelines, and platform settings.

**Detailed**: The ODH Dashboard serves as the central management console for the OpenShift AI platform. It consists of a Fastify-based Node.js backend (BFF) and a React/PatternFly frontend. The backend acts as a Kubernetes API proxy, forwarding user requests to the Kubernetes API server using the authenticated user's token (passed via `x-forwarded-access-token` header from kube-rbac-proxy). It does not maintain its own data store; all state is stored as Kubernetes resources (CRDs, ConfigMaps, Secrets).

The dashboard employs a **modular architecture** using Webpack Module Federation, allowing feature-specific UI modules (Model Registry, Gen AI, MaaS) to run as separate containers within the same pod, each serving their own frontend bundles and backend APIs on dedicated ports. The main dashboard container proxies requests to these federated modules. External access is provided via Gateway API HTTPRoute through the platform's `data-science-gateway`, with kube-rbac-proxy handling authentication and RBAC enforcement.
