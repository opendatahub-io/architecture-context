## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Feast (Feature Store) deployments for machine learning feature serving.

**Detailed**: The Feast Operator is a controller-runtime based Go operator that manages `FeatureStore` custom resources to deploy and configure Feast feature store infrastructure on Kubernetes and OpenShift. It automates the deployment of Feast's core services: online store (feature serving), offline store (historical feature retrieval), registry (feature metadata), and an optional UI server.

The operator dynamically creates Deployments, Services, PersistentVolumeClaims, ConfigMaps, ServiceAccounts, RBAC resources (Roles, RoleBindings, ClusterRoles, ClusterRoleBindings), CronJobs, and on OpenShift clusters, Routes for external access. It supports multiple persistence backends including file-based (SQLite, DuckDB, Dask), database-backed (PostgreSQL, Snowflake, Redis, Cassandra, etc.), and object store (S3, GCS) configurations. The operator also provides Kubernetes RBAC-based and OIDC-based authorization for Feast services.

On OpenShift, the operator automatically configures TLS using OpenShift service serving certificates and creates Routes for the UI service. It also manages a namespace registry ConfigMap that tracks all deployed FeatureStore instances across the cluster, and a Notebook integration controller that creates ConfigMaps with Feast client configurations for Kubeflow Notebooks.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| feast-operator controller-manager | Go Operator (controller-runtime) | Reconciles FeatureStore CRs; dynamically creates Deployments, Services, PVCs, ConfigMaps, ServiceAccounts, RBAC resources, CronJobs, and OpenShift Routes |
| FeatureStore Controller | Reconciler | Watches FeatureStore CRs and owned resources (ConfigMap, Deployment, Service, PVC, ServiceAccount, RoleBinding, Role, CronJob, Route); deploys all Feast services |
| NotebookConfigMap Controller | Reconciler | Watches Kubeflow Notebooks with `opendatahub.io/feast-integration` label; creates per-notebook ConfigMaps with Feast client configs |
| Authorization Module | Controller logic | Deploys Kubernetes RBAC (Role, RoleBinding, ClusterRole, ClusterRoleBinding) or OIDC authorization for Feast services |
| Namespace Registry | Controller logic | Maintains a central ConfigMap (`feast-configs-registry`) tracking all FeatureStore deployments across the cluster |
| Feature Server | Python container | Feast feature server image serving online/offline/registry/UI endpoints; deployed by the operator |
| CronJob Runner | Container (origin-cli) | Executes periodic `feast apply` and `feast materialize-incremental` commands against deployed feature stores |

### OpenShift TLS Auto-Configuration

On OpenShift clusters, the operator automatically:
1. Annotates Services with `service.beta.openshift.io/serving-cert-secret-name` to request service serving certificates
2. For dual-protocol registry (gRPC + REST), uses SANs (`service.beta.openshift.io/serving-cert-sans`) to include both service hostnames in one certificate
3. Creates a CA ConfigMap annotated with `service-ca.crt` for remote registry TLS verification
4. Mounts TLS secrets and CA bundles into feature server pod containers
