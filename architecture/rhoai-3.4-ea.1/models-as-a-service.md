## Purpose
**Short**: A platform API service that provides tiered, policy-managed access to LLM inference models via an OpenAI-compatible interface with token management, tier-based rate limiting, and API key lifecycle.

**Detailed**: Models as a Service (MaaS) is a Go-based API server that sits in front of KServe LLMInferenceService instances to provide a unified management layer for LLM model access. It discovers available models by querying LLMInferenceService resources in the cluster and checking which ones are associated with a designated MaaS Gateway. Users authenticate via Kubernetes TokenReview (through Kuadrant/Authorino), receive tier classifications (free/premium/enterprise) based on group membership, and can issue ephemeral service account tokens or persistent API keys for programmatic access to models.

The platform uses Gateway API (HTTPRoute) for traffic routing through an OpenShift-managed gateway, with Kuadrant providing authentication (AuthPolicy), authorization (SubjectAccessReview against LLMInferenceService resources), and token-based rate limiting (TokenRateLimitPolicy). The MaaS API provides the tier lookup service that Authorino calls during auth policy evaluation to enrich identity metadata. Observability is built-in via Istio telemetry, ServiceMonitors, and Grafana dashboards that track usage per user, tier, and model.

The maas-controller component exists as a placeholder (not yet implemented) for future controller logic. The API supports pluggable storage backends: in-memory (default/ephemeral), local disk (SQLite file), or external PostgreSQL for HA multi-replica deployments.
