## Purpose
**Short**: A FIPS-compliant authentication and authorization reverse proxy combining OAuth2/OIDC authentication (kube-auth-proxy) and Kubernetes RBAC authorization (kube-rbac-proxy) into a single container image for securing services in OpenShift AI environments.

**Detailed**: kube-auth-proxy is a focused, FIPS-compliant authentication proxy derived from oauth2-proxy, streamlined specifically for OpenShift Data Hub (ODH) and Red Hat OpenShift AI (RHOAI) environments. It provides secure authentication through two mechanisms: external OIDC providers and OpenShift's internal OAuth service. The project serves as a drop-in replacement for existing oauth-proxy sidecars, maintaining argument and header compatibility while adding FIPS compliance.

The container image ships three binaries selected via an entrypoint dispatcher: `kube-auth-proxy` (OAuth2/OIDC authentication proxy), `kube-rbac-proxy` (Kubernetes SubjectAccessReview-based authorization proxy), and `entrypoint` (mode selector defaulting to RBAC mode). The kube-rbac-proxy component is an embedded fork of brancz/kube-rbac-proxy that performs RBAC authorization against the Kubernetes API using SubjectAccessReview. Together, these provide both authentication (authn) and authorization (authz) capabilities for upstream services in the RHOAI platform.

The proxy also supports Kubernetes service account token validation via the TokenReview API, enabling service accounts to authenticate alongside human users independently of the configured provider.
