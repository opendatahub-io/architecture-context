## Purpose
**Short**: Kubeflow Pipelines provides an end-to-end ML workflow orchestration platform for building, deploying, and managing machine learning pipelines on Kubernetes.

**Detailed**: Data Science Pipelines (based on Kubeflow Pipelines v2) is the ML pipeline orchestration component of Red Hat OpenShift AI. It enables data scientists and ML engineers to define, compile, run, and monitor reproducible ML workflows as directed acyclic graphs (DAGs) of containerized steps. The system uses Argo Workflows as its underlying workflow engine.

The component consists of several microservices: an API server that exposes both gRPC (v1beta1 and v2beta1) and REST APIs for pipeline CRUD operations and run management; a persistence agent that watches Argo Workflow resources and reports their status back to the API server; a scheduled workflow controller that manages recurring pipeline runs based on cron or periodic schedules; a cache server that implements execution caching via Kubernetes mutating admission webhooks; a frontend UI for visual pipeline management; a metadata (MLMD) gRPC service for ML artifact lineage tracking; and driver/launcher binaries that execute within pipeline step pods to coordinate artifact I/O and metadata logging.

All Go binaries in the RHOAI distribution are built with FIPS-compliant crypto (`GOEXPERIMENT=strictfipsruntime`) using UBI9 base images. The system stores pipeline definitions and run metadata in a relational database (MySQL or PostgreSQL) and pipeline artifacts in S3-compatible object storage (MinIO or external).

| ml-pipeline-persistenceagent | Go Agent | Watches Argo Workflow CRs and syncs run status to the API server database |
| ml-pipeline-scheduledworkflow | Go Controller | Manages ScheduledWorkflow CRs to trigger recurring pipeline runs via Argo Workflows |
| ml-pipeline-viewer-crd-controller | Go Controller | Manages Viewer CRs to deploy Tensorboard instances for experiment visualization |
| cache-server | Go Service | Mutating admission webhook for pipeline step caching; intercepts pod creation to check for cached results |

| ml-pipeline-visualizationserver | Python Service | Generates HTML visualizations for pipeline run outputs |

| metadata-envoy | Envoy Proxy | Envoy sidecar proxy for metadata gRPC service |
| kfp-driver | Go Binary | In-pod binary that resolves inputs, manages caching decisions, and orchestrates pipeline step execution |
| kfp-launcher | Go Binary | In-pod binary that executes user containers and handles artifact upload/download |
