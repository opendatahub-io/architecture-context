## Purpose
**Short**: A Kubernetes controller that performs intelligent autoscaling for LLM inference model servers based on KV cache saturation and queue depth analysis.

**Detailed**: The Workload Variant Autoscaler (WVA) is a global autoscaler for inference model servers serving Large Language Models (LLMs). It continuously monitors vLLM inference server metrics via Prometheus - specifically KV cache utilization and request queue depth - to determine optimal replica counts for deployments. WVA works alongside standard Kubernetes HPA and external autoscalers like KEDA by emitting custom Prometheus metrics (desired replicas, current replicas, desired ratio) that HPA/KEDA can use for scaling decisions.

WVA introduces the `VariantAutoscaling` CRD (API group `llmd.ai/v1alpha1`) which maps a model variant to a scalable deployment target. Each VariantAutoscaling resource specifies a `scaleTargetRef` (Deployment), a `modelID`, and an optional `variantCost` for cost-weighted scaling decisions. The controller supports two optimization engines: a saturation engine that runs a 30-second polling loop analyzing per-model KV cache and queue metrics, and a scale-from-zero engine that runs a 100ms polling loop detecting pending requests via Gateway API Inference Extension EPP flow control metrics. The controller also supports GPU-aware limiting (constraining replicas to available GPU resources), scale-to-zero policies, and multi-controller isolation via instance labels.

WVA integrates with the Gateway API Inference Extension (`sigs.k8s.io/gateway-api-inference-extension`) by watching `InferencePool` resources (both `v1` and `v1alpha2` API versions) to discover endpoint picker (EPP) services for flow control metrics used in scale-from-zero decisions.
