## Purpose
**Short**: Optimized inference request routing scheduler for the llm-d framework, extending the Kubernetes Gateway API Inference Extension (GIE) with llm-d-specific scheduling plugins and Prefill/Decode disaggregation support.

**Detailed**: The llm-d-inference-scheduler provides two main components: an Endpoint Picker (EPP) service and a P/D routing sidecar. The EPP integrates with Envoy via the ext-proc (External Processing) filter protocol to make intelligent routing decisions for LLM inference requests. It extends the upstream Gateway API Inference Extension (GIE) with custom scheduling plugins including KV-cache-aware scoring, load-aware scoring, session affinity, prefix cache optimization, and Prefill/Decode (P/D) disaggregation support.

The P/D routing sidecar runs alongside decoder vLLM instances and orchestrates disaggregated inference by forwarding prefill requests to designated prefill workers and managing the decode stage locally. It supports multiple P/D connector protocols (NIXL v2, LMCache, SGLang) and provides SSRF protection through InferencePool-based allowlisting. The sidecar also supports data-parallel routing for multi-GPU vLLM configurations.

The scheduler uses a pluggable architecture with configurable filters (by-label, decode-filter, prefill-filter), scorers (prefix-cache, load-aware, active-request, session-affinity, no-hit-lru, precise-prefix-cache), and profile handlers (P/D profile, data-parallel profile) that are composed via YAML configuration (EndpointPickerConfig).

| Metrics | Prometheus | Custom metrics for P/D decision tracking (llm_d_inference_scheduler_pd_decision_total) |
