## Purpose
**Short**: Kubernetes-native operator for managing distributed training jobs across multiple ML/AI frameworks.

**Detailed**: The Kubeflow Training Operator (KFTO) is a Kubernetes operator that manages the full lifecycle of distributed training jobs for machine learning and AI models. It provides Custom Resource Definitions (CRDs) for six training frameworks: PyTorchJob, TFJob (TensorFlow), XGBoostJob, MPIJob, PaddleJob, and JAXJob. For each training job, the operator reconciles the desired state by creating and managing worker pods, headless services for inter-pod communication, and optional gang scheduling via Volcano or scheduler-plugins PodGroups.

The operator runs as a single-replica deployment in the `opendatahub` namespace on RHOAI. It includes a validating webhook server for admission control of training job CRs, with self-managed TLS certificate rotation using the OPA cert-controller library. The PyTorch controller additionally creates per-job NetworkPolicy resources to restrict inter-pod traffic and allow metrics scraping from the `openshift-monitoring` namespace. The operator also supports Horizontal Pod Autoscaler (HPA) for elastic PyTorch training workloads and init containers for PyTorch rendezvous bootstrapping.

| PodGroup (Volcano) | All controllers (when gang scheduling enabled) | Gang scheduling coordination |
| PodGroup (scheduler-plugins) | All controllers (when gang scheduling enabled) | Gang scheduling coordination |
