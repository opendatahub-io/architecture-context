## Purpose
**Short**: A pluggable KV-Cache indexing library and service for KV-Cache-aware routing and scheduling in vLLM-based LLM inference platforms.

**Detailed**: The llm-d-kv-cache component provides a high-performance KV-Cache indexer that maintains a global, near-real-time view of KV-Cache block locality across a fleet of vLLM pods. It enables intelligent scheduling by scoring vLLM pods based on their cached KV-blocks, allowing schedulers (like llm-d-inference-scheduler) to route inference requests to the pod with the most relevant cached data already loaded.

The system operates via two primary data paths: a Read Path where the scheduler queries the indexer to score pods for a given prompt (tokenize, hash into block keys, look up which pods have those blocks, score by longest prefix match), and a Write Path where KV-cache events (BlockStored, BlockRemoved, AllBlocksCleared) are streamed from vLLM pods via ZeroMQ pub/sub, decoded from msgpack, and used to update the in-memory block index in near-real-time.

The component also includes a Unix Domain Socket (UDS) tokenizer sidecar service (Python-based gRPC server) that handles tokenization and chat template rendering, and a filesystem-based KV-cache offloading connector (`llmd_fs_backend`) for shared-storage KV-cache transfer between GPU and disk.
