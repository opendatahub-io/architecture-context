## Purpose
**Short**: Kubernetes operator that manages TrustyAI AI/ML trustworthiness services including model explainability, LLM evaluation, guardrails orchestration, NeMo guardrails, and an evaluation hub.

**Detailed**: The TrustyAI Service Operator is a multi-controller Go operator built with Kubebuilder that manages five distinct services for AI/ML trustworthiness within the OpenShift AI (RHOAI) platform. It provisions and manages: (1) TrustyAIService - the core explainability and fairness monitoring service for ML models served via KServe/ModelMesh, (2) LMEvalJob - a job-based controller for running LLM evaluation benchmarks using the lm-evaluation-harness framework with Kueue integration for workload scheduling, (3) GuardrailsOrchestrator - a deployment controller that orchestrates guardrails detection services around LLM inference endpoints with auto-configuration capabilities, (4) NemoGuardrails - a controller for deploying NVIDIA NeMo Guardrails server instances, and (5) EvalHub - an evaluation hub service providing a centralized API for evaluation management with PostgreSQL or SQLite storage.

Each controller dynamically creates Kubernetes resources including Deployments, Services, OpenShift Routes, ConfigMaps, ServiceAccounts, and ClusterRoleBindings. The operator uses kube-rbac-proxy sidecars for authentication on secured endpoints and supports both PVC and database-backed storage for the TrustyAI service. It integrates with Istio service mesh for traffic management (DestinationRules, VirtualServices) and with Prometheus for metrics monitoring via ServiceMonitors.

| Component | Type | Purpose |
|-----------|------|---------|
| TrustyAIService Controller | Go Controller (controller-runtime) | Manages TrustyAI explainability service deployments, creates Services, Routes, PVCs, kube-rbac-proxy ConfigMaps, ServiceMonitors, and patches InferenceService env vars for ModelMesh payload processing |
| LMEvalJob Controller | Go Controller (controller-runtime) | Creates and manages Pods for LLM evaluation jobs using lm-evaluation-harness, with driver sidecar for progress monitoring and result collection |
| LMEvalJob Kueue Integration (Job Manager) | Go Controller (Kueue JobFramework) | Integrates LMEvalJob with Kueue workload scheduling for resource-aware job queuing |
| GuardrailsOrchestrator Controller | Go Controller (controller-runtime) | Deploys guardrails orchestrator with optional gateway sidecar, built-in detectors, auto-configuration from InferenceServices, kube-rbac-proxy sidecars, Routes, and ServiceMonitors |
| NemoGuardrails Controller | Go Controller (controller-runtime) | Deploys NVIDIA NeMo Guardrails server instances with CA bundle management, kube-rbac-proxy sidecars, Services, and Routes |
| EvalHub Controller | Go Controller (controller-runtime) | Deploys EvalHub evaluation management service with kube-rbac-proxy sidecar, PostgreSQL/SQLite storage, MLFlow integration, and OpenShift Routes |
| Operator Manager | Go Binary | Main entrypoint that starts all enabled controllers (TAS, LMES, GORCH, NEMO_GUARDRAILS, EVALHUB) with leader election |
