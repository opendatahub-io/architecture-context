## Purpose
**Short**: Provides GPU-accelerated training runtime container images and end-to-end integration tests for the RHOAI distributed workloads stack.

**Detailed**: The distributed-workloads repository serves two primary purposes within the RHOAI platform. First, it builds and maintains a set of container images used as training runtimes — these images bundle PyTorch with CUDA or ROCm GPU libraries, InfiniBand/RDMA networking support, and ML acceleration libraries (FlashAttention, NCCL, Mamba SSM). These images are consumed by PyTorchJobs, TrainJobs, and RayJobs when running distributed training workloads on OpenShift AI.

Second, the repository houses a comprehensive Go-based integration test suite that validates the distributed workloads stack across multiple components: Kueue (job queuing and resource management), Kubeflow Training Operator (PyTorchJob orchestration), Kubeflow Trainer v2 (TrainJob orchestration with ClusterTrainingRuntimes), KubeRay (RayCluster and RayJob orchestration), and JobSet. The tests verify end-to-end workflows including model fine-tuning (SFT, LoRA, OSFT), MNIST training, hyperparameter optimization with Ray Tune, and DeepSpeed/FSDP distributed training. Tests also validate Kueue integration for quota management and workload scheduling, including upgrade scenarios.

The repository also includes "universal" training images — hybrid images that can function both as Jupyter notebook workbenches and as headless training runtimes, controlled by an entrypoint script. All images are built via Konflux (Tekton-based CI/CD) and published to quay.io/rhoai/.
