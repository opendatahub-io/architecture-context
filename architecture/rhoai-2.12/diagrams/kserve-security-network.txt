KEY SECURITY CONTROLS
═══════════════════════════════════════════════════════════════════════════════════════════════

1. Webhook Authentication: mTLS client certificates (enforced by kube-apiserver)
2. Controller Authorization: ServiceAccount Token with ClusterRole RBAC
3. Inference Endpoint Auth: Bearer Token (optional) + Istio AuthorizationPolicy (custom per InferenceService)
4. Service Mesh Encryption: mTLS (optional - PERMISSIVE or STRICT mode)
5. External Storage Auth: Cloud provider credentials (AWS IAM, GCP SA, Azure SAS)
6. Pod Security: Non-root UID 1000, no privilege escalation
7. Certificate Management: cert-manager with auto-rotation (90 days)
8. Leader Election: ConfigMaps and Leases in kserve namespace
