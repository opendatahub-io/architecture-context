  • Type: Opaque
  • Purpose: OAuth proxy cookie secret (random base64 string)
  • Provisioned By: ODH Notebook Controller
  • Auto-Rotate: No (static per notebook)
  • Contains: session_secret key

Secret: {notebook-name}-tls (per notebook)
  • Type: kubernetes.io/tls
  • Purpose: OAuth proxy TLS certificate
  • Provisioned By: service.beta.openshift.io/serving-cert-secret-name annotation
  • Auto-Rotate: Yes (OpenShift Service CA)
  • Mounted To: OAuth proxy sidecar

ServiceAccount ImagePullSecrets:
  • Automatically created ServiceAccount: {notebook-name}-sa
  • Controller verifies ImagePullSecrets are mounted before removing reconciliation lock
  • Uses exponential backoff (3 steps, 1s initial, 5.0x factor) to wait for secrets

Data Classification & Compliance Notes
═══════════════════════════════════════════════════════════════════════════════════

Sensitive Data Flows:
  • User credentials: Handled by OpenShift OAuth (external to this component)
  • OAuth cookies: Encrypted via TLS, signed with session_secret
  • Notebook data: User-controlled, stored in notebook containers (ephemeral or PV)
  • ImagePullSecrets: Automatically provisioned, managed by OpenShift

Encryption in Transit:
  • External access: TLS 1.3 (OpenShift Router)
  • Internal webhook: TLS 1.2+ with mTLS (Kubernetes API ↔ Webhook)
  • OAuth proxy: TLS 1.2+ (Router ↔ OAuth proxy sidecar)
  • Notebook backend: HTTP/8888 (localhost only, no encryption needed)
  • Kubernetes API: TLS 1.2+ (all controller API calls)

Encryption at Rest:
  • Secrets: Encrypted at rest by Kubernetes etcd encryption (if enabled)
  • ConfigMaps: Plain text (contains CA certificates only, public by nature)
  • Notebook data: Depends on user PVC configuration (not managed by this controller)

Authentication Mechanisms:
  • Controller → K8s API: ServiceAccount token (JWT)
  • Webhook ← K8s API: mTLS (Kubernetes API client certificate)
  • User → Notebook (OAuth): OpenShift OAuth + Bearer token + SAR check
  • User → Notebook (Non-OAuth): None (public access)

Authorization Enforcement:
  • Controller actions: RBAC (manager-role ClusterRole)
  • User access to notebooks: OpenShift RBAC via SAR (notebooks-view/edit roles)
  • Webhook mutations: Kubernetes admission control (MutatingWebhookConfiguration)
