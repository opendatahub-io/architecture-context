NETWORK POLICIES & SECURITY CONTROLS
═══════════════════════════════════════════════════════════════════════════

Ingress Controls:
  - OpenShift Route: External access via auto-generated hostname
  - TLS Termination: Reencrypt (router terminates external TLS, re-encrypts to service)
  - Service Type: ClusterIP (internal only)
  - OAuth Enforcement: All requests (except /metrics, /oauth/healthz, /api/health)

Authentication Mechanisms:
  1. User Authentication (Browser):
     - OpenShift OAuth 2.0 flow via OAuth Proxy
     - Session cookies (encrypted with dashboard-oauth-config-generated)
     - Bearer tokens (JWT from OpenShift OAuth)
     - Requirement: User must have 'list projects' permission

  2. API Authentication (Backend):
     - Bearer Token validation via K8s TokenReview API
     - ServiceAccount tokens for K8s API access
     - User token passthrough for service proxying

  3. Service-to-Service:
     - ServiceAccount tokens (mounted at /var/run/secrets/kubernetes.io/serviceaccount/)
     - OAuth client secret for OAuth flow

Public Endpoints (Auth Bypassed):
  - /metrics (Prometheus scraping)
  - /oauth/healthz (OAuth proxy health)
  - /api/health (Backend health)

WebSocket Support:
  - Endpoint: /wss/k8s/*
  - Protocol: WebSocket over HTTP
  - Port: 8080/TCP
  - Auth: Bearer Token
  - Purpose: Real-time K8s resource watches

High Availability:

  - Readiness Probe: HTTP /api/health

  - Rolling Update Strategy: MaxUnavailable=1, MaxSurge=1
