Secrets:

1. webhook-server-cert (kubernetes.io/tls)
   - Purpose: TLS certificate for admission webhooks (9443/TCP)
   - Provisioned By: cert-manager or OLM (Operator Lifecycle Manager)
   - Rotation: Manual (no auto-rotation)
   - Used By: Webhook Server component

2. prometheus-tls (kubernetes.io/tls)
   - Purpose: TLS certificate for Prometheus service (9091/TCP)
   - Provisioned By: service.alpha.openshift.io/serving-cert-secret-name annotation
   - Rotation: Automatic (OpenShift certificate rotation)
   - Used By: Prometheus service

3. Generated component secrets (Opaque)
   - Purpose: Component-specific credentials (DB passwords, API tokens, etc.)
   - Provisioned By: SecretGenerator controller
   - Rotation: Manual (no auto-rotation)
   - Examples: Database passwords, OAuth client secrets, S3 credentials

4. odh-trusted-ca-bundle (ConfigMap, not Secret)
   - Purpose: Cluster-wide and custom CA certificates for TLS verification
   - Provisioned By: CertConfigmapGenerator controller
   - Rotation: Automatic (when cluster CA changes)
   - Used By: All components for external HTTPS connections
   - Content: Merged cluster CA + custom CA bundles

Trust Boundaries:

1. External → API Server
   - Boundary: OpenShift OAuth authentication
   - Encryption: TLS 1.2+
   - Trust Level: Zero Trust (authenticate every request)

2. API Server → Operator Webhook
   - Boundary: mTLS (API Server validates webhook cert)
   - Encryption: TLS 1.2+ with client certificates
   - Trust Level: High (cluster control plane)

3. Operator → API Server
   - Boundary: ServiceAccount Token authentication
   - Encryption: TLS 1.2+
   - Trust Level: Medium (RBAC-controlled)

4. Operator → GitHub
   - Boundary: Public internet
   - Encryption: TLS 1.2+
   - Trust Level: Low (public repositories, verify content via git hash)

5. Monitoring → Operator
   - Boundary: kube-rbac-proxy authentication
   - Encryption: TLS 1.2+ (prometheus-tls secret)
   - Trust Level: Medium (internal monitoring)

6. Service Mesh (if enabled)
   - Boundary: mTLS between all mesh members
   - Encryption: mTLS (Istio manages certs)
   - Trust Level: High (mesh internal communication)
