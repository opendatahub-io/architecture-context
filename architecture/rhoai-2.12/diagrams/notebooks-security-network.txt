RBAC Summary (Typical Notebook Pod Permissions):
- ServiceAccount: default or custom SA in user namespace
- ClusterRole/Role: notebook-controller-role (managed by Notebook Controller)
  - API Groups: "" (core)
  - Resources: pods, services, persistentvolumeclaims
  - Verbs: get, list, watch, create, delete

  - API Groups: apps
  - Resources: deployments, statefulsets
  - Verbs: get, list, watch

  - API Groups: route.openshift.io
  - Resources: routes
  - Verbs: get, list, watch, create, delete

  - API Groups: kubeflow.org
  - Resources: notebooks
  - Verbs: get, list, watch, create, update, patch, delete

Secrets (Consumed by Notebook Pods):
- [user]-notebook-token (Opaque) - Jupyter auth token, Notebook Controller, No rotation
- aws-connection-* (Opaque) - AWS credentials for S3/Boto3, User/Data Scientist, No rotation
- [database]-credentials (Opaque) - Database credentials, User/Data Scientist, No rotation
- git-credentials (kubernetes.io/basic-auth or ssh-auth) - Git access, User/Data Scientist, No rotation
- pull-secret (kubernetes.io/dockerconfigjson) - Container registry auth, Platform/User, No rotation

Secrets (Build Time Only):
- rhel-subscription-secret (Opaque) - RHEL subscription for BuildConfigs, Platform Admin, No rotation

Authentication Flow:
1. External User → OpenShift Route (HTTPS/443, TLS1.2+, OAuth Token)
2. Route → OAuth Proxy Sidecar (HTTP/8888, TLS termination at route)
3. OAuth Proxy validates token with OpenShift OAuth server
4. OAuth Proxy → JupyterLab Container (HTTP/8888, localhost, proxied auth)
5. JupyterLab may also have Jupyter token auth (configured by controller)

Network Policies (if deployed):
- Ingress: Allow from OpenShift Router namespace on port 8888
- Egress: Allow to Kubernetes API (6443), PyPI (443), S3 (443), Databases (various)
- Default deny all other traffic (optional, depends on cluster policy)

Security Considerations:
- Notebook runs as UID 1001 (non-root)
- OAuth Proxy provides authentication layer
- ServiceAccount RBAC limits cluster access
- Secrets mounted as volumes (not environment variables for sensitive data)
- TLS encryption for all external communication
- Optional: PodSecurityPolicy/SecurityContextConstraints enforcement
- Optional: NetworkPolicy for namespace isolation
- Optional: Resource quotas and limits on notebook pods
