RBAC Summary:
  No Kubernetes RBAC — this is a containerized service, not an operator.
  RBAC is managed by KServe which deploys vLLM as an InferenceService runtime.

Secrets:
  - SSL certificate/key (kubernetes.io/tls) - TLS termination, cert-manager or manual, auto-rotate via --enable-ssl-refresh
  - API key (Opaque) - Bearer token auth, manual/platform config, no auto-rotate
  - HuggingFace token (Opaque) - HF_TOKEN env var for model repo access, manual, no auto-rotate

Crypto / FIPS Notes:
  - Base image: UBI 9 (system OpenSSL for FIPS-validated crypto)
  - Python ssl module delegates to system OpenSSL
  - hashlib.md5 used for config hashing (SHA256 fallback in FIPS mode via safe_hash())
  - blake3 default multimodal hasher (set VLLM_MM_HASHER_ALGORITHM=sha256 for FIPS)
  - OTLP gRPC exporter defaults to insecure channel (configure TLS for FIPS)

Security Recommendations:
  1. Restrict CORS (default allows all origins/methods/headers)
  2. Network policy to restrict access to unauthenticated operational endpoints
  3. Enable TLS (--ssl-certfile/--ssl-keyfile) in production
  4. Set VLLM_MM_HASHER_ALGORITHM=sha256 for FIPS compliance
  5. Configure OTLP exporter with TLS for FIPS compliance
  6. Protect /update_weights and /collective_rpc via service mesh AuthorizationPolicy
