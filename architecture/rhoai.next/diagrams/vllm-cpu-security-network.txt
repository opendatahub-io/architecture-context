RBAC Summary:
  No RBAC — this is a containerized service, not a Kubernetes operator.
  RBAC is managed by KServe InferenceService.

Authentication & Authorization:
  - /v1/* paths: Bearer Token (API key, SHA256-hashed, constant-time compare)
    → Reject 401 if token invalid
  - /health, /version, /load: Open access (no auth)
  - /tokenize, /detokenize, /reset_*: Open access (no auth)
  - /pause, /resume, /sleep, /wake_up: Open access — operational endpoints
  - /update_weights, /collective_rpc: Open access — ⚠️ sensitive operational
  - OPTIONS (all paths): Open access (CORS preflight)
  - CORS: Defaults to allowing ALL origins, methods, headers
    ⚠️ Should be restricted in production

Secrets:
  - SSL certificate/key (kubernetes.io/tls)
    Purpose: TLS termination for HTTPS
    Provisioned by: cert-manager or manual mount
    Auto-rotate: Yes (--enable-ssl-refresh, async watchfile)
  - API key (Opaque)
    Purpose: Bearer token authentication (--api-key or VLLM_API_KEY)
    Provisioned by: Manual / platform config
    Auto-rotate: No
  - HuggingFace token (Opaque)
    Purpose: Model repository access (HF_TOKEN env)
    Provisioned by: Manual
    Auto-rotate: No

FIPS Compliance:
  - Base image: UBI 9 (ubi-minimal) — OpenSSL from system
  - Python ssl module delegates to system OpenSSL
  - ⚠️ MD5 used by default for config hashing (SHA256 fallback in FIPS mode)
  - ⚠️ blake3 used by default for multimodal hashing
    (set VLLM_MM_HASHER_ALGORITHM=sha256 for FIPS)
  - ⚠️ OTLP gRPC exporter uses insecure channel by default

Security Recommendations:
  1. Restrict CORS origins in production (currently allows all)
  2. Use network policies to protect unauthenticated operational endpoints
     (/pause, /resume, /sleep, /wake_up, /update_weights, /collective_rpc)
  3. Set VLLM_MM_HASHER_ALGORITHM=sha256 for FIPS-compliant deployments
  4. Enable TLS on OTLP exporter endpoint
  5. Enable TLS on gRPC server (50051/TCP currently unencrypted)
  6. Restrict access to profiling endpoints (/start_profile, /stop_profile)
