INTEGRATION POINTS:
- KServe: Deploys vLLM-CPU as ServingRuntime container, routes traffic (8000/TCP HTTP)
- Prometheus: Scrapes /metrics endpoint (8000/TCP HTTP, no auth)
- MCP Tool Servers: Optional tool calling via Model Context Protocol (configurable, HTTP)

RBAC Summary:
- No ClusterRoles or RoleBindings defined
- vLLM-CPU runs as a regular workload pod managed by KServe
- Container runs as non-root user (UID 2000, GID 0)

Secrets:
- (HF_TOKEN)                              Opaque (env var) - HF Hub auth, Manual, No auto-rotate
- (S3_ACCESS_KEY_ID/S3_SECRET_ACCESS_KEY) Opaque (env var) - S3 auth, Manual, No auto-rotate
- (VLLM_API_KEY)                          Opaque (env var) - Client API key auth, Manual, No auto-rotate
- (ssl_keyfile/ssl_certfile)              kubernetes.io/tls - HTTPS serving, cert-manager/Manual, Yes (--enable-ssl-refresh)

Security Notes:
- ⚠️  gRPC port (50051) is insecure only - no TLS support
- ⚠️  HTTP port (8000) defaults to plaintext - TLS requires explicit --ssl-certfile/--ssl-keyfile
- ✅  API Key authentication available for /v1/* endpoints
- ✅  Non-root container execution (UID 2000)
- ✅  Usage stats telemetry disabled by default (VLLM_NO_USAGE_STATS=1)
- ✅  TLS certificate hot-reload supported via --enable-ssl-refresh
- ✅  CORS configurable via --allowed-origins
