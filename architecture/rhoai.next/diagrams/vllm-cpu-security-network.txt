RBAC Summary:
- No Kubernetes RBAC — vllm-cpu runs as a non-root user (UID 2000, GID 0)
  within a container and does not interact with the Kubernetes API.

Authentication Summary:
- HTTP API (/v1/*): Bearer Token via --api-key flag
  → FastAPI AuthenticationMiddleware rejects 401 if api-key is set and token is missing/invalid
- Health/Metrics (/health, /ping, /metrics, /version, /load): No auth required
- gRPC (50051/TCP): No built-in authentication

Secrets:
- HF_TOKEN (Opaque env var) — HuggingFace Hub auth for gated models, no auto-rotate
- VLLM_API_KEY (Opaque env var) — API authentication for HTTP endpoints, no auto-rotate
- S3_ACCESS_KEY_ID / S3_SECRET_ACCESS_KEY (Opaque env var) — AWS S3 credentials, no auto-rotate
- SSL cert/key (kubernetes.io/tls via --ssl-certfile/--ssl-keyfile) — TLS termination,
  auto-rotate via SSLCertRefresher hot-reload

FIPS Compliance:
- Python service inherits FIPS crypto from UBI9 base image's OpenSSL
- Python ssl module delegates to system OpenSSL (FIPS mode when cluster FIPS enabled)
- Non-FIPS risk: blake3 hash used for multimodal content dedup (not security-critical)
- No explicit FIPS opt-in/opt-out in application code

Build Security:
- Build: Non-hermetic (hermetic: "false" in Tekton pipelines)
- No lock files (rpms.lock.yaml, uv.lock) on main branch
- Dependencies fetched from PyPI, PyTorch CPU index, IBM DevPI at build time
- Supply chain risk: builds not reproducible
