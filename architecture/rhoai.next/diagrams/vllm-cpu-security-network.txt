RBAC Summary:
- No RBAC definitions — this is a stateless service container, not an operator.
- RBAC is managed externally by KServe and the platform operator.
- Pod runs as non-root user 2000, group 0 (OpenShift best practice).

Secrets:
- SSL key/cert files (kubernetes.io/tls) - TLS encryption for HTTP server
  - Provisioned by: Manual or cert-manager
  - Auto-Rotate: Yes, via --enable-ssl-refresh (SSLCertRefresher watches file changes)

- HF_TOKEN (Opaque) - Hugging Face Hub authentication
  - Provisioned by: Manual
  - Auto-Rotate: No

- S3_ACCESS_KEY_ID / S3_SECRET_ACCESS_KEY (Opaque) - S3 model storage auth
  - Provisioned by: Manual
  - Auto-Rotate: No

- VLLM_API_KEY (Opaque) - API key for request authentication
  - Provisioned by: Manual
  - Auto-Rotate: No

Authentication & Authorization:
- /v1/* endpoints: Bearer Token (API key in Authorization header)
  - Enforcement: FastAPI AuthenticationMiddleware
  - Policy: Reject with 401 if key invalid; skip if --api-key not set
- /health, /metrics, /version, /ping: No auth required (always accessible)
- /start_profile, /stop_profile: Dev mode only (VLLM_SERVER_DEV_MODE env var)
- gRPC 50051/TCP: No authentication (⚠️ security concern for production)

FIPS Compliance:
- Python service on UBI9 base image — inherits FIPS-compliant OpenSSL
- Python ssl module uses system OpenSSL (FIPS-approved in FIPS mode)
- MD5 usage: hashlib.md5(usedforsecurity=False) for content hashing only
- No Go binaries — check-payload validation N/A for Go constraints
- Non-hermetic build (hermetic: false) on main branch — expected for EA

Build Security:
- Non-hermetic build: Dependencies fetched from PyPI at build time
- No lock files (rpms.lock.yaml, uv.lock) — pinned versions only
- Container runs as non-root (USER 2000, GROUP 0)
- Base image: UBI9 ubi-minimal (Red Hat supported, security patched)
