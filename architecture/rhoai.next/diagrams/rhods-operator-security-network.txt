  Provisioned by: Operator (gateway controller), No auto-rotate (regenerated on change)
- kube-auth-proxy-tls (kubernetes.io/tls) - TLS certificate for kube-auth-proxy service
  Provisioned by: OpenShift Service CA Operator, Auto-rotate: Yes
- data-science-prometheus-cluster-proxy-kube-rbac-proxy (Opaque) - kube-rbac-proxy auth config
  Provisioned by: Operator (monitoring controller), No auto-rotate

Webhook Configuration:
- Mutating webhooks: /mutate-* on port 9443/TCP (HTTPS, TLS cert-manager/service CA)
- Validating webhooks: /validate-* on port 9443/TCP (HTTPS, TLS cert-manager/service CA)
- Only Kubernetes API server can call webhooks (TLS client cert authentication)

Authentication Modes:
1. Integrated OAuth: OpenShift OAuth Server → kube-auth-proxy → ext_authz EnvoyFilter
2. External OIDC: External OIDC Provider → kube-auth-proxy → ext_authz EnvoyFilter
3. No-Auth: External management (gateway deployed without auth proxy)
