RBAC Summary:
- No RBAC manifests in this repository
- RBAC is managed externally by the deploying operator (rhods-operator)
- Service runs as a stateless container; no Kubernetes API access required

Secrets:
- TLS server cert (kubernetes.io/tls) — Server TLS certificate for guardrails API
  Provisioned by: Manual / cert-manager (external), No auto-rotation (file-based)
- TLS server key (kubernetes.io/tls) — Server TLS private key
  Provisioned by: Manual / cert-manager (external), No auto-rotation (file-based)
- TLS client CA cert (Opaque) — Client CA certificate for mTLS verification
  Provisioned by: Manual / cert-manager (external), No auto-rotation (file-based)
- Per-service TLS certs (Opaque) — Client TLS certificates for upstream connections
  Provisioned by: Manual / cert-manager (external), configured via YAML
- API tokens (Opaque) — Bearer tokens for upstream HTTP services
  Provisioned by: Manual, via environment variable references

Authentication & Authorization:
- All /api/* endpoints: Header passthrough (X-Forwarded-Access-Token rewritable
  to Bearer); orchestrator does NOT enforce auth itself
- Optional mTLS: When tls_client_ca_cert_path is configured, clients must present
  valid certificates signed by the specified CA
- Health endpoints (/health, /info): Open access, no authentication

TLS Configuration:
- Library: rustls 0.23.36 with ring crypto backend
- Protocols: TLS 1.2+ (hyper-rustls tls12 feature enabled)
- Certificate Formats: PEM (PKCS#1, PKCS#8, SEC1 key formats)
- Named TLS configs shareable across upstream services via YAML
- INSECURE MODE: NoVerifier bypasses ALL certificate verification when
  "insecure: true" is configured — SECURITY RISK in production

FIPS Compliance:
- ⚠️  SIGNIFICANT GAP: Application uses rustls/ring for all TLS operations
- ring is NOT FIPS 140-2/3 validated
- OpenSSL is installed in container image (satisfies check-payload)
- Runtime TLS does NOT use OpenSSL — uses ring instead
- DISA STIG remediation script sets system FIPS policy but this only affects
  OpenSSL-based applications, not the rustls-based orchestrator binary
