RBAC Summary:
  No RBAC manifests in this repository. RBAC is managed by the deploying operator
  (rhods-operator). The orchestrator does not interact with the Kubernetes API
  directly and does not require its own ClusterRole or RoleBinding.

TLS Configuration:
  - Server TLS: rustls 0.23.36 with ring crypto backend
  - TLS versions: TLS 1.2+ (hyper-rustls tls12 feature enabled)
  - mTLS: Optional, via WebPkiClientVerifier when tls_client_ca_cert_path is set
  - Client TLS: Per-service configurable via named TLS configs in YAML
  - INSECURE MODE: NoVerifier bypasses ALL cert verification when insecure: true
  - FIPS STATUS: NOT FIPS-compliant (ring is not FIPS 140-2/3 validated)

Secrets:
  - TLS server cert (kubernetes.io/tls) - Server TLS certificate, manual/cert-manager, no auto-rotate
  - TLS server key (kubernetes.io/tls) - Server TLS private key, manual/cert-manager, no auto-rotate
  - TLS client CA cert (Opaque) - Client CA cert for mTLS verification, manual/cert-manager, no auto-rotate
  - Per-service TLS certs (Opaque) - Client TLS certs for upstream connections, manual/cert-manager, no auto-rotate
  - API tokens (Opaque) - Bearer tokens for upstream HTTP services (env var references), manual, no auto-rotate

FIPS Compliance Gaps:
  - CRITICAL: All TLS operations use rustls with ring crypto backend, which is NOT FIPS 140-2/3 validated
  - OpenSSL is installed in container image (satisfies check-payload) but is NOT used for runtime TLS
  - NoVerifier implementation completely bypasses certificate verification when insecure: true
  - DISA STIG remediation script sets crypto policy to FIPS, but only affects OpenSSL-based apps
  - Recommendation: Switch to FIPS-validated TLS backend (openssl-based) for FIPS-compliant deployments

Authentication:
  - Orchestrator does NOT enforce authentication itself
  - Designed to sit behind an auth proxy (kube-rbac-proxy, oauth-proxy)
  - Configurable header passthrough with X-Forwarded-Access-Token → Bearer rewrite
  - Upstream detector services may validate Bearer tokens passed through
