RBAC Summary:
  No RBAC manifests in this repository. RBAC is managed by the deploying
  operator (rhods-operator).

TLS Configuration:
  - Library: rustls 0.23.36 with ring crypto backend
  - Features: TLS 1.2+ enabled (hyper-rustls tls12 feature)
  - Server TLS: rustls::ServerConfig with WebPkiClientVerifier for mTLS
  - Client TLS: Per-service configurable TLS with named TLS configurations
  - INSECURE MODE: NoVerifier implementation bypasses all certificate
    verification when insecure: true is configured
  - FIPS GAP: ring is NOT FIPS 140-2/3 validated; OpenSSL is installed in
    container image but not used for TLS operations

Secrets:
  - TLS server cert (kubernetes.io/tls) - Server TLS certificate, manual/cert-manager
  - TLS server key (kubernetes.io/tls) - Server TLS private key, manual/cert-manager
  - TLS client CA cert (Opaque) - Client CA cert for mTLS verification
  - Per-service TLS certs (Opaque) - Client TLS certs for upstream connections
  - API tokens (Opaque) - Bearer tokens for upstream HTTP services (env var refs)

Authentication & Authorization:
  - All /api/* endpoints: Header passthrough (no enforcement by orchestrator)
  - X-Forwarded-Access-Token rewritten to Authorization: Bearer header
  - Designed to operate behind auth proxy (kube-rbac-proxy / oauth-proxy)
  - Optional mTLS: When tls_client_ca_cert_path is configured, clients must
    present valid certificates signed by the specified CA
  - Health endpoints: Open access (no auth)

FIPS Compliance Warning:
  - Container image: OpenSSL 3.5.1-7.el9_7 installed (passes check-payload)
  - Runtime TLS: rustls + ring (NOT FIPS-certified)
  - DISA STIG remediation script sets system crypto policy to FIPS, but only
    affects OpenSSL-based applications, not the rustls-based binary
  - Status: SIGNIFICANT FIPS COMPLIANCE GAP
