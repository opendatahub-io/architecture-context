TLS Configuration:
- Server Listener: TLS 1.2+ minimum, ECDHE cipher suites only
  - AES-128-GCM-SHA256, AES-256-GCM-SHA384, CHACHA20-POLY1305-SHA256
  - Self-signed RSA-4096 if no cert path provided (development mode)
- Prefiller Connection: Optional TLS via -prefiller-use-tls flag
  - InsecureSkipVerify available via -prefiller-tls-insecure-skip-verify
- Decoder Connection: Optional TLS via -decoder-use-tls flag
  - InsecureSkipVerify available via -decoder-tls-insecure-skip-verify

SSRF Protection:
- Optional, enabled via -enable-ssrf-protection=true
- Watches InferencePool CR to discover pod selector
- Watches matching Pods to build allowlist of IPs and hostnames
- Validates x-prefiller-host-port header against allowlist
- Returns HTTP 403 if prefiller target not in allowlist
- Tracks host identifiers only (not ports); any port on allowed host is permitted

FIPS Compliance:
- Konflux build: CGO_ENABLED=1, dynamic linking against OpenSSL (FIPS-compatible)
- Upstream build: CGO_ENABLED=0, static linking (would fail check-payload)
- GOEXPERIMENT=strictfipsruntime NOT set in Konflux Dockerfile
- All crypto: stdlib crypto/tls, crypto/rsa, crypto/x509, crypto/rand
