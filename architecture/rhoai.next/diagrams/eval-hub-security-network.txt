RBAC Summary:
- Role (managed by RHOAI operator):
  - API Groups: batch          Resources: jobs                    Verbs: create, get, list, delete
  - API Groups: ""             Resources: configmaps              Verbs: create, get, list, delete, update
  - API Groups: ""             Resources: secrets                 Verbs: get
  - API Groups: authentication.k8s.io  Resources: tokenreviews   Verbs: create
  - API Groups: authorization.k8s.io   Resources: subjectaccessreviews  Verbs: create

- Role Bindings:
  - eval-hub role → eval-hub service account (eval-hub namespace)
  - job role → <instance-name>-<instance-namespace>-job SA (job namespace)

Authentication & Authorization:
- /api/v1/evaluations/*: Bearer token → K8s TokenReview (AuthN) → SubjectAccessReview (AuthZ)
  - HTTP verb mapping: GET→get, POST→create, PUT→update, PATCH→patch, DELETE→delete
  - Resource attributes are configurable
- /api/v1/health, /openapi.yaml, /docs, /metrics: Unauthenticated

Secrets:
- Database password (Opaque) — PostgreSQL credentials, provisioned by operator/admin, no auto-rotate
- OCI credentials (kubernetes.io/dockerconfigjson) — Registry auth, user-provided, no auto-rotate
- Model auth (Opaque) — Model endpoint API keys/tokens, user-provided, no auto-rotate
- Test data S3 credentials (Opaque) — AWS credentials, user-provided, no auto-rotate
- Service CA ConfigMap (service-ca.crt) — Internal TLS CA, OpenShift auto-provisioned, auto-rotated
- MLflow token (Projected ServiceAccountToken) — K8s auth for MLflow, auto-rotated (1h expiry)

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamic linking, UBI9 base
- Runtime: TLS 1.2–1.3 enforced, stdlib crypto/tls via OpenSSL
- Guard: Server refuses InsecureSkipVerify when /proc/sys/crypto/fips_enabled is active

No Ingress Defined:
- eval-hub is an internal ClusterIP service
- External access provided by platform operator via HTTPRoute (RHOAI 3.x) or Route (RHOAI 2.x)
