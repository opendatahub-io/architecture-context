RBAC Summary:
- ClusterRole (implied by TokenReview): authentication.k8s.io/tokenreviews → create
- ClusterRole (implied by SAR): authorization.k8s.io/subjectaccessreviews → create
- ClusterRole (implied by Job mgmt): batch/jobs → create, get, list, delete
- ClusterRole (implied by ConfigMap mgmt): core/configmaps → create, get, list, delete, update
- RoleBinding (operator-managed): control plane namespace, eval-hub ClusterRole → eval-hub SA
- RoleBinding (operator-managed): evaluation namespaces → {evalhub-instance}-{namespace}-job SA

Authorization Rules (SAR-based):
- trustyai.opendatahub.io/evaluations:{verb} — evaluation job operations
- trustyai.opendatahub.io/collections:{verb} — benchmark collection management
- trustyai.opendatahub.io/providers:{verb} — evaluation provider management
- trustyai.opendatahub.io/status-events:create — job status event posting
- mlflow.kubeflow.org/experiments:{verb} — MLflow experiment operations
- Namespace derived dynamically from X-Tenant HTTP header

Secrets:
- TLS cert/key (kubernetes.io/tls) — eval-hub server TLS, cert-manager / OpenShift service CA, auto-rotate
- MLflow token (Projected SA Token) — MLflow kubernetes-auth, kubelet rotation, auto-rotate
- OCI credentials (kubernetes.io/dockerconfigjson) — OCI registry push auth, user-created, no auto-rotate
- Model auth (Opaque) — model serving endpoint auth, user-created, no auto-rotate
- S3 credentials (Opaque) — AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, etc., user-created, no auto-rotate
- Database credentials (Opaque) — PostgreSQL connection string, operator/user, no auto-rotate

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1 (Dockerfile.konflux)
- Crypto: stdlib crypto/tls via OpenSSL (strictfipsruntime)
- TLS: MinVersion=TLS1.2, MaxVersion=TLS1.3
- InsecureSkipVerify: blocked when FIPS mode detected (/proc/sys/crypto/fips_enabled)
- Base image: ubi9/ubi-minimal (OpenSSL included)
