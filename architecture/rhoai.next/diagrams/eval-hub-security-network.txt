Credential Injection Pattern:
  1. User creates Secret with real model credentials (api-key, *_api-key, *_url)
  2. eval-hub K8s runtime detects proxy-injectable keys
  3. Creates ephemeral "{job}-{guid}-model-ref" Secret with synthetic ref-token values
  4. Adapter container receives ref-token Secret (never real credentials)
  5. Sidecar receives real credentials Secret
  6. Sidecar intercepts model requests, replaces ref-token with real API key
  → Adapter framework code NEVER sees real credentials

FIPS Compliance:
  - Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
  - Base image: UBI9 ubi-minimal (OpenSSL included)
  - TLS: MinVersion=TLS 1.2, MaxVersion=TLS 1.3
  - FIPS mode rejects InsecureSkipVerify
  - No non-FIPS crypto imports detected (no md5, rc4, des)
