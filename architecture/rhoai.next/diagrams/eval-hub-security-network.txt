RBAC Summary:
- eval-hub does NOT define its own ClusterRoles or RoleBindings
- Authentication: Delegated to Kubernetes API (TokenReview)
- Authorization: Delegated to Kubernetes API (SubjectAccessReview)
- Auth Resource Mappings:
  - trustyai.opendatahub.io/evaluations → /api/v1/evaluations/jobs
  - trustyai.opendatahub.io/collections → /api/v1/evaluations/collections
  - trustyai.opendatahub.io/providers → /api/v1/evaluations/providers
  - trustyai.opendatahub.io/status-events → /api/v1/evaluations/jobs/{id}/events
  - mlflow.kubeflow.org/experiments → MLflow experiment access (on job create)
- Multi-tenancy: X-Tenant header mapped to K8s namespace for SAR checks
- Public endpoints (no auth): /api/v1/health, /metrics, /openapi.yaml, /docs

Secrets:
- TLS cert/key (kubernetes.io/tls) - HTTPS for eval-hub server, cert-manager, auto-rotate
- Database credentials (Opaque) - PostgreSQL connection URL, operator-provisioned, no auto-rotate
- MLflow token (Projected SA Token) - MLflow API auth, 3600s expiry, auto-rotate by kubelet
- S3 credentials (Opaque) - AWS access key, secret key, region, endpoint, user-provisioned
- OCI credentials (kubernetes.io/dockerconfigjson) - Docker registry auth, user-provisioned
- Model auth (Opaque) - Model endpoint authentication, user-provisioned
- Service CA (ConfigMap) - OpenShift service CA certificate, service-ca operator, auto-rotate

Kueue Integration:
- Evaluation Jobs labelled with kueue.x-k8s.io/queue-name
- Priority-based scheduling via Kueue admission controller
