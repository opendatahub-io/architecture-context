SECURITY NOTES:

⚠️  KFP Pipeline Pods → Llama Stack Server communication is unencrypted HTTP
    without authentication. The KUBEFLOW_LLAMA_STACK_URL must be accessible
    from KFP pods. This is a security concern for sensitive evaluation data.

⚠️  Llama Stack Server external endpoint (8321/TCP) has no TLS or auth by
    default. Encryption and authentication are server-configurable but not
    enforced by this provider.

Secrets:
- (user-provided S3 credentials) [Opaque] - AWS credentials for S3 result storage
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_DEFAULT_REGION
  - Provisioned: Manual (oc create secret generic)
  - Auto-Rotate: No

- kubeflow-pipelines-token [Opaque] - KFP API bearer token
  - Provisioned: Manual
  - Auto-Rotate: No

- llama-stack-inference-model-secret [Opaque] - vLLM URL, API token, TLS config
  - Keys: vLLM URL, API token, TLS verification flag, model name
  - Provisioned: Manual
  - Auto-Rotate: No

- aws-credentials [Opaque] - AWS credentials for S3 from LlamaStack pod
  - Provisioned: Manual
  - Auto-Rotate: No

RBAC Summary:
- No RBAC roles defined by this component
- Relies on Llama Stack server's ServiceAccount permissions
- KFP pods use credentials injected via kubernetes.use_secret_as_env()

Authentication Mechanisms:
- Kubeflow Pipelines: Bearer Token (from KUBEFLOW_PIPELINES_TOKEN env var or kubeconfig)
- S3 Storage: AWS IAM credentials (injected via Kubernetes Secret into KFP pods)
- Kubernetes API: ServiceAccount token or kubeconfig (for ConfigMap reads)
- Llama Stack Server: None (server-configurable, not provider-enforced)

Base Image Resolution Chain (Remote Provider):
1. Environment variable (highest priority)
2. Kubernetes ConfigMap: trustyai-service-operator-config
   - Namespace: redhat-ods-applications or opendatahub
3. Hardcoded default Quay image (lowest priority)
