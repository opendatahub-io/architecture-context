RBAC Summary:
- No ClusterRoles or ClusterRoleBindings defined by this component
- Provider relies on Llama Stack server's ServiceAccount
- KFP pods use credentials injected via Kubernetes Secrets

Secrets:
- (user-provided S3 credentials) (Opaque) - AWS credentials for S3 results, Manual, No auto-rotate
- kubeflow-pipelines-token (Opaque) - KFP API token for pipeline submission, Manual, No auto-rotate
- llama-stack-inference-model-secret (Opaque) - vLLM URL/API token/TLS/model name, Manual, No auto-rotate
- aws-credentials (Opaque) - AWS credentials for S3 from LlamaStack pod, Manual, No auto-rotate

Security Concerns:
⚠️  KFP pipeline pods communicate with Llama Stack server over HTTP (8321/TCP)
    without TLS or authentication. The KUBEFLOW_LLAMA_STACK_URL must be
    accessible from KFP pods, and traffic is unencrypted within the cluster.
⚠️  No RBAC roles defined - relies entirely on the host server's SA permissions.
⚠️  All secrets are manually provisioned with no auto-rotation.

Authentication Flow:
- KFP API: Bearer token from KUBEFLOW_PIPELINES_TOKEN env var, or extracted
  from local kubeconfig (~/.kube/config) as fallback
- S3: AWS IAM credentials injected into KFP pods via Kubernetes Secrets
  using kfp-kubernetes use_secret_as_env()
- Kubernetes API: ServiceAccount token for ConfigMap reads; kubeconfig for
  KFP token fallback
- Base image resolution chain: env var → ConfigMap (trustyai-service-operator-config)
  → hardcoded default (quay.io)
