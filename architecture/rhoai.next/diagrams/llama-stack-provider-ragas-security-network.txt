================================================================================
  SECURITY NETWORK DIAGRAM: llama-stack-provider-ragas
================================================================================

  This component is a library/plugin deployed in two modes:
  - Inline: runs within the Llama Stack Server process
  - Remote: submits KFP pipeline runs (ephemeral pods)

  No Kubernetes Services or Ingress defined by this component.
  All external access is via egress connections.

================================================================================
  EGRESS CONNECTIONS
================================================================================

  (1) Kubernetes API Server
      Port:        6443/TCP
      Protocol:    HTTPS
      Encryption:  TLS 1.2+
      Auth:        Bearer Token (kubeconfig)
      Purpose:     Read ConfigMap (trustyai-service-operator-config)
                   for ragas-provider-image; extract kubeconfig token
      Namespaces:  redhat-ods-applications, opendatahub (fallback)

  (2) Kubeflow Pipelines API
      Port:        443/TCP
      Protocol:    HTTPS
      Encryption:  TLS 1.2+
      Auth:        Bearer Token (OpenShift OAuth)
      Token:       KUBEFLOW_PIPELINES_TOKEN env var or kubeconfig extraction
      Token Type:  Pydantic SecretStr (redacted in logs)
      Purpose:     Submit pipeline runs (create_run_from_pipeline_func),
                   check run status (get_run), cancel runs

  (3) S3-compatible Object Storage
      Port:        443/TCP
      Protocol:    HTTPS
      Encryption:  TLS 1.2+
      Auth:        AWS IAM credentials (ACCESS_KEY_ID, SECRET_ACCESS_KEY)
      Credentials: From Kubernetes Secret mounted as env vars in KFP pods
      Purpose:     Write evaluation results (JSONL), read results back
      Libraries:   s3fs, pandas (for result retrieval)

  (4) Llama Stack Server (from KFP Pod: retrieve_data)
      Port:        8321/TCP
      Protocol:    HTTP
      Encryption:  None (cluster-internal)
      Auth:        None
      Purpose:     Retrieve dataset rows for evaluation

  (5) Llama Stack Server (from KFP Pod: run_ragas)
      Port:        8321/TCP
      Protocol:    HTTP
      Encryption:  None (cluster-internal)
      Auth:        None
      Purpose:     LLM inference and embedding generation for Ragas metrics

  (6) S3 Object Storage (from KFP Pod: run_ragas)
      Port:        443/TCP
      Protocol:    HTTPS
      Encryption:  TLS 1.2+
      Auth:        AWS IAM credentials (from Kubernetes Secret env vars)
      Purpose:     Write evaluation results as JSONL

================================================================================
  DEV-ONLY EGRESS (not in production)
================================================================================

  Ollama Server (dev distribution only)
      Port:        11434/TCP
      Protocol:    HTTP
      Encryption:  None
      Auth:        None
      Purpose:     Inference backend for sample distribution/run.yaml

================================================================================
  SECRETS
================================================================================

  Secret: {KUBEFLOW_S3_CREDENTIALS_SECRET_NAME} (user-defined)
    Type:          Opaque
    Data Keys:     AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_DEFAULT_REGION
    Purpose:       S3 credentials for evaluation result storage
    Provisioned:   Manual (oc create secret)
    Rotation:      No auto-rotation
    Exposure:      Mounted as env vars in KFP pods via use_secret_as_env()

  Token: KUBEFLOW_PIPELINES_TOKEN
    Type:          SecretStr (Pydantic)
    Purpose:       Bearer token for KFP API authentication
    Provisioned:   Manual env var or kubeconfig extraction
    Rotation:      No auto-rotation
    Redaction:     Yes (Pydantic SecretStr - redacted in logs/serialization)

================================================================================
  RBAC
================================================================================

  No RBAC manifests defined by this component.

  RBAC is inherited from:
  - The KFP execution environment (ServiceAccount running pipeline pods)
  - The ServiceAccount running the Llama Stack server
  - KFP pods require: ConfigMap read access in operator namespaces

================================================================================
  AUTHENTICATION & AUTHORIZATION
================================================================================

  | Endpoint                 | Auth Mechanism                  | Enforcement       |
  |--------------------------|---------------------------------|-------------------|
  | KFP API /apis/v2beta1/* | Bearer Token (OpenShift OAuth)  | KFP server        |
  | Llama Stack Server       | None (cluster-internal)         | Application-level |
  | S3 Object Storage        | AWS IAM credentials             | S3 service        |
  | Kubernetes API           | Bearer Token (kubeconfig)       | API server        |

================================================================================
  FIPS COMPLIANCE
================================================================================

  - Base image: UBI9 python-312 (OpenSSL included)
  - Crypto library: cryptography==46.0.2 (FIPS-capable via OpenSSL)
  - TLS config: Delegated to upstream HTTP clients (requests, kfp, s3fs)
  - Certificate handling: System trust store (UBI9)
  - No direct crypto usage in application code
  - No non-FIPS crypto risks detected
