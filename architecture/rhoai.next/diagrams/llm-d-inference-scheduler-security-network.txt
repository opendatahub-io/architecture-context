Authentication & Authorization:
- /metrics endpoint: TokenReview + SubjectAccessReview (when --metrics-endpoint-auth=true)
  - Enforcement: EPP metrics server validates via Kubernetes API
  - Policy: Kubernetes RBAC
- PD Sidecar endpoints (/v1/chat/completions, /v1/completions, /v1/responses):
  - Auth: SSRF allowlist from InferencePool pod IPs (when --enable-ssrf-protection=true)
  - Enforcement: PD sidecar validates destination IPs against InferencePool pod list
  - Policy: Only forward to pods listed in watched InferencePool resource
- gRPC ext-proc (9002/TCP): No auth (Envoy-internal trust boundary)
- gRPC Health (9003/TCP): No auth (open health checks)

Secrets:
- EPP TLS cert/key pair (kubernetes.io/tls)
  - Purpose: gRPC server TLS for ext-proc and health endpoints
  - Provisioned By: Self-signed or external
  - Rotation: fsnotify cert reloading with 250ms debounce
- PD Sidecar TLS cert/key (kubernetes.io/tls)
  - Purpose: HTTPS server TLS for sidecar proxy
  - Provisioned By: Self-signed (RSA 4096-bit, 10-year validity) or external
  - Rotation: fsnotify cert reloading

Network Security Notes:
- EPP-to-Envoy communication uses TLS 1.2+ when --secure-serving=true (default)
- PD Sidecar supports per-stage TLS configuration (prefiller, decoder, encoder)
- SSRF protection prevents header-injection attacks on PD sidecar routing
- ZMQ subscriber (5557/TCP) uses plaintext TCP - internal cluster only
- UDS Tokenizer uses Unix Domain Socket (no network exposure)
- Metrics endpoint supports optional Kubernetes RBAC authentication
