Security Configuration:
─────────────────────

RBAC:
  - Not applicable (Python library, not a Kubernetes component)
  - No ClusterRoles, Roles, or RoleBindings defined
  - RBAC is managed by the caikit runtime's Kubernetes deployment

Secrets / Credentials:
  - TLS CA cert (ca_cert_file)       — File path — Verifies TGIS server TLS cert
  - TLS client cert (client_cert_file) — File path — Client cert for mTLS to TGIS
  - TLS client key (client_key_file)   — File path — Client key for mTLS to TGIS
  - Provisioned by: Manual / platform configuration
  - Auto-rotate: No

Authentication & Authorization:
  - TGIS connection: mTLS (optional), enforced at TLS handshake
  - Configuration: runtime_config.yaml → tls → ca_cert_file, client_cert_file, client_key_file
  - HuggingFace Hub: API token (optional), via HF_TOKEN environment variable

Security-Relevant Configuration:
  - allow_downloads: false (default) — Prevents arbitrary URL fetching during model bootstrap
  - trust_remote_code: false (default) — Prevents execution of untrusted code in model repos
  - base_models_dir: Restricts filesystem access for model loading
  - source_prompt_base: Restricts filesystem access for prompt loading
  - TWO_DOTS_REGEX: Path validation to prevent directory traversal attacks

FIPS Compliance:
  - No direct crypto in caikit-nlp source code
  - TLS delegated to grpcio (uses system OpenSSL)
  - Base image: UBI9-minimal (provides FIPS-validated OpenSSL)
  - Risk: Not using AIPCC base images (evaluated for migration)

Build Hermeticity:
  - WARNING: No lock files at any layer (RPM, Python, artifacts)
  - pip install without --require-hashes or --no-deps
  - No Hermeto/cachi2 prefetch integration
  - Dependencies resolved at build time from PyPI (supply chain risk)
