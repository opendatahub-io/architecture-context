RBAC Summary:
- No Kubernetes RBAC — caikit-nlp is a Python library, not a Kubernetes operator
- No ClusterRoles or RoleBindings defined
- Authentication enforced externally by platform (kube-rbac-proxy sidecar)

TLS Configuration:
- Server TLS (gRPC + HTTP):
  - RUNTIME_TLS_SERVER_KEY: TLS private key (file path, manual provisioning)
  - RUNTIME_TLS_SERVER_CERT: TLS certificate (file path, manual provisioning)
  - RUNTIME_TLS_CLIENT_CERT: Client certificate for mTLS (file path, optional)
  - No auto-rotation configured

- TGIS Backend TLS (outbound gRPC):
  - ca_cert_file: CA certificate for server verification (config file path)
  - client_cert_file: Client certificate for mutual TLS (config file path)
  - client_key_file: Client private key for mutual TLS (config file path)
  - Per-connection configuration via caikit config

Security Notes:
- Model downloads from HuggingFace Hub DISABLED by default (allow_downloads=false)
- PyTorch distributed training uses PLAINTEXT TCP on port 29550 (no encryption)
- No secrets stored in Kubernetes — all TLS materials are file-path references
- Authentication is NOT enforced at the library level
- Platform-level auth via kube-rbac-proxy is the primary security boundary
