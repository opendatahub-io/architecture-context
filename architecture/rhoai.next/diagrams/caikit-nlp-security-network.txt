RBAC Summary:
- No RBAC resources — caikit-nlp is a Python library, not a Kubernetes operator
- Authentication and authorization are handled by the caikit runtime and
  the platform's ingress layer (kube-rbac-proxy, Gateway API)
- Container runs as non-root user: UID 1001 (caikit)

Security Configuration:
- allow_downloads: false (default) — prevents arbitrary model downloads
- trust_remote_code: false (default) — prevents execution of untrusted code
- TGIS TLS: Optional, configurable via ca_cert_file, client_cert_file, client_key_file
- Base image: UBI 9 minimal (registry.access.redhat.com/ubi9/ubi-minimal:latest)

FIPS Compliance:
- Python library on UBI 9 — FIPS-validated OpenSSL inherited from base image
- No custom crypto operations — TLS delegated to grpcio (uses system OpenSSL)
- No Go binaries (N/A for check-payload static linking checks)
- OLM FIPS annotation: Not present (not an OLM-managed operator)

Secrets:
- No secrets defined by caikit-nlp itself
- TGIS TLS certificates configured via runtime_config.yaml (ca_cert_file, client_cert_file, client_key_file)
- HuggingFace API tokens provided via environment variable (optional)

- ⚠ No lock files present on main branch (rpms.lock.yaml, pip lock, artifacts.lock.yaml)
