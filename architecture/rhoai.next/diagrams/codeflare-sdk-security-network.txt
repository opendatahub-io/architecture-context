Authentication Model (3-Tier):
1. Kubernetes Cluster Auth:
   - Bearer Token (OpenShift OAuth) or kubeconfig
   - Validated by Kubernetes API Server
   - Required for all SDK operations

2. Ray Dashboard Auth (OpenShift):
   - Bearer Token forwarded from K8s auth context
   - Enforced by OpenShift Route + oauth-proxy
   - Required for job submission/status

3. Ray Client mTLS Auth (Optional):
   - CA cert read from {cluster}-ca-secret-* Kubernetes Secret
   - Client cert/key generated locally by SDK (cryptography library)
   - Written to local filesystem: tls-{cluster}-{ns}/
   - Environment variables: RAY_USE_TLS, RAY_TLS_SERVER_CERT, RAY_TLS_CA_CERT

Secrets:
- {cluster-name}-ca-secret-* (Opaque: ca.crt, ca.key) - CA for Ray mTLS, provisioned by KubeRay/RHOAI, no auto-rotate
- Local TLS files (tls-{cluster}-{ns}/) - Client cert/key, generated by SDK generate_cert module, no auto-rotate

Trusted CA Bundle:
- ConfigMap: odh-trusted-ca-bundle
- Mount path: /etc/pki/tls/custom-certs/ca-bundle.crt (RHOAI workbench default)
- Fallback: CF_SDK_CA_CERT_PATH env var → explicit parameter
- Purpose: Custom CA trust chain for Ray pods generated by SDK
