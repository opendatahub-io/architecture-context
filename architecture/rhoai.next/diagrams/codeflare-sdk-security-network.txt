NOTE: The CodeFlare SDK is a CLIENT LIBRARY, not a deployed service.
It runs inside user environments (Jupyter notebooks / workbenches) and
initiates OUTBOUND connections only. It does not expose any network services.

SDK → Ray Dashboard Connection:
          │
          │ Port: 8265/TCP (direct) or 443/TCP (via Route)
          │ Protocol: HTTP or HTTPS
          │ Encryption: TLS when via OpenShift Route
          │ Auth: Bearer Token (OpenShift) or None (non-OpenShift)
          │ Purpose: Job submission, status, logs, deletion
          ▼

SDK → Ray Client Connection:
          │
          │ Port: 10001/TCP
          │ Protocol: TCP (ray:// protocol)
          │ Encryption: Optional mTLS (client cert from generate_cert)
          │ Auth: Client TLS certificate (RSA 2048, SHA-256)
          │ Purpose: Direct Ray client interactive access
          ▼

  RBAC SUMMARY (Required user permissions)

  The SDK does NOT install RBAC resources. It operates with the authenticated
  user's credentials. The following API access is required:

  Role Name         API Group                    Resources       Verbs
  ─────────────────────────────────────────────────────────────────────────

  (user-provided)   networking.k8s.io            ingresses       list, get

================================================================================
  FIPS COMPLIANCE NOTES
================================================================================

  - Python library, not a container image (no check-payload validation)
  - Crypto library: cryptography==40.0.2 (OpenSSL backend, FIPS-compatible
    when running on FIPS-enabled host)
  - Certificate generation: RSA 2048-bit, SHA-256 (FIPS-approved algorithms)
  - Risk: verify_ssl=False when skip_tls=True disables TLS verification
    entirely (development only, never use in production)
  - CA trust: Configurable via CF_SDK_CA_CERT_PATH or workbench default path
    /etc/pki/tls/custom-certs/ca-bundle.crt
