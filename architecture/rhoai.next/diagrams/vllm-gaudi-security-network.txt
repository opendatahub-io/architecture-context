RBAC Summary:
- No cluster roles or role bindings defined by this component
- RBAC is managed entirely by the deploying platform (rhods-operator)
- kube-rbac-proxy sidecar enforces authorization at the pod level

Secrets:
- HF_TOKEN (Opaque, env var) - HuggingFace Hub auth for model downloads
  - Provisioned by: User / platform
  - Auto-rotate: No
- QUANT_CONFIG (Opaque, env var) - Path to quantization config file
  - Provisioned by: User / platform
  - Auto-rotate: No

FIPS Compliance:
- OpenSSL FIPS provider EXPLICITLY REMOVED (rpm -e --nodeps openssl-fips-provider-so)
- Container will NOT operate in FIPS mode even on FIPS-enabled clusters
- TLS handled by platform sidecar, not by vLLM process
- hashlib used only for build-time wheel checksums (install_nixl.py)

Build Security:
- Tekton PipelineRun: hermetic=false (non-hermetic builds)
- Dependencies: requirements.txt with >= ranges (not pinned)
- Source: Clones vLLM and vllm-gaudi from public GitHub at build time
- Version control: Build ARGs (VLLM_GAUDI_COMMIT, VLLM_PROJECT_COMMIT)
- Lock files: Absent on main branch (likely added on release branches)

Network Security Concerns:
1. HCCL and NIXL traffic between workers is UNENCRYPTED
2. Ray cluster communication is UNENCRYPTED
3. vLLM API server serves HTTP without TLS (relies on sidecar)
4. FIPS provider removed — incompatible with FIPS-certified deployments
5. Non-hermetic builds with unpinned dependencies — supply chain risk
