RBAC Summary:
  No RBAC resources defined — this is a container image, not an operator.
  RBAC is managed by KServe and the RHOAI operator.
  The container runs under the ServiceAccount assigned by the KServe InferenceService.

Authentication & Authorization:
  - /v1/* endpoints: Bearer Token (JWT) or None
    Enforcement: kube-rbac-proxy sidecar (RHOAI 3.x) or oauth-proxy (RHOAI 2.x)
    Policy: RHOAI platform auth policy
  - /health, /metrics: Unauthenticated (direct access, no sidecar interception)

Secrets:
  - Model storage credentials (Opaque) — S3/MinIO access for model weight download
    Provisioned by: User / RHOAI Dashboard
    Auto-Rotate: No
  - HuggingFace token (Opaque) — HuggingFace Hub auth for gated models
    Provisioned by: User
    Auto-Rotate: No

FIPS Compliance:
  ⚠️  WARNING: Container explicitly removes OpenSSL FIPS provider
  - Dockerfile line 63: rpm -e --nodeps openssl-fips-provider-so || true
  - Container runs with standard (non-FIPS) OpenSSL
  - Reason: Likely Habana SynapseAI runtime compatibility
  - Impact: Container does NOT operate in FIPS-compliant mode

Build Hermeticity:
  ⚠️  Non-hermetic builds (hermetic: false in Tekton PipelineRun)
  - No rpms.lock.yaml
  - No pip lock file
  - No artifacts.lock.yaml
  - Dockerfile clones from GitHub and installs from PyPI at build time

Plaintext Communication Risks:
  ⚠️  HCCL tensor parallelism: Plaintext, no auth (in-cluster only)
  ⚠️  NIXL KV cache transfer: Plaintext HTTP + RDMA, no auth (in-cluster only)
  ⚠️  Ray distributed orchestration: Plaintext, no auth (in-cluster only)
  These are mitigated by being cluster-internal traffic only.
