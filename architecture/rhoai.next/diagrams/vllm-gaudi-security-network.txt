RBAC Summary:
- No RBAC resources defined by this component
- RBAC is managed by:
  - KServe deployment (ServingRuntime CRDs)
  - RHOAI platform operator (rhods-operator)
  - kube-rbac-proxy sidecar (Bearer token validation against K8s API server)

Authentication & Authorization:
- /v1/* endpoints:
  - Auth: Bearer Token (delegated to kube-rbac-proxy sidecar)
  - Enforcement: kube-rbac-proxy on port 8443/TCP
  - Policy: RBAC on Kubernetes ServiceAccount or user token
- /health, /metrics:
  - Auth: None (when accessed pod-internally)
  - Exposure: Informational, pod-local access only

Secrets:
- HF_TOKEN (Opaque, env var)
  - Purpose: HuggingFace Hub authentication for model downloads
  - Provisioned By: User/admin
  - Auto-Rotate: No
  - Sensitivity: Medium (grants read access to gated model repositories)

Container Security:
- Base Image: UBI 9.6 (Red Hat Universal Base Image)
- Runtime User: non-root (UID 2000)
- Build: Multi-stage Dockerfile via Konflux pipeline
- No capabilities or privilege escalation required

Network Notes:
- vLLM itself does NOT implement authentication or TLS
- All external auth/TLS is handled by kube-rbac-proxy sidecar (injected by platform)
- Intra-cluster distributed communication (Ray, HCCL, NIXL) is plaintext
  - Acceptable: Pod-to-pod within trusted cluster network
  - Risk: No encryption on RDMA/TCP channels for model weights and KV caches
- Egress to HuggingFace Hub requires HF_TOKEN if accessing gated models
- Egress to S3 requires AWS IAM credentials (typically via IRSA or env vars)
