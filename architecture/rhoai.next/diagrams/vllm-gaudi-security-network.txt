RBAC Summary:
- No RBAC defined — this is a container image, not an operator.
- RBAC is managed by the deploying platform (KServe, RHOAI operator).

Secrets:
- No secrets defined in this repository.
- Model access credentials (HuggingFace tokens, S3 credentials) are injected
  by the platform at deployment time via environment variables or mounted secrets.

Authentication & Authorization:
- Endpoint: /v1/*
  - Methods: GET, POST
  - Auth Mechanism: None (by default)
  - Enforcement: Platform-level (kube-rbac-proxy sidecar in RHOAI 3.x)
  - The vLLM server itself has NO built-in authentication.

FIPS Compliance:
- CRITICAL: OpenSSL FIPS provider explicitly REMOVED from container image
  (`rpm -e --nodeps openssl-fips-provider-so` in Dockerfile.konflux.gaudi:63)
- Container CANNOT operate in FIPS mode on FIPS-enforcing OpenShift clusters
- Root cause: Habana Synapse drivers incompatible with FIPS OpenSSL provider

Build Hermeticity:
- Non-hermetic build: No lock files (rpms.lock.yaml, uv.lock, poetry.lock)
- vLLM and vllm-gaudi cloned from GitHub during Docker build
- Packages fetched from public PyPI and Habana private artifactory
- AIPCC Gaudi base image currently DISABLED (AIPCC-3471)

Container Security:
- Runs as non-root user (UID 2000) for OpenShift compatibility
- Base image: RHEL 9.6 UBI (registry.access.redhat.com/ubi9/ubi:9.6)
