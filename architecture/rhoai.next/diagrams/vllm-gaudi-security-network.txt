RBAC Summary:
- No RBAC resources defined by this component
- RBAC is managed by KServe deployment and platform operator (rhods-operator)
- kube-rbac-proxy sidecar enforces RBAC on incoming requests:
  - Validates Bearer tokens against Kubernetes API server
  - Enforces RBAC on ServiceAccount or user identity

Secrets:
- HF_TOKEN (Opaque, env var) - HuggingFace Hub authentication for model downloads
  - Provisioned by: User/admin
  - Auto-Rotate: No
  - Purpose: Bearer token for accessing gated models on HuggingFace Hub

Container Security:
- Base Image: UBI 9.6 (Red Hat Universal Base Image)
- Run-as User: non-root (UID 2000) for OpenShift compatibility
- Build System: Konflux (multi-stage Dockerfile)
- Version Pins: vLLM v0.16.0, vllm-gaudi v0.16.0, SynapseAI 1.23.0

Network Security Notes:
- Distributed inference channels (Ray, HCCL, NIXL) use PLAINTEXT
  - These are intra-cluster communications between pods in the same namespace
  - No encryption or authentication on distributed inference traffic
  - Risk: If cluster network is compromised, inference data could be intercepted
- External model downloads use TLS 1.2+ with proper authentication
- Client-facing traffic is TLS 1.3 terminated at kube-rbac-proxy sidecar
