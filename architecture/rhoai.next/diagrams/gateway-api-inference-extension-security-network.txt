Security Considerations:
- ⚠ InsecureSkipVerify=true by default for model server metrics scraping
  - Risk: MITM attacks on metrics data used for scheduling decisions
  - Mitigation: Configure proper TLS certs on model servers, set InsecureSkipVerify=false
- ⚠ Self-signed TLS certs with 10-year validity and no client auth
  - Risk: No mutual authentication between Gateway and EPP/BBR
  - Mitigation: Use cert-manager provided certs with shorter rotation
- ⚠ Latency prediction subsystem uses plaintext HTTP (no TLS)
  - Risk: Unencrypted model data in transit between EPP, training, and prediction servers
  - Mitigation: Deploy within same pod (sidecar pattern), network policy isolation
- ⚠ No FIPS compliance in upstream build (CGO_ENABLED=0, distroless base)
  - Risk: Would fail check-payload validation
  - Mitigation: Downstream RHOAI build with Konflux Dockerfile, UBI base, GOEXPERIMENT=strictfipsruntime

FIPS Compliance:
- Build: CGO_ENABLED=0, static linking, distroless/static base → NOT FIPS compliant
- Crypto: stdlib crypto/tls, crypto/rsa, crypto/x509 (standard Go crypto, not FIPS)
- Downstream requirement: Konflux Dockerfile with CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, UBI base
