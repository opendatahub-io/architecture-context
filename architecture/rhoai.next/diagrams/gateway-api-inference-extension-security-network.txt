Security Observations:

  ⚠ InsecureSkipVerify=true is DEFAULT for model server metrics scraping
    - Risk: No certificate verification when scraping model server /metrics via HTTPS
    - Mitigation: Configurable per plugin; enforce verification in production

  ⚠ ext-proc gRPC connections are unencrypted by default
    - Risk: Request/response data flows in plaintext between gateway and EPP/BBR
    - Mitigation: Enable --secure-serving flag; or rely on Istio mTLS (STRICT)

  ⚠ Latency predictor subsystem has no authentication
    - Risk: Training/prediction HTTP servers accept unauthenticated requests
    - Mitigation: Cluster-internal only; network policies recommended

  ⚠ Python dependencies are unpinned (latencypredictor/requirements.txt)
    - Risk: Supply chain attack vector; non-reproducible builds
    - Mitigation: Downstream RHOAI builds likely pin deps on release branches

Service Mesh Configuration (when Istio is the gateway provider):
  - DestinationRule: SIMPLE TLS to EPP service
  - EnvoyFilter: Inserts BBR ext-proc filter in gateway filter chain
  - mTLS: Relies on Istio PeerAuthentication if configured (not created by this chart)
