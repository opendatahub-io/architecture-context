Secrets:
- Data Connection Secret (Opaque) - S3 credentials (AWS_ACCESS_KEY_ID,
  AWS_SECRET_ACCESS_KEY, AWS_S3_ENDPOINT) - Provisioned by user via RHOAI
  Dashboard or manually - No auto-rotation
- HuggingFace Access Token (Opaque, env injection) - Access token for private
  HuggingFace models/datasets - Provisioned by user - No auto-rotation

Security Considerations:
1. In-pod metrics server (28080/TCP) binds 0.0.0.0 with NO authentication.
   Accessible to any pod in the namespace without NetworkPolicy enforcement.
2. verify_cloud_storage_ssl=False option disables SSL verification for
   S3-compatible storage. Logged as WARNING. Enables use with self-signed certs.
3. RHAI code injection pattern: Python source extracted via inspect.getsource()
   and injected as heredoc into TrainJob container command. No SDK dependencies
   required in training image — fully self-contained injected code.
4. SDK is a client library — security boundary is the user's kubeconfig/SA
   permissions. No additional server-side enforcement.

Authentication Summary:
- Kubernetes API: Bearer Token / kubeconfig / In-cluster SA → RBAC enforcement
- In-pod Metrics Server: None (polled by RHOAI controller within cluster)
- Model Registry: Bearer Token (optional) / Custom CA support
- S3 Storage: AWS IAM credentials from Data Connection secret
- HuggingFace Hub: Bearer Token (optional, for private repos)
