Authentication Modes:
- auto: ServiceAccount token (single-user, default)
- kubeconfig: Local kubeconfig file (development)
- token: Bearer token passthrough
- OIDC: JWT validation + Kubernetes Impersonation (multi-user production)

Secrets:
- User-created data connections (Opaque) - S3 credentials (AWS_ACCESS_KEY_ID,
  AWS_SECRET_ACCESS_KEY, etc.) - Created via MCP tool, no auto-rotation
- Workflow HMAC secret (in-memory) - Auto-generated or RHOAI_MCP_WORKFLOW_HMAC_SECRET
  env var - Used for stateless workflow token signing

FIPS Considerations:
- UBI 9 base provides FIPS-validated OpenSSL
- PyJWT[crypto] uses 'cryptography' library (FIPS-capable on UBI 9)
- hashlib.md5 used for non-cryptographic name generation (may trigger FIPS scanners)
- skip_tls_verify option available for Model Registry (disabled by default)
