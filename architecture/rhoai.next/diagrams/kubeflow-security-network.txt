Authentication & Authorization:

  Notebook (auth enabled):
  - Methods: All
  - Auth: Bearer Token → TokenReview + SubjectAccessReview
  - Enforcement: kube-rbac-proxy sidecar
  - Policy: User must have "get" verb on notebooks.kubeflow.org in notebook namespace

  Notebook (auth disabled):
  - Methods: All
  - Auth: None (relies on NetworkPolicy)
  - Enforcement: NetworkPolicy
  - Policy: Only controller namespace can reach port 8888

  Webhook endpoint:
  - Methods: POST
  - Auth: Kubernetes API server mutual TLS
  - Enforcement: API server
  - Policy: Admission webhook called by API server only

  - {notebook}-kube-rbac-proxy-tls (kubernetes.io/tls)
    Purpose: TLS certificate for kube-rbac-proxy sidecar service
    Provisioned By: OpenShift service-ca (via annotation)
    Auto-Rotate: Yes

  - ds-pipeline-config (Opaque)
    Purpose: Elyra runtime config with DSPA connection info
    Provisioned By: odh-notebook-controller

    Purpose: TLS certificate for webhook HTTPS endpoint
    Provisioned By: OpenShift service-ca (via annotation)

TLS Configuration:

  - OpenShift TLS security profile integration via config.openshift.io/v1 APIServer
  - Fallback: Mozilla Intermediate (TLS 1.2 minimum, ECDHE+AES-GCM, CHACHA20-POLY1305)
  - FIPS: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamic linking
  - SecurityProfileWatcher triggers graceful shutdown on TLS profile changes
  - Cipher suites: FIPS-approved only (ECDHE+AES-GCM, CHACHA20-POLY1305)

    Resources: Notebook

    Restart avoidance: Blocks pod-template changes on running notebooks
