NetworkPolicy Summary:
──────────────────────
  Per notebook, two NetworkPolicies:
    1. Port 8888/TCP: Ingress restricted to controller namespace only
       (only gateway proxy / culler can reach unprotected notebook port)
    2. Port 8443/TCP: All ingress allowed
       (kube-rbac-proxy enforces auth before forwarding)

Authentication Flow (Auth Mode):
────────────────────────────────
  Browser → Gateway (TLS 1.3)
    → kube-rbac-proxy:8443 (TLS, service cert)
      → TokenReview (K8s API, TLS 1.2+)
      → SubjectAccessReview (K8s API, TLS 1.2+)
        check: verb=get, resource=notebooks, apiGroup=kubeflow.org,
               name={notebook-name}, namespace={ns}
      → Notebook:8888 (HTTP, localhost, X-Auth-Request-User header)
