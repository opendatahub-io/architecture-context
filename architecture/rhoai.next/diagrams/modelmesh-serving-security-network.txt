==============================================================================
  NETWORK POLICIES
==============================================================================

NetworkPolicy: modelmesh-controller
  - Namespace: controller namespace
  - Ingress: Allow port 8080/TCP (metrics) from Prometheus
  - Ingress: Allow port 9443/TCP (webhook) from API server

NetworkPolicy: modelmesh-runtimes
  - Namespace: model namespace
  - Ingress: Allow port 8033/TCP (gRPC inference)
  - Ingress: Allow port 8443/TCP (REST HTTPS via oauth-proxy)
  - Ingress: Allow port 8008/TCP (REST HTTP, internal only)
  - Ingress: Allow port 2112/TCP (metrics) from Prometheus

NetworkPolicy: modelmesh-webhook
  - Namespace: controller namespace
  - Ingress: Allow port 9443/TCP from Kubernetes API server

==============================================================================
  SECURITY CONCERNS
==============================================================================

1. InsecureSkipVerify: true in ServiceMonitor TLS configuration
   (service_controller.go:346). Prometheus metrics scraping does NOT validate
   the server certificate. TODO comment indicates this needs to be updated
   to use the CA certificate.

2. REST inference on port 8008/TCP has NO authentication (internal use only).
   Must be protected by NetworkPolicy to prevent unauthorized access.

3. etcd credentials (username/password) stored in Secrets without automatic
   rotation. Per-namespace secrets are auto-created by the controller.
