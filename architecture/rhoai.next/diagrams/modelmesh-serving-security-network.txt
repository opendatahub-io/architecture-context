    - apps: deployments (CRUD+watch)
    - serving.kserve.io: inferenceservices, predictors, servingruntimes, clusterservingruntimes + finalizers + status (full CRUD+watch)
    - autoscaling: horizontalpodautoscalers (full CRUD+watch)
    - monitoring.coreos.com: servicemonitors (CRUD+watch)
    - coordination.k8s.io: leases (create, get, list, update)

- ClusterRole: modelmesh-controller-restricted-scc-role
  - ServiceAccounts: modelmesh, modelmesh-serving-sa
  - security.openshift.io: securitycontextconstraints (resourceName: restricted) - use

OpenShift SCC:
- restricted SCC bound to modelmesh and modelmesh-serving-sa service accounts

Authentication & Authorization:
- gRPC inference (8033): Optional mTLS client certificates (MM_TLS_CLIENT_AUTH: REQUIRE/OPTIONAL)
- REST inference (8443): OpenShift OAuth token → SAR check (get services in namespace)
- Webhook (9443): Kubernetes API server TLS, cert-manager CA bundle
- Debug pprof (9999): HTTP Basic Auth (etcd credentials)

⚠️ Security Asymmetry: REST and gRPC inference endpoints have DIFFERENT security postures:
  - REST (8443): OAuth-enforced access control via oauth-proxy SAR
  - gRPC (8033): Relies on network policy + optional mTLS (no OAuth enforcement)

Secrets:
- model-serving-etcd (Opaque) - etcd endpoints, credentials, TLS certs, root prefix [Manual, No auto-rotate]
- storage-config (Opaque) - S3/GCS/Azure/PVC storage credentials [User-managed, No auto-rotate]
- TLS secret (kubernetes.io/tls) - ModelMesh gRPC TLS cert/key [cert-manager/Manual, Depends]
- model-serving-proxy-tls (kubernetes.io/tls) - OAuth proxy HTTPS cert [OpenShift serving-cert-signer, Auto-rotate]

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamic linking
- Runtime: OpenSSL via UBI9 ubi-minimal base image
- Risks: ServiceMonitor insecureSkipVerify: true (monitoring only, not data path)
         oauth-proxy uses hardcoded cookie-secret placeholder

NetworkPolicies:
- networkpolicy-controller: Controls ingress to controller namespace
- networkpolicy-runtimes: Controls ingress to runtime pods (removed in ODH overlay)
- networkpolicy-webhook: Controls ingress to webhook service
- networkpolicy-etcd: Added in ODH overlay for etcd access
