- runtime NetworkPolicy: Restricts runtime pod network access (removed in ODH overlay)

- etcd NetworkPolicy: Controls etcd access (ODH overlay)

Secrets:
- model-serving-etcd (Opaque) - etcd connection config, Manual provisioning, No auto-rotate
- storage-config (Opaque) - Storage backend credentials (S3/GCS/Azure/PVC), User provisioned, No auto-rotate
- TLS secret (kubernetes.io/tls) - ModelMesh gRPC TLS cert/key, cert-manager/Manual, Depends on provisioner
- model-serving-proxy-tls (kubernetes.io/tls) - oauth-proxy HTTPS cert, OpenShift serving-cert-signer, Yes auto-rotate

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamic linking
- Runtime: stdlib crypto/tls with FIPS via OpenSSL (strictfipsruntime)
- Risks: ServiceMonitor insecureSkipVerify:true (monitoring only), oauth-proxy hardcoded cookie-secret

Security Asymmetry Note:
- REST path (8443): OAuth-enforced access control via oauth-proxy SAR check
- gRPC path (8033): Network policy + optional mTLS only (no OAuth)
- Different security postures between REST and gRPC endpoints
