RBAC Summary:
  Not applicable — autogluon-server is a model server process, not an operator.
  It does not interact with the Kubernetes API directly.
  RBAC for the pod is configured by the KServe controller via the
  InferenceService's service account.

Secrets:
  - storage-config (Opaque, optional) — S3/GCS/Azure credentials for model
    download. Provisioned by user / platform admin. No auto-rotation.

Authentication & Authorization:
  - All inference endpoints: Auth is delegated to the KServe ingress layer
    (kube-rbac-proxy / Istio sidecar / OpenShift Route).
  - The autogluon-server itself performs NO authentication or authorization.
  - TLS is NOT terminated at the server — it runs plaintext HTTP/gRPC.
  - TLS termination occurs at the ingress layer.

FIPS Compliance:
  - Base image: UBI9 (registry.redhat.io/rhai/base-image-cpu-rhel9)
  - OpenSSL: System OpenSSL from UBI (FIPS-validated)
  - cryptography library: >=46.0.5 (uses system OpenSSL, FIPS-capable)
  - No direct use of weak ciphers (hashlib.md5, pycryptodome) detected
  - Server runs plaintext; FIPS TLS enforcement is at ingress layer

Build Hermeticity:
  - Python deps: pinned via autogluon-all-requirements.txt
    (Red Hat internal PyPI index, RHOAI-specific builds)
  - OS packages: No rpms.lock.yaml (not fully hermetic at OS level)
  - Container base: registry.redhat.io/rhai/base-image-cpu-rhel9
