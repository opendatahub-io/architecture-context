RBAC Summary:
- No Kubernetes RBAC configured by this component
- This is a model server process, not an operator
- RBAC for the pod is configured by the KServe controller via
  the InferenceService's service account

Secrets:
- storage-config (Opaque, optional) — S3/GCS/Azure credentials for model download
  - Provisioned by: User / platform admin
  - Auto-Rotate: No
  - Contains: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, or equivalent

Authentication & Authorization:
- Server-level auth: NONE
- Auth enforcement: Delegated entirely to KServe ingress layer
  - kube-rbac-proxy (for OpenShift)
  - Istio sidecar (for service mesh)
  - OpenShift Route (for external access)

FIPS Compliance:
- Server binds plaintext HTTP/gRPC (no TLS at application level)
- TLS termination handled at ingress layer
- Crypto library: cryptography>=46.0.5 (Python OpenSSL bindings)
  - Links against system OpenSSL on UBI9 (FIPS-validated)
- No direct use of weak ciphers (md5, etc.) detected
- Base image: UBI9 with FIPS-validated OpenSSL

Network Security Notes:
- Server listens on plaintext HTTP (8080) and gRPC (8081)
- No mTLS at application level — relies on service mesh sidecar if configured
- All egress to model storage uses TLS 1.2+
- Pod runs as non-root user (OpenShift pod security context)
