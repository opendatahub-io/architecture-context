RBAC Summary:
- No ClusterRoles or RoleBindings — this is a model server process, not an operator
- Pod RBAC is configured by the KServe controller via the InferenceService's ServiceAccount
- The server does not interact with the Kubernetes API directly

Secrets:
- storage-config (Opaque, optional) — S3/GCS/Azure credentials for model download
  - Provisioned by: User / platform admin
  - Auto-Rotate: No
  - Contains: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY (or GCS/Azure equivalents)

Authentication & Authorization:
- All inference endpoints: No authentication at the server level
- Auth is delegated entirely to the KServe ingress layer:
  - kube-rbac-proxy (Kubernetes RBAC)
  - Istio RequestAuthentication + AuthorizationPolicy
  - OpenShift Route + OAuth proxy
- The enforcement point is OUTSIDE this component

FIPS Compliance:
- Base image: UBI9 (registry.redhat.io/rhai/base-image-cpu-rhel9)
- OpenSSL: System OpenSSL from UBI (FIPS-validated)
- cryptography lib: >=46.0.5, uses system OpenSSL (FIPS-capable)
- No direct TLS — server runs plaintext HTTP/gRPC
- TLS termination handled by ingress layer
- No non-FIPS crypto detected in application code

Container Security:
- Runs as non-root user (OpenShift restricted SCC)
- Writable CWD workaround: runtime_paths.py changes CWD to /tmp or $HOME
  (AutoGluon SeasonalNaive creates ./AutogluonModels — fails as non-root at /)
- MPLCONFIGDIR env var set to prevent matplotlib cache issues
