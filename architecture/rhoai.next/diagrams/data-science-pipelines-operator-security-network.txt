NetworkPolicy Summary:
- ds-pipeline-{name} policy:
  - Ingress from: monitoring namespaces, pipeline step pods
    (pipelines.kubeflow.org/v2_component=true), workbench pods
    (opendatahub.io/workbenches=true), same-namespace DSP components
  - Ports allowed: 8888/TCP, 8887/TCP, 8443/TCP

- mariadb-{name} policy:
  - Ingress from: API server pod, MLMD gRPC pod (same namespace only)
  - Ports allowed: 3306/TCP

- metadata-grpc-{name} policy:
  - Ingress from: pipeline step pods, MLMD Envoy, API server
  - Ports allowed: 8080/TCP

- mlmd-envoy dashboard access policy:
  - Ingress from: monitoring namespaces
  - Ports allowed: 9090/TCP

TLS Certificate Management:
- All TLS certs provisioned via OpenShift service-ca (annotation-based)
- Annotation: service.beta.openshift.io/serving-cert-secret-name
- Auto-rotation managed by service-ca operator
- Pod-to-Pod TLS enabled by default (PodToPodTLS=true)
- CA trust chain: openshift-service-ca.crt ConfigMap + odh-trusted-ca-bundle
