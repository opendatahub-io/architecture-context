NetworkPolicy Topology:
- MariaDB: Allows ONLY from DSPO operator pod, API Server pod, MLMD pod
- API Server (8888/8887): Allows DSP components, pipeline steps
                          (pipelines.kubeflow.org/v2_component: true),
                          monitoring namespaces, workbenches
                          (opendatahub.io/workbenches: true)
- API Server (8443): Open to all sources (route traffic)
- MLMD gRPC (8080): Allows pipeline step pods, DSP component pods

Secrets:
- ds-pipeline-db-{name} (Opaque) - MariaDB password, 12-char generated, No auto-rotate
- ds-pipeline-s3-{name} (Opaque) - MinIO access key 16-char / secret key 24-char, No auto-rotate
- ds-pipelines-proxy-tls-{name} (kubernetes.io/tls) - kube-rbac-proxy cert, OpenShift Service CA, Auto-rotated
- ds-pipeline-metadata-grpc-tls-certs-{name} (kubernetes.io/tls) - MLMD gRPC TLS, Service CA or user-provided
- mariadb-{name}-secret (Opaque) - MariaDB root password, generated or user-provided
- minio-{name}-secret (Opaque) - MinIO access/secret keys, generated or user-provided

CA Bundle Aggregation:
- Sources: odh-trusted-ca-bundle (platform) + user custom CA + OpenShift service-ca
- Output: dsp-trusted-ca-bundle-{name} ConfigMap mounted into all components
- Auto-refresh: Watches platform CA bundle, triggers reconciliation on change
