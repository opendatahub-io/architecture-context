Authentication Summary:
- Primary: OAuth2 JWT validation via JWKS endpoint
  - JWKS keys cached with configurable recheck period
  - Supports RS256/RS384/RS512 signing algorithms
- Secondary: RFC 7662 token introspection (for opaque tokens)
  - Client authenticates to introspection endpoint via HTTP Basic Auth
- Tertiary: Custom auth endpoint (delegated token validation)
- Unauthenticated: /v1/health, /v1/version, /docs, /redoc, /openapi.json

Authorization Summary:
- Cedar-inspired ABAC (Attribute-Based Access Control)
  - Policies: permit/forbid rules per (principal, action, resource)
  - Actions: CREATE, READ, UPDATE, DELETE
  - Resources: Qualified as type::identifier (e.g., model::llama-3)
  - Conditions: user is owner, user with <value> in <attribute>, user in owners <attribute>
- Quota enforcement: Per-client rate limiting
  - Anonymous: 100 requests/day (configurable)
  - Authenticated: 1000 requests/day (configurable)

Secrets:
- TLS certificate (tls_certfile) — HTTPS server certificate — Manual/cert-manager — No auto-rotate
- TLS key (tls_keyfile) — HTTPS server private key — Manual/cert-manager — No auto-rotate
- TLS CA (tls_cafile) — CA certificate for mTLS client verification — Manual/cert-manager — No auto-rotate
- Provider API keys — Environment variables — Authentication to remote inference/tool providers — Manual — No auto-rotate
- OAuth2 introspection client_secret — Config field — Token introspection endpoint authentication — Manual — No auto-rotate

TLS Configuration:
- Server TLS: Optional, configured via tls_certfile + tls_keyfile
- mTLS: Optional, enabled by additionally setting tls_cafile
- Minimum TLS version: TLS 1.2 (Uvicorn/ssl module default)
- All external HTTPS egress: TLS 1.2+
- Ollama connection: plaintext HTTP (intended for localhost/development only)

Note: No Kubernetes RBAC — this is a Python application service.
Access control is entirely at the application level via Cedar ABAC policies.
Kubernetes-level security (NetworkPolicy, PodSecurity, ServiceAccount)
is managed by the deploying operator (e.g., rhods-operator).
