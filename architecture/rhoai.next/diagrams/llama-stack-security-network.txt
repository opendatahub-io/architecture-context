APPLICATION-LEVEL ACCESS CONTROL (Cedar-inspired ABAC):
- No Kubernetes RBAC — Llama Stack uses application-level policies
- Policy Format: permit/forbid rules with principal, action, resource matching
- Actions: create, read, update, delete
- Resources: type-qualified (e.g., model::my-model, shield::llama-guard)
- Conditions: when/unless clauses, attribute matching, ownership checks
- Evaluation: First match wins; default deny if no rules match
- Scoping: Per-resource, per-user, per-group/team/project/namespace

Authentication Summary:
- OAuth2 JWT: Validates via JWKS URI or RFC 7662 token introspection
  - Claims extracted: sub, groups, team, project, tenant, namespace
  - Claims mapped to: roles, teams, projects, namespaces (configurable)
  - Supports: configurable audience, issuer, TLS verification
- Custom Auth: Delegates to external HTTP endpoint
  - Sends: token + request context
  - Receives: principal + attributes
- Unauthenticated: /health, /version, /docs, /redoc, /openapi.json

Rate Limiting:
- Authenticated clients: 1000 requests/day (default, configurable)
- Anonymous clients: 100 requests/day (default, configurable)
- Identification: Bearer token hash (auth) or IP address (anonymous)
- Backend: SQLite KV store for counter persistence

Secrets:
- TLS certificate (PEM)          - HTTPS server encryption, Manual/cert-manager, No auto-rotate
- TLS private key (PEM)          - HTTPS server encryption, Manual/cert-manager, No auto-rotate
- TLS CA certificate (PEM)       - Mutual TLS client verification, Manual/cert-manager, No auto-rotate
- Provider API keys (env var)    - Auth to remote providers (OpenAI, Anthropic, etc.), Manual, No auto-rotate
- OAuth2 client secret (YAML)   - Token introspection authentication, Manual, No auto-rotate
- Database passwords (env/YAML)  - PostgreSQL, Redis, MongoDB auth, Manual, No auto-rotate

FIPS Compliance Notes:
- Pure Python service; relies on container base image (UBI) for FIPS-validated OpenSSL
- RISK: python-jose uses pure-Python RSA/ECDSA (not system OpenSSL) for JWT validation
- hashlib.md5(usedforsecurity=False) usage is FIPS-compliant per PEP 456
- Recommendation: Migrate from python-jose to PyJWT+cryptography for FIPS JWT validation
- No Konflux Dockerfile on upstream main branch (expected on downstream RHOAI branches)
