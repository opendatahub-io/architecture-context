AUTHENTICATION & AUTHORIZATION:
- AuthenticationMiddleware (ASGI): Validates tokens before routing
  - Strategy 1: JWT validation via JWKS (periodic key refresh)
  - Strategy 2: Token introspection (RFC 7662) with client credentials
  - Strategy 3: Custom auth endpoint (sends token in body)
- ABAC (Cedar-like): Attribute-based access control
  - Rules: permit/forbid with when/unless conditions
  - Attributes: Extracted from JWT claims (user, roles, groups)
  - Resource-level: Per-resource access policies
- Rate Limiting: QuotaMiddleware
  - Anonymous: 100 req/day
  - Authenticated: 1000 req/day (configurable)

SECRETS:
- TLS key/cert pair (file-based: tls_keyfile, tls_certfile)
  - Purpose: HTTPS for server
  - Provisioned: Manual or cert-manager
  - Rotation: Manual (requires restart)
- TLS CA certificate (file-based: tls_cafile)
  - Purpose: mTLS client cert validation
  - Provisioned: Manual
- Provider API keys (environment variables / config)
  - Purpose: Auth to remote inference providers (OpenAI, Anthropic, etc.)
  - Provisioned: Manual
- OAuth2 introspection client secret (config)
  - Purpose: Token introspection with OAuth2 provider
  - Provisioned: Manual
- PostgreSQL credentials (config: host, user, password)
  - Purpose: Database auth for pgvector, KV store, SQL store
  - Provisioned: Manual
- Redis password (config)
  - Purpose: Redis authentication
  - Provisioned: Manual
- HuggingFace token (environment variable)
  - Purpose: Model downloads from HuggingFace Hub
  - Provisioned: Manual

FIPS COMPLIANCE NOTES:
- Python ssl module: Inherits OpenSSL FIPS mode from OS (RHEL/UBI base)
- Crypto libs: python-jose (JWT), ecdsa, rsa — use Python stdlib/OpenSSL backend
- hashlib.md5: Used with usedforsecurity=False (PEP 452 compliant)
  - cli/verify_download.py (download verification)
  - providers/inline/vector_io/sqlite_vec/sqlite_vec.py (deterministic UUIDs)
  - providers/remote/vector_io/milvus/milvus.py (content hashes)
- No Konflux Dockerfile in repo — FIPS posture depends on downstream build
