Application-Level Access Control (ABAC — No Kubernetes RBAC):
──────────────────────────────────────────────────────────────
- Policy Engine: Cedar-inspired attribute-based access control
- Actions: CREATE, READ, UPDATE, DELETE
- Resource Types: model, shield, vector_db, dataset, toolgroup, scoring_fn, benchmark
- Resource Qualification: type::identifier (e.g., model::llama-3)
- Conditions:
  - "user is owner" — resource creator has full access
  - "user with <value> in <attribute>" — attribute-based matching
  - "user in owners <attribute>" — ownership group membership
- Default: Permit (when no access control policies configured)

Authentication Configuration:
──────────────────────────────
- Strategy 1 — OAuth2 JWT (JWKS):
  - Provider: Configurable JWKS URI
  - Token format: JWT Bearer
  - Validation: Signature verification via cached JWKS
  - Claims extraction: Subject, roles, custom attributes
  - Key caching: Yes (configurable recheck period)

- Strategy 2 — Token Introspection (RFC 7662):
  - Provider: Configurable introspection endpoint
  - Auth to IdP: HTTP Basic (client_id / client_secret)
  - Response: Active/inactive + user attributes

- Strategy 3 — Custom Auth Provider:
  - Delegates to arbitrary HTTPS endpoint
  - Passes Bearer token for external validation
  - Extracts user attributes from response

Quota / Rate Limiting:
──────────────────────
- Anonymous clients: 100 requests/day (configurable)
- Authenticated clients: 1000 requests/day (configurable)
- Tracking: Persistent KV store (SQLite by default)
- Scope: Per-client (identified by user principal)

Secrets:
────────
- TLS certificate (tls_certfile)         — File path — HTTPS server certificate — Manual/cert-manager — No auto-rotate
- TLS private key (tls_keyfile)          — File path — HTTPS server private key — Manual/cert-manager — No auto-rotate
- TLS CA certificate (tls_cafile)        — File path — CA cert for mTLS client verification — Manual/cert-manager — No auto-rotate
- Provider API keys                      — Environment variable — Auth to remote inference providers — Manual — No auto-rotate
- OAuth2 introspection client_secret     — Config field — Token introspection endpoint auth — Manual — No auto-rotate
