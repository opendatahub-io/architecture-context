ABAC Access Control Summary:
- Policy Engine: Cedar-like rule evaluation
- Attributes: Extracted from JWT claims (roles, namespaces, teams)
- Rules: permit/forbid with when/unless conditions
  - Example: "user in owners roles" → permit
  - Example: "user with admin in roles" → permit
- Scope: Per-resource (models, datasets, vector-dbs, shields, benchmarks)
- Default: Configurable (typically deny-by-default)

Secrets:
- TLS key/cert pair (file-based: tls_keyfile, tls_certfile) - HTTPS server, manual provisioning, no auto-rotate
- TLS CA certificate (file-based: tls_cafile) - mTLS client cert validation, manual provisioning
- Provider API keys (env vars) - Remote inference providers (OpenAI, Anthropic, etc.), manual
- OAuth2 introspection client secret (config) - Token introspection, manual
- PostgreSQL credentials (config: host, user, password) - Database authentication, manual
- Redis password (config) - Redis authentication, manual
- HuggingFace token (env var) - Model downloads, manual

FIPS Compliance Notes:
- No Konflux Dockerfile in repository (build process delegated to downstream)
- Python ssl module inherits OS OpenSSL FIPS mode on FIPS-enabled RHEL/UBI
- hashlib.md5 usage in 3 locations flagged with usedforsecurity=False (PEP 452 compliant)
- python-jose 3.4.0 for JWT validation (uses Python stdlib crypto backend)
- TLS configured via Python ssl.create_default_context()

Middleware Execution Order:
1. ClientVersionMiddleware - Checks client SDK version compatibility
2. AuthenticationMiddleware - JWT/introspection/custom token validation
3. QuotaMiddleware - Rate limiting (100/day anon, 1000/day auth)
4. TracingMiddleware - Adds x-trace-id response header
5. Route Handlers - FastAPI dynamic routes from Protocol classes
