APPLICATION-LEVEL ACCESS CONTROL (No Kubernetes RBAC):

  Authentication Providers:
  - OAuth2 JWT: Validates via JWKS URI or RFC 7662 introspection
    - Claim extraction: sub, groups, team, project, tenant, namespace
    - Configurable audience, issuer, TLS verification
  - Custom: Delegates to external HTTP endpoint
    - Sends token + request context; receives principal + attributes

  Access Control Model (Cedar-inspired ABAC):
  - permit/forbid rules evaluated in order (first match wins)
  - Scoped by: principal, actions (CRUD), resource (type-qualified)
  - Conditions: when/unless clauses
    - Attribute matching: "user with X in roles"
    - Ownership checks: "user is owner"
    - Group membership: "user in owners teams"
  - Default: DENY (if no rules match)

  Rate Limiting:
  - Authenticated clients: 1000 req/day (default)
  - Anonymous clients: 100 req/day (default, identified by IP)
  - Counter storage: SQLite KV backend

Secrets:
  - TLS certificate (PEM file) - HTTPS server encryption
    Provisioned by: Manual / cert-manager | Auto-rotate: No
  - TLS private key (PEM file) - HTTPS server encryption
    Provisioned by: Manual / cert-manager | Auto-rotate: No
  - TLS CA certificate (PEM file) - Mutual TLS client verification
    Provisioned by: Manual / cert-manager | Auto-rotate: No
  - Provider API keys (env var) - Auth to remote providers (OpenAI, Anthropic, etc.)
    Provisioned by: Manual | Auto-rotate: No
  - OAuth2 client secret (YAML config) - Token introspection authentication
    Provisioned by: Manual | Auto-rotate: No
  - Database passwords (env var / config) - PostgreSQL, Redis, MongoDB auth
    Provisioned by: Manual | Auto-rotate: No

FIPS Compliance Notes:
  - Pure Python service; FIPS depends on UBI base image OpenSSL
  - python-jose uses pure-Python RSA/ECDSA (bypasses FIPS OpenSSL)
  - hashlib.md5 uses usedforsecurity=False (FIPS-compliant per PEP 456)
  - Risk: JWT validation via python-jose may not use FIPS-validated crypto
  - Recommendation: Migrate to PyJWT with cryptography backend
