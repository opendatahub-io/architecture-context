Application-Level Access Control (No Kubernetes RBAC):

  Llama Stack implements application-level access control, not Kubernetes RBAC.
  No ClusterRoles, Roles, or RoleBindings are defined.

  Access Control Model:
  - Engine: Cedar-inspired policy language (permit/forbid rules)
  - Actions: create, read, update, delete
  - Resource scoping: type-qualified (e.g., model::my-model, shield::llama-guard)
  - Principal attributes: roles, teams, projects, namespaces
  - Conditions: when/unless clauses, ownership checks, group membership
  - Evaluation: First match wins, default deny

  Authentication Providers:
  - OAuth2 JWT: JWKS URI or RFC 7662 Token Introspection
    - Claim mapping: sub → user, groups → roles, team/project/namespace → attributes
    - Configurable: audience, issuer, TLS verification
  - Custom: External HTTP endpoint delegation
    - Sends: token + request context
    - Receives: principal + attributes

  Rate Limiting:
  - Authenticated clients: 1000 requests/day (configurable)
  - Anonymous clients: 100 requests/day (configurable)
  - Identity: Token hash (auth) / IP address (anon)
  - Backend: SQLite KV store

Secrets:

  | Secret                  | Type           | Purpose                              | Rotation |
  |-------------------------|----------------|--------------------------------------|----------|
  | TLS certificate         | File (PEM)     | HTTPS server encryption              | Manual   |
  | TLS private key         | File (PEM)     | HTTPS server encryption              | Manual   |
  | TLS CA certificate      | File (PEM)     | Mutual TLS client verification       | Manual   |
  | Provider API keys       | Env var        | Auth to remote providers (OpenAI...) | Manual   |
  | OAuth2 client secret    | Config (YAML)  | Token introspection auth             | Manual   |
  | Database passwords      | Env var/Config | PostgreSQL, Redis, MongoDB auth      | Manual   |

FIPS Compliance Notes:

  - Pure Python service; FIPS relies on UBI base image OpenSSL
  - RISK: python-jose uses pure-Python RSA/ECDSA (bypasses FIPS OpenSSL)
  - Mitigation: Migrate to PyJWT with cryptography backend
  - hashlib.md5(usedforsecurity=False) usage is FIPS-compliant per PEP 456
  - No Konflux Dockerfile on upstream main branch (expected on release branches)
