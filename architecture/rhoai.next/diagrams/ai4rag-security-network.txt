RBAC Summary:
  No Kubernetes RBAC — ai4rag is a Python library, not a Kubernetes component.

Secrets:
  No Kubernetes secrets.
  API keys are provided by the user via:
    - Environment variables
    - Direct constructor parameters (e.g., LlamaStackClient(base_url=..., api_key=...))
    - OpenAI(api_key=..., base_url=...)

Cryptographic Usage:
  - hashlib SHA256: Document deduplication only (not security-sensitive)
  - TLS: Delegated entirely to httpx and openai client libraries
  - No custom TLS configuration, no certificate handling
  - FIPS compliance depends on runtime environment's OpenSSL

Authentication & Authorization:
  - Llama Stack API: API Key (Bearer Token), enforced by Llama Stack server
  - OpenAI API: API Key (Bearer Token), enforced by OpenAI API server
  - No mutual TLS, no service mesh identity
  - No authorization policies — library operates with caller's credentials
