RBAC Summary:
  - No Kubernetes RBAC — ai4rag is a library, not a Kubernetes component.
  - No ClusterRoles, Roles, or RoleBindings.
  - No ServiceAccounts.

Secrets:
  - No Kubernetes Secrets.
  - API keys are provided by the user via:
    - Environment variables (e.g., LLAMA_STACK_API_KEY, OPENAI_API_KEY)
    - Direct constructor parameters (e.g., LlamaStackClient(api_key=...))
  - The library does NOT store, cache, or persist credentials.

Service Mesh:
  - Not applicable — ai4rag is a library, not a deployed service.
  - No PeerAuthentication, AuthorizationPolicy, or sidecar injection.

FIPS Compliance:
  - No container image produced (pure Python library).
  - No custom TLS/crypto — delegates to httpx and openai SDK libraries.
  - SHA256 used only for document deduplication (not security-sensitive).
  - FIPS compliance depends on the runtime environment's OpenSSL configuration.

Build Hermeticity:
  - No lock files on main branch (rpms.lock.yaml, uv.lock, poetry.lock).
  - pyproject.toml uses version pinning constraints (e.g., langchain~=1.1.3).
  - Consumers resolve dependencies at install time via pip/uv.
