RBAC Summary:
  N/A — ai4rag is a Python library, not a Kubernetes component.
  No ClusterRoles, Roles, or RoleBindings are created or required.

Service Mesh Configuration:
  N/A — ai4rag does not deploy any Kubernetes Services or Pods.
  When running inside a Pod (e.g., in a Jupyter Notebook on RHOAI),
  the service mesh configuration is determined by the hosting environment.

Secrets:
  No Kubernetes Secrets are created or managed by ai4rag.
  API keys and base URLs are provided by the user via:
  - Constructor parameters: LlamaStackClient(base_url=..., api_key=...)
  - Environment variables (application-specific)
  - Direct parameter passing to model/vector store constructors

Cryptographic Operations:
  - hashlib SHA256: Used ONLY for document deduplication in ChromaVectorStore
    (not security-sensitive — deterministic content hashing)
  - TLS: Delegated entirely to httpx (Llama Stack) and openai SDK
  - No custom TLS configuration, certificate handling, or crypto operations
  - FIPS compliance depends on the runtime environment's OpenSSL configuration

Build & Deployment:
  - No container images produced (no Dockerfile/Containerfile)
  - Not subject to Konflux check-payload validation
  - Distributed as pip-installable Python package via PyPI
  - No OLM operator, no CSV manifest
  - Version pinning via pyproject.toml constraints (no resolved lock file)
