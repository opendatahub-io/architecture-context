- No ClusterRoles or Roles created by the Helm chart

- No Kubernetes API permissions required
- Service Accounts:
  - {release}-batch-gateway-apiserver
  - {release}-batch-gateway-processor
  - {release}-batch-gateway-gc

Secrets:
- batch-gateway-secrets (Opaque)
  - redis-url: Redis/Valkey connection URL
  - postgresql-url: PostgreSQL connection URL
  - inference-api-key: API key for inference gateway
  - s3-secret-access-key: AWS S3 secret access key
  - Provisioned by: Manual (kubectl create secret)
  - Auto-Rotate: No

- {release}-batch-gateway-apiserver-tls (kubernetes.io/tls)
  - TLS certificate and key for API server HTTPS
  - Provisioned by: cert-manager (optional)
  - Auto-Rotate: Yes (cert-manager)

Security Hardening:
- All pods: readOnlyRootFilesystem, runAsNonRoot (UID 65532)
- All pods: drop ALL capabilities, seccompProfile RuntimeDefault
- Secret file reads use os.OpenInRoot() to prevent path traversal
- API key read from /etc/.secrets/ or named file
- Cross-tenant access returns 404 (prevents information leakage)
- Slowloris protection: read_header_timeout=10s

Observability Exposure:
- API Server: /health, /ready, /metrics on 8081/TCP (HTTP, unauthenticated)
- Processor: /health, /ready, /metrics on 9090/TCP (HTTP, unauthenticated)
- GC: No exposed ports
- Prometheus scrape via ServiceMonitor (apiserver) and PodMonitor (processor)
