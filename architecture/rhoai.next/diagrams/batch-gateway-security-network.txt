RBAC Summary:
  No ClusterRoles or Roles are created by the Helm chart.
  All components run with service account tokens but require no
  Kubernetes API permissions.

  Service Accounts:
  - {release}-batch-gateway-apiserver
  - {release}-batch-gateway-processor
  - {release}-batch-gateway-gc

Secrets:
  - batch-gateway-secrets (Opaque)
    - redis-url: Redis/Valkey connection URL
    - postgresql-url: PostgreSQL connection URL
    - inference-api-key: Bearer token for inference gateway
    - s3-secret-access-key: AWS S3 credentials
    Provisioned by: Manual (kubectl create secret)
    Auto-rotate: No

  - {release}-batch-gateway-apiserver-tls (kubernetes.io/tls)
    - tls.crt: TLS certificate for API server HTTPS
    - tls.key: TLS private key
    Provisioned by: cert-manager (optional)
    Auto-rotate: Yes (cert-manager)

Security Hardening:
  - All pods: readOnlyRootFilesystem: true
  - All pods: runAsNonRoot: true, runAsUser: 65532
  - All pods: drop ALL capabilities
  - All pods: seccompProfile: RuntimeDefault
  - Secret reads: os.OpenInRoot() prevents path traversal
  - Tenant isolation: All queries scoped to tenant ID
  - Cross-tenant access: Returns 404 (not 403) to prevent info leakage
  - Slowloris protection: read_header_timeout_seconds: 10

Auth Flow:
  External Client → Kuadrant/Authorino (AuthN) → X-MaaS-Username injection
  → API Server (tenant scoping) → Data Layer (tenant-filtered queries)

  Inference Auth:
  Processor → Inference Gateway
  - Option 1: API key from named file
  - Option 2: API key from K8s secret mount (/etc/.secrets/)
  - Option 3: ServiceAccount token (standard mount)
  - Option 4: mTLS with custom CA certificate
