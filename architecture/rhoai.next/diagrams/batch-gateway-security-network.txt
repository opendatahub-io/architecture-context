Secrets:
- batch-gateway-secrets (Opaque)
  - redis-url: Redis/Valkey connection URL
  - postgresql-url: PostgreSQL connection URL
  - inference-api-key: Bearer token for inference gateway
  - s3-secret-access-key: AWS S3 secret access key
  - Provisioned by: Manual (kubectl create secret)
  - Auto-Rotate: No

- {release}-batch-gateway-apiserver-tls (kubernetes.io/tls)
  - tls.crt, tls.key: TLS certificate and private key for apiserver HTTPS
  - Provisioned by: cert-manager (when enabled) or manual
  - Auto-Rotate: Yes (cert-manager) / No (manual)

RBAC:
- No ClusterRoles or RoleBindings defined
- ServiceAccounts used for pod identity only:
  - {release}-batch-gateway-apiserver
  - {release}-batch-gateway-processor
  - {release}-batch-gateway-gc

Authentication Model:
- No authentication at batch-gateway layer
- Tenant identity extracted from HTTP header (default: X-MaaS-Username)
- Relies on upstream auth proxy (e.g., kube-rbac-proxy) for identity validation
- All data queries scoped by tenant ID
- File storage paths use SHA256-derived folder names incorporating tenant ID

FIPS Compliance:
- Currently builds with CGO_ENABLED=0 (static linking) — would FAIL check-payload
- Konflux Dockerfiles expose CGO_ENABLED as build ARG for downstream override
- stdlib crypto/tls only (no third-party crypto)
- TLS MinVersion: TLS 1.2
- InsecureSkipVerify available for testing (default: false)
