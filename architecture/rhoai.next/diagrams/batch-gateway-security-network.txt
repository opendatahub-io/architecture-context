Multi-Tenant Security:
  - Tenant isolation via X-MaaS-Username header
  - All DB queries filtered by tenant_id
  - S3 files organized by t-{SHA256(tenant_id)} folder prefix
  - Cross-tenant access returns HTTP 404 (not 403) to hide resource existence
  - Assumes upstream auth proxy injects tenant header (Envoy ext_authz pattern)
  - LastHeaderValue utility handles auth-injected header after client-supplied value

FIPS Compliance:
  - Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
  - Runtime: stdlib crypto/tls backed by OpenSSL (UBI 9 base image)
  - TLS: MinVersion=tls.VersionTLS12 enforced
  - Crypto: crypto/sha256 for tenant folder hashing
  - Risk: InsecureSkipVerify configurable (testing only, logged as WARNING)
  - No use of MD5, RC4, DES, or other non-FIPS algorithms
