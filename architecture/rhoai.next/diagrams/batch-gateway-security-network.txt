RBAC Summary:
- No Kubernetes RBAC roles defined
- The component operates as a user-space service with database-level access control
- Multi-tenancy enforced at application layer via tenant ID scoping (X-MaaS-Username header)
- Cross-tenant isolation: 404 response (not 403) prevents information leakage

Authentication & Authorization:
- /v1/batches/* : API key, SA token, or OpenShift user token
  - Enforcement: Kuadrant/Authorino (gateway layer)
  - Policy: Batch API admission policy
- /v1/files/* : API key, SA token, or OpenShift user token
  - Enforcement: Kuadrant/Authorino (gateway layer)
  - Policy: Batch API admission policy
- Inference endpoints : API key (Bearer) or mTLS client certs
  - Enforcement: Inference gateway
  - Policy: Per-model inference authorization
- /health, /ready, /metrics : Unauthenticated
  - Enforcement: None
