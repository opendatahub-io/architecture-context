================================================================================
  RBAC Summary
================================================================================

No ClusterRoles, Roles, or RoleBindings defined.
Batch Gateway runs as a stateless service without Kubernetes API access.
Pod identity is provided by ServiceAccounts created by the Helm chart:
  - {release}-apiserver
  - {release}-processor
  - {release}-gc

================================================================================
  Secrets
================================================================================

Secret: batch-gateway-secrets (Opaque)
  - redis-url: Redis/Valkey connection URL
  - postgresql-url: PostgreSQL connection URL
  - inference-api-key: Bearer token for inference gateway
  - s3-secret-access-key: AWS S3 secret access key
  Provisioned by: Manual (kubectl create secret)
  Auto-Rotate: No

Secret: {release}-apiserver-tls (kubernetes.io/tls)
  - tls.crt: TLS certificate for API server
  - tls.key: TLS private key for API server
  Provisioned by: cert-manager (optional) or manual
  Auto-Rotate: Yes (when cert-manager)

================================================================================
  Authentication & Authorization Model
================================================================================

- AuthN is FULLY DELEGATED to the gateway layer (Kuadrant / Authorino)
- API server does NOT perform authentication itself
- Tenant ID extracted from last value of X-MaaS-Username header
  (Envoy appends verified identity after client-supplied values)
- All data access scoped to authenticated tenant
- Cross-tenant access returns HTTP 404 (prevents tenant enumeration)
- Observability endpoints (/health, /ready, /metrics) are unauthenticated
  and served on separate ports (8081, 9090) not exposed externally
