  - scheduledworkflow-sa-token (Projected SA Token) - API server auth, Kubernetes projected volume, 1h expiry auto-rotate

Authentication Mechanisms:
  - /apis/v2beta1/* (multi-user): Bearer Token (JWT) → TokenReview + SubjectAccessReview
  - /apis/v2beta1/* (single-user): No auth enforced
  - /webhooks/*: Kubernetes API server client certificate (mutual TLS)
  - Cache webhook: MutatingWebhookConfiguration (K8s API server auth)
  - Persistence Agent → API Server: Projected SA Token (1h, pipelines.kubeflow.org audience)
  - Scheduled Workflow Controller → API Server: Projected SA Token (1h, pipelines.kubeflow.org audience)

FIPS Compliance Notes:
  - API Server, Driver, Persistence Agent, Scheduled Workflow Controller: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamically linked
  - Launcher: Ships BOTH launcher-v2 (non-FIPS, static) and launcher-v2-fips (FIPS, dynamic)
  - FIPS binary selection controlled by DSPO operator (not in this repo)
  - Potential gap: Default entrypoint uses non-FIPS binary
