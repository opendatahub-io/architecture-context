  RBAC Summary
================================================================================

  - kubeflow-pipelines-cache-deployer-clusterrole
    • certificates.k8s.io: certificatesigningrequests → create, delete, get, update
    • certificates.k8s.io: certificatesigningrequests/approval → create, delete, get, update
    • admissionregistration.k8s.io: mutatingwebhookconfigurations → create, delete, get, list, patch
    • certificates.k8s.io: signers → approve

    Provisioned By: Kubernetes (projected volume)
    Auto-Rotate: Yes (auto-refresh)

================================================================================
  Authentication & Authorization Chain
================================================================================

  End-User Requests:
    1. Request arrives via OpenShift Route (TLS edge termination)
    2. API Server extracts identity:
       a. HTTP header extraction (for identity-aware proxies like Istio)
       b. Kubernetes TokenReview (for bearer tokens)
    3. Authorization via SubjectAccessReview on pipelines.kubeflow.org API group
    4. Namespace-scoped RBAC enforcement
    Note: In single-user mode, all authorization is bypassed

  Service-to-Service (Internal):
    - Persistence Agent → API Server: Projected SA token (audience: pipelines.kubeflow.org)
    - Scheduled Workflow Controller → API Server: Projected SA token (audience: pipelines.kubeflow.org)
    - Both validated via TokenReview at the API Server

  Admission Webhooks:
    - PipelineVersion webhook (8443/TCP): TLS, Kubernetes admission
    - Cache server webhook (443/TCP): TLS (webhook-server-tls Secret), Kubernetes admission

================================================================================
  FIPS Compliance Notes
================================================================================

  - All production binaries: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
  - Go module: godebug fips140=on
  - Base image: UBI 9 minimal with OpenSSL 3.5.1
  - Exception: Launcher ships non-FIPS static binary (launcher-v2, CGO_ENABLED=0)
    alongside FIPS binary (launcher-v2-fips) — non-FIPS binary will fail check-payload
