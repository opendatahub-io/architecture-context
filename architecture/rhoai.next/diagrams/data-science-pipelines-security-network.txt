Service Mesh Configuration (multi-user mode):
===============================================
- PeerAuthentication: STRICT (ISTIO_MUTUAL) — all traffic mTLS
- AuthorizationPolicy: Restricts ml-pipeline access by source principal
- DestinationRule: ISTIO_MUTUAL for service-to-service communication
- VirtualService: Routes via kubeflow-gateway

Secrets:
========
- mysql-secret (Opaque) — Database username/password, Manual/DSPA operator, No auto-rotate
- mlpipeline-minio-artifact (Opaque) — S3/MinIO access key & secret key, Manual/DSPA operator, No auto-rotate
- webhook-server-tls (kubernetes.io/tls) — Cache webhook TLS cert, Cache deployer (CSR) or cert-manager, Auto-rotated

Pod Security:
=============
- runAsNonRoot: true
- runAsUser: 1000 (removed on OpenShift — SCC manages)
- Capabilities: drop ALL
- seccompProfile: RuntimeDefault
- FIPS: GOEXPERIMENT=strictfipsruntime, CGO enabled, UBI9 base images
