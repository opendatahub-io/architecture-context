RBAC Summary:
- No Kubernetes RBAC — stateless service with no K8s API access
- No ClusterRoles, no RoleBindings
- No ServiceAccount privileges beyond default

Secrets:
- LLM API Key (env var: varies by provider) — Opaque — Manual / platform operator — No auto-rotate
- OTEL credentials (optional) — Opaque — Manual — No auto-rotate

Authentication & Authorization:
- /v1/chat/completions: Bearer Token forwarded via X-Authorization to LLM provider
  Enforcement: Application-level (NeMo Guardrails server)
  Policy: K8s/proxy Authorization header NOT forwarded; only X-Authorization
- /v1/guardrail/checks: Same as chat/completions
- /v1/models: Bearer Token from Authorization header forwarded to upstream
  Enforcement: Application-level (passthrough)

FIPS Compliance:
- TLS: No TLS termination — relies on platform ingress. Outbound HTTPS uses system OpenSSL
- Crypto: cryptography==44.0.3 (via presidio-anonymizer, OpenSSL backend on RHEL)
- Risk: hashlib.md5 used for cache keys (non-cryptographic)
  - utils.py: FIPS-aware fallback to SHA-256 ✓
  - embeddings/cache.py: Missing FIPS fallback ✗ (would fail in strict FIPS)
- Certificate handling: System trust store (UBI9 base image)
