- No RBAC resources defined — standalone service, not an operator
- RBAC is managed by the deploying entity (rhods-operator or user)
- Platform-level auth via kube-rbac-proxy sidecar or Gateway API

Secrets:
- LLM API Key (env: varies per provider) - Opaque - Auth to upstream LLM - Manual - No auto-rotate
- Guardrail API Keys (env: varies per guardrail) - Opaque - Auth to external guardrail services - Manual - No auto-rotate
- Redis credentials (env: REDIS_PASSWORD) - Opaque - Auth to Redis - Manual - No auto-rotate

Authentication & Authorization:
- All /v1/* endpoints: No built-in auth — relies on platform-level auth
- Upstream LLM calls: Bearer Token / API Key forwarded from client request headers
- External guardrail APIs: API Key from environment variables per provider
- CORS: Disabled by default (explicit opt-in via env vars)

FIPS Compliance:
- Base image: UBI9 Python 3.12 (FIPS-validated OpenSSL)
- cryptography==44.0.3 (transitive) uses system OpenSSL backend
- No custom cipher suites or non-FIPS crypto detected
- FIPS compliance inherited from UBI9 when host is in FIPS mode

Security Controls:
- Thread ID minimum length: 16 characters
- Path traversal protection in configuration loading (regex + resolved-path check)
- Auth token redaction in log output
- DO_NOT_TRACK=1 (telemetry opt-out by default)
- Guardrail profile filtering removes closed-source guardrails from build
