RBAC Summary:
- No Kubernetes RBAC resources defined — stateless service container
- No ClusterRoles, ClusterRoleBindings, Roles, or RoleBindings

Authentication & Authorization:
- No built-in authentication or authorization
- Authorization header passthrough pattern:
  - Client sends Authorization header
  - Server forwards header to upstream LLM provider
  - Per-request auth delegation (no stored API keys required)
- Alternative: environment variable-based API keys (api_key_env_var in config)
  - Shared key for all requests from env var
- Auth tokens redacted from invocation parameter logs (logging/callbacks.py)
- Service assumes platform-level auth enforcement (kube-rbac-proxy / Gateway API)

Secrets:
- LLM API keys (env vars) — Opaque — Manual provisioning — No auto-rotation
  - Contains: API authentication credentials for upstream LLM providers
  - Provisioned by: External secret management or manual configuration

FIPS Compliance:
- Python service on UBI9 — inherits FIPS-validated OpenSSL from OS
- cryptography==44.0.3 (transitive via presidio-anonymizer) uses system OpenSSL
- hashlib.md5 usage in utils.py has FIPS-aware SHA-256 fallback (lines 410-416)
- hashlib.md5 usage in embeddings/cache.py — no FIPS fallback (cache keys only, non-security)
- Certificate handling: System trust store (UBI9 ca-certificates)
