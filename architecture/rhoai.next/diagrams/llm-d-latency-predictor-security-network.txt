Security Concerns:
  1. No TLS on any endpoint — all traffic is plaintext HTTP
  2. No authentication or authorization on any endpoint
  3. /add_training_data_bulk accepts arbitrary data (model poisoning risk)
  4. /model/{name}/download serves joblib/pickle objects over plaintext
     (deserialization attack vector + data integrity risk)
  5. MD5 used for model checksums (non-security context, but weak hash)
  6. Default ServiceAccount used (no least-privilege RBAC)
  7. LoadBalancer services expose endpoints externally without protection
  8. In RHOAI production: requires service mesh (mTLS) or kube-rbac-proxy
