  - core (endpoints, pods, pods/log): get, list, watch
  - apps/deployments: create, delete, get, list, patch, update, watch
  - route.openshift.io (routes, routes/custom-host): create, delete, get, list, patch, update, watch
  - user.openshift.io/groups: create, delete, get, list, patch, update, watch
  - rbac.authorization.k8s.io (roles, rolebindings, clusterrolebindings): CRUD

  - components.platform.opendatahub.io/modelregistries: get, list, watch
  - services.platform.opendatahub.io/auths: get, list, watch
  - config.openshift.io (ingresses, apiservers): get, list, watch

- ClusterRoleBinding: {registry-name}-auth-delegator → system:auth-delegator
- ClusterRoleBinding: model-catalog-auth-delegator → system:auth-delegator
- RoleBinding: {registry-name} → {registry-name}-users Group
- RoleBinding: leader-election-rolebinding → leader-election-role

NetworkPolicy:
- Per-registry NetworkPolicy restricting ingress to kube-rbac-proxy port

Secrets:
- {registry-name}-kube-rbac-proxy (kubernetes.io/tls) - OpenShift serving cert, auto-rotated
- {registry-name}-postgres-credentials (Opaque) - Random 16-byte password, NOT auto-rotated
- model-catalog-postgres (Opaque) - Catalog DB credentials, NOT auto-rotated
- User-provided DB secrets (Opaque) - PostgreSQL/MySQL credentials, NOT auto-rotated

Authentication Modes:
- Traditional OpenShift: user.openshift.io Groups + kube-rbac-proxy SAR
- BYOIDC (OpenShift 4.20+): kube-rbac-proxy SAR only, Group creation skipped
- Plain Kubernetes: kube-rbac-proxy SAR only, no Routes/Groups

FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
- Base image: registry.access.redhat.com/ubi9/ubi-minimal
- TLS: Dynamic cluster TLS security profile integration
- No InsecureSkipVerify in production code
