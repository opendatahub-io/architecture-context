RBAC Summary:

OpenShift:
- Inherited from oc adm must-gather framework
- ServiceAccount has broad read access via must-gather mechanism
- No custom ClusterRole required

xKS (Non-OpenShift):
- ClusterRole: must-gather-reader
  - API Groups: * (all)
  - Resources: * (all)
  - Verbs: get, list, watch
- ClusterRoleBinding: must-gather-reader-binding
  - ServiceAccount: must-gather-sa (user-chosen namespace)

Secrets:
- redhat-pull-secret (xKS only, kubernetes.io/dockerconfigjson)
  - Purpose: Image pull secret for registry.redhat.io
  - Provisioned by: Manual (user creates from podman/docker auth)
  - Auto-Rotate: No

Authentication & Authorization:
- All API access uses ServiceAccount bearer token
- Read-only operations only (GET, LIST, WATCH)
- No write access to any cluster resources
- No network egress outside cluster (only talks to kube-apiserver)

FIPS Compliance:
- Pure Bash scripts — no compiled binaries
- FIPS inherited from base image (ose-must-gather-rhel9)
- TLS delegated to oc/kubectl (OpenSSL in base image)
- No custom crypto operations
