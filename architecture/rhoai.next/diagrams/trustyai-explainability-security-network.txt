RBAC Summary:
- ClusterRole: trustyai-role
  - API Groups: "" (core)
  - Resources: configmaps
  - Verbs: create, delete, get, list, patch, update, watch
- ClusterRoleBinding: trustyai-clusterrolebinding
  - ServiceAccount: trustyai-serviceaccount (cluster-wide)

Authentication & Authorization:
- TrustyAI service implements NO built-in authentication or authorization
- Auth is delegated to sidecar proxies injected by TrustyAI Service Operator:
  - RHOAI 3.x: kube-rbac-proxy sidecar
  - RHOAI 2.x: oauth-proxy sidecar
- /q/metrics: Bearer token (SA token) for Prometheus scraping
- All other endpoints: Rely on auth proxy + network isolation

Secrets:
- TLS certificate (/etc/tls/internal/)
  - Type: kubernetes.io/tls
  - Purpose: HTTPS on port 4443/TCP
  - Provisioned by: OpenShift service serving certificates or manual
  - Auto-Rotate: No

- Database credentials
  - Type: Opaque (env vars)
  - Purpose: MariaDB/MySQL username and password
  - Provisioned by: Manual / operator-provisioned
  - Auto-Rotate: No

- MinIO credentials
  - Type: Opaque (env vars)
  - Purpose: S3 access key and secret key
  - Provisioned by: Manual / operator-provisioned
  - Auto-Rotate: No

RHOAI Distribution Notes:
- Explainer endpoints (SHAP, LIME, CF, TS Saliency) DISABLED in RHOAI profile
- Fairness metrics (SPD, DIR) and drift detection ENABLED
- Image: registry.redhat.io/rhods/odh-trustyai-service-rhel8:latest
- PVC storage default: 1Gi ReadWriteOnce (limits to single-replica)
