RBAC Summary:
- ClusterRole: trustyai-role
  - API Groups: "" (core)
  - Resources: configmaps
  - Verbs: create, delete, get, list, patch, update, watch

- ClusterRoleBinding: trustyai-clusterrolebinding
  - Scope: cluster-wide
  - Role: trustyai-role (ClusterRole)
  - ServiceAccount: trustyai-serviceaccount

Authentication & Authorization:
- TrustyAI service has NO built-in authentication
- All auth is enforced by sidecar proxy injected by TrustyAI Service Operator:
  - RHOAI 3.x: kube-rbac-proxy
  - RHOAI 2.x: oauth-proxy
- /q/metrics endpoint: Bearer token (ServiceAccount token) via ServiceMonitor bearerTokenFile
- Network isolation: Service is ClusterIP (internal only), Route provides external access

Secrets:
- TLS certificate (kubernetes.io/tls) - Mounted at /etc/tls/internal/, enables HTTPS on 4443/TCP
  - Provisioned by: OpenShift service serving certs or manual
  - Auto-Rotate: No
- Database credentials (Opaque) - MariaDB/MySQL username and password
  - Provisioned by: Manual / operator-provisioned
  - Auto-Rotate: No
- MinIO credentials (Opaque) - S3 access key and secret key
  - Provisioned by: Manual / operator-provisioned
  - Auto-Rotate: No

Storage Modes (SERVICE_STORAGE_FORMAT env var):
- PVC (default): CSV flat files on 1Gi ReadWriteOnce PersistentVolumeClaim
- DATABASE: Hibernate ORM with MariaDB/MySQL via JDBC
- MINIO: S3-compatible object storage
- MEMORY: In-memory (development only)

RHOAI Profile Restrictions:
- Explainer endpoints DISABLED: /explainers/local/shap, /explainers/local/lime,
  /explainers/local/cf, /explainers/local/tssaliency, /explainers/global/lime,
  /explainers/global/pdp
- Fairness metrics ENABLED: SPD, DIR
- Drift metrics ENABLED: KS, approxKS, FourierMMD, Meanshift
