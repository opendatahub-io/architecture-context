================================================================================
  RBAC SUMMARY
================================================================================

OVMS does not define its own RBAC resources. All RBAC is managed by KServe.

Container Security Context:
- Runs as: UID 5000 / GID 5000 (non-root)
- Base image: UBI 9 (Red Hat Universal Base Image 9)
- No privileged capabilities required

Authentication Enforcement:
- REST API: Optional API key via --api_key_file flag
  - Validated by: HttpRestApiHandler::isAuthorized()
  - Exempted: /v2/health/live, /v2/health/ready, /metrics
- gRPC API: No built-in authentication
  - Relies on: kube-rbac-proxy sidecar (8443/TCP, HTTPS)
- RHOAI deployment: kube-rbac-proxy enforces RBAC for all external access
