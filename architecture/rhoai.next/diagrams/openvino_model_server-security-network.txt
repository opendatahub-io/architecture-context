RBAC Summary:
  OVMS is a standalone inference server and does NOT define Kubernetes RBAC roles.
  RBAC for the KServe InferenceService is managed by the platform operator.

Authentication Summary:
  - REST API: Optional Bearer Token via --api_key_file or API_KEY env var
  - gRPC API: No built-in auth; relies on external proxy (nginx mTLS,
    kube-rbac-proxy, oauth-proxy, or Istio)
  - Health probes (/v2/health/ready, /v2/health/live): Unauthenticated
  - Metrics (/metrics): Unauthenticated; enabled via --metrics_enable flag

Security Controls:
  - Path traversal prevention: isPathEscaped() blocks ../ in model paths
  - API key auth: validates Authorization: Bearer header against configured key
  - Platform-level auth: kube-rbac-proxy (RHOAI 3.x) / oauth-proxy (RHOAI 2.x)
  - Optional mTLS: nginx sidecar (extras/nginx-mtls-auth/)

Secrets:
  - AWS credentials (env vars): AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY,
    AWS_SESSION_TOKEN - S3 storage access - No auto-rotation
  - GCS credentials (ADC): Application Default Credentials - GCS storage
    access - No auto-rotation
  - Azure credentials (connection string): Azure Blob/File storage access -
    No auto-rotation
  - HuggingFace token (env var): HF_TOKEN - HF Hub model access -
    No auto-rotation
  - API key (file or env var): --api_key_file or API_KEY - REST API
    authentication - No auto-rotation
