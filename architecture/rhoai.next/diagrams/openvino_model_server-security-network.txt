RBAC Summary:
- OVMS does not define Kubernetes RBAC roles.
- RBAC for the KServe InferenceService is managed by the platform operator.
- No Kubernetes API interactions from the OVMS container.

Authentication Summary:
- REST API: Optional Bearer Token via --api_key_file or API_KEY env var
  - Validated in-process by HttpRestApiHandler
  - Authorization header: "Bearer <api_key>"
- gRPC API: No built-in authentication
  - Relies on external proxy (nginx mTLS, kube-rbac-proxy, or Istio)
- /metrics endpoint: Unauthenticated (enabled via --metrics_enable flag)
- /v2/health/ready, /v2/health/live: Unauthenticated health probes
- Production auth: kube-rbac-proxy sidecar (RHOAI 3.x) or oauth-proxy (RHOAI 2.x)

Security Controls:
- Path traversal prevention: isPathEscaped() blocks ../ sequences in model paths
- No built-in mTLS or OIDC support
- Optional nginx mTLS sidecar available (extras/nginx-mtls-auth/)
- Max message size: 1GB (gRPC and REST)
- CORS support configurable via Drogon HTTP server

Secrets:
- AWS credentials (Environment variables) - S3 access, user-provisioned, no auto-rotate
- GCS credentials (Application Default Credentials) - GCS access, user-provisioned, no auto-rotate
- Azure credentials (Connection string) - Azure storage access, user-provisioned, no auto-rotate
- HuggingFace token (HF_TOKEN env var) - HF Hub access, user-provisioned, no auto-rotate
- API key (--api_key_file or API_KEY env) - REST API auth, user-provisioned, no auto-rotate
