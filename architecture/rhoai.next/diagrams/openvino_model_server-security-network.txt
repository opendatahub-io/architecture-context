RBAC Summary:
- OVMS is a standalone server binary — it does NOT define Kubernetes RBAC resources
- RBAC is managed by KServe ServingRuntime deployment
- kube-rbac-proxy sidecar enforces AuthN (TokenReview) and AuthZ (SubjectAccessReview)

FIPS Compliance Notes:
- C++ binary — not Go, so no GOEXPERIMENT=strictfipsruntime applicable
- Links dynamically against system OpenSSL on UBI 9 (FIPS-capable)
- gRPC server uses InsecureServerCredentials() — no built-in TLS
- Uses OpenSSL MD5 for file integrity checksumming (non-security, non-FIPS-approved hash)
- SDL hardening flags: -fstack-protector, -D_FORTIFY_SOURCE=1, -fstack-clash-protection
- No OLM FIPS annotation (not an OLM operator)

Security Concerns:
1. No built-in TLS — all transport security relies on kube-rbac-proxy sidecar
2. gRPC uses InsecureServerCredentials() — plaintext within pod
3. Optional API key auth only on /v3/* endpoints — /v1/*, /v2/* have no built-in auth
4. MD5 used for file integrity (non-FIPS-approved, but non-security use case)
5. No secret auto-rotation for any credentials
6. Python interpreter module (pybind11) executes arbitrary Python in custom nodes
