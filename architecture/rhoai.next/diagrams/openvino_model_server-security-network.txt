FIPS Compliance Notes:
- No TLS termination in OVMS itself (delegates to platform)
- gRPC uses InsecureServerCredentials
- OpenSSL via UBI 9 base image (respects FIPS mode when enabled at OS level)
- MD5 used in src/filesystem.hpp for file content deduplication (not security)
  WARNING: openssl/md5.h triggers strict FIPS audit warnings
- CURL SSL configuration delegates to system OpenSSL

Crypto Flow:
  OVMS → libcurl → system OpenSSL (UBI 9) → S3/GCS/Azure/HuggingFace
  OVMS → libgit2 → system OpenSSL (UBI 9) → HuggingFace (git clone)
  System trust store: /etc/pki/tls/certs/ca-bundle.crt → ca-certificates.crt
