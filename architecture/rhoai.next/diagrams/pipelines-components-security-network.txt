Security Concerns:
⚠️  SSL verification disabled as fallback in multiple components:
  - boto3 verify=False on self-signed cert errors (S3 connections)
  - httpx verify=False on self-signed cert errors (Llama Stack, OpenAI endpoints)
  - ssl.CERT_NONE fallback in text extraction components
  - This is a development convenience that weakens TLS in non-production deployments
⚠️  Kubeflow Model Registry uses plaintext HTTP (8080/TCP) with no authentication
⚠️  All secrets are user-provisioned with no auto-rotation

Certificate Handling:
- Honors REQUESTS_CA_BUNDLE and SSL_CERT_FILE env vars for custom CA bundles
- Falls back to system trust store (UBI9 OpenSSL)
- SSL fallback pattern: try TLS → catch SSLError → retry with verify=False

FIPS Compliance:
- Python-only (no Go binaries, no CGO)
- UBI9 base images provide FIPS-validated OpenSSL
- Python ssl module delegates to system OpenSSL on UBI9
- ⚠️  SSL verify=False fallback bypasses FIPS TLS validation
