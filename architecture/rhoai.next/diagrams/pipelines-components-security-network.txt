Security Concerns:
- ⚠ SSL verification disabled as fallback in multiple components (boto3 verify=False, httpx verify=False)
  Affected: automl_data_loader, text_extraction, documents_indexing, build_responses_request_bodies
  Risk: Weakened TLS in non-production deployments, potential MITM
  Mitigation: Set REQUESTS_CA_BUNDLE / SSL_CERT_FILE env vars for custom CA bundles

- ⚠ Model Registry communication is plaintext HTTP on port 8080 with no authentication
  Risk: Model metadata tamper, unauthorized model registration
  Mitigation: Network policies to restrict access to Model Registry pod

- ⚠ No secrets auto-rotation configured for any credentials
  Risk: Long-lived credentials increase exposure window
  Mitigation: Manual credential rotation by users

FIPS Compliance:
- Python-based (no Go binaries, no GOEXPERIMENT=strictfipsruntime needed)
- UBI9 base images provide system OpenSSL (FIPS-capable when FIPS mode enabled on host)
- Python ssl module uses system OpenSSL (FIPS-compliant on FIPS-enabled RHEL/OpenShift)
- ⚠ SSL fallback pattern (verify=False) bypasses FIPS TLS validation
