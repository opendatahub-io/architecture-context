INGRESS: None
- No HTTP/gRPC endpoints exposed
- No persistent services
- All components run as ephemeral batch KFP task pods

RBAC Summary:
- No RBAC definitions in this repository
- Pipeline tasks use the KFP pipeline runner ServiceAccount configured by DSPA
- DSPA provisions appropriate RBAC for pipeline execution

Secrets:
- s3-secret / train_data_secret / input_data_secret (Opaque) - AWS S3 credentials, User/Admin provisioned, No auto-rotation
- hf-token (Opaque) - Hugging Face API token, User provisioned, No auto-rotation
- llm-credentials / llama_stack_secret (Opaque) - LLM API key and endpoint, User/Admin provisioned, No auto-rotation
- kubernetes-credentials (Opaque) - K8s API server URL and auth token for Model Registry, User/Admin provisioned, No auto-rotation
- oci-pull-secret-model-download (Opaque) - OCI registry credentials, User/Admin provisioned, No auto-rotation
- s3-connection (Opaque) - DSPA S3 connection credentials, DSPA Operator provisioned, No auto-rotation

Authentication Summary:
- S3 Storage: AWS IAM Credentials (AccessKeyId/SecretAccessKey) via K8s Secret
- Hugging Face Hub: Bearer Token (HF_TOKEN) via K8s Secret
- Llama Stack API: API Key via K8s Secret
- Kubeflow Model Registry: Bearer Token (K8s SA token) via K8s ServiceAccount
- KFP API Server: ServiceAccount Token via DSPA RBAC
- OCI Registry: Registry credentials via K8s Secret
- Kubernetes API: ServiceAccount Token (auto-mounted)

Base Image Governance:
- Allowlisted registries: registry.redhat.io, ghcr.io/kubeflow, quay.io/opendatahub, quay.io/rhoai
- Enforced by: validate_base_images.py during CI
- Image references: Operator-overridable via RELATED_IMAGE_* env vars
