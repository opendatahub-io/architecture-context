RBAC Summary:
  No direct RBAC definitions. Pipeline tasks inherit RBAC from the DSPA
  pipeline runner ServiceAccount, which is configured by the Data Science
  Pipelines Application operator.

  DSPA Pipeline Runner SA Permissions (inherited):
  - Read Secrets (for S3 creds, HF tokens, API keys)
  - Create/manage PVCs (for shared workspaces)
  - Access KFP API (for artifact storage, pipeline status)

Secrets:
  - s3-secret / train_data_secret / input_data_secret (Opaque)
    AWS S3 credentials: endpoint, access key, secret key, region, bucket
    Provisioned by: User/Admin | Auto-Rotate: No

  - hf-token (Opaque)
    Hugging Face API token for model/dataset downloads
    Provisioned by: User | Auto-Rotate: No

  - llm-credentials / llama_stack_secret (Opaque)
    LLM API key and endpoint for inference and embedding
    Provisioned by: User/Admin | Auto-Rotate: No

  - kubernetes-credentials (Opaque)
    Kubernetes API server URL and auth token for Model Registry
    Provisioned by: User/Admin | Auto-Rotate: No

  - oci-pull-secret-model-download (Opaque)
    OCI registry credentials for model artifact pulls
    Provisioned by: User/Admin | Auto-Rotate: No

  - s3-connection (Opaque)
    DSPA S3 connection credentials for pipeline artifact storage
    Provisioned by: DSPA Operator | Auto-Rotate: No

Authentication & Authorization Summary:
  - S3 Storage:            AWS IAM Credentials (AccessKeyId/SecretAccessKey)
  - Hugging Face Hub:      Bearer Token (HF_TOKEN)
  - Llama Stack API:       API Key
  - LLM API (OpenAI):      API Key
  - Model Registry:        Bearer Token (K8s SA token)
  - KFP API Server:        ServiceAccount Token
  - OCI Registry:          Registry credentials (username/password)
  - Kubernetes API:        ServiceAccount Token (auto-mounted)

Network Notes:
  - NO INGRESS: All components are batch KFP task pods with no inbound traffic
  - NO PERSISTENT SERVICES: All pods are ephemeral, created per pipeline run
  - ALL EGRESS: Outbound connections only, to external APIs and storage
  - ENCRYPTION: All external connections use TLS 1.2+ minimum
  - CREDENTIALS: All sensitive credentials stored in Kubernetes Secrets
  - GPU ACCESS: Training pods request nvidia.com/gpu accelerator resources
  - STORAGE: Shared PVCs with nfs-csi (RWX) for multi-pod training workflows
