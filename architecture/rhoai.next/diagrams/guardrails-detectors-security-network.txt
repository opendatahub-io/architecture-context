RBAC Summary:
- No RBAC roles defined by the component
- Detectors run as application workloads and rely on KServe
  and platform-level RBAC
- Example RoleBinding: user-one-view → ClusterRole/view (demo only)

Service Mesh Configuration:
- Istio sidecar injection: sidecar.istio.io/inject: "true"
  (configured via KServe InferenceService annotations)
- mTLS: Enforced via Istio PeerAuthentication (platform-level)
- Auth: security.opendatahub.io/enable-auth: "true"
  (KServe annotation for platform authentication)

Secrets:
- aws-connection-minio-data-connection-guardrails-guardian (Opaque)
  - Keys: ACCESS_KEY, SECRET_KEY, ENDPOINT, BUCKET, REGION
  - Purpose: S3 credentials for HuggingFace model storage
  - Provisioned: Manual (example); platform operator in production
  - Auto-Rotate: No

TLS Configuration:
- Application-level TLS: Optional (uvicorn ssl_ca_certs in config)
- No hardcoded TLS configuration in detector code
- System trust store: UBI9 default
- FIPS: Python on UBI9 inherits FIPS-compliant OpenSSL
- Konflux HF image pins OpenSSL 3.5.1-5.el9_7

Security Notes:
- Custom detector subsystem runs AST-based static analysis before
  loading user Python code
- Forbidden imports: os, subprocess, sys, shutil
- Forbidden calls: eval, exec, open, compile, input
- Exception: os.environ, os.getenv allowed
- LLM Judge → vLLM connection has NO encryption and NO auth
  (relies on network-level isolation)
