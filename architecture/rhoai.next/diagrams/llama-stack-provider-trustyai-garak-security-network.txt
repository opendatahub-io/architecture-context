================================================================================
Security Notes
================================================================================

1. Reference deployment manifests contain placeholder/dev credentials
   (default PostgreSQL password, TLS verification disabled for vLLM)
2. All authentication delegated to Kubernetes Secrets and ServiceAccount tokens
3. RBAC follows least-privilege: only get/create on DSPA API resources
4. API keys use __FROM_ENV__ pattern for K8s Secret injection
5. Process isolation via os.setsid() for Garak subprocess management
6. Async concurrency control via asyncio.Semaphore for inline mode
7. No service mesh (mTLS) — cluster-internal HTTP is plaintext
