Secrets:
- llama-stack-inference-model-secret (Opaque) - vLLM API token, Manual, No rotation
- model-auth (Opaque) - Multi-role API keys (api-key, SDG_API_KEY, JUDGE_API_KEY,
  ATTACKER_API_KEY, EVALUATOR_API_KEY), Manual, No rotation
- Data Connection Secret (Opaque) - AWS S3 credentials (ACCESS_KEY, SECRET_KEY,
  ENDPOINT, BUCKET), OpenShift AI Data Connection, No rotation
- kubeflow-credentials (Opaque, optional) - KFP API token, Manual, No rotation

API Key Security:
- Keys resolved at runtime via resolve_api_key(role) with layered fallback:
  1. Volume mount: /mnt/model-auth/{ROLE}_API_KEY
  2. Fallback: /mnt/model-auth/api-key (generic)
  3. Fallback: Environment variable
- Configs always redacted (redact_api_keys) before upload to S3, Files API, or OCI
- Placeholder values (DUMMY, __FROM_ENV__, ***) replaced at pod startup

FIPS Compliance:
- Python application on UBI9 base image (system OpenSSL)
- TLS via httpx (system ssl module) and boto3 (system OpenSSL)
- No direct use of MD5, RC4, DES, or custom crypto
- Certificate verification configurable (GARAK_TLS_VERIFY)
