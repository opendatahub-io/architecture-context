RBAC Summary:
- No ClusterRoles defined in this repository
- Runtime requirement (documented, not provisioned by this repo):
  - Role: secret-reader (target namespace)
    - API Groups: ""
    - Resources: secrets
    - Verbs: get
    - Service Account: Job service account (configured by eval-hub)

Secrets:
- model-auth (Opaque)
  - Keys: API keys for target/judge/attacker/evaluator/SDG models
  - Mount: /mnt/model-auth (volume mount, optional=True)
  - Provisioned by: User / Operator
  - Auto-rotate: No
  - Key resolution: role-specific env var → generic fallback → mounted file → "DUMMY"

- Data Connection secret (Opaque, configurable name)
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_S3_ENDPOINT,
           AWS_S3_BUCKET, AWS_DEFAULT_REGION
  - Injection: use_secret_as_env (KFP pods) or env vars (adapter pod)
  - Provisioned by: OpenShift AI Data Connection
  - Auto-rotate: No

TLS Configuration:
- httpx client with configurable TLS verification (verify=True default)
- Supports custom CA path via `tls_verify` parameter
- System trust store used by default
- Crypto: cryptography==48.0.0 (transitive, FIPS-capable via UBI/OpenSSL)
- No non-FIPS crypto detected (no MD5, RC4, DES, custom RNG)
- Shell injection prevented: subprocess shell=False

Security Design Notes:
- API keys never embedded in KFP pipeline args (__FROM_ENV__ placeholders)
- config.json redacted before upload (api_key values → ***)
- Garak subprocess: os.setsid + SIGTERM → 5s → SIGKILL process group cleanup
- stdout/stderr streaming via daemon threads with bounded deque tails
