Secrets:
- llama-stack-inference-model-secret (Opaque) - vLLM API token, Manual provisioning, No auto-rotate
- model-auth (Opaque) - Multi-role API keys (api-key, SDG_API_KEY, JUDGE_API_KEY, ATTACKER_API_KEY, EVALUATOR_API_KEY), Manual, No auto-rotate
- Data Connection Secret (Opaque) - AWS S3 credentials (ACCESS_KEY_ID, SECRET_ACCESS_KEY, S3_ENDPOINT, S3_BUCKET), OpenShift AI Data Connection, No auto-rotate
- kubeflow-credentials (Opaque, optional) - KFP API token (alternative to SA RBAC), Manual, No auto-rotate

API Key Resolution Strategy:
- KFP mode: Secrets mounted as volumes at /mnt/model-auth
- resolve_api_key(role) reads role-specific files with fallback to generic api-key and env vars
- Configs always redacted (redact_api_keys) before upload to S3/Files API/OCI
- Placeholder values (DUMMY, __FROM_ENV__, ***) replaced with real keys at pod startup

Security Controls:
- No HTTP server exposed (library/batch job only)
- Process isolation: garak runs in separate process group (os.setsid)
- Timeout enforcement: SIGTERM → SIGKILL for entire process tree
- Secret redaction: API keys stripped from persisted configs
- TLS verification: Configurable per endpoint (GARAK_TLS_VERIFY)
- FIPS: Inherits UBI9 system OpenSSL; no custom crypto; no MD5/RC4/DES usage
- Build hermeticity: Konflux + Hermeto prefetch (pip, cargo, RPM lockfiles)
