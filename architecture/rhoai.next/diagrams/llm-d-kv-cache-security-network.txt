SECURITY CONCERNS:
⚠️  ZMQ event transport (5557/TCP) has NO encryption and NO authentication
    - Any pod with network access can inject or observe cache metadata
    - Typical for intra-cluster comms but a risk in multi-tenant clusters
⚠️  Redis backend defaults to plaintext (TLS available via rediss:// scheme)
⚠️  Go binary NOT built with GOEXPERIMENT=strictfipsruntime
    - Uses Go standard crypto, not FIPS-validated OpenSSL backend
⚠️  No crypto/tls usage detected in application code
