RBAC Summary:
- No RBAC resources defined in this repository
- Component runs as sidecar using the pod's service account
- RBAC is managed by the parent ModelMesh Serving deployment
- Pod service account inherited from ModelMesh controller configuration

Secrets:
- storage-config (Opaque) - Cloud storage credentials
  - Contents: Access keys, service account JSON, connection strings
  - Mount path: /storage-config
  - Provisioned by: User / Platform operator
  - Auto-rotate: No
  - Security note: Credentials stripped from LoadModel requests before
    forwarding to runtime adapters (puller.go lines 220-221)

Security Notes:
- FIPS-compliant build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
- Base image: UBI 9 Minimal (registry.access.redhat.com/ubi9/ubi-minimal)
- No inter-pod TLS (all adapter communication is localhost within pod)
- Credential isolation: pullman uses FNV-64a hashing for cache keys
  to avoid storing raw credentials in memory-resident cache structures
- Model path sanitization: Prevents directory traversal attacks via
  path joining validation (internal/util/join.go)
