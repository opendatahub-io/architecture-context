RBAC Summary:
- No RBAC resources defined in this repository
- Component runs as sidecar using the pod's ServiceAccount
- RBAC is managed by the parent ModelMesh Serving deployment
- No ClusterRoles, Roles, or RoleBindings defined

Secrets:
- storage-config (Opaque) - Cloud storage credentials
  - Mounted at: /storage-config
  - Contains: Per-storage-type credential files (access keys, service accounts, connection strings)
  - Provisioned by: User/Platform operator
  - Auto-Rotate: No
  - Security: Credentials stripped from gRPC requests before forwarding to runtime adapters
  - Cache: FNV-64a hashed keys (raw credentials not stored in cache)

Build Security:
- FIPS Compliance: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
- Base Image: registry.access.redhat.com/ubi9/ubi-minimal (Red Hat UBI 9)
- Build: Konflux pipeline (Dockerfile.konflux)

Network Security Notes:
- ALL inter-process communication is plaintext gRPC/HTTP over localhost
- No network-level encryption between sidecar containers (same pod network namespace)
- Security boundary is at the pod level, managed by ModelMesh Serving
- Only egress traffic to cloud storage uses TLS encryption
- No ingress from external networks - all inbound is pod-internal from ModelMesh
