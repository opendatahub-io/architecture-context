RBAC Summary:
- No Kubernetes RBAC — this component runs as a sidecar and does NOT
  interact with the Kubernetes API server.
- Uses the pod's service account from the parent deployment (ModelMesh).

Secrets:
- storage-config (Opaque) - Mounted volume at /storage-config
  - Contains: JSON files with storage credentials
    - S3: access_key_id, secret_access_key, endpoint_url, region, bucket
    - GCS: service_account JSON (private_key, client_email, token_uri)
    - Azure: client_id, client_secret, tenant_id OR connection_string
    - HTTP: custom headers, CA cert PEM, client cert/key PEM
  - Provisioned By: User / modelmesh-serving controller
  - Auto-Rotate: No

FIPS Compliance:
- Build flags: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, -tags strictfipsruntime
- Linking: Dynamic (CGO_ENABLED=1) — uses OpenSSL via UBI9 base image
- TLS configuration: Go stdlib crypto/tls (FIPS-compliant via OpenSSL in strict mode)
  - No explicit MinVersion set (defaults to TLS 1.2 in FIPS mode)
  - Custom CA pools from PEM for storage providers
  - Client cert/key pairs for mutual TLS (HTTP provider)
- No non-FIPS crypto detected (no MD5, RC4, DES usage)

Authentication & Authorization:
- Pod-internal: NO authentication (all plaintext gRPC on localhost)
- S3 egress: Static credentials (access_key_id/secret_access_key)
- GCS egress: Service account JSON (private_key/client_email/token_uri)
- Azure egress: ClientSecretCredential OR connection string OR no auth
- HTTP egress: Optional custom headers + optional mTLS client certs

Network Policies:
- No NetworkPolicy defined by this component
- Relies on platform-level network policies from ModelMesh/RHOAI
