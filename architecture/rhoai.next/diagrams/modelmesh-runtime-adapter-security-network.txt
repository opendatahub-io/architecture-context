RBAC Summary:
- No ClusterRoles or RoleBindings
- Component runs as a sidecar container and does not interact with the Kubernetes API
- No ServiceAccount RBAC requirements

Secrets:
- storage-config (Opaque) - Mounted at /storage-config
  - Purpose: Cloud storage credentials (S3 keys, GCS service account, Azure principal, HTTP certs)
  - Provisioned By: User/Admin
  - Auto-Rotate: No
  - Fields per provider:
    - S3: type, access_key_id, secret_access_key, endpoint_url, region, default_bucket, certificate
    - GCS: type, service_account (JSON with private_key, client_email)
    - Azure: type, client_id, client_secret, tenant_id, container_name (or connection_string)
    - HTTP: type, headers (custom), client_certificate, client_key, certificate (CA)

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, -tags strictfipsruntime
- Linking: Dynamic (CGO_ENABLED=1), uses OpenSSL at runtime via UBI 9 base
- All five Go binaries compiled with FIPS-compliant flags
- Python (TensorFlow/Keras): Inherits FIPS from UBI 9 OpenSSL
- No non-FIPS crypto algorithms detected (no MD5, RC4, DES usage)
- Custom CA certificate support for S3 and HTTP providers
- mTLS client certificate support for HTTP provider

Network Security Notes:
- All inter-container communication within the pod is plaintext over localhost
  (acceptable for pod-internal traffic - containers share the same network namespace)
- External storage egress uses TLS 1.2+ with cloud provider authentication
- No ingress from outside the cluster (pod-internal services only)
- Path traversal protection via cyphar/filepath-securejoin library
- Model state serialization via per-model goroutines (25-item channel buffer)
