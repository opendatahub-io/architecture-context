SECURITY SUMMARY
================

Trust Boundaries:
  1. Pod boundary — all internal gRPC is plaintext localhost, no auth required
  2. Pod-to-external — all cloud storage uses TLS 1.2+ with provider credentials

Credential Management:
  - Secret: storage-config (Opaque)
    - Mounted at: /storage-config
    - Contains: S3 access keys, Azure service principal, GCS service account JSON,
                HTTP custom headers, mTLS client certificates, custom CA bundles
    - Provisioned by: User / ModelMesh Serving operator
    - Auto-rotate: No

FIPS Compliance:
  - Build flags: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, -tags strictfipsruntime
  - Linking: Dynamic (CGO_ENABLED=1)
  - Crypto: stdlib crypto/tls and crypto/x509 (FIPS-compliant via OpenSSL with strictfipsruntime)
  - Base image: UBI9 ubi-minimal (includes OpenSSL)
  - Non-FIPS crypto risks: None detected (no MD5, RC4, DES usage)

Path Traversal Protection:
  - All adapters use util.SecureJoin() wrapping cyphar/filepath-securejoin
  - Prevents directory traversal in user-supplied model paths from LoadModelRequest

RBAC:
  - No Kubernetes RBAC roles defined
  - This component runs as a sidecar and does not interact with the Kubernetes API
  - RBAC is managed by the parent ModelMesh Serving deployment

Authentication & Authorization:
  - Pod-internal endpoints (8084/TCP, 8085/TCP): No auth — localhost only
  - Cloud storage: Provider-specific credentials read from /storage-config volume
  - No external ingress — component is not exposed outside the pod

Service Mesh:
  - No dedicated service mesh configuration
  - Pod-internal communication only; service mesh applies at pod level via ModelMesh Serving
