  - S3 credentials: access_key_id, secret_access_key, endpoint_url, region,

  - GCS credentials: private_key, client_email, type, project_id, token_uri
  - Azure credentials: client_id, client_secret, tenant_id, account_name,
    container_name (or connection_string)
  - HTTP credentials: custom headers, client_certificate, client_key
  - Provisioned by: User/Admin
  - Auto-rotate: No

FIPS Compliance:
- All Go binaries: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1,
  -tags strictfipsruntime
- Dynamic linking to OpenSSL via UBI 9 base image
- Python (TensorFlow/Keras): Inherits FIPS from UBI 9 OpenSSL
- No non-FIPS crypto detected (no MD5, RC4, DES usage)

Path Traversal Protection:
- All storage providers use cyphar/filepath-securejoin v0.2.4
- Prevents model path escape from /models root directory

Network Summary:
- All pod-internal communication: plaintext (acceptable for localhost)
- All external egress: TLS 1.2+ with cloud provider credentials
- No ingress from outside the pod
- No Kubernetes API interaction
- No service mesh participation (pod-internal only)
