RBAC Summary:
- No Kubernetes API interaction
- No ClusterRoles or RoleBindings
- Component runs as a sidecar container with no RBAC requirements
- All authorization is at the cloud storage provider level

Secrets:
- storage-config (Opaque) - Cloud storage credentials
  - Mounted at: /storage-config
  - Contains: Per-storage-backend credential files (JSON)
  - S3: access_key_id, secret_access_key, endpoint_url, region, certificate
  - GCS: private_key, client_email, token_uri
  - Azure: client_id, client_secret, tenant_id OR connection_string
  - HTTP: custom headers, client_certificate, client_key, certificate
  - Provisioned by: User/Admin
  - Auto-Rotate: No

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, -tags strictfipsruntime
- Runtime: Go crypto via OpenSSL (UBI 9 base image)
- TLS: Custom CA bundle support, mTLS client cert support (HTTP provider)
- No non-FIPS crypto detected (no MD5, RC4, DES usage)
- Python (TensorFlow/Keras): Inherits FIPS from UBI 9 OpenSSL

Security Notes:
- All pod-internal communication is plaintext (acceptable for localhost)
- External egress uses TLS 1.2+ with cloud provider authentication
- Path traversal protection via cyphar/filepath-securejoin library
- Model state serialization via per-model goroutine channels (25-item buffer)
- No network policies defined (relies on pod-level isolation)
- No ServiceMesh/mTLS for pod-internal traffic (not needed for localhost)
