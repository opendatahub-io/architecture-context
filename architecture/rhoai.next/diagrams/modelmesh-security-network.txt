RBAC Summary:
  No RBAC defined — ModelMesh is a sidecar service that does not interact
  with the Kubernetes API. RBAC is managed by the modelmesh-serving controller.

Secrets:
  - model-mesh-tls-secret (kubernetes.io/tls)
    Purpose: TLS certificate and private key for external gRPC and inter-instance
    Provisioned by: modelmesh-serving controller / manual
    Auto-Rotate: No

  - model-mesh-icd-etcd (Opaque)
    Purpose: etcd connection configuration (JSON-format connection string)
    Provisioned by: modelmesh-serving controller / manual
    Auto-Rotate: No

  - Trust certs (Opaque)
    Purpose: CA certificates for mTLS client authentication
    Provisioned by: Manual
    Auto-Rotate: No

Authentication & Authorization Summary:
  - gRPC External (8033): Optional mTLS — configurable via MM_TLS_CLIENT_AUTH
    (NONE / REQUIRE / OPTIONAL). TLS termination at Netty/gRPC handler.
  - gRPC Inter-Instance (8033): TLS with shared certificates, mutual auth.
  - gRPC Runtime (8085/UDS): No auth — pod-internal communication only.
  - Metrics (2112): Self-signed TLS, no client auth.
  - Health Probes (8089): Plaintext HTTP, no auth.

FIPS Compliance:
  - Konflux Dockerfile: SunPKCS11 + NSS configured (FIPS-capable when host in FIPS mode)
  - Non-Konflux Dockerfile: Explicitly disables Java FIPS
  - Metrics endpoint self-signed cert (BouncyCastle): Outside FIPS boundary
  - Netty OpenSSL: Prefers native OpenSSL (FIPS-capable) over JDK crypto

Network Policy:
  - Headless Service (ClusterIP: None) — no external ingress
  - NetworkPolicy restricts access per config/base/networkpolicy.yaml
