Headless Service:
- Name: model-mesh
- Type: ClusterIP (clusterIP: None)
- Port: 8033/TCP → grpc (8033)
- Protocol: gRPC/HTTP2
- Purpose: Service discovery for ModelMesh instances via DNS SRV records

RBAC Summary:
- ModelMesh does not define its own RBAC resources
- RBAC is managed by the modelmesh-serving controller
- No ClusterRoles, Roles, or RoleBindings are defined

Secrets:
- model-mesh-tls-secret (kubernetes.io/tls) - TLS cert and private key for gRPC
  server and internal communication; provisioned manually or via cert-manager; no
  auto-rotation
- model-mesh-icd-etcd (Opaque) - etcd connection configuration (host, port, TLS
  certs); provisioned manually or by platform operator; no auto-rotation

NetworkPolicy:
- Defined in config/base/networkpolicy.yaml
- Restricts access to ModelMesh pod ports
- Prometheus metrics (:2112) restricted by NetworkPolicy rules
- Health probes (:8089) restricted by NetworkPolicy rules

TLS Configuration:
- External gRPC (:8033): Optional TLS via MM_TLS_KEY_CERT_PATH / MM_TLS_PRIVATE_KEY_PATH
- Client Auth: Optional mTLS via MM_TLS_CLIENT_AUTH (REQUIRE/OPTIONAL)
- Trust Store: MM_TLS_TRUST_CERT_PATH for client cert validation
- Inter-instance (:8080): Optional TLS via MM_ENABLE_SSL
- Inter-instance mTLS: Optional via MM_INTRACLUSTER_CLIENTAUTH (default on when TLS enabled)
- Prometheus (:2112): Self-signed TLS cert generated via BouncyCastle at startup
- Runtime comm (:8085/UDS): Always plaintext (pod-local only)
