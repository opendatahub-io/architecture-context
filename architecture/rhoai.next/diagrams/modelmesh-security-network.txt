- model-mesh-tls-secret (kubernetes.io/tls)

  - Auto-Rotate: No

- model-mesh-icd-etcd (Opaque)
  - etcd connection configuration (JSON-format connection string)
  - Provisioned by: modelmesh-serving controller / manual
  - Auto-Rotate: No

- Trust certs (Opaque)
  - CA certificates for mTLS client authentication
  - Provisioned by: Manual
  - Auto-Rotate: No

RBAC:
- No Kubernetes RBAC defined — ModelMesh is a sidecar service that does not
  interact with the Kubernetes API. RBAC is managed by the modelmesh-serving controller.

FIPS Compliance:
- Konflux Dockerfile: SunPKCS11 with NSS configured (FIPS-capable)
- Non-Konflux Dockerfile: Java FIPS explicitly disabled (security.useSystemPropertiesFile=false)
- Metrics endpoint: Self-signed cert via BouncyCastle — operates outside FIPS boundary
- TLS: Netty OpenSSL (prefers native OpenSSL, FIPS-capable) with JDK fallback

Network Policy:
- Defined in config/base/networkpolicy.yaml
- Headless Service (ClusterIP: None) for pod-to-pod discovery
