Secrets:
- model-mesh-tls-secret (kubernetes.io/tls)
  - Purpose: TLS private key and certificate for external gRPC and inter-pod
  - Format: PKCS8 required
  - Provisioned by: Manual or cert-manager
  - Rotation: Manual (see docs/configuration/tls.md)
  - Used by: ModelMeshApi (8033/TCP), Litelinks (8080/TCP)

- model-mesh-icd-etcd (Opaque)
  - Purpose: etcd connection configuration (host, port, credentials)
  - Provisioned by: Manual
  - Rotation: No auto-rotate

RBAC Summary:
- No RBAC manifests defined
- ModelMesh is a service, not a Kubernetes operator
- Does not interact with the Kubernetes API server
- RBAC for the parent controller (modelmesh-serving) is in a separate repository

FIPS Compliance Warning:
- Java FIPS mode is EXPLICITLY DISABLED: -Dcom.redhat.fips=false
- Reason: "disable java FIPS which breaks when TLS is enabled" (start.sh:394)
- SunPKCS11 provider reconfigured in Dockerfile
- security.useSystemPropertiesFile=false (disables FIPS system properties)
- BouncyCastle (not FIPS-validated) used for self-signed cert generation
- JVM does NOT operate in FIPS mode despite FIPS-enabled UBI base image
- IMPACT: Gap for FIPS-mandated deployments

NetworkPolicy (from manifests):
- Allows ingress to gRPC port 8033/TCP
- Allows ingress to metrics port 2112/TCP from all sources
- Allows ingress to health probe port 8089/TCP
- Allows inter-pod communication on port 8080/TCP (Thrift/Litelinks)
