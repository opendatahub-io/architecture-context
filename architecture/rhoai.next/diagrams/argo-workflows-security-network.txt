Aggregate Roles:
  - argo-aggregate-to-view: read-only access to Argo CRDs
  - argo-aggregate-to-edit: read-write access to Argo CRDs
  - argo-aggregate-to-admin: full access including workflowtasksets

Secrets:
────────
- argo-workflows-agent-ca-certificates (Opaque) - CA certs for agent TLS, manual, no rotation
- argo-server-tls (kubernetes.io/tls) - TLS cert for argo-server HTTPS, manual/cert-manager
- argo-workflows-webhook-clients (Opaque) - Webhook secrets for GitHub/GitLab/Bitbucket
- argo-server-sso (Opaque) - SSO client ID, secret, redirect URL
- database credentials (Opaque) - PostgreSQL/MySQL username and password
- artifact repository credentials (Opaque) - S3/GCS/Azure credentials

FIPS Compliance:
────────────────
- workflow-controller: FIPS-only build (CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime)
- argoexec: Dual binary (argoexec-fips + argoexec standard)
- Go FIPS module: godebug fips140=on (Go 1.24+ native FIPS 140-3)
- TLS minimum: 1.2 (configurable via TLS_MIN_VERSION)
- Risk: InsecureSkipVerify=true in SSO config, self-signed TLS, agent HTTP client

Authentication Flow:
────────────────────
1. Bearer Token → gatekeeper.go → maps to Kubernetes client → RBAC via K8s API
2. SSO/OIDC → gatekeeper.go + sso.go → validates JWT → maps to SA via RBAC config
3. Webhook → webhook/interceptor.go → HMAC-SHA256 signature verification
