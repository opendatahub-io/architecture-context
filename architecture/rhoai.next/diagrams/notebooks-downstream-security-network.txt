RBAC Summary:
- No ClusterRoles or RoleBindings defined by this repository
- RBAC is managed per-workbench by the ODH Notebook Controller
- kube-rbac-proxy sidecar enforces per-user access via SubjectAccessReview

Container Security:
- All containers run as non-root: UID 1001, GID 0
- Compatible with OpenShift random UID security model
- No privileged capabilities required
- Base images: Red Hat UBI 9 (hardened, CVE-patched)
- RStudio exception: CentOS Stream 9 base (R package availability)

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription for RStudio BuildConfig
  - Provisioned by: Manual
  - Auto-Rotate: No
  - Used only during image build, not at runtime

Authentication Flow:
1. User authenticates via OIDC/OAuth at platform gateway
2. Gateway forwards Bearer Token via HTTPRoute to kube-rbac-proxy
3. kube-rbac-proxy validates token via Kubernetes SubjectAccessReview
4. If authorized, request proxied to workbench container (no auth internally)
5. Workbench images explicitly disable internal auth (--auth none)

Image Registry:
- All images published to: quay.io/modh/
- Image references pinned by SHA256 digest in params.env
- ImageStream tags: N (current) through N-5 (5 versions back)
- Traceability: opendatahub.io/notebook-build-commit annotation
