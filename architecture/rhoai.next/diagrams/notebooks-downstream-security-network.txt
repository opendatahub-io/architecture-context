RBAC Summary:
- No RBAC definitions in this repository
- RBAC is managed by the deploying odh-notebook-controller
- Authentication enforced externally via OAuth + kube-rbac-proxy sidecar
- Per-user namespace isolation policy

Secrets:
- rhel-subscription-secret (Opaque) - RHEL entitlement certificates for RStudio RHEL9 builds
  Provisioned by: Manual / build infrastructure, No auto-rotation
- {{ git_auth_secret }} (Opaque) - Git credentials for Tekton/Konflux pipeline runs
  Provisioned by: Pipelines as Code, Auto-rotation: Yes

Authentication & Authorization:
- JupyterLab (8888/TCP): OAuth/kube-rbac-proxy sidecar, per-user namespace isolation
- Code Server (8585/TCP): OAuth/kube-rbac-proxy sidecar, per-user namespace isolation
- RStudio (8787/TCP): OAuth/kube-rbac-proxy sidecar, per-user namespace isolation
- No authentication implemented within the container images themselves
- All auth enforced at platform level by odh-notebook-controller

FIPS Compliance:
- mongocli binary built with CGO_ENABLED=1 -tags strictfipsruntime (data science images only)
- Python crypto relies on UBI9 OpenSSL → FIPS-capable when host FIPS mode enabled
- No explicit FIPS opt-in for Python processes
- TLS termination at platform ingress, not within workbench containers

Build Hermeticity:
- Builds are NOT hermetic (hermetic: false)
- No Hermeto/cachi2 prefetch configured
- Dependencies pulled from internet at build time
- Python deps locked via Pipfile.lock (codeserver, rstudio) and requirements.txt (jupyter, runtimes)
