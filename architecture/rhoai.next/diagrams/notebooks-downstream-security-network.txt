RBAC Summary:
- No RBAC resources defined in this repository
- These are container images, not operators
- RBAC for running notebooks is managed by odh-notebook-controller
- The controller creates ServiceAccounts and RoleBindings per-notebook

Authentication Architecture:
- Notebook images DISABLE authentication intentionally
  - JupyterLab: --ServerApp.token='' --ServerApp.password=''
  - code-server/RStudio: No auth configured (nginx proxy only)
- Authentication is enforced by sidecar containers:
  - RHOAI 3.x: kube-rbac-proxy on port 8443/TCP (TokenReview + SubjectAccessReview)
  - RHOAI 2.x: oauth-proxy on port 8443/TCP (OpenShift OAuth)
- Sidecar is injected by odh-notebook-controller at pod creation

SECURITY CONCERN:
- If images are deployed WITHOUT odh-notebook-controller (e.g., using
  per-image kustomize manifests in jupyter/*/kustomize/base/), notebooks
  are accessible WITHOUT authentication on port 8888/TCP or 8787/TCP
- Per-image kustomize manifests do NOT include sidecar containers

Secrets:
- No secrets defined in this repository's manifests
- Image pull secrets managed externally by platform operator
- Notebook credentials managed by odh-notebook-controller

FIPS Compliance:
- mongocli binary: Built with CGO_ENABLED=1 -tags strictfipsruntime (FIPS-compliant)
- Python runtime: Inherits OpenSSL from UBI9 base image (FIPS-capable when FIPS mode enabled)
- JupyterLab: Runs plaintext HTTP on 8888/TCP (no custom TLS configuration)
- No Konflux Dockerfiles: check-payload validation applies to downstream build system

Container Security:
- UID: 1001 (non-root)
- GID: 0 (root group for OpenShift compatibility)
- Base images: UBI9 (registry.access.redhat.com/ubi9/python-3XX) or C9S (quay.io/sclorg/python-311-c9s)
- Dependencies locked: Pipfile.lock for Python, go.sum for Go utilities
- RPM versions: Specified via Dockerfile ARG/ENV but not fully locked (no rpms.lock.yaml)
