RBAC Summary:
  None defined by this component.
  RBAC for workbench pods is managed by the ODH Notebook Controller.
  Workbench pods run as UID 1001 (non-root) with GID 0 for
  OpenShift random UID compatibility.

Secrets:
  - rhel-subscription-secret (Opaque)
    Purpose: RHEL subscription credentials for entitled RStudio RHEL9 builds
    Mounted at: /opt/app-root/src/.sec
    Provisioned by: Manual (build environment)
    Auto-Rotate: No

  - git_auth_secret (kubernetes.io/basic-auth)
    Purpose: Git authentication for Konflux Tekton pipelines
    Provisioned by: Konflux / Pipelines as Code
    Auto-Rotate: Yes

Authentication Model:
  - All workbench images DISABLE built-in authentication
    (e.g., --ServerApp.token='' for JupyterLab)
  - Authentication enforced EXTERNALLY by ODH Notebook Controller:
    - RHOAI 3.x: kube-rbac-proxy sidecar
    - RHOAI 2.x: oauth-proxy sidecar
  - Enforcement point: Sidecar container intercepts traffic before
    reaching the workbench container

FIPS Compliance:
  - Images inherit FIPS from UBI 9 base (system OpenSSL is FIPS-validated)
  - No custom TLS configuration — workbenches listen on plaintext HTTP
  - TLS terminated at platform ingress layer
  - No custom cryptographic code introduced by Dockerfiles
  - RStudio installs compat-openssl11 but links to system OpenSSL

Build Hermeticity:
  - Python deps: LOCKED (Pipfile.lock — 30 files)
  - OS packages (RPM): NOT LOCKED (dnf/yum pulls latest)
  - Go deps: LOCKED (go.sum — 2 files, build tools only)
  - No Hermeto/cachi2 prefetch integration
  - ⚠️ Hermeticity gap at OS package layer on main branch
