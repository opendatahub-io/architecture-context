RBAC Summary:
- No ClusterRoles or RoleBindings defined by this component
- RBAC for workbench pods is managed by the ODH Notebook Controller
- Workbench pods run as UID 1001 (non-root) with OpenShift random UID support (GID 0)

Authentication Model:
- CRITICAL: All workbench images DISABLE built-in authentication
  - JupyterLab: --ServerApp.token='' (empty token)
  - RStudio: No password authentication
  - code-server: No password authentication
- Authentication is EXTERNALLY enforced by ODH Notebook Controller sidecar:
  - RHOAI 3.x: kube-rbac-proxy sidecar
  - RHOAI 2.x: oauth-proxy sidecar
- WARNING: These images must NEVER be deployed without the Notebook Controller's
  sidecar injection - direct exposure would be UNAUTHENTICATED

FIPS Compliance:
- Images inherit FIPS from UBI 9 base (Python links against system OpenSSL)
- No custom TLS: workbenches listen on plaintext HTTP
- TLS termination handled by platform ingress layer
- RStudio installs compat-openssl11 (C9S variant)

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription for entitled builds, manual, no rotation
- git_auth_secret (kubernetes.io/basic-auth) - Git auth for Konflux pipelines, auto-provisioned

Container Security:
- Base OS: Red Hat UBI 9 (or CentOS Stream 9 for RStudio C9S)
- User: Non-root (UID 1001), supports OpenShift random UID via GID 0
- Python deps: Locked via Pipfile.lock (hermetic)
- OS packages: NOT locked (non-hermetic dnf/yum installs)
- No Hermeto/cachi2 prefetch integration on main branch
