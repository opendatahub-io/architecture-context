RBAC Summary:
- No ClusterRoles or RoleBindings defined by this repository
- RBAC is managed per-workbench by the ODH Notebook Controller
- Each workbench pod gets its own ServiceAccount in the user's namespace
- kube-rbac-proxy sidecar enforces SubjectAccessReview for user access

Security Posture:
- All containers run as non-root (UID 1001, GID 0)
- Supports OpenShift's random UID security model
- No internal authentication — all auth delegated to sidecar proxy
- No TLS termination in workbench containers — TLS handled by platform ingress
- Base images: Red Hat UBI 9 (RHEL-based, FIPS-validated crypto)
- CUDA/ROCm drivers installed from vendor repositories

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription for RStudio BuildConfig builds, Manual provisioning, No auto-rotation

Image Registries:
- Source: registry.access.redhat.com (UBI9 base images)
- Source: quay.io/sclorg (CentOS Stream 9 for RStudio)
- Source: developer.download.nvidia.com (CUDA toolkit)
- Destination: quay.io/modh/ (published workbench images)
