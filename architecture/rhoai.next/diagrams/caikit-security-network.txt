RBAC Summary:
- No Kubernetes RBAC defined (Python library, not an operator)
- RBAC is managed by the platform operator (RHOAI)
- In RHOAI: kube-rbac-proxy sidecar enforces Bearer Token auth

Secrets:
- TLS server key/cert (file path reference) - gRPC/HTTP TLS termination, external provisioning, no auto-rotate
- TLS client CA cert (file path reference) - mTLS client verification, external provisioning, no auto-rotate
- S3 credentials (config/environment) - S3 storage authentication, external provisioning, no auto-rotate
- OTEL TLS certs (file path reference) - Trace exporter mTLS, external provisioning, no auto-rotate

Authentication & Authorization:
- Transport layer: Optional mTLS at caikit runtime level (validates client cert against CA)
- Application layer: None at caikit level
- Platform layer (RHOAI): kube-rbac-proxy sidecar enforces Bearer Token / Kubernetes RBAC
- Note: insecure_verify field available in remote connection config (InsecureSkipVerify equivalent)

FIPS Compliance:
- Pure Python library; FIPS depends on downstream container image (UBI-based)
- Crypto: Python ssl module (delegates to system OpenSSL), grpcio (BoringSSL internally)
- Non-crypto hash: reversible_hasher.py uses hashlib for training ID obfuscation (not security-sensitive)
- No compiled binaries in this repository

Build Hermeticity:
- No lock files (pip, rpms.lock.yaml, artifacts.lock.yaml) on main branch
- No Dockerfile in this repository
- Downstream RHOAI release branches add lock files in container build repos
