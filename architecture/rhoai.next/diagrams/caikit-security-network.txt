- Auto-Rotate: No (externally managed)

- CA cert:      runtime.trace.tls.ca
- Client key:   runtime.trace.tls.client_key
- Client cert:  runtime.trace.tls.client_cert
- Provisioned by: External

RBAC Summary:
- No Kubernetes RBAC resources defined in this repository.
- Caikit is a Python library; RBAC is managed by the deploying operator
  (e.g., KServe, ODH operator).

Authentication & Authorization:
- All gRPC/HTTP endpoints: No built-in auth
- Auth is fully delegated to platform layer:
  - kube-rbac-proxy sidecar
  - Istio mTLS + AuthorizationPolicy
  - KServe auth mechanisms
- ModelMesh Unix socket: Pod-local only (Unix socket file permissions)

FIPS Compliance Notes:
- Pure Python library — no container builds in this repo
- Uses Python ssl module via grpcio and requests (OpenSSL internally)
- FIPS compliance depends on consuming container image's UBI base + OpenSSL
- No MD5, RC4, or other non-FIPS algorithms detected
- Health probe verify=False is a controlled mTLS detection workaround,
  not a production data path

Security Risks:
- No built-in authentication — any deployment without platform-level auth
  is completely open
- Health probe uses verify=False for mTLS readiness detection
  (SSLError treated as passing health check — controlled workaround)
- No lock file for dependency pinning (pyproject.toml uses version ranges)
