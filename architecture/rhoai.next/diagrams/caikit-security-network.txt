RBAC Summary:
  No Kubernetes RBAC defined. Caikit is a Python runtime; RBAC is managed by
  the platform operator and KServe. ServiceAccount permissions are inherited
  from the KServe/ModelMesh deployment configuration.

TLS Configuration:
  - Server TLS: Configurable via runtime config (use_tls: true/false)
    - Key file: tls.key (runtime.tls.server.key config path)
    - Cert file: tls.crt (runtime.tls.server.cert config path)
    - Client CA: ca.crt (runtime.tls.server.ca config path, enables mTLS)
  - gRPC: Uses grpc.ssl_server_credentials with require_client_auth
  - HTTP: Uses Python ssl.SSLContext with CERT_REQUIRED when CA configured
  - Both protocols support: plaintext, TLS, mTLS modes

Secrets:
  - TLS server key (kubernetes.io/tls) - gRPC/HTTP server TLS termination
    Provisioned by: Platform operator / cert-manager
    Auto-rotate: No (file-based)
  - TLS server cert (kubernetes.io/tls) - gRPC/HTTP server certificate
    Provisioned by: Platform operator / cert-manager
    Auto-rotate: No (file-based)
  - TLS client CA cert (kubernetes.io/tls) - Client cert verification for mTLS
    Provisioned by: Platform operator
    Auto-rotate: No (file-based)
  - S3 credentials (Opaque) - accessKey, secretKey, IAM_id, IAM_api_key
    Provisioned by: User / platform
    Auto-rotate: No
  - OTEL TLS certs (Opaque) - CA, client key, client cert for trace exporter
    Provisioned by: Platform operator
    Auto-rotate: No

Authentication & Authorization:
  - gRPC Server: Mutual TLS (configurable)
    Enforcement: grpc.ssl_server_credentials with require_client_auth
    Policy: Client cert required when client CA configured
  - HTTP Server: Mutual TLS (configurable)
    Enforcement: Python ssl.SSLContext with CERT_REQUIRED
    Policy: Client cert required when client CA configured
  - /health endpoint: No auth (always accessible for probes)
  - ModelMesh Unix Socket: No auth (pod-local only, namespace isolation)

Service Mesh Configuration:
  Not directly managed by Caikit. When deployed within KServe/ModelMesh pods,
  service mesh (Istio/OSSM) configuration is managed by the platform operator.
  Caikit's own TLS can be layered with service mesh mTLS.
