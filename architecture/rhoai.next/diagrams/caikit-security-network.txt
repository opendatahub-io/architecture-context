RBAC Summary:
  None. Caikit is a Python library/runtime, not a Kubernetes operator.
  RBAC is managed by the platform operator (RHOAI).

Authentication & Authorization:
  - Transport-layer: Optional TLS/mTLS (configurable via YAML config)
    - Server TLS: cert/key file paths (grpcio + uvicorn)
    - Client mTLS: CA cert file path for client certificate verification
  - Application-layer: None in caikit itself
    - In RHOAI: kube-rbac-proxy sidecar enforces Bearer Token / K8s RBAC
  - insecure_verify option: Available in remote client config (InsecureSkipVerify equivalent)

Secrets:
  - TLS server key/cert (file path reference) - gRPC/HTTP server TLS termination
    - Provisioned by: External (cert-manager or manual)
    - Auto-rotate: No (file-based, requires restart)
  - TLS client CA cert (file path reference) - mTLS client verification
    - Provisioned by: External (cert-manager or manual)
    - Auto-rotate: No (file-based)
  - S3 credentials (config/environment) - S3 training data authentication
    - Provisioned by: External
    - Auto-rotate: No
  - OTEL TLS certs (file path reference) - Trace exporter mTLS
    - Provisioned by: External
    - Auto-rotate: No

FIPS Compliance:
  - Pure Python library, no compiled binaries in this repo
  - TLS via Python ssl module (delegates to system OpenSSL)
  - gRPC uses BoringSSL internally (C-extension)
  - FIPS compliance depends on downstream UBI-based container image
  - Non-cryptographic hashing: reversible_hasher.py uses hashlib for training ID obfuscation (not security-sensitive)
