RBAC Summary:
  None. Caikit is a Python library/runtime, not a Kubernetes operator.
  RBAC is managed by the platform operator (RHOAI).
  In RHOAI deployments, kube-rbac-proxy sidecar enforces platform RBAC.

Secrets:
  - TLS server key/cert (file path) - gRPC and HTTP server TLS termination
    Provisioned by: External (cert-manager or manual)
    Auto-Rotate: No (file-based, requires restart)
  - TLS client CA cert (file path) - mTLS client certificate verification
    Provisioned by: External (cert-manager or manual)
    Auto-Rotate: No (file-based)
  - S3 credentials (config/environment) - S3 training data source auth
    Provisioned by: External
    Auto-Rotate: No
  - OTEL TLS certs (file path) - Trace exporter mTLS to collector
    Provisioned by: External
    Auto-Rotate: No

Authentication & Authorization:
  - Transport-level: Optional mTLS on all gRPC/HTTP endpoints
    - Server validates client certs against configured CA
    - Enforcement: caikit runtime TLS config
  - Application-level (RHOAI only): Bearer Token via kube-rbac-proxy sidecar
    - Enforcement: Platform sidecar (external to caikit)
  - Caikit itself does NOT implement application-level auth/authz

FIPS Compliance:
  - Pure Python library; FIPS depends on downstream container (UBI-based)
  - TLS via Python ssl module (delegates to system OpenSSL)
  - gRPC uses BoringSSL internally (via grpcio C-extension)
  - Non-crypto hash usage: reversible_hasher.py (training ID obfuscation only)
  - Risk: ConnectionTlsInfo.insecure_verify flag available (InsecureSkipVerify equivalent)

Build Hermeticity:
  - No Dockerfiles in this repo (upstream SDK only)
  - No lock files on main branch
  - Downstream RHOAI release branches add lock files
