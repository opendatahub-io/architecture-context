- N/A — Caikit is a Python library/runtime, not a Kubernetes operator
- No ClusterRoles, Roles, or RoleBindings defined
- RBAC is managed by the platform operator (KServe/ModelMesh/ODH)

Service Mesh Configuration:
- N/A — Service mesh integration managed by platform operator
- Caikit supports TLS/mTLS at the application level (configurable)
- When deployed in a service mesh, sidecar handles mTLS transparently

Secrets:
- runtime.tls.server.key (TLS private key) — gRPC/HTTP server TLS termination
  Provisioned by: KServe/platform operator, Auto-Rotate: N/A
- runtime.tls.server.cert (TLS certificate) — gRPC/HTTP server TLS certificate
  Provisioned by: KServe/platform operator, Auto-Rotate: N/A
- runtime.tls.client.cert (TLS CA/client cert) — mTLS client authentication
  Provisioned by: KServe/platform operator, Auto-Rotate: N/A
- S3 credentials (HMAC keys or IAM API key) — S3 storage access
  Provisioned by: User, Auto-Rotate: No

TLS Configuration:
- Server TLS: Configurable via runtime.tls.server.key/cert config keys
- Client mTLS: Configurable via runtime.tls.client.cert config key
- Health probe: Supports TLS/mTLS for both gRPC and HTTP probing
- Certificate loading: From file path or inline content
- CA bundle support: For client cert validation
- FIPS: Delegates to system OpenSSL via grpcio/uvicorn (UBI base image)
