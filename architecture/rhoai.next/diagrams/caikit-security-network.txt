RBAC Summary:
- No Kubernetes RBAC — Caikit is a Python library, not a Kubernetes operator.
- Access control is delegated to the consuming deployment:
  - KServe: Uses kube-rbac-proxy or oauth-proxy sidecars
  - Model Mesh: Pod-level RBAC managed by modelmesh-serving operator
  - Platform-level auth enforced externally (not by Caikit itself)

TLS Configuration:
- Server TLS Mode: Configurable via config.yml (runtime.tls section)
  - Modes: None (default), TLS (server cert only), mTLS (mutual TLS)
  - gRPC: ssl_server_credentials with optional require_client_auth
  - HTTP: Uvicorn ssl_certfile/ssl_keyfile with optional ssl_cert_reqs=CERT_REQUIRED
  - Certificate Sources: File paths or inline PEM content
  - Crypto: Python ssl module → OpenSSL (FIPS depends on container image)
- Health Probe TLS: Reuses the same TLS config as the runtime servers

Secrets:
- TLS server key/cert (kubernetes.io/tls) - gRPC and HTTP server TLS
  - Provisioned by: Manual or cert-manager (via consuming deployment)
  - Auto-Rotate: Depends on provisioner
- TLS client CA cert (Opaque) - mTLS client certificate verification
  - Provisioned by: Manual or cert-manager
  - Auto-Rotate: Depends on provisioner
- OTLP TLS CA/key/cert (Opaque) - TLS for OpenTelemetry trace exporter
  - Provisioned by: Manual
  - Auto-Rotate: No

FIPS Compliance:
- Caikit delegates TLS to the Python ssl module (via grpcio and uvicorn)
- FIPS compliance determined by container image (UBI = FIPS-enabled OpenSSL)
- No direct cryptographic operations in the library
- No non-FIPS crypto risks detected (no MD5, RC4, DES, custom RNG)
