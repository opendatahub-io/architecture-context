RBAC Summary:
- No RBAC resources (ClusterRoles, RoleBindings) are defined by this component.
- Container images run as non-root: UID 1001, GID 0 (OpenShift pattern).
- RBAC for operators that consume these images is defined in their respective components
  (Training Operator, KubeRay, Notebooks Controller, Kueue).

Secrets:
- SSH key pair (Opaque) — MPI inter-node auth, provisioned per-job by Trainer v2 controller, no rotation
- HuggingFace token (Opaque) — Model download auth, provisioned by user, no auto-rotation
- AWS credentials (Opaque) — S3 storage access, provisioned by user, no auto-rotation

Security Observations:
1. SSH in OpenMPI images uses StrictModes=no and StrictHostKeyChecking=no
   - Trade-off: Required for OpenShift random UID compatibility
   - Risk: No SSH host key verification between MPI peers
   - Mitigation: SSH key pairs are generated per-job and scoped to pod network

2. Ray cluster communication (GCS:6379, Client:10001, Dashboard:8265) has NO encryption and NO auth
   - Risk: Unencrypted inter-pod communication
   - Mitigation: Cluster-internal only, relies on network policies for isolation

3. NCCL/RCCL AllReduce over InfiniBand RDMA has NO encryption
   - Risk: GPU gradient data transmitted in plaintext
   - Mitigation: Direct GPU memory access, cluster-internal, high-performance requirement

4. Jupyter workbench (8888/TCP) uses HTTP internally
   - Mitigation: TLS terminated at platform ingress, OAuth enforced by Notebooks controller

FIPS Compliance:
- No Go binaries in production images (Python-only)
- OpenSSL inherited from UBI9/AIPCC base images
- cryptography==46.0.5 (OpenSSL-backed, FIPS-compatible)
- System trust store from UBI9 base
- Risk: SSH with disabled StrictModes reduces transport security (see above)

Build-Time Egress (not runtime):
- NVIDIA, AMD, Mellanox repos only accessed during container image build
- RHEL AI PyPI only accessed during build (universal/OpenMPI images)
- These are NOT runtime egress destinations
