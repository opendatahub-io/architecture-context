RBAC Summary:
- No RBAC manifests bundled with the CLI
- CLI operates using the caller's kubeconfig credentials
- Pre-flight RBAC check: SelfSubjectAccessReview before mutating operations
- Reader/Writer interface split:
  - Lint checks: Reader interface only (cannot mutate)
  - Migration actions: Full Client interface (read + write)
  - All other commands: Reader interface only

Authentication Chain:
- CLI flags: --token, --kubeconfig, --server, --certificate-authority,
  --client-certificate, --client-key
- Standard Kubernetes auth chain via kubectl ConfigFlags
- Bearer token reused for TrustyAI Service Route access
