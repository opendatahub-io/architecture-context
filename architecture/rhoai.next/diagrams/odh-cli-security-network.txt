RBAC Summary:
- No ClusterRoles or RoleBindings are installed by the CLI
- CLI relies entirely on the calling user's existing RBAC permissions
- Lint commands enforce read-only at the Go type level (client.Reader)
- Write operations restricted to: deps install, components enable/disable, migrate
- Exit code 4 returned when RBAC is insufficient (graceful degradation)
- Permission errors treated as non-fatal for lint/status (skip check, continue)

Authentication Summary:
- Primary: kubeconfig file (Bearer Token / Client Certificate / OIDC)
- Alternative: --token flag for direct Bearer Token authentication
- Custom CA: --certificate-authority flag for non-standard CAs
- GitHub: No authentication required (public repository access)

FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
- Linking: Dynamic (links against UBI9 OpenSSL)
- Runtime: No direct crypto operations — all TLS via client-go → system OpenSSL
- yq binary: Also built with FIPS flags
- Python helpers: Use system OpenSSL via kubernetes Python client
- No non-FIPS crypto detected (no MD5, RC4, DES imports)

Secrets Handling:
- CLI does not create or manage Secrets
- Reads Secrets only during: backup (--dependencies=true), lint (config validation)
- Backup output containing Secrets should be stored in encrypted locations

Security Mitigations:
- CWE-829: OLM catalog source allow-list restricts installable operators
- CWE-269: System namespace targets rejected during dependency installation
- Reader/Writer type split prevents accidental mutations in lint checks
- 6-level exit code system (0=success, 1=warning, 2=error, 3=validation, 4=auth, 5=connection)
