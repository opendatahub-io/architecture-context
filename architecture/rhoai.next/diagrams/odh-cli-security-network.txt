RBAC Summary:
=============
- No ClusterRoles or RoleBindings installed by this tool
- Operates under the user's existing kubeconfig RBAC context
- Required permissions:
  - READ: 20+ API groups, ~40 resource types (get, list)
  - WRITE: datascienceclusters (patch), subscriptions (create) - migration only
  - SCOPE: Cluster-wide (ClusterQueues, ClusterVersions) + all namespaces (workloads)

Authentication:
===============
- Primary: Bearer Token from kubeconfig
- Alternative: Client Certificate, OIDC
- Kubernetes RBAC enforcement at kube-apiserver
- No service accounts created or managed
- No secrets created or managed

Security Notes:
===============
- FIPS-compliant build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime
- Read-heavy client: lint checks NEVER mutate cluster state
- Write operations limited to migrate and components commands
- Backup strips cluster metadata (uid, resourceVersion, status)
- Secrets read during backup are written to local files (operator responsibility to secure)
- OLM integration degrades gracefully (nil-safe) when OLM unavailable
- No network listeners: CLI tool has no attack surface from inbound connections
- Container image based on ose-cli-rhel9 (Red Hat certified base)

Data Classification:
====================
- Reads: Cluster configuration, workload definitions, operator status
- Writes (local): Sanitized YAML exports (metadata stripped)
- Writes (cluster): OLM Subscriptions, DSC patches (migration only)
- Sensitive data handled: Kubernetes Secrets (read for backup, written to local files)
- No data stored persistently by the tool itself
