- No ClusterRoles/RoleBindings installed by odh-cli
- Operates under user's existing kubeconfig RBAC context
- Requires cluster-admin or equivalent for full lint coverage
- Read access to 25+ API groups across all namespaces
- Write access limited to:
  - DataScienceCluster (PATCH) - component management, migration
  - OLM Subscriptions (CREATE) - RHBOK installation

Security Considerations:
- FIPS compliance: Built with CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime
- No network listeners: CLI tool, no inbound attack surface
- Backup sensitivity: backup command writes Secrets to local filesystem
  - Cluster metadata stripped (uid, resourceVersion, managedFields)
  - Secret values preserved for restore capability
- Elevated K8s client settings: QPS=50, Burst=100 (vs kubectl default 5/100)
- OLM nil-safe: Graceful degradation when OLM unavailable
- Dry-run support: All write operations support --dry-run flag
