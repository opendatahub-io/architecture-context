RBAC Summary:
  MLServer does not define Kubernetes RBAC resources. It runs as a
  container within KServe-managed pods and does not interact with
  the Kubernetes API (except reading namespace from service account
  mount for CloudEvents headers).

Secrets:
  MLServer does not define or consume Kubernetes Secrets directly.
  Model artifacts are mounted via KServe storage initializer.

Authentication & Authorization:
  - Application-level: No authentication on any port (8080, 8081, 8082)
  - Platform-level: kube-rbac-proxy sidecar enforces Bearer Token auth
    on port 8443 before forwarding to application ports
  - Runtime security: Trusted runtime allowlist at
    /etc/mlserver/trusted-runtimes.json prevents arbitrary code execution
  - Production mode restrictions:
    * environment_tarball and environment_path are forbidden
    * Wildcard CORS origins (*) and regex patterns are forbidden
    * Only pre-approved runtime import paths can be loaded

Crypto / FIPS:
  - No TLS on any application port (all plaintext)
  - TLS termination handled by kube-rbac-proxy sidecar
  - hashlib.sha256 used for environment tarball hashing (FIPS-safe via
    OpenSSL on RHEL)
  - OpenTelemetry exporter uses insecure=True (no TLS on trace export)
  - No MD5, RC4, DES, or custom RNG usage detected
  - Base image: AIPCC cpu:3.5.0 (RHEL 9.6 with FIPS-capable OpenSSL)
