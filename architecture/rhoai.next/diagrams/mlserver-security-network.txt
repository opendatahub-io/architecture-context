RBAC Summary:
- No Kubernetes RBAC defined (service workload, not an operator)
- RBAC for the pod is managed by KServe and the platform operator (rhods-operator)
- Reads namespace from ServiceAccount mount (read-only filesystem access)

Application-Level Security:
- Trusted Runtimes Allowlist:
  - File: /etc/mlserver/trusted-runtimes.json (baked into image)
  - Mode: PRODUCTION (when file exists)
  - Validation: Only allowlisted model implementations can be loaded
  - Enforcement: ModelSettings Pydantic validator + implementation property accessor
  - Regex: Strict dotted module paths with capitalized class names

- CORS Policy (Production Mode):
  - Wildcard CORS origins: BLOCKED
  - Regex-based CORS origins: BLOCKED

- Environment Security (Production Mode):
  - environment_tarball parameter: REJECTED (prevents sys.path injection)
  - environment_path parameter: REJECTED (prevents sys.path injection)

Secrets:
- No Kubernetes Secrets consumed directly
- TLS certificates: Not managed (handled by platform ingress layer)
- Model credentials: Managed by KServe storage initializer (external to MLServer)

FIPS Compliance:
- Base image: quay.io/aipcc/base-images/cpu (UBI-based, FIPS-capable OpenSSL)
- Crypto usage: hashlib.sha256 only (delegates to OpenSSL, FIPS-safe on UBI)
- TLS: Not configured (plaintext); TLS at platform ingress layer
- Risk: OTLP exporter uses insecure=True (no TLS to OTel Collector)
