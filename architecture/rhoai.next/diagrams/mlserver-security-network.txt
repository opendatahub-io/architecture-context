RBAC Summary:
- MLServer does NOT define Kubernetes RBAC resources
- Runs as non-root user (UID 1000)
- No ServiceAccount roles or bindings
- No Kubernetes API server access required

Runtime Security:
- PRODUCTION Mode: /etc/mlserver/trusted-runtimes.json
  - Allowed runtimes: LightGBM, ONNX, scikit-learn, XGBoost
  - Blocks: Arbitrary Python imports, wildcard CORS, custom environments
- DEVELOPMENT Mode: No restrictions (all runtimes allowed)
- Security endpoint: /v2/runtimes (REST) / RuntimeSecurity (gRPC)

Authentication & Authorization:
- Application-level: None (all endpoints unauthenticated)
- Platform-level: Enforced by KServe/Istio/kube-rbac-proxy at ingress
- Network isolation: Kubernetes NetworkPolicy (if configured)

Secrets:
- MLServer does NOT directly consume Kubernetes Secrets
- Model credentials and TLS certificates managed by KServe
- No secret volumes mounted by MLServer itself

Container Security:
- Non-root execution: UID 1000
- No privileged capabilities required
- Filesystem: Model artifacts mounted read-only at /mnt/models
- trusted-runtimes.json baked into container image at build time (Konflux)
