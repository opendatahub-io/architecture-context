RBAC Summary:
- None. MLServer runs as a userspace process and does not interact with the
  Kubernetes API. No ClusterRoles, Roles, or RoleBindings are defined.
- Auth is delegated to kube-rbac-proxy sidecar (RHOAI 3.x) or oauth-proxy (RHOAI 2.x).

Secrets:
- trusted-runtimes.json (File at /etc/mlserver/)
  - Type: Allowlist file baked into container image
  - Purpose: Restricts which Python runtime classes can be loaded
  - Provisioned by: Container build (Dockerfile.konflux TRUSTED_RUNTIMES build arg)
  - Auto-Rotate: No (immutable in image)

Security Notes:
- MLServer does NOT terminate TLS on any port
- gRPC explicitly uses add_insecure_port() — no option for application-level TLS
- REST uses Uvicorn without ssl_keyfile/ssl_certfile configuration
- All three ports (8080, 8081, 8082) are plaintext by design
- TLS and auth are delegated entirely to the platform layer:
  - kube-rbac-proxy sidecar (RHOAI 3.x) for auth
  - Istio/Envoy sidecar for mTLS (if Service Mesh enabled)
  - KServe networking layer for TLS termination

FIPS Compliance:
- No direct crypto at application level
- AIPCC CPU base image provides OpenSSL from RHEL 9.6
- FIPS mode active when host RHEL kernel has FIPS enabled
- ssl_verify_path available for alibi-explain remote calls (uses system OpenSSL)
