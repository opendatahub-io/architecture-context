Webhook Configuration:
- MutatingWebhookConfiguration:
  - Pod mutating webhook (Ray TLS init container injection)
  - InferenceService mutating webhook (defaults, annotations)
- ValidatingWebhookConfiguration:
  - NIM Account webhook (singleton-per-namespace validation)
  - InferenceGraph webhook (ServiceURL reference validation)
  - InferenceService webhook (storage validation)
- Failure Policy: Fail (fail-closed)
- Namespace Selector: Managed by operator

Secrets:
- odh-model-controller-webhook-cert (kubernetes.io/tls) - Webhook TLS, OpenShift serving cert, auto-rotate
- model-serving-api-tls (kubernetes.io/tls) - API server TLS, OpenShift serving cert, auto-rotate
- ray-ca-tls (Opaque) - Self-signed CA, RSA 4096, 5yr validity, no auto-rotate
- ray-tls (Opaque) - Per-pod Ray certs, signed by CA, rotates on pod IP change
- storage-config (Opaque) - Consolidated S3 credentials, rotates on data connection change
- {account-name}-pull (dockerconfigjson) - nvcr.io pull creds, rotates on API key change

Egress Destinations:
- authn.nvidia.com:443/TCP (HTTPS, TLS 1.2+, Basic Auth) - NIM token acquisition
- api.ngc.nvidia.com:443/TCP (HTTPS, TLS 1.2+, Bearer Token) - NIM catalog/validation
- nvcr.io:443/TCP (HTTPS, TLS 1.2+, Bearer Token) - NIM image manifest verification
- Kubeflow Model Registry (configurable, HTTP/HTTPS, Bearer Token) - Model metadata sync
- OTLP Collector (configurable, gRPC, TLS) - Distributed tracing (model-serving-api only)
