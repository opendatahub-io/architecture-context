- Ingress: NONE
- Egress: NONE (file I/O only, no outbound network)

RBAC Summary:
- NOT APPLICABLE — OpenVINO is a native library, not a Kubernetes workload
- No ServiceAccounts, Roles, ClusterRoles, or RoleBindings
- Access control is enforced by OVMS and the RHOAI platform operator

Secrets:
- NOT APPLICABLE — OpenVINO does not consume or manage secrets
- Model storage credentials are handled by OVMS

FIPS Compliance:
- OpenVINO performs NO cryptographic operations
- Uses protobuf for serialization (not encryption)
- FIPS compliance is enforced at the OVMS container image level
- Build-time: No Konflux Dockerfiles (library, not container)
- Application-level: No TLS, no crypto libraries, no certificate handling
