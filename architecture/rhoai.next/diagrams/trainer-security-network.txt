Secrets:
  - kubeflow-trainer-webhook-cert (kubernetes.io/tls)
    - Purpose: TLS certificate for webhook server (CA + server cert)
    - Provisioned by: cert-controller (OPA), auto-rotated
  - {trainjob-name}-mpi-ssh-auth (Opaque)
    - Purpose: SSH RSA key pair for MPI distributed training
    - Provisioned by: Controller (dynamically per MPI TrainJob)
    - Rotation: None (lifecycle tied to TrainJob)

Security Considerations:
  ⚠ Plaintext HTTP on port 28080/TCP between controller and training pods
    - Mitigated by: NetworkPolicy restricting ingress to controller namespace pods
    - Risk: Metrics data (training progress) traverses pod network unencrypted
  ⚠ golang.org/x/crypto used for SSH key generation (MPI)
    - Mitigated by: strictfipsruntime GOEXPERIMENT forces OpenSSL backend
  ⚠ PodMonitor uses insecureSkipVerify for metrics scraping
    - Mitigated by: Internal cluster traffic only

FIPS Compliance:
  - Build: CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime
  - Base image: registry.access.redhat.com/ubi9/ubi-minimal (OpenSSL included)
  - TLS: stdlib crypto/tls (FIPS via OpenSSL at runtime)
  - HTTP/2 disabled by default (CVE-2023-44487 mitigation)
