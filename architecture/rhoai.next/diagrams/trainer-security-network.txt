Dynamically Created NetworkPolicy:
- Applied to: training pods in user namespaces
- Ingress rules:
  - Allow from controller namespace (opendatahub) pods with matching labels
  - Allow from same-job pods (intra-training-job communication)
- Ports: 28080/TCP (metrics endpoint)
- Purpose: Isolates training pods; restricts metrics access to controller only

Secrets:
- kubeflow-trainer-webhook-cert (kubernetes.io/tls) - Webhook TLS cert, cert-controller auto-rotation
- {trainjob-name}-mpi-ssh-auth (Opaque) - SSH RSA key pair for MPI, dynamically created per MPI TrainJob

Certificate Management:
- Provider: OPA cert-controller library (not cert-manager)
- Rotation: Automatic
- Storage: kubeflow-trainer-webhook-cert Secret
- Mount: /tmp/k8s-webhook-server/serving-certs
- Readiness: Controller reports not-ready until certs generated

Security Considerations:
- ⚠️  Plaintext HTTP (port 28080) between controller and training pods for progression metrics
  - Mitigated by: NetworkPolicy restricting access to controller pods only
  - Risk: Metrics data traverses pod network unencrypted
- ⚠️  golang.org/x/crypto used for SSH key generation (MPI)
  - Mitigated by: strictfipsruntime routes to OpenSSL backend
- ⚠️  PodMonitor uses insecureSkipVerify for metrics scraping (self-signed cert)

FIPS Compliance:
- Build: CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime
- Linking: Dynamic (uses OpenSSL from UBI9 base image)
- TLS: HTTP/2 disabled by default (CVE-2023-44487 mitigation)
