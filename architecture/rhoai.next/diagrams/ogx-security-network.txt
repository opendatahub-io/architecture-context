Authentication & Authorization Summary:

  Authentication (OPTIONAL - disabled by default):
  - OAuth2/OIDC: JWT validation via JWKS or token introspection (RFC 7662)
  - Custom: External endpoint token validation
  - Bearer token extracted from Authorization header
  - Claims mapped to user attributes (roles, teams, projects, namespaces)

  Access Control (OPTIONAL):
  - ABAC with Cedar-like policy language
  - Rules: permit/forbid with when/unless conditions
  - Principals: matched against sub/username JWT claims
  - Actions: create, read, update, delete
  - Resources: typed identifiers (model::my-model, vector_db::*)
  - Default: owners have full access (backward compatibility)

  Rate Limiting (OPTIONAL):
  - Per-client rate limiting via QuotaMiddleware
  - Separate limits for authenticated vs anonymous clients
  - Client ID derived from authentication context

Secrets:
  - API keys (various providers)  [env vars]   - Remote provider auth         - Manual - No rotation
  - TLS key/cert files            [PEM files]  - HTTPS server encryption      - Manual / cert-manager
  - TLS CA file                   [PEM file]   - mTLS client verification     - Manual / cert-manager
  - OAuth2 client credentials     [YAML config] - Token introspection auth    - Manual - No rotation

FIPS Compliance:
  - No Konflux Dockerfiles in repository
  - Python ssl module with ssl.create_default_context() (inherits OS OpenSSL FIPS mode)
  - python-jose for JWT validation (ecdsa/rsa/pyasn1)
  - hashlib.md5 used with usedforsecurity=False (non-security: UUID generation)
  - FIPS compliance depends on container base image and build pipeline
