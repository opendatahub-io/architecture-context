  DSPAs, RayClusters, Training jobs, FeatureStores, MLflow, ModelRegistries)

INGRESS ARCHITECTURE (IN TRANSITION):
- Gateway API: data-science-gateway Gateway in platform namespace
  HTTPRoutes cross-namespace to user Services via ReferenceGrants
  Used by: KServe, Kubeflow Notebooks, KubeRay, Model Registry, MLflow, Feast
- OpenShift Routes: Legacy per-workload Routes in user namespaces
  Used by: CodeFlare (RayCluster Dashboard), odh-deployer, MaaS

AUTHENTICATION MECHANISMS (6 COEXISTING):
1. kube-rbac-proxy sidecar (dominant RHOAI 3.x pattern)
   - Ports: 8443/8444, TokenReview + SubjectAccessReview
   - Components: TrustyAI, Model Registry, Kubeflow Notebooks, Dashboard,
     KServe, OpenVINO, vllm-spyre, kserve-autogluon, Eval Hub
   - FIPS 140-2 cipher suites hardcoded
2. Kuadrant AuthPolicy (next-gen, Gateway API)
   - AuthPolicy CRs target Gateway/HTTPRoute, Authorino token validation
   - Components: odh-model-controller, KServe LLMInferenceService, MaaS
3. OAuth proxy sidecar (legacy RHOAI 2.x)
   - OpenShift OAuth, SAR for pods:get
   - Components: CodeFlare (RayCluster Dashboard)
4. Kubernetes RBAC (application-level)
   - SelfSubjectAccessReview, aggregate ClusterRoles
   - Components: MLflow, Feast
5. mTLS (mutual TLS)
   - Components: KubeRay (cert-manager), CodeFlare (openssl), ModelMesh (etcd),
     KServe (Istio)
6. HMAC-SHA256 JWT (internal)
   - Feast operator-to-registry internal communication
