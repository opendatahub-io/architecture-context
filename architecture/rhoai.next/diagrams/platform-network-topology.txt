NAMESPACES:
- redhat-ods-operator:      Operator namespace (rhods-operator)
- redhat-ods-applications:  Platform controllers and operators (16+ operators)
- redhat-ods-monitoring:    Prometheus, Thanos, Tempo, OTEL, Perses
- istio-system / ingress:   Envoy Gateway, data-science-gateway, kube-auth-proxy
- User namespaces:          Notebooks, InferenceServices, Pipelines, RayClusters,
                            TrainJobs, FeatureStores, ModelRegistries, MLflow

GATEWAY CONFIGURATION:
- Implementation: Gateway API with Envoy data plane
- Central Gateway CR: data-science-gateway
- Auth Proxy: kube-auth-proxy (OIDC/OAuth2), 3 modes:
  * Integrated OpenShift OAuth
  * External OIDC provider
  * Unmanaged (external management)
- HTTPRoutes: Per-component routing rules
- Legacy: OpenShift Routes maintained for backward compatibility

TLS CONFIGURATION:
- Webhook Servers: OLM-managed or service-ca TLS (9443/TCP or 8443/TCP)
- Metrics Endpoints: kube-rbac-proxy with service-ca TLS (8443/TCP)
- Gateway: TLS termination at Envoy, reencrypt to backends
- Ray Inter-Node: mTLS via cert-manager (per-cluster Issuer + Certificate)
- Internal Services: OpenShift service-ca auto-injection

RBAC SUMMARY (Top Privileged ClusterRoles):
- opendatahub-operator-manager-role: DSC/DSCI, all component CRDs, namespaces, secrets
- kserve-manager-role: InferenceService, LLMInferenceService, ServingRuntime, LocalModelCache
- kueue-manager-role: ClusterQueue, Workload, ResourceFlavor, AdmissionCheck
- kuberay-operator-role: RayCluster, RayJob, RayService, HTTPRoutes
- dspo-manager-role: DataSciencePipelinesApplication, deployments, secrets, routes
- maas-controller-role: ExternalModel, MaaSModelRef, MaaSAuthPolicy, Tenant

NETWORK POLICIES:
- llama-stack-k8s-operator: Restricts ingress/egress to known patterns
- odh-deployer: Legacy network policies
- trainer: Per-training-job network policies
- modelmesh-serving: Pod-to-pod communication restrictions
