NAMESPACES:
  - redhat-ods-applications: Primary platform namespace (all operators + dashboard)
  - redhat-ods-monitoring: Monitoring stack (Prometheus, Thanos, Tempo, OTEL, Perses)
  - openshift-ingress: Gateway infrastructure (Envoy, kube-auth-proxy, nginx redirect)
  - models-as-a-service: MaaS platform (MaaS CRs, maas-api)
  - kuadrant-system: Rate limiting infrastructure (Authorino, Limitador)
  - istio-system: Service mesh control plane (when enabled)
  - knative-serving: Knative Serving for KServe serverless mode
  - User namespaces: Notebooks, InferenceServices, DSPAs, training jobs

SERVICE MESH CONFIGURATION:
  - PeerAuthentication: STRICT mTLS for mesh-enrolled InferenceService pods
  - DestinationRule: SIMPLE TLS (insecureSkipVerify) for gateway → backend
  - EnvoyFilter: ext_authz + Lua filter at data-science-gateway
  - mTLS (cert-manager): Per-cluster certificates for CodeFlare/Ray clusters
  - mTLS (self-signed): Controller-generated for LLMInferenceService workloads
  - Status: MIXED — not all services are mesh-enrolled

TLS SUMMARY:
  - Webhooks (9443/TCP): cert-manager or OpenShift service-ca
  - Metrics (8443/TCP): kube-rbac-proxy or direct TLS
  - kube-auth-proxy: OpenShift service-ca
  - LLMInferenceService workloads (8000/TCP): Self-signed (controller-generated)
  - DSP pod-to-pod: OpenShift service-ca (PodToPodTLS)
  - MaaS API (8443/TCP): Self-signed RSA-4096 or configured cert
  - fms-guardrails-orchestrator (8443/TCP): Configurable mTLS
  - Ray clusters: cert-manager mTLS
  - vllm-cpu (8000/TCP): Certificate hot-reloading

RBAC SUMMARY (Top Privileged ClusterRoles):
  - opendatahub-operator-manager-role: datascienceclusters, dscinitializations,
    16 component CRDs, services CRDs, namespaces, deployments, routes, oauthclients,
    gateways, httproutes, envoyfilters
  - kserve-manager-role: inferenceservices, servingruntimes, deployments, services,
    httproutes, virtualservices, scaledobjects
  - llmisvc-manager-role: llminferenceservices, inferencepools, inferencemodels,
    httproutes, leaderworkersets, variantautoscalings, destinationrules
  - kueue-manager-role: clusterqueues, localqueues, workloads, admissionchecks,
    resourceflavors, pytorchjobs, tfjobs, rayclusters, rayjobs
  - maas-controller-role: externalmodels, maasmodelrefs, maasauthpolicies,
    maassubscriptions, tenants, httproutes, authpolicies, tokenratelimitpolicies

AUTHENTICATION MECHANISMS:
  1. kube-rbac-proxy (SubjectAccessReview) — Dashboard, DSP, TrustyAI, KServe,
     ModelMesh, rhods-operator metrics endpoints
  2. kube-auth-proxy (OAuth2/OIDC) — All gateway-routed services
  3. API Key (sk-oai-*) — models-as-a-service via Kuadrant/Authorino
  4. Bearer Token (K8s SA) — kube-auth-proxy token validation, KServe EPP
  5. mTLS (cert-manager) — CodeFlare/Ray clusters
  6. mTLS (self-signed) — LLMInferenceService workloads
  7. Platform-delegated (none) — vLLM, TGIS, Caikit, OpenVINO containers
