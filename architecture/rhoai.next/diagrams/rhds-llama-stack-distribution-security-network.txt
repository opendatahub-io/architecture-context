RBAC Summary:
- No RBAC resources defined in this repository
- This is a container image, not a Kubernetes operator
- RBAC for the service account running this pod is managed by the platform operator
- Kubernetes API access requires ServiceAccount token (for LMEval K8s mode only)

Authentication Architecture:
- INBOUND: No authentication at application layer
  - All /v1/* endpoints are unauthenticated
  - Platform ingress (kube-rbac-proxy, Gateway API, or external auth) enforces access control
  - RISK: Misconfigured deployment could expose server without auth
- OUTBOUND: Provider-specific authentication via environment variables
  - Bearer tokens: VLLM_API_TOKEN, AWS_BEARER_TOKEN_BEDROCK, KUBEFLOW_PIPELINES_TOKEN
  - API keys: AZURE_API_KEY, WATSONX_API_KEY, OPENAI_API_KEY, QDRANT_API_KEY,
              BRAVE_SEARCH_API_KEY, TAVILY_SEARCH_API_KEY
  - Credentials: POSTGRES_PASSWORD, PGVECTOR_PASSWORD, AWS_ACCESS_KEY_ID/SECRET_ACCESS_KEY
  - Tokens: MILVUS_TOKEN
  - Certs: FMS_SSL_CERT_PATH, MILVUS_CA_PEM_PATH, MILVUS_CLIENT_PEM_PATH
  - ADC: Google Application Default Credentials (Vertex AI)
