RBAC Summary:
- No RBAC resources. This component is a container image with no Kubernetes
  operator or controller. It defines no ClusterRoles, Roles, or RoleBindings.

Authentication Summary:
- ⚠️  Application-level: NONE. All /v1/* endpoints are unauthenticated.
- Platform-level: Relies on deploying platform's ingress layer
  (kube-rbac-proxy sidecars, Gateway API with auth enforcement).
- Standalone: API is completely open when run outside platform (e.g., podman).

Secrets Summary:
- rhel-ai-private-index-auth/BOT_PAT (Build-time only)
  Type: GitLab PAT
  Purpose: Download wheel releases from internal package registry
  Provisioned by: Manual (Konflux secret)
  Auto-rotate: No

- Runtime secrets (all via environment variables, no rotation mechanism):
  - VLLM_API_TOKEN / VLLM_EMBEDDING_API_TOKEN (vLLM auth)
  - AWS_BEARER_TOKEN_BEDROCK (AWS Bedrock)
  - WATSONX_API_KEY (IBM Watsonx)
  - AZURE_API_KEY (Azure AI)
  - OPENAI_API_KEY (OpenAI)
  - MILVUS_TOKEN (Milvus remote)
  - QDRANT_API_KEY (Qdrant)
  - KUBEFLOW_PIPELINES_TOKEN (Kubeflow Pipelines)
  - BRAVE_SEARCH_API_KEY (Brave Search)
  - TAVILY_SEARCH_API_KEY (Tavily Search)
  - PostgreSQL credentials (username/password via connection URI)
  - pgvector credentials (username/password)

FIPS Compliance:
- Python application; FIPS depends on AIPCC base image's OpenSSL config
- TLS delegated to Python libraries using system OpenSSL
- Base image: UBI9/RHEL 9.6 (AIPCC CPU base)
- No non-FIPS crypto risks detected in application code

Security Warnings:
⚠️  1. No application-level auth on HTTP API (port 8321)
⚠️  2. PostgreSQL connection is plaintext (no TLS visible in config)
⚠️  3. pgvector connection defaults to plaintext
⚠️  4. Broad egress surface (many external services)
⚠️  5. All credentials via env vars with no secrets rotation mechanism
⚠️  6. No build hermeticity (no lock files on main branch)
