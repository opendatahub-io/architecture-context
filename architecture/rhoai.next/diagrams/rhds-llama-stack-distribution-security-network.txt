RBAC Summary:
  No RBAC resources defined in this repository.
  This is a container image, not a Kubernetes operator.
  RBAC for the service account running this pod is managed by the platform operator
  (rhods-operator or opendatahub-operator).

Authentication & Authorization:
  - Application-level auth: NONE
  - All /v1/* endpoints are unauthenticated at the application layer
  - Auth enforcement: Entirely delegated to platform ingress layer
    (kube-rbac-proxy, Gateway API, or external auth proxy)
  - Outbound auth: Per-provider API keys and tokens via environment variables

Secrets (Environment Variables):
  - VLLM_API_TOKEN            - vLLM inference backend auth (Bearer Token)
  - AWS_BEARER_TOKEN_BEDROCK  - AWS Bedrock API auth (12h TTL, auto-rotate)
  - AZURE_API_KEY             - Azure OpenAI API auth
  - WATSONX_API_KEY           - IBM WatsonX API auth
  - OPENAI_API_KEY            - OpenAI API auth
  - POSTGRES_PASSWORD         - PostgreSQL database auth
  - MILVUS_TOKEN              - Milvus vector database auth
  - PGVECTOR_PASSWORD         - pgvector database auth
  - QDRANT_API_KEY            - Qdrant vector database auth
  - BRAVE_SEARCH_API_KEY      - Brave Search API auth
  - TAVILY_SEARCH_API_KEY     - Tavily Search API auth
  - AWS_ACCESS_KEY_ID /
    AWS_SECRET_ACCESS_KEY     - S3 file storage auth
  - KUBEFLOW_PIPELINES_TOKEN  - Kubeflow Pipelines API auth
  - FMS_SSL_CERT_PATH         - TLS certificate path for FMS Orchestrator

FIPS Compliance:
  - Python application on UBI9 with system Python 3.12
  - FIPS compliance inherited from UBI9 base image (FIPS-validated OpenSSL)
  - Python ssl module uses system OpenSSL automatically
  - Konflux build uses FIPS-compliant base image (quay.io/aipcc/base-images/cpu)
  - No non-FIPS crypto operations detected in repository code

Security Considerations:
  - No built-in authentication: Misconfigured deployment could expose unauthenticated API
  - Environment-variable-driven providers: Misconfigured vars silently disable providers
  - PostgreSQL single point of failure: All state in one database instance
  - Numerous outbound credentials: 13+ secrets managed via environment variables
  - In-cluster traffic is plaintext (HTTP 8321/TCP) — relies on network policy
