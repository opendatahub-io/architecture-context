RBAC Summary:
- No RBAC resources defined in this repository
- This is a container image, not a Kubernetes operator
- RBAC for the pod's ServiceAccount is managed by the platform operator
  (rhods-operator or opendatahub-operator)

Authentication Architecture:
- Application Layer: NO authentication — all /v1/* endpoints are unauthenticated
- Ingress Layer: Platform-enforced (kube-rbac-proxy, Gateway API, or external auth proxy)
- SECURITY NOTE: Misconfigured deployment could expose the server without auth
- All secrets/tokens in config are for OUTBOUND calls only (to backends)

- VLLM_API_TOKEN (env var) - vLLM backend auth - Manual/Platform - No rotation
- AWS_BEARER_TOKEN_BEDROCK (env var) - AWS Bedrock auth - AWS IAM - 12h TTL auto-rotate
