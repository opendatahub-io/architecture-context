RBAC Summary:
- No RBAC roles defined by this component
- Runs as a container without direct Kubernetes API access
- TrustyAI LMEval provider may use Kubernetes API (when TRUSTYAI_LMEVAL_USE_K8S=true)
  but RBAC is managed by the platform operator (rhods-operator)

Authentication Architecture:
- Application-level auth: NONE (all /v1/* endpoints unauthenticated)
- Auth enforcement: Delegated entirely to platform ingress
- Platform ingress uses kube-rbac-proxy or OAuth proxy for AuthN/AuthZ
- Egress auth: Per-backend credentials via environment variables (see Secrets below)

Secrets (all Environment Variables):
- VLLM_API_TOKEN              - vLLM inference auth (Bearer Token)
- VLLM_EMBEDDING_API_TOKEN    - vLLM embedding auth (Bearer Token)
- AWS_BEARER_TOKEN_BEDROCK    - AWS Bedrock auth (short-lived, 12h)
- WATSONX_API_KEY             - IBM WatsonX auth
- AZURE_API_KEY               - Azure OpenAI auth
- OPENAI_API_KEY              - OpenAI auth
- POSTGRES_PASSWORD           - PostgreSQL auth
- MILVUS_TOKEN                - Remote Milvus auth
- PGVECTOR_PASSWORD           - pgvector PostgreSQL auth
- QDRANT_API_KEY              - Qdrant auth
- BRAVE_SEARCH_API_KEY        - Brave Search auth
- TAVILY_SEARCH_API_KEY       - Tavily Search auth
- AWS_ACCESS_KEY_ID           - S3 storage auth
- AWS_SECRET_ACCESS_KEY       - S3 storage auth
- KUBEFLOW_PIPELINES_TOKEN    - Kubeflow Pipelines auth
- BOT_PAT (Konflux build only) - GitLab PAT for private wheel downloads

Security Notes:
- No secrets auto-rotation configured for any credential
- PostgreSQL connection is plaintext within the cluster (no TLS)
- All cloud backends use TLS 1.2+ for egress connections
- Provider activation is environment-variable-driven (conditional)
- Container runs on UBI9 Python 3.12 base image
- No network policies defined by this component (managed by platform)
