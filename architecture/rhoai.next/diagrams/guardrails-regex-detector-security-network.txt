Security Concerns:
1. PLAINTEXT HTTP: All traffic including PII data transmitted without encryption
   - Mitigation: Service mesh mTLS or NetworkPolicy enforcement required
2. NO AUTHENTICATION: Any pod in the cluster can call the detection endpoint
   - Mitigation: AuthorizationPolicy restricting to orchestrator SA
3. ROOT CONTAINER: Dockerfile has no USER directive
   - Mitigation: SecurityContext in Deployment manifest (managed externally)
4. REGEX COMPILATION: Custom regex patterns compiled per-request (no caching)
   - Risk: ReDoS (Regular Expression Denial of Service) with malicious patterns
   - Mitigation: Orchestrator should validate/sanitize regex patterns upstream

Network Policy Recommendations:
- Ingress: Allow only from FMS Guardrails Orchestrator namespace/pod selector
- Egress: Deny all (service makes no outbound connections)
