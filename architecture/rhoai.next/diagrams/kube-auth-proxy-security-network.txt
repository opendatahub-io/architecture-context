FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
- Linking: Dynamic (uses OpenSSL via UBI9 FIPS module)
- Base image: UBI9 ubi-minimal (OpenSSL FIPS-validated)
- TLS: configurable MinVersion TLS 1.2, MaxVersion TLS 1.3
- Session crypto: AES-CFB + HMAC-SHA256 (cookie), AES-GCM (cipher option)
- No MD5/RC4/DES usage detected
- InsecureSkipVerify: available as opt-in (not enabled by default)

Secrets:
- (cookie-secret)     Opaque     AES key for session cookies (16/24/32 bytes)    Operator     No auto-rotate
- (client-secret)     Opaque     OAuth2 client secret                            Operator     No auto-rotate
- (tls-cert/key)      TLS        HTTPS listener cert/key                         cert-manager Yes (file watcher)
- (htpasswd-file)     Opaque     Basic auth credentials                          Admin        Yes (file watcher)
- (redis-password)    Opaque     Redis auth password                             Operator     No auto-rotate

RBAC Model:
- This component does NOT install its own RBAC resources
- RBAC is configured by the deploying operator (rhods-operator) per component
- Auth mode: delegates authn to OIDC/OAuth provider, no K8s authz
- RBAC mode: delegates authn via TokenReview and authz via SubjectAccessReview
  to the Kubernetes API server — the requesting user must have appropriate
  RBAC permissions on the target resource
- /metrics endpoint: hardcoded auto-allow for
  system:serviceaccount:openshift-monitoring:prometheus-k8s

Deployment Model:
- Deployed as sidecar container by rhods-operator (not self-deployed)
- Single container image with entrypoint dispatcher
- PROXY_MODE env var selects auth or rbac mode
- No CRDs, no operator, no kustomize manifests
