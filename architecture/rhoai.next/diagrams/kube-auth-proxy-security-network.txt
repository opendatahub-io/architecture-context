RBAC Summary:
=============
- No RBAC manifests in repository (RBAC managed by deploying operator: rhods-operator)
- Required RBAC for K8s TokenReview feature:
  - API Group: authentication.k8s.io
  - Resource: tokenreviews
  - Verbs: create
- Required RBAC for OpenShift OAuth logout:
  - API Group: oauth.openshift.io
  - Resource: oauthaccesstokens
  - Verbs: delete
- Required RBAC for OpenShift user info:
  - API Group: user.openshift.io
  - Resource: users
  - Verbs: get

Secrets:
========
- Cookie secret (Opaque)
  - Purpose: AES-128/192/256 encryption key for session cookies (16/24/32 bytes)
  - Provisioned by: Manual or deploying operator
  - Auto-Rotate: No

- OAuth client credentials (Opaque)
  - Purpose: Client ID and Client Secret for OIDC/OpenShift OAuth
  - Provisioned by: Manual or deploying operator
  - Auto-Rotate: No

- TLS certificate (kubernetes.io/tls)
  - Purpose: TLS key and cert for HTTPS listener
  - Provisioned by: cert-manager or deploying operator
  - Auto-Rotate: Depends on provisioner

- JWT signing key (Opaque)
  - Purpose: Optional JWT signing key (PEM format)
  - Provisioned by: Manual
  - Auto-Rotate: No

- Redis credentials (Opaque)
  - Purpose: Optional Redis username/password for session store
  - Provisioned by: Manual
  - Auto-Rotate: No

Authentication Mechanisms (ordered by middleware chain priority):
================================================================
1. K8s Service Account Token → TokenReview API validation
2. OAuth Bearer Token → JWT verification via JWKS or sha256~ token via OpenShift API
3. Basic Auth → htpasswd file validation
4. Stored Session Cookie → AES-encrypted cookie or Redis lookup

Authorization Policies:
=======================
- Email domain allowlist (--email-domain)
- Authenticated emails file (--authenticated-emails-file)
- Group-based authorization (--allowed-group)
- Trusted IP bypass (--trusted-ip)
- Per-request ext_authz params: allowed_groups, allowed_emails, allowed_email_domains

Non-FIPS Crypto Risks:
======================
- InsecureSkipVerify: Available for upstream, Redis, OpenShift provider (disabled by default)
- InsecureCipherSuites: Parseable via tls.InsecureCipherSuites() but only if explicitly configured
- No use of MD5, RC4, or DES
