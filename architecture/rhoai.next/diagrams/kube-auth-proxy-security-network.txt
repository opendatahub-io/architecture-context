- ServiceAccount: Created per-component by rhods-operator
  - API Group: authentication.k8s.io
    Resources: tokenreviews
    Verbs: create
  - API Group: authorization.k8s.io
    Resources: subjectaccessreviews
    Verbs: create

Hardcoded Authorization (kube-rbac-proxy):
- Subject: system:serviceaccount:openshift-monitoring:prometheus-k8s
  Resource: /metrics (GET)
  Policy: Always Allow (hardcoded in kube-rbac-proxy)

Secrets:
- cookie-secret (Opaque) - AES encryption key for session cookies, provisioned by platform operator, no auto-rotation
- client-secret (Opaque) - OAuth2 client secret for OIDC/OpenShift OAuth, provisioned by IdP admin, no auto-rotation
- tls-cert (kubernetes.io/tls) - TLS cert for kube-rbac-proxy HTTPS listener, cert-manager/self-signed, rotation depends on source
- redis-password (Opaque, optional) - Redis auth credentials, provisioned by platform operator, no auto-rotation
- jwt-signing-key (Opaque, optional) - JWT signing key for token verification, provisioned by platform operator, no auto-rotation

FIPS Compliance:
- Build: CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime -tags strictfipsruntime
- Crypto: Go stdlib crypto via system OpenSSL (UBI 9 Minimal FIPS-validated)
- Algorithms: AES-CFB (cookie), AES-GCM (session tickets), HMAC-SHA256 (signing), SHA-256
- No non-FIPS primitives (MD5, RC4, DES) detected

Encryption Summary:
- Pod-local (auth mode HTTP): No encryption (localhost only, never exposed externally)
- RBAC mode listener: TLS 1.2+ with configurable cipher suites
- K8s API calls: TLS 1.2+ with in-cluster CA
- OIDC/OAuth calls: TLS 1.2+ with system/custom CA bundles
- Redis (optional): TLS optional, data encrypted at rest with AES-GCM
- Session cookies: AES-CFB encrypted, HMAC-SHA256 signed, auto-split >4KB
