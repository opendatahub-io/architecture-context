RBAC Summary:
- No RBAC roles defined in this repository (managed by rhods-operator)
- Required ServiceAccount permissions (provisioned by platform operator):
  - authentication.k8s.io: tokenreviews - create (TokenReview authentication)
  - authorization.k8s.io: subjectaccessreviews - create (SAR authorization, rbac mode)
- Hardcoded authorization rule (kube-rbac-proxy):
  - system:serviceaccount:openshift-monitoring:prometheus-k8s → GET /metrics (always allowed)

Session Security:
- Cookie-based sessions: AES-CFB encryption + HMAC-SHA256 signing
  - Cookie splitting for >4000 byte payloads
  - cookie-secret: 16/24/32-byte AES key
- Redis-based sessions: AES-GCM per-session encryption + ticket system
  - Cookie stores session ID only
  - Distributed locking prevents concurrent refresh races

Secrets:
- cookie-secret (Opaque) - AES encryption key for session cookies, Platform operator, No auto-rotate
- client-secret (Opaque) - OAuth2 client secret for OIDC/OpenShift, Platform operator/IdP admin, No auto-rotate
- tls-cert (kubernetes.io/tls) - TLS cert for kube-rbac-proxy HTTPS, cert-manager/self-signed, Depends on source
- redis-password (Opaque, optional) - Redis auth credentials, Platform operator, No auto-rotate
- jwt-signing-key (Opaque, optional) - JWT signing key for token verification, Platform operator, No auto-rotate

FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
- Linking: Dynamic (CGO_ENABLED=1) - OpenSSL via UBI 9 Minimal
- Crypto: AES-CFB, AES-GCM, HMAC-SHA256, SHA-256 (all FIPS-approved)
- No non-FIPS primitives detected (no MD5, RC4, DES)
- TLS: configurable min version (TLS 1.2+), configurable cipher suites (k8sapiflag)

Certificate Trust:
- System trust store for OIDC/OAuth providers
- Custom CA bundles: --provider-ca-files, --upstream-ca-file
- In-cluster CA: /var/run/secrets/kubernetes.io/serviceaccount/ca.crt
- kube-rbac-proxy: TLS cert hot-reload (filesystem watcher)
