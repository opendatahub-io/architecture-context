Secrets:
- cookie-secret (Opaque) - AES encryption key for session cookies (16/24/32 bytes), provisioned by platform operator, no auto-rotate
- client-secret (Opaque) - OAuth2 client secret for OIDC/OpenShift OAuth, provisioned by platform operator / IdP admin, no auto-rotate
- tls-cert (kubernetes.io/tls) - TLS certificate for kube-rbac-proxy HTTPS listener, provisioned by cert-manager / self-signed
- redis-password (Opaque, optional) - Redis authentication credentials, provisioned by platform operator, no auto-rotate
- jwt-signing-key (Opaque, optional) - JWT signing key for token verification, provisioned by platform operator, no auto-rotate

FIPS Compliance:
- Build: CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime -tags strictfipsruntime
- Crypto: AES-CFB (cookies), AES-GCM (session tickets), HMAC-SHA256 (signing) - all FIPS-approved
- TLS: crypto/tls with system OpenSSL via strictfipsruntime
- Base Image: UBI 9 Minimal (FIPS-validated OpenSSL)
- No non-FIPS crypto detected (no MD5, RC4, DES)

Session Security:
- Cookie store: AES-CFB encryption + HMAC-SHA256 signing, auto-split for >4000 bytes
- Redis store: Per-session AES-GCM encryption, ticket-based (cookie holds session ID only)
- Distributed locking: Redis-based locks prevent concurrent session refresh races
