    Verbs: create

Secrets:
  - cookie-secret (Opaque) - AES-CFB session cookie encryption key, provisioned by operator, no auto-rotate
  - client-secret (Opaque) - OAuth2 client secret for IdP, provisioned by operator, no auto-rotate
  - tls-cert + tls-key (kubernetes.io/tls) - TLS cert for kube-rbac-proxy HTTPS, cert-manager/operator/self-signed
  - jwt_signing_key.pem (File) - Empty placeholder (GCP App Engine compat), embedded at build time

Authentication Modes:
  Auth Mode (kube-auth-proxy):
    1. K8s SA Token → TokenReview API (checked first for M2M)
    2. JWT Bearer Token → OIDC verification
    3. Basic Auth → htpasswd file
    4. Session Cookie → AES-CFB decrypted, optional refresh
    Session: Cookie-based (AES-CFB encrypted, per-session nonce) or Redis-backed
    ext_authz: /oauth2/auth endpoint for Envoy external authorization

  RBAC Mode (kube-rbac-proxy):
    1. Delegated TokenReview → k8s.io/apiserver framework
    2. OIDC JWT → Direct verification
    3. X.509 Client Certificate → CA validation
    Authorization: SubjectAccessReview with configurable ResourceAttributes
    Template rewrites: HTTP request → K8s resource attributes (query params, headers)

  - Build tags: strictfipsruntime

  - Architectures: amd64, arm64, ppc64le, s390x
