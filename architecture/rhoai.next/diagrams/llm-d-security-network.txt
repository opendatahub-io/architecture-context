Security Summary:
================

Container Security:
- Non-root user (uid 2000, vllm) for all model server containers
- OpenShift-compatible security context
- Trivy vulnerability scanning in CI (CRITICAL/HIGH/MEDIUM)
- Multi-stage builds minimize attack surface (runtime stage only)
- Pinned dependency versions via docker/common-versions

Network Security:
- InferencePool is namespace-scoped for security isolation
- Gateway API controls ingress routing
- NIXL RDMA transfers are unencrypted (GPU memory direct - network isolation required)
- ZMQ pub/sub is unencrypted (cluster-internal only)
- Gateway failure mode configurable: FailOpen or FailClose

CI/CD Security:
- DCO sign-off required on all commits
- Fork gate prevents unauthorized builds from external PRs
- Signed commit verification (ci-signed-commits.yaml)
- SARIF upload to GitHub Security tab

Port Summary:

|-------|-----------|--------------------------------------|----------------|
| 80    | HTTP      | Gateway listener (external)          | TLS optional   |
| 8000  | HTTP      | vLLM API / Routing sidecar           | None (internal)|

| 5600  | NIXL/RDMA | KV cache transfer                    | None (RDMA)    |
| 5556  | ZMQ       | Prefix cache events                  | None (internal)|
| 9002  | HTTP      | EPP Prometheus metrics               | None (internal)|
| 9090  | HTTP      | EPP Prometheus metrics (alt)         | None (internal)|

RBAC Summary:
=============
- ServiceAccount: sa (model server pods)
- InferencePool: Requires RBAC to watch/list pods in namespace
- EPP: Requires RBAC for Kubernetes API (pods, CRDs, services)
- Namespace-scoped InferencePool enforces security isolation
