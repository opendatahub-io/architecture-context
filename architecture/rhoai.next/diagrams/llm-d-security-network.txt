Security Posture Notes:
- vLLM API serves PLAINTEXT HTTP (no TLS) - relies on network isolation
- RDMA/NIXL transport is UNENCRYPTED (L2 transport, no app-level encryption)
- No built-in authentication - security layered by deployment environment
- FIPS: No FIPS build flags detected (upstream project)
- Threat model: Work-in-progress per THREAT-MODEL.md
- Gateway auth is provider-dependent (Istio AuthorizationPolicy, etc.)
