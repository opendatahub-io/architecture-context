RBAC Summary:
- ServiceAccount: sa (per-namespace, model server pods)
- ServiceAccount: (guide-specific) for EPP
- InferencePool: Namespace-scoped CRD (security isolation)
- Non-root containers: uid 2000 (vllm user)
- OpenShift: Arbitrary UID compatible

Security Considerations:
- Gateway HTTP (port 80): No TLS by default; TLS depends on gateway provider
- ext-proc gRPC: In-cluster communication (loopback/sidecar)
- Model server: HTTP (no mTLS by default); relies on network policies
- NIXL RDMA: No application-layer encryption; hardware-level isolation
- ZMQ (5556): Unencrypted; in-cluster only
- Container images: Trivy vulnerability scanning (CRITICAL/HIGH/MEDIUM)
- CI/CD: DCO sign-off required, fork gate for external PRs
- Image signing: SARIF upload to GitHub Security tab

Secrets:
- Model storage credentials (HuggingFace token, AWS IAM, etc.)
- Image pull secrets (for private registries)
- Gateway TLS certificates (provider-managed)
- OTEL endpoint configuration
