================================================================================
SECURITY NOTES
================================================================================

1. MODEL SERVER SECURITY POSTURE:
   - Plaintext HTTP on port 8000 with NO authentication or TLS
   - Security enforcement is DELEGATED to Gateway/proxy layer
   - This is an intentional architectural decision for cluster-internal traffic

2. WIDE EXPERT-PARALLELISM ELEVATED PRIVILEGES:
   - LeaderWorkerSet manifests run as root (UID 0)
   - Capabilities: IPC_LOCK, SYS_RAWIO
   - Required for RDMA/NVSHMEM GPU Direct operations
   - Represents a security trade-off for multi-node GPU communication

3. NIXL KV TRANSFER:
   - Port 5600/TCP with no encryption or authentication
   - Transfers KV cache blocks between prefill and decode pods
   - Cluster-internal only, relies on network policies for isolation

4. FIPS COMPLIANCE:
   - No FIPS build flags (no GOEXPERIMENT, CGO_ENABLED, strictfipsruntime)
   - No explicit TLS configuration in model servers
   - Crypto operations delegated to gateway layer
   - Expected for upstream CNCF project; downstream RHOAI would add compliance

5. BUILD HERMETICITY:
   - Partial dependency pinning via uv constraints (PyTorch, NIXL)
   - No full lock files (rpms.lock.yaml, artifacts.lock.yaml absent)
   - Images depend on network access at build time
   - Downstream would add Konflux integration for hermetic builds
