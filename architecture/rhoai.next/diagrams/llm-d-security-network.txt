RBAC Summary:
- Model server pods use minimal RBAC:
  - ServiceAccount with automountServiceAccountToken: false
  - No ClusterRoles defined in deployment guides
  - RBAC delegated to EPP and gateway controllers (sibling repos)
  - Non-root execution (UID 2000)

Authentication & Authorization:
- /v1/* endpoints: No auth (delegated to upstream proxy layer)
- /health endpoint: No auth (liveness probe)
- /metrics endpoint: No auth (cluster-internal, NetworkPolicy if configured)
- Gateway-level auth: Configurable per GatewayClass implementation

Secrets:
- hf-secret (Opaque) — HuggingFace API token for model downloads
  - Provisioned: Manual (kubectl create secret)
  - Auto-Rotate: No
- AWS sccache credentials (Docker build secret) — S3 for sccache (build-time only)
  - Provisioned: AWS IAM role
  - Auto-Rotate: No

Container Security:
- Base images: NVIDIA CUDA (UBI9), Ubuntu, AMD ROCm, Intel Habana
- Running as: non-root (UID 2000)
- ServiceAccount token: disabled (automountServiceAccountToken: false)
- FIPS: Depends on base image (UBI9 provides FIPS-capable OpenSSL)
- No custom cryptographic code in llm-d

RHOAI Distribution Security Gaps:
- No AIPCC base images (required for RHOAI product security)
- No kube-rbac-proxy sidecar (used by other RHOAI components)
- No Konflux Dockerfiles
- No hermetic build support (rpms.lock.yaml, cachi2/hermeto)
- No comprehensive Python lock files (partial constraints.txt only)
