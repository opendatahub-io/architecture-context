RBAC Summary:
  No RBAC resources defined in this repository.
  RBAC is managed by KServe and the RHOAI platform operator.
  The container runs under whatever ServiceAccount KServe assigns to the
  InferenceService pod.

Secrets:
  No secrets defined in this repository.
  Model storage credentials and TLS certificates are managed by KServe
  and the platform infrastructure.

Container Security:
  - Runtime user: UID 2000 (non-root)
  - Base image: registry.redhat.io/rhaiis/vllm-spyre-rhel9:3.2.2
    (Red Hat certified image)
  - Package sources: Inherited from RHAIIS base image (Red Hat repos)

Authentication & Authorization:
  - /v1/* HTTP endpoints: Auth delegated to platform
    (kube-rbac-proxy or Gateway API auth)
  - TGIS gRPC endpoint: Auth delegated to platform (KServe)
  - Enforcement point: KServe / RHOAI Gateway (not this component)

Network Policy:
  Not defined by this component. Network policies are expected to be
  managed by KServe, the RHOAI operator, or cluster-level policies.

Architectural Notes:
  - This container exposes NO authentication or TLS of its own
  - ALL security is delegated to the platform layer (KServe, RHOAI Gateway)
  - The container runs in a KServe-managed pod with platform-provided
    service mesh, network policies, and ingress controls
  - Dual-protocol (HTTP + gRPC) runs in a single process via uvicorn ASGI
  - DISABLE_LOGPROBS_DURING_SPEC_DECODING=false overrides upstream vLLM
    to maintain API contract consistency during speculative decoding
