RBAC Summary:
  No RBAC resources defined in this repository.
  RBAC is managed by KServe and the platform operator (rhods-operator).
  - Container runs as non-root user (UID 2000)
  - SCC: restricted-v2 (standard OpenShift non-root SCC)
  - ServiceAccount: Created by KServe per InferenceService

Secrets:
  No secrets defined in this repository.
  - Model storage credentials injected by KServe via:
    - ServiceAccount annotations (IRSA for S3)
    - Secret references in InferenceService CR spec
  - TLS certificates managed by platform ingress (not by container)

Authentication & Authorization:
  - Container exposes UNAUTHENTICATED endpoints on 8000/TCP and 8033/TCP
  - All AuthN/AuthZ is DELEGATED to platform infrastructure:
    - kube-rbac-proxy sidecars (older RHOAI versions)
    - Gateway API with EnvoyFilter (newer RHOAI versions)
    - OAuth proxy (alternative configuration)
  - IMPORTANT: Container MUST NOT be exposed directly without platform
    ingress providing authentication

FIPS Compliance:
  - Base image: RHAIIS vllm-spyre-rhel9:3.2.2 (UBI9, FIPS-compliant)
  - Crypto: System OpenSSL from UBI9 base image
  - TLS: Inherited from Python ssl module backed by system OpenSSL
  - No custom crypto configuration in this repository

Build Security:
  - Single-layer Dockerfile (ENV/LABEL/USER/ENTRYPOINT only)
  - No package installation (hermetic by design)
  - All dependencies inherited from RHAIIS base image
  - Konflux build pipeline (Dockerfile.konflux.spyre)
