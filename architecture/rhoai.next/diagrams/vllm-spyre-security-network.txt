RBAC Summary:
- No RBAC resources defined in this repository
- RBAC is managed by KServe and the platform operator (rhods-operator)
- Container runs as non-root UID 2000
- Platform enforces access control at ingress layer

Authentication & Authorization Summary:
- Container Level: NONE — all endpoints are unauthenticated
- Platform Level: Auth enforced by KServe ingress stack
  - Option 1: kube-rbac-proxy sidecar
  - Option 2: Gateway API with EnvoyFilter
  - Option 3: OAuth proxy
- Trust Model: Container trusts all in-cluster traffic; security boundary is at platform ingress

Secrets:
- No secrets defined in this repository
- Model storage credentials injected by KServe via:
  - ServiceAccount annotations (for AWS IAM/IRSA)
  - Secret references in InferenceService CR
  - Environment variables from platform configuration

FIPS Compliance:
- Build: No Go build — Python entrypoint on UBI9 base
- Crypto: Inherited from RHAIIS base image (system OpenSSL on UBI9)
- TLS: System trust store from UBI9; TLS config inherited from base
- Assessment: FIPS posture depends entirely on RHAIIS base image

- Non-root execution: UID 2000
- Base image: registry.redhat.io/rhaiis/vllm-spyre-rhel9:3.2.2 (Red Hat signed)
- No package installation in Dockerfile (hermetic by delegation)
- SCC compatible: restricted-v2 (non-root, no capabilities)
