RBAC Summary:
  No RBAC resources defined in this repository.
  This is a container image, not an operator.
  RBAC is managed by KServe and the platform operator (rhods-operator).

Authentication & Authorization:
  - HTTP API (8000/TCP): No auth at container level
  - gRPC API (8033/TCP): No auth at container level
  - Enforcement Point: KServe / platform ingress
    (kube-rbac-proxy sidecar, Gateway API with EnvoyFilter, or OAuth proxy)
  - Auth is fully delegated to platform infrastructure

Secrets:
  No secrets defined in this repository.
  Model storage credentials are injected by KServe via:
  - ServiceAccount annotations (e.g., IRSA for AWS)
  - Secret references in InferenceService CR spec

Container Security:
  - Runs as non-root user (UID 2000)
  - Base image: registry.redhat.io/rhaiis/vllm-spyre-rhel9:3.2.2 (RHEL 9 UBI)
  - FIPS compliance: Inherited from RHAIIS base image (UBI9 OpenSSL)
  - No custom TLS configuration in this repository

FIPS Compliance:
  - No Go binaries — Python entrypoint only
  - Crypto libraries: System OpenSSL via Python ssl module (from UBI9 base)
  - Certificate handling: System trust store (UBI9)
  - FIPS posture depends entirely on RHAIIS base image
