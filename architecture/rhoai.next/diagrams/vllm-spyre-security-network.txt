RBAC Summary:
- No RBAC resources defined by this component (container image, not an operator)
- RBAC is managed by KServe and the RHOAI platform operator
- kube-rbac-proxy sidecar enforces SubjectAccessReview for all API access

Authentication & Authorization:
- HTTP API (8000/TCP): No auth at container level — enforced by kube-rbac-proxy sidecar
- gRPC API (8033/TCP): No auth at container level — enforced by kube-rbac-proxy sidecar
- kube-rbac-proxy (8443/TCP): Bearer Token + SubjectAccessReview (platform-managed)

Secrets:
- Model storage credentials: Injected at runtime by KServe via ServingRuntime configuration
- HF_TOKEN (optional): Hugging Face Hub access token for model downloads
- All secrets are externally provisioned — this image defines no secret resources

FIPS Compliance:
- Entirely inherited from RHAIIS vllm-spyre-rhel9 base image
- No compilation, no package installation, no source code in this Dockerfile
- Base image: UBI9/RHEL9-based with system OpenSSL
- FIPS posture determined by base image build flags and crypto libraries

Container Security:
- Non-root user: UID 2000
- OpenShift restricted-v2 SCC compliant
- No privileged capabilities required (beyond IBM Spyre device access)
- Image architectures: x86_64, ppc64le, s390x
