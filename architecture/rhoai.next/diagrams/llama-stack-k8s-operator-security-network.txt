- manager-rolebinding → ClusterRole:manager-role → SA:controller-manager (cluster-wide)
- leader-election-rolebinding → Role:leader-election-role → SA:controller-manager (system ns)
- proxy-rolebinding → ClusterRole:proxy-role → SA:controller-manager (cluster-wide)
- {instance-name}-rolebinding → ClusterRole:system:openshift:scc:anyuid → SA:{instance-name}-sa (per instance)

TLS Certificate Management:
- Webhook TLS: cert-manager (vanilla K8s) or OpenShift service-ca (auto-rotate: Yes)
  Secret: ogx-k8s-operator-webhook-cert (kubernetes.io/tls)
- User TLS: User-provided secrets for managed OGX server TLS (spec.network.tls.secretName)
- Provider credentials: Opaque secrets with API keys, passwords, AWS credentials (user-managed)

CA Trust:
- odh-trusted-ca-bundle ConfigMap auto-detected in operator namespace (optional)
- Custom CA bundles via spec.tls.trust.caCertificates ConfigMap refs
- X.509 PEM validation with 10MB / 1000 cert limits
