NetworkPolicy Configuration:
- Default: Enabled (feature flag: enableNetworkPolicy)
- Allow rules:
  - Same namespace as instance
  - Operator namespace (redhat-ods-applications / opendatahub)
  - Configurable additional namespaces (via CR spec)
  - Configurable label-based selectors

Authentication Endpoints:
- /metrics (8443/TCP): Bearer Token via kube-rbac-proxy → RBAC auth_proxy_role
- {instance}-service (8321/TCP): No authentication (cluster-internal, NetworkPolicy enforced)
- /healthz, /readyz (8081/TCP): No authentication (operator health probes)

CA Bundle Management:
- Sources: User ConfigMap (spec.server.tlsConfig.caBundle) + odh-trusted-ca-bundle (auto-detected)
- Validation: PEM/X.509 parsing, max 10MB, max 1000 certificates
- Output: Managed ConfigMap {instance}-ca-bundle → mounted at /etc/ssl/certs/ca-bundle/ca-bundle.crt
- Env: SSL_CERT_FILE=/etc/ssl/certs/ca-bundle/ca-bundle.crt

FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime,openssl
- OLM annotation: features.operators.openshift.io/fips-compliant: "true"
- Runtime crypto: stdlib crypto/x509 for CA bundle validation only; HTTP client uses plaintext (cluster-internal)

Secrets:
- No operator-managed secrets
- No TLS certificates (operator-to-server communication is plaintext HTTP)
