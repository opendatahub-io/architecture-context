RBAC Summary:
  No RBAC permissions required. The rest-proxy does not interact with the
  Kubernetes API server. It is a stateless reverse-proxy sidecar.

Secrets:
  - TLS cert/key (referenced by env vars):
    - MM_TLS_KEY_CERT_PATH: TLS certificate file path for HTTPS listener
    - MM_TLS_PRIVATE_KEY_PATH: TLS private key file path for HTTPS listener
    - Provisioned by: ModelMesh Serving operator or manual configuration
    - Auto-rotate: Depends on provisioner

Authentication & Authorization:
  - No authentication at the proxy level
  - Auth is handled by the upstream ModelMesh infrastructure
  - The proxy is a transparent pass-through for auth purposes

TLS Configuration:
  - Inbound (REST listener):
    - Enabled when MM_TLS_KEY_CERT_PATH and MM_TLS_PRIVATE_KEY_PATH are set
    - Mode: SIMPLE (server certificate only)
    - tls.Config: default (no MinVersion set explicitly)
  - Outbound (gRPC to ModelMesh):
    - Enabled via REST_PROXY_USE_TLS=true env var
    - InsecureSkipVerify: configurable via REST_PROXY_SKIP_VERIFY (default: false)
    - Uses system trust store for CA certificates

FIPS Compliance:
  - Konflux builds: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
  - Links dynamically against OpenSSL via UBI9 ubi-minimal base
  - Upstream builds: CGO_ENABLED=0 (static, NOT FIPS compliant)
  - No hardcoded non-FIPS cipher suites detected
  - Risk: No explicit MinVersion on tls.Config (mitigated by strictfipsruntime)
  - Risk: REST_PROXY_SKIP_VERIFY can disable cert verification

Multi-Architecture Support:
  - linux/x86_64, linux/ppc64le, linux/s390x, linux/arm64
