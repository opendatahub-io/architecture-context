NO EXTERNAL EGRESS from rest-proxy:
  - All communication is pod-internal (localhost:8033)
  - No outbound internet connections
  - No connections to external services

RBAC Summary:
  - No RBAC resources defined by this component
  - Inherits pod-level ServiceAccount from ModelMesh Serving deployment
  - No Kubernetes API access required at runtime

Secrets:
  - TLS cert/key (kubernetes.io/tls)
    - Purpose: TLS termination on REST listener + TLS for gRPC upstream
    - Referenced via: MM_TLS_KEY_CERT_PATH, MM_TLS_PRIVATE_KEY_PATH env vars
    - Provisioned by: ModelMesh controller / cert-manager
    - Auto-rotate: Depends on provisioner

FIPS Compliance:
  - Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
  - Crypto: stdlib crypto/tls backed by OpenSSL (FIPS-validated via strictfipsruntime)
  - Risk: REST_PROXY_SKIP_VERIFY env var can disable cert verification (mitigated: localhost only)
  - No non-FIPS crypto imports detected (no crypto/md5, crypto/rc4, crypto/des)

Environment Variables (Security-Relevant):
  - REST_PROXY_USE_TLS: Enable TLS on gRPC upstream connection (default: false)
  - REST_PROXY_SKIP_VERIFY: Skip TLS certificate verification on gRPC upstream (default: false)
  - MM_TLS_KEY_CERT_PATH: Path to TLS certificate for HTTPS listener
  - MM_TLS_PRIVATE_KEY_PATH: Path to TLS private key for HTTPS listener
  - REST_PROXY_LISTEN: HTTP/HTTPS listen address (default: :8008)
  - REST_PROXY_GRPC_PORT: Upstream gRPC port (default: 8033)

Attack Surface:
  - HTTP/HTTPS listener on 8008/TCP (exposed within pod/service boundary)
  - No authentication at proxy level (delegated to platform ingress)
  - Custom JSON unmarshaling of tensor data (potential for malformed input)
  - TLS certificate verification can be disabled via env var
