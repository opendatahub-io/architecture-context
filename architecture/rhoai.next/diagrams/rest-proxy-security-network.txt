RBAC Summary:
- None. This component does not interact with the Kubernetes API.
- No ClusterRoles, Roles, or RoleBindings defined.

Service Mesh Configuration:
- N/A - rest-proxy runs as a sidecar within the ModelMesh pod.
- Network isolation relies on pod-level boundary (localhost communication).
- Any service mesh (Istio/OSSM) policies apply at the ModelMesh Service level,
  not at the rest-proxy sidecar level.

Secrets:
- TLS cert/key (via MM_TLS_KEY_CERT_PATH / MM_TLS_PRIVATE_KEY_PATH)
  - Type: kubernetes.io/tls
  - Purpose: TLS termination for the HTTP listener
  - Provisioned By: ModelMesh Serving operator
  - Auto-Rotate: Unknown

Authentication & Authorization:
- No authentication enforced at the proxy layer.
- Security relies on:
  1. Network-level isolation (pod-local gRPC to ModelMesh)
  2. Platform ingress authentication (upstream of rest-proxy)
  3. ModelMesh-level auth (if configured)

FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
- Linking: Dynamic (CGO enabled for OpenSSL)
- Base Image: registry.access.redhat.com/ubi9/ubi-minimal
- Crypto: stdlib crypto/tls backed by OpenSSL (FIPS mode)
- Risk: REST_PROXY_SKIP_VERIFY env var can disable cert verification

Environment Variables:
- REST_PROXY_GRPC_PORT (default: 8033) - upstream gRPC port
- REST_PROXY_LISTEN_PORT (default: 8008) - HTTP listener port
- REST_PROXY_USE_TLS - enable TLS for gRPC upstream
- REST_PROXY_SKIP_VERIFY - skip TLS cert verification (SECURITY RISK)
- MM_TLS_KEY_CERT_PATH - TLS certificate file path
- MM_TLS_PRIVATE_KEY_PATH - TLS private key file path
- MM_SERVING_HOSTNAME - hostname for gRPC connection (default: localhost)
