Authentication:
- kube-rbac-proxy sidecars on all managed services
  - TLS 1.2+ with FIPS-approved cipher suites
  - OAuth via tokenreviews + subjectaccessreviews
  - EvalHub uses SAR with X-Tenant header for multi-tenant scoping
- Conditional auth via annotation: security.opendatahub.io/enable-auth
  (applies to GORCH and NemoGuardrails)

- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamic linking

- kube-rbac-proxy: FIPS-approved cipher suites only
