  - "": services, pods, configmaps, pvcs, secrets, serviceaccounts, events → CRUD
  - route.openshift.io: routes → CRUD
  - serving.kserve.io: inferenceservices → list, watch, get, update, patch, delete
  - networking.istio.io: destinationrules, virtualservices → CRUD
  - monitoring.coreos.com: servicemonitors → list, watch, create
  - rbac.authorization.k8s.io: clusterrolebindings → get, list, watch, create, update, delete

ClusterRole: lmes-manager-role
  - trustyai.opendatahub.io: lmevaljobs → CRUD
  - "": pods, pods/exec, configmaps, secrets, pvcs → CRUD

ClusterRole: gorch-manager-role
  - trustyai.opendatahub.io: guardrailsorchestrators → CRUD
  - serving.kserve.io: servingruntimes, inferenceservices → get, list, watch
  - apps: deployments → CRUD

ClusterRole: nemo-guardrails-manager-role
  - trustyai.opendatahub.io: nemoguardrails → CRUD
  - "": configmaps, services, secrets, serviceaccounts → CRUD
  - route.openshift.io: routes → CRUD

ClusterRole: evalhub-manager-role
  - trustyai.opendatahub.io: evalhubs, evalhubs/proxy, status-events → CRUD
  - batch: jobs → get, list, watch, patch, delete
  - "": namespaces → get, list, watch
  - rbac.authorization.k8s.io: clusterrolebindings, roles, rolebindings → CRUD

ClusterRole: proxy-role
  - authentication.k8s.io: tokenreviews → create
  - authorization.k8s.io: subjectaccessreviews → create

User-Facing Roles:
  - trustyaiservice-editor-role: CRUD on trustyaiservices
  - trustyaiservice-viewer-role: read-only on trustyaiservices
  - lmeval-user-role: CRUD on lmevaljobs (bound to system:authenticated)
  - nemoguardrail-editor-role: CRUD on nemoguardrails
  - nemoguardrail-viewer-role: read-only on nemoguardrails

Authentication Architecture:
  - All user-facing services use kube-rbac-proxy sidecar for SAR-based auth
  - GORCH: up to 3 kube-rbac-proxy instances (orchestrator, gateway, built-in)
  - EvalHub: SAR + X-Tenant header for multi-tenant namespace scoping
  - Operator metrics: Bearer token + metrics-reader ClusterRole

Secrets:
  - {tas-instance}-internal (kubernetes.io/tls) - OpenShift service CA, auto-rotate
  - {tas-instance}-db-credentials (Opaque) - DB connection creds, user-provisioned
  - {tas-instance}-db-ca (Opaque) - DB TLS CA cert, user-provisioned
  - {evalhub-instance}-tls (kubernetes.io/tls) - OpenShift service CA, auto-rotate
  - {gorch-instance}-predictor-serving-cert (kubernetes.io/tls) - KServe auto-discovered, auto-rotate
  - S3 credentials (Opaque, SecretKeySelectors) - AWS creds for LMES, user-provisioned
  - OCI credentials (Opaque, SecretKeySelectors) - Registry auth for LMES, user-provisioned

FIPS Compliance:
  - Build: CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime -tags strictfipsruntime
  - Base image: registry.access.redhat.com/ubi9/ubi-minimal (OpenSSL-backed crypto)
  - TLS: MinVersion TLS 1.2 enforced
  - Risk: InsecureSkipVerify struct field in EvalHub OTEL config (defaulted to false)

Conversion Webhook:
  - conversion.trustyaiservices.trustyai.opendatahub.io
  - Converts TrustyAIService v1alpha1 ↔ v1
