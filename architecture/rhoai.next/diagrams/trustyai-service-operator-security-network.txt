Authentication Pattern:
  All user-facing services use kube-rbac-proxy sidecar pattern:
  1. OpenShift Route terminates external TLS (edge or reencrypt)
  2. kube-rbac-proxy terminates service TLS (service-ca certificates)
  3. kube-rbac-proxy performs SubjectAccessReview against K8s API
  4. Authorized requests forwarded to application on localhost
