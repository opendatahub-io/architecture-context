  Secrets
================================================================================

  feast-{name}-{service}-tls (kubernetes.io/tls)
    - Purpose: TLS certificate for Feast service
    - Provisioned By: OpenShift Service CA (auto)
    - Auto-Rotate: Yes (managed by OpenShift)

  User-provided TLS secret (kubernetes.io/tls)
    - Purpose: Custom TLS certificate
    - Provisioned By: Manual / cert-manager
    - Auto-Rotate: No (user managed)

  Backend store credentials (Opaque)
    - Purpose: Database/cloud credentials for offline/online stores
    - Provisioned By: Manual
    - Auto-Rotate: No (user managed)

  OIDC client secret (Opaque)
    - Purpose: OIDC client credentials for auth
    - Provisioned By: Manual
    - Auto-Rotate: No (user managed)

================================================================================
  Authentication & Authorization
================================================================================

  Registry REST API (/api/v1/*):
    Methods: GET, POST, DELETE
    Auth: Bearer JWT (Kubernetes or OIDC)
    Enforcement: FastAPI middleware (inject_user_details)
    Policy: Permission-based (GroupBasedPolicy, NamespaceBasedPolicy)

  Registry gRPC / Online Store gRPC / Offline Store gRPC:
    Auth: Bearer JWT (Kubernetes or OIDC)
    Enforcement: gRPC interceptor
    Policy: Permission-based

  Operator Metrics (8443/TCP):
    Auth: Bearer token (TokenReview)
    Enforcement: kube-rbac-proxy pattern
    Policy: metrics-reader ClusterRole

  Inter-service communication:
    Auth: HMAC-SHA256 JWT (intra-comm token)
    Enforcement: Feast auth middleware
    Policy: Bypass per-user auth

================================================================================
  FIPS Compliance Notes
================================================================================

  Operator (Go):
    - GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, -tags strictfipsruntime
    - Dynamic linking with UBI9 OpenSSL
    - HTTP/2 disabled (CVE safety)
    - InsecureSkipVerify: true for cluster-internal registry calls (documented gap)

  Feature Server (Python):
    - FIPS via UBI9 base image OpenSSL
    - PyJWT for inter-service tokens
    - No explicit FIPS Python crypto configuration
