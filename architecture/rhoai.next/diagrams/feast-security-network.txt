- Store connection secrets (Opaque) - Database credentials, user-provided, no auto-rotate
- OIDC secret (Opaque) - auth_discovery_url, client_id, client_secret, user-provided, no auto-rotate
- OpenLineage API key (Opaque) - API key, user-provided, no auto-rotate
- Intra-communication token (ConfigMap) - Random JWT, operator-generated, rotates per reconcile

Authentication & Authorization:
- /get-online-features: Bearer Token (OIDC/K8s) → SecurityManager → READ_ONLINE permission
- /push: Bearer Token (OIDC/K8s) → SecurityManager → WRITE_ONLINE / WRITE_OFFLINE permission
- /materialize: Bearer Token (OIDC/K8s) → SecurityManager → UPDATE permission
- /health: No auth (public)
- /api/v1/permissions: Intra-communication JWT (operator-only)
- /metrics (operator): Bearer Token via kube-rbac-proxy (metrics-reader ClusterRole)
