Authentication & Authorization:
- Feature server: Bearer JWT (K8s TokenReview or OIDC), enforced via FastAPI middleware
- Registry gRPC: gRPC interceptor, permission-based
- Operator internal: HMAC-SHA256 intra-comm token for operator→registry calls
- Operator metrics: kube-rbac-proxy, Bearer Token

TLS Configuration:
- Operator: HTTP/2 disabled by default (CVE mitigation)
- Go feature server: MinVersion=TLS 1.2, P256+X25519MLKEM768 curves
- Services: OpenShift service-serving CA (automatic) or user-provided certs
- CA bundle: config.openshift.io/inject-trusted-cabundle ConfigMap supported

Secrets:
- feast-{name}-{service}-tls (kubernetes.io/tls) - Service TLS certs, OpenShift CA, auto-rotate
- OIDC client secret (Opaque) - client_id, client_secret, issuer URL, user-provided
- Database credentials (Opaque) - PostgreSQL/Redis/cloud DB connection strings, user-provided

Security Concerns:
- Operator registry client uses InsecureSkipVerify=true for cluster-internal certs
- Go Redis client creates empty tls.Config{} (uses system defaults)
- Feature server Python image does NOT use AIPCC base images (uses UBI 9 Python 3.12 directly)
