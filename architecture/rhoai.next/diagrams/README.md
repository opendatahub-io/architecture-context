### Component Dependency Graph
Shows how RHOAI's 71 components depend on each other, organized into logical groups: Meta-Operator, Core Controllers, User-Facing Services, Inference Runtimes, LLM Infrastructure (new in 3.x), Guardrails & Safety, Evaluation, ModelMesh, Llama Stack, Pipeline Infrastructure, Training, Gateway Infrastructure, and External Dependencies. Central components (rhods-operator, KServe, odh-dashboard, Kueue, MaaS) are highlighted. Useful for understanding blast radius of changes and planning upgrades.

### Platform Network Topology
Complete network architecture showing the 3-tier namespace topology (operator, platform, workload), all ingress points via centralized Gateway API (Envoy), service mesh communication (Istio mTLS STRICT for KServe/Knative), and egress destinations (S3, container registries, OAuth providers). Includes exact port numbers, protocols, encryption, and authentication mechanisms.

**Two formats provided**:
- **Mermaid (.mmd)**: Visual diagram with color-coded trust zones (ingress, platform, mesh, workload, egress). Great for presentations and architecture discussions.
- **ASCII (.txt)**: Precise text format with full namespace topology, authentication architecture (4 layers), RBAC summary (top 5 privileged roles), and TLS configuration summary. Required for SAR submissions and compliance documentation.

Both formats contain the same information — use Mermaid for visual clarity, ASCII for security reviews.

### Cross-Component Workflows
Five sequence diagrams showing end-to-end workflows:
1. **Model Deployment**: Dashboard → KServe → runtime pod → odh-model-controller → Gateway API routing
2. **Pipeline Execution**: Dashboard → DSPO → Argo Workflows → MLMD → Model Registry
3. **Distributed Training**: CodeFlare SDK → Kueue → KubeRay → Ray cluster (with mTLS, NetworkPolicies)

5. **Model Evaluation**: Dashboard → TrustyAI LMES → Kueue → lm-eval-harness → Eval Hub → MLflow

### Platform Security Overview
Visual representation of RHOAI's defense-in-depth security architecture:
- **6 authentication mechanisms**: kube-auth-proxy (OAuth2/OIDC), kube-rbac-proxy (SAR), Kuadrant/Authorino, Bearer passthrough, Ray mTLS, platform-delegated
- **10 key ServiceAccounts** with their ClusterRole bindings

- **Critical secrets** (storage credentials, OAuth config, webhook TLS, Ray CA)

### Platform Observability
