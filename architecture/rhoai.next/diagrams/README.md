## Diagram Descriptions

### Component Dependency Graph
Shows how RHOAI's 76 components depend on each other. rhods-operator is the platform root deploying all component operators. KServe is the serving hub with the most inbound dependencies. odh-dashboard aggregates APIs from nearly every component. Color-coded by category: dark for root operator, blue for central components, green for model management, yellow for data/experiments, red for training/evaluation, purple for notebooks/GenAI.

### Platform Network Topology
Complete network architecture showing the Gateway API ingress stack (Envoy + kube-auth-proxy + EnvoyFilter), 4 namespace tiers (operator, platform, monitoring, user), all component communication paths, and external egress destinations.

**Two formats provided**:
- **Mermaid (.mmd)**: Visual diagram with color-coded trust zones — orange for ingress, green for platform services, blue for monitoring, yellow for user namespaces, red for egress
- **ASCII (.txt)**: Precise text format listing every port, protocol, encryption method, authentication mechanism, and ServiceAccount. Required for SAR submissions

Both formats contain the same information — use Mermaid for visual clarity, ASCII for security reviews.

### Cross-Component Workflows
Five sequence diagrams showing end-to-end flows:
1. **LLM Model Deployment & Inference** - Dashboard -> KServe -> vLLM -> llm-d EPP -> Envoy routing
2. **Notebook Workbench Lifecycle** - Dashboard -> Notebook Controller -> auto-inject connections -> idle culling
3. **ML Pipeline Execution** - Dashboard -> DSPA -> Argo Workflows -> Driver/Launcher -> MLMD + S3
4. **Model Evaluation** - Dashboard -> EvalHub -> Kueue admission -> lm-eval-harness -> MLflow tracking
5. **Models as a Service** - Dashboard -> MaaS Controller -> AI Gateway -> Authorino -> External LLM Provider

Each workflow shows protocol details (mTLS, HTTPS, gRPC), authentication at boundary crossings, and ext_proc routing decisions.

### Platform Security Overview
Visual representation of RHOAI's defense-in-depth security architecture:
- **6 authentication mechanisms**: Gateway ext_authz, kube-rbac-proxy, Kuadrant AuthPolicy, API keys, mTLS, OAuth proxy (legacy)
- **Gateway authentication stack**: Envoy -> EnvoyFilter -> kube-auth-proxy -> token injection flow
- **Key ServiceAccounts and ClusterRoles**: Privilege levels highlighted in red/orange
- **Critical secrets**: Credential secrets in yellow, TLS certificates in light yellow
- **TLS configuration**: 5 TLS patterns across webhooks, metrics, gateway, service-to-service, inter-pod
- **Container security**: 8 patterns including UBI 9, FIPS crypto, non-root, digest pinning, capability drops
- **FIPS compliance gaps**: 3 identified gaps highlighted in red
- **Network security**: NetworkPolicies, namespace isolation, allowed ingress sources

### Platform Observability
Maps all monitoring and observability infrastructure:
- **Prometheus scrape targets**: 19+ endpoints categorized as operator metrics (kube-rbac-proxy on 8443), service metrics, inference metrics, pipeline metrics
- **Health probes**: 15+ components with liveness/readiness probes (controllers on 8081, services on application ports)
- **Distributed tracing**: 4 components sending OTLP gRPC traces to OpenTelemetry Collector
- **Alerting**: Key alert categories for component health, inference latency, pipeline failures, idle notebooks, resource exhaustion
- **Monitoring stack**: Prometheus -> Thanos Querier -> Perses dashboards + AlertManager routing

### Platform Maturity Dashboard
High-level metrics organized into 6 categories:
- **Platform Metrics**: 76 components, 18 operators, 120+ images, 22+ webhooks
- **Gateway API Coverage**: Comprehensive with 4 ext_proc services, ModelMesh legacy still pending
- **Security Posture**: MIXED mTLS, 6 auth patterns, 3 FIPS gaps, all non-root containers
- **CRD API Maturity**: Mix of v1alpha2 through v1 — color-coded green (GA), blue (beta), orange (alpha)
- **Deployment Readiness**: Leader election HA, HPA/KEDA/WVA autoscaling, disconnected support, 4-arch builds
- **Integration Complexity**: 7 integration patterns with component counts
