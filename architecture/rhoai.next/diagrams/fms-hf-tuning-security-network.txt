RBAC Summary:
  No RBAC resources (ClusterRoles, RoleBindings) are defined by this component.
  RBAC is managed externally by the Kubeflow Training Operator for PyTorchJob pods.
  Training pods run under user-provided ServiceAccounts within project namespaces.

Secrets:
  - HF_TOKEN (env var / Opaque secret)
    - Purpose: HuggingFace Hub authentication for gated model downloads
    - Provisioned by: User/ConfigMap
    - Auto-Rotate: No

  - imagePullSecrets (kubernetes.io/dockerconfigjson)
    - Purpose: Container registry credentials for pulling fms-hf-tuning image
    - Provisioned by: User
    - Auto-Rotate: No

Security Notes:
  - ⚠️  NCCL inter-rank communication (port 29500/TCP) is PLAINTEXT
    - No TLS or authentication between distributed training ranks
    - Relies on Kubernetes network policies for isolation
    - Consider: Service mesh sidecar injection for mTLS if sensitive data
  - Container image uses UBI9 base with CUDA 12.1
    - Multi-stage build strips dev packages and dnf to minimize CVE surface
  - Error handling distinguishes user errors (exit 1) vs internal errors (exit 203)
  - Termination logs written to /dev/termination-log for K8s status reporting
  - simpleeval library used for safe expression evaluation in trainer controller rules
    - Restricts imports and dunder access
  - Jinja2 templates run in sandboxed mode for data formatting
