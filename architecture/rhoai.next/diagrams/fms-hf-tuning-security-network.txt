- imagePullSecrets (kubernetes.io/dockerconfigjson) - Pull image from Quay.io, manual provisioning, no auto-rotate
- HF_TOKEN (env var, Opaque) - Authenticate to HuggingFace Hub for gated models, manual provisioning, no auto-rotate

Security Risks:
1. FSDP inter-worker communication (29500/TCP) is PLAINTEXT
   - Gradient tensors and model parameters transmitted unencrypted
   - Standard for distributed ML training (encryption overhead prohibitive)
   - Mitigation: NetworkPolicy or namespace isolation required
2. No pip lock file present - Python dependency resolution is non-hermetic
   - Different builds may resolve different dependency versions
   - Supply chain risk for reproducibility
3. No RPM lock file (rpms.lock.yaml) present on main branch
   - Expected for upstream; downstream release branches add lockfiles

FIPS Compliance:
- Python-only component (no Go binaries, no custom crypto)
- TLS delegated to Python runtime + UBI 9 OpenSSL
- No direct crypto library imports detected
- FIPS compliance inherited from UBI 9 base image OpenSSL configuration

Build Security:
- Base image: UBI 9 (registry.access.redhat.com/ubi9/ubi)
- Release image strips Python 3 system packages and dnf (reduced CVE surface)
- No package manager available in runtime container
- Tekton/Konflux pipeline builds via .tekton/ pipeline definitions
- Image pushed to quay.io/modh/fms-hf-tuning (rhoai-tenant namespace)
