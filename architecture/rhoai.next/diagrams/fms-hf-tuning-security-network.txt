SECURITY NOTES:

⚠️  FSDP Inter-Worker Communication (29500/TCP):
   - Gradient synchronization between distributed training workers is PLAINTEXT
   - Standard practice for distributed ML (encryption overhead prohibitive at training speeds)
   - Relies on network-level isolation (Kubernetes NetworkPolicy / namespace isolation)
   - Risk: gradient data exposure if network is compromised

RBAC Summary:
   - No RBAC roles defined by this component
   - RBAC is managed by Kubeflow Training Operator and PyTorchJob service account
   - Pod runs with ServiceAccount permissions inherited from PyTorchJob

Secrets:
   - imagePullSecrets (kubernetes.io/dockerconfigjson) - Pull image from Quay.io, manual provisioning, no auto-rotate
   - HF_TOKEN (Opaque env var) - Authenticate to HuggingFace Hub for gated models, manual provisioning, no auto-rotate

FIPS Compliance:
   - Base image: UBI 9 (registry.access.redhat.com/ubi9/ubi) — provides FIPS-validated OpenSSL
   - No custom crypto libraries or TLS configuration
   - All TLS operations delegated to Python runtime / system OpenSSL
   - FIPS compliance inherited from base image in FIPS-enabled OpenShift clusters

Build Security:
   - ⚠️  No lock files present (pip, rpm, artifacts) — non-hermetic dependency resolution
   - Supply chain risk: builds may resolve different dependency versions within constraint ranges
   - Tekton/Konflux pipeline without cachi2/Hermeto prefetch integration

Container Security:
   - Release image strips Python system packages and dnf (rpm -e)
   - Reduced CVE surface but no package manager for runtime debugging
   - Multi-stage Docker build with UBI 9 base
