NETWORK SUMMARY:
- Ingress: NONE (batch job, no inbound connections)
- Egress: Hugging Face Hub (443/TCP), AimStack (configurable), MLflow (configurable), ClearML (configurable)
- Internal: NCCL inter-pod communication (29500/TCP, unencrypted) for multi-GPU FSDP
- Build-time only: NVIDIA CUDA repos (443/TCP), PyPI (443/TCP)

RBAC Summary:
- No Kubernetes RBAC defined by this component
- RBAC is managed by the Kubeflow Training Operator (KFTO)
- Pod runs as non-root user (tuning, UID 1000)

Secrets:
- SFT_TRAINER_CONFIG_JSON_ENV_VAR (Opaque) - Base64-encoded JSON training config, provisioned by user/job orchestrator, no auto-rotation
- SFT_TRAINER_CONFIG_JSON_PATH (Opaque) - Path to JSON config file (mounted via ConfigMap/Secret), provisioned by user/job orchestrator, no auto-rotation

Security Concerns:
- pickle.loads() fallback in tuning/utils/config_utils.py:167 — deserialization vulnerability if attacker controls env var
- NCCL inter-pod communication (29500/TCP) is unencrypted — acceptable within trusted cluster network
- No FIPS-specific crypto configuration — inherits from UBI 9 base image OpenSSL
- No lock files for build hermeticity — dependency versions pinned by ranges only

Container Security:
- Base image: registry.access.redhat.com/ubi9/ubi:latest
- Non-root user: tuning (UID 1000)
- pip removed from final image (no runtime package installation)
- Python, dnf, crypto-policies-scripts stripped in release-base stage
- CUDA dev libraries only in build stages (not in final image)
