Authentication & Authorization Model:
======================================

  Layer 1 - Gateway:
    - TLS termination at Gateway API
    - No user-level auth (handled downstream)

  Layer 2 - kube-rbac-proxy:
    - SubjectAccessReview: user must be able to list OpenShift projects
    - Injects identity headers: x-forwarded-access-token, x-auth-request-user, x-auth-request-groups

  Layer 3 - Fastify Backend:
    - Admin routes: checks user group membership (secureAdminRoute())
    - K8s API proxy: passthrough user token (K8s RBAC enforced server-side)
    - Service proxy: user token forwarded to upstream services

  Layer 4 - BFF Sidecars:
    - SubjectAccessReview per API call (BFF middleware)
    - User must have access to target service in namespace
    - Token forwarded to upstream domain services

  dashboard-proxy-tls (kubernetes.io/tls)

FIPS Compliance:
================

  Host (Node.js):
    - OpenSSL from UBI 9 base image (FIPS-compliant when FIPS mode enabled on host)
    - TLS via Node.js https.globalAgent with system CA bundles

  BFF Sidecars (Go):
    - Build: CGO_ENABLED=1, -tags strictfipsruntime
    - Linking: Dynamic (CGO) - uses system OpenSSL
    - TLS minimum: TLS 1.3 (crypto/tls)
    - esbuild Go binaries explicitly removed from production images
    - Custom CA bundle support via BUNDLE_PATHS (5 system paths loaded)

  Risk: InsecureSkipVerify available as dev flag (default: false)
