Authentication & Authorization Summary:
========================================

Layer 1 — kube-rbac-proxy (port 8443):
  - Mechanism: OpenShift RBAC (SubjectAccessReview)
  - Policy: User must have "list" on project.openshift.io/projects
  - Enforces authentication before any request reaches backend

Layer 2 — Fastify Backend (port 8080):
  - Mechanism: X-Auth-Request-User/Groups headers (from kube-rbac-proxy)
  - Policy: secureRoute (authenticated user), secureAdminRoute (admin group check)
  - Admin check via OpenShift group membership

Layer 3 — Go BFF Modules (ports 8043-8843):
  - Mechanism: x-forwarded-access-token header
  - Policy: SubjectAccessReview per namespace/resource
  - Middleware: AttachNamespace → RequireAccessToService

Layer 4 — Kubernetes API passthrough (/api/k8s/*):
  - Mechanism: User's Bearer token forwarded directly
  - Policy: Native Kubernetes RBAC
