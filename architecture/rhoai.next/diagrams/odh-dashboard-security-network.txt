- kube-rbac-proxy validates Bearer Token via SubjectAccessReview
- Resource: "list" on "projects" (project.openshift.io)
- On success: forwards with x-forwarded-access-token and x-auth-request-user headers
- Dashboard backend applies additional app-level authorization:
  - secureAdminRoute: Admin group membership required
  - secureRoute: Namespace ownership + resource name contains username
