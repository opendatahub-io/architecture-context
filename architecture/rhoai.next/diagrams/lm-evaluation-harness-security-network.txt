RBAC Summary:
- Not applicable — RBAC is configured by the trustyai-service-operator
  when creating the Kubernetes Job. The lmes-job container itself does
  not define ClusterRoles or RoleBindings.
- Pod ServiceAccount is set by the operator based on LMEvalJob CR spec.

Secrets Summary:
- Model auth secret (model.auth.secret_ref):
  Type: Opaque
  Contains: API key, HF token (hf-token key), optional CA cert
  Provisioned by: User / trustyai-service-operator
  Auto-Rotate: No

- OCI registry auth:
  Type: Docker config (kubernetes.io/dockerconfigjson)
  Contains: Registry authentication credentials
  Provisioned by: User / platform config
  Auto-Rotate: No

- OPENAI_API_KEY:
  Type: Environment variable
  Derived from: Model auth secret
  Purpose: Bearer token for model inference API
  Auto-Rotate: No

- HF_TOKEN:
  Type: Environment variable
  Derived from: Model auth secret (hf-token key) or user env
  Purpose: HuggingFace Hub access for gated resources
  Auto-Rotate: No

FIPS Compliance:
- Build: Python application on UBI 9 base — OpenSSL provided by base image
- Runtime: ssl.create_default_context() respects system FIPS configuration
- Crypto: SHA-256 for cache keys (FIPS-approved), no non-FIPS crypto detected
- TLS: Configurable verify_certificate with custom CA bundle support
- Symlink CA paths explicitly rejected for security (api_models.py:136-137)

Network Security Notes:
- No ingress ports — batch job is egress-only
- All egress connections use TLS 1.2+ minimum
- Model inference supports custom CA bundles for private PKI
- evalhub_insecure flag available for development (not recommended for production)
- Offline mode available for air-gapped/disconnected environments
- Concurrent inference requests clamped to [1, 128] range
