NETWORK TOPOLOGY NOTES:
═══════════════════════
- NO INBOUND TRAFFIC: This is a batch Job container with NO Services,
  NO Ingress, NO exposed ports. All traffic is EGRESS-ONLY.
- The Job pod is created by the TrustyAI LMEval operator and terminates
  after evaluation completes (one-shot execution).

RBAC Summary:
═════════════
- No RBAC resources defined by this component.
- RBAC for the Job pod is managed by the LMEval operator
  (trustyai-service-operator) which creates the Job.

Secrets:
════════
- Model auth secret (Opaque)
  - Keys: OPENAI_API_KEY, hf-token (HF_TOKEN), ca-cert
  - Provisioned by: User / LMEval operator
  - Auto-rotate: No

- OCI registry auth (docker config.json)
  - Purpose: Authentication for OCI artifact push
  - Provisioned by: User / Platform
  - Auto-rotate: No

- git-auth (kubernetes.io/basic-auth)
  - Purpose: Git authentication for Tekton pipeline builds
  - Provisioned by: Pipelines as Code
  - Auto-rotate: No

Security Controls:
══════════════════
- Credential sanitization: _sanitize_error_message() redacts Bearer tokens,
  API keys, passwords, and URL credentials from error messages before
  sending to EvalHub callbacks
- Path traversal protection: _resolve_job_spec_path_for_read() ensures
  job spec resolves under /meta only
- TLS validation: verify_certificate rejects symlinks, validates file
  existence, uses ssl.create_default_context()
- FIPS: Python on UBI 9 delegates to OpenSSL (FIPS-capable when enabled
  at OS level); no non-FIPS crypto libraries detected
