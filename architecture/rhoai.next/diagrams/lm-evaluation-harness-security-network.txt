RBAC Summary:
- RBAC is managed by the TrustyAI operator, not by this component
- The Job pod runs under a ServiceAccount created by the operator
- No ClusterRoles or ClusterRoleBindings defined by this component

Secrets:
- Model auth secret (model.auth.secret_ref) — Opaque
  Contents: API key, HF token, CA certificate for model endpoint
  Provisioned by: User / TrustyAI operator
  Auto-Rotate: No

- git_auth_secret — Opaque
  Contents: Git authentication for Konflux pipeline builds
  Provisioned by: Konflux / PipelinesAsCode
  Auto-Rotate: No

- OCI auth config (/tmp/.docker/config.json) — Docker config
  Contents: OCI registry push authentication
  Provisioned by: EvalHub / Kubernetes mount
  Auto-Rotate: No

- AWS credentials (env vars) — Environment
  Contents: S3 access key, secret key for asset downloads
  Provisioned by: User / Kubernetes Secret mount
  Auto-Rotate: No

Security Controls:
- Container runs as non-root user (UID 65532)
- CPU-only execution (no GPU access)
- CWE-22 path traversal prevention for job spec loading
- Credential redaction in error messages (Bearer tokens, key=value,
  JSON-encoded secrets, URL credentials)
- Symlink traversal prevention for CA certificate paths
- Custom CA certificate support for model endpoints
- Auto-detection of air-gapped/offline mode

FIPS Compliance:
- Python application on UBI9 (inherits system OpenSSL with FIPS support)
- ssl.create_default_context() inherits FIPS-compliant cipher suites
- No non-FIPS crypto libraries detected (no pycryptodome, pycrypto)
- S3_VERIFY_SSL flag can disable TLS verification (not a FIPS issue)
