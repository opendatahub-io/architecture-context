RBAC Summary:
- No RBAC defined in this repository
- RBAC is managed by the TrustyAI LMEval Operator (trustyai-service-operator)
- The operator creates ServiceAccounts and RBAC bindings for the Job pods

Secrets:
- model.auth.secret_ref (Opaque) - API key for inference endpoint + optional HF token
  Provisioned by: User / LMEvalJob CR, No auto-rotation
- /tmp/.docker/config.json (Docker config) - OCI registry authentication
  Provisioned by: Platform / Operator, No auto-rotation
- AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY (Env vars) - S3 storage credentials
  Provisioned by: User / ConfigMap, No auto-rotation
- verify_certificate path (TLS CA bundle) - Custom CA for inference endpoint TLS
  Provisioned by: Platform / Operator, No auto-rotation

Security Notes:
- Container runs as non-root user (UID 65532, GID 65532)
- No inbound network exposure (no Services, no Ingress)
- All external communication is egress-only
- TLS 1.2+ on all external connections
- Custom CA certificate support for air-gapped/private PKI environments
- Error messages are sanitised to avoid leaking raw exception details
- No CRDs owned (batch job, not an operator)
- Offline mode disables all HuggingFace network calls via environment flags
