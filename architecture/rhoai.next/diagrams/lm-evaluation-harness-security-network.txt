RBAC Summary:
- Not directly applicable — lm-evaluation-harness is a batch job container
- RBAC for the Job pod is configured by the trustyai-service-operator
- The operator's ServiceAccount creates Jobs and ConfigMaps in user namespaces

Secrets:
- Model auth secret (model.auth.secret_ref) (Opaque) — API key, HF token, optional CA cert
  - Provisioned by: User / trustyai-service-operator
  - Auto-rotate: No
- OCI registry auth (Docker config) — Registry authentication for artifact push
  - Provisioned by: User / platform config
  - Auto-rotate: No
- OPENAI_API_KEY (env var) — Bearer token for model inference API
  - Derived from: model auth secret
  - Auto-rotate: No
- HF_TOKEN (env var) — HuggingFace Hub token for gated resources
  - Derived from: model auth secret (hf-token key) or user env
  - Auto-rotate: No

TLS Configuration:
- Python ssl.create_default_context() for custom CA bundles
- System trust store via UBI 9 base image
- aiohttp TCPConnector with configurable ssl context
- requests.post(verify=...) for sync requests
- Symlink paths for CA certificates explicitly rejected (security hardening)
- verify_certificate: True (system trust), False (skip), or path to CA bundle

FIPS Compliance:
- Python application — no Go binary (check-payload N/A for Go checks)
- OpenSSL present via UBI 9 base image
- Uses hashlib.sha256 (FIPS-approved) for cache keys
- No non-FIPS crypto (MD5, RC4, DES) detected
- ssl module delegates to system OpenSSL (FIPS via UBI 9)

Network Policy Notes:
- No ingress required (batch job — egress only)
- All egress uses HTTPS/TLS 1.2+ minimum
- Offline mode eliminates egress 5 (HuggingFace Hub) — data pre-fetched via init container
- Custom CA bundles supported for all TLS connections
