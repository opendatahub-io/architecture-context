RBAC Summary:
  No RBAC resources — TGIS runs as a workload pod, not a controller.
  RBAC is managed by KServe (which creates the pod).

Service Details:
  - Service: inference-server (ClusterIP, headless)
  - Port: 8033/TCP (gRPC)
  - Exposure: Internal (cluster only; external via KServe Route)

Secrets:
  - TLS keypair (TLS_CERT_PATH, TLS_KEY_PATH)
    Type: kubernetes.io/tls
    Purpose: gRPC/HTTP TLS termination
    Provisioned by: cert-manager or manual mount
    Auto-Rotate: Depends on provisioner

  - Client CA cert (TLS_CLIENT_CA_CERT_PATH)
    Type: Opaque
    Purpose: mTLS client certificate verification
    Provisioned by: manual mount
    Auto-Rotate: No

FIPS Considerations:
  - Dual TLS stack: rustls 0.22.4 (ring crate) + openssl 0.10.64 / native-tls 0.2.13
  - rustls uses ring crate which may NOT use FIPS-validated crypto module
  - No explicit FIPS mode enforcement in Rust TLS configuration
  - UBI 9.5 base image provides FIPS-validated OpenSSL for Python layer
  - NCCL inter-GPU communication is plaintext (localhost only in single-node)

Volumes:
  - PVC (models-pvc): Mounted at /shared_model_storage for model weights
  - emptyDir (/dev/shm): Shared memory for NCCL multi-GPU communication
