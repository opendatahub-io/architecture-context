Services:
- inference-server: ClusterIP (headless), 8033/TCP gRPC, configurable TLS/mTLS, internal only

RBAC Summary:
- No RBAC resources — TGIS runs as a workload pod, not a controller
- RBAC is managed externally by KServe
- TGIS does not interact with the Kubernetes API

Secrets:
- TLS keypair (TLS_CERT_PATH, TLS_KEY_PATH)
  - Type: kubernetes.io/tls
  - Purpose: gRPC/HTTP TLS termination
  - Provisioned by: cert-manager or manual mount
  - Auto-Rotate: Depends on provisioner

- Client CA cert (TLS_CLIENT_CA_CERT_PATH)
  - Type: Opaque
  - Purpose: mTLS client certificate verification
  - Provisioned by: Manual mount
  - Auto-Rotate: No

Authentication & Authorization:
- gRPC GenerationService (Generate, GenerateStream, Tokenize, ModelInfo):
  - Auth: mTLS client certificates (optional)
  - Enforcement: Tonic gRPC server in router
  - Policy: Client cert must be signed by TLS_CLIENT_CA_CERT_PATH CA
- HTTP /health, /metrics:
  - Auth: None (unauthenticated probe endpoints)

FIPS Compliance Notes:
- Dual TLS stack: rustls 0.22.4 (ring crate) + openssl 0.10.64 + native-tls 0.2.13
- rustls uses ring crate which may NOT use FIPS-validated crypto module
- No explicit FIPS mode enforcement in Rust TLS configuration
- UBI 9.5 base image provides FIPS-validated OpenSSL system library
- Python: cryptography >=43.0.1 (wraps system OpenSSL)
- Risk: gRPC TLS via Tonic may use rustls path instead of FIPS-validated OpenSSL
