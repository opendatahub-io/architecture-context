RBAC Summary:
- No RBAC resources — TGIS runs as a workload pod, not a controller
- RBAC is fully managed by KServe for the InferenceService lifecycle
- No Kubernetes API interaction from TGIS processes

Service Configuration:
- Service: inference-server (headless ClusterIP)
  - Port: 8033/TCP (gRPC)
  - Target Port: 8033
  - Selector: matches TGIS deployment pods

TLS Configuration:
- Dual TLS stack: rustls 0.22.4 (via Tonic) + openssl 0.10.64 (system)
- rustls uses ring crate — may NOT use FIPS-validated crypto module
- No explicit FIPS mode enforcement or cipher suite restrictions
- Certificate sources: PEM files mounted at TLS_CERT_PATH, TLS_KEY_PATH
- mTLS: Client CA cert at TLS_CLIENT_CA_CERT_PATH (optional)

FIPS Compliance Concerns:
- Dual TLS stack (rustls + openssl) — TLS may not use FIPS-validated path
- ring crate (used by rustls) has its own crypto implementation
- No OLM FIPS annotation (TGIS is not an OLM-managed operator)
- UBI 9.5 base image provides FIPS-validated OpenSSL

Secrets:
- TLS keypair (TLS_CERT_PATH, TLS_KEY_PATH)
  Type: kubernetes.io/tls
  Purpose: gRPC/HTTP TLS termination
  Provisioner: cert-manager or manual mount
  Rotation: Depends on provisioner

- Client CA cert (TLS_CLIENT_CA_CERT_PATH)
  Type: Opaque
  Purpose: mTLS client certificate verification
  Provisioner: Manual mount
  Rotation: No auto-rotate
