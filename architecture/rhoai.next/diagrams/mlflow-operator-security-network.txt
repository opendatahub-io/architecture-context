Secrets:
- mlflow-tls (kubernetes.io/tls) - TLS cert/key for MLflow HTTPS, OpenShift service-ca, auto-rotated
- mlflow-artifact-connection (Opaque) - Per-namespace S3 credentials, user-provisioned, no auto-rotation

TLS Architecture:
- Inbound: uvicorn direct SSL termination using service-ca provisioned certs (mlflow-tls Secret)
- Outbound: Combined CA bundle from 3 sources:
  1. System CAs (/etc/pki/tls/certs/ca-bundle.crt)
  2. Platform CAs (odh-trusted-ca-bundle ConfigMap)
  3. User CAs (caBundleConfigMap spec field)
- CA bundle injected via: SSL_CERT_FILE, REQUESTS_CA_BUNDLE, CURL_CA_BUNDLE, AWS_CA_BUNDLE, PGSSLROOTCERT, MLFLOW_MYSQL_CA
- Init container creates initial bundle; sidecar regenerates every 30s on ConfigMap changes

NetworkPolicy Details:
- Ingress: Port 8443 from ANY namespace (required: gateway, dashboard, Prometheus in varying namespaces)
- Egress: Explicit allowlist (DNS, HTTPS, K8s API, DB, object storage)
- Extensible: networkPolicyAdditionalEgressRules spec field for non-standard ports
- Note: Port 5353 included for OVN-Kubernetes post-DNAT CoreDNS evaluation

Authentication:
- MLflow API: Kubernetes Bearer Token → SelfSubjectAccessReview → maps to RBAC verbs on pseudo-resources
- Metrics (operator): Bearer Token → TokenReview + SubjectAccessReview
- Metrics (MLflow): Bearer Token + TLS (service-ca)
- Security middleware: MLFLOW_SERVER_DISABLE_SECURITY_MIDDLEWARE=false (hardcoded, blocked in CR spec validation)
