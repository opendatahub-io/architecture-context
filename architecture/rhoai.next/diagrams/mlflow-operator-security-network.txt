  - Resources: namespaces, secrets (mlflow-artifact-connection), mlflowconfigs

  - mlflow-view: Read-only access to mlflows, mlflowconfigs, datasets, experiments, registeredmodels
  - mlflow-edit: Full CRUD on mlflows, mlflowconfigs, datasets, experiments, registeredmodels, gatewaysecrets
  - mlflow-integration: Cross-component integration access to datasets, experiments, registeredmodels, gatewayendpoints

Service Mesh / Network Policy:
- NetworkPolicy: Operator-created for MLflow pods
  - Egress allowed: DNS (53, 5353), HTTPS (443), K8s API (6443), PostgreSQL (5432), MySQL (3306), MinIO (9000), SeaweedFS (8333-8334)
  - Ingress: Platform gateway → MLflow 8443/TCP

Authentication & Authorization:
- MLflow API: Kubernetes RBAC via SelfSubjectAccessReview
  - Enforcement: MLflow kubernetes-auth app plugin
  - Checks pseudo-resources in mlflow.kubeflow.org API group
  - Security middleware hardcoded enabled (CEL-validated CRD field)
- Operator metrics: Bearer Token with authn/authz filter
- MLflow health: Unauthenticated

Secrets:
- mlflow-tls (kubernetes.io/tls) - MLflow server TLS cert, provisioned by OpenShift service-ca, auto-rotated
- controller-manager-metrics-tls (kubernetes.io/tls) - Operator metrics TLS cert, service-ca provisioned, auto-rotated
- mlflow-artifact-connection (Opaque) - Per-namespace S3 credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_S3_BUCKET, AWS_S3_ENDPOINT), manually created, no auto-rotation

FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
- Runtime: Dynamic linking, OpenSSL via UBI 9 minimal
- TLS: crypto/tls with HTTP/2 disabled, cipher selection via system crypto policy
- MLflow: UVICORN_SSL_CIPHERS=PROFILE=SYSTEM (delegates to platform crypto policy)
