  - Role: system:auth-delegator
  - ServiceAccount: {raycluster}-oauth-proxy
  - Purpose: OAuth proxy SubjectAccessReview delegation

Secrets:
- codeflare-operator-webhook-server-cert (kubernetes.io/tls)
  - Purpose: Webhook server TLS cert
  - Provisioned: cert-controller (auto-rotated)
  - Rotation: Automatic

- {raycluster}-proxy-tls-secret (kubernetes.io/tls) — per RayCluster
  - Purpose: OAuth proxy TLS serving cert
  - Provisioned: OpenShift serving-cert annotation
  - Rotation: Automatic (OpenShift managed)

- {raycluster}-oauth-config (Opaque) — per RayCluster
  - Purpose: OAuth cookie secret (SHA1 of cluster name)
  - Provisioned: Operator (generated)
  - Rotation: None

- ca-secret-{raycluster} (Opaque) — per RayCluster
  - Purpose: CA private key + cert for Ray mTLS
  - Provisioned: Operator (RSA 2048-bit, self-signed)
  - Validity: 1 year (no auto-rotation)

NetworkPolicy Rules:
- head-{name}: Allows traffic from cluster pods, KubeRay operator ns,
  openshift-monitoring (port 8080), any source on secured ports (8443, 10001)
- worker-{name}: Only accepts traffic from pods in the same RayCluster

Webhook Configuration:
- MutatingWebhookConfiguration: codeflare-operator-mutating-webhook
  - RayClusters: injects OAuth proxy + mTLS
  - AppWrappers: admission control (optional)
- ValidatingWebhookConfiguration: codeflare-operator-validating-webhook
  - RayClusters: enforces immutability of injected resources
  - AppWrappers: authorization via SAR (optional)
