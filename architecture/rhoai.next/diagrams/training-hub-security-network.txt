NO CRDs, NO RBAC, and NO secrets of its own. The diagram below shows

RBAC Summary:
- N/A -- training-hub is a Python library with no Kubernetes RBAC

Service Mesh Configuration:
- N/A -- training-hub is not deployed as a Kubernetes service

Secrets:
- No secrets managed by training-hub itself
- Runtime secrets are managed by the consuming environment:
  - HF_TOKEN (Optional) - HuggingFace access token for gated models
  - WANDB_API_KEY (Optional) - Weights & Biases API key
  - MLFLOW_TRACKING_URI (Optional) - MLflow server URI

Security Notes:
- litellm dependency pinned to <1.82.7 due to supply chain attack (TeamPCP, 2026-03-24)
- GRPO backends use subprocess isolation to prevent GPU memory conflicts
- No cryptographic operations performed by the library itself
- No TLS termination or certificate handling
- No container image -- FIPS compliance is responsibility of consuming environment
- All inter-process communication is local (loopback/shared memory)
