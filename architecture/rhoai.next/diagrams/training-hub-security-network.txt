RBAC Summary:
- No Kubernetes RBAC resources (training-hub is a Python library)
- No ServiceAccounts, ClusterRoles, or RoleBindings
- Authentication is user-managed via environment variables:
  - HF_TOKEN: HuggingFace Hub access (Bearer token)
  - WANDB_API_KEY: Weights & Biases API key
  - MLFLOW_TRACKING_URI: MLflow server endpoint

Secrets Management:
- No Kubernetes Secrets (no cluster deployment)
- Credentials managed via environment variables at runtime
- No credential storage or caching by the library

Supply Chain Security:
- litellm pinned to <1.82.7 due to TeamPCP supply chain attack (2026-03-24)
- No lock file (library uses version ranges in pyproject.toml)
- Exact versions resolved at install time by consuming applications

Network Security Notes:
- NCCL distributed training has no encryption (standard limitation)
  - Mitigation: Network isolation of training cluster
- vLLM subprocess binds to localhost only (no external exposure)
- All external API calls are outbound-only over HTTPS/TLS 1.2+
- No inbound network listeners in the library itself
- No service mesh integration (not a deployed service)

FIPS Compliance:
- N/A for library itself (pure Python, no container images)
- Inherits FIPS posture of host environment
- TLS handled by urllib3/requests using system OpenSSL
- No direct crypto imports in training-hub code
