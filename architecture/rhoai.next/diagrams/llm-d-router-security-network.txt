  - inference.networking.k8s.io / inferencepools: get, watch, list
  - llm-d.ai / inferenceobjectives, inferencemodelrewrites: get, watch, list

Namespace Role (EPP - HA leader election):
  - coordination.k8s.io / leases: get, list, watch, create, update, patch, delete
  - core / events: create, patch

Role Bindings:
  - EPP RoleBinding → EPP Role → EPP ServiceAccount (namespace)
  - EPP ClusterRoleBinding → EPP ClusterRole → EPP ServiceAccount (cluster)
  - EPP Leader Election RoleBinding → EPP Leader Election Role → EPP ServiceAccount (namespace)

Authentication & Authorization:
  - /metrics (EPP): Bearer token + TokenReview + SubjectAccessReview (--metrics-endpoint-auth)
  - ext_proc gRPC: TLS client certificate (optional); no app-level auth, network trust
  - Health gRPC: None (open for probes)
  - Sidecar routes: None; SSRF allowlist via InferencePool (--enable-ssrf-protection)

Secrets:
  - TLS certificates (EPP): kubernetes.io/tls, self-signed or cert-path mount, hot-reload optional
  - TLS certificates (sidecar): kubernetes.io/tls, self-signed or cert-path mount, no auto-rotate
  - inference-gateway-sa-metrics-reader-secret: kubernetes.io/service-account-token, Kubernetes-managed auto-rotate

FIPS Compliance:
  - EPP: CGO_ENABLED=1, dynamic linking, UBI 9 OpenSSL (missing strictfipsruntime tag)
  - Sidecar: CGO_ENABLED=1, -tags strictfipsruntime, UBI 9 OpenSSL (FIPS enforced)
  - Base Image: registry.access.redhat.com/ubi9/ubi-minimal:9.7

Security Concerns:
  - InsecureSkipVerify=true on metrics scraping HTTP clients (default)
  - EPP gRPC uses self-signed certs (encryption but no authentication)
  - EPP missing strictfipsruntime build tag (inconsistent FIPS posture vs sidecar)
  - Sidecar SSRF protection is optional (--enable-ssrf-protection, off by default)
