- alertmanager-proxy (Opaque) - OAuth proxy session secret, deploy.sh (openssl rand), no auto-rotate
- prometheus-secret (service-account-token) - SA token for Prometheus scraping, Kubernetes, auto-rotate
- anaconda-ce-access (Opaque) - Anaconda CE credentials (empty placeholder), deploy.sh, no auto-rotate
- odh-segment-key (Opaque) - Segment.io analytics key (base64), deploy.sh, no auto-rotate
- redhat-rhods-pagerduty (Opaque) - PagerDuty service key, Addon/SRE pre-provisioned, no auto-rotate
- redhat-rhods-deadmanssnitch (Opaque) - DMS URL, Addon/SRE pre-provisioned, no auto-rotate
- redhat-rhods-smtp (Opaque) - SMTP credentials (host, port, user, pass), Addon/SRE pre-provisioned, no auto-rotate
- addon-managed-odh-parameters (Opaque) - Addon params incl. notification-email, Addon/SRE, no auto-rotate

Authentication & Authorization:
- Prometheus Route (9091/TCP): OAuth Proxy → SAR: get namespaces in redhat-ods-monitoring
- Alertmanager Route (10443/TCP): OAuth Proxy → SAR: get namespaces in redhat-ods-monitoring
- Blackbox Exporter (9114/TCP): OAuth Proxy → SAR: get namespaces in redhat-ods-monitoring
- Prometheus /metrics: No auth (skip-auth-regex ^/metrics bypasses OAuth)

Security Observations:
- anyuid SCC granted to multiple service accounts (permissive but required)
- Blackbox exporter uses insecure_skip_verify: true (internal probing only)
- Segment.io analytics key committed in base64 (not truly secret)
- oc CLI download in Dockerfile uses unpinned "latest" URL without checksum (supply chain risk)
- Prometheus/Alertmanager OAuth proxy session secrets regenerated on every deployment
