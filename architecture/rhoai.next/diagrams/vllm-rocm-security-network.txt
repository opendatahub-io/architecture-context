RBAC Summary:
- No RBAC resources defined in this repository
- RBAC is managed by KServe and the platform operator for the
  inference pod's service account

Security Configuration:
- Container runs as non-root user (UID 2000)
- No TLS or authentication at container level
- Security enforcement delegated to platform ingress stack:
  - Gateway API with kube-rbac-proxy sidecars (RHOAI 3.x)
  - RequestAuthentication / AuthorizationPolicy (platform-level)
- DISABLE_LOGPROBS_DURING_SPEC_DECODING=false (overrides upstream
  performance optimization for API consistency)

Secrets:
- (model credentials) (Opaque) - S3/storage credentials for model
  download, provisioned by user/platform operator, no auto-rotation

Network Policy:
- No NetworkPolicy defined in this repository
- Network isolation managed by KServe and platform namespace policies

Trust Boundaries:
1. EXTERNAL → INGRESS: TLS termination at platform ingress
2. INGRESS → CONTAINER: Plaintext HTTP/gRPC (auth enforced upstream)
3. CONTAINER → GPU: Hardware bus (no network encryption)
4. CONTAINER → EGRESS: TLS 1.2+ to external services (S3, HF Hub)
