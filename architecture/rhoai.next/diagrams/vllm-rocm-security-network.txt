RBAC Summary:
- No RBAC resources defined in this repository
- RBAC is managed by KServe and the platform operator for the inference pod's
  service account
- The container itself has no Kubernetes API access requirements

Security Configuration:
- Container User: UID 2000 (non-root)
- TLS: Not implemented at container level (delegated to platform)
- Authentication: Not implemented at container level (delegated to platform)
- Authorization: Not implemented at container level (delegated to platform)
- Security Hardening: HTTP header count/size limits, removal of eval() for
  unknown types

Secrets:
- (model credentials) (Opaque) - S3/storage credentials for model download,
  provisioned by user/platform operator, no auto-rotation

Network Policy Notes:
- Container does not define NetworkPolicy resources
- Network isolation is managed by the platform/KServe namespace policies
- All container ports are unencrypted; encryption is enforced at the
  platform ingress layer
