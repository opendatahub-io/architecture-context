RBAC Summary:
  No RBAC roles or bindings — this component is a container image, not an operator.
  Authentication and authorization are enforced by the surrounding KServe infrastructure:
  - kube-rbac-proxy sidecar (RHOAI 3.x): Bearer Token + Kubernetes SubjectAccessReview
  - oauth-proxy sidecar (RHOAI 2.x): OAuth token validation

Secrets:
  No secrets defined in this repository.
  Model access credentials are injected by KServe InferenceService spec:
  - HF_TOKEN (env var) — HuggingFace Hub bearer token
  - AWS credentials — S3 access via IAM role or access key/secret

Service Mesh Configuration:
  TLS and mTLS are managed by KServe/Istio infrastructure, not by the application.
  The vLLM container serves plaintext HTTP/gRPC; encryption is handled at the pod boundary.

FIPS Compliance:
  - Application serves plaintext; TLS handled externally by kube-rbac-proxy / Istio
  - OpenSSL from RHEL 9 base image provides FIPS-validated crypto when host FIPS mode enabled
  - No Go binaries (Python application) — no GOEXPERIMENT=strictfipsruntime needed
  - Build: hermetic=false (appropriate for thin wrapper — hermeticity in AIPCC base image)

Container Security:
  - Runs as non-root user (UID 1001)
  - Base image: RHEL 9 (rhaiis/vllm-rocm-rhel9:3.2.1)
  - No additional packages installed beyond base image
  - DISABLE_LOGPROBS_DURING_SPEC_DECODING=false (UX trade-off: logprobs always available)
