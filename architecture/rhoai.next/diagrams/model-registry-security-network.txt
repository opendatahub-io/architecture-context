Security Notes:
- ⚠️  Core model-registry API has NO built-in authentication — relies entirely on Istio mTLS + AuthorizationPolicy
- ⚠️  AuthorizationPolicy is ALLOW all within mesh — any pod in the mesh can access the registry API
- ⚠️  InsecureSkipVerify defaults to true unless explicitly configured (gap in defense-in-depth)
- ✅  TLS config has insecure cipher rejection and FIPS-compatible cipher suites
- ✅  FIPS build flags: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
- ✅  Controller metrics endpoint protected by TokenReview + SubjectAccessReview
- ✅  NetworkPolicy: allow-metrics-traffic restricts controller metrics access
