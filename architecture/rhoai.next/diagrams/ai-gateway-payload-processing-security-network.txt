Network Isolation:
- ext_proc gRPC (9004/TCP): NO TLS, NO authentication
  - Security relies on network isolation — must be in same namespace as Gateway
  - Only Envoy proxy connects via EnvoyFilter targetRefs
- Health check (9005/TCP): NO TLS, NO authentication
  - Liveness/readiness probes only

Secrets:
- User-created Secrets (Opaque): API keys for external model providers
  - Label: inference.networking.k8s.io/bbr-managed=true
  - Watched by secretReconciler, cached in-memory (secretStore)
  - No auto-rotation
  - Sensitive: contains cloud provider API keys (OpenAI, Anthropic, Azure, etc.)
  - Persist only in process memory after reconciliation

FIPS Compliance:
- Build-time enforcement: GOEXPERIMENT=strictfipsruntime in Dockerfile.konflux
- Runtime: Uses Go 1.25 FIPS-validated crypto

Security Considerations:
- ext_proc runs plaintext gRPC — relies entirely on pod-local network isolation
- API keys are held in process memory (secretStore) — no disk persistence
- No mTLS between Envoy and ext_proc (same-pod / same-namespace communication)
- NeMo Guardrails calls may be plaintext HTTP depending on configuration
