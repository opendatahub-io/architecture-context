  - API Group: mlflow.kubeflow.org
    - Resources: mlflowconfigs

  - API Group: "" (core)

    - Verbs: get, list, watch
  - API Group: authorization.k8s.io
    - Resources: selfsubjectaccessreviews
    - Verbs: create

Authentication Model:
- Workspace RBAC: Kubernetes SelfSubjectAccessReview against
  mlflow.kubeflow.org pseudo-resources per workspace
- Token source: K8s ServiceAccount token (Bearer)
- Authorization delegation: Every authenticated API call triggers
  a SelfSubjectAccessReview to the K8s API server
- Default on missing permission: NO_PERMISSIONS (deny by default)

Secrets:
- MLFLOW_CRYPTO_KEK_PASSPHRASE (Opaque env var) - AES-256-GCM KEK for
  gateway credential encryption, Manual/Operator provisioned, No auto-rotate
- MLFLOW_FLASK_SERVER_SECRET_KEY (Opaque env var) - Flask session signing,
  Manual/Operator provisioned, No auto-rotate
- Database connection URI (Opaque env var) - PostgreSQL/MySQL connection
  string with credentials, Operator provisioned, No auto-rotate
- S3/artifact credentials (Opaque env var) - AWS_ACCESS_KEY_ID,
  AWS_SECRET_ACCESS_KEY, Platform config provisioned, No auto-rotate

FIPS Compliance:
- Python service on UBI9-minimal (OpenSSL linked via system RPM)
- cryptography >=43,<47 (uses OpenSSL backend, FIPS-compatible)
- KEK encryption: AES-256-GCM with PBKDF2-HMAC-SHA256 (600,000 iterations)
- No non-FIPS crypto detected (no MD5, RC4, DES, custom RNG)
- TLS handled at platform ingress gateway, not by MLflow directly

Build Hermeticity:
- RPM: rpms.lock.yaml (rpm-lockfile-prototype)
- Python: uv.lock + konflux-*.txt (pip --require-hashes)
- JavaScript: yarn.lock (Yarn Berry immutable install)
- Hermeto/Cachi2 prefetch for all layers
- Base images: SHA256-pinned UBI9
- Multi-arch: amd64, arm64, ppc64le, s390x
