NetworkPolicy (Helm-managed, when enabled):
  Egress Rules:
  - Allow DNS: 53/UDP, 53/TCP, 5353/UDP, 5353/TCP (CoreDNS + OVN-K)
  - Allow HTTPS: 443/TCP (S3, LLM providers, external services)
  - Allow PostgreSQL: 5432/TCP
  - Allow MySQL: 3306/TCP
  - Allow S3 (MinIO): 9000/TCP, 8333/TCP, 8334/TCP
  - Allow Kubernetes API: 6443/TCP

RBAC Summary:
  - ServiceAccount: mlflow (namespace-scoped)
  - ServiceAccount token automount: disabled by default
  - Role/ClusterRole: configurable via Helm values
    - namespace_rbac.extraRules: namespace-scoped RBAC
    - cluster_rbac.extraRules: cluster-scoped RBAC
  - No default RBAC rules created

Secrets:
  - mlflow-tls (kubernetes.io/tls)
    - TLS certificate and key for HTTPS
    - Provisioned: Manual / cert-manager
    - Auto-Rotate: No
  - Backend store secret (Opaque)
    - Database connection URI (PostgreSQL/MySQL)
    - Provisioned: Manual
    - Auto-Rotate: No
  - MLFLOW_CRYPTO_KEK_PASSPHRASE (env var)
    - Passphrase for KEK derivation (AES-256-GCM)
    - WARNING: Default dev passphrase is a production risk if not overridden
    - Manual rotation via CLI (`mlflow gateway rotate-kek`)

FIPS Compliance:
  - pyca/cryptography (>=43.0.0) delegates to system OpenSSL (FIPS-validated on RHEL/UBI9)
  - PBKDF2-HMAC-SHA256 with 600,000 iterations (NIST SP 800-132 compliant)
  - AES-256-GCM encryption (NIST-approved algorithm)
  - hashlib.md5 used in 4 places with usedforsecurity=False (non-cryptographic, FIPS-compliant)
  - No Go binaries (Python service, N/A for GOEXPERIMENT=strictfipsruntime)

Container Security:
  - Base image: registry.access.redhat.com/ubi9/ubi-minimal:9.7
  - Non-root user: UID 1000
  - Read-only root filesystem
  - No privilege escalation
  - All capabilities dropped
  - Seccomp profile: RuntimeDefault
  - Multi-arch: amd64, arm64, ppc64le, s390x
