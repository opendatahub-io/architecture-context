RBAC Summary:
- RBAC roles and bindings are managed by the MLflow Operator (separate repo)
- Runtime authorization is enforced by the kubernetes-auth plugin via
  SelfSubjectAccessReview against mlflow.kubeflow.org pseudo-resources
- Per-request: caller's Bearer Token is submitted as a SAR to verify
  access to the target workspace (Kubernetes namespace)

Secrets:
- MLFLOW_CRYPTO_KEK_PASSPHRASE (Opaque env var)
  - Purpose: KEK passphrase for AES-256-GCM envelope encryption
  - Provisioned by: Server admin (env var or .env file)
  - Rotation: Manual via CLI (KEK rotation supported without re-encrypting values)
  - WARNING: Default passphrase exists for dev; production MUST override

- Gateway endpoint secrets (encrypted in DB)
  - Purpose: API keys for LLM providers (OpenAI, Anthropic, Gemini, etc.)
  - Encryption: AES-256-GCM with per-secret DEK, wrapped by KEK
  - AAD binding: Ciphertext bound to secret metadata (prevents substitution)
  - Provisioned by: User via API

- TLS certificates (kubernetes.io/tls)
  - Purpose: TLS termination inside the MLflow pod
  - Provisioned by: MLflow Operator / cert-manager
  - Rotation: Depends on provisioner

Authentication Mechanisms:
- /api/*, /ajax-api/*, /gateway/*, /v1/traces:
  - Bearer Token (K8s ServiceAccount or user token)
  - Enforcement: kubernetes-auth plugin → SelfSubjectAccessReview
  - Workspace scoping via X-MLFLOW-WORKSPACE header
- /health, /version, /static-files/*, /:
  - No authentication (exempt endpoints)

FIPS Compliance:
- Python on UBI9 inherits FIPS mode from host OS OpenSSL
- cryptography>=43.0.0 uses FIPS-validated OpenSSL primitives when FIPS enabled
- hashlib.md5(usedforsecurity=False) calls are FIPS-compliant (Python 3.9+)
- No custom cipher suite configuration (relies on system defaults)
