Authentication Flow:
- Every authenticated API request triggers:
  1. Bearer token extraction from Authorization header
  2. Workspace resolution from X-MLFLOW-WORKSPACE header
  3. SelfSubjectAccessReview against K8s API (mlflow.kubeflow.org pseudo-resources)
  4. workspace_aware_mixin enforces query-level isolation
- Latency dependency: each MLflow API call → at least 1 K8s API call

Secrets:
- MLFLOW_CRYPTO_KEK_PASSPHRASE (Opaque/env) - KEK for AES-256-GCM encryption of AI gateway credentials, Manual/Operator, No auto-rotate
- MLFLOW_FLASK_SERVER_SECRET_KEY (Opaque/env) - Flask session signing key, Manual/Operator, No auto-rotate
- Database connection URI (Opaque/env) - PostgreSQL/MySQL connection string with credentials, Operator, No auto-rotate
- S3/artifact credentials (Opaque/env) - AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, Platform config, No auto-rotate
