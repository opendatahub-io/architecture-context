RBAC Summary:
- ClusterRole: mlflow-executor (required for K8s project backend)
  - API Groups: batch
  - Resources: jobs
  - Verbs: create, get, list, watch, delete

Secrets:
- MLFLOW_CRYPTO_KEK_PASSPHRASE (Opaque/env) - KEK for gateway secrets envelope encryption
  - Provisioned by: Manual (admin)
  - Auto-Rotate: No (manual via `mlflow crypto rotate-kek`)
- MLFLOW_FLASK_SERVER_SECRET_KEY (Opaque/env) - Flask session cookie signing
  - Provisioned by: Manual (admin)
  - Auto-Rotate: No
- Database credentials (Opaque/connection string) - PostgreSQL URI with username/password
  - Provisioned by: Manual or Operator
  - Auto-Rotate: No
- AI Gateway provider API keys (Opaque/encrypted in DB) - LLM provider keys
  - Provisioned by: Admin via Gateway API
  - Auto-Rotate: No (KEK rotation available)

Authentication Flow:
1. External request arrives at Platform Gateway (HTTPS/443)
2. Platform Gateway routes to kube-rbac-proxy sidecar (HTTPS/8443)
3. kube-rbac-proxy validates Kubernetes SA token
4. kube-rbac-proxy forwards to MLflow Server (HTTP/5000, pod-local)
5. kubernetes-namespaced auth provider extracts workspace from token
6. MLflow auth middleware checks per-resource permissions (READ/USE/EDIT/MANAGE)

Workspace Isolation:
- Kubernetes namespaces map 1:1 to MLflow workspaces
- Composite primary keys on database tables include workspace column
- Request-scoped ContextVar propagates workspace context
- X-MLFLOW-WORKSPACE header injected by auth provider
- Workspace deletion modes: RESTRICT, CASCADE, SET_DEFAULT

Security Controls:
- DNS rebinding protection via CORS + Host header validation
- Cross-origin state change blocking
- Localhost-only restriction for assistant endpoints (127.0.0.1/::1)
- AI Gateway requires explicit MLFLOW_ENABLE_AI_GATEWAY=true
- KEK envelope encryption for provider API keys in database
