Authentication & Authorization:
- /api/*, /ajax-api/*:
  - Primary: Bearer Token (K8s ServiceAccount) via kubernetes-auth plugin
  - Mechanism: SelfSubjectAccessReview against mlflow.kubeflow.org pseudo-resource
  - Fallback: HTTP Basic Auth (basic_auth.ini) with READ/USE/EDIT/MANAGE roles
- /health, /version: No auth (exempt)
- CORS: State-changing cross-origin requests blocked unless origin is trusted
- Host validation: DNS rebinding protection; only localhost and private IPs by default

Security Headers:
- X-Frame-Options: DENY
- X-Content-Type-Options: nosniff

Secrets:
- mlflow-tls (kubernetes.io/tls) - TLS cert for HTTPS termination, cert-manager or manual, no auto-rotate
- Database credentials (Opaque) - PostgreSQL/MySQL connection URI, manual or external secret operator, no auto-rotate
- MLFLOW_CRYPTO_KEK_PASSPHRASE (env var) - AES-GCM KEK for gateway secret envelope encryption, admin-provisioned, no auto-rotate

Cryptography:
- AES-GCM envelope encryption (cryptography library v46.0.7, OpenSSL backend)
- PBKDF2HMAC for KEK derivation from passphrase
- hashlib.sha256 for job param hashing (non-security)
- hashlib.sha1(usedforsecurity=False) for conda env hashing (FIPS-safe)
- HMAC-SHA256 for auth credential caching
- OpenSSL inherited from UBI9 base image (FIPS-compliant when host in FIPS mode)
