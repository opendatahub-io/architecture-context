RBAC Summary:
- No Kubernetes RBAC resources. This is a stateless service that does not
  interact with the Kubernetes API server.
- No ClusterRoles, Roles, ClusterRoleBindings, or RoleBindings defined.
- No ServiceAccount defined in this repository (managed by deployer).

Secrets:
- TLS cert (/etc/tls/private/tls.crt)
  Type: kubernetes.io/tls
  Purpose: Client certificate for mTLS with orchestrator
  Provisioned by: External (cert-manager or platform operator)
  Auto-Rotate: Unknown

- TLS key (/etc/tls/private/tls.key)
  Type: kubernetes.io/tls
  Purpose: Private key for mTLS with orchestrator
  Provisioned by: External (cert-manager or platform operator)
  Auto-Rotate: Unknown

- CA cert (/etc/tls/ca/service-ca.crt)
  Type: Opaque
  Purpose: Custom CA certificate for validating orchestrator server cert
  Provisioned by: External (OpenShift service CA operator)
  Auto-Rotate: Unknown

Authentication & Authorization:
- Gateway performs NO authentication or authorization itself.
- Authorization header from clients is forwarded as-is to orchestrator.
- x-forwarded-* headers from clients are forwarded to orchestrator.
- Authentication enforcement is delegated to:
  1. Upstream proxy (e.g., kube-rbac-proxy sidecar) — for client auth
  2. FMS Guardrails Orchestrator — for backend auth

FIPS Compliance:
- Rust binary dynamically links against system OpenSSL (UBI 9 base image)
- openssl crate (0.10.73): PEM → PKCS#12 certificate conversion only
- native-tls crate (0.2.12): Delegates TLS to system OpenSSL
- No pure-Rust crypto (ring, rustls) used
- FIPS-capable when OS FIPS mode is enabled (RHEL 9 / UBI 9)
- Risk: danger_accept_invalid_hostnames(true) gated on hostname=="localhost"
  (development mode only, not triggered in production)

Network Security Notes:
- Gateway listens on plaintext HTTP (8090/TCP) — no TLS termination
- TLS termination expected at platform ingress layer
- Outbound to orchestrator uses conditional mTLS (degrades gracefully)
- No health/readiness/liveness probe endpoints exposed
- No Prometheus metrics endpoint exposed
- No ingress resources defined (managed externally)
