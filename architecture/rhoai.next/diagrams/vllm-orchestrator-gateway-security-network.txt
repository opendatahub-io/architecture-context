Security Observations:
  1. Gateway performs NO authentication or authorization — all auth delegated to orchestrator
  2. Listening port 8090/TCP is plaintext HTTP (no TLS termination at gateway)
  3. mTLS to orchestrator is opportunistic — activated only when cert files exist at well-known paths
  4. Code contains danger_accept_invalid_hostnames(true) for localhost development
  5. No NetworkPolicy, PeerAuthentication, or AuthorizationPolicy resources in repository
  6. No service mesh sidecar configuration in repository
  7. Authorization header is forwarded verbatim (no validation, sanitization, or scoping)
  8. X-Forwarded-* headers forwarded without validation (potential spoofing if not behind trusted proxy)

Network Flow Summary:
  Client → [8090/TCP HTTP plaintext] → Gateway → [8085/TCP HTTP/HTTPS optional mTLS] → Orchestrator → Detectors + vLLM
