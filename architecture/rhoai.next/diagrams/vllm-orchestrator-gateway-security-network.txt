RBAC Summary:
- No RBAC resources — this is a standalone service, not a Kubernetes operator
- No ClusterRoles, Roles, or RoleBindings defined

Secrets:
- TLS cert (/etc/tls/private/tls.crt)
  - Type: kubernetes.io/tls
  - Purpose: mTLS client certificate for authenticating to orchestrator
  - Provisioned by: Platform/cert-manager (external)
  - Auto-Rotate: Unknown

- TLS key (/etc/tls/private/tls.key)
  - Type: kubernetes.io/tls
  - Purpose: mTLS private key for orchestrator communication
  - Provisioned by: Platform/cert-manager (external)
  - Auto-Rotate: Unknown

- CA cert (/etc/tls/ca/service-ca.crt)
  - Type: Opaque
  - Purpose: Custom CA certificate for trusting orchestrator's server cert
  - Provisioned by: Platform/service-ca-operator (external)
  - Auto-Rotate: Unknown

Authentication & Authorization:
- No local auth enforcement at the gateway
- Authorization header from clients is transparently forwarded to orchestrator
- x-forwarded-* headers are forwarded to orchestrator
- mTLS (when TLS certs present) provides transport-level authentication to orchestrator

TLS Configuration:
- Uses openssl crate (0.10.73) for PKCS#12 identity construction
- Uses native-tls (0.2.12) for TLS transport via reqwest
- Delegates cipher suite selection to system OpenSSL defaults
- danger_accept_invalid_hostnames(true) ONLY for localhost connections (dev/test)
- On FIPS-enabled RHEL/OpenShift, system OpenSSL operates in FIPS mode automatically

Network Notes:
- No Kubernetes Service manifest in repository (provided by deploying operator)
- No Ingress resources in repository (managed externally)
- Gateway binds to 0.0.0.0:8090 by default
- Upstream orchestrator connection is configurable via config.yaml
