RBAC Summary:
  None — this is a stateless HTTP service binary, not a Kubernetes operator.
  No ClusterRoles, Roles, or RoleBindings are defined in the repository.

Secrets:
  - /etc/tls/private/tls.crt (kubernetes.io/tls) - Client certificate for mTLS to orchestrator
    Provisioned by: Platform (cert-manager or service-serving-signer)
    Auto-Rotate: Unknown

  - /etc/tls/private/tls.key (kubernetes.io/tls) - Private key for mTLS client identity
    Provisioned by: Platform (cert-manager or service-serving-signer)
    Auto-Rotate: Unknown

  - /etc/tls/ca/service-ca.crt (Opaque) - Custom CA for validating orchestrator's server cert
    Provisioned by: Platform (service-ca-operator)
    Auto-Rotate: Unknown

Authentication & Authorization:
  - Gateway does NOT enforce authentication itself
  - Authorization header is passed through from client to orchestrator
  - Orchestrator is the enforcement point for authorization policies
  - mTLS client identity (when configured) provides service-level authentication

Crypto Details:
  - TLS backend: native-tls (backed by system OpenSSL on Linux)
  - Certificate handling: openssl crate 0.10.73 for PEM parsing + PKCS#12 construction
  - FIPS: Relies on system OpenSSL defaults (dynamic linking against openssl-libs)
  - Risk: No explicit cipher suite restrictions; depends on system OpenSSL configuration
  - Risk: danger_accept_invalid_hostnames(true) when host is "localhost"

Network Exposure:
  - No ingress resources defined in repository
  - Expected to be fronted by platform mechanisms (HTTPRoute, OpenShift Route,
    InferenceService/ServingRuntime configuration)
  - Service type: ClusterIP (internal only)
