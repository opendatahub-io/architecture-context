Security Notes:
  ⚠ danger_accept_invalid_hostnames(true) is enabled in the TLS client
    builder for localhost development. This should be reviewed and
    conditionally disabled for production deployments.
  ⚠ Gateway performs no authentication or authorization itself — it is
    a pure routing and policy (content filtering) layer.
  ⚠ TLS is activated opportunistically based on certificate file presence;
    there is no configuration to require TLS.
