RBAC Summary:
  No RBAC roles or bindings defined in this repository.
  RBAC is managed by KServe and the platform operator.

  Authentication enforcement:
  - kube-rbac-proxy sidecar validates JWT tokens against Kubernetes RBAC
  - Applies to both HTTP (/v1/*) and gRPC (TGIS 8033) endpoints
  - vLLM container itself has no authentication - relies entirely on sidecar

Secrets:
  - (HuggingFace token) - Opaque - Model download auth - User-provisioned - No auto-rotate
  - (S3 credentials)    - Opaque - Model storage access - User-provisioned / platform operator - No auto-rotate

Container Security:
  - Runs as non-root user (UID 2000)
  - No privileged capabilities requested (GPU access via device plugin)
  - Base image: RHAIIS vllm-cuda-rhel9:3.2.2 (RHEL9/UBI9)
  - FIPS compliance inherited from RHAIIS base image

FIPS Compliance:
  - No cryptographic code in this repository
  - All TLS/crypto handled by RHAIIS base image (RHEL9 OpenSSL)
  - FIPS assessment should target RHAIIS vLLM CUDA source repository

Build Security:
  - Tekton pipeline: hermetic=false (appropriate - no package installation)
  - Multi-arch: amd64, arm64
  - No lock files (no dependencies installed at this layer)
