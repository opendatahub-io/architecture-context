RBAC Summary:
- No RBAC resources defined in this repository
- RBAC is managed by the deploying operator (KServe, odh-model-controller)
- Container runs as non-root user (UID 2000)

Secrets:
- (model storage credentials) (Opaque) - S3/storage access for model download
  - Provisioned by: User / platform operator
  - Auto-Rotate: No

Authentication & Authorization:
- Container-level: No authentication (by design)
- All auth enforced at platform layer:
  - kube-rbac-proxy sidecar handles TLS termination and authn
  - RHOAI Gateway enforces per-InferenceService policies
  - Bearer Token / OAuth validated at gateway level

Container Security:
- Runs as UID 2000 (non-root)
- Base image: registry.redhat.io/rhaiis/vllm-cuda-rhel9:3.2.2
- HTTP header count and size limited (security hardening)
- eval() usage removed for unknown types (security fix)
- Multi-arch: amd64, arm64
- Build security: Clair scanning, preflight checks via Konflux
