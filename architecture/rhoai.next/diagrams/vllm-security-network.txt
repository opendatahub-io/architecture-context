RBAC Summary:
- No RBAC resources defined in this repository
- RBAC is managed by the deploying operator (KServe, odh-model-controller)
- kube-rbac-proxy sidecar enforces SubjectAccessReview for inference endpoints

Container Security:
- Runs as non-root user: UID 2000
- Base image: registry.redhat.io/rhaiis/vllm-cuda-rhel9:3.2.2
- Multi-arch: amd64, arm64
- Build pipeline: Konflux with Clair vulnerability scanning and preflight checks
- HTTP header count and size limited (security hardening)
- No eval() for unknown type conversion (security fix)

Secrets:
- (model storage credentials) (Opaque) - S3/storage access for model download
  - Provisioned by: User / platform operator
  - Auto-rotate: No

Authentication & Authorization:
- No auth at container level
- All auth enforced by platform layer:
  - kube-rbac-proxy sidecar (per-pod)
  - RHOAI Gateway (per-route)
  - KServe InferenceService auth policies

Network Exposure:
- Container ports 8000/TCP and 8033/TCP are NOT directly exposed externally
- External access only through RHOAI Gateway → kube-rbac-proxy chain
- Intra-pod communication (kube-rbac-proxy ↔ vLLM) is localhost-only (no encryption needed)
