RBAC Summary:
  None. This is a Python library, not deployed with its own ServiceAccount.
  RBAC is inherited from the consuming application's ServiceAccount.

Service Mesh Configuration:
  N/A. This library does not configure Service Mesh resources.
  mTLS may be enforced by the consuming application's namespace-level
  PeerAuthentication policy, which would apply to the gRPC egress traffic.

Secrets:
  No secrets are defined by this library.
  TLS certificate paths are provided via runtime configuration by the consuming
  application:
  - ca_cert_file    → CA certificate for TGIS server validation
  - client_cert_file → Client certificate for mTLS authentication
  - client_key_file  → Client private key for mTLS authentication

TLS Configuration Summary:
  - Crypto library: grpcio (delegates to gRPC C core → OpenSSL)
  - FIPS compliance: Inherited from container's OpenSSL (FIPS-validated on UBI)
  - Certificate validation: CA cert required before client cert can be set
  - Client cert/key: Must be provided as a pair
  - No hardcoded cipher suites (uses gRPC/OpenSSL defaults)
