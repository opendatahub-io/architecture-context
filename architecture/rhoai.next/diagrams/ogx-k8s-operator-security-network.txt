NetworkPolicy Configuration:
- Per-instance NetworkPolicy applied to each OGXServer
- Ingress allowed from: same namespace + operator namespace only
- Configurable via spec.network in OGXServer CR

Secrets:
- webhook-server-cert (kubernetes.io/tls) - TLS cert for webhook, provisioned by cert-manager or service-serving-cert-signer, auto-rotated
- User-referenced secrets (Opaque) - Provider API keys, DB connection strings, Redis passwords; manual provisioning, no auto-rotation

Authentication Summary:
- Webhook endpoint: Kubernetes API server mTLS (failurePolicy: Fail)
- OGXServer managed pods: No default auth; NetworkPolicy-based isolation
- K8s API access: ServiceAccount token-based auth
- OCI registries: Registry credentials (optional)
- OGX server health checks: No auth (internal HTTP)
