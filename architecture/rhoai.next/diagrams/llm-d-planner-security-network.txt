- MODEL_CATALOG_SOURCE_ID: redhat_ai_validated_models

TLS Configuration:
- External: TLS edge termination at OpenShift Router (Routes)
- Internal: No encryption (HTTP between all pods)
- Model Catalog: TLS 1.2+ with service-serving CA bundle injection
  - CA bundle: /etc/pki/service-ca (annotated ConfigMap, auto-injected by OpenShift)
  - SSL verification: Configurable via MODEL_CATALOG_VERIFY_SSL (default: true)
- HuggingFace: TLS 1.2+ (system default)
- Kubernetes API: TLS 1.2+ (ServiceAccount token auth)

Security Gaps:
1. No authentication on any of the 29 backend API endpoints
2. No service mesh or mTLS for internal pod-to-pod traffic
3. CORS allows all origins with credentials (CORS_ORIGINS=*)
4. PostgreSQL uses password auth over unencrypted connection
5. Ollama accessed over unencrypted HTTP
6. hashlib.sha256 used for config ID generation (non-security, acceptable)

FIPS Notes:
- No Konflux Dockerfiles — FIPS check-payload validation does not apply
- Python application — no compiled binaries with crypto linking
- TLS handled by system OpenSSL (via requests library for Model Catalog)
- hashlib.sha256 used only for deterministic dedup (non-cryptographic)
