RBAC Summary:
  ClusterRole: planner-gpu-reader
    - API Groups: "" (core)
    - Resources: nodes
    - Verbs: list
  ClusterRoleBinding: planner-gpu-reader
    - ServiceAccount: planner/planner-backend
    - Scope: Cluster-wide

  NOTE: Backend also performs kubectl apply/delete for InferenceService resources.
  The RBAC for these operations is NOT explicitly defined in the repository manifests.
  The planner-backend ServiceAccount would need additional RBAC for:
    - serving.kserve.io: inferenceservices (create, get, list, delete)
    - autoscaling: horizontalpodautoscalers (create, get, list, delete)
    - monitoring.coreos.com: servicemonitors (create, get, list, delete)

Secrets:
  planner-secrets (Opaque)
    - POSTGRES_PASSWORD: PostgreSQL database password
    - MODEL_CATALOG_TOKEN: Bearer token for RHOAI Model Catalog
    - Provisioned By: Manual (deploy-all.sh)
    - Auto-Rotate: No

  planner-service-ca (ConfigMap, annotation-injected)
    - service-ca.crt: OpenShift service-serving CA certificate
    - Provisioned By: OpenShift service-ca operator
    - Auto-Rotate: Yes (managed by OpenShift)

Authentication Summary:
  CRITICAL: All 27 Backend API endpoints are UNAUTHENTICATED
  - No auth mechanism on any /api/v1/* endpoint
  - Destructive operations exposed without auth:
    - POST /api/v1/db/reset (wipe benchmark database)
    - POST /api/v1/deploy-to-cluster (deploy to K8s cluster)
    - DELETE /api/v1/deployments/{id} (delete InferenceService)
    - POST /api/v1/db/upload-benchmarks (inject benchmark data)
  - CORS: allow_origins=["*"] (unrestricted cross-origin access)
  - OpenShift Routes: TLS edge termination only, no auth enforcement
  - Status: Incubation project — security hardening is documented as future work

FIPS Compliance:
  - Base Image: python:3.14-slim (Debian) — NOT FIPS-enabled
  - Crypto: Delegates to system OpenSSL via requests/urllib3
  - Risk: FIPS compliance depends on base image OpenSSL configuration
  - Recommendation: Switch to UBI-based image for RHOAI production deployment
