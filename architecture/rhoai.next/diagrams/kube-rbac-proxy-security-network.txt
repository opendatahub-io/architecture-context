RBAC Summary:
- ClusterRole: kube-rbac-proxy
  - API Groups: authentication.k8s.io
  - Resources: tokenreviews
  - Verbs: create
  - API Groups: authorization.k8s.io
  - Resources: subjectaccessreviews
  - Verbs: create
- ClusterRoleBinding: kube-rbac-proxy
  - Scope: cluster-wide
  - ServiceAccount: kube-rbac-proxy (in deploying namespace)

Authentication Modes (mutually exclusive at startup):
- Delegating (default): TokenReview via K8s API (2-minute cache TTL)
- OIDC: JWT verification with configurable claim mappings (--oidc-issuer)
- x509: Client certificate verification (--client-ca-file)

Authorization Chain (union, priority order):
1. Hardcoded: system:serviceaccount:openshift-monitoring:prometheus-k8s → allow GET /metrics
2. Static: Config-file driven (--config-file) resource/non-resource attribute matching
3. SubjectAccessReview: Delegated to Kubernetes API server

Secrets:
- TLS cert/key (--tls-cert-file, --tls-private-key-file)
  - Type: kubernetes.io/tls
  - Provisioned by: cert-manager / platform operator / self-signed fallback
  - Auto-rotate: Yes (CertReloader hot-reload at --tls-reload-interval)
- Client CA (--client-ca-file)
  - Type: CA bundle
  - Provisioned by: Platform operator
  - Auto-rotate: Yes (DynamicFileCAContent)
- Upstream client cert (--upstream-client-cert-file, --upstream-client-key-file)
  - Type: x509 keypair
  - Provisioned by: Platform operator
  - Auto-rotate: No
- OIDC CA (--oidc-ca-file)
  - Type: CA bundle
  - Provisioned by: Platform operator
  - Auto-rotate: Yes (DynamicFileCAContent)

Security Controls:
- Token sanitization: SanitizingFilter masks bearer tokens in klog output
- FIPS compliance: strictfipsruntime + CGO_ENABLED=1 + UBI 9 OpenSSL
- Path bypass: --ignore-paths skips auth for matching paths (use with caution)
- Deprecated insecure mode: --insecure-listen-address (plaintext HTTP, deprecated)
