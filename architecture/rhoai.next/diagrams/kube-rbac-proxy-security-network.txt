Secrets / Certificates:

  - Type: kubernetes.io/tls
  - Purpose: Server TLS certificate for HTTPS listener
  - Provisioned by: Deploying operator / cert-manager
  - Auto-rotate: Yes (hot-reload via CertReloader at --tls-reload-interval)

- Client CA (--client-ca-file)
  - Type: Opaque (CA bundle)
  - Purpose: CA for validating client TLS certificates (mTLS authn)
  - Provisioned by: Deploying operator
  - Auto-rotate: Yes (dynamic reload via DynamicFileCAContent)

- OIDC CA (--oidc-ca-file)
  - Type: Opaque (CA bundle)
  - Purpose: CA for validating OIDC provider's TLS certificate
  - Provisioned by: Deploying operator
  - Auto-rotate: Yes (dynamic reload via DynamicFileCAContent)

- Upstream client cert (--upstream-client-cert-file)
  - Type: kubernetes.io/tls
  - Purpose: Client certificate for mTLS to upstream
  - Provisioned by: Deploying operator
  - Auto-rotate: No (loaded once at startup)
