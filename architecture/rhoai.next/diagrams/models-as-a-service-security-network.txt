- CronJob cleanup: Namespace-restricted access to /internal/v1/api-keys/cleanup
- Controller monitoring: Allow Prometheus scrape on 8080/TCP

Secrets:
- maas-db-config (Opaque) - PostgreSQL connection URL, manual provisioning, no auto-rotate
- maas-api-serving-cert (kubernetes.io/tls) - TLS cert, OpenShift service-ca, auto-rotate
- ExternalModel credentialRef (Opaque) - Provider API key, manual per-model, no auto-rotate
- Gateway TLS cert (kubernetes.io/tls) - Gateway listener cert, manual or cert-manager

Authentication Flow:
1. API Key: Client sends sk-oai-{key_id}_{secret} → Authorino calls maas-api
   /internal/v1/api-keys/validate → SHA-256 hash lookup (key_id as salt,
   null byte delimiter) → constant-time comparison → frozen group snapshot returned
2. K8s Token: Client sends OpenShift token → Authorino calls TokenReview API →
   User identity + groups extracted
3. OIDC JWT: Client sends JWT → Authorino validates against cluster issuer →
   Claims extracted

Security Design Principles:
- Fail-closed: Gateway default policies deny unconfigured paths (401/403 + 0 rate)
- Credential isolation: BBR strips Authorization header before reaching model backends
- Key security: SHA-256 hashed with per-key salt, plaintext shown once, constant-time compare
- IDOR protection: Authorization failures return 404 (not 403) to prevent key enumeration
- Subscription immutability: Binding frozen at key creation to prevent subscription-swap attacks
- Group snapshot freezing: Groups captured at key creation to prevent privilege escalation
