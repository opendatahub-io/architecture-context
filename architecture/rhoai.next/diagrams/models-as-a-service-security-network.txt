Authentication & Authorization:

  Layer 1 - Authorino (AuthPolicy per model HTTPRoute):
  - API key: sk-oai-{key_id}_{secret} → HTTP callback to maas-api /internal/v1/api-keys/validate
  - Kubernetes token: kubernetesTokenReview → K8s API TokenReview
  - OIDC JWT: issuer validation with cluster-audience
  - Authorization: OPA policy checking group membership + subscription validity

  Layer 2 - Limitador (TokenRateLimitPolicy per model):
  - Token-based rate limits per user (auth.identity.userid)
  - Rates aggregated from MaaSSubscription CRs
  - Priority-based subscription selection

  Layer 3 - BBR ext-proc (Security controls):
  - Strips original Authorization header before model backends
  - Injects model-specific credentials from Kubernetes Secrets
  - Prevents credential exfiltration to untrusted models

  Default Gateway Policies (fail-closed):
  - gateway-default-auth: 401/403 for unconfigured paths
  - gateway-default-deny: 0 tokens/min rate limit for unconfigured paths

Network Policies:

  - maas-api internal endpoints (/internal/v1/*): NetworkPolicy restricts access to Authorino namespace only
  - CronJob cleanup endpoint: NetworkPolicy restricts to same namespace
  - Monitoring: NetworkPolicy allows Prometheus scrape on metrics ports

FIPS Compliance:

  - Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamic linking
  - Runtime: UBI9 base with OpenSSL, stdlib crypto/tls + crypto/sha256 via OpenSSL
  - TLS: Configurable MinVersion (default TLS 1.2), supports TLS 1.2 and 1.3
  - Risk: InsecureSkipVerify=true for cluster-internal model probing (acceptable for internal traffic)
  - No MD5/RC4/DES usage detected
