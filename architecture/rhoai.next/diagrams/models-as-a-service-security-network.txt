Authentication & Authorization:

  Public Endpoints (/v1/*):
  - Auth Methods: API key (sk-oai-* Bearer), Kubernetes TokenReview, External OIDC (JWT)
  - Enforcement: Kuadrant Authorino via AuthPolicy on HTTPRoute
  - Policy: maas-api-auth-policy + per-model maas-auth-* AuthPolicies
  - Actions: Strip Authorization header before forwarding to backend

  Internal Endpoints (/internal/v1/*):
  - Auth Methods: None (network-restricted)
  - Enforcement: NetworkPolicy (maas-authorino-allow)
  - Allowed Sources: Authorino pods and CronJob only

  Model Inference (per-model HTTPRoutes):
  - Auth Methods: API key, OpenShift token, or OIDC
  - Enforcement: Kuadrant Authorino + Kuadrant Limitador
  - Rate Limiting: TokenRateLimitPolicy per subscription/model

  Gateway Default:
  - Pattern: deny-unconfigured-models
  - Default Rate Limit: 0 tokens/min (gateway-default-deny TRLP)

Secrets:

  Name: maas-db-config
  - Type: Opaque
  - Purpose: PostgreSQL connection URL (DB_CONNECTION_URL key)
  - Provisioned By: Administrator / deploy script
  - Auto-Rotate: No

  Name: maas-api-serving-cert
  - Type: kubernetes.io/tls
  - Purpose: TLS certificate for maas-api HTTPS
  - Provisioned By: OpenShift service-ca (annotation-based)
  - Auto-Rotate: Yes

  Name: ExternalModel credentialRef secrets
  - Type: Opaque
  - Purpose: External provider API keys (api-key data key)
  - Provisioned By: Administrator
  - Auto-Rotate: No

Network Policies:

  maas-authorino-allow:
  - Purpose: Allow Authorino pods to reach maas-api /internal/v1/* endpoints
  - Source: Authorino pods (label selector)
  - Destination: maas-api port 8080

  maas-cleanup-networkpolicy:
  - Purpose: Allow CronJob to reach maas-api /internal/v1/api-keys/cleanup
  - Source: Cleanup CronJob pods
  - Destination: maas-api port 8080

  maas-controller-metrics:
  - Purpose: Allow Prometheus to scrape controller metrics
  - Source: Prometheus pods
  - Destination: maas-controller port 8080
