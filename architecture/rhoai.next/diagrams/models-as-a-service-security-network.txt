  FIPS COMPLIANCE
================================================================================

  Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, UBI9 base (OpenSSL)
  TLS: Configurable min version (1.2 or 1.3), stdlib crypto/tls (FIPS via OpenSSL)
  Crypto: SHA-256 for API key hashing, RSA 4096-bit for self-signed certs
  Non-FIPS risks: None detected (no MD5, RC4, DES usage)

================================================================================
  SECURITY DESIGN PATTERNS
================================================================================

  - Defense-in-depth: NetworkPolicies + AuthPolicies + RBAC
  - Gateway default-deny: TokenRateLimitPolicy with zero tokens catches unprotected routes
  - API key storage: SHA-256 hash with per-key salt (key_id + null byte + secret)
  - IDOR protection: Returns 404 (not 403) for unauthorized key access
  - CEL injection prevention: Group names validated against regex allowlist
  - SAR caching: Asymmetric TTLs (30s positive, 2s negative)
  - Tenant isolation: Per-tenant maas-api Deployment, TENANT_NAME scopes all DB queries
  - Cross-tenant prevention: Tenant mismatch validation at service layer
