  - Istio mTLS: Enabled via service mesh sidecars
  - DestinationRules: TLS origination for external providers, internal TLS config
  - EnvoyFilters: Two-stage ext_proc pipeline (pre-auth + post-auth)
  - ServiceEntries: Register external LLM provider endpoints in mesh

Secrets:
  - maas-db-config (Opaque) — PostgreSQL connection URL, manual provisioning, no auto-rotate
  - maas-api-serving-cert (kubernetes.io/tls) — maas-api HTTPS cert, OpenShift service-ca, auto-rotate
  - maas-controller-webhook-cert (kubernetes.io/tls) — Webhook TLS cert, OpenShift service-ca, auto-rotate
  - ExternalModel credentialRef (Opaque) — External LLM provider API key ("api-key" data key), manual, no auto-rotate

  - vaitenant.kb.io (validating) — AITenant must be in configured infrastructure namespace
  - vmaasauthpolicy.kb.io (validating) — MaaSAuthPolicy requires existing Tenant in namespace
  - vmaassubscription.kb.io (validating) — MaaSSubscription requires existing Tenant in namespace
