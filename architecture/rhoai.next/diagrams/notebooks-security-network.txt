================================================================================
  RBAC SUMMARY
================================================================================

Workbench images do NOT define RBAC resources.
RBAC for workbench pods is managed by:
  - kubeflow notebook controller (injects sidecar, creates RBAC bindings)
  - rhods-operator / opendatahub-operator (deploys platform RBAC)

Workbench pods run with:
  - ServiceAccount: per-notebook SA (created by notebook controller)
  - Permissions: limited to user namespace

  FIPS COMPLIANCE
================================================================================

  Build-Time:
  - mongocli Go binary: CGO_ENABLED=1, -tags strictfipsruntime
  - Dynamic linking (CGO) for FIPS-compliant crypto
  - UBI9 base provides system OpenSSL
  - compat-openssl11 installed for check-payload scanning

  Application-Level:
  - Python uses system OpenSSL from UBI9 (FIPS-capable)
  - No custom crypto libraries (pycryptodome, etc.)
  - CA bundle mounted by notebook controller
  - No non-FIPS crypto risks detected
