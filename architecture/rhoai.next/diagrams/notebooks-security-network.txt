RBAC Summary:
- This component does NOT define its own RBAC resources
- RBAC for notebook pods is managed by:
  - ODH Notebook Controller (creates ServiceAccounts, RoleBindings)
  - Platform operator (manages namespace-level permissions)
- Authentication enforced by kube-rbac-proxy sidecar (injected by Notebook Controller)
  - Mechanism: Platform SSO (OAuth/OIDC)
  - Policy: User must have access to the namespace

Secrets:
- No secrets defined by this component
- User-provisioned secrets may be mounted for:
  - Data source credentials (S3, database)
  - Git configuration
  - Configured via Dashboard UI, injected by Notebook Controller

FIPS Compliance:
- mongocli binary: Built with CGO_ENABLED=1 -tags strictfipsruntime (FIPS-compliant)
- Python/R/Node.js: Use system OpenSSL from UBI9 base (FIPS mode via crypto-policies)
- TLS: Inherits UBI9 system defaults (OpenSSL crypto-policies)
- OpenSSL: UBI9 base + compat-openssl11 package installed
- Risk: User-installed Python packages may introduce non-FIPS crypto at runtime

Build Hermeticity:
- All builds are fully hermetic via Konflux/cachi2
- Lock files: rpms.lock.yaml, pylock.*.toml, go.sum, package-lock.json
- Zero network access during builds (--no-index flags, /cachi2/output/deps/ mounts)
- Supply chain defense: exclude-newer = "3 days" in pyproject.toml
  (refuses Python packages published <3 days ago)

Image Versioning:
- RHOAI: 3 concurrent versions per image (N=3.4, N-1=2025.2, N-2=2025.1)
- ODH: 2 concurrent versions per image (N, N-1)
- Images referenced by immutable @sha256: digests in production
- N-2 images marked as outdated (deprecation warning in Dashboard)
