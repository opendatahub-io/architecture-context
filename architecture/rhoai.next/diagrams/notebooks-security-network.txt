RBAC Summary:
- No RBAC resources defined in this repository
- Workbench pod RBAC is managed by the ODH Notebook Controller and platform operator
- kube-rbac-proxy sidecar enforces namespace-scoped access control
- Users must have access to the namespace to reach workbench UI

Authentication Architecture:
- External access: OpenShift OAuth → Route/HTTPRoute → kube-rbac-proxy sidecar
- kube-rbac-proxy: Validates OAuth Bearer Token, enforces namespace RBAC
- Internal (pod): All services run plaintext HTTP (auth already validated at pod boundary)
- IDE auth disabled: JupyterLab (no token), code-server (--auth=none), RStudio (no auth)
- Idle culling: ServiceAccount Bearer Token via kube-rbac-proxy → internal HTTP

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription for RStudio BuildConfig builds
  - Provisioned by: Manual (admin)
  - Auto-rotate: No
  - Used at: Build-time only (not runtime)

Supply Chain Security:
- Hermetic builds: No network access during container image build
- Dependency prefetch: cachi2 fetches all deps before build starts
- Hash verification: pylock TOML lockfiles with sha256 hashes
- Freshness window: 3-day exclude-newer prevents supply chain attacks
- Image pinning: All ImageStream references use @sha256: digests
