RBAC Summary:
- This component does NOT define RBAC resources (ClusterRoles, RoleBindings, etc.)

- kube-rbac-proxy sidecar enforces namespace-level access control per user

Note: Authentication is NOT implemented within the workbench images themselves.
The ODH Notebook Controller injects a kube-rbac-proxy sidecar that handles all auth.

Secrets:
- No secrets defined by this component
- User-provisioned secrets (data source credentials, git config, S3 access)
  are configured via Dashboard UI and injected by the Notebook Controller

FIPS Compliance:
- mongocli: Built with CGO_ENABLED=1 -tags strictfipsruntime (FIPS compliant)
- Python/R/Node.js: Rely on system OpenSSL FIPS mode via UBI9 base image
- TLS: Inherits UBI9 system defaults (crypto-policies)
- Risk: User-installed packages may introduce non-FIPS crypto at runtime

Build Hermeticity:
- All images built hermetically via Konflux/cachi2 (zero network during build)
- Lock files: rpms.lock.yaml, pylock.*.toml, go.sum, package-lock.json
- Supply chain defense: exclude-newer = "3 days" blocks freshly published packages
