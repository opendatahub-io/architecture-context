RBAC Summary:
- No RBAC resources defined in this repository
- Workbench pod RBAC managed by ODH Notebook Controller and platform operator
- kube-rbac-proxy sidecar enforces namespace-level access control via OAuth

Authentication Architecture:
- All workbench UIs: OAuth via kube-rbac-proxy sidecar (injected by Notebook Controller)
- kube-rbac-proxy validates OAuth tokens against OpenShift API server
- Internal workbench processes (JupyterLab, code-server, RStudio): --auth=none
- Trust boundary: kube-rbac-proxy sidecar is the sole authentication enforcement point
- All internal pod communication is plaintext HTTP (localhost only)

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription for RStudio BuildConfig builds
  - Provisioned by: Manual (admin)
  - Auto-Rotate: No
  - Used during: Build-time only (not runtime)

Build-Time Security:
- Hermetic builds via Konflux (no network access during build)
- All dependencies prefetched by cachi2 with hash verification
- 3-day exclude-newer window in pyproject.toml (supply chain protection)
- Images pinned by sha256 digest in ImageStream manifests

Idle Culling Security Note:
- Code-Server and RStudio implement /api/kernels/ via nginx → httpd → CGI
- All communication within pod (localhost), no network exposure
- CGI scripts run as same user as workbench process
- Acknowledged as fragile; planned replacement with single Go reverse proxy
