RBAC Summary:
- No RBAC resources defined in this repository
- RBAC for deployed notebooks is managed by ODH Notebook Controller and platform operator
- kube-rbac-proxy enforces per-user authorization to Notebook CR

Authentication & Authorization:
- JupyterLab (8888/TCP): OAuth2/OIDC via kube-rbac-proxy sidecar (8443/TCP)
  Policy: User must have RBAC access to Notebook CR
- code-server (8787/TCP): OAuth2/OIDC via kube-rbac-proxy sidecar (8443/TCP)
  Policy: Same as JupyterLab — nginx proxies internal traffic

FIPS Compliance:
- mongocli binary: CGO_ENABLED=1 -tags strictfipsruntime (Go FIPS)
- Python crypto: OpenSSL backend via AIPCC-provided wheels (FIPS-compliant)
- TLS: Inherited from JupyterLab / code-server defaults
- compat-openssl11 explicitly installed in all Konflux Dockerfiles
- System trust store from UBI9/c9s base image

Build Hermeticity:
- OS packages: rpms.lock.yaml (rpm-lockfile-prototype)
- Python deps: pylock.{flavor}.toml + requirements.{flavor}.txt (uv, PEP 751)
- Go deps: go.sum (go mod, mongocli submodule)
- npm deps: package-lock.json (code-server submodule)
- Artifacts: artifacts.lock.yaml (Hermeto)
- All Konflux Dockerfiles enforce fully hermetic builds (zero network during build)

Secrets:
- No secrets defined in this repository
- OAuth/TLS secrets for deployed notebooks managed by ODH Notebook Controller
