RBAC Summary:
- This component does NOT define its own RBAC resources
- RBAC for workbench pods is managed by:
  - odh-notebook-controller: Creates/manages pod security context
  - kube-rbac-proxy sidecar: Enforces OAuth/RBAC at ingress
  - User must have access to the data science project namespace

Secrets:
- DSPA connection secret (Opaque) - Pipeline endpoint URL + credentials
  - Provisioned by: Notebook controller (notebook_dspa_secret.go)
  - Auto-Rotate: No
  - Mount: /opt/app-root/runtimes/
- CA certificate bundle (Opaque) - Custom CA certs
  - Provisioned by: Notebook controller (mutating webhook)
  - Auto-Rotate: No
  - Mount: /etc/pki/tls/custom-certs/ca-bundle.crt
- ServiceAccount token (kubernetes.io/service-account-token)
  - Provisioned by: Kubernetes
  - Auto-Rotate: Yes
  - Mount: /var/run/secrets/kubernetes.io/serviceaccount/

FIPS Compliance:
- mongocli binary: CGO_ENABLED=1, -tags strictfipsruntime (FIPS-compliant Go)
- Python crypto: System OpenSSL from AIPCC/UBI base (FIPS via system)
- compat-openssl11 installed in all images
- No custom TLS configuration in JupyterLab (uses system defaults)
- No non-FIPS crypto risks detected in build logic

Build Hermeticity:
- All builds run with NO network access
- RPMs locked via rpms.lock.yaml
- Python locked via uv.lock (CI) + pylock.*.toml (workbench)
- Go locked via go.sum (mongocli submodule)
- npm locked via package-lock.json (code-server submodule)
- Artifacts locked via artifacts.lock.yaml
