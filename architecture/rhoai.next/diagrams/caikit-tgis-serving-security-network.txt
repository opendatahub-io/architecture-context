RBAC Summary:
- No RBAC resources defined by this component
- RBAC is managed by KServe and the platform operator
- ServiceAccount: default (no custom SA defined)

Service Mesh Configuration:
- PeerAuthentication: STRICT (pod-level)
  - portLevelMtls:
    - 8086: PERMISSIVE (allows Prometheus scraping without mTLS)
- mTLS: STRICT mode for all inference traffic (8080, 8085)
- Sidecar: Istio sidecar injection on inference pods
- Gateway: knative-ingress-gateway (TLS SIMPLE, wildcard-certs)
- Gateway: knative-local-gateway (plaintext, cluster-local)

Secrets:
- wildcard-certs (kubernetes.io/tls) - TLS cert for Knative ingress, Manual/cert-manager, No auto-rotate
- minio-secret (Opaque) - S3 credentials for MinIO model storage, Manual (demo), No auto-rotate
- git_auth_secret (Opaque) - Git auth for Konflux pipeline runs, Pipelines as Code, No auto-rotate

NetworkPolicy:
- Allows ingress from openshift-user-workload-monitoring namespace to port 8086
- Purpose: Permits Prometheus to scrape Caikit metrics

Authentication & Authorization:
- HTTP inference (8080/TCP): mTLS (sidecar) + Istio authorization, PeerAuthentication STRICT
- gRPC inference (8085/TCP): mTLS (sidecar) + Istio authorization, PeerAuthentication STRICT
- Metrics (8086/TCP): PERMISSIVE mTLS (allows plaintext scraping)
- Optional: Authorino token-based authorization for inference endpoints
