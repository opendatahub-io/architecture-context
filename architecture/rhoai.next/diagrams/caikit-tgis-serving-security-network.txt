RBAC Summary:
  No RBAC resources (ClusterRoles, RoleBindings) are defined by this component.
  RBAC is managed by KServe and the platform operator.

Service Mesh Configuration:
  - PeerAuthentication: STRICT (pod-level)
  - PeerAuthentication: PERMISSIVE on port 8086 (portLevelMtls)
    - Allows Prometheus to scrape metrics without mTLS certificates
  - Istio sidecar injection: Required on inference pods

Authentication Summary:
  - HTTP inference (8080/TCP): mTLS (Istio sidecar) + Istio AuthorizationPolicy
  - gRPC inference (8085/TCP): mTLS (Istio sidecar) + Istio AuthorizationPolicy
  - Metrics (8086/TCP): PERMISSIVE mTLS (allows plaintext scraping)
  - TGIS backend (8033/TCP): None (localhost loopback, no auth required)
  - Authorino (optional): Token-based authorization for inference endpoints

Secrets:
  - wildcard-certs (kubernetes.io/tls) — TLS cert for Knative ingress gateway, Manual/cert-manager, No auto-rotate
  - minio-secret (Opaque) — S3 credentials for MinIO model storage (demo), Manual, No auto-rotate
  - git_auth_secret (Opaque) — Git auth for Konflux pipeline runs, Pipelines as Code, No auto-rotate

NetworkPolicy:
  - Allows Prometheus (openshift-user-workload-monitoring) to scrape port 8086/TCP
  - Inference traffic controlled by Istio AuthorizationPolicy

FIPS Compliance:
  - Python container — no direct crypto operations
  - Relies on UBI 9 base image for FIPS-validated OpenSSL
  - Relies on Istio sidecar for mTLS encryption
  - No custom CA bundles or certificate handling
  - Build: Non-hermetic (hermetic: false), poetry.lock for Python deps, no rpms.lock.yaml
