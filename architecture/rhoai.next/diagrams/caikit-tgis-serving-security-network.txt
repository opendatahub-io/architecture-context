RBAC Summary:
  No RBAC resources defined by this component.
  RBAC is managed by KServe and the platform operator (RHOAI).

Service Mesh Configuration:
  - PeerAuthentication: STRICT (pod-level, all ports)
    - Exception: port 8086 — PERMISSIVE (allows Prometheus scraping without mTLS)
  - Istio sidecar injection: Enabled on inference pod
  - mTLS mode: STRICT between all service mesh members
  - Ingress Gateway: knative-ingress-gateway (TLS SIMPLE, wildcard-certs)
  - Local Gateway: knative-local-gateway (plaintext HTTP/8081, cluster-local)

Secrets:
  - wildcard-certs (kubernetes.io/tls) - TLS cert for Knative ingress gateway, manual/cert-manager, no auto-rotation
  - minio-secret (Opaque) - S3 credentials for MinIO model storage (demo), manual provisioning
  - git_auth_secret (Opaque) - Git auth for Konflux pipeline (Tekton), Pipelines as Code provisioning

Network Policies:
  - networkpolicy-uwm: Allows ingress from openshift-user-workload-monitoring namespace to port 8086/TCP

FIPS Compliance:
  - Base image: UBI 9 minimal (FIPS-validated OpenSSL)
  - Application crypto: Delegated to Istio sidecar (mTLS) and system OpenSSL
  - No direct cryptographic operations in application code
  - Build: Non-hermetic (hermetic: false in Tekton PipelineRun)
