Service Mesh Configuration:
- PeerAuthentication: STRICT (namespace-scoped)
  - Exception: Port 8086 set to PERMISSIVE for Prometheus metrics scraping
- NetworkPolicy: Allows ingress from openshift-user-workload-monitoring namespace
  - Destination: port 8086/TCP
  - Purpose: Enable User Workload Monitoring Prometheus scrape access
- Istio Sidecar: Injected into all ServingRuntime pods
  - Provides mTLS encryption for all inter-service communication
  - TLS Passthrough enabled for gRPC streaming (enablePassthrough: "true")

Secrets:
- wildcard-certs (kubernetes.io/tls) - TLS certificate for Knative ingress gateway, manual provisioning, no auto-rotation
- S3 storage credentials (Opaque) - AWS IAM / S3 access credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY), user/operator provisioned, no auto-rotation
- git_auth_secret (Opaque) - Git authentication for Konflux CI/CD builds, Pipelines-as-Code provisioned
