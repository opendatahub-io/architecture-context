RBAC Summary:
- No Kubernetes RBAC resources defined by this component.
- The MLMD server does not interact with the Kubernetes API.
- RBAC is managed by the deploying platform operator (rhods-operator / opendatahub-operator).

Service Mesh Configuration:
- No service mesh sidecar injection defined by this component.
- Network policy and mesh configuration managed by the platform operator.

Secrets:
- Database credentials (Opaque) — MySQL/PostgreSQL username and password
  - Provisioned by: Platform operator / DSP controller
  - Auto-Rotate: No
- TLS certificates (kubernetes.io/tls) — Optional server key, cert, CA for gRPC TLS and mTLS
  - Provisioned by: Platform operator / cert-manager
  - Auto-Rotate: Depends on provisioner

Crypto Notes:
- gRPC TLS via grpc::SslServerCredentials with configurable PEM key/cert/CA
- Default: InsecureServerCredentials() (plaintext) when no SSLConfig provided
- gRPC internally uses BoringSSL; on RHEL/UBI linked against system OpenSSL
- Non-FIPS risk: BoringSSL may use non-FIPS cipher suites unless OS enforces FIPS mode
- Database SSL configurable independently via MySQLDatabaseConfig.SSLOptions / PostgreSQLDatabaseConfig.SSLOptions

Build & Runtime:
- Container: odh-mlmd-grpc-server on UBI9-minimal
- User: non-root (UID 65534)
- Architectures: x86_64, arm64, ppc64le
- Build hermeticity: Not fully enforced (hermetic=false in Tekton PipelineRun)
