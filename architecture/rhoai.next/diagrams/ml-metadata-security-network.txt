Ingress Summary:
- No Kubernetes Ingress resources defined
- MLMD gRPC server is accessed only from within the cluster
- Service type: ClusterIP (not exposed externally)

Egress Summary:
- MySQL/MariaDB: 3306/TCP, MySQL protocol, Optional SSL, Username/password
- PostgreSQL: 5432/TCP, PostgreSQL protocol, Optional SSL, Username/password

Secrets:
- Database credentials (Opaque) — MySQL/PostgreSQL username and password
  - Provisioned by: Platform operator / DSP controller
  - Auto-rotate: No
- TLS certificates (kubernetes.io/tls) — Server key, cert, CA for gRPC TLS and optional mTLS
  - Provisioned by: Platform operator / cert-manager
  - Auto-rotate: Depends on provisioner

RBAC Summary:
- No Kubernetes RBAC resources defined in this component
- MLMD server does not interact with the Kubernetes API
- RBAC is managed by the deploying platform operator (rhods-operator)

Authentication & Authorization:
- Endpoint: MetadataStoreService (gRPC, all 46 RPCs)
- Auth Mechanism: Optional mTLS (client_verify=true in SSLConfig)
- Enforcement Point: gRPC server credentials (C++ grpc::SslServerCredentials)
- Policy: If SSLConfig is provided with client_verify=true, server requires
  and verifies client certificate; otherwise no authentication enforced

FIPS Compliance Notes:
- C++ binary (not Go) — no GOEXPERIMENT/CGO_ENABLED applicable
- Dynamic linking against system libraries via Bazel on UBI9
- OpenSSL available in image (openssl-devel in builder, UBI9-minimal runtime)
- gRPC's BoringSSL may use non-FIPS cipher suites by default unless
  OS-level FIPS mode is enforced
- Server falls back to InsecureServerCredentials() when no SSLConfig provided
