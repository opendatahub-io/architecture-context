RBAC Summary:
- No Kubernetes RBAC defined in this repository
- Container runs as non-root user (UID 65534:65534)
- RBAC is managed by the deploying operator (rhods-operator or opendatahub-operator)

Secrets:
- metadata-store-server-config (Opaque)
  - MetadataStoreServerConfig protobuf text format
  - Contains: database connection details, TLS configuration
  - Provisioned by: Platform operator
  - Auto-Rotate: No

- mysql/postgres credentials (Opaque)
  - Contains: Database user/password for MySQL or PostgreSQL
  - Provisioned by: Platform operator
  - Auto-Rotate: No

- grpc-tls (kubernetes.io/tls)
  - Contains: server_key, server_cert, custom_ca
  - Provisioned by: Platform operator / cert-manager
  - Auto-Rotate: Depends on provisioner

Authentication & Authorization:
- Endpoint: MetadataStoreService (all 40+ RPCs)
- Auth Mechanism: Optional mTLS (when SSLConfig.client_verify=true)
- Enforcement Point: gRPC server (BoringSSL)
- Policy: Client certificate verification against custom CA

Network Exposure:
- NO external ingress (ClusterIP service only)
- NO HTTP endpoints (gRPC only)
- Egress limited to database backends (MySQL 3306/TCP, PostgreSQL 5432/TCP)

Security Notes:
- Connection-per-RPC pattern (no connection pooling) - thread-safe but higher overhead
- No service mesh integration defined in repo (managed by deploying operator)
- TLS is optional but configurable for both client-facing and database connections
- Multi-architecture build: x86_64, arm64, ppc64le on UBI9-minimal
