RBAC Summary:
- No Kubernetes RBAC resources defined in this repository
- MLMD server does NOT interact with the Kubernetes API
- All RBAC is managed by the deploying operator (rhods-operator / opendatahub-operator)

Secrets:
- Database credentials (Opaque)
  - Purpose: MySQL/PostgreSQL username and password
  - Provisioned by: Platform operator / DSP controller
  - Auto-rotate: No

- TLS certificates (kubernetes.io/tls) [Optional]
  - Purpose: Server key, cert, CA for gRPC TLS and optional mTLS
  - Provisioned by: Platform operator / cert-manager
  - Auto-rotate: Depends on provisioner

Authentication & Authorization:
- Endpoint: MetadataStoreService (gRPC, all 46 RPCs)
- Auth Mechanism: Optional mTLS (client_verify=true in SSLConfig)
- Enforcement: grpc::SslServerCredentials (C++)
- Policy: If SSLConfig provided with client_verify=true, server requires
  and verifies client certificate; otherwise NO authentication
- Default: InsecureServerCredentials() (no TLS, no auth)

FIPS Compliance Notes:
- C++ binary (not Go) — no GOEXPERIMENT/CGO_ENABLED applicable
- Dynamic linking against system libraries on UBI9
- OpenSSL available in image (openssl-devel in builder, system OpenSSL in runtime)
- gRPC uses BoringSSL internally — may use non-FIPS cipher suites unless
  OS-level FIPS mode is enforced
- No explicit cipher suite restriction in application code
- Falls back to InsecureServerCredentials() when no SSLConfig provided

Network Exposure:
- Service Type: ClusterIP (internal only)
- No Ingress resources
- No external-facing endpoints
- All access is cluster-internal from DSP and pipeline UI components
