INGRESS:
- None. MLMD has no ingress resources.
- Accessible only by in-cluster clients via direct gRPC calls.

RBAC Summary:
- No Kubernetes RBAC defined in this repository.
- MLMD is a standalone gRPC service that does not interact with the Kubernetes API.
- RBAC for the MLMD pod is defined by the deploying operator (Data Science Pipelines Operator).

Authentication & Authorization:
- Endpoint: MetadataStoreService (all ~45 RPCs)
- Mechanism: Optional mTLS via SSLConfig
  - server_key: Path to server private key
  - server_cert: Path to server certificate
  - custom_ca: Path to custom CA for client cert validation
  - client_verify: Boolean flag to require client certificates
- Enforcement: gRPC server (BoringSSL)
- Default: InsecureServerCredentials when no SSLConfig provided
- No application-level authentication or authorization

Secrets:
- No secrets defined in the repository.
- Database credentials provided externally via:
  - MetadataStoreServerConfig protobuf configuration file
  - Command-line flags (--mysql_config_*, --postgres_config_*)
- TLS certificates provided externally via SSLConfig fields.

FIPS Compliance Notes:
- Crypto library: BoringSSL (Google fork of OpenSSL, statically linked)
- FIPS mode: NOT enabled (-DBORINGSSL_FIPS flag not set in Bazel build)
- Risk: BoringSSL crypto may not be FIPS-validated
- Server defaults to InsecureServerCredentials when SSLConfig absent

Deployment Management:
- Deployed by: Data Science Pipelines Operator
- No self-managed Kubernetes manifests in this repository
- No kustomize overlays or Helm charts
