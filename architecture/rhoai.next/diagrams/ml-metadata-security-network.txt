INGRESS:
- None. The gRPC server is accessed cluster-internally only.
- No Route, Ingress, or Gateway resources.

RBAC Summary:
- No Kubernetes RBAC defined in this repository.
- Container runs as non-root user (UID 65534:65534).
- RBAC is managed by the deploying operator (rhods-operator or opendatahub-operator).

Service Mesh Configuration:
- No service mesh configuration defined in this repository.
- Optional mTLS is handled natively by the gRPC server via BoringSSL.
- SSLConfig protobuf: server_key, server_cert, custom_ca, client_verify fields.

Secrets:
- metadata-store-server-config (Opaque) - MetadataStoreServerConfig protobuf text format
  containing database connection details and TLS configuration.
  Provisioned by: Platform operator. Auto-Rotate: No.

- mysql/postgres credentials (Opaque) - Database user/password for MySQL or PostgreSQL backend.
  Provisioned by: Platform operator. Auto-Rotate: No.

- grpc-tls (kubernetes.io/tls) - Optional TLS certificates for secure gRPC channel
  (server_key, server_cert, custom_ca).
  Provisioned by: Platform operator / cert-manager. Auto-Rotate: Depends on provisioner.

Authentication & Authorization:
- Endpoint: MetadataStoreService (all RPCs)
- Mechanism: Optional mTLS (when SSLConfig.client_verify=true)
- Enforcement Point: gRPC server (BoringSSL)
- Policy: Client certificate verification against custom CA

Notes:
- No HTTP endpoints - gRPC only.
- No CRDs - standalone service, not an operator.
- Connection-per-request pattern (no connection pooling at service layer).
- Multi-architecture support: x86_64, arm64, ppc64le.
- Deployment manifests managed externally by platform operator.
