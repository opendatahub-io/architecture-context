INGRESS:
  No ingress resources defined. Ingress is managed by the consuming
  deployment (e.g., platform operator, service mesh sidecar, or
  kube-rbac-proxy).

RBAC Summary:
  No Kubernetes RBAC resources. MLMD runs as a standalone gRPC service.
  Access control relies entirely on:
  - Network-level isolation (ClusterIP, NetworkPolicies)
  - Optional mTLS (client_verify in SSLConfig)
  - Upstream proxy (e.g., kube-rbac-proxy in RHOAI)

FIPS Considerations:
  - Crypto library: BoringSSL (statically linked via Bazel, NOT system OpenSSL)
  - BoringSSL is FIPS-capable but FIPS mode is NOT explicitly enabled in this build
  - C++ binary — no Go GOEXPERIMENT/strictfipsruntime flags applicable
  - Risk: Server binary uses BoringSSL from Bazel rather than dynamically linking
    against system OpenSSL, potentially bypassing FIPS enforcement on the host

Security Concerns:
  - No per-RPC authorization: any authenticated client can perform all operations
  - Default mode is InsecureServerCredentials (no TLS, no auth)
  - No connection pooling (new DB connection per RPC) — potential DoS vector
  - No RPM-level lock file — OS package versions not pinned
  - No Hermeto/cachi2 prefetch — Bazel fetches from internet during build
