## Purpose

**Short**: Caikit-NLP is a Python library providing NLP capabilities (text generation, embedding, reranking, classification, tokenization) built on the caikit framework, designed to run as a module within the caikit runtime.

**Detailed**: Caikit-NLP extends the caikit AI toolkit framework with Natural Language Processing capabilities. It provides a collection of caikit modules that wrap HuggingFace Transformers and sentence-transformers models, exposing them through caikit's standardized task-based API. The library supports both local model execution and remote inference via TGIS (Text Generation Inference Server) backends.

The library is not a standalone service — it is packaged as a Python wheel (`caikit_nlp`) and installed into the caikit runtime container. When loaded as a runtime library (configured via `RUNTIME_LIBRARY=caikit_nlp`), it registers its modules with caikit's module registry, making them available through caikit's gRPC and HTTP serving infrastructure. This architecture allows caikit-nlp models to be served via KServe's InferenceService CRDs in the RHOAI platform, where the caikit runtime container acts as the model server.

Key capabilities include: text generation (local HuggingFace models and remote TGIS), PEFT prompt tuning (vanilla and multitask), text embeddings via sentence-transformers, cross-encoder reranking, sequence classification, token classification with span filtering, and regex-based sentence splitting. The library also includes a TGIS auto-finder for automatic discovery of text generation models available on remote TGIS servers.

## Architectural Analysis

Caikit-NLP follows a modular plugin architecture built on the caikit framework. Each NLP capability (text generation, embedding, reranking, classification) is implemented as a caikit `@module` — a class that declares which caikit tasks it implements (e.g., `EmbeddingTask`, `TextGenerationTask`) and registers task methods via `@TaskName.taskmethod()` decorators. This design decouples the NLP implementation from the serving infrastructure: caikit's runtime automatically discovers registered modules and exposes their task methods through gRPC and HTTP servers. The library's `__init__.py` calls `caikit.configure(CONFIG_PATH)` at import time, bootstrapping the module registry.

A distinctive pattern in caikit-nlp is the local/remote duality for text generation. The `TextGeneration` module runs HuggingFace models directly in-process, while `TextGenerationTGIS` delegates to a remote TGIS server via gRPC. The same duality exists for PEFT prompt tuning: `PeftPromptTuning` handles local training and inference, while `PeftPromptTuningTGIS` runs inference against TGIS with cached prompt vectors. The TGIS modules use `caikit_tgis_backend` for connection management and prompt artifact loading. The `TGISAutoFinder` extends this further by providing automatic model discovery — it checks whether a TGIS server can serve a given model ID and creates the appropriate module config dynamically.

The embedding module (`EmbeddingModule`) extends `SentenceTransformer` with a custom `SentenceTransformerWithTruncate` class that adds explicit truncation control and thread-safe tokenization. A notable implementation detail is the per-thread tokenizer copy (`self.tokenizers[thread_id]`), which works around "Already borrowed" errors when the HuggingFace fast tokenizer is used concurrently with different truncation settings. The same thread-safety pattern appears in `CrossEncoderWithTruncate`. This suggests the caikit runtime serves requests concurrently, and the module implementations must be thread-safe.

The training infrastructure supports two paradigms: full fine-tuning via `TextGeneration.train()` (using HuggingFace's `Trainer` API with FSDP for multi-GPU) and parameter-efficient prompt tuning via `PeftPromptTuning.train()` (using the PEFT library with HuggingFace Accelerate). The prompt tuning module exports trained prompt vectors as PyTorch tensors (`encoder.pt`, `decoder.pt`) that can be loaded into TGIS for serving. The torch distributed launch configuration (`toolkit/torch_run.py`) automatically detects available GPUs and configures elastic launch parameters, enabling multi-GPU training without explicit user configuration.

Security-wise, the library is conservative by default: `allow_downloads` is set to `false` in `config.yml`, preventing arbitrary model downloads from the internet. The `trust_remote_code` setting defaults to `false` for embedding models. TGIS connections support optional TLS with CA certificate and client certificate authentication. The container runs as a non-root user (UID 1001, `caikit` user) with a dedicated group.
