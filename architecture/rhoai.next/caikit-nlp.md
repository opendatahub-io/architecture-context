## Purpose

**Short**: Caikit-NLP is a Python library providing Natural Language Processing (NLP) capabilities — text generation, embedding, reranking, classification, and tokenization — built on top of the Caikit AI framework.

**Detailed**: Caikit-NLP serves as a runtime library (not a standalone service) that extends the Caikit AI framework with NLP-specific modules. It is loaded by the Caikit runtime via the `RUNTIME_LIBRARY=caikit_nlp` environment variable and provides a set of AI task modules that the runtime exposes over gRPC and HTTP endpoints. The library wraps HuggingFace Transformers, Sentence Transformers, and PEFT (Parameter-Efficient Fine-Tuning) to provide production-ready NLP capabilities including text generation (local and remote via TGIS), text embedding with sentence-transformers, cross-encoder reranking, sequence classification, token classification, and sentence tokenization.

The library supports two execution modes for text generation: local inference using HuggingFace models directly, and remote inference through the TGIS (Text Generation Inference Server) backend via gRPC. For embeddings and reranking, it runs models locally using sentence-transformers and cross-encoder models. Prompt tuning (vanilla and multitask) is supported through PEFT, enabling fine-tuning of large language models with parameter-efficient methods. The library integrates with PyTorch for GPU acceleration, supports IPEX (Intel Extension for PyTorch) optimization, and includes distributed training via torch elastic launch.

## Architectural Analysis

Caikit-NLP follows the Caikit framework's module-based architecture, where each NLP capability is implemented as a self-contained module registered with the framework via the `@module()` decorator. This design enables the Caikit runtime to auto-discover modules and auto-generate gRPC/HTTP API surfaces from task method signatures, without the library defining explicit API routes. The library itself has no network-facing code — all serving infrastructure is delegated to the Caikit runtime.

The text generation subsystem has a notable dual-mode architecture: `TextGeneration` runs HuggingFace models locally in-process, while `TextGenerationTGIS` delegates inference to a remote TGIS server over gRPC. This split is mirrored in the PEFT modules (`PeftPromptTuning` for local, `PeftPromptTuningTGIS` for remote). The `TGISAutoFinder` model finder automatically detects which models are available remotely and creates the appropriate module configuration, enabling transparent local-vs-remote model resolution. TGIS connections support TLS with configurable CA certificates and client certificates for mTLS, though these are optional and default to plaintext.

The embedding subsystem (`EmbeddingModule`) includes a custom `SentenceTransformerWithTruncate` wrapper that adds thread-safe tokenization (per-thread tokenizer copies to avoid "Already borrowed" errors from Rust-based fast tokenizers), configurable input token truncation, and input token counting. The `CrossEncoderModule` follows the same pattern with `CrossEncoderWithTruncate`. Both modules support configurable optimization: PyTorch 2.0 compile, IPEX optimization for Intel hardware, and autocast for bfloat16 inference.

Security-wise, the library defaults to `allow_downloads: false` in its configuration, preventing arbitrary model downloads from HuggingFace Hub. Model paths are validated with regex to prevent directory traversal attacks (the `TWO_DOTS_REGEX` check in `peft_config.py`). The Dockerfile runs as a non-root user (UID 1001) in the `caikit` group. However, the container image uses a generic UBI base rather than AIPCC base images, which means Python dependencies are not sourced from Red Hat's secure build pipeline — this is a notable gap for a component that ships PyTorch and HuggingFace libraries with native extensions.

The training architecture supports multi-GPU distributed training via PyTorch elastic launch (`torch.distributed.launcher.api.elastic_launch`), with automatic world size detection based on available CUDA devices. The PEFT prompt tuning module uses HuggingFace Accelerate for mixed-precision training (fp16/bf16) and gradient accumulation. Training metadata (loss history per step/epoch) is persisted as JSONL files alongside model artifacts, enabling training observability.
