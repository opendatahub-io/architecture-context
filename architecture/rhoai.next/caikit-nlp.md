## Purpose

**Short**: Caikit-NLP is a Python library providing NLP capabilities (text generation, embeddings, reranking, classification, tokenization) built on the caikit AI framework, designed to be loaded by the caikit runtime for serving NLP models via gRPC and HTTP.

**Detailed**: Caikit-NLP is an extension library for the caikit AI toolkit that provides Natural Language Processing modules. It is not a standalone service — it is packaged as a Python wheel (`caikit_nlp`) and loaded by the caikit runtime server at startup via the `RUNTIME_LIBRARY=caikit_nlp` environment variable. The caikit runtime then exposes these NLP modules through its gRPC and HTTP serving infrastructure.

The library implements several NLP task modules: text generation (both local HuggingFace transformers execution and remote TGIS backend delegation), text embedding and similarity using sentence-transformers, document reranking using both bi-encoder (embedding) and cross-encoder models, prompt tuning via PEFT (Parameter-Efficient Fine-Tuning), text classification, token classification, and tokenization. Each module follows the caikit module interface pattern with `load()`, `save()`, `run()`, and optionally `train()` methods.

A key architectural feature is the dual-backend design for text generation: the `TextGeneration` module runs models locally using HuggingFace transformers, while `TextGenerationTGIS` and `PeftPromptTuningTGIS` delegate inference to a remote TGIS (Text Generation Inference Server) backend over gRPC. This allows the same caikit runtime to serve models either locally or via a high-performance remote inference server. The TGIS integration is provided by the `caikit-tgis-backend` package, which manages model connections and gRPC communication.

## Architectural Analysis

Caikit-NLP follows a modular library architecture where each NLP capability is encapsulated as a caikit "module" — a class decorated with `@module()` that implements standardized `load()`, `save()`, `run()`, and optionally `train()` methods. This design allows the caikit runtime to dynamically discover and expose these capabilities without caikit-nlp needing to know about network protocols or serving infrastructure. The separation is clean: caikit-nlp provides domain logic, the caikit runtime provides serving.

The dual-backend pattern for text generation is architecturally significant. The `TextGeneration` module loads and runs HuggingFace models locally (suitable for smaller models or development), while `TextGenerationTGIS` delegates to a remote TGIS server (suitable for production serving of large models). The TGIS modules use a `@module(backend_type=TGISBackend.backend_type, base_module=TextGeneration)` decorator pattern that tells the caikit runtime to prefer the TGIS-backed module when a TGIS backend is configured, transparently routing requests without client awareness. This same pattern applies to `PeftPromptTuning` (local) vs `PeftPromptTuningTGIS` (remote). The `TGISAutoFinder` model finder further automates this by detecting TGIS-accessible models and creating appropriate module configurations.

The embedding and reranking modules (`EmbeddingModule` and `CrossEncoderModule`) implement a custom truncation layer (`SentenceTransformerWithTruncate`, `CrossEncoderWithTruncate`) on top of the sentence-transformers library. This addresses a practical production concern: input texts that exceed model token limits must be handled gracefully. The truncation logic supports three modes — error on truncation (default, for safety), silent truncation to a specified limit, or pass-through to the model's default behavior. The per-thread tokenizer copying pattern (`self.tokenizers[thread_id]`) works around thread-safety issues in HuggingFace's fast tokenizers, which raise "Already borrowed" errors when concurrent threads modify truncation settings.

The training subsystem supports distributed multi-GPU training via PyTorch's elastic launch (`torch.distributed.launcher.api.elastic_launch`). For full fine-tuning (`TextGeneration.train()`), it uses FSDP (Fully Sharded Data Parallel) when multiple GPUs are available. For prompt tuning (`PeftPromptTuning.train()`), it uses the HuggingFace Accelerate library with gradient accumulation and mixed precision. The prompt tuning module exports trained prompt vectors as PyTorch tensors (`encoder.pt`, `decoder.pt`) compatible with TGIS's prompt loading interface, enabling a workflow where prompts are tuned locally and then deployed to a remote TGIS server.

The configuration system layers library-specific config (`caikit_nlp/config/config.yml`) on top of the caikit core configuration, and integrates with a runtime configuration (`runtime_config.yaml`) that manages TGIS connections, model finders, and backend priorities. Security-relevant settings include `allow_downloads` (disabled by default, preventing arbitrary URL fetching during model bootstrap), `trust_remote_code` (disabled by default for embeddings), and TLS certificate paths for TGIS connections. The `base_models_dir` and `source_prompt_base` settings restrict filesystem access for model and prompt loading. A notable pattern is path validation for base model names using regex to prevent directory traversal attacks (`TWO_DOTS_REGEX` in `peft_config.py`).
