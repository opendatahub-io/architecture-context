════════════════════════════════════════════════════════════════════════════
TRUST BOUNDARIES
════════════════════════════════════════════════════════════════════════════

1. External → Ingress (DMZ)
   - Trust Level: Untrusted → Semi-trusted
   - Enforcement: OpenShift Router (TLS termination or passthrough)
   - AuthN: Optional Bearer Token (per ISVC configuration)

2. Ingress → Service Mesh
   - Trust Level: Semi-trusted → Trusted
   - Enforcement: Istio Ingress Gateway + PeerAuthentication (mTLS STRICT)
   - AuthN: mTLS client certificates (service identity)

3. Service Mesh → Model Container
   - Trust Level: Trusted → Trusted
   - Enforcement: Localhost (same pod)
   - AuthN: None (implicit trust within pod)

4. Monitoring → Service Mesh
   - Trust Level: Trusted → Trusted
   - Enforcement: NetworkPolicy + RBAC
   - AuthN: ServiceAccount Token

5. Controller → Kubernetes API
   - Trust Level: Trusted → Trusted
   - Enforcement: Kubernetes RBAC + ServiceAccount Token
   - AuthN: ServiceAccount Token
