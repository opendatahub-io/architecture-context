═══════════════════════════════════════════════════════════════════════════
DEPLOYMENT MODES
═══════════════════════════════════════════════════════════════════════════

Serverless Mode (Default):
├─ Uses Knative Serving for autoscaling and scale-to-zero
├─ Istio VirtualService for traffic routing
├─ Traffic flow: Istio Gateway → Knative Activator → Predictor Pod
└─ Ports: 443/TCP (external) → 8012/TCP (activator) → 8080/TCP (predictor)

RawDeployment Mode:
├─ Uses standard Kubernetes Deployments
├─ Kubernetes Ingress for traffic routing
├─ Traffic flow: Kubernetes Ingress → Predictor Service → Predictor Pod
└─ Ports: 443/TCP (external) → 8080/TCP (service) → 8080/TCP (pod)

ModelMesh Mode:
├─ High-density model serving with shared model servers
├─ Not detailed in this diagram (different architecture)
└─ Purpose: Serve many small models efficiently
