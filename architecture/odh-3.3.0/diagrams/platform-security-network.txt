SERVICE MESH CONFIGURATION
═══════════════════════════════════════════════════════════════════════════

PeerAuthentication (Namespace-scoped):
- Mode: PERMISSIVE (default) or STRICT (recommended for production)
- mTLS: Automatic (Istio sidecar injection)
- Applies to: KServe predictors, Model Registry, internal services

AuthorizationPolicy Examples:
- kserve-predictor-access
  - Allows: cluster.local/ns/knative-serving/sa/activator
  - To: {user-namespace}/predictor pods
  - Methods: GET, POST

- model-registry-access
  - Allows: cluster.local/ns/{user-namespace}/sa/{predictor-sa}
  - To: opendatahub/model-registry service
  - Methods: GET (gRPC)

NetworkPolicy:
- Default: Deny all ingress/egress (if enabled)
- Explicit allow rules:
  • ODH Dashboard → Kubernetes API (6443/TCP)
  • Notebooks → MLflow, Model Registry, S3
  • KServe → Model Registry (9090/TCP gRPC), S3 (443/TCP)
  • All pods → kube-dns (53/UDP)

SECRETS MANAGEMENT
═══════════════════════════════════════════════════════════════════════════

TLS Certificates (kubernetes.io/tls):
- *-webhook-server-cert (All operators)
  - Issuer: cert-manager (ClusterIssuer: selfsigned or ACME)
  - Rotation: 90 days (automatic via cert-manager)
  - Used for: Webhook servers (ValidatingWebhookConfiguration)

- odh-dashboard-tls, model-registry-tls, mlflow-tls
  - Issuer: OpenShift Router or cert-manager
  - Used for: HTTPS endpoints

Storage Credentials (Opaque):
- storage-config (per user-namespace)
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_DEFAULT_REGION
  - Used by: KServe, Notebooks, Pipelines
  - Alternative: IRSA (IAM Roles for Service Accounts)

Database Credentials (Opaque):
- model-registry-db-secret
  - Keys: username, password, host, port, database
- mlflow-db-secret
  - Keys: username, password, host, port, database
- pipelines-db-secret
  - Keys: username, password, host, port, database

OAuth Certificates (kubernetes.io/tls):
- odh-dashboard-oauth-proxy-cert
- model-registry-oauth-proxy-cert
- mlflow-oauth-proxy-cert
  - Used for: kube-rbac-proxy sidecar authentication

API Keys (Opaque):
- huggingface-token (TrustyAI, Notebooks)
  - Key: HF_TOKEN
- ngc-api-key (odh-model-controller for NVIDIA NIM)
  - Key: NGC_API_KEY

Pull Secrets (kubernetes.io/dockerconfigjson):
- {component}-pull-secret
  - Used for: Pulling images from private registries (quay.io, NGC)

EXTERNAL DEPENDENCIES
═══════════════════════════════════════════════════════════════════════════

Required for Platform Operation:
- Istio (service mesh) - OPTIONAL but recommended for mTLS
- Knative Serving - REQUIRED for KServe autoscaling
- cert-manager - REQUIRED for webhook TLS certificates

Required for Component Functionality:
- S3-compatible storage - REQUIRED for model artifacts, data, pipelines
- PostgreSQL/MySQL - REQUIRED for Model Registry, MLflow, Pipelines metadata
- GPU Drivers (NVIDIA/AMD) - OPTIONAL for GPU workloads

Optional Integrations:
- HuggingFace Hub - For LLM model downloads
- Container Registry - For custom images (built-in OpenShift registry available)
- LDAP/OAuth Provider - For user authentication (OpenShift OAuth by default)
