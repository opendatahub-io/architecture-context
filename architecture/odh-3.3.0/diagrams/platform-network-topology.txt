NOTES:
- Service mesh (Istio) integration is optional; when enabled, provides mTLS for
  service-to-service communication
- External dependencies (PostgreSQL, Redis, S3, BigQuery) are deployment-specific;
  endpoints and credentials are configurable per installation
- Multi-tenancy is implemented via Kubernetes namespaces; user workspaces are isolated
- All webhook certificates are managed by cert-manager or OLM
- OAuth integration requires OpenShift; on upstream Kubernetes, alternative auth
  mechanisms (OIDC, Dex) can be configured
