NAMESPACES:
- opendatahub-operator-system: OpenDataHub Operator, Controller Manager, Webhooks
- opendatahub: ODH Dashboard, shared platform resources
- redhat-ods-applications: Notebook Controller, component operators
- feast-operator-system: Feast Operator controller
- {user-namespaces}: User notebooks, FeatureStore instances, MLflow workspaces

SERVICE MESH CONFIGURATION:
- Mode: PERMISSIVE mTLS (optional, configurable per component)
- Istio Integration: Optional for Feast and Kubeflow Notebooks
- PeerAuthentication: Optional configuration for Feast services, Notebook services
- AuthorizationPolicy: Optional per-component (Feast feature server, Notebook endpoints)
- Certificate Rotation: Managed by Istio control plane when enabled

RBAC SUMMARY (Top 5 Most Privileged ClusterRoles):
1. controller-manager (OpenDataHub Operator)
   - datasciencecluster.opendatahub.io/*, dscinitialization.opendatahub.io/*
   - components.platform.opendatahub.io/* (all component CRDs)
   - Core resources: namespaces, deployments, roles, rolebindings, secrets, configmaps (*)

2. notebook-controller-role (Kubeflow Notebooks)
   - kubeflow.org/notebooks (*)
   - datasciencepipelinesapplications.opendatahub.io/* (get, list, watch, create, delete, patch, update)
   - oauth.openshift.io/oauthclients (delete, get, list, patch, update, watch)

3. odh-dashboard (ODH Dashboard)
   - kubeflow.org/notebooks (get, list, watch, create, update, patch, delete)
   - modelregistry.opendatahub.io/modelregistries (get, list, watch, create, update, patch, delete)
   - feast.dev/featurestores (get, list, watch)

4. manager-role (Feast Operator)
   - feast.dev/featurestores, featurestores/status, featurestores/finalizers (*)
   - apps/deployments, batch/cronjobs (get, list, watch, create, update, delete)

5. mlflow-k8s-workspace-provider (MLflow)
   - Core/namespaces (list, watch)
   - mlflow.kubeflow.org/mlflowconfigs (list, watch)
   - authorization.k8s.io/subjectaccessreviews (create)

AUTHENTICATION FLOWS:
1. External User → ODH Dashboard:
   - Browser → OpenShift Route (TLS termination)
   - Route → kube-rbac-proxy (OAuth validation)
   - kube-rbac-proxy → Dashboard Backend (user headers injected)

2. Notebook → MLflow Experiment Tracking:
   - Notebook Pod → MLflow SDK (Bearer Token from SA)
   - MLflow SDK → Istio Gateway (TLS 1.2+)
   - Istio Gateway → MLflow Server (optional mTLS)
   - MLflow Server → K8s API (SubjectAccessReview for RBAC)

3. Notebook → Feast Feature Retrieval:
   - Notebook Pod → Feast Python SDK (injected config from ConfigMap)
   - Feast SDK → Feast Online Feature Server (gRPC/6566 or HTTP/80, optional mTLS)
   - Feast Server → Redis/PostgreSQL (credentials from secret)
