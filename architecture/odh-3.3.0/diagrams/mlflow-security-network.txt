SECURITY CONTROLS:
═══════════════════

Authentication & Authorization:
  • External → Gateway: HTTPS with TLS 1.2+ termination at Istio Gateway
  • Client → MLflow: Bearer Token (Kubernetes ServiceAccount token)
    - Passed in Authorization header or X-Forwarded-Access-Token
  • MLflow → K8s API: ServiceAccount token for workspace provider and auth plugin
  • RBAC Enforcement: Every MLflow API request validated via SubjectAccessReview
  • User permissions checked against mlflow.kubeflow.org API group resources

RBAC Roles:
  • mlflow-k8s-workspace-provider ClusterRole:
    - namespaces: list, watch
    - mlflowconfigs: list, watch
    - secrets (mlflow-artifact-connection): get
    - subjectaccessreviews: create (for RBAC enforcement)

Custom API Resources (mlflow.kubeflow.org):
  • experiments: get, list, create, update, delete
  • datasets: get, list, create, update, delete
  • registeredmodels: get, list, create, update, delete (includes prompts)
  • assistants: get, create, update (localhost only)
  • gatewaysecrets: get, list, create, update, delete
  • gatewaysecrets/use: create
  • gatewayendpoints: get, list, create, update, delete
  • gatewayendpoints/use: create
  • gatewaymodeldefinitions: get, list, create, update, delete
  • gatewaymodeldefinitions/use: create

Encryption:
  • External → Gateway: TLS 1.2+ (HTTPS/443)
  • Gateway → MLflow Service: HTTP (TLS termination at gateway)
  • Service Mesh: Optional mTLS via Istio
  • MLflow → PostgreSQL: TLS 1.2+ optional (5432/TCP)
  • MLflow → S3: TLS 1.2+ (HTTPS/443)
  • MLflow → K8s API: TLS 1.2+ (HTTPS/6443)

Secrets Management:
  • mlflow-artifact-connection: S3 credentials (user-provided, no auto-rotate)
    - AWS_S3_BUCKET, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
  • mlflow-db-credentials: PostgreSQL credentials (ODH operator, no auto-rotate)

Multi-Tenancy:
  • Kubernetes Workspace Provider maps namespaces to MLflow workspaces
  • Per-namespace artifact storage override via MLflowConfig CRD
  • RBAC enforcement ensures users can only access resources in authorized namespaces

Monitoring:
  • Prometheus exporter exposes MLflow server metrics
  • Health endpoints: /health, /version (unauthenticated)
  • UI telemetry: /ajax-api/3.0/mlflow/ui-telemetry
