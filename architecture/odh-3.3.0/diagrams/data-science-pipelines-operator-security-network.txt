RBAC SUMMARY
────────────────────────────────────────────────────────────────────────────────

ClusterRoles:
  • manager-role
    - Manages ConfigMaps, Secrets, ServiceAccounts, Services, PVCs, PVs
    - Full control over Pods (create, delete, exec, logs)
    - Manages Deployments, ReplicaSets
    - Manages DataSciencePipelinesApplication CRDs
    - Manages Argo Workflows (Workflow, WorkflowTemplate, CronWorkflow)
    - Manages KFP Pipelines and PipelineVersions
    - Manages ScheduledWorkflows
    - Manages OpenShift Routes
    - Manages Admission Webhooks

  • argo-cluster-role
    - Manages Pods (create, exec, get, list, watch, update, patch, delete)
    - Reads ConfigMaps
    - Manages Argo Workflows and WorkflowTemplates

RoleBindings:
  • manager-rolebinding: Binds manager-role to controller-manager SA
  • leader-election-rolebinding: Binds leader-election-role to controller-manager SA
  • ds-pipeline-argo-binding: Binds argo-cluster-role to ds-pipeline-argo SA

────────────────────────────────────────────────────────────────────────────────
SECRETS MANAGEMENT
────────────────────────────────────────────────────────────────────────────────

Secrets:
  • ds-pipeline-db-{name} (Opaque)
    - Purpose: MariaDB connection credentials
    - Provisioned: DSPO
    - Auto-Rotate: No

  • ds-pipeline-s3-{name} (Opaque)
    - Purpose: S3/Minio access credentials
    - Provisioned: DSPO
    - Auto-Rotate: No

  • ds-pipeline-tls (kubernetes.io/tls)
    - Purpose: API server TLS certificates
    - Provisioned: cert-manager
    - Auto-Rotate: Yes

  • webhook-server-cert (kubernetes.io/tls)
    - Purpose: Webhook server TLS certificates
    - Provisioned: cert-manager
    - Auto-Rotate: Yes

────────────────────────────────────────────────────────────────────────────────
AUTHENTICATION & AUTHORIZATION
────────────────────────────────────────────────────────────────────────────────

Endpoint: /apis/v2beta1/*
  • Auth: Bearer Token (Kubernetes ServiceAccount token)
  • Enforcement: API Server
  • Policy: Kubernetes RBAC

Endpoint: ML Metadata gRPC
  • Auth: mTLS client certificates
  • Enforcement: Envoy Proxy
  • Policy: Certificate validation

Endpoint: MariaDB
  • Auth: Username/Password
  • Enforcement: MariaDB Server
  • Policy: Database authentication

Endpoint: Minio
  • Auth: Access Key/Secret Key
  • Enforcement: Minio Server
  • Policy: IAM-style authentication

────────────────────────────────────────────────────────────────────────────────
SERVICE MESH CONFIGURATION
────────────────────────────────────────────────────────────────────────────────

Service Mesh: Not explicitly configured
  • mTLS enabled for ML Metadata gRPC endpoints
  • TLS 1.2+ required for API server and database connections
  • Internal services use HTTP without encryption (persistence agent, scheduled workflow)
  • External Routes use Edge TLS termination
