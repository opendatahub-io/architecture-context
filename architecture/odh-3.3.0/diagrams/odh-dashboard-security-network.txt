Authentication Flow:
1. User requests dashboard URL (HTTPS/443)
2. OpenShift Router redirects to OAuth Server
3. User authenticates with OpenShift credentials
4. OAuth Server issues Bearer Token (stored in cookie)
5. User browser sends authenticated request to kube-rbac-proxy (HTTPS/8443)
6. kube-rbac-proxy validates token with OAuth Server
7. kube-rbac-proxy forwards to backend with X-Auth-Request-User header (HTTP/8080)
8. Backend uses ServiceAccount token for Kubernetes API calls (HTTPS/6443)
