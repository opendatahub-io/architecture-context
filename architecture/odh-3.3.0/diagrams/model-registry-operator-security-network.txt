Trust Boundaries
═══════════════════════════════════════════════════════════════════════════

1. EXTERNAL → INGRESS
   - TLS termination at OpenShift Route (Edge mode)
   - User authentication via Session Cookie or Bearer Token

2. INGRESS → APPLICATION
   - OAuth Proxy validates Bearer Tokens with OAuth Service
   - RBAC enforcement via registry-user Role

3. APPLICATION → DATABASE
   - Optional TLS encryption (model-registry-db-tls secret)
   - Password-based authentication
   - NetworkPolicy restricts access to Model Registry namespace only

4. OPERATOR → KUBERNETES API
   - ServiceAccount Token authentication
   - ClusterRole manager-role with RBAC permissions

5. APPLICATION → EXTERNAL SERVICES
   - OAuth Service: HTTPS/TLS 1.2+ for token validation
   - S3 Storage: HTTPS/TLS 1.2+ with AWS credentials
   - Prometheus: HTTPS/TLS 1.2+ with mTLS
