SECURITY CONTROLS:
═══════════════════

Authentication & Authorization:
  • Webhook: mTLS client certificates (K8s API Server → webhook)
  • Notebook Access: OAuth Proxy or Istio AuthorizationPolicy
  • Controller → K8s API: ServiceAccount tokens
  • User → Notebook: OAuth token via OpenShift OAuth Server

RBAC Roles:
  • notebook-controller-role: Full control over Notebooks, StatefulSets, Services
  • odh-notebook-controller-manager-role: Manages OAuth clients, Routes,
    NetworkPolicies, RBAC, DSPA integration, Gateway API resources
  • notebook-controller-leader-election-role: Leader election via ConfigMaps/Leases

Encryption:
  • External → Ingress: TLS 1.2+ (HTTPS/443)
  • Ingress → OAuth Proxy: HTTP/80 (internal)
  • OAuth Proxy → Notebook: HTTP (internal, protected by NetworkPolicy)
  • Controller → K8s API: TLS 1.2+ (HTTPS/6443)
  • Webhook: TLS 1.2+ with mTLS

Secrets Management:
  • odh-notebook-controller-webhook-cert: TLS cert for webhook
    (OpenShift Service CA: auto-rotate, cert-manager: manual)
  • {notebook-name}-oauth-config: OAuth client secret (manual, no auto-rotate)
  • ds-pipeline-{dspa-name}-config: DSPA credentials (manual, no auto-rotate)

Network Policies:
  • Created by odh-notebook-controller for each notebook
  • Controls ingress/egress traffic for notebook pods
  • Enforces isolation between notebook instances

Service Mesh Configuration:
  • Istio VirtualServices: Optional for notebook ingress routing
  • Istio AuthorizationPolicy: Alternative to OAuth for notebook access
  • Gateway API: HTTPRoute + ReferenceGrant for cross-namespace routing

Monitoring:
  • Prometheus scrapes:
    - odh-notebook-controller metrics (8080/TCP HTTP)
    - Notebook pod metrics (notebook-port HTTP)
    - Culling controller uses metrics for idle detection

Culling:
  • Notebook culler monitors /metrics endpoint on notebook pods
  • Stops notebooks idle for CULL_IDLE_TIME (default: 24h)
  • Checks every IDLENESS_CHECK_PERIOD (default: 1 min)
