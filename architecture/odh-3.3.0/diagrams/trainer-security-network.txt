Security Considerations:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️  PLAINTEXT COMMUNICATION:
    - Inter-worker communication (23456/TCP) is UNENCRYPTED
    - Risk: Potential eavesdropping on distributed training traffic
    - Mitigation: Use NetworkPolicies to restrict to same TrainJob pods
    - Alternative: Enable mTLS via service mesh (not default)

✅  TLS ENCRYPTION:
    - All external communications use TLS 1.2+
    - Webhook server uses TLS with certificate validation
    - S3, HuggingFace, PyPI use HTTPS

⚠️  MODEL REGISTRY HTTP:
    - Model Registry API uses HTTP (no encryption)
    - Risk: Model metadata exposed in transit
    - Mitigation: Deploy in trusted cluster network, use service mesh

🔐  SECRET MANAGEMENT:
    - User-provided secrets (S3, HuggingFace) require manual rotation
    - Recommendation: Use external secret management (Vault, AWS Secrets Manager)
    - Webhook certificates auto-rotate via cert-manager
