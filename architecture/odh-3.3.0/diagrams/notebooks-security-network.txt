RBAC Summary:
- No custom ClusterRoles defined in notebooks component
- Workbench pods run with permissions granted by Notebook Controller's ServiceAccount
- RBAC is managed by the ODH Notebook Controller component
- Minimal permissions for workbench pod execution

Service Configuration:
- Service Name: notebook
- Type: ClusterIP
- Port: 8888/TCP → 8888/TCP
- Protocol: HTTP
- Encryption: None (internal cluster traffic)
- Auth: Jupyter Notebook Token

Secrets:
- notebook-tls-secret (kubernetes.io/tls)
  - Purpose: TLS certificates for HTTPS ingress
  - Provisioned By: cert-manager / OpenShift
  - Auto-Rotate: Yes

- notebook-oauth-secret (Opaque)
  - Purpose: OAuth proxy authentication token
  - Provisioned By: ODH Dashboard
  - Auto-Rotate: No

- aws-credentials (Opaque)
  - Purpose: S3 bucket access credentials for datasets
  - Content: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
  - Provisioned By: User-provided
  - Auto-Rotate: No

- git-credentials (kubernetes.io/ssh-auth)
  - Purpose: SSH keys for private git repository access
  - Provisioned By: User-provided
  - Auto-Rotate: No

Authentication & Authorization:
- Jupyter Lab Endpoints (/lab, /api):
  - Auth Mechanism: Jupyter Token
  - Enforcement Point: Jupyter Server
  - Policy: Token-based access control

- Ingress HTTPS (OpenShift Route):
  - Auth Mechanism: OAuth Proxy
  - Enforcement Point: OpenShift Route
  - Policy: User authentication via OpenShift OAuth

Network Policies:
- Ingress: Allow traffic from OpenShift Router to notebook pods on 8888/TCP
- Egress: Allow outbound HTTPS (443/TCP) to external services (PyPI, S3, Git)
- Internal: Allow HTTP traffic to Model Registry API (8080/TCP)
