SECURITY CONTROLS:
═══════════════════

Authentication & Authorization:
  • Istio Ingress: Bearer Token authentication for client requests
  • Webhook API: mTLS with certificate validation
  • Service Mesh: mTLS between all internal services
  • LLM API: Bearer Token (API key) for OpenAI-compatible endpoints
  • K8s API: ServiceAccount tokens for controller operations

RBAC Roles:
  • kserve-manager-role: Full control over InferenceServices, CRDs, Deployments
  • llmisvc-manager-role: Manages LLM inference services
  • localmodel-manager-role: Manages local model caching

Encryption:
  • External → Ingress: TLS 1.2+ (HTTPS/443)
  • Ingress → Services: mTLS via Istio service mesh
  • Controller → K8s API: TLS 1.3 (HTTPS/6443)
  • Model Download: TLS 1.2+ with AWS SigV4/OAuth2/Azure AD

Secrets Management:
  • kserve-webhook-server-cert: TLS cert (cert-manager, auto-rotate)
  • storage-config: S3/GCS/Azure credentials (manual, no auto-rotate)
  • {isvc-name}-sa-token: Service account tokens (auto-rotate)
  • llm-api-key: LLM provider API keys (manual, no auto-rotate)

Service Mesh Configuration:
  • Istio AuthorizationPolicy: RBAC enforcement via K8s SA tokens
  • VirtualServices: Traffic routing and host-based routing
  • DestinationRules: mTLS configuration and CA-signed certificates

Network Policies:
  • Ingress: Only via Istio Gateway (443/TCP HTTPS)
  • Egress: S3/Minio (443/9000), GCS (443), Azure (443), Hugging Face (443)
  • Internal: All traffic via service mesh with mTLS

Monitoring:
  • Prometheus scrapes metrics from:
    - Model servers (8080/8082 HTTP TLS 1.2+)
    - Istio Envoy sidecars (15090 HTTP)
    - KServe controllers (8080 HTTP)
