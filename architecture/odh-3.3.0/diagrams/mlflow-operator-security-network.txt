Authentication Flow:
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. User sends request to MLflow with Bearer Token                       │
│    → Header: Authorization: Bearer <k8s-token>                          │
│                                                                          │
│ 2. MLflow receives request (kubernetes-auth enabled)                    │
│    → Extracts token from Authorization header                           │
│                                                                          │
│ 3. MLflow calls Kubernetes API SubjectAccessReview                      │
│    → POST /apis/authorization.k8s.io/v1/subjectaccessreviews            │
│    → Body: {token: <token>, namespace: <ns>, verb: <verb>}              │
│                                                                          │
│ 4. Kubernetes API validates token and checks RBAC                       │
│    → Returns: {allowed: true/false, reason: "..."}                      │
│                                                                          │
│ 5. MLflow allows/denies request based on response                       │
│    → If allowed: Process request                                         │
│    → If denied: Return 403 Forbidden                                     │
└─────────────────────────────────────────────────────────────────────────┘

TLS Certificate Provisioning:
┌─────────────────────────────────────────────────────────────────────────┐
│ OpenShift service-ca (Automatic):                                       │
│   1. MLflow Service created with annotation:                            │
│      service.beta.openshift.io/serving-cert-secret-name: mlflow-tls     │
│   2. service-ca controller watches annotated Services                   │
│   3. Generates TLS cert/key for Service DNS name                         │
│      → CN: mlflow.{namespace}.svc                                       │
│   4. Creates/updates Secret: mlflow-tls                                  │
│   5. Auto-rotates certificate before expiry (90 days)                   │
│                                                                          │
│ Manual (User-provided):                                                 │
│   1. User creates Secret with tls.crt and tls.key                       │
│   2. MLflow Pod mounts Secret and uses for HTTPS                        │
│   3. User responsible for rotation                                      │
└─────────────────────────────────────────────────────────────────────────┘
