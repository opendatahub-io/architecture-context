NETWORK POLICY RECOMMENDATIONS (Not included by default)
════════════════════════════════════════════════════════════════════════════

1. Isolate Feast namespace from other namespaces (except Istio system)
2. Allow ingress to Feature Server only from:
   - OpenShift Router / Istio Gateway (external traffic)
   - Authorized ML application namespaces (internal)
3. Allow egress from Feature Server to:
   - Registry Service (6570/TCP gRPC, 6572/TCP HTTP)
   - Transformation Service (6569/TCP gRPC)
   - External databases (5432/TCP PostgreSQL, 6379/TCP Redis)
   - Cloud storage (443/TCP HTTPS)
4. Allow egress from Operator to:
   - Kubernetes API Server (6443/TCP HTTPS)
   - All namespaces (for resource management)
5. Deny all other ingress/egress by default
