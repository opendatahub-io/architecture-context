SECURITY CONTROLS:
═══════════════════

Authentication & Authorization:
  • Ray Dashboard: mTLS optional (client certificates)
  • Ray Serve API: Bearer Token optional
  • Inter-node gRPC: mTLS optional (operator-managed certificates)
  • Kubernetes API: ServiceAccount tokens for operator
  • External Redis: Password authentication

RBAC Roles:
  • kuberay-operator: Full control over RayClusters, RayJobs, RayServices
  • kuberay-operator: Manages Pods, Services, ConfigMaps, ServiceAccounts
  • kuberay-operator: Creates NetworkPolicies, Routes, Ingresses
  • kuberay-operator: Manages Secrets for mTLS and authentication

Encryption:
  • External → Ingress: TLS 1.2+ (HTTPS/443, Edge Termination)
  • Ingress → Services: HTTP or HTTPS (TLS 1.2+ optional)
  • Ray Head ↔ Workers: mTLS optional (gRPC)
  • Worker ↔ Worker: mTLS optional (Object Store 2384/TCP)
  • Operator → K8s API: TLS 1.2+ (HTTPS/6443)
  • Workers → S3: TLS 1.2+ (HTTPS/443)
  • Ray → Redis: TLS 1.2+ optional (TCP/6379)

Secrets Management:
  • ray-mtls-ca-secret: CA cert for mTLS (KubeRay, auto-rotate)
  • ray-mtls-cert-secret: mTLS certs for nodes (cert-manager/KubeRay, auto-rotate)
  • ray-redis-password: Redis password (user-provided, no auto-rotate)
  • aws-s3-credentials: S3 access credentials (user-provided, no auto-rotate)

Network Policies:
  • Created by NetworkPolicy Controller for each RayCluster
  • Controls ingress/egress for Ray pods
  • Enforces isolation between Ray clusters

Service Mesh Configuration:
  • Optional mTLS for all Ray inter-node communication
  • Operator manages certificate lifecycle
  • Optional integration with Istio for advanced routing

Monitoring:
  • Prometheus scrapes:
    - KubeRay Operator (8080/TCP HTTP)
    - Ray Dashboard (8265/TCP HTTP)
  • Metrics for cluster health, job status, resource utilization

Autoscaling:
  • Ray Autoscaler monitors workload via GCS
  • Scales workers based on resource demands
  • Integrates with Kubernetes HPA
  • Optional integration with Kueue/Volcano for job queuing
