SECURITY CONTROLS:
═══════════════════

Authentication & Authorization:
  • Llama Stack API: API Key (Bearer Token) authentication
  • Kubernetes API: ServiceAccount tokens for operator
  • Ollama Backend: None (internal, protected by NetworkPolicy)
  • Hugging Face: API token for model downloads

RBAC Roles:
  • manager-role: Full control over LlamaStackDistributions, Deployments, Services,
    PVCs, HPAs, NetworkPolicies, Routes
  • llsd-editor-role: Create, update, delete LlamaStackDistributions
  • llsd-viewer-role: Read-only access to LlamaStackDistributions

Encryption:
  • External → Route: TLS 1.2+ (HTTPS/443, Edge Termination)
  • Route → Service: HTTP or HTTPS (TLS 1.2+)
  • Service → Pod: HTTP or HTTPS (TLS 1.2+)
  • Operator → K8s API: TLS 1.3 (HTTPS/6443)
  • Pod → Hugging Face: TLS 1.3 (HTTPS/443)
  • Pod → Ollama: HTTP (internal, unencrypted)

Secrets Management:
  • hf-token-secret: Hugging Face API token (user-provided, no auto-rotate)
  • {llsd-name}-api-key: Llama Stack API key (operator-provisioned, no auto-rotate)

Network Policies:
  • Created by operator for each LlamaStackDistribution
  • Controls ingress/egress for Llama Stack pods
  • Namespace isolation

Route Exposure:
  • Optional external exposure via OpenShift Route
  • Controlled by exposeRoute parameter in LlamaStackDistribution CR
  • TLS edge termination

Monitoring:
  • Operator health: /healthz, /readyz (8081/TCP HTTP)
  • Llama Stack health: /health (8000/TCP HTTP)

Autoscaling:
  • Optional HorizontalPodAutoscaler
  • Scales based on CPU/memory utilization
