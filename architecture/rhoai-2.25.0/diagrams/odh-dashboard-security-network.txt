/metrics               GET       None (auth bypass)          OAuth Proxy Sidecar    Publicly accessible for monitoring
/api/* (internal)      ALL       ServiceAccount Token        Kubernetes API Server  RBAC (odh-dashboard ClusterRole)
Backend → K8s API      ALL       ServiceAccount Token        Kubernetes API Server  odh-dashboard ClusterRole permissions

OAuth Flow Details:
1. User accesses dashboard URL (443/TCP HTTPS)
2. OpenShift Router forwards to OAuth Proxy (8443/TCP HTTPS)
3. OAuth Proxy redirects to OpenShift OAuth Server for authentication
4. User authenticates with OpenShift credentials
5. OAuth Server issues Bearer Token, stored in encrypted cookie
6. OAuth Proxy validates token on subsequent requests
7. Validated requests forwarded to backend (8080/TCP HTTP with Bearer header)

ServiceAccount Token Usage:
- Backend container mounts SA token from /var/run/secrets/kubernetes.io/serviceaccount/token
- All K8s API calls include SA token in Authorization header
- RBAC policies enforced by K8s API Server based on odh-dashboard ClusterRole
- Token auto-rotated by Kubernetes (1 year expiry, refreshed before expiry)

DATA CLASSIFICATION

Data Handled:
- User authentication tokens (OAuth Bearer tokens in cookies)
- Kubernetes resource definitions (notebooks, serving runtimes, model registries)
- Configuration data (feature flags, dashboard settings)
- Metrics data (Prometheus queries)
- User and group information (OpenShift users/groups)
- Model metadata (from model registries)
- Pipeline run data (from data science pipelines)

Sensitive Data:
- OAuth tokens (encrypted in transit via TLS, stored in encrypted cookies)
- ServiceAccount tokens (mounted securely, transmitted via TLS)
- Secrets (configmaps, PVC, secrets - managed via K8s RBAC)

Data Protection:
- All external communication encrypted via TLS 1.2+
- OAuth tokens protected by HttpOnly, Secure, SameSite cookie flags
- Secrets managed via Kubernetes Secrets API
- Authorization headers redacted from logs (Pino logger configuration)
- Admin activity logged separately in /usr/src/app/logs/adminActivity.log
