Security Best Practices Applied
═══════════════════════════════════════════════════════════════════════════════════════

✓ mTLS STRICT enforced across service mesh (Istio PeerAuthentication)
✓ TLS 1.2+ for all external communications (ingress, storage, event sinks)
✓ Certificate rotation automated via cert-manager (90-day lifecycle)
✓ Service Account isolation (one SA per InferenceService)
✓ RBAC least privilege (controller has minimal required permissions)
✓ Secret management (credentials stored in Kubernetes Secrets, not code)
✓ AWS IRSA / GCP Workload Identity supported (no long-lived credentials)
✓ Path traversal vulnerabilities patched (RHOAIENG-47407, commits bb03ddbbe, 16e11a862)
✓ Network policies limit ingress/egress to required flows
✓ Webhook authentication (Kubernetes API server validates webhook calls)
✓ Bearer token authentication supported at inference endpoints (optional)
✓ Request/response logging to event sinks (audit trail)
✓ Security context constraints on OpenShift (SCC: openshift-ai-llminferenceservice-scc)
