Data Flows Summary:
===============================================================================
1. Job Submission:
   User → K8s API (443/TCP HTTPS Bearer)
   → Webhook (9443/TCP HTTPS mTLS)
   → Controller watches (443/TCP HTTPS ServiceAccount)
   → Admission decision
   → Patch job to unsuspend (443/TCP HTTPS ServiceAccount)

2. Metrics Collection:
   Prometheus → kueue-metrics-service (8443/TCP HTTPS Bearer)

3. Visibility API:
   User → K8s API (443/TCP HTTPS Bearer)
   → Visibility server (in-process HTTP K8s RBAC)
   → In-memory cache

4. MultiKueue:
   Controller → Remote K8s API (443/TCP HTTPS Kubeconfig)

5. Provisioning Request:
   Controller → K8s API (443/TCP HTTPS ServiceAccount)
   → Cluster Autoscaler watches → Cloud Provider API

Integration Points:
===============================================================================
- Kubernetes API Server: REST API client (watches CRs, patches jobs)
- Webhook Server: Admission webhook (mutates/validates jobs on creation)
- Aggregated API Server: Visibility API (on-demand pending workload queries)
- Prometheus: Metrics endpoint (queue state, admission metrics)
- cert-manager: Certificate requests (webhook TLS)
- Batch/Kubeflow/Ray/JobSet Operators: CRD watch/patch (job admission)
- Cluster Autoscaler: ProvisioningRequest CRD (trigger node provisioning)
