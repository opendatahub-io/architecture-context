Network Policies
════════════════════════════════════════════════════════════════════════════

Note: Feast operator does not deploy default NetworkPolicies. Network isolation
should be configured at the cluster/namespace level using Kubernetes NetworkPolicy
resources or OpenShift SDN features.

Recommended NetworkPolicy for production:
  - Allow ingress from OpenShift Router to Feature Store Services (80/443)
  - Allow ingress from Prometheus to Operator Metrics Service (8443)
  - Allow ingress from same namespace to all Feature Store Services
  - Allow egress to PostgreSQL, Redis, S3, Git (specific IPs/CIDRs)
  - Allow egress to Kubernetes API (6443)
  - Allow egress to OIDC Provider (443)
  - Deny all other ingress/egress by default
