High Availability & Fault Tolerance:
─────────────────────────────────────
Operator HA:
  - Deployment: Single replica (Recreate strategy)
  - Leader election: Supported but disabled by default
  - Can be enabled with --enable-leader-election flag

Ray Cluster HA:
  - GCS Fault Tolerance: Requires external Redis (optional)
  - Worker Fault Tolerance: Automatic pod recreation on failure
  - Head Fault Tolerance: Requires external GCS with FT enabled
  - Autoscaling: Dynamic worker scaling based on demand

RayService HA:
  - Zero-Downtime Updates: Blue-green deployment pattern
  - Health Monitoring: Continuous checks of Serve endpoints
  - Automatic Rollback: Failed deployments don't affect traffic
  - Multi-Replica Serve: Load balancing across replicas
