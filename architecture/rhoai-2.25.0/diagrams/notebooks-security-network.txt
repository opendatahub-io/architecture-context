RBAC SUMMARY
═══════════════════════════════════════════════════════════════════════════

This component does not define ClusterRoles or RoleBindings.

RBAC is managed by ODH Notebook Controller which:
  - Creates ServiceAccount: {notebook-name}-sa
  - Assigns default namespace permissions
  - Mounts ServiceAccount token for Kubernetes API access

User permissions:
  - Enforced by: OAuth Proxy (OpenShift OAuth)
  - Policy: Users can only access notebooks in their authorized namespaces
  - Mechanism: OpenShift RBAC controls namespace access

SECRETS

Pull Secret (kubernetes.io/dockerconfigjson):
  - Name: Varies (configured at cluster level)
  - Purpose: Authenticate to quay.io, registry.redhat.io

  - Auto-rotate: No (manual management)

  - Name: User-defined (e.g., aws-credentials, s3-secret)
  - Purpose: Data access (S3, databases, APIs)
  - Provisioned by: End users or administrators
  - Auto-rotate: No
  - Mounted: User specifies in notebook configuration
  - Contains: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, connection strings

OAuth Proxy Configuration (Opaque):
  - Name: {notebook-name}-oauth-config
  - Purpose: OAuth proxy session secret
  - Provisioned by: ODH Notebook Controller
  - Auto-rotate: No
  - Contains: Session encryption key
