RBAC Summary:
- This component does not define ClusterRoles or RoleBindings
- RBAC is managed by ODH Notebook Controller
- Workbench pods run with ServiceAccount: {notebook-name}-sa (assigned by controller)
- ServiceAccount has minimal permissions (no cluster-level access)

Authentication & Authorization:
- External Access: OpenShift OAuth enforced by OAuth proxy sidecar
  - Policy: Only authenticated OpenShift users can access their own notebooks
  - Enforcement Point: OAuth2 Proxy Sidecar Container
  - Path: /notebook/{namespace}/{username}/*
- Container Registry: Pull secrets for authenticated image pulls
  - Enforcement Point: Kubelet
  - Policy: Only authenticated service accounts can pull images

Secrets:
- Pull Secret (kubernetes.io/dockerconfigjson)
  - Purpose: Pull images from authenticated registries (Quay.io, registry.redhat.io)
  - Provisioned By: OpenShift Installation / Cluster Admin
  - Auto-Rotate: No
  - Mounted in: Namespace service accounts

- User Credentials (Opaque)
  - Purpose: S3 credentials, cloud credentials for data access
  - Provisioned By: End Users
  - Auto-Rotate: No
  - Mounted in: Workbench pods (user-managed)

Network Policies:
- No explicit NetworkPolicies defined by this component
- Default OpenShift SDN allows pod-to-pod communication
- External access controlled by Routes with OAuth proxy

Service Mesh:
- Not used by default
- Workbench containers do not participate in service mesh
- TLS termination at OpenShift Router (edge mode)
- Internal traffic between OAuth proxy and workbench is unencrypted (localhost)

Egress Traffic:
- quay.io (443/TCP, HTTPS, TLS 1.2+, Pull Secret) - Container image pulls
- registry.redhat.io (443/TCP, HTTPS, TLS 1.2+, Pull Secret) - Red Hat registry
- mirror.openshift.com (443/TCP, HTTPS, TLS 1.2+, None) - OpenShift CLI downloads
- pypi.org (443/TCP, HTTPS, TLS 1.2+, None) - Python package installation at build time
- S3 Storage (443/TCP, HTTPS, TLS 1.2+, AWS SigV4) - User data access from notebooks
- Object Storage (443/TCP, HTTPS, TLS 1.2+, S3 credentials) - Kubeflow Pipelines artifacts

Security Notes:
- Workbench containers run as non-root user (UID 1001 or higher)
- Images built on UBI9 (security-hardened RHEL base)
- Weekly security patches applied via patch releases
- Vulnerability scanning via Trivy in Konflux pipeline
- No elevated privileges required (securityContext: runAsNonRoot: true)
