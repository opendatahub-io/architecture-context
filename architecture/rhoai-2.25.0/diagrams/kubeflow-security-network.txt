Known Security Limitations:
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. Culling requires unauthenticated access to /api/kernels on notebook  │
│    pods (HTTP, no auth) - potential information disclosure              │
│                                                                          │
│ 2. Metrics endpoint (:8080) exposed over HTTP (no TLS) - relies on      │
│    network policies for security                                        │
│                                                                          │
│ 3. Health probes (:8081) exposed over HTTP (no auth) - information      │
│    disclosure risk if exposed externally                                │
│                                                                          │
│ 4. Notebook pods run with user-specified images - container security    │
│    depends on image provenance and vulnerability scanning               │
└─────────────────────────────────────────────────────────────────────────┘
