2. {name}-oauth-cookie-secret (Opaque)
   - Purpose: OAuth proxy session cookie encryption key
   - Provisioned By: Operator (auto-generated)
   - Auto-Rotate: No
   - Mounted In: oauth-proxy container

3. {db-secret-name} (Opaque)
   - Purpose: Database password
   - Provisioned By: User
   - Referenced In: spec.postgres.passwordSecret or spec.mysql.passwordSecret
   - Auto-Rotate: No
   - Used By: rest-container

4. {ssl-cert-secret} (Opaque)
   - Purpose: Database client SSL certificates (optional)
   - Provisioned By: User
   - Auto-Rotate: No
   - Used By: rest-container (PostgreSQL SSL connections)

   - Purpose: Webhook server TLS certificate
   - Provisioned By: cert-manager OR OpenShift Service CA
   - Auto-Rotate: Yes
   - Mounted In: controller-manager

Authentication & Authorization:
=================================

OAuth Mode (Recommended for External Access):
- Endpoint: /api/model_registry/v1alpha3/*
- Port: 8443/TCP HTTPS
- Auth Mechanism: Bearer Token (OpenShift OAuth)
- Enforcement Point: OAuth Proxy Sidecar
- Policy: User must have GET permission on service/{name} in namespace
- Token Validation: OpenShift OAuth Server (443/TCP HTTPS)

Non-OAuth Mode (Internal Cluster Access Only):
- Endpoint: /api/model_registry/v1alpha3/*
- Port: 8080/TCP HTTP
- Auth Mechanism: None
- Exposure: Internal ClusterIP only (no Route)
- Security: Relies on network isolation

Operator Metrics:
- Endpoint: /metrics
- Port: 8443/TCP HTTPS
- Auth Mechanism: Bearer Token (ServiceAccount)
- Enforcement Point: kube-rbac-proxy
- Policy: Requires tokenreviews/subjectaccessreviews permissions

Admission Webhooks:
- Endpoints: /validate-*, /mutate-*
- Port: 443/TCP HTTPS
- Auth Mechanism: Webhook Client Certificate
- Enforcement Point: Kubernetes API Server
- Policy: API server validates webhook CA bundle

Data Flow Summary:
===================

External → Internal (OAuth Mode):
1. User → OpenShift Router (443/TCP HTTPS, TLS 1.2+, Bearer Token)
2. Router → model-registry Service (8443/TCP HTTPS, Edge Termination)
3. Service → OAuth Proxy Container (8443/TCP HTTPS, TLS 1.2+, Bearer Token)
4. OAuth Proxy → OpenShift OAuth Server (443/TCP HTTPS, Service Account)
5. OAuth Proxy → REST Container (8080/TCP HTTP, localhost, no auth)
6. REST Container → PostgreSQL/MySQL (5432/3306 TCP, TLS optional, user/pass)

Internal Cluster Access (Non-OAuth Mode):
1. Pod → model-registry Service (8080/TCP HTTP, no auth)
2. Service → REST Container (8080/TCP HTTP, no auth)
3. REST Container → PostgreSQL/MySQL (5432/3306 TCP, TLS optional, user/pass)

Operator Control Plane:
1. controller-manager → Kubernetes API (6443/TCP HTTPS, TLS 1.2+, SA Token)
2. Kubernetes API → webhook-service (443/TCP HTTPS, TLS, Webhook Cert)
3. Prometheus → controller-manager-metrics-service (8443/TCP HTTPS, Bearer Token)
