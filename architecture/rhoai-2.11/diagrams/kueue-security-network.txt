AUTHENTICATION & AUTHORIZATION FLOWS
═══════════════════════════════════════════════════════════════════════════

Flow 1: User Submits Job
├── User → Kubernetes API Server
│   ├── Port: 443/TCP
│   ├── Protocol: HTTPS
│   ├── Encryption: TLS 1.2+
│   ├── Auth: Bearer Token (user credentials)
│   └── Enforcement: Kubernetes RBAC (user must have batch/jobs create permission)
└── Result: Job created (suspended=true by default)

Flow 2: Webhook Validation/Mutation
├── Kubernetes API Server → kueue-webhook-service
│   ├── Port: 443/TCP (service) → 9443/TCP (pod)
│   ├── Protocol: HTTPS
│   ├── Encryption: TLS 1.2+ (mTLS)
│   ├── Auth: Client certificate (Kubernetes API Server identity)
│   └── Enforcement: Kubernetes validates webhook certificate
└── Result: Job mutated with queue labels and annotations

Flow 3: Controller Watches Resources
├── kueue-controller-manager → Kubernetes API Server
│   ├── Port: 443/TCP
│   ├── Protocol: HTTPS
│   ├── Encryption: TLS 1.2+
│   ├── Auth: ServiceAccount token (kueue-controller-manager)
│   └── Enforcement: Kubernetes RBAC (kueue-manager-role)
└── Result: Controller receives watch events and reconciles workloads

Flow 4: Metrics Collection (Unauthenticated)
├── Prometheus → kueue-metrics-service
│   ├── Port: 8080/TCP
│   ├── Protocol: HTTP
│   ├── Encryption: None
│   └── Auth: None (internal service, no external exposure)
└── Result: Metrics scraped

Flow 5: Metrics Collection (Authenticated)
├── Prometheus → kueue-controller-manager-metrics-service
│   ├── Port: 8443/TCP
│   ├── Protocol: HTTPS
│   ├── Encryption: TLS 1.2+
│   ├── Auth: Bearer Token (ServiceAccount) via kube-rbac-proxy
│   └── Enforcement: kube-rbac-proxy validates token and enforces RBAC
└── Result: Metrics scraped (authenticated)
