AUTHENTICATION & AUTHORIZATION POLICIES
══════════════════════════════════════════════════════════════════════════

Endpoint: /api/* (All API endpoints)
  - Methods: GET, POST, PATCH, DELETE
  - Auth Mechanism: Bearer Token (OpenShift OAuth JWT)
  - Enforcement Point: OAuth Proxy (port 8443)
  - Policy: Requires 'projects:list' permission via OpenShift RBAC
  - Flow:
    1. User authenticates with OpenShift OAuth
    2. OAuth proxy validates JWT token
    3. Proxy forwards request with Bearer token to backend
    4. Backend uses token for Kubernetes API impersonation

Endpoint: /oauth/healthz
  - Methods: GET
  - Auth Mechanism: None
  - Enforcement Point: OAuth Proxy
  - Policy: Public endpoint for health checks

Endpoint: /api/health
  - Methods: GET
  - Auth Mechanism: None (skip-auth-regex)
  - Enforcement Point: Backend (bypasses OAuth)
  - Policy: Public endpoint for liveness/readiness probes

Endpoint: /metrics
  - Methods: GET
  - Auth Mechanism: None (skip-auth-regex)
  - Enforcement Point: Backend (bypasses OAuth)
  - Policy: Public endpoint for Prometheus scraping

══════════════════════════════════════════════════════════════════════════
NETWORK POLICIES & TRUST BOUNDARIES
══════════════════════════════════════════════════════════════════════════

Trust Zones:
  1. EXTERNAL: User browsers, untrusted internet traffic
  2. INGRESS (DMZ): OpenShift Router with TLS termination/reencryption
  3. APPLICATION: Dashboard pods (OAuth proxy + backend containers)
  4. PLATFORM: Kubernetes API, OAuth server, monitoring services
  5. EGRESS: External storage (S3)

Traffic Flows:
  - External → Ingress: TLS 1.2+ (public CA or cluster CA)
  - Ingress → Application: TLS 1.2+ (Service CA, reencrypt)
  - OAuth Proxy → Backend: HTTP (localhost, same pod)
  - Application → Platform: HTTPS with ServiceAccount tokens
  - Application → External: HTTPS with AWS credentials

Deployment Configuration:
  - Replicas: 2 (high availability)
  - Pod Anti-Affinity: Preferred across availability zones
  - Resource Requests: 500m CPU, 1Gi memory (per container)
  - Resource Limits: 1000m CPU, 2Gi memory (per container)
