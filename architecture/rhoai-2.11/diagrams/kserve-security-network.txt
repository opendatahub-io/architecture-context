TRUST BOUNDARIES
═══════════════════════════════════════════════════════════════════════════════════════════

1. EXTERNAL → CONTROL PLANE
   - Entry: Kubernetes API Server (6443/TCP)
   - Auth: Bearer Token (kubeconfig)
   - Encryption: TLS 1.2+
   - Trust: User credentials validated by K8s RBAC

2. CONTROL PLANE → KSERVE CONTROL PLANE
   - Entry: kserve-webhook-server (443/TCP)
   - Auth: mTLS client certificates
   - Encryption: TLS 1.2+ (cert-manager issued)
   - Trust: Kubernetes API server client certificate

3. EXTERNAL → DATA PLANE INGRESS
   - Entry: Istio Gateway (443/TCP)
   - Auth: Bearer Token (optional) / mTLS
   - Encryption: TLS 1.2+
   - Trust: RequestAuthentication validates JWT

4. DATA PLANE INGRESS → SERVICE MESH
   - Entry: Knative Service (80/TCP)
   - Auth: Service Identity (Istio)
   - Encryption: mTLS (ISTIO_MUTUAL STRICT)
   - Trust: Istio PeerAuthentication enforces mTLS

5. SERVICE MESH → EXTERNAL SERVICES
   - Entry: S3/GCS/Azure (443/TCP), Model Registry (443/TCP)
   - Auth: AWS IAM/GCP SA/Azure creds, API tokens
   - Encryption: TLS 1.2+
   - Trust: Cloud provider IAM, API authentication
