WORKBENCH LIFECYCLE FLOW:
┌─────────────────────────────────────────────────────────────────────────┐
│  1. User → ODH Dashboard (HTTPS/443, OAuth2 token)                     │
│     - Select workbench type, size, GPU accelerators                     │
│                                                                          │
│  2. Dashboard → Kubernetes API (HTTPS/6443, mTLS, SA token)            │
│     - Create Notebook CR (custom resource)                              │
│                                                                          │
│  3. Notebook Controller → Kubernetes API (HTTPS/6443, mTLS)            │
│     - Watch Notebook CR                                                  │
│     - Create StatefulSet, Service, Route, OAuth proxy config            │
│                                                                          │
│  4. Kubelet → Quay.io (HTTPS/443, TLS 1.2+, registry auth)             │
│     - Pull workbench image                                               │
│                                                                          │
│  5. Kubelet → Start notebook pod                                        │
│     - Mount PVC, secrets (git, S3)                                       │
│     - Start OAuth proxy sidecar + notebook container                    │
│                                                                          │
│  6. User → OpenShift Route (HTTPS/443, TLS 1.2+, OAuth2 token)         │
│     - Access JupyterLab/RStudio/Code Server UI                          │
│                                                                          │
│  7. Notebook code → External services (S3, GitHub, PyPI)               │
│     - Data access, Git operations, package installation                 │
│                                                                          │
│  8. User → Stop workbench (via Dashboard)                               │
│     - Controller deletes pod, service, route                            │
│     - PVC persists for future use                                        │
└─────────────────────────────────────────────────────────────────────────┘
