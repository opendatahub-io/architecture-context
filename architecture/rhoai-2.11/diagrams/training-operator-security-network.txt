Network Security Notes:
┌─────────────────────────────────────────────────────────────────────────┐
│ ⚠️  SECURITY CONSIDERATIONS:                                            │
│                                                                          │
│ 1. Metrics Endpoint (8080/TCP)                                           │
│    - No TLS encryption (plaintext HTTP)                                  │
│    - No authentication                                                   │
│    - Mitigation: Internal cluster access only, use NetworkPolicies      │
│                                                                          │
│ 2. Training Job Pod-to-Pod Communication                                 │
│    - No encryption (plaintext TCP/23456 for PyTorch)                    │
│    - No authentication between pods                                      │
│    - Mitigation: Kubernetes network isolation, consider Istio for       │
│      sensitive workloads                                                 │
│                                                                          │
│ 3. Istio Sidecar Disabled                                                │
│    - Operator pods do not use Istio mTLS                                │
│    - Training job pods MAY enable Istio if needed                        │
│                                                                          │
│ 4. Multi-Tenancy                                                          │
│    - Jobs run in user namespaces (isolation via Kubernetes RBAC)        │
│    - Resource quotas should be enforced per namespace                    │
│    - Network policies recommended for stronger isolation                 │
