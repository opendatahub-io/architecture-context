Authentication Flow:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User → OpenShift Route (443/TCP HTTPS)
2. Route → Service → OAuth Proxy (8443/TCP HTTPS)
3. OAuth Proxy → Checks for valid session cookie
4. If no valid cookie → Redirect to OpenShift OAuth Server (443/TCP HTTPS)
5. User authenticates → OAuth Server issues token
6. OAuth Proxy validates token → Performs SAR check (user must have pod get permission)
7. If authorized → OAuth Proxy forwards to TrustyAI Service (8080/TCP HTTP)
8. TrustyAI Service processes request → Returns response
