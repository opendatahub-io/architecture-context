Security Considerations:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️  Plaintext Communications:
  - ModelMesh ↔ etcd (2379/TCP HTTP plaintext)
  - ModelMesh ↔ Model Servers (8001/TCP gRPC plaintext)
  - Internal ModelMesh gRPC (8085/TCP plaintext)
  - Mitigation: Network segmentation, NetworkPolicies, optional Service Mesh

⚠️  No Authentication on Internal Services:
  - etcd has no authentication (relies on network isolation)
  - ModelMesh internal gRPC has no auth
  - Model servers have no auth
  - Mitigation: NetworkPolicies, namespace isolation, Service Mesh mTLS

✅  Encrypted External Communications:
  - K8s API: TLS 1.2+ with ServiceAccount tokens
  - S3: TLS 1.2+ with AWS IAM or access keys
  - Webhook: TLS 1.2+ with K8s API server authentication

✅  Optional Service Mesh (Istio):
  - Provides mTLS for all inference traffic
  - Service identity validation
  - Traffic encryption and authorization

Recommended Security Enhancements:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Enable Service Mesh (Istio) for mTLS encryption
2. Implement strict NetworkPolicies to isolate etcd
3. Use AWS IAM Roles for ServiceAccounts (IRSA) for S3 access
4. Enable etcd authentication and TLS
5. Rotate storage-config Secret credentials regularly
6. Implement PodSecurityPolicies/PodSecurityStandards
7. Enable audit logging for CRD operations
8. Use private S3 endpoints (VPC endpoint or PrivateLink)
