AUTHENTICATION MECHANISMS
═══════════════════════════════════════════════════════════════════════════════════════════════════

1. OpenShift OAuth (Bearer Tokens)
   - Components: Dashboard, Notebooks, DSP UI, Ray Dashboard, TrustyAI, Prometheus, Alertmanager
   - Enforcement: oauth-proxy sidecars (validate token via OpenShift OAuth server)
   - Token flow: User → OpenShift OAuth → SAR check → oauth-proxy validates → backend

2. mTLS (Service Mesh - Istio)
   - Components: KServe InferenceServices, predictor pods
   - Enforcement: Istio sidecars (envoy proxy), PeerAuthentication STRICT
   - Certificate issuance: Istio CA (SPIFFE identities)

3. mTLS (Kubernetes Webhooks)
   - Components: All operator webhooks
   - Enforcement: Kubernetes API Server
   - Certificates: cert-manager (webhook-server-cert secrets)

4. ServiceAccount Tokens (Kubernetes API)
   - Components: All operators, all pods
   - Enforcement: Kubernetes API Server RBAC
   - Token: Mounted at /var/run/secrets/kubernetes.io/serviceaccount/token

5. AWS IAM / S3 Credentials
   - Components: KServe, ModelMesh, Notebooks, Training, Pipelines
   - Enforcement: Application-level (AWS SDK)
   - Credentials: From storage-config secrets

6. API Keys (Authorino)
   - Components: KServe (when authentication enabled)
   - Enforcement: Authorino (validates AuthConfigs)
   - Policy: ODH Model Controller creates AuthConfigs per InferenceService

NETWORK POLICIES (Recommended - not enforced by default)
═══════════════════════════════════════════════════════════════════════════════════════════════════

Namespace Isolation:
  - Default: Allow all traffic within OpenShift cluster
  - Recommendation: Create NetworkPolicies to isolate user namespaces
  - Critical paths to allow:
    * Operators → Kubernetes API
    * Prometheus → ServiceMonitors (all namespaces)
    * Predictor pods → S3 egress
    * Notebook pods → PyPI, GitHub egress
    * Training pods → inter-pod communication (same job)

Egress Filtering:
  - Default: Allow all egress
  - Recommendation: Restrict egress to known endpoints (S3, registries, package repos)
