## Purpose
**Short**: A Kubernetes operator that manages TrustyAI explainability services, LM evaluation jobs, and guardrails orchestration for AI/ML models on OpenShift.

**Detailed**: The TrustyAI Service Operator provides three core services for AI trustworthiness in Red Hat OpenShift AI (RHOAI). The first service (TAS) deploys and manages TrustyAI Service instances that provide model explainability, fairness metrics, and data monitoring capabilities for ModelMesh-served and KServe inference services. It creates Deployments with kube-rbac-proxy sidecars for authentication, PVC or database-backed storage, OpenShift Routes, and Istio VirtualService/DestinationRule resources when service mesh is present.

The second service (LMES) manages LMEvalJob custom resources for running language model evaluation tasks using the lm-evaluation-harness framework. It creates evaluation Pods with a driver sidecar that manages the evaluation lifecycle, supports offline storage via S3 or PVC, and integrates with Kueue for workload queuing and resource management.

The third service (GORCH) manages GuardrailsOrchestrator custom resources that deploy guardrails infrastructure for AI model safety. It creates multi-container Deployments with the orchestrator, optional built-in regex detectors, optional sidecar gateways for vLLM integration, and kube-rbac-proxy sidecars for authentication. It supports auto-configuration by detecting KServe InferenceServices to automatically configure generator and detector endpoints.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| TrustyAI Service Controller (TAS) | Go Controller | Reconciles TrustyAIService CRs; creates Deployments with kube-rbac-proxy sidecars, Services, Routes, PVCs, ServiceMonitors, Istio DestinationRules/VirtualServices |
| LMEvalJob Controller (LMES) | Go Controller | Reconciles LMEvalJob CRs; creates evaluation Pods with driver sidecar containers, manages job lifecycle and results |
| Job Manager Controller | Go Controller | Kueue integration for LMEvalJob workload scheduling and resource management |
| GuardrailsOrchestrator Controller (GORCH) | Go Controller | Reconciles GuardrailsOrchestrator CRs; creates multi-container Deployments (orchestrator, detectors, gateway, kube-rbac-proxy), Services, Routes, ServiceMonitors |
| Operator Manager | Go Binary | controller-runtime manager that initializes and runs all three controllers with leader election |
