## Purpose
**Short**: A centralized metadata repository for managing ML models, versions, artifacts, and serving environments in the Red Hat OpenShift AI platform.

**Detailed**: Model Registry provides a REST API service for recording, retrieving, and managing metadata associated with ML developer and data scientist workflows. It stores metadata about registered models, model versions, model artifacts, inference services, serving environments, experiments, and experiment runs. The service replaces the original C++ gRPC ml-metadata project with a Go-based REST API server that supports MySQL and PostgreSQL backends.

The repository contains multiple components: (1) the core REST API proxy server that exposes the Model Registry API, (2) an InferenceService controller that watches KServe InferenceService CRs and synchronizes them with the registry, (3) a UI Backend-for-Frontend (BFF) service that provides an API layer between the React frontend and the Model Registry API, (4) a Model Catalog service for browsing curated model catalogs from sources like HuggingFace, (5) a CSI storage initializer that enables KServe to pull model artifacts using `model-registry://` URIs, and (6) an async upload job for transferring model artifacts between storage backends. The system uses an embedded metadata store (EmbedMD) backed by either MySQL or PostgreSQL with GORM ORM, and supports Istio service mesh integration for mTLS and traffic management. FIPS-compliant builds are produced via Konflux using `strictfipsruntime` Go build tags.
