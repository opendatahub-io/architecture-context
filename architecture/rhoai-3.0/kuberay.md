## Purpose
**Short**: KubeRay Operator manages the lifecycle of Ray clusters, jobs, and services on Kubernetes/OpenShift for distributed computing workloads within Red Hat OpenShift AI.

**Detailed**: The KubeRay operator is a Kubernetes controller-runtime based operator that provides three Custom Resource Definitions (RayCluster, RayJob, RayService) for managing Ray distributed computing clusters. It automates the provisioning, scaling, and lifecycle management of Ray head and worker pods, including autoscaler support, batch scheduler integration (Volcano, Yunikorn), and GCS fault tolerance with Redis cleanup.

In the RHOAI 3.x distribution, the operator extends upstream KubeRay with OpenShift-specific features including: Gateway API integration (HTTPRoute creation for the data-science-gateway), kube-rbac-proxy sidecar injection for authentication, mTLS certificate management via cert-manager, NetworkPolicy enforcement for secure trusted networking, and SecurityContextConstraints for running Ray pods as a dedicated user. The operator's authentication controller creates HTTPRoutes in the platform namespace with cross-namespace ReferenceGrants, enabling Ray dashboard access through the RHOAI platform gateway.

The operator is deployed by rhods-operator using the kustomize overlay at `ray-operator/config/openshift/`, which enables webhooks (mutating and validating for RayCluster), configures the Gateway API environment variables, and sets up the SecurityContextConstraints. The mutating webhook automatically enforces the `odh.ray.io/secure-trusted-network` annotation on all RayClusters created in OpenShift environments.

**Authentication Controller - Gateway API Pattern (RHOAI 3.x)**:
- Creates `HTTPRoute` CRs in the platform namespace (APPLICATION_NAMESPACE, typically `redhat-ods-applications`)
- HTTPRoutes reference parent Gateway `data-science-gateway` in `openshift-ingress` namespace
- Routes traffic to the head service at port 8265 with path prefix `/ray/{namespace}/{cluster}`
- URL rewrite filter strips the prefix before forwarding to the Ray dashboard backend
- Cross-namespace service references enabled via `ReferenceGrant` (one shared per namespace: `kuberay-gateway-access`)
- Injects `kube-rbac-proxy` sidecar (port 8443/TCP) into head pods for authentication enforcement
