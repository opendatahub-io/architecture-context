## Purpose
**Short**: KServe is a Kubernetes-native model serving platform that provides serverless inference, model management, and LLM-optimized deployment orchestration for AI/ML models on OpenShift AI.

**Detailed**: KServe is the model inference serving component of Red Hat OpenShift AI (RHOAI). It provides a custom resource-based API for deploying, scaling, and managing machine learning model inference endpoints on Kubernetes/OpenShift clusters. The controller manages the full lifecycle of inference services including model storage initialization, autoscaling, traffic routing, and observability.

In RHOAI 3.0, KServe introduces the LLMInferenceService (llmisvc) CRD, purpose-built for large language model serving with features such as disaggregated prefill/decode architectures, multi-node GPU deployments via LeaderWorkerSets, Gateway API-based inference routing with llm-d inference scheduler (ext_proc), and InferencePool/InferenceModel integration from the Gateway API Inference Extension. The default deployment mode is RawDeployment (as opposed to Knative Serverless), and ingress is handled through Istio Gateway with kube-rbac-proxy authentication replacing the legacy oauth-proxy pattern.

The controller dynamically creates Deployments, Services, HTTPRoutes, VirtualServices, InferencePools, InferenceModels, DestinationRules, HPA/KEDA ScaledObjects, ServiceMonitors, RBAC resources, and self-signed TLS certificates based on the InferenceService and LLMInferenceService CRD specifications.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| kserve-controller-manager | Go Operator (controller-runtime) | Main controller managing InferenceService, LLMInferenceService, TrainedModel, InferenceGraph CRDs; dynamically creates Deployments, Services, HTTPRoutes, VirtualServices, InferencePools, HPA, KEDA ScaledObjects |
| kserve-storage-initializer | Init Container | Downloads model artifacts from storage (S3, GCS, PVC, HuggingFace, OCI) into serving containers |
| kserve-agent | Sidecar Container | Provides model agent capabilities including multi-model config management, inference logging, and request batching |
| kserve-router | Container | Routes requests across InferenceGraph nodes, propagates Authorization headers |
| kube-rbac-proxy | Sidecar Container (RHOAI 3.x auth) | Provides RBAC-based authentication/authorization proxy for inference endpoints, replacing legacy oauth-proxy |
| llm-d inference-scheduler | Deployment (per LLMInferenceService) | Gateway API ext_proc-based inference scheduler for LLM request routing; creates InferencePool and InferenceModel resources |
| llm-d routing-sidecar | Init Container (restartable sidecar) | NIXL-based routing sidecar for disaggregated prefill/decode LLM serving; proxies requests between vLLM and the scheduler |
| Webhook Server | Admission Webhook | Mutating and validating webhooks for InferenceService, ServingRuntime, TrainedModel, InferenceGraph, LLMInferenceService, LocalModelCache CRDs |

**Ingress Configuration (RHOAI 3.0 ODH Overlay)**:
- Default deployment mode: `RawDeployment` (not Knative Serverless)
- `enableGatewayApi`: false (Istio VirtualService used for InferenceService by default)
- `disableIngressCreation`: true (ingress disabled by default for InferenceService)
- `kserveIngressGateway`: `openshift-ingress/openshift-ai-inference`
- kube-rbac-proxy replaces oauth-proxy for authentication (RHOAI 3.x pattern)
- LLMInferenceService always uses Gateway API (HTTPRoute) with parent Gateway references
- LLMInferenceService creates InferencePool + InferenceModel for ext_proc-based routing via llm-d scheduler
