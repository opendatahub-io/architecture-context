## Purpose
**Short**: An Endpoint Picker (EPP) service that makes optimized routing decisions for LLM inference requests within the llm-d inference framework.

**Detailed**: The llm-d Inference Scheduler provides an Endpoint Picker (EPP) component for the llm-d inference framework. It schedules incoming inference requests arriving via a Kubernetes Gateway API-based gateway to optimal model-serving pods using a pluggable architecture of filters, scorers, and scrapers.

The EPP extends the upstream Gateway API Inference Extension (GIE) project, adding llm-d-specific capabilities such as disaggregated Prefill/Decode (P/D) execution, precise KV-cache locality scoring via real-time cache state tracking, session affinity, load-aware scoring, and active request tracking. The scheduler operates as a gRPC-based external processing (ext-proc) filter for Envoy, receiving inference requests from the gateway's data plane and selecting the best target pod based on configurable scheduling profiles and plugin chains.

The architecture supports both standard (single-profile) and disaggregated P/D (multi-profile) scheduling modes. In P/D mode, the scheduler selects separate pods for prefill and decode stages of inference, enabling specialized resource utilization and improved throughput for large language model serving.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| EPP (Endpoint Picker) | Go gRPC Service | Main binary that receives ext-proc callbacks from Envoy and selects target pods for inference requests |
| Plugin Registry | Go Package | Registers all llm-d-specific filter, scorer, pre-request, and profile handler plugins |
| ByLabel Filter | Filter Plugin | Filters pods based on label values (generic label filtering) |
| ByLabelSelector Filter | Filter Plugin | Filters pods using standard Kubernetes label selectors |
| Decode Filter | Filter Plugin | Filters pods to only include decode-capable workers (label `llm-d.ai/role` = `decode` or `both`) |
| Prefill Filter | Filter Plugin | Filters pods to only include prefill workers (label `llm-d.ai/role` = `prefill`) |
| PrecisePrefixCacheScorer | Scorer Plugin | Scores pods based on real-time KV-cache locality using llm-d-kv-cache-manager indexer and ZMQ-based KV events |
| LoadAware Scorer | Scorer Plugin | Scores pods based on queue load; pods with fewer waiting requests get higher scores |
| SessionAffinity Scorer | Scorer Plugin | Routes subsequent session requests to the same pod using x-session-token header |
| ActiveRequest Scorer | Scorer Plugin | Tracks in-flight requests per pod with TTL and scores pods with fewer active requests higher |
| PrefillHeaderHandler | Pre-Request Plugin | Injects x-prefiller-host-port header when P/D scheduling selects a prefill worker |
| PdProfileHandler | Profile Handler Plugin | Manages prefill/decode scheduling profile selection and result processing |
| Inference Gateway | Gateway API Resource | Kubernetes Gateway API Gateway resource for routing inference traffic |
| InferencePool | GIE CRD | Groups model-serving pods and references the EPP as extension |
| Istio Control Plane | Infrastructure | Istiod (rev: llm-d-gateway) for service mesh data plane management |
