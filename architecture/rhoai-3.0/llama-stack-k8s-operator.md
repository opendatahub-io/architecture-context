## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of Llama Stack server distributions on OpenShift/Kubernetes clusters.

**Detailed**: The Llama Stack K8s Operator automates the deployment and management of Llama Stack AI inference server distributions on Kubernetes and OpenShift clusters. It provides a custom resource (`LlamaStackDistribution`) that allows users to declaratively specify a Llama Stack server configuration, including the distribution type (e.g., "rh-dev" for RHOAI), container resources, persistent storage, user configuration via ConfigMaps, and custom TLS CA bundle certificates.

The operator uses controller-runtime to reconcile the desired state by dynamically creating and managing Deployments, Services, ServiceAccounts, PersistentVolumeClaims, RoleBindings, and NetworkPolicies per LlamaStackDistribution CR instance. It monitors the health of deployed Llama Stack servers by querying their `/v1/health`, `/v1/providers`, and `/v1/version` endpoints, and reflects the operational status back into the CR's status subresource. The operator supports automatic detection and injection of ODH trusted CA bundles, content-based ConfigMap change detection for rolling restarts, and a feature flag system for enabling/disabling NetworkPolicy creation.

In the RHOAI distribution, the operator is deployed by the rhods-operator via kustomize overlays. The RHOAI overlay injects the `RELATED_IMAGE_RH_DISTRIBUTION` environment variable to configure the Red Hat-supported Llama Stack server image, overriding the upstream community distributions embedded in `distributions.json`.
