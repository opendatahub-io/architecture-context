## Purpose
**Short**: A Kubernetes controller that automates model serving infrastructure on OpenShift AI by reconciling KServe InferenceService, LLMInferenceService, and NVIDIA NIM Account resources.

**Detailed**: The odh-model-controller is a controller-runtime based Kubernetes operator that manages the lifecycle of model serving infrastructure in Red Hat OpenShift AI (RHOAI). It watches KServe InferenceService and LLMInferenceService custom resources and automatically creates/reconciles the necessary supporting resources such as OpenShift Routes, NetworkPolicies, ClusterRoleBindings, ServiceMonitors, PodMonitors, KEDA TriggerAuthentications, AuthPolicies (Kuadrant), and EnvoyFilters (Istio) for inference endpoints.

The controller also manages NVIDIA NIM Account integration, including API key validation, ServingRuntime template creation, model configuration ConfigMaps, and image pull secrets. Additionally, it handles data connection secrets by aggregating them into a unified `storage-config` secret, manages CA certificate bundles for KServe, and provides multi-node (Ray) TLS certificate management for distributed inference workloads. The controller deploys a set of ServingRuntime templates (vLLM CUDA/ROCm/Gaudi/CPU/Spyre, OpenVINO, Guardrails) via OpenShift Templates.

For RHOAI 3.x, the LLMInferenceService controller integrates with the Gateway API pattern, managing AuthPolicy CRs targeting both Gateway and HTTPRoute resources, and EnvoyFilter CRs for SSL authentication on the inference gateway. The InferenceService controller continues to manage OpenShift Routes for KServe RawDeployment mode ingress.

**Ingress Architecture Notes**:
- **InferenceService (RawDeployment mode)**: Controller creates OpenShift Route CRs per InferenceService when labeled with `networking.kserve.io/visibility: exposed`. Route uses TLS Edge termination by default, or Reencrypt when auth is enabled via `security.opendatahub.io/enable-auth` annotation.
- **LLMInferenceService (RHOAI 3.x Gateway API pattern)**: Controller creates Kuadrant AuthPolicy CRs targeting both the parent Gateway (`openshift-ai-inference` in `openshift-ingress` namespace by default) and individual HTTPRoute CRs. Also creates Istio EnvoyFilter CRs for SSL authentication on the inference gateway. These resources are Gateway API-centric; the actual HTTPRoute and Gateway are managed by KServe and the rhods-operator respectively.
