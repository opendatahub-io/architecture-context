## Purpose
**Short**: Provides GPU-accelerated training runtime container images and comprehensive integration tests for the RHOAI Distributed Workloads stack (KubeRay, Kueue, Kubeflow Training Operator, and Kubeflow Trainer v2).

**Detailed**: The distributed-workloads repository serves two primary functions within the RHOAI ecosystem. First, it builds and ships GPU-accelerated container images for distributed training workloads. These images include PyTorch runtimes with CUDA (NVIDIA) and ROCm (AMD) GPU support across multiple Python/PyTorch version combinations, as well as Ray cluster base images with GPU support. These images are built via Konflux (Tekton-based CI/CD) and published to `quay.io/rhoai/` and `quay.io/modh/`.

Second, it contains a comprehensive Go-based end-to-end test suite that validates the distributed workload components on OpenShift clusters. The tests exercise PyTorchJob (Kubeflow Training Operator), TrainJob (Kubeflow Trainer v2), RayCluster/RayJob (KubeRay), and Kueue resource queuing integration. The tests validate GPU training workflows, model fine-tuning (SFT, DreamBooth, DeepSpeed), notebook-based workloads, and upgrade scenarios. The test framework supports OCP, OSD, Hypershift, and KIND cluster types.
