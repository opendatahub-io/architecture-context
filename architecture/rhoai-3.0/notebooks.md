## Purpose

**Short**: Provides pre-built container images for interactive data science workbenches (Jupyter, Code Server, RStudio) and pipeline runtime environments used in Red Hat OpenShift AI.

**Detailed**: The Notebooks repository is a container image build system and manifest repository that produces the workbench images available in RHOAI. It does not contain operator or controller code — instead, it provides OpenShift ImageStream and BuildConfig manifests that define the notebook images available to users in the RHOAI Dashboard. These manifests are consumed by the rhods-operator via kustomize overlays.

The repository builds a comprehensive set of workbench images across multiple dimensions: IDE type (JupyterLab, Code Server, RStudio), accelerator support (CPU-only, NVIDIA CUDA, AMD ROCm), ML framework (PyTorch, TensorFlow, TrustyAI, LLM Compressor), and Python version (3.11, 3.12). Each workbench type maintains multiple versioned tags following an N through N-6 versioning scheme, providing approximately one year of image support. The images are built using Konflux CI/CD pipelines defined in the `.tekton/` directory, with multi-architecture support for amd64, arm64, ppc64le, and s390x.

In addition to interactive workbench images, the repository also produces pipeline runtime images for Elyra, enabling execution of data science pipelines from workbenches. These runtime images are available for Minimal, Data Science, PyTorch, TensorFlow, ROCm PyTorch, ROCm TensorFlow, and PyTorch LLM Compressor configurations.

### Authentication & Authorization

When deployed as workbench pods:
- **RHOAI 3.x pattern**: kube-rbac-proxy sidecar (injected by notebook controller) enforces RBAC authentication on port 8443/TCP before proxying to the workbench application
- **JupyterLab**: Runs with `--ServerApp.ip="" --ServerApp.allow_origin="*" --ServerApp.open_browser=False` — authentication is delegated entirely to the proxy sidecar
- Notebook images run as non-root user (UID 1001)
