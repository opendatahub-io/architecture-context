## Purpose
**Short**: REST API orchestrator that coordinates AI text generation with safety guardrails by invoking content detectors on LLM input and output.

**Detailed**: The FMS Guardrails Orchestrator serves as middleware between clients and language models (LLMs), providing content safety detection, filtering, and secure text generation. It orchestrates calls to multiple detector services, chunking services, and LLM backends through unified HTTP APIs.

The orchestrator supports multiple generation backends including TGIS (Text Generation Inference Server) via gRPC and OpenAI-compatible APIs via HTTP. Text is optionally split into chunks before being sent to detector services for safety analysis. Detectors classify text content for harmful patterns (e.g., hate speech, profanity) and the orchestrator filters or blocks responses based on configurable thresholds.

The service supports both unary and streaming request patterns, including Server-Sent Events (SSE) for streaming text generation with detection, and NDJSON for streaming content detection. It includes built-in support for TLS and mTLS encryption, OpenTelemetry-based observability (traces and metrics), and header passthrough for authentication token forwarding.

## Security

### TLS Configuration

The orchestrator supports three TLS modes for server-side connections, controlled via environment variables or CLI args:

| Mode | Configuration | Description |
|------|---------------|-------------|
| Plaintext | No TLS env vars set | No encryption (default) |
| TLS | `TLS_KEY_PATH` + `TLS_CERT_PATH` | Server-side TLS encryption |
| mTLS | TLS + `TLS_CLIENT_CA_CERT_PATH` | Mutual TLS with client certificate verification |

Client-side TLS is configured per-service in the YAML config file, supporting:
- Certificate path (`cert_path`)
- Private key path (`key_path`)
- CA certificate path (`client_ca_cert_path`)
- Insecure mode flag (`insecure`)
- Named TLS configurations for reuse across services
