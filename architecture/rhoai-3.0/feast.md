## Purpose
**Short**: Feast Operator manages the lifecycle of Feast feature store deployments on Kubernetes/OpenShift, automating the provisioning of online stores, offline stores, registries, and supporting services.

**Detailed**: The Feast Operator is a Kubernetes operator built with controller-runtime that watches `FeatureStore` custom resources and reconciles the desired state by creating and managing Deployments, Services, ConfigMaps, PersistentVolumeClaims, ServiceAccounts, RBAC resources, CronJobs, and OpenShift Routes. It supports multiple deployment topologies including ephemeral and persistent storage, local and remote registries, and optional offline store servers.

On OpenShift clusters, the operator automatically configures TLS using OpenShift service serving certificates, creates Routes for UI access, and integrates with the ODH trusted CA bundle for custom certificate authorities. The operator supports both Kubernetes RBAC-based and OIDC-based authorization models for securing Feast services. It also manages a namespace registry ConfigMap that tracks all deployed FeatureStore instances across namespaces, enabling service discovery.

The operator is deployed by the rhods-operator using kustomize overlays. The RHOAI overlay deploys into the `redhat-ods-applications` namespace, injecting RELATED_IMAGE environment variables for the operator itself, the feature server, and the CronJob CLI image. The Konflux build uses FIPS-compliant Go compilation flags (`GOEXPERIMENT=strictfipsruntime`).
