## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of Model Registry instances and the Model Catalog service for ML model metadata management in Red Hat OpenShift AI.

**Detailed**: The Model Registry Operator is a controller-runtime based Go operator that reconciles `ModelRegistry` custom resources to deploy and manage model registry services. Each ModelRegistry CR provisions a complete model registry stack consisting of a REST API service (model-registry), a gRPC metadata server (mlmd-grpc-server), and optionally a kube-rbac-proxy sidecar for authentication. The operator supports PostgreSQL and MySQL backends, with optional auto-provisioning of PostgreSQL databases.

In RHOAI 3.0, the operator has migrated from OAuth proxy to kube-rbac-proxy for authentication. It automatically detects and migrates existing OAuth proxy configurations to kube-rbac-proxy via both the mutating webhook and the reconcile loop. The operator also manages the Model Catalog, a separate service that provides a curated catalog of ML models with metadata collection, benchmark data, and configurable catalog sources backed by a PostgreSQL database.

The operator is deployed by the rhods-operator using the kustomize overlay at `config/overlays/odh`. Image references and configuration parameters are injected via `params.env` replacements at deploy time. The operator includes admission webhooks (mutating and validating) for ModelRegistry CRs and supports CRD storage version migration (v1alpha1 to v1beta1).

| Operator Manager | Go Binary | Main entry point; starts controller-runtime manager with health/readiness probes (8081), secure metrics endpoint (8443), leader election, and webhook server |
