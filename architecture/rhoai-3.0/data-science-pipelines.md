## Purpose
**Short**: Kubeflow Pipelines provides an end-to-end ML workflow orchestration platform for building, deploying, and managing reusable machine learning pipelines on Kubernetes.

**Detailed**: Data Science Pipelines (based on Kubeflow Pipelines) is the ML pipeline orchestration engine in Red Hat OpenShift AI. It enables data scientists and ML engineers to define, compile, and execute multi-step ML workflows as reusable pipelines. Each pipeline step executes as a container on Kubernetes, orchestrated by Argo Workflows as the execution engine.

The system consists of several microservices: an API server providing REST (HTTP) and gRPC endpoints for pipeline management, a persistence agent that synchronizes workflow state to the database, a scheduled workflow controller for recurring runs, a cache server for step-level execution caching, a metadata gRPC service for ML Metadata (MLMD), and a frontend UI. Pipelines can be stored either in a relational database (MySQL/PostgreSQL) or as Kubernetes CRDs (Pipeline and PipelineVersion resources). The v2 architecture uses a driver/launcher model where the driver resolves inputs and the launcher executes user containers within Argo Workflow steps.

For RHOAI, the images are built on UBI9 with FIPS-compliant Go build tags (`strictfipsruntime`), shipped through Konflux CI, and deployed by the data-science-pipelines-operator (DSPO) which manages the lifecycle of these services within user namespaces.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| ml-pipeline (API Server) | Go Service | Central API server exposing REST (8888/TCP) and gRPC (8887/TCP) endpoints; manages pipelines, experiments, runs, recurring runs, artifacts; includes webhook server (8443/TCP) for PipelineVersion CRD validation/mutation |
| ml-pipeline-ui (Frontend) | TypeScript/Node.js Service | Web UI (3000/TCP) for pipeline visualization, run management, artifact viewing; proxies API requests to API server |
| ml-pipeline-persistenceagent | Go Agent | Watches Argo Workflow CRs and ScheduledWorkflow CRs, reports status back to API server via gRPC |
| ml-pipeline-scheduledworkflow | Go Controller | Controller for ScheduledWorkflow CRD; creates Argo Workflow CRs on schedule based on cron or periodic triggers |
| cache-server | Go Webhook Server | Mutating admission webhook (8443/TCP) for caching pipeline step execution results; injects cached outputs when cache key matches |
| ml-pipeline-viewer-crd | Go Controller | Controller for Viewer CRD; creates Deployments and Services for TensorBoard viewer instances |
| metadata-grpc-server | Go/gRPC Service | ML Metadata (MLMD) gRPC service (8080/TCP) for storing and querying pipeline artifacts and execution metadata |
| metadata-envoy | Envoy Proxy | Envoy sidecar proxy (9090/TCP) for metadata gRPC service; provides gRPC-Web transcoding |
| metadata-writer | Python Agent | Watches Argo Workflows and writes execution metadata to MLMD gRPC service |
| kfp-driver | Go Binary | V2 pipeline step driver; resolves inputs, handles caching, manages artifact I/O; runs as init container in Argo Workflow pods |
| kfp-launcher | Go Binary | V2 pipeline step launcher; launches user containers and reports outputs; runs as sidecar in Argo Workflow pods. Includes both standard and FIPS-compliant builds |
| visualization-server | Python Service | Generates visualizations from pipeline data (8888/TCP) |
