## Purpose
**Short**: A high-throughput, memory-efficient LLM inference and serving engine optimized for Intel Gaudi (Habana) AI accelerators, providing OpenAI-compatible and gRPC APIs.

**Detailed**: vLLM Gaudi is a fork of the upstream vLLM project, adapted to run on Intel Gaudi 2 and Gaudi 3 AI accelerators (formerly Habana Labs HPU). It provides high-performance large language model inference with support for continuous batching, PagedAttention, tensor parallelism, and pipeline parallelism on Gaudi hardware. The component is built on PyTorch with the Habana PyTorch plugin and the `vllm-hpu-extension` library.

The container image ships two runtime modes: (1) an OpenAI-compatible HTTP API server built with FastAPI/Uvicorn that serves chat completions, text completions, embeddings, tokenization, scoring, reranking, and audio transcription endpoints; and (2) a gRPC adapter (`vllm-tgis-adapter`) that provides a TGIS (Text Generation Inference Service) compatible gRPC interface on port 8033 alongside the HTTP API on port 8000. This dual-protocol support enables integration with both OpenAI API clients and KServe/TGIS-based model serving infrastructure in Red Hat OpenShift AI.

The Gaudi-specific worker (`HPUWorker`, `HPUModelRunner`) handles model execution, KV cache management, and distributed inference across multiple Gaudi accelerators. The runtime supports tensor parallelism and pipeline parallelism for large model inference across multiple HPU devices. The container is built on RHEL 9.4 UBI base with Intel Gaudi Software Suite 1.21.3 and PyTorch 2.6.0.
