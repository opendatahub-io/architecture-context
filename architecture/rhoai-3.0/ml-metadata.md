## Purpose
**Short**: ML Metadata (MLMD) is a gRPC server and library for recording and retrieving metadata associated with ML workflows, including artifacts, executions, and contexts.

**Detailed**: ML Metadata provides a metadata store service that tracks the lineage and provenance of ML artifacts. It stores information about artifacts (datasets, models, metrics), executions (training runs, preprocessing steps), and contexts (experiments, pipelines) along with the events that connect them. This enables ML pipeline orchestration systems like Kubeflow Pipelines to record and query the complete history of ML workflow executions.

The RHOAI distribution ships a containerized C++ gRPC server (`odh-mlmd-grpc-server`) built from this repository. The server supports MySQL, PostgreSQL, and SQLite as backend metadata storage databases. It exposes a comprehensive gRPC API defined via Protocol Buffers for CRUD operations on types, artifacts, executions, contexts, events, and lineage graphs. The server supports optional TLS/SSL encryption for both the gRPC endpoint and database connections.

This component is a core dependency of Kubeflow Pipelines within the RHOAI platform, providing the persistent metadata storage layer for pipeline run tracking, artifact lineage, and experiment management.
