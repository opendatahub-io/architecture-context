## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates Ragas (Retrieval Augmented Generation Assessment) metrics for LLM evaluation, supporting both inline (in-process) and remote (Kubeflow Pipelines) execution modes.

**Detailed**: The llama-stack-provider-ragas component implements the Ragas evaluation framework as a pluggable provider for Meta's Llama Stack platform. It exposes Ragas' RAG evaluation metrics (answer relevancy, context precision, faithfulness, context recall) through the Llama Stack Eval API, enabling standardized LLM evaluation workflows within the RHOAI ecosystem.

The provider ships in two modes: an **inline provider** that executes Ragas evaluations directly within the Llama Stack server process, and a **remote provider** that offloads evaluation workloads to Kubeflow Pipelines (Data Science Pipelines) running on OpenShift. The remote provider constructs KFP pipeline definitions, submits them to a Kubeflow Pipelines endpoint, and retrieves results from S3-compatible storage. Both modes wrap the Llama Stack inference API (for LLM completions and embeddings) into Ragas-compatible LangChain-style interfaces.

The component is built and shipped as a container image (`odh-trustyai-ragas-lls-provider-dsp-rhel9`) via Konflux pipelines, based on UBI9 Python 3.12. It is consumed as a base image for Kubeflow Pipeline components and as a provider plugin within Llama Stack distributions deployed on OpenShift through the LlamaStackDistribution custom resource.
