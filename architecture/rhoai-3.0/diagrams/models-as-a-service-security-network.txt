Authentication & Authorization Flow:
  1. Client sends Bearer Token (OpenShift user token or SA token)
  2. Kuadrant AuthPolicy validates via Kubernetes TokenReview
     - Audiences: maas-default-gateway-sa
  3. AuthPolicy enriches identity with tier via maas-api /v1/tiers/lookup callback
  4. For model inference: SubjectAccessReview checks llminferenceservices:post
  5. Rate limiting applied based on resolved tier (free/premium/enterprise)
  6. Request forwarded to backend with validated identity

Secrets:
  - default-gateway-tls (kubernetes.io/tls)
    - Purpose: TLS certificate for maas-default-gateway HTTPS listener
    - Provisioned By: cert-manager or cluster default
    - Auto-Rotate: Yes

Tier-to-Group Mapping:
  - ConfigMap: tier-to-group-mapping
  - Maps OpenShift user groups to subscription tiers
  - Tiers: free, premium, enterprise (with precedence levels)
