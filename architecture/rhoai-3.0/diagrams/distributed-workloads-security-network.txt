Authentication & Authorization:
- Kubernetes API: Bearer Token (ServiceAccount) → kube-apiserver RBAC
- Notebook (RHOAI 3.0): kube-rbac-proxy sidecar (inject-auth annotation)
- Prometheus API: Bearer Token → OpenShift OAuth
- Ray Dashboard: No auth (cluster-internal only, ClusterIP service)

Secrets:
- AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY (env var) - S3 access, user-provided, no auto-rotate
- HF_TOKEN (env var) - HuggingFace model access, user-provided, no auto-rotate
- NOTEBOOK_USER_TOKEN (env var) - OpenShift user auth for notebook tests, OpenShift OAuth

Security Notes:
- NCCL/RDMA inter-node communication is plaintext (performance-critical path)
- Ray internal communication (GCS, Object Store) is plaintext (cluster-internal)
- Ray Dashboard has no authentication (relies on ClusterIP network isolation)
- All external egress uses TLS 1.2+
- Notebook auth migrated from inject-oauth to inject-auth in RHOAI 3.0
