    * "" (core): pods, services, serviceaccounts, configmaps, events, secrets, endpoints
    * apps: deployments
    * batch: jobs
    * ray.io: rayclusters, rayjobs, rayservices (including status and finalizers)
    * networking.k8s.io: ingresses, networkpolicies, ingressclasses
    * route.openshift.io: routes
    * cert-manager.io: certificates, issuers (including status)
    * gateway.networking.k8s.io: gateways, httproutes, referencegrants
    * rbac.authorization.k8s.io: roles, rolebindings, clusterroles, clusterrolebindings
    * coordination.k8s.io: leases
    * authentication.k8s.io: tokenreviews
    * authorization.k8s.io: subjectaccessreviews
    * config.openshift.io: authentications, oauths
    * operator.openshift.io: kubeapiservers
  - Verbs: get, list, watch, create, update, patch, delete

- ClusterRoleBinding: kuberay-operator
  - Binds: kuberay-operator (ClusterRole)
  - To: kuberay-operator (ServiceAccount, opendatahub namespace)

Network Policies:
- Created by NetworkPolicy Controller for each RayCluster
- Isolates Ray cluster network traffic
- Allows communication between Ray head and worker pods
- Allows ingress to Ray Dashboard (8265/TCP)
- Allows ingress to Ray GCS (6379/TCP)

Security Context Constraints (OpenShift):
- SCC: run-as-ray-user
  - Run As User: 1000 (Ray user)
  - Capabilities: Drop ALL
  - Privilege Escalation: Denied
  - Purpose: Allows Ray pods to run as non-root user 1000

Secrets:
- kuberay-webhook-server-cert (kubernetes.io/tls)
  - TLS certificate for webhook server
  - Provisioned by: OpenShift Service CA
  - Auto-Rotate: Yes

- Ray cluster mTLS certs (kubernetes.io/tls)
  - mTLS certificates for Ray internal communication
  - Provisioned by: cert-manager
  - Auto-Rotate: Yes
  - Optional: Only when cert-manager integration is enabled

Authentication & Authorization:
- Webhook endpoints: mTLS client certificate (Kubernetes API Server only)
- Metrics endpoint (/metrics): None (open to cluster for Prometheus scraping)
- Health/Readiness endpoints: None (internal cluster probes)
- Kubernetes API access: Bearer Token (Service Account kuberay-operator)
