Authentication Mechanisms:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Kubernetes API Server → Webhook Server
   - Method: mTLS (mutual TLS)
   - Client Certificate: Kubernetes API Server client cert
   - Server Certificate: kuberay-webhook-server-cert
   - Enforcement: Kubernetes API server validates webhook TLS

2. Operator → Kubernetes API Server
   - Method: Bearer Token (ServiceAccount token)
   - ServiceAccount: kuberay-operator
   - Namespace: opendatahub
   - Token: Auto-mounted at /var/run/secrets/kubernetes.io/serviceaccount/token

3. User → Kubernetes API Server
   - Method: Bearer Token (user credentials)
   - Source: kubectl context, OpenShift OAuth, etc.

4. Ray Worker → Ray Head (Optional mTLS)
   - Method: mTLS (if enabled via cert-manager)
   - Certificates: Provisioned by cert-manager
   - Managed by: mTLS Controller

5. Prometheus → Operator Metrics
   - Method: None (unauthenticated, cluster-internal only)
   - Endpoint: /metrics (8080/TCP)

Network Isolation:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

NetworkPolicy Controller:
  - Automatically creates NetworkPolicy resources for Ray clusters
  - Isolates Ray cluster traffic to pods within the same cluster
  - Allows ingress/egress based on RayCluster configuration
  - Default: Deny all traffic except explicitly allowed

Typical NetworkPolicy:
  - Ingress:
    * Allow from Ray Head to Ray Workers (GCS)
    * Allow from external ingress (if configured)
  - Egress:
    * Allow to Kubernetes DNS (53/UDP)
    * Allow to Kubernetes API Server (6443/TCP)
    * Allow to external services (if configured)
