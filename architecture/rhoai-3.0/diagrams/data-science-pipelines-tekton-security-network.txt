RBAC Summary:

  ClusterRoles:
  - ds-pipeline-persistenceagent-clusterrole
    - argoproj.io: workflows [get, list, watch]
    - kubeflow.org: scheduledworkflows [get, list, watch]
    - tekton.dev: pipelineruns, taskruns, conditions [create, get, list, watch, update, patch, delete]

  - tekton-pipelineloop-controller-cluster-access
    - tekton.dev: runs, taskruns, pipelineruns [get, list, create, update, delete, patch, watch]
    - tekton.dev: runs/status, taskruns/status, pipelineruns/status, runs/finalizers [full access]
    - custom.tekton.dev: pipelineloops [full access]

  - tekton-pipelineloop-webhook-cluster-access
    - apiextensions.k8s.io: CRDs, CRDs/status [get, list, update, patch, watch]
    - admissionregistration.k8s.io: webhookconfigurations [list, watch, get, update]

  Namespaced Roles:
  - ds-pipeline: pods, tekton CRs, scheduledworkflows, SAR/TR, pipelineloops
  - ds-pipeline-scheduledworkflow-role: tekton CRs, scheduledworkflows/finalizers, pipelineloops
  - kubeflow-pipelines-cache-role: taskruns, pods, configmaps
  - ml-pipeline-viewer-controller-role: viewers/finalizers, deployments, services
  - pipeline-runner: broad access to tekton CRs, kubeflow.org/*, batch/jobs, pods, secrets, etc.

Secrets:
  - mysql-secret (Opaque) - DB credentials (username/password), Admin/Operator provisioned, No auto-rotate
  - mlpipeline-minio-artifact (Opaque) - S3 credentials (accesskey/secretkey/host/port), Admin/Operator provisioned, No auto-rotate
  - webhook-server-tls (kubernetes.io/tls) - Cache webhook TLS certs, cache-deployer Job provisioned, No auto-rotate
  - ds-pipelines-ui-proxy-tls (kubernetes.io/tls) - OAuth proxy serving cert, OpenShift cert signer, Yes auto-rotate

Authentication & Authorization:
  - ds-pipeline API (8888/8887): kubeflow-userid header / Bearer Token, enforced by API Server interceptor (SAR, TR)
  - ds-pipeline-ui (8443): OAuth Proxy (OpenShift OAuth), enforced by oauth-proxy sidecar
  - cache-server (443): K8s Admission Webhook TLS, enforced by Kubernetes API Server
  - ml-pipeline (multi-user/Istio): Istio mTLS + AuthorizationPolicy, enforced by Istio sidecar
