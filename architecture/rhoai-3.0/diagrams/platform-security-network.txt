EXTERNAL EGRESS:
──────────────────────────────────────────────────────────────────────────────────────

1. **Container Registries**
   - Destinations: quay.io, registry.redhat.io, registry.access.redhat.com
   - Port: 443/TCP (HTTPS)
   - Auth: Pull secrets (Opaque, .dockerconfigjson)
   - Purpose: Container image pulls for all workloads

2. **S3-Compatible Storage** (AWS, GCS, Azure, Minio)
   - Port: 443/TCP (HTTPS)
   - Encryption: TLS 1.2+
   - Auth: AWS IAM (SigV4), Access Key + Secret Key
   - Callers: KServe, DSP, Notebooks, Training Jobs, Feast

3. **External Databases** (PostgreSQL, MySQL, Snowflake)
   - Ports: 5432/TCP (PostgreSQL), 3306/TCP (MySQL)
   - Encryption: TLS 1.2+ (optional, configurable)
   - Auth: Username/Password, TLS client certs
   - Callers: Model Registry, DSP, Feast, TrustyAI

4. **Python/R Package Repositories**
   - PyPI: files.pythonhosted.org:443/TCP (HTTPS)
   - CRAN: cran.rstudio.com:443/TCP (HTTPS)
   - Auth: None (public)
   - Callers: Workbench pods (pip, conda, R)

5. **Git Repositories**
   - GitHub, GitLab: 443/TCP (HTTPS), 22/TCP (SSH)
   - Auth: Personal Access Token, SSH keys
   - Secrets: git-credentials (basic-auth)
   - Callers: Workbench pods (git clone)

6. **HuggingFace Hub**
   - Endpoint: huggingface.co:443/TCP (HTTPS)
   - Auth: API Token (HF_TOKEN)
   - Callers: KServe, Llama Stack, Notebooks

7. **Inference Provider Backends** (vLLM, TGI, Ollama)
   - Ports: Varies (HTTP/HTTPS)
   - Auth: Varies by provider
   - Callers: Llama Stack Operator

SERVICE MESH CONFIGURATION:
──────────────────────────────────────────────────────────────────────────────────────

PeerAuthentication (namespace-scoped):
- Mode: PERMISSIVE (default)
  - Accepts: mTLS + plaintext
  - Applied to: InferenceService pods in namespaces with serverless mode

AuthorizationPolicy (optional):
- Name: kserve-inference-access
- Action: ALLOW
- Principals: All authenticated users
- Optional: JWT validation via RequestAuthentication + AuthorizationPolicy

VirtualService (dynamic per workload):
- InferenceService: Routes /v1/models/{name}, /v2/models/{name}
- Notebooks: Routes /notebook/{namespace}/{name}
- Gateways: istio-ingressgateway, knative-serving/knative-ingress-gateway

DestinationRule (per service):
- TrafficPolicy: mTLS mode ISTIO_MUTUAL
- ConnectionPool: tcp.maxConnections, http.http1MaxPendingRequests
- OutlierDetection: consecutiveErrors, interval, baseEjectionTime

EnvoyFilter (ODH Model Controller):
- Purpose: SSL passthrough for LLM services
- Match: Gateway listener on 8443/TCP
- Action: Bypass TLS termination

COMPLIANCE NOTES:
──────────────────────────────────────────────────────────────────────────────────────

- **FIPS 140-2**: All components built with FIPS-compliant base images (UBI9)
- **TLS Minimum**: TLS 1.2 enforced across all external ingress/egress
- **mTLS**: Optional for service mesh (PERMISSIVE mode), required for webhooks
- **Secret Rotation**: TLS certificates auto-rotated by cert-manager (90-day default)
- **Audit Logging**: Kubernetes API audit logs capture all CR modifications
- **Network Policies**: Not shown (optional, can restrict pod-to-pod traffic)
- **Pod Security**: OpenShift SCC (restricted, nonroot) enforced on all pods
- **Image Scanning**: Konflux CI/CD scans all images for CVEs before promotion
