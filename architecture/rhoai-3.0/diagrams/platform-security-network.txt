Authentication Patterns
================================================================================

1. Gateway ext_authz (kube-auth-proxy)
   - Components: All Gateway-fronted services
   - Mechanism: OAuth2/OIDC session validation
   - Port: 8443/HTTPS
   - Flow: EnvoyFilter → kube-auth-proxy → OAuth/OIDC provider

2. kube-rbac-proxy sidecar
   - Components: Dashboard, Notebooks, Model Registry, TrustyAI, Guardrails
   - Mechanism: Kubernetes SubjectAccessReview
   - Port: 8443/HTTPS per pod
   - Flow: Validates user can perform action before proxying to backend

3. AuthPolicy (Kuadrant)
   - Components: LLMInferenceService
   - Mechanism: Gateway API policy attachment
   - Flow: Policy on Gateway + HTTPRoute

4. API Key
   - Components: vLLM, OVMS (optional)
   - Mechanism: Bearer Token in Authorization header
   - Port: 8000/HTTP

5. mTLS
   - Components: ML Metadata, TGIS, KServe (optional)
   - Mechanism: Client certificates
   - Flow: Mutual TLS for service-to-service

================================================================================
TLS Certificate Provisioning
================================================================================

Method: OpenShift service-ca (auto-rotate: Yes)
  Components: Dashboard, Model Registry, TrustyAI, Guardrails, kube-auth-proxy
  Secret: service.beta.openshift.io/serving-cert-secret-name annotation

Method: self-signed cert-controller (auto-rotate: Yes)
  Components: Training Operator
  Secret: webhook-cert

Method: OpenShift default ingress cert (auto-rotate: Depends)
  Components: data-science-gateway
  Secret: Gateway TLS secret

Method: self-managed CA (auto-rotate: No)
  Components: odh-model-controller (Ray TLS)
  Secrets: ray-ca-tls, ray-tls

================================================================================
Container Security Defaults
================================================================================

- Run as non-root: Yes (all components)
- UIDs: 1001, 2000, 5000, 65532, 65534 (per component)
- Base images: UBI9 / UBI9-minimal / UBI9-micro
- Seccomp: RuntimeDefault (Training Operator, TrustyAI Operator)
- Capabilities: drop ALL (Training Operator, TrustyAI Operator, TGIS)
- FIPS: GOEXPERIMENT=strictfipsruntime (Go), UBI crypto policies (all)

================================================================================
Network Policies
================================================================================

- redhat-ods-applications: Allow inter-component + ingress traffic
- rhods-dashboard-allow-ports: Allow 8443, 8043, 8143 from all
- per-PyTorchJob: Isolate training pods + allow monitoring
- per-DSPA: Pipeline component isolation
