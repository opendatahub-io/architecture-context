NOTES:
- Pod-to-Pod TLS: Enabled by default using OpenShift service-serving certificates
- Multi-tenancy: Each DSPA instance isolated to its own namespace
- Production deployment: Use external S3 and database; embedded Minio/MariaDB for dev/test only
- FIPS compliance: Built with FIPS-enabled Go runtime (GOEXPERIMENT=strictfipsruntime)
- Artifact storage: Pipeline artifacts in S3; metadata/lineage in relational database
- TLS everywhere: All internal communication encrypted (pod-to-pod, external routes)
- Namespace isolation: DSPA instances do not share resources across namespaces
