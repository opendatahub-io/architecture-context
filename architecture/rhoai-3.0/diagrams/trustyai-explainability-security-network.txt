RBAC Summary:
- ClusterRole: trustyai-role
  - API Groups: "" (core)
  - Resources: configmaps
  - Verbs: create, delete, get, list, patch, update, watch

- ClusterRoleBinding: trustyai-clusterrolebinding
  - Scope: Cluster-scoped
  - Role: trustyai-role (ClusterRole)
  - ServiceAccount: trustyai-serviceaccount

Authentication & Authorization:
- Service-level: No built-in auth (all REST endpoints open internally)
- Production: Auth proxy sidecar injected by TrustyAI Operator
  - RHOAI 2.x: oauth-proxy
  - RHOAI 3.x: kube-rbac-proxy
- /q/metrics: Bearer Token (SA token, ServiceMonitor config)

Secrets:
- TLS certificate (kubernetes.io/tls) - HTTPS on 4443/TCP, provisioned by Operator/cert-manager
- Database credentials (Opaque) - MariaDB/MySQL username/password, provisioned by Operator
- MinIO credentials (Opaque) - Access key/secret key, provisioned by Operator

Storage Backends:
- PVC: PersistentVolumeClaim at /inputs (default, CSV files)
- MinIO: S3-compatible object storage
- Database: MariaDB/MySQL/H2 via Hibernate ORM
- Memory: In-process (dev/test only)
