Secrets Management
──────────────────

1. odh-model-controller-webhook-cert (kubernetes.io/tls)
   - Purpose: TLS certificate for webhook server
   - Provisioned by: OpenShift Service CA (via service annotation)
   - Auto-rotate: Yes (Service CA managed)
   - Mounted at: /tmp/k8s-webhook-server/serving-certs/

2. NIM API Key Secrets (Opaque)
   - Purpose: Nvidia NGC API keys for NIM account validation
   - Provisioned by: User/Administrator
   - Auto-rotate: No (manual management)
   - Referenced by: NIM Account CR (apiKeySecret field)
   - Used by: Controller to call NGC API

3. Image Pull Secrets (kubernetes.io/dockerconfigjson)
   - Purpose: Container registry credentials for NIM model images
   - Provisioned by: Controller (from NIM Account)
   - Auto-rotate: No (synced from NGC account)
   - Used by: Predictor pods to pull NIM model images

4. Managed Secrets (Opaque)
   - Purpose: InferenceService-related secrets
   - Provisioned by: Controller
   - Label: opendatahub.io/managed=true
   - Watched by: Secret controller (label-based caching)

════════════════════════════════════════════════════════════════════════════════════════

Network Isolation
─────────────────

- Controller runs in dedicated namespace (typically 'opendatahub')
- Controller creates NetworkPolicies for InferenceService predictor pods
- Webhook endpoint only accessible from Kubernetes API Server (ClusterIP service)
- Metrics endpoint only accessible within cluster (ClusterIP service)
- Health probes only accessible within pod (not exposed via service)
- No direct external exposure (controller itself)
- External communication via OpenShift Router for created InferenceServices

════════════════════════════════════════════════════════════════════════════════════════

Security Controls
─────────────────

1. Pod Security:
   - Non-root user: UID 2000
   - FIPS-compliant base image: ubi9/ubi-minimal
   - Resource limits: CPU 500m, Memory 2Gi

2. Network Security:
   - TLS 1.2+ for all HTTPS endpoints
   - mTLS for webhook endpoints (TLS client cert validation)
   - Bearer token authentication for Kubernetes API
   - API key authentication for Nvidia NGC
   - Optional TLS verification skip for Model Registry (MR_SKIP_TLS_VERIFY)

3. RBAC Controls:
   - Least privilege ServiceAccount (odh-model-controller)
   - ClusterRole with explicit resource/verb permissions
   - Separate leader-election role
   - Separate metrics-auth role

4. Admission Control:
   - Validating webhooks for Pods, InferenceServices, InferenceGraphs, LLMInferenceServices, Accounts
   - Mutating webhooks for Pods, InferenceServices
   - Webhook fail-closed policy (default Kubernetes behavior)

5. Secrets Management:
   - Service CA for webhook certificates (auto-rotation)
   - Label-based caching to limit secret exposure
   - Explicit secret management for NIM accounts
