Network Policies:
─────────────────────────────────────────────────────────────────────────

The operator creates NetworkPolicies for training jobs if configured, but
does not enforce network policies on itself. Training jobs communicate over
plaintext TCP (port 23456) within the cluster network.

Trust Boundaries:
─────────────────────────────────────────────────────────────────────────

1. EXTERNAL → KUBERNETES API: User authentication via kubeconfig
2. KUBERNETES API → OPERATOR: ServiceAccount token authentication
3. KUBERNETES API → WEBHOOK: mTLS client certificate validation
4. OPERATOR → KUBERNETES API: ServiceAccount token (RBAC enforced)
5. TRAINING PODS → TRAINING PODS: No encryption, framework-specific auth
6. TRAINING PODS → EXTERNAL SERVICES: HTTPS/TLS with credentials

Security Notes:
─────────────────────────────────────────────────────────────────────────

1. Webhook endpoints use mTLS for authentication (K8s API Server validates
   client certificates)
2. Metrics endpoint (/metrics:8080) has no authentication - relies on
   network isolation within the cluster
3. Training pod inter-communication (port 23456) is plaintext TCP - training
   frameworks (PyTorch, TensorFlow, etc.) handle their own authentication
4. Operator runs with minimal privileges (non-root, read-only filesystem,
   FIPS-compliant)
5. Training jobs inherit security context from job specification (user-defined)
6. Secrets (image pull secrets, storage credentials) are mounted into training
   pods as needed
