DEPLOYMENT MODES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Serverless Mode (Default):
- Dependencies: Knative Serving 1.8+, Istio 1.17+
- Autoscaling: Request-based with scale-to-zero
- Traffic: Istio VirtualService for routing and mTLS
- mTLS: STRICT mode enforced
- Canary: Traffic splitting via DestinationRule

Raw Deployment Mode:
- Dependencies: Kubernetes 1.25+ only
- Autoscaling: HorizontalPodAutoscaler or KEDA
- Traffic: Kubernetes Ingress or OpenShift Route
- mTLS: Optional (via service mesh annotation)
- Canary: Manual via multiple deployments

ModelMesh Mode:
- Dependencies: ModelMesh controller
- Use Case: High-density multi-model serving
- CRD: TrainedModel instead of InferenceService
- Traffic: Internal ClusterIP routing only
