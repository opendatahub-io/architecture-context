RBAC Summary:
==============

ClusterRole: manager-role
  - Resources: configmaps, secrets, services, deployments, modelregistries (full CRUD)
  - Resources: routes (OpenShift), networkpolicies, rolebindings, roles, clusterrolebindings
  - Resources: tokenreviews, subjectaccessreviews (authentication/authorization)
  - Resources: storageversionmigrations (for v1alpha1→v1beta1 migration)
  - Verbs: get, list, watch, create, update, patch, delete

ClusterRole: modelregistry-editor-role
  - Resources: modelregistries (all verbs + status get)
  - Purpose: Allow users to manage ModelRegistry CRs

ClusterRole: modelregistry-viewer-role
  - Resources: modelregistries (get, list, watch + status get)
  - Purpose: Read-only access to ModelRegistry CRs

ClusterRole: metrics-reader
  - Resources: /metrics (non-resource URL)
  - Verbs: get
  - Purpose: Allow Prometheus to scrape metrics

Role: leader-election-role (Namespace-scoped)
  - Resources: configmaps, leases (coordination.k8s.io)
  - Verbs: get, list, watch, create, update, patch, delete
  - Purpose: Operator leader election

Per-Registry RBAC:
  - RoleBinding: registry-user-{registry-name}
  - Role: registry-user-{registry-name}
  - Purpose: Grant access to specific registry service
  - Enforcement: kube-rbac-proxy performs SubjectAccessReview

Secrets:
=========

Operator Secrets (Namespace: opendatahub/redhat-ods-applications):
  - webhook-server-cert (kubernetes.io/tls)
    - Purpose: Webhook server TLS certificate
    - Provisioner: cert-manager or OpenShift CA
    - Rotation: Automatic (OpenShift CA) or cert-manager renewal

Per-Registry Secrets (Namespace: {user-namespace}):
  - {registry-name}-kube-rbac-proxy (kubernetes.io/tls)
    - Purpose: kube-rbac-proxy TLS cert/key
    - Provisioner: OpenShift Service CA (service.beta.openshift.io/serving-cert-secret-name)
    - Rotation: Automatic (90 days, OpenShift Service CA)

  - model-registry-db (Opaque)
    - Keys: database-password, database-user, database-name
    - Purpose: Database connection credentials
    - Rotation: Manual

  - model-registry-db-credential (Opaque) [Optional]
    - Keys: ca-bundle.crt
    - Purpose: Database TLS CA certificates
    - Rotation: Manual

  - {registry-name}-postgres-ssl-cert (Opaque) [Optional]
    - Keys: tls.crt
    - Purpose: PostgreSQL client certificate for mutual TLS
    - Rotation: Manual

  - {registry-name}-postgres-ssl-key (Opaque) [Optional]
    - Keys: tls.key
    - Purpose: PostgreSQL client private key
    - Rotation: Manual

  - {registry-name}-mysql-ssl-cert (Opaque) [Optional]
    - Keys: tls.crt
    - Purpose: MySQL client certificate for mutual TLS
    - Rotation: Manual

  - {registry-name}-mysql-ssl-key (Opaque) [Optional]
    - Keys: tls.key
    - Purpose: MySQL client private key
    - Rotation: Manual

Service Mesh Integration (Optional):
=====================================

When Istio integration is enabled:
  - Istio Gateway: Configured via service-mesh-refs ConfigMap
  - Control Plane: Specified in DEFAULT_CONTROL_PLANE env var
  - Ingress Gateway: Specified in DEFAULT_ISTIO_INGRESS env var (default: ingressgateway)
  - mTLS: Managed by service mesh (automatic sidecar injection)
  - Custom domains: Configured via Istio Gateway CR

Container Security:
===================

Operator Controller:
  - Base Image: registry.access.redhat.com/ubi9/ubi-minimal
  - SecurityContext:
    - runAsNonRoot: true
    - runAsUser: 65532
    - runAsGroup: 65532
    - capabilities.drop: [ALL]
  - FIPS: strictFIPS mode enabled (via Dockerfile.konflux)

Model Registry Instances:
  - REST Image: quay.io/opendatahub/model-registry:latest
  - gRPC Image: quay.io/opendatahub/mlmd-grpc-server:latest
  - Proxy Image: quay.io/openshift/origin-kube-rbac-proxy:latest
  - SecurityContext:
    - runAsNonRoot: true
    - capabilities.drop: [ALL]
  - SCC: restricted-v2 (OpenShift)

Network Flow Summary:
======================

External Access (Authenticated):
  Client → Router (443 HTTPS) → Service (8443 HTTPS) → kube-rbac-proxy (validates token)
  → REST (8080 HTTP localhost) → DB (5432/3306 TLS optional)

Internal gRPC Access:
  Internal Client → Service (9090 gRPC) → gRPC Container → DB (5432/3306 TLS optional)

Operator Reconciliation:
  Controller → K8s API (6443 HTTPS) → Watch/Create/Update Resources

Webhook Validation:
  K8s API → webhook-service (9443 HTTPS, TLS client cert) → Controller

Metrics Collection:
  Prometheus → controller-manager-metrics-service (8443 HTTPS, Bearer Token) → /metrics
