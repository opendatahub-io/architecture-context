RBAC Summary:
  No RBAC manifests shipped with this repository.
  RBAC is configured by the deploying component (e.g., RHOAI operator).

  In rbac mode (kube-rbac-proxy):
  - The proxy's ServiceAccount needs permission to create SubjectAccessReview resources
  - Performs SAR checks against Kubernetes API server using caller's bearer token

Authentication & Authorization:
  - /* (auth mode):     OAuth2/OIDC Cookie or Bearer JWT → redirect to IdP on missing session
  - /oauth2/auth:       Cookie/Bearer → 202 (authenticated) or 401 (unauthenticated)
  - /* (rbac mode):     Bearer Token + SubjectAccessReview → Kubernetes RBAC check
  - /ping, /ready:      No auth required (health checks)

Secrets:
  - kube-auth-proxy-secret (Opaque) - cookie-secret for session encryption,
    optionally client-secret for OIDC. Provisioned by deploying component. No auto-rotate.
  - TLS cert/key (kubernetes.io/tls) - TLS termination for HTTPS listener.
    Provisioned by deploying component / cert-manager. Rotation depends on provisioner.
  - ServiceAccount token (kubernetes.io/service-account-token) - OpenShift OAuth
    client credentials (auto-detected from pod). Auto-rotated by Kubernetes.
  - JWT signing key (Opaque) - JWT token signing (optional, embedded placeholder).
    Provisioned by build process. No auto-rotate.
