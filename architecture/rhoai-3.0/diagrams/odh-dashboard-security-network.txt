Authentication Flow:
  1. User accesses dashboard via data-science-gateway (Gateway API)
  2. kube-rbac-proxy performs SubjectAccessReview:
     - API Group: project.openshift.io
     - Resource: projects
     - Verb: list
  3. On success, user identity forwarded via headers:
     - X-Auth-Request-User
     - X-Auth-Request-Groups
  4. Backend passes user token (X-Forwarded-Access-Token) to Kubernetes API
  5. Fine-grained RBAC enforced by Kubernetes API on every operation
