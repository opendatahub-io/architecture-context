RBAC Summary:
- ServiceAccount: {notebook-name}-sa (per workbench instance)
  - Namespace: {user-namespace}
  - Permissions: Limited to namespace resources (pods, services, secrets)
  - No cluster-wide permissions
  - OAuth proxy manages authentication via OpenShift OAuth

- Workbench Image Selection:
  - Managed by ODH Dashboard via ImageStream API
  - Dashboard reads ImageStreams from openshift namespace
  - User selects image via UI, Notebook Controller creates pod

Service Mesh Configuration:
- OAuth Proxy: Mandatory sidecar for all workbench pods
  - Config Secret: {notebook-name}-oauth-config
  - Cookie Secret: {notebook-name}-oauth-cookie-secret
  - Validates OpenShift tokens
  - Enforces namespace access control

Network Policies:
- Ingress: Only from OpenShift Router
- Egress: Controlled by namespace NetworkPolicy
  - Allow: DNS (53/UDP), HTTPS (443/TCP), K8s API (6443/TCP)
  - Deny: All other outbound by default (configurable per namespace)

Secrets:
- {notebook-name}-oauth-config (Opaque)
  - Purpose: OAuth proxy configuration
  - Provisioned By: ODH Notebook Controller
  - Auto-Rotate: No

- {notebook-name}-oauth-cookie-secret (Opaque)
  - Purpose: OAuth session cookies
  - Provisioned By: ODH Notebook Controller
  - Auto-Rotate: No

- git-credentials (kubernetes.io/basic-auth)
  - Purpose: Git repository authentication
  - Provisioned By: User (mounted as volume)
  - Keys: username, password/token

- aws-credentials (Opaque)
  - Purpose: S3 storage access
  - Provisioned By: User (environment variables)
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY

- database-credentials (Opaque)
  - Purpose: Database connection credentials
  - Provisioned By: User
  - Keys: DB_HOST, DB_USER, DB_PASSWORD

Container Image Security:
- Build Process: Konflux CI/CD pipelines
- Vulnerability Scanning: Clair, Snyk (no HIGH/CRITICAL CVEs)
- Image Signing: Cosign signatures
- Base Images: Red Hat UBI9 (security-hardened)
- Updates: Security patches within 14 days of disclosure

Resource Quotas (per workbench):
- CPU Request: 0.5-4 cores (configurable)
- CPU Limit: 2-16 cores
- Memory Request: 1-8 GiB
- Memory Limit: 4-32 GiB
- Storage: 20-100 GiB PVC
- GPU: 0-1 (dedicated, no sharing)

Health Monitoring:
- Liveness Probe: /healthz (HTTP GET 8888)
- Readiness Probe: /api/status (HTTP GET 8888)
- Startup Timeout: 120 seconds
- Restart Policy: Always
