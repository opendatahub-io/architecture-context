RBAC Summary:
- This component does NOT define RBAC resources (ClusterRoles, Roles, RoleBindings)
- RBAC for workbench access is managed by:
  - odh-notebook-controller (creates per-notebook RBAC)
  - kube-rbac-proxy sidecar (enforces RBAC at pod level)
  - rhods-operator (platform-level RBAC)

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription credentials for RStudio
  BuildConfig package installation. Provisioned manually by admin. No auto-rotation.

Container Security:
- Run as user: 1001 (non-root)
- Base OS: UBI 9 (Red Hat Universal Base Image)
- FIPS compliance: Enabled (via UBI 9 crypto policies)
- Disconnected support: Yes (images use digest-pinned references)
- Package management: dnf with --nobest --skip-broken for CVE remediation
- Architectures: amd64, arm64, ppc64le, s390x

Authentication & Authorization:
- RHOAI 3.x pattern: kube-rbac-proxy sidecar injected by notebook controller
  enforces RBAC authentication on port 8443/TCP before proxying to workbench
- JupyterLab runs with authentication disabled (delegated to proxy sidecar)
- Notebook images run as non-root user (UID 1001)
