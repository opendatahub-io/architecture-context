CONTAINER SECURITY:
- Base Image: registry.access.redhat.com/ubi9/ubi-minimal:9.6
- Run As User: 2000 (non-root)
- Run As Group: 0 (root group, OpenShift compatibility)
- FIPS Compliant: Yes (enabled operator feature)
- Disconnected Support: Yes (HF_HUB_OFFLINE=1)
- Multi-Arch: amd64, arm64, ppc64le, s390x

TLS CONFIGURATION:
- TLS Termination: Optional (via --ssl-keyfile, --ssl-certfile, --ssl-ca-certs)
- Client Certificate Auth: Optional (via --ssl-cert-reqs)
- Certificate Hot-Reload: Yes (via --enable-ssl-refresh, uses watchfiles)
- TLS Version: Depends on system OpenSSL defaults

AUTHENTICATION:
- /v1/* endpoints: API Key via Authorization header (optional, set via --api-key)
- /health, /ping, /version: No auth required (health probes)
- /metrics: No auth required (Prometheus scraping)
- gRPC 8033: No built-in auth; relies on network-level security
