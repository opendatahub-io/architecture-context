## Purpose
**Short**: OpenVINO Model Server is a high-performance C++ inference serving runtime that hosts AI/ML models and exposes them via gRPC and REST APIs compatible with TensorFlow Serving, KServe v2, and OpenAI standards.

**Detailed**: OpenVINO Model Server (OVMS) is an Intel-developed inference serving solution optimized for deployment on Intel architectures. It serves as one of the model serving runtimes within the Red Hat OpenShift AI (RHOAI) platform, enabling users to deploy and serve machine learning models at scale in containerized environments.

The server supports multiple inference frontends: TensorFlow Serving API (v1), KServe Inference Protocol v2, and OpenAI-compatible API (v3) for chat completions, completions, embeddings, reranking, and image generation. Models can be loaded from local storage, Amazon S3, Google Cloud Storage, Azure Blob Storage, or pulled directly from HuggingFace Hub. OVMS supports advanced features including DAG (Directed Acyclic Graph) pipelines for chaining models, MediaPipe graph execution, model versioning, dynamic batching, continuous batching for LLMs, and Prometheus-compatible metrics.

In the RHOAI platform, OVMS is deployed as a container image used by the KServe InferenceService or ModelMesh serving infrastructure. It is not an operator itself -- it is a serving runtime binary packaged in a container that receives inference requests from the platform's ingress infrastructure. The RHOAI build uses Konflux CI/CD with Link-Time Optimization (LTO) enabled, builds OpenVINO from source (opendatahub-io fork), and includes GPU driver support.
