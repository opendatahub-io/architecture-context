## Purpose

**Short**: ODH Notebook Controller extends the upstream Kubeflow Notebook Controller with OpenShift/RHOAI-specific capabilities including Gateway API ingress, kube-rbac-proxy authentication, network policies, CA certificate injection, and Elyra pipeline integration.

**Detailed**: The ODH Notebook Controller is a Kubernetes controller and mutating webhook that operates alongside the upstream Kubeflow Notebook Controller. While the upstream controller manages the lifecycle of Notebook StatefulSets, this controller adds OpenShift and RHOAI-specific enhancements to each Notebook CR.

In RHOAI 3.x, the controller creates HTTPRoute CRs (Gateway API) in a central namespace to expose each notebook through the platform's data-science-gateway. It manages cross-namespace routing using ReferenceGrants, creates NetworkPolicies for notebook pod isolation, and optionally injects kube-rbac-proxy sidecars for RBAC-based authentication. The mutating webhook intercepts Notebook create/update operations to inject the kube-rbac-proxy sidecar container, mount CA certificate bundles, resolve ImageStream references, and mount pipeline runtime configuration.

The controller also handles migration from RHOAI 2.x by cleaning up legacy OAuthClient resources via finalizers. It integrates with Data Science Pipelines (DSPA) to generate Elyra runtime configuration secrets, enabling notebooks to execute pipelines. The controller is built with FIPS-compliant Go compilation flags and runs as a non-root user.
