## Purpose
**Short**: A high-performance text generation inference server (TGIS) that serves large language models via gRPC and HTTP APIs, originally forked from HuggingFace Text Generation Inference and developed by IBM.

**Detailed**: Text Generation Inference Server (TGIS) is a production-grade inference runtime for large language models (LLMs) used within Red Hat OpenShift AI. It provides optimized serving of causal language models (decoder-only) and sequence-to-sequence models (encoder-decoder) with support for tensor parallelism across multiple GPUs via NCCL.

The system consists of three main components: a Rust-based **launcher** that orchestrates process lifecycle, a Rust-based **router** that handles incoming gRPC/HTTP requests and performs batching and scheduling, and a Python-based **server** (model shard) that loads models and performs GPU inference using PyTorch. The router communicates with model shards over Unix domain sockets using gRPC. The server supports multiple inference engines including HuggingFace Transformers, IBM FMS (Foundation Model Stack), DeepSpeed, ONNX Runtime, and a native TGIS engine with custom CUDA kernels for flash attention and GPTQ quantization.

TGIS is deployed as a container image running on GPU-equipped nodes in OpenShift/Kubernetes. It is typically used as the backend runtime for KServe InferenceService custom resources, serving models through the RHOAI model serving stack.
