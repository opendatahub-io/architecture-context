## Purpose
**Short**: A reverse proxy sidecar that enables disaggregated prefill/decode (P/D) inference by routing LLM chat completion requests between prefiller and decoder vLLM instances.

**Detailed**: The llm-d-routing-sidecar is a lightweight HTTP/HTTPS reverse proxy designed to run as a sidecar container alongside a vLLM decoder instance in a disaggregated LLM serving architecture. It intercepts OpenAI-compatible chat completion and legacy completion API requests, and when a prefiller target is specified via the `x-prefiller-host-port` HTTP header, it orchestrates a two-phase inference flow: first sending a prefill request to a remote prefiller pod to compute KV cache data, then forwarding the decode request (with KV transfer parameters) to the local decoder vLLM instance.

The sidecar supports multiple P/D connector protocols — NIXL v2 (primary), NIXL v1 (deprecated), and LMCache (deprecated) — and includes SSRF (Server-Side Request Forgery) protection that dynamically maintains an allowlist of valid prefill targets by watching Kubernetes InferencePool resources and their associated pods. TLS is enabled by default with configurable certificate paths, and the proxy supports both secure and insecure modes for prefiller and decoder connections.

### SSRF Protection

When enabled via `--enable-ssrf-protection`:
- Watches the specified InferencePool CR in the given namespace
- Extracts pod label selectors from InferencePool spec
- Maintains a dynamic allowlist of pod IPs and names matching the selector
- Validates `x-prefiller-host-port` header against the allowlist (host/IP only, any port allowed)
- Returns HTTP 403 Forbidden for requests targeting pods not in the allowlist
- Allowlist auto-updates via Kubernetes informers with 30-second resync
