## Purpose
**Short**: TrustyAI is a Java library and Quarkus REST service providing AI fairness metrics, explainability algorithms, and data drift detection for ML models served via KServe and ModelMesh.

**Detailed**: TrustyAI Explainability is the core engine behind TrustyAI's responsible AI capabilities within Red Hat OpenShift AI. It consists of a multi-module Java project that provides both a reusable library (`explainability-core`) and a containerized Quarkus service (`explainability-service`) that exposes REST APIs for fairness metrics (Statistical Parity Difference, Disparate Impact Ratio), drift detection (KS Test, Meanshift, Fourier MMD), and local/global explainability algorithms (LIME, SHAP, Counterfactual, Partial Dependence Plots, Time-Series Saliency).

The service integrates with KServe and ModelMesh inference platforms by consuming inference payloads -- either via direct HTTP POST from ModelMesh's payload processor mechanism or via Knative CloudEvents from KServe's inference logger. It reconciles input/output pairs, stores them in configurable backends (PVC filesystem, MinIO, or relational database via Hibernate), and computes scheduled fairness/drift metrics that are exported as Prometheus gauges (prefixed `trustyai_`). The service supports both ODH and RHOAI profiles, with RHOAI disabling explainer endpoints (SHAP, LIME, Counterfactual, TSSaliency, global explainers) while keeping fairness and drift metrics enabled.

The `explainability-connectors` module provides protocol-level integration with KServe v1/v2 inference APIs, including generated gRPC stubs from the KServe v2 Predict protocol (`grpc_predict_v2.proto`), enabling TrustyAI to invoke model inference directly for explanation computation.
