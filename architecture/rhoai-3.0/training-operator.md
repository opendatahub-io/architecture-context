## Purpose
**Short**: Kubernetes operator for managing distributed machine learning training jobs across multiple frameworks (PyTorch, TensorFlow, XGBoost, MPI, PaddlePaddle, JAX).

**Detailed**: The Kubeflow Training Operator (KFTO) is a Kubernetes-native operator that enables scalable distributed training of AI/ML models. It provides Custom Resource Definitions (CRDs) for six distinct ML training frameworks: PyTorchJob, TFJob, XGBoostJob, MPIJob, PaddleJob, and JAXJob. Each CRD abstracts the complexity of distributed training by managing the lifecycle of training pods, headless services for inter-pod communication, and optional gang scheduling via Volcano or scheduler-plugins.

The operator uses controller-runtime and follows the Kubebuilder operator pattern. For each training job CR, it creates and manages worker/master pods, headless ClusterIP services for peer discovery, and optionally PodGroups for gang scheduling. The PyTorch controller additionally creates NetworkPolicy resources to isolate training job traffic and manages HorizontalPodAutoscaler resources for elastic training. The operator also provides validating webhooks for all supported job types to enforce correctness of training job specifications at admission time, with self-managed TLS certificate rotation.

In RHOAI, the operator is deployed via kustomize overlays applied by the rhods-operator. The RHOAI overlay configures the operator image, webhook certificate secret names, and Prometheus metrics monitoring via a PodMonitor. The operator runs in the `opendatahub` namespace with cluster-wide scope to manage training jobs across all namespaces.

### Network Policies (Dynamically Created)

The PyTorch controller dynamically creates a NetworkPolicy per PyTorchJob that:
1. Allows ingress traffic between pods belonging to the same job (matched by `training.kubeflow.org/job-name` label)
2. Allows ingress from `openshift-monitoring` namespace on port 8080/TCP for Prometheus scraping

## Kustomize Deployment

The RHOAI overlay is located at `manifests/rhoai/kustomization.yaml` and is applied by the rhods-operator.

**Key overlay configuration**:
- **Namespace**: `opendatahub`
- **Name prefix**: `kubeflow-` (all resources prefixed)
- **Image injection**: `odh-training-operator-controller-image` parameter from `params.env` replaces the deployment container image AND the PyTorch init container image argument
- **Webhook cert secret**: Renamed to `kubeflow-training-operator-webhook-cert`
- **Monitoring**: PodMonitor added for Prometheus metrics scraping
- **Service patch**: Monitoring port (8080) removed from Service; metrics exposed only via direct pod scraping
- **Additional RBAC**: `training-edit` and `training-view` aggregated ClusterRoles for user access
