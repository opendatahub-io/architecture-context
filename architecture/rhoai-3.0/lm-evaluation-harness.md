## Purpose
**Short**: A framework for evaluating language models against standardized benchmarks, packaged as a batch job container for RHOAI's TrustyAI LMEval Service.

**Detailed**: lm-evaluation-harness (originally by EleutherAI) is a comprehensive framework for evaluating language models across 156+ evaluation task families and 54+ metric types. It supports multiple model backends including HuggingFace Transformers, vLLM, OpenAI-compatible APIs, IBM watsonx.ai, Anthropic, SGLang, GGUF, and others.

In the RHOAI context, this repository is packaged as the `odh-ta-lmes-job-rhel9` container image, which serves as the execution environment for the TrustyAI LMEval Service (LMES). The container is designed for one-shot, batch-oriented evaluation workloads: it receives model and task configuration, runs evaluations using lm-evaluation-harness, and outputs structured results. It includes offline metric support via bundled metrics (in the `metrics/` directory), Unitxt framework integration for custom evaluation tasks, and support for disconnected/air-gapped environments.

The container is built via Konflux CI/CD pipelines with multi-architecture support (amd64, arm64, ppc64le, s390x), with specialized build stages for PyTorch and Apache Arrow compilation on s390x and ppc64le architectures. It runs as a non-root user (UID 65532) for security compliance.
