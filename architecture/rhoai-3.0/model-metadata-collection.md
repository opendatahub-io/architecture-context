## Purpose
**Short**: A Go-based build-time tool that extracts, enriches, and catalogs metadata from Red Hat AI model container images, producing static YAML catalog files shipped in a minimal container image.

**Detailed**: Model Metadata Collection is a build-time data processing pipeline, not a runtime service. It discovers AI models from HuggingFace collections, inspects OCI container images (ModelCar format) in container registries to extract model cards, enriches metadata with information from the HuggingFace API, and produces comprehensive YAML catalogs of model metadata. The output catalogs include model names, providers, descriptions, licenses, tasks, languages, tags, and OCI artifact references with timestamps.

The tool follows a strict data priority hierarchy: (1) data extracted from `modelcard.md` files embedded in container image layers takes highest priority, (2) HuggingFace YAML frontmatter and API data serve as secondary/enrichment sources, (3) registry metadata provides artifact-level details, and (4) generated defaults fill remaining gaps. The final output is a `models-catalog.yaml` and `validated-models-catalog.yaml` files that are committed to the repository by CI/CD and then packaged into a minimal `ubi9-micro` container image via Konflux.

This container image is consumed by other RHOAI components (such as the Model Catalog UI) as an init container or volume mount to provide the model catalog data at runtime. The Go binaries themselves (`model-extractor` and `metadata-report`) are build-time tools only and are NOT included in the shipped container image.
