## Purpose
**Short**: A platform for exposing LLM inference services with tiered API key management, rate limiting, token-based billing, and policy enforcement via Gateway API and Kuadrant.

**Detailed**: Models as a Service (MaaS) provides a comprehensive platform for serving Large Language Models (LLMs) with real-time policy management on OpenShift. The core component (`maas-api`) is a Go HTTP service built with the Gin framework that manages API token issuance, model discovery via KServe `InferenceService` and `LLMInferenceService` CRDs, and tier-based access control. It creates Kubernetes ServiceAccounts and namespaces per user/tier and issues short-lived SA tokens that are validated by Kuadrant's Authorino at the Gateway level.

The platform uses OpenShift's native Gateway API implementation with a `maas-default-gateway` Gateway resource in `openshift-ingress` namespace. Kuadrant policies (AuthPolicy, RateLimitPolicy, TokenRateLimitPolicy) are applied at the Gateway level to enforce authentication via Kubernetes TokenReview, tier-based authorization via SubjectAccessReview against `llminferenceservices` resources, and tiered rate limiting (free/premium/enterprise). Traffic routing is handled by HTTPRoute resources that direct requests to the maas-api service and model inference backends.

Additionally, the repository includes development/demo applications: a React frontend for policy management and monitoring, a Node.js/Express backend-for-frontend (BFF) that proxies cluster interactions, and a QoS Prioritizer service for tier-based request queuing. These are development tools, not production-deployed components. Only the `maas-api` Go service is built and shipped via Konflux for RHOAI.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| maas-api | Go HTTP Service (Gin framework) | Core API for token issuance/revocation, model discovery, and tier lookup. Dynamically creates Namespaces and ServiceAccounts per user/tier. |
| maas-default-gateway | Gateway API Gateway | Primary ingress gateway in `openshift-ingress` namespace. Handles HTTPS termination and routes traffic to maas-api and model backends. |
| gateway-auth-policy | Kuadrant AuthPolicy | Gateway-level authentication (Kubernetes TokenReview) and authorization (SubjectAccessReview for llminferenceservices). Enriches identity with tier metadata via maas-api callback. |
| gateway-rate-limits | Kuadrant RateLimitPolicy | Tiered request rate limiting: free (5/2m), premium (20/2m), enterprise (50/2m) per user. |
| gateway-token-rate-limits | Kuadrant TokenRateLimitPolicy | Tiered token-based rate limiting: free (100/1m), premium (50000/1m), enterprise (100000/1m) per user. |
| tier-to-group-mapping | ConfigMap | Maps OpenShift user groups to subscription tiers (free/premium/enterprise) with precedence levels. |
| openshift-ai-inference | Gateway API Gateway | Default inference gateway for LLMInferenceService resources (HTTP on port 80, all namespaces). |
| frontend (dev) | React/TypeScript App | Development UI for policy management, metrics dashboard, QoS monitoring, and request simulation. |
| backend BFF (dev) | Node.js/Express/TypeScript App | Development backend-for-frontend proxying cluster API calls, metrics, policies, tokens, and QoS service. |
| qos-prioritizer (dev) | Node.js/TypeScript App | Development QoS service with Socket.IO for tier-based request queuing and prioritization. |
