## Purpose
**Short**: Build configuration repository that defines the OLM operator bundle, file-based catalog (FBC), and image references for all RHOAI 3.0 shipped container images.

**Detailed**: RHOAI-Build-Config is the central build configuration repository for Red Hat OpenShift AI (RHOAI) version 3.0. It does not contain operator source code or controller logic; instead, it defines the artifacts needed to package and distribute the RHOAI operator via the Operator Lifecycle Manager (OLM). This includes the ClusterServiceVersion (CSV), Custom Resource Definitions (CRDs), RBAC manifests, and all related container image references that constitute the RHOAI 3.0 product.

The repository is built entirely through Konflux (Red Hat's CI/CD platform based on Tekton), producing two primary artifacts: (1) the `odh-operator-bundle` - an OLM bundle image containing the CSV, CRDs, and metadata for the rhods-operator, and (2) the `rhoai-fbc-fragment` - a File-Based Catalog fragment for each supported OCP version (v4.19, v4.20) that allows the operator to be discovered and installed via OLM. GitHub Actions workflows automate the processing of bundle patches and image updates, while Tekton PipelineRuns handle the actual container image builds. The repository tracks 73 shipped container images via RELATED_IMAGE environment variables, enabling disconnected/air-gapped installations.
