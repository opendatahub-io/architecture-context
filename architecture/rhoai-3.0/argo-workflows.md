## Purpose

**Short**: Argo Workflows is a Kubernetes-native workflow engine that orchestrates parallel jobs as DAGs or steps, used as the execution backend for Data Science Pipelines in RHOAI.

**Detailed**: Argo Workflows provides a container-native workflow engine for Kubernetes, enabling users to define and execute multi-step workflows where each step runs as a container. It supports DAG-based and step-based workflow definitions through Custom Resource Definitions (CRDs), enabling complex pipeline orchestrations including parallel execution, conditional logic, artifact passing, and retry logic.

In the RHOAI distribution, Argo Workflows serves as the core execution engine for Data Science Pipelines (DSP). The workflow controller watches for Workflow CRs and orchestrates pod creation to execute each workflow step. The argoexec executor runs as a sidecar/init container within workflow pods to manage artifact collection, output capture, and container lifecycle. The Argo Server provides both a gRPC and REST API (via grpc-gateway) as well as a web UI for workflow management and monitoring.

The RHOAI build uses Konflux pipelines with FIPS-compliant builds (`GOEXPERIMENT=strictfipsruntime`) and produces two container images: the workflow controller and argoexec. The argoexec image includes both a FIPS-enabled binary (`argoexec-fips`) and a standard statically-linked binary (`argoexec`), built on UBI 9 base images.
