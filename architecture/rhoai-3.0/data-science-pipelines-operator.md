## Purpose
**Short**: Manages the lifecycle of Data Science Pipelines (DSP) application stacks on OpenShift, deploying and reconciling all components needed for ML pipeline orchestration.

**Detailed**: The Data Science Pipelines Operator (DSPO) is a Kubernetes operator that watches `DataSciencePipelinesApplication` (DSPA) custom resources and reconciles the full stack of components required for running Kubeflow Pipelines v2 on OpenShift. For each DSPA CR, the operator deploys an API server, persistence agent, scheduled workflow controller, Argo workflow controller, ML Metadata (MLMD) gRPC server with Envoy proxy, and optionally a MariaDB database and MinIO object store.

The operator uses Go templates (manifestival) to generate Kubernetes manifests dynamically during reconciliation, injecting DSPA-specific configuration such as database credentials, object storage endpoints, TLS certificates, and image references. It performs health checks against both the database and object storage backends before deploying dependent components, and publishes Prometheus metrics reflecting the readiness state of each sub-component.

DSPO supports pod-to-pod TLS (enabled by default) using OpenShift service-ca signed certificates, custom CA bundles for external storage/database connections, FIPS-compliant builds, and kube-rbac-proxy sidecars for API authentication via Kubernetes RBAC SubjectAccessReview.

**Note**: DSPO uses OpenShift Routes with `Reencrypt` TLS termination for API Server and MLMD Envoy external access. The kube-rbac-proxy sidecar on port 8443 handles authentication via Kubernetes SubjectAccessReview, enforcing RBAC on the `datasciencepipelinesapplications/api` subresource. Routes are conditionally created based on `spec.apiServer.enableOauth` and `spec.mlmd.envoy.deployRoute` settings in the DSPA CR.
