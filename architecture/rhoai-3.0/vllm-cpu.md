## Purpose
**Short**: CPU-only build of vLLM, a high-throughput and memory-efficient inference and serving engine for Large Language Models (LLMs).

**Detailed**: vLLM CPU is the CPU-optimized variant of the vLLM inference engine, packaged as a container image for Red Hat OpenShift AI (RHOAI). It provides an OpenAI-compatible REST API server for serving LLM inference requests without requiring GPU hardware, making it suitable for environments where GPU resources are unavailable or cost-prohibitive.

The container image ships in two build stages/targets: `vllm-openai` (the primary OpenAI-compatible HTTP API server) and `vllm-grpc-adapter` (which adds the vllm-tgis-adapter for gRPC compatibility with the TGIS protocol used by KServe/Caikit). The gRPC adapter stage is the final image target used for RHOAI deployments, providing both HTTP (port 8000) and gRPC (port 8033) serving endpoints. The image runs as a non-root user (UID 2000) for OpenShift compatibility.

This repository is a downstream fork of the upstream vllm-project/vllm, with Red Hat-specific modifications including UBI-based container images, multi-architecture build support (s390x, ppc64le), architecture-specific build scripts, and integration with the vllm-tgis-adapter for KServe compatibility. The build is managed entirely through Konflux CI/CD pipelines.
