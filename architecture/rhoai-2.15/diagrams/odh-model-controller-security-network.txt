DEPLOYMENT MODE ROUTING
═══════════════════════════════════════════════════════════════════════════

Mode 1: ModelMesh (Multi-model serving)
├─ Resources Created: OpenShift Routes, ServiceAccounts, ClusterRoleBindings
├─ Service Mesh: Not used (ModelMesh handles internal routing)
├─ Exposure: External via OpenShift Route (443/TCP HTTPS Edge TLS)
└─ Use Case: CPU-optimized multi-model serving

Mode 2: KServe Serverless (Knative-based autoscaling)
├─ Resources Created: Routes, VirtualServices, Gateways, PeerAuthentications,
│                      ServiceMonitors, AuthConfigs, NetworkPolicies,
│                      ServiceMeshMembers, Telemetries
├─ Service Mesh: Full integration (mTLS STRICT)
├─ Exposure: External via OpenShift Route + Internal via VirtualService
├─ Webhook: Validates Knative Services for mesh compatibility
└─ Use Case: GPU-accelerated LLM inference with autoscaling

Mode 3: KServe Raw (Direct Kubernetes deployment)
├─ Resources Created: OpenShift Routes (minimal)
├─ Service Mesh: Not used
├─ Exposure: External via OpenShift Route (443/TCP HTTPS Edge TLS)
└─ Use Case: Simple deployments without Knative or service mesh
