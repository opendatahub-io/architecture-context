Network Policies:

- Ingress: External → Istio Gateway (443/TCP)
- Ingress: Istio Gateway → Model Registry Service (8080/9090 via mTLS)
- Egress: Model Registry → Database (5432/3306 TCP)
- Egress: Operator → Kubernetes API (443/TCP)
- Egress: Authorino → Kubernetes API (443/TCP for TokenReview/SubjectAccessReview)

Authentication & Authorization:
1. External Client → Model Registry (with Istio):
   - Client presents Bearer Token (JWT) in Authorization header
   - Istio Gateway forwards to Authorino via ext-authz (gRPC/50051)
   - Authorino validates token via Kubernetes TokenReview API
   - Authorino checks GET permission on service/{registry-name} via SubjectAccessReview
   - If authorized, request forwarded to Model Registry Service via mTLS
   - Without valid token or permissions: 401 Unauthorized or 403 Forbidden

2. External Client → Model Registry (without Istio):
   - No authentication (plaintext HTTP on internal service)
   - Network policies provide only isolation

3. Operator → Kubernetes API:
   - Service Account Token (controller-manager)
   - RBAC: manager-role ClusterRole with full CRUD on managed resources

4. Admission Webhooks:
   - mTLS (Kubernetes API server validates webhook certificate)
   - API server presents client certificate

5. Metrics Endpoint:
   - mTLS client certificate
   - RBAC: Service account must have metrics access

Database Connection Security:
- PostgreSQL:
  - SSL Mode: disable (no encryption), require (TLS without verification),
             verify-ca (verify CA cert), verify-full (verify CA + hostname)
  - Client Cert: Optional (tls.crt, tls.key from {ssl-cert-secret})
  - Password: From {db-password-secret}

- MySQL:
  - SSL: true/false (enable/disable TLS)
  - Client Cert: Optional (tls.crt, tls.key, ca.crt from {ssl-cert-secret})
  - Password: From {db-password-secret}

Trust Boundaries:
1. External (Untrusted): External clients, internet
2. Ingress (DMZ): OpenShift Router, Istio Ingress Gateway
3. Service Mesh (Trusted): Model Registry pods with mTLS
4. Operator Control Plane: Operator pods managing resources
5. External Services: PostgreSQL/MySQL databases
