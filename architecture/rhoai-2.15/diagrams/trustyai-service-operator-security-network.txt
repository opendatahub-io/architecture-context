TRUST BOUNDARIES
═══════════════════════════════════════════════════════════════════════════

1. EXTERNAL → INGRESS
   - Trust: Untrusted users → DMZ
   - Protection: TLS termination, OAuth authentication

2. INGRESS → APPLICATION
   - Trust: Authenticated users → Internal services
   - Protection: TLS reencryption, OAuth proxy SAR checks

3. APPLICATION → OPERATOR
   - Trust: Managed services → Control plane
   - Protection: ServiceAccount RBAC, Kubernetes API auth

4. APPLICATION → SERVICE MESH
   - Trust: Services in mesh → Services in mesh
   - Protection: mTLS (STRICT), service identity

5. APPLICATION → STORAGE
   - Trust: Application pods → Storage backends
   - Protection: TLS (database), filesystem ACLs (PVC)

SECURITY CONTROLS
═══════════════════════════════════════════════════════════════════════════

Authentication:
  - External users: OpenShift OAuth (Bearer tokens)
  - Service-to-service: ServiceAccount tokens
  - Database: Username/Password + optional TLS client certificates
  - Metrics: Bearer Token (Prometheus ServiceAccount)

Authorization:
  - OAuth Proxy: Subject Access Review (SAR)
    - Check: namespace:{ns}, resource:pods, verb:get
  - RBAC: ClusterRole/RoleBinding for operator
  - Database: User privileges (managed externally)

Encryption:
  - In-transit (external): TLS 1.2+ (OpenShift Router → OAuth Proxy)
  - In-transit (internal): HTTP plaintext (localhost OAuth → TrustyAI)
  - In-transit (mesh): mTLS (if Istio enabled)
  - In-transit (database): TLS 1.2+ (optional, configurable)
  - At-rest: Storage class dependent (PVC), database encryption (external)

Certificate Management:
  - OAuth Proxy TLS: OpenShift service-serving-cert (auto-rotation)
  - Internal TLS: OpenShift service-serving-cert (auto-rotation)
  - Database TLS: User-provided CA certificate

Security Context (Operator Pod):
  - Run as non-root: true
  - Seccomp profile: RuntimeDefault
  - Capabilities: Drop ALL
  - Allow privilege escalation: false

Security Context (TrustyAI Service Pod):
  - OAuth Proxy: Standard security context
  - TrustyAI Container: Standard security context

Network Policies:
  - Not explicitly defined (relies on OpenShift/Kubernetes network policies)
  - Service mesh policies via Istio (if enabled)
