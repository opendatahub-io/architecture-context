TRUST BOUNDARIES:

1. EXTERNAL → KUBERNETES API
   - TLS 1.2+ encryption
   - Bearer token authentication
   - RBAC authorization

2. KUBERNETES API → CONTROLLER
   - ServiceAccount token authentication
   - TLS 1.2+ encryption
   - RBAC controls resource access

3. CONTROLLER → DATA PLANE
   - Creates resources via Kubernetes API
   - No direct network connection
   - RBAC enforces least privilege

4. INFERENCE CLIENT → RUNTIME PODS
   - Optional mTLS via Service Mesh
   - Optional token-based auth via Istio
   - Network Policies restrict ingress

5. RUNTIME PODS → EXTERNAL SERVICES
   - etcd: Optional TLS + client certificates
   - S3: TLS + AWS SigV4 or access keys
   - All egress controlled by Network Policies

SECURITY NOTES:

1. Webhook Security:
   - Uses mTLS client certificate validation
   - Kubernetes API Server validates webhook cert
   - cert-manager automates certificate rotation (90 days)

2. Metrics Security:
   - Controller metrics (8443) protected by kube-rbac-proxy
   - Requires tokenreviews/subjectaccessreviews RBAC permissions
   - ModelMesh metrics (2112) are unprotected (internal only)

3. Inference Security:
   - Default: No authentication/encryption
   - Optional: mTLS via Service Mesh (PeerAuthentication STRICT)
   - Optional: Token-based auth via Istio AuthorizationPolicy
   - Optional: TLS termination via modelmesh-serving-tls secret

4. Storage Security:
   - S3: AWS SigV4 or access key authentication
   - TLS encryption configurable (recommended for production)
   - Credentials stored in storage-config secret

5. etcd Security:
   - Optional TLS encryption
   - Optional client certificate authentication
   - Connection config in model-serving-etcd secret
   - Recommended: TLS + client certs for production

6. Controller High Availability:
   - Leader election via ConfigMaps or Leases
   - Multiple replicas supported (set replicas > 1)
   - Leader election prevents split-brain scenarios

7. Network Isolation:
   - Network Policies restrict ingress to required ports
   - Service Mesh provides additional layer of mTLS encryption
   - Runtime pods isolated per namespace
