Service Mesh Configuration:
═══════════════════════════════════════════════════════════════════════════

Ray Cluster mTLS:
- Enabled by default (configurable)
- CA certificate generated per RayCluster
- Client certificates validated for Ray client API (port 10001)
- Ray head and worker pods communicate via mTLS

OAuth Proxy (OpenShift):
- Automatic integration when RayCluster created on OpenShift
- OAuth sidecar container injected into Ray head pod
- Service CA provides TLS certificate for OAuth service
- Delegates authentication to OpenShift OAuth server
- Dashboard access (port 8265) protected by OAuth

Network Isolation:
- NetworkPolicies created per RayCluster
- Head pods: Allow from cluster pods (all), same namespace (8265, 10001), KubeRay namespaces
- Worker pods: Allow only from same cluster pods
- Egress not restricted (relies on cluster-level policies)
