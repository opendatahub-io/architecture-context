Secrets:

- ds-pipelines-proxy-tls-{name} (kubernetes.io/tls)
  • Purpose: OAuth proxy TLS certificate
  • Provisioned by: OpenShift service-ca operator
  • Auto-rotate: Yes (service-ca handles rotation)

- ds-pipelines-mariadb-tls-{name} (kubernetes.io/tls)
  • Purpose: MariaDB TLS certificate (when podToPodTLS=true)
  • Provisioned by: OpenShift service-ca operator
  • Auto-rotate: Yes

- ds-pipelines-envoy-proxy-tls-{name} (kubernetes.io/tls)
  • Purpose: MLMD Envoy proxy TLS certificate
  • Provisioned by: OpenShift service-ca operator
  • Auto-rotate: Yes

- mariadb-{name} (Opaque)
  • Purpose: MariaDB root password
  • Provisioned by: DSPO controller (auto-generated)
  • Auto-rotate: No (manual rotation required)

- minio-{name} (Opaque)
  • Purpose: Minio S3 access/secret keys
  • Provisioned by: DSPO controller (auto-generated) or user-provided
  • Auto-rotate: No

- {user-provided} (Opaque)
  • Purpose: External database credentials or external S3 credentials
  • Provisioned by: User
  • Auto-rotate: No (user managed)

Service Mesh Configuration (Optional):

- All inter-component communication uses TLS 1.3
- Service CA certificates auto-provisioned
- MariaDB, API Server gRPC, and MLMD gRPC encrypted
- Compatible with Istio Service Mesh for additional mTLS
