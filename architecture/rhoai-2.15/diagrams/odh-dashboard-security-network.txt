Authentication & Authorization Flow:
  1. User → Route (443/TCP HTTPS) - No auth required initially
  2. Route → oauth-proxy (8443/TCP HTTPS service cert) - OAuth validation
  3. oauth-proxy → OpenShift OAuth Server (443/TCP HTTPS) - OAuth flow
  4. OAuth Server validates user → Returns token
  5. oauth-proxy sets cookie → Proxies to backend (8080/TCP HTTP, Bearer token)
  6. Backend → Kubernetes API (443/TCP HTTPS, ServiceAccount token)
  7. Kubernetes API validates ServiceAccount → RBAC check → Response

Special Cases:
  - /metrics endpoint: Auth skipped via skip-auth-regex, no OAuth validation
  - /api/health endpoint: No authentication required at backend
  - Internal port 8080: Backend uses ServiceAccount token for K8s API operations

Network Policies:
  - Default: Allow all egress from dashboard namespace
  - Ingress: Restricted to OpenShift Router and Prometheus scraping

    - Readiness: HTTP GET /api/health on 8080, /oauth/healthz on 8443
