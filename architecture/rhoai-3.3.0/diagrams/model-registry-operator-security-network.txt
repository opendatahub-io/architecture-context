Authentication & Authorization Mechanisms:
==========================================

1. Model Registry REST API (8443/TCP - Secured):
   - Enforcement: kube-rbac-proxy sidecar
   - Mechanism: Bearer Token (Kubernetes user or ServiceAccount)
   - Process:
     a) Client presents Bearer token in Authorization header
     b) Proxy performs SubjectAccessReview against Service resource
     c) If authorized, request forwarded to REST API on 8080/TCP
     d) If denied, 403 Forbidden returned

2. Model Registry REST API (8080/TCP - Internal):
   - Enforcement: None
   - Mechanism: No authentication
   - Access: Internal cluster pods only (ClusterIP)

3. Admission Webhooks (9443/TCP):
   - Enforcement: Kubernetes API Server
   - Mechanism: mTLS client certificate
   - Process: API server validates webhook CA certificate

4. PostgreSQL Database:
   - Enforcement: PostgreSQL server
   - Mechanism: Password authentication or client certificate
   - Credentials: Stored in Kubernetes Secret

NetworkPolicies:

  - Ingress: Allow only from model-registry pods (label selector)
  - Egress: Default allow
  - Purpose: Isolate database from unauthorized pod access
