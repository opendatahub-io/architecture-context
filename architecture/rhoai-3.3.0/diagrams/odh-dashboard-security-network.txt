SECURITY CONTROLS
═══════════════════════════════════════════════════════════════════════════════════════════

Container Security:
  - Run as non-root: UID 1001, GID 0
  - Capabilities dropped: ALL (modular architecture containers)
  - Privilege escalation: Disabled (allowPrivilegeEscalation: false)
  - Read-only root filesystem: No (logs written to /usr/src/app/logs/)

TLS Configuration:
  - Minimum version: TLS 1.2
  - Preferred version: TLS 1.3
  - Cipher suites: Modern/secure ciphers only (OpenShift defaults)
  - HSTS enabled: max-age=31536000;includeSubDomains;preload

Authentication Mechanisms:
  1. User Authentication: OpenShift OAuth (Bearer tokens)
  2. ServiceAccount Authentication: Kubernetes ServiceAccount tokens
  3. Token Validation: TokenReview API
  4. Authorization: SubjectAccessReview + RBAC

Attack Mitigations:
  - CSRF: OAuth token requirements, SameSite cookie policies
  - XSS: DOMPurify sanitization, Content Security Policy headers
  - MITM: TLS everywhere (reencrypt route, service-serving-cert)
  - Authentication Bypass: kube-rbac-proxy OAuth enforcement
  - Privilege Escalation: Least-privilege RBAC, user impersonation
