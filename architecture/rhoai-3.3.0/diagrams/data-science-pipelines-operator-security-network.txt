NOTES
════════════════════════════════════════════════════════════════════════════════════

Deployment Isolation:
  - Each DataSciencePipelinesApplication (DSPA) CR creates fully isolated infrastructure
  - All components are namespace-scoped (controllers, storage, DB)
  - Multiple DSPA instances can coexist in different namespaces

Optional Components:
  - MariaDB: Optional for dev/test; use external MySQL for production
  - Minio: Optional for dev/test; use external S3 for production
  - ML Metadata: Optional artifact lineage tracking system

mTLS Support:
  - Optional pod-to-pod mTLS between components
  - Configurable via DSPA spec
  - Uses service-cert for automatic certificate management

FIPS Compliance:
  - FIPS-enabled builds available via GOEXPERIMENT=strictfipsruntime

Metrics:
  - data_science_pipelines_application_apiserver_ready
  - data_science_pipelines_application_persistenceagent_ready
  - data_science_pipelines_application_scheduledworkflow_ready
  - data_science_pipelines_application_ready
