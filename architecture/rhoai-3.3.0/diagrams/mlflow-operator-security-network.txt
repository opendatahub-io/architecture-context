SECRETS:

1. mlflow-tls (kubernetes.io/tls)
   - Purpose: TLS certificate for MLflow server HTTPS endpoint
   - Provisioned by: service-ca-operator (OpenShift automatic)
   - Auto-Rotate: Yes (service-ca-operator handles rotation)
   - Used by: MLflow server for :8443 HTTPS endpoint

2. mlflow-db-credentials (Opaque) - OPTIONAL
   - Purpose: Database connection URIs with embedded credentials
   - Keys: BACKEND_STORE_URI, REGISTRY_STORE_URI
   - Format: postgresql://user:pass@host:5432/mlflow
   - Provisioned by: User (manual creation)
   - Auto-Rotate: No (user-managed)

3. aws-credentials (Opaque) - OPTIONAL
   - Purpose: AWS access keys for S3 artifact storage
   - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY
   - Provisioned by: User (manual creation)
   - Auto-Rotate: No (user-managed)

AUTHENTICATION & AUTHORIZATION:

Endpoint: /metrics (Operator)
- Method: GET
- Port: 8443/TCP
- Auth: Bearer Token (ServiceAccount)
- Enforcement: kube-rbac-proxy
- Policy: ServiceAccount must have metrics reader role

Endpoint: /api/2.0/* (MLflow Server)
- Methods: ALL (GET, POST, PUT, DELETE, PATCH)
- Port: 8443/TCP
- Auth: Bearer Token (kubernetes-auth mode)
- Enforcement: MLflow server (self_subject_access_review)
- Policy: Token must be valid Kubernetes ServiceAccount token

Endpoint: HTTPRoute Traffic (Gateway)
- Methods: ALL
- Port: 443/TCP (external) → 8443/TCP (internal)
- Auth: Passthrough (Bearer Token forwarded to MLflow)
- Enforcement: Gateway handles external TLS termination, forwards HTTPS to MLflow
- Policy: Authentication enforced by MLflow server, not gateway

SECURITY FEATURES:

1. TLS Everywhere: Operator metrics (8443/HTTPS TLS1.2+), MLflow server (8443/HTTPS TLS1.3)
2. Non-Root Containers: All containers run as UID 1001 (non-root)
3. Read-Only Root Filesystem: Operator and MLflow containers use read-only rootfs
4. No Privilege Escalation: allowPrivilegeEscalation: false
5. Capabilities Dropped: ALL Linux capabilities dropped
6. Seccomp Profile: RuntimeDefault seccomp profile enforced
7. Network Policies: Ingress restricted to TCP/8443 only
8. Kubernetes-Native Auth: Bearer token authentication with RBAC (self_subject_access_review)
9. Secret References: Database credentials and AWS keys stored in Kubernetes secrets
10. Service CA Integration: Automatic TLS cert rotation via service-ca-operator (OpenShift)
