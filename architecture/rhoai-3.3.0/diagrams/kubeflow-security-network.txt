CROSS-NAMESPACE ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════════════

Controller Namespace: redhat-ods-applications
  Resources Created:
    - HTTPRoute: nb-{user-namespace}-{notebook-name}
    - ReferenceGrant: Permits HTTPRoute → Service cross-namespace references

User Namespace: {user-namespace}
  Resources Created:
    - Service: {notebook-name} (Kubeflow Notebook Controller)
    - Service: {notebook-name}-kube-rbac-proxy (ODH Notebook Controller)
    - NetworkPolicy: {notebook-name}-ctrl-np
    - NetworkPolicy: {notebook-name}-kube-rbac-proxy-np
    - ServiceAccount: {notebook-name}
    - Secret: {notebook-name}-tls-proxy-serving-cert
    - Secret: {notebook-name}-{dspa-name}-token (if DSPA integration enabled)
    - RoleBinding: {notebook-name}-sa-rb (if DSPA integration enabled)

Gateway Namespace: openshift-ingress
  Resources Referenced:
    - Gateway: data-science-gateway

Naming Convention:
  - HTTPRoutes: nb-{user-namespace}-{notebook-name} (prevents conflicts)
  - Services: {notebook-name}, {notebook-name}-kube-rbac-proxy
  - NetworkPolicies: {notebook-name}-ctrl-np, {notebook-name}-kube-rbac-proxy-np

Security Isolation:
  - HTTPRoutes in controller namespace (prevents user manipulation)
  - ReferenceGrants permit cross-namespace service access
  - NetworkPolicies restrict traffic to controller namespace only
