AUTHENTICATION & AUTHORIZATION
═══════════════════════════════════════════════════════════════════════════

Endpoint: /get-online-features
  Auth Mechanisms:
    - Bearer Token (JWT): OIDC token validation against configured provider
    - mTLS: Client certificate validation (X.509)
    - Kubernetes RBAC: SubjectAccessReview for FeatureStore resource
  Enforcement Point: Online Feature Server (Python FastAPI)

Endpoint: /get-historical-features
  Auth Mechanisms:
    - Bearer Token (JWT): OIDC token validation
    - mTLS: Client certificate validation
  Enforcement Point: Offline Feature Server

Endpoint: /registry/* (REST API)
  Auth Mechanisms:
    - Bearer Token (JWT): OIDC token validation
    - Kubernetes RBAC: SubjectAccessReview for FeatureStore resource
  Enforcement Point: Registry Server

gRPC Services: ServingService, RegistryServer
  Auth Mechanisms:
    - Bearer Token (JWT): Via gRPC metadata headers
    - mTLS: Client certificate validation
  Enforcement Point: Feature/Registry Servers

Operator Metrics Endpoint: /metrics (8443/TCP)
  Auth Mechanism: Kubernetes TokenReview API
  Enforcement Point: Operator kube-rbac-proxy sidecar

NETWORK POLICIES (Recommended)
═══════════════════════════════════════════════════════════════════════════

Ingress Rules:
  - Allow from OpenShift Routers to Feature Server Pods (6566, 6567, 8815, 8816, 6570-6573, 8888, 8443)
  - Allow from Prometheus namespace to Feature Server Pods (8000/TCP)
  - Allow from same namespace to all services (internal communication)

Egress Rules:
  - Allow to PostgreSQL: 5432/TCP (if configured)
  - Allow to Redis: 6379/TCP (if configured)
  - Allow to S3: 443/TCP (if configured)
  - Allow to OIDC Provider: 443/TCP (if configured)
  - Allow to Kubernetes API: 6443/TCP (for operator)
  - Deny all other egress by default
