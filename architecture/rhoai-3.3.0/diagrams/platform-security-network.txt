Trust Zones:

1. EXTERNAL (Untrusted):
   - External clients, data scientists
   - Public internet endpoints
   - Encryption: TLS 1.2+ required
   - Auth: OpenShift OAuth

2. INGRESS (DMZ):
   - OpenShift Router, Gateway API, Istio Gateway
   - TLS termination points
   - OAuth/Bearer token validation
   - Rate limiting, DDoS protection (OpenShift Router)

3. APPLICATION (Trusted):
   - Platform components (operators, services)
   - User workloads (notebooks, inference, training)
   - Service mesh (optional mTLS)
   - Kubernetes RBAC enforcement

4. EXTERNAL SERVICES:
   - S3, PostgreSQL, HuggingFace Hub, NGC API
   - Always encrypted (TLS 1.2+)
   - Credential-based authentication
   - Network egress policies recommended
