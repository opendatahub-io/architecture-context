Notes
═══════════════════════════════════════════════════════════════════════════════════════════

Security Considerations:
  1. Metrics endpoint currently uses HTTP (8080/TCP) instead of HTTPS
     - TODO: Migrate to HTTPS on port 8443 with proper TLS encryption
     - Bearer token auth provided via kube-rbac-proxy sidecar

  2. Webhook server uses TLS 1.2+ with certificates from OpenShift service-ca-operator
     - Automatic rotation handled by service-ca-operator
     - mTLS authentication from Kubernetes API server

  3. Controller requires OpenShift-specific features:
     - Routes for external access
     - Templates for NIM serving runtimes
     - service-ca-operator for webhook certificates

  4. External network dependencies:
     - NVIDIA NGC API for NIM validation (outbound HTTPS/443)
     - Model Registry API (optional, outbound HTTPS/443)
     - Kubernetes API server (inbound watches, outbound API calls)

  5. FIPS Compliance:
     - Built with strictfipsruntime build tags
     - Uses UBI9 minimal base image
     - Non-root user (UID 2000)

  6. HTTP/2 disabled to mitigate HTTP/2 Stream Cancellation vulnerability (CVE-2023-44487)
