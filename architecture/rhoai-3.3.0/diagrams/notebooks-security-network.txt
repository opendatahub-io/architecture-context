  - allowPrivilegeEscalation: false
  - capabilities: drop ALL

- No privileged containers
- Read-only root filesystem: No (Jupyter needs writable filesystem for packages)

AUTHENTICATION MODEL:

External Access:
1. User authenticates with OpenShift OAuth (handled by oauth-proxy sidecar)
2. OAuth proxy validates token with OpenShift OAuth server
3. Valid requests forwarded to Jupyter Server (HTTP/8888, no auth)
4. Session maintained with secure HTTP-only cookie

Internal Access:
- Jupyter Server authentication DISABLED (token='', password='')
- Assumes OAuth proxy handles all authentication
- Direct access to port 8888 blocked (ClusterIP service, no external route)

NETWORK POLICIES:

Egress (allowed):
- pypi.org:443 (Python package installation)
- quay.io:443, registry.redhat.io:443 (image pulls - node-level)
- mirror.openshift.com:443 (OpenShift CLI download)
- Kubernetes API: 6443 (kubectl/oc commands)
- User-configured S3/Git endpoints: 443
- DNS: 53/UDP (cluster DNS resolution)

Ingress (allowed):
- OpenShift Router → oauth-proxy:8443 (external user access)
- Prometheus → metrics endpoint (if monitoring enabled)

Egress (blocked by default):
- Other namespaces (cross-namespace traffic restricted)
- Node-level services (unless explicitly allowed)
