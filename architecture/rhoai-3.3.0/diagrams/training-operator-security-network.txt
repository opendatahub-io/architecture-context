- Users can configure NetworkPolicies for training pods if required
- No built-in network isolation for training pod communication

Security Considerations:

1. Plaintext Training Communication: Framework-to-framework communication
   (e.g., PyTorch workers on port 23456/TCP) uses plaintext protocols.

2. No Built-in Encryption: Operator does not enforce encryption on training
   data or model parameters. Users must implement at application layer.

3. No Service Mesh: Operator explicitly disables Istio sidecar injection to
   avoid webhook conflicts. Training pods have no automatic mTLS.

4. Secrets in Job Specs: Users may include secrets in job specifications
   (e.g., for S3 access). Should use Kubernetes Secrets, not plaintext.

5. Non-root Execution: Operator deployment runs as non-root (UID 65532).
   Training pod security context is user-specified.

6. CVE-2026-2353: Secrets RBAC restricted to namespace-scoped Role
   (not cluster-wide ClusterRole).
