SECURITY CONSIDERATIONS
═══════════════════════════════════════════════════════════════════════════

1. Webhook Security
   ├─ TLS certificates auto-rotated by cert-controller
   ├─ Prevents MITM during admission control
   └─ API Server validates webhook endpoint using mTLS

2. RBAC Least Privilege
   ├─ Secrets access restricted to namespace-scoped Role (CVE-2026-2353)
   ├─ ClusterRole provides necessary permissions for job management
   └─ ServiceAccount token used for API Server authentication

3. Training Pod Security
   ├─ Security context: User-specified (not enforced by operator)
   ├─ Operator deployment: runAsNonRoot, UID 65532
   └─ No default pod security policies applied to training jobs

4. Network Security
   ├─ Training communication: PLAINTEXT (framework-dependent)
   ├─ No built-in encryption for gradient synchronization
   ├─ No NetworkPolicies by default
   └─ Users must configure network isolation at cluster level if required

5. Secrets in Job Specifications
   ├─ Users may reference Kubernetes Secrets for credentials
   ├─ WARNING: Do not include secrets in plaintext in job specs
   └─ Use Kubernetes Secrets or external secret management

6. Known Vulnerabilities Addressed
   ├─ CVE-2026-2353: Secrets RBAC restricted to namespace scope
   └─ CVE-2025-61726: Go version upgraded to 1.25
