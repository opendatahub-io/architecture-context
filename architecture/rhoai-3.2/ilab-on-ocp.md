## Purpose
**Short**: InstructLab LLM fine-tuning pipeline for Red Hat OpenShift AI, orchestrating synthetic data generation, distributed multi-node GPU training, and model evaluation via Kubeflow Pipelines.

**Detailed**: ilab-on-ocp is the central repository for running the InstructLab large language model (LLM) fine-tuning workflow on OpenShift clusters with Red Hat OpenShift AI (RHOAI). It packages the full InstructLab pipeline as a Kubeflow Pipelines v2 (KFP) pipeline definition that is automatically provisioned by the DataSciencePipelinesApplication (DSPA) resource.

The pipeline orchestrates a multi-phase workflow: (1) prerequisites validation (checking model servers, Model Registry, Training Operator availability, OCI registry, and SDG parameters), (2) Synthetic Data Generation (SDG) using a teacher model via OpenAI-compatible API, (3) data processing to prepare training data, (4) two-phase distributed training using PyTorchJob resources via the Kubeflow Training Operator across multiple GPU-enabled nodes with FSDP, (5) model evaluation using MT-Bench and MMLU benchmarks with a judge model, and (6) output model upload to an OCI registry and optional registration in Model Registry.

The repository also produces a container image (`odh-ml-pipelines-runtime-generic`) built via Konflux that serves as the base runtime for pipeline tasks. This image bundles the pipeline code, Python dependencies, a Mixtral tokenizer, and the `skopeo` tool for OCI image operations. Supplementary Kubernetes manifests are included for deploying teacher (Mixtral) and judge (Prometheus) model servers via KServe InferenceService resources.
