## Purpose
**Short**: High-throughput LLM inference and serving engine optimized for Intel Gaudi (Habana) AI accelerators, providing OpenAI-compatible and gRPC APIs.

**Detailed**: vLLM Gaudi is a fork of the vLLM project adapted to run on Intel Gaudi (Habana Labs) HPU accelerators. It provides a high-throughput, memory-efficient inference and serving engine for large language models, exposing both an OpenAI-compatible REST API and a gRPC interface via the vllm-tgis-adapter. The container image is built on RHEL 9.6 UBI using the Konflux build system and ships with the Habana software stack (driver libraries, RDMA support, OpenMPI) for distributed multi-accelerator inference.

The component supports tensor parallelism and pipeline parallelism across multiple Gaudi devices using MPI-based communication (with SSH for mpi-operator v2 coordination). It includes PagedAttention for efficient KV cache management, continuous batching for maximizing throughput, and supports quantization methods (FP8, compressed-tensors, INC, AWQ, GPTQ, bitsandbytes) on HPU. The serving layer is built on FastAPI/Uvicorn with optional TLS support and Prometheus metrics export.
