## Purpose
**Short**: Sidecar container image providing model storage retrieval and runtime-specific protocol adaptation for ModelMesh Serving inference pods.

**Detailed**: The ModelMesh Runtime Adapter is a multi-purpose sidecar container that runs alongside model server containers (Triton, MLServer, OpenVINO Model Server, TorchServe) within ModelMesh Serving pods. It contains two logical categories of subcomponents: a model-serving puller and runtime-specific adapters.

The **model-serving puller** is a gRPC server that intercepts model load/unload requests from the ModelMesh controller. When a model load is requested, the puller downloads model artifacts from remote storage backends (S3, Azure Blob Storage, Google Cloud Storage, HTTP, or PVC mounts) to a local filesystem, then forwards the rewritten request to the downstream model runtime adapter. The puller library ("pullman") supports configurable storage credentials read from mounted Kubernetes Secrets.

The **runtime adapters** (Triton, MLServer, OVMS, TorchServe) implement the ModelMesh `ModelRuntime` gRPC service interface and translate model lifecycle operations (load, unload, size prediction, runtime status) into runtime-specific API calls. Each adapter connects to the colocated model server via gRPC (or gRPC + HTTP for TorchServe) and handles runtime-specific model layout adaptation, configuration file generation, and model format conversion (e.g., Keras to TensorFlow SavedModel via Python). The image is built with FIPS-compliant Go compilation flags (`strictfipsruntime`).
