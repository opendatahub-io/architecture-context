## Purpose
**Short**: KServe is the model serving controller for RHOAI that manages the lifecycle and networking of ML inference services on Kubernetes/OpenShift.

**Detailed**: KServe provides a Kubernetes-native way to deploy, scale, and manage machine learning models for inference. It introduces several Custom Resource Definitions (CRDs) including InferenceService, LLMInferenceService, InferenceGraph, ServingRuntime, and TrainedModel to declaratively define model serving workloads. The controller reconciles these CRDs into Kubernetes Deployments, Services, and networking resources (HTTPRoutes via Gateway API, or Istio VirtualServices).

In the RHOAI distribution, KServe operates primarily in "RawDeployment" mode (not Knative Serverless), creating standard Kubernetes Deployments and Services for model serving. The controller dynamically creates HTTPRoute resources (Gateway API) for ingress when `enableGatewayApi` is enabled, or uses Istio VirtualServices for traffic routing. The RHOAI overlay uses `kube-rbac-proxy` (referenced as "oauthProxy" in the config) for authentication, replacing the upstream oauth-proxy pattern. The LLMInferenceService controller provides advanced LLM-specific features including disaggregated prefill/decode, multi-node serving via LeaderWorkerSets, inference scheduling via Gateway API Inference Extensions (InferencePool/InferenceModel), and routing sidecars.

KServe also includes auxiliary components: a storage-initializer init container for downloading models from various storage backends (S3, GCS, Azure, PVC, OCI, HuggingFace), a model agent sidecar for multi-model serving and inference logging, and a router component for InferenceGraph routing.

| InferenceService Controller | Reconciler | Reconciles InferenceService CRs into predictor/transformer/explainer Deployments, Services, and ingress (HTTPRoute or VirtualService). Supports RawDeployment and Serverless modes |

| kserve-storage-initializer | Init Container | Downloads model artifacts from S3, GCS, Azure, PVC, OCI, HuggingFace to local storage before inference container starts |
| kserve-agent | Sidecar Container | Model agent for multi-model serving, inference logging, and batching |
| kserve-router | Container | InferenceGraph router that routes requests between inference services in a graph topology |
| kube-rbac-proxy | Sidecar Container | Authentication/authorization proxy for inference endpoints (RHOAI 3.x pattern) |
