## Purpose
**Short**: Kubeflow Pipelines provides end-to-end ML workflow orchestration on Kubernetes, enabling reusable, portable, and scalable machine learning pipelines.

**Detailed**: Data Science Pipelines (based on Kubeflow Pipelines) is a platform for building and deploying portable, scalable machine learning workflows. It provides an API server (v1beta1 and v2beta1) with both gRPC and REST interfaces for managing pipelines, experiments, runs, and recurring runs. The platform uses Argo Workflows as its workflow execution engine and stores pipeline metadata in a relational database (MySQL or PostgreSQL).

The system consists of several cooperating microservices: an API server that exposes pipeline management operations, a persistence agent that synchronizes workflow status back to the API server, a scheduled workflow controller that manages recurring pipeline executions, a cache server that provides execution caching via Kubernetes admission webhooks, a metadata (MLMD) gRPC server for artifact/execution tracking, and a React-based frontend UI. In the RHOAI distribution, all Go components are built with FIPS-compliant cryptography using UBI9 base images and Konflux CI/CD.

The v2 pipeline architecture introduces a driver/launcher pattern where the driver binary handles DAG orchestration and the launcher binary manages individual task execution, both injected as init/sidecar containers into pipeline step pods. Kubernetes-native pipeline storage is supported via CRDs (Pipeline and PipelineVersion) with admission webhooks for validation and mutation.
