## Purpose
**Short**: High-performance model inference server using OpenVINO for optimized AI model serving via gRPC and REST APIs.

**Detailed**: OpenVINO Model Server (OVMS) is a high-performance inference serving system implemented in C++ and optimized for deployment on Intel architectures. It hosts AI models and makes them accessible over standard network protocols — gRPC and REST — supporting multiple API specifications including KServe Predict V2, TensorFlow Serving, and OpenAI-compatible generative AI APIs (chat completions, embeddings, reranking, image generation).

OVMS supports loading models from local filesystem, AWS S3, Google Cloud Storage, Azure Blob Storage, and HuggingFace Hub. It provides advanced features including model versioning, dynamic model updates, DAG pipeline scheduling, MediaPipe graph execution, Python code execution within graphs, GPU acceleration via OpenCL, and Prometheus-compatible metrics. The server is built using Bazel and uses OpenVINO as its inference backend.

In the RHOAI context, OVMS is shipped as a container image (`odh-openvino-model-server-rhel9`) built via Konflux on a UBI 9 minimal base, and serves as a runtime for KServe InferenceService workloads. It does not include its own Kubernetes manifests — deployment is managed by the KServe/ModelMesh infrastructure.

| Service Name | Type | Port | Target Port | Protocol | Encryption | Auth | Exposure |
|--------------|------|------|-------------|----------|------------|------|----------|

| REST listener | N/A (in-process) | configurable (default 0 = disabled)/TCP | same | HTTP/1.1 | plaintext | None | Internal |

**Default port behavior**: Both gRPC (`--port`) and REST (`--rest_port`) default to 0 (disabled). The deploying infrastructure (KServe ServingRuntime) configures the actual port values. Common conventions in KServe context: gRPC on 8085/TCP, REST on 8080/TCP.
