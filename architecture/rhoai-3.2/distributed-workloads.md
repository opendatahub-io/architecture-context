## Purpose
**Short**: Provides training runtime container images and end-to-end integration tests for the RHOAI Distributed Workloads stack (Kueue, KubeRay, Kubeflow Training Operator/Trainer).

**Detailed**: The `distributed-workloads` repository is the central repository for building GPU-accelerated training runtime container images and comprehensive integration tests that validate the RHOAI Distributed Workloads stack. This stack enables users to run distributed machine learning training workloads on OpenShift AI using components such as Kueue (job queuing and resource quota management), KubeRay (Ray cluster orchestration), and the Kubeflow Training Operator / Kubeflow Trainer v2 (PyTorchJob and TrainJob orchestration).

The repository does **not** contain any operator or controller code itself. Instead, it houses: (1) Dockerfiles and build configurations for training runtime images that combine CUDA/ROCm GPU drivers, PyTorch, and ML libraries (Flash Attention, DeepSpeed, NCCL, RDMA/InfiniBand) into UBI9-based container images; (2) a Go-based integration test suite that exercises the full distributed workloads lifecycle against a running RHOAI or ODH cluster; and (3) example notebooks and scripts demonstrating distributed training patterns (LLM fine-tuning with DeepSpeed, Stable Diffusion DreamBooth, HPO with Ray Tune). The Tekton/Konflux pipelines in `.tekton/` drive the automated builds of these training images for the RHOAI product.

Additionally, the repository provides "universal" training images that combine Jupyter workbench and ML runtime capabilities in a single image, enabling users to both interactively develop and batch-execute distributed training jobs.
