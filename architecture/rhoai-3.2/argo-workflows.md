## Purpose
**Short**: Argo Workflows is a container-native workflow engine for orchestrating parallel jobs on Kubernetes, used as the execution engine for Data Science Pipelines in RHOAI.

**Detailed**: Argo Workflows implements a Kubernetes-native workflow orchestration engine that defines workflows as Custom Resources. It provides a workflow controller that watches Workflow CRs and orchestrates pod-based step execution, a CLI/API server (argo-server) exposing both gRPC and REST APIs for workflow management, and an executor (argoexec) that runs as a sidecar within workflow pods to manage artifact passing, output capture, and container lifecycle.

In the RHOAI distribution, Argo Workflows serves as the backend execution engine for Data Science Pipelines (DSP). The workflow controller manages the lifecycle of ML pipeline runs by translating pipeline DAGs into Kubernetes pod executions. The RHOAI builds use Konflux CI with UBI9-based container images, including FIPS-compliant builds (using `strictfipsruntime` Go build tags with CGO enabled). Two container images are shipped: the workflow controller and the argoexec executor. The argo-server (CLI/API server) is not separately built in the RHOAI Konflux pipelines, as DSP manages pipeline interactions through its own API layer.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| workflow-controller | Go Binary (Deployment) | Watches Workflow CRs, orchestrates pod creation for workflow steps, manages DAG execution, handles cron workflows, artifact GC, and workflow lifecycle |
| argoexec | Go Binary (Sidecar/Init Container) | Runs inside workflow pods using the emissary executor pattern; manages artifact collection, output parameter capture, script execution, and container lifecycle |
| argo-server | Go Binary (Deployment) | API server exposing gRPC + REST (via grpc-gateway) APIs on port 2746/TCP; serves the Argo Workflows UI; handles SSO authentication |
| Workflow CRDs | Custom Resource Definitions | 8 CRDs defining the workflow data model (Workflow, CronWorkflow, WorkflowTemplate, ClusterWorkflowTemplate, WorkflowTaskSet, WorkflowTaskResult, WorkflowArtifactGCTask, WorkflowEventBinding) |
| UI (React) | Frontend (embedded in argo-server) | Web-based UI for workflow visualization and management, built with React/TypeScript and bundled into the argo-server binary |
