SECURITY CONSIDERATIONS
═══════════════════════════════════════════════════════════════════════════════════════

1. FIPS Compliance:
   - Built with FIPS-enabled Go toolchain (GOEXPERIMENT=strictfipsruntime)
   - Uses Red Hat UBI9 minimal base images with FIPS-validated cryptography

2. Non-root Execution:
   - Controller runs as UID 1001 (non-root user 'rhods')
   - Compatible with OpenShift restricted SCC

3. Service Account Permissions:
   - Requires cluster-wide permissions for managing StatefulSets, Services, Notebooks
   - Optional Istio VirtualService creation requires networking.istio.io API access

4. Notebook Isolation:
   - Each notebook runs in its own StatefulSet with namespace-level isolation
   - No built-in multi-tenancy controls (relies on Kubernetes namespace isolation)

5. Culling Security Risk:
   - CullingReconciler accesses notebook /api/kernels endpoint without authentication
   - RECOMMENDATION: Deploy Istio AuthorizationPolicy in production (see docs)

6. Network Policies:
   - Not defined by controller (user responsibility)
   - Culling requires HTTP access to notebook pods on port 8888

═══════════════════════════════════════════════════════════════════════════════════════
KEY NETWORK FLOWS
═══════════════════════════════════════════════════════════════════════════════════════

1. User → Notebook Access:
   External User → Istio Gateway (443/HTTPS TLS1.2+) → VirtualService →
   Notebook Service (80/HTTP) → Notebook Pod (8888/HTTP plaintext)

2. Controller → API Server:
   Notebook Controller (manager) → Kubernetes API (443/HTTPS TLS1.2+, SA token) →
   Watch/Manage: Notebooks, StatefulSets, Services, VirtualServices

3. Culling → Notebook:
   CullingReconciler → Notebook Service (8888/HTTP plaintext, no auth) →
   GET /api/kernels → Update Notebook CR with STOP_ANNOTATION

4. Metrics Collection:
   Prometheus → kube-rbac-proxy (8443/HTTPS TLS1.2+, RBAC) →
   Manager /metrics (8080/HTTP localhost) → Metrics response

5. Notebook Creation:
   jupyter-web-app → Kubernetes API (443/HTTPS TLS1.2+) → Create Notebook CR →
   NotebookReconciler watches → Create StatefulSet, Service, VirtualService

═══════════════════════════════════════════════════════════════════════════════════════
CONFIGURATION VARIABLES (Security-Relevant)
═══════════════════════════════════════════════════════════════════════════════════════

USE_ISTIO (default: true on kubeflow, false on openshift)
├─ Controls: VirtualService creation
└─ Security Impact: Affects external exposure and mTLS enforcement

ENABLE_CULLING (default: false)
├─ Controls: CullingReconciler activation
└─ Security Impact: Enables unauthenticated HTTP access to notebook pods (8888/TCP)

ADD_FSGROUP (default: true)
├─ Controls: Automatic fsGroup:100 injection to pod security context
└─ Security Impact: Affects file permission behavior in notebook pods

DEV (default: false)
├─ Controls: Developer mode (uses kubectl proxy for local testing)
└─ Security Impact: DO NOT enable in production (bypasses network isolation)
