RBAC Summary:
- No RBAC resources are defined by vLLM CPU directly
- vLLM CPU runs as a workload container, not an operator
- RBAC is managed by KServe or the deploying operator
- ServiceAccount managed by KServe InferenceService

Authentication & Authorization:
- HTTP endpoints: API Key (Bearer header), optional, configured via --api-key or VLLM_API_KEY env
- /metrics endpoint: No authentication (always open)
- mTLS: Optional client certificate verification via --ssl-cert-reqs
- gRPC (TGIS): No authentication

Secrets:
- ssl-keyfile (File mount) - TLS private key for HTTPS, User/Operator provisioned, auto-rotate via SSLCertRefresher
- ssl-certfile (File mount) - TLS certificate for HTTPS, User/Operator provisioned, auto-rotate via SSLCertRefresher
- ssl-ca-certs (File mount) - CA certificates for client verification, User/Operator provisioned, auto-rotate via SSLCertRefresher
- hf-token (Env: HF_TOKEN) - HuggingFace Hub auth, User provisioned, no auto-rotate
- s3-credentials (Env: S3_ACCESS_KEY_ID, S3_SECRET_ACCESS_KEY) - S3 access, User provisioned, no auto-rotate
- vllm-api-key (Env: VLLM_API_KEY) - API key for endpoint auth, User provisioned, no auto-rotate

Encryption Summary:
- HTTP API (8000/TCP): Optional TLS with certificate hot-reload
- gRPC TGIS (8033/TCP): Plaintext (relies on service mesh for encryption)
- Egress to HuggingFace/S3: TLS 1.2+
- Egress to OTLP Collector: Optional TLS
