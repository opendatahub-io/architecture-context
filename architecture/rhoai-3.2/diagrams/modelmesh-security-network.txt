Network Policies:
- Policy: model-mesh
  - Pod Selector: app=model-mesh, component=model-mesh
  - Ingress Rules:
    - Port 8033/TCP, 8080/TCP from pods with matching labels
    - Port 8033/TCP from any (inference traffic)
    - Port 2112/TCP from any (metrics scraping)

TLS Configuration:
- External gRPC (8033): TLS with PKCS8 key, optional mTLS
  - Cert Source: Kubernetes Secret (model-mesh-tls-secret)
  - Config: MM_TLS_KEY_CERT_PATH, MM_TLS_PRIVATE_KEY_PATH
  - Client Auth: MM_TLS_CLIENT_AUTH=REQUIRE/OPTIONAL/NONE
- Inter-pod (8080, Litelinks): TLS (same certs as external)
  - Client Auth: MM_INTRACLUSTER_CLIENTAUTH (configurable)
- Internal runtime (localhost): plaintext (not encrypted)
- Prometheus metrics (2112): HTTPS with auto-generated self-signed cert (BouncyCastle)

FIPS Compliance:
- Java FIPS mode disabled (-Dcom.redhat.fips=false) in start.sh
- Due to compatibility issues when TLS is enabled
- Operator feature flag: fips-compliant (enabled)

Secrets:
- model-mesh-tls-secret (kubernetes.io/tls)
  - Purpose: TLS key pair for external gRPC and inter-pod communication
  - Provisioned by: Administrator / cert-manager
  - Auto-Rotate: No (manual rotation, care needed for rolling updates)
- model-mesh-icd-etcd (Opaque)
  - Purpose: etcd connection configuration (JSON-formatted)
  - Provisioned by: Administrator
  - Auto-Rotate: No

Authentication & Authorization:
- External gRPC (8033): mTLS client certificates (optional)
  - Enforcement: Netty SSL handler
  - Policy: MM_TLS_CLIENT_AUTH env var
- Model Identification: mm-model-id / mm-vmodel-id gRPC metadata header
  - Enforcement: ModelMeshApi interceptor
  - Required for inference routing
