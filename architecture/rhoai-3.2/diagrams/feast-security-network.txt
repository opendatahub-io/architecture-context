Feature Store Authorization Modes
════════════════════════════════════════════════════════════════════════════

1. No Auth (Default - Development Only)
   - Mechanism: None
   - Configuration: Default when authz not configured
   - Use case: Local development, testing

2. Kubernetes RBAC
   - Mechanism: ServiceAccount + Roles
   - Configuration: authz.kubernetes.roles in FeatureStore CR
   - Enforcement: Feature server validates K8s tokens via TokenReview
   - Use case: Production with K8s-native access control

3. OIDC (External IdP)
   - Mechanism: OAuth 2.0/OpenID Connect
   - Configuration: authz.oidc.secretRef → {name}-oidc secret
   - Secret keys: client_id, client_secret, username, password
   - Enforcement: Feature server validates JWT tokens from IdP
   - Use case: Enterprise SSO (Auth0, Keycloak, Azure AD)
