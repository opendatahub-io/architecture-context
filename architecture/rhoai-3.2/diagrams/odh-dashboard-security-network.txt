Authentication & Authorization Flow
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User → OpenShift OAuth: Authenticate with OpenShift credentials
2. OAuth → User: Issue OAuth Bearer Token (cookie)
3. User → Route: All requests include OAuth Bearer Token
4. Route → kube-rbac-proxy: Forward with token (reencrypt TLS)
5. kube-rbac-proxy: Validate token with OpenShift OAuth
6. kube-rbac-proxy → Backend: Inject X-Auth-Request-User and
                               X-Auth-Request-Groups headers (HTTP/8080)
7. Backend: Enforce resource-level RBAC based on user/groups
8. Backend → K8s API: Use ServiceAccount token for operations
9. Backend → User: Return response (proxied through kube-rbac-proxy/Route)

Public Endpoints (No Auth):
  - /api/health (8080/TCP) - Backend health check
  - /healthz (8444/TCP) - kube-rbac-proxy health check

Security Considerations
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. OAuth Integration: All user requests authenticated via OpenShift OAuth
   before reaching backend

2. TLS Encryption:
   - External traffic: TLS 1.2+ (user → Route → kube-rbac-proxy)
   - Internal traffic: Unencrypted HTTP (kube-rbac-proxy → backend, localhost)
   - K8s API: TLS 1.2+ (backend → API server)

3. RBAC Enforcement: Two-layer authorization
   - Layer 1: kube-rbac-proxy validates OAuth token
   - Layer 2: Backend enforces resource-level permissions via K8s RBAC

4. Service Account Permissions:
   - Broad cluster-level read access (users, groups, nodes, CRs)
   - Namespace-level write access for managed resources
   - Can create/delete RoleBindings (privilege escalation risk)

5. Secret Management:
   - TLS certificates auto-rotated by OpenShift
   - Trusted CA bundle managed by cluster operator
   - No manual credential management required

6. Network Isolation:
   - Ingress restricted to specific ports (8443, 8043, 8143/TCP)
   - Egress unrestricted (requires K8s API, Thanos, external services)

7. Content Security:
   - Frontend uses DOMPurify for XSS protection
   - User-generated content sanitized before rendering

8. Dependency Management:
   - Automated Renovate updates for npm dependencies
   - Base images (UBI9/Node.js 22) regularly patched
