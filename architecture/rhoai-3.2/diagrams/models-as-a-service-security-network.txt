  - core: namespaces (get, list, watch, create)
  - core: serviceaccounts (get, list, watch, create, delete)

  - gateway.networking.k8s.io: httproutes (get, list, watch)
  - core: pods, services, endpoints (get, list, watch)

- ClusterRoleBinding: maas-api
  - Binds: ClusterRole/maas-api -> SA maas-api (deployment namespace)

Authentication & Authorization:
===============================
- Gateway level: Kuadrant AuthPolicy with KubernetesTokenReview
  - Validates SA tokens with audience: maas-default-gateway-sa
  - SubjectAccessReview for model route access (post llminferenceservices)
  - Tier enrichment via callback to maas-api /v1/tiers/lookup
- maas-api level: Kuadrant AuthPolicy with KubernetesTokenReview
  - Audiences: https://kubernetes.default.svc + maas-default-gateway-sa
  - Injects X-MaaS-Username / X-MaaS-Group headers

Secrets:
========
- default-ingress-cert (kubernetes.io/tls)
  - Purpose: TLS certificate for maas-default-gateway HTTPS listener
  - Provisioned by: OpenShift default ingress
  - Auto-Rotate: Yes (OpenShift managed)

Token Management:
=================
- Ephemeral tokens: Kubernetes SA tokens via TokenRequest API
  - Min expiration: 10 minutes
  - Default expiration: 4 hours
  - Audience: maas-default-gateway-sa
- API keys: Long-lived named tokens tracked in metadata store
- Token revocation: Deletes/recreates user's ServiceAccount
- Tier namespace isolation: {instance}-tier-{tier} namespaces
