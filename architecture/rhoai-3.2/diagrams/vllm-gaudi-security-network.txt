RBAC Summary:
- No Kubernetes RBAC resources defined by this component
- Runs as non-root user (UID 2000, GID 0) for inference
- SSHD starts as root, inference process runs as vllm user
- RBAC managed by deploying infrastructure (KServe, ModelMesh)

Container Security:
- Capabilities: SYS_NICE (required for Habana device scheduling)
- IPC Namespace: host (required for shared memory between Gaudi devices)
- Network Namespace: host (recommended for RDMA inter-device communication)
- User: root → UID 2000 (vllm) after sshd startup

Secrets:
- HF_TOKEN (Opaque env var) - HuggingFace Hub auth, user-provided, no auto-rotate
- SSL key/cert files (kubernetes.io/tls) - TLS encryption, user/cert-manager, auto-rotate via SSLCertRefresher
- API Key (Opaque env var) - API authentication, user-provided, no auto-rotate

Authentication & Authorization:
- HTTP endpoints: Optional API Key via X-API-Key header (enforced by FastAPI middleware when --api-key set)
- /metrics, /health, /ping: Always accessible (no auth)
- gRPC (8033/TCP): No built-in auth (relies on network-level security)
