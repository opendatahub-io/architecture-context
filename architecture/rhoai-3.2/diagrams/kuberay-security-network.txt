Service Mesh Configuration:
============================

NetworkPolicy Support:
  - Enabled when RayCluster has annotation: ray.io/enable-secure-trusted-network: "true"

  Head Pod NetworkPolicy:
    Ingress:
      - Allow from: Ray worker pods (label: ray.io/cluster-name={cluster})
      - Allow from: KubeRay operator namespace
      - Allow from: Same namespace
    Egress:
      - Allow to: Kubernetes DNS (kube-system/kube-dns)
      - Allow to: Kubernetes API server
      - Allow to: Ray worker pods (label: ray.io/cluster-name={cluster})

  Worker Pod NetworkPolicy:
    Ingress:
      - Allow from: Ray head pod (label: ray.io/node-type=head)
      - Allow from: Other worker pods (label: ray.io/cluster-name={cluster})
    Egress:
      - Allow to: Kubernetes DNS (kube-system/kube-dns)
      - Allow to: Ray head pod (label: ray.io/node-type=head)
      - Allow to: Other worker pods (label: ray.io/cluster-name={cluster})

mTLS Configuration:
  - Enabled when RayCluster spec includes: head.enableRayAutoscaler.rayStartParams.ray-client-server-use-tls: "true"
  - cert-manager creates:
    • CA Certificate and Issuer ({raycluster}-ca-secret)
    • Ray pod certificates signed by CA ({raycluster}-tls-secret)
  - Certificates mounted into Ray head/worker pods at /etc/ray/tls/
  - Ray GCS and client API use mTLS for encryption and authentication

Network Flows:
==============

1. User → Kubernetes API → Webhook → Operator → Ray Pods (RayCluster creation)
2. User → OpenShift Route → OAuth Proxy → Ray Dashboard (authenticated access)
3. Ray Workers → Ray Head GCS (6379/TCP, optional mTLS)
4. Operator → Kubernetes API (watch CRDs, manage resources)
5. Operator → cert-manager (create Certificates and Issuers)
6. OAuth Proxy → OpenShift OAuth Server (validate tokens)
7. Ray Pods → External Redis (optional, GCS fault tolerance)
8. Ray Pods → Container Registry (pull images)
9. Prometheus → Operator metrics endpoint (8080/TCP)
