  - secrets: controller secret access

Authentication Modes:
- Client Mode: K8s ServiceAccount token impersonation (user presents SA token)
- SSO Mode: OIDC JWT validated against configured issuer
- Server Mode: Uses argo-server's own SA permissions (least restrictive)

- sso (Opaque) - SSO cookie encryption key, auto-generated by argo-server, no rotation
- argo-workflows-agent-ca-certificates (Opaque) - CA certs for agent pods, user-provisioned
- workflow-controller-configmap (ConfigMap) - Controller config (artifact repo creds, DB, SSO)

- workflow-controller: RunAsNonRoot, ReadOnlyRootFS, No PrivEsc, Drop ALL capabilities
- argoexec: RunAsNonRoot (UID 2000)
