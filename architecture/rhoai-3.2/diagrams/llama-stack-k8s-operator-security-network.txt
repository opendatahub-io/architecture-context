TLS/ENCRYPTION SUMMARY
═══════════════════════════════════════════════════════════════════════════

Operator → Kubernetes API:
  - TLS 1.3, Bearer Token (ServiceAccount)
  - Certificate validation: Yes

kube-rbac-proxy → Prometheus:
  - TLS 1.2+ (HTTPS on 8443/TCP)
  - Client auth: Bearer Token with SubjectAccessReview

Llama Stack Server → HuggingFace:
  - TLS 1.2+ (HTTPS on 443/TCP)
  - Optional HF Token for authentication

Llama Stack Server → vLLM/Ollama:
  - Typically HTTP (no encryption) for internal services
  - TLS possible for external/remote inference providers
  - Optional API key authentication

Llama Stack API (8321/TCP):
  - Default: HTTP (no encryption, no authentication)
  - Recommendation: Use external proxy/gateway for TLS and auth
  - CA Bundle support: spec.server.tlsConfig for custom certificates

Container Image Pull:
  - TLS 1.2+ (HTTPS on 443/TCP)
  - Registry credentials via Pull Secrets
