RBAC Summary:
- No RBAC resources defined by OVMS
- Runs as non-privileged container (UID 5000, GID 5000)
- No Kubernetes API access required
- All RBAC enforcement delegated to KServe infrastructure

Security Configuration:
- No built-in TLS: gRPC uses InsecureServerCredentials
- No built-in authentication on any endpoint
- Authentication enforced by infrastructure:
  - KServe: kube-rbac-proxy sidecar or Gateway API auth proxy
  - ModelMesh: infrastructure-level auth
  - Istio: mTLS between pods (when service mesh enabled)
- CORS: Configurable via --allow_credentials, --allowed_origins,
         --allowed_methods, --allowed_headers flags

Secrets:
- AWS credentials (Environment variables) - S3 access, User/KServe provisioned, No auto-rotate
- GCS credentials (Service account JSON) - GCS access, User/KServe provisioned, No auto-rotate
- Azure credentials (Environment variables) - Azure access, User/KServe provisioned, No auto-rotate
- HF_TOKEN (Environment variable) - HuggingFace auth, User provisioned, No auto-rotate

Metrics:
- Prometheus-compatible metrics at /metrics endpoint
- Exposed on REST port (plaintext HTTP)
- No authentication on metrics endpoint
