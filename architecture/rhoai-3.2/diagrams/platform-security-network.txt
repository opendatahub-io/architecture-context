AUTHENTICATION PATTERNS
════════════════════════════════════════════════════════════════════════════════════════

1. OpenShift OAuth (Bearer Token JWT):
   - Components: ODH Dashboard, Notebooks, Ray Dashboard, Routes
   - Enforcement: kube-rbac-proxy, OAuth proxy
   - Flow: User → OpenShift OAuth → JWT token → Service

2. Kubernetes ServiceAccount Tokens:
   - Components: All operators, internal service calls
   - Enforcement: Kubernetes API Server (RBAC)
   - Flow: Pod → ServiceAccount → Token → K8s API

3. mTLS Client Certificates (SPIFFE):
   - Components: KServe (Istio service mesh), Ray GCS, DSP MLMD
   - Enforcement: Istio Envoy sidecar, Ray GCS, Envoy proxy
   - Flow: Service → Istio → mTLS handshake → Target service

4. AWS IAM / S3 SigV4:
   - Components: Data Science Pipelines, Notebooks, Training, MLflow
   - Enforcement: S3-compatible storage
   - Flow: Pod → IRSA/Access Key → S3 signed request → Storage

5. Application-level RBAC (self_subject_access_review):
   - Components: MLflow, Feast, Model Registry
   - Enforcement: Application validates K8s RBAC via API call
   - Flow: User → Bearer token → App → K8s API (RBAC check) → Allow/Deny

6. Kuadrant AuthPolicy (Authorino):
   - Components: KServe InferenceServices (optional), LLMInferenceServices
   - Enforcement: Authorino (external auth service)
   - Flow: Request → Istio → Authorino (JWT validation) → Service

7. API Keys:
   - Components: NVIDIA NGC, HuggingFace Hub
   - Enforcement: External API provider
   - Flow: Request → API key header → External validation → Response
