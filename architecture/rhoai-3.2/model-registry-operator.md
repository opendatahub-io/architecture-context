## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Model Registry and Model Catalog instances on OpenShift/Kubernetes clusters.

**Detailed**: The Model Registry Operator automates the deployment and management of ML model registry services within the RHOAI platform. It reconciles `ModelRegistry` custom resources to create and manage REST API deployments backed by PostgreSQL or MySQL databases, with kube-rbac-proxy sidecars providing RBAC-based authentication. The operator migrated from oauth-proxy to kube-rbac-proxy for the 3.x release.

The operator includes two controllers: the `ModelRegistryReconciler` manages per-instance model registry deployments (each `ModelRegistry` CR results in a separate deployment with its own database, service, route, and auth proxy), and the `ModelCatalogReconciler` manages a singleton model catalog service that aggregates metadata across registries. The operator supports platform-aware behavior, detecting OpenShift capabilities at startup including BYOIDC mode (OpenShift 4.20+), and conditionally creating platform-specific resources like Routes and Groups.

The operator also includes a storage version migration system that monitors CRD version changes and triggers data migration from v1alpha1 to v1beta1, supporting both Kubernetes StorageVersionMigration API and manual fallback strategies.
