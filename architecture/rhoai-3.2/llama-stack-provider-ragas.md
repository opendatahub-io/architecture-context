## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates the Ragas evaluation framework for LLM/RAG quality assessment, with both inline (in-process) and remote (Kubeflow Pipelines) execution modes.

**Detailed**: The llama-stack-provider-ragas component implements [Ragas](https://github.com/explodinggradients/ragas) as an external evaluation provider for the [Llama Stack](https://github.com/meta-llama/llama-stack) platform. It exposes Ragas' evaluation metrics (answer relevancy, context precision, faithfulness, context recall) through Llama Stack's standard Eval API, enabling users to evaluate LLM and RAG pipeline quality using standardized benchmarks and datasets.

The provider supports two execution modes: an **inline provider** that runs Ragas evaluations in the same process as the Llama Stack server (suitable for development and small-scale evaluations), and a **remote provider** that submits evaluation jobs to Kubeflow Pipelines for distributed execution on OpenShift (suitable for production workloads). The remote provider orchestrates a two-step KFP pipeline: data retrieval from a Llama Stack server followed by Ragas evaluation, with results stored in S3-compatible object storage.

The component wraps Llama Stack's inference APIs (both LLM completions and embeddings) into Ragas-compatible interfaces (LangChain-compatible `BaseRagasLLM` and `BaseRagasEmbeddings`), enabling Ragas to use whatever LLM and embedding backends are configured in the Llama Stack distribution. For RHOAI 3.2, the Konflux build produces a UBI9-based container image (`odh-trustyai-ragas-lls-provider-dsp-rhel9`) used as the base image for Kubeflow Pipeline components.

### Llama Stack Eval API (Provider Interface)

This component implements the Llama Stack `Eval` API interface:

| Method | Purpose | Provider |
|--------|---------|----------|
| `run_eval(benchmark_id, benchmark_config)` | Submit an evaluation job | Inline + Remote |
| `job_status(benchmark_id, job_id)` | Check evaluation job status | Inline + Remote |
| `job_result(benchmark_id, job_id)` | Retrieve evaluation results | Inline + Remote |
| `job_cancel(benchmark_id, job_id)` | Cancel a running evaluation job | Remote only |
| `register_benchmark(task_def)` | Register a benchmark for evaluation | Inline + Remote |
| `evaluate_rows(...)` | Evaluate individual rows (not implemented) | Neither |
