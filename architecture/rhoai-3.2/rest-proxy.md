## Purpose
**Short**: A reverse-proxy sidecar that translates KServe V2 REST inference requests into gRPC calls for ModelMesh Serving backends.

**Detailed**: The KServe V2 REST Proxy leverages gRPC-Gateway to create a reverse-proxy server that translates RESTful HTTP API calls into gRPC messages. This enables clients to send inference requests using the KServe V2 REST Predict Protocol to model serving platforms (ModelMesh) that natively expect the gRPC V2 Predict Protocol.

The proxy runs as a sidecar container within ModelMesh Serving deployments. It listens for incoming HTTP REST requests on a configurable port (default 8008), transforms the JSON request payloads into protobuf messages, and forwards them as gRPC calls to the local inference service on a configurable port (default 8033). It supports TLS for both the incoming HTTP listener and the outgoing gRPC connection, handles all KServe V2 tensor data types (BOOL, INT8-64, UINT8-64, FP16/32/64, BYTES), and performs custom JSON marshaling/unmarshaling to properly handle nested tensor data and base64-encoded byte content. The container image is built using Konflux with FIPS-compliant strict runtime (`GOEXPERIMENT=strictfipsruntime`).
