## Purpose
**Short**: A central repository for recording and retrieving metadata associated with ML model developer and data scientist workflows.

**Detailed**: Model Registry is a REST API server that stores metadata for ML applications, providing a centralized repository for managing registered models, model versions, model artifacts, inference services, serving environments, experiments, and experiment runs. It is built on the Kubeflow Model Registry project and serves as the metadata backbone for ML model lifecycle management within the RHOAI platform.

The component consists of multiple sub-components: (1) the **proxy server** — the primary REST API service backed by an embedded metadata store using MySQL or PostgreSQL; (2) the **InferenceService controller** — a Kubernetes controller that watches KServe InferenceService CRs and synchronizes their state with the Model Registry API; (3) the **CSI storage initializer** — a KServe ClusterStorageContainer that downloads model artifacts from Model Registry using the `model-registry://` URI scheme; (4) the **catalog server** — a REST API for managing a model catalog sourced from external providers (e.g., HuggingFace); and (5) the **async-upload job** — a Python-based Kubernetes Job image that copies models between storage back-ends and registers them in Model Registry.

The system supports FIPS-compliant builds via `strictfipsruntime` Go build tags, and configurable TLS for database connections with cipher suite validation. It integrates with Istio service mesh for mTLS and traffic management within the Kubeflow/RHOAI platform.
