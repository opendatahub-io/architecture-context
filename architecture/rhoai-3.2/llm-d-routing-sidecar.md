## Purpose
**Short**: A reverse proxy sidecar that enables disaggregated prefill/decode (P/D) inference by routing requests between prefiller and decoder vLLM instances using NIXL or LMCache protocols.

**Detailed**: The llm-d-routing-sidecar is an HTTP/HTTPS reverse proxy deployed as a sidecar container alongside a vLLM decoder instance. It intercepts OpenAI-compatible chat completion and legacy completion API requests, inspects the `x-prefiller-host-port` header to determine if disaggregated prefilling should be performed, and orchestrates the two-phase prefill-then-decode inference workflow.

When a request arrives with a prefiller target specified, the sidecar first forwards a modified request to the designated prefiller pod (with streaming disabled and `max_tokens=1`), collects KV transfer parameters from the prefiller's response, then constructs a decode request enriched with these parameters and forwards it to the local vLLM decoder. This enables KV-cache transfer between separate prefiller and decoder GPU instances using NIXL (v1/v2) or LMCache protocols.

The sidecar also includes SSRF (Server-Side Request Forgery) protection that watches Kubernetes InferencePool resources (from the Gateway API Inference Extension) to maintain a dynamic allowlist of valid prefill targets, preventing requests from being forwarded to unauthorized hosts.

### SSRF Protection

When enabled (`--enable-ssrf-protection`), the AllowlistValidator:
1. Watches a specific InferencePool resource in the configured namespace
2. Extracts the pod selector from the InferencePool spec
3. Watches pods matching the selector and builds an allowlist of pod IPs and names
4. Validates `x-prefiller-host-port` header targets against the allowlist (host/IP only, port is not checked)
5. Returns HTTP 403 for unauthorized prefill targets

## Data Flows

### Flow 1: Disaggregated Prefill/Decode (NIXL v2)

| Step | Source | Destination | Port | Protocol | Encryption | Auth |
|------|--------|-------------|------|----------|------------|------|
| 1 | Client / Gateway | Sidecar Proxy | 8000/TCP | HTTPS | TLS 1.2+ | None |
| 2 | Sidecar Proxy | SSRF Validator | - | In-process | - | Allowlist check |
| 3 | Sidecar Proxy | Remote Prefiller | Dynamic | HTTP/HTTPS | Optional TLS 1.2+ | None |
| 4 | Sidecar Proxy | Local vLLM Decoder | 8001/TCP | HTTP/HTTPS | Optional TLS 1.2+ | None |
| 5 | Local vLLM Decoder | Client / Gateway | 8000/TCP | HTTPS | TLS 1.2+ | None |

**Protocol details (NIXL v2)**:
- Step 3: Sends modified request to prefiller with `kv_transfer_params.do_remote_decode=true`, `stream=false`, `max_tokens=1`
- Step 3 response: Receives `kv_transfer_params` containing remote engine ID, block IDs, host, and port
- Step 4: Sends original request to decoder with `kv_transfer_params` from prefiller response (original stream/max_tokens restored)

### Flow 2: Passthrough (No Disaggregation)

| Step | Source | Destination | Port | Protocol | Encryption | Auth |
|------|--------|-------------|------|----------|------------|------|
| 1 | Client / Gateway | Sidecar Proxy | 8000/TCP | HTTPS | TLS 1.2+ | None |
| 2 | Sidecar Proxy | Local vLLM Decoder | 8001/TCP | HTTP/HTTPS | Optional TLS 1.2+ | None |

When no `x-prefiller-host-port` header is present, requests pass through directly to the local decoder.
