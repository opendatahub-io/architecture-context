## Purpose
**Short**: Feast Operator manages the lifecycle of Feast feature store deployments on Kubernetes/OpenShift via the FeatureStore custom resource.

**Detailed**: The Feast Operator is a Kubernetes operator built with controller-runtime that automates the deployment and management of Feast feature stores. Feast is an open-source feature store for machine learning that manages and serves features to models in production. The operator watches FeatureStore custom resources and reconciles the desired state by creating and managing Deployments, Services, ConfigMaps, PersistentVolumeClaims, ServiceAccounts, CronJobs, and (on OpenShift) Routes.

The operator supports deploying multiple Feast service types within a single Deployment pod: registry server (gRPC and/or REST), online store server, offline store server, and a UI server. It handles TLS configuration automatically on OpenShift using service serving certificates, supports Kubernetes RBAC and OIDC authorization models, manages persistent storage for feature data, and provides a CronJob mechanism for scheduled feature materialization. A secondary controller (NotebookConfigMapReconciler) watches Kubeflow Notebook CRs and creates ConfigMaps containing Feast client configuration for notebook integration.

In the RHOAI distribution, the operator is deployed to the `redhat-ods-applications` namespace via kustomize overlays processed by the rhods-operator. Image references are injected at deploy time through `RELATED_IMAGE_*` environment variables for disconnected/air-gapped support.

| FeatureStoreReconciler | Controller | Primary reconciler for FeatureStore CR; deploys all feast services, authz, client config, namespace registry, and CronJobs |
| NotebookConfigMapReconciler | Controller | Watches Kubeflow Notebook CRs with `opendatahub.io/feast-integration` label; creates per-notebook ConfigMaps with Feast client config |
| FeastAuthorization | Authorization Module | Deploys Kubernetes RBAC (Roles, RoleBindings, ClusterRoles, ClusterRoleBindings) or configures OIDC auth for Feast services |
| Feature Server | Python Application | Multi-container pod running Feast servers (registry, online, offline, UI) using the `feast` CLI |
| CronJob | Kubernetes CronJob | Scheduled `feast apply` and `feast materialize-incremental` commands via `kubectl exec` into the Feast deployment |
| Namespace Registry | ConfigMap | Cluster-wide ConfigMap (`feast-configs-registry`) tracking deployed FeatureStore instances across namespaces |
