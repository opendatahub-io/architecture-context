## Purpose
**Short**: ModelMesh is a distributed LRU cache and routing layer for high-scale, high-density model serving that manages model lifecycle (loading, unloading, scaling) across a cluster of model runtime containers.

**Detailed**: ModelMesh is a mature, general-purpose model serving management and routing layer originally developed by IBM. It acts as a sidecar container that runs alongside model runtime containers within Kubernetes pods, functioning as a distributed LRU cache for serving runtime models. It coordinates model placement, loading, unloading, and request routing across multiple pod replicas using etcd or ZooKeeper as a key-value store for state coordination.

The framework is designed for use cases with high model density (many models per pod), frequent model changes, and high-scale inference traffic. It implements intelligent model placement and auto-scaling of model copies within existing pods based on request rates or latency. ModelMesh exposes a gRPC API for model management (registration, unregistration, status) and transparently proxies inference requests to the appropriate model runtime, handling cache misses by loading models on-demand. It supports virtual models (VModels) for zero-downtime model updates.

ModelMesh communicates with colocated model runtime containers via a local gRPC interface (either TCP on port 8085/8056 or Unix Domain Sockets) using the ModelRuntime proto service. The external service interface is exposed via gRPC on a configurable port (default 8033) and supports TLS with optional mutual authentication. Within RHOAI, ModelMesh is deployed as part of the modelmesh-serving stack alongside KServe for model inference.
