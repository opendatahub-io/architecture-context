## Purpose
**Short**: NeMo Guardrails is an open-source toolkit for adding programmable guardrails to LLM-based conversational systems, controlling and securing LLM outputs.

**Detailed**: NeMo Guardrails, originally developed by NVIDIA, provides a framework for defining programmable guardrails ("rails") that control large language model outputs in conversational applications. Rails can enforce topic boundaries, prevent jailbreaks and prompt injections, perform fact-checking, detect sensitive data, moderate content, and enforce predefined dialog paths using the Colang domain-specific language.

The RHOAI distribution packages NeMo Guardrails as a containerized server built on FastAPI/Uvicorn, exposing an OpenAI-compatible chat completions API. The server loads guardrail configurations at startup and applies them to all LLM interactions. The container image is built via Konflux using a multi-stage Dockerfile (`Dockerfile.konflux`) based on UBI9/Python 3.12, and ships with pre-downloaded ML models (spaCy, sentence-transformers, FastEmbed) for air-gapped/disconnected operation. A build-time profile system (`filter_guardrails.py`) removes closed-source guardrail libraries (e.g., ActiveFence, Cleanlab, Fiddler) from the open-source distribution.

The toolkit includes a rich library of guardrail modules covering content safety, hallucination detection, jailbreak detection, injection detection, sensitive data detection (PII via Presidio), topic safety, and integrations with external safety services. It supports multiple LLM providers through LangChain, including NVIDIA NIM endpoints, OpenAI, and HuggingFace models.
