## Purpose
**Short**: Caikit-TGIS-Serving is a container image providing a KServe-compatible serving runtime for Large Language Model (LLM) inference using the Caikit AI toolkit and Text Generation Inference Server (TGIS) backend.

**Detailed**: Caikit-TGIS-Serving packages the Caikit AI framework together with the caikit-nlp and caikit-tgis-backend libraries into a container image that runs as a KServe ServingRuntime. It acts as a "transformer container" in a multi-container KServe pod, sitting alongside a TGIS inference engine container. The Caikit runtime provides developer-friendly APIs (both HTTP/REST and gRPC) for text generation tasks, while TGIS handles the underlying model loading and GPU-accelerated inference.

The runtime is designed for deployment on Red Hat OpenShift AI (RHOAI) via KServe InferenceService custom resources. Models are loaded from S3-compatible storage or PersistentVolumeClaims in Caikit format. The Caikit runtime connects to the co-located TGIS container over localhost gRPC (port 8033) to delegate actual model inference. The stack integrates with OpenShift Service Mesh (Istio) for mTLS networking and Knative Serverless for autoscaling and traffic management.

This repository does not contain an operator or controller — it builds a container image that is consumed as a runtime by KServe. The repository includes demo manifests showing how to deploy the runtime, configure metrics collection, and set up the required infrastructure (Service Mesh, Serverless, KServe).
