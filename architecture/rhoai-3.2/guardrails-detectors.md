## Purpose
**Short**: A collection of detector algorithm microservices for the FMS Guardrails Orchestrator, providing text content analysis including PII detection, file-type validation, HuggingFace model-based classification, and LLM-as-a-judge evaluation.

**Detailed**: Guardrails Detectors is a Python-based repository that provides three distinct detector microservices designed to work with the IBM FMS Guardrails Orchestrator. Each detector runs as an independent FastAPI-based HTTP service, implementing a common detector API contract (`/api/v1/text/contents`).

The **built-in detector** bundles lightweight heuristic detection functions (regex-based PII detection for emails, credit cards, SSNs, phone numbers, IP addresses; file-type validation for JSON, XML, YAML including schema validation; and a custom detector framework allowing user-defined Python detection functions). The **HuggingFace detector** loads transformer models (AutoModelForSequenceClassification or GraniteForCausalLM) for ML-based content classification including toxicity, harm, jailbreak, and bias detection. The **LLM Judge detector** integrates the vllm_judge library to use LLM-as-a-judge evaluation against an external vLLM-compatible inference server, supporting built-in metrics like safety, accuracy, helpfulness, and custom criteria.

All detectors are built as container images using Konflux CI pipelines and are intended for deployment as KServe InferenceService instances on OpenShift AI (RHOAI). They expose Prometheus metrics for observability and support multi-architecture builds (amd64, arm64, ppc64le, s390x).
