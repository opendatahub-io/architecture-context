## Purpose
**Short**: Manages the lifecycle of Data Science Pipelines (DSP) infrastructure by reconciling DataSciencePipelinesApplication (DSPA) custom resources.

**Detailed**: The Data Science Pipelines Operator (DSPO) is a Kubernetes operator built with controller-runtime that watches for DataSciencePipelinesApplication (DSPA) custom resources and deploys a complete Kubeflow Pipelines V2 stack per namespace. For each DSPA instance, the operator dynamically creates and manages the API server, persistence agent, scheduled workflow controller, Argo Workflow controller, ML Metadata (MLMD) gRPC server, MLMD Envoy proxy, MariaDB database (or configures external DB), and Minio object storage (or configures external S3).

The operator uses Go templates (located in `config/internal/`) to render Kubernetes manifests at runtime, injecting parameters from the DSPA CR spec and environment variables. It performs active health checks against the database (MariaDB/MySQL) and object storage (S3-compatible) before deploying dependent components. Pod-to-pod TLS is enabled by default using OpenShift service-ca signed certificates. The operator also manages NetworkPolicies to restrict traffic between DSP components and exposes the API server through an OpenShift Route with kube-rbac-proxy for RBAC-based authentication.

The operator is deployed by the rhods-operator using kustomize overlays from `config/overlays/rhoai/`, which references the base configuration and Argo Workflow CRDs. Container images are injected at deploy time through RELATED_IMAGE environment variables for disconnected/air-gapped support. FIPS compliance is supported through a dedicated build mode using `strictfipsruntime` Go build tags.
