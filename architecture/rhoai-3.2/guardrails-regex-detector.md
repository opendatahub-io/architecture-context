## Purpose
**Short**: A lightweight regex-based text detection microservice that identifies PII and custom patterns in text content, designed for integration with the FMS Guardrails Orchestrator.

**Detailed**: The Guardrails Regex Detector is a Rust-based HTTP microservice built with the Axum web framework. It provides a REST API endpoint that accepts text content and applies regex-based pattern matching to detect sensitive information such as email addresses, Social Security numbers, and credit card numbers. These are classified as built-in PII detectors.

In addition to built-in patterns, the service supports arbitrary custom regular expressions submitted at request time. Each detection result includes positional information (start/end offsets), the matched text, a detection type classification, and a confidence score. The service is designed to operate as a sidecar or backend detector within the FMS Guardrails Orchestrator pipeline, where it evaluates LLM inputs and outputs for sensitive content that should be flagged or redacted.

### Request/Response Format

**Request Body** (`POST /api/v1/text/contents`):
```json
{
  "contents": ["array of text strings to scan"],
  "detector_params": {
    "regex": ["ssn", "email", "credit-card", "^custom-regex$"]
  }
}
```

**Response Body**:
```json
[[
  {
    "start": 0,
    "end": 16,
    "text": "matched text",
    "detection_type": "pii|custom",
    "detection": "EmailAddress|SocialSecurity|CreditCard|custom",
    "score": 1.0
  }
]]
```
