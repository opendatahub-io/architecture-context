## Purpose
**Short**: A high-performance text generation inference server (TGIS) forked from HuggingFace TGI, providing gRPC and HTTP APIs for serving large language models in production.

**Detailed**: Text Generation Inference Server (TGIS) is an early fork of HuggingFace's text-generation-inference, developed and extended by IBM. It serves as a core inference runtime within the Red Hat OpenShift AI (RHOAI) platform, enabling deployment and serving of large language models (LLMs) including causal LM and sequence-to-sequence models.

The server has a multi-process architecture consisting of three main components: a **Launcher** (Rust) that orchestrates the startup of model shards and the router, a **Router** (Rust) that handles incoming gRPC and HTTP requests with continuous batching and request queuing, and a **Python Server** that performs the actual model inference using PyTorch. The router communicates with the Python server shards via Unix domain sockets using an internal gRPC protocol.

Key differentiating features from upstream HF TGI include: a gRPC front-end interface (FMaaS GenerationService), batch input support, independent tokenization API, configurable inference engine abstraction (supporting HF Transformers, IBM FMS, ONNX Runtime, DeepSpeed), flash attention and paged attention support, tensor parallelism for sharded model deployments, GPTQ quantization, and PyTorch 2 compile support. The server is built on UBI 9 base images with CUDA 12.1 support for GPU-accelerated inference.
