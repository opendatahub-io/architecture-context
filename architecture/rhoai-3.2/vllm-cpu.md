## Purpose
**Short**: CPU-optimized build of vLLM providing high-throughput LLM inference serving via OpenAI-compatible REST API and gRPC (via TGIS adapter).

**Detailed**: vLLM CPU is a Red Hat-maintained fork of the vLLM inference engine, specifically built and optimized for CPU-only deployments without GPU hardware. It provides an OpenAI-compatible HTTP API server for large language model (LLM) inference, supporting completions, chat completions, embeddings, tokenization, classification, scoring, transcription, and the OpenAI Responses API. The engine uses PagedAttention for efficient KV-cache memory management and supports advanced features like speculative decoding, LoRA adapters, structured output (guided generation via xgrammar/outlines), and multimodal inputs.

The container image ships in two variants built from a single Dockerfile: `vllm-openai` (primary, serving the OpenAI-compatible REST API on port 8000) and `vllm-grpc-adapter` (serving both REST on port 8000 and gRPC on port 8033 via the vllm-tgis-adapter for KServe/TGIS protocol compatibility). Platform-specific builds exist for x86_64 (with Intel Extension for PyTorch), aarch64, ppc64le, and s390x, each with tailored build scripts and optimized dependencies. The v1 engine architecture is enabled by default (`VLLM_USE_V1=1`) in the production container.
