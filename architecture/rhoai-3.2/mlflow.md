## Purpose
**Short**: MLflow is an open-source ML lifecycle platform providing experiment tracking, model registry, tracing, and artifact management, extended for RHOAI with Kubernetes-native workspace isolation and RBAC authorization.

**Detailed**: MLflow provides a comprehensive platform for managing the machine learning lifecycle. Its core capabilities include experiment tracking (logging parameters, metrics, and artifacts for ML runs), a model registry for versioning and managing ML models, LLM observability via OpenTelemetry-based tracing, model evaluation, and prompt management. The server exposes both REST and GraphQL APIs and includes a React-based web UI for browsing experiments, runs, models, and traces.

The RHOAI distribution extends upstream MLflow with a Kubernetes workspace provider plugin (`mlflow-kubernetes-workspace-provider`) that maps MLflow workspaces 1:1 to Kubernetes namespaces. This enables multi-tenant isolation where each team's namespace becomes an MLflow workspace. An accompanying authorization plugin (`kubernetes-auth`) enforces Kubernetes RBAC on every MLflow API request by performing `SelfSubjectAccessReview` or `SubjectAccessReview` calls against the `mlflow.kubeflow.org` API group. This integration supports both direct bearer token authentication and proxy-forwarded headers from `kube-rbac-proxy`.

The container image is built on UBI9 with Python 3.11 and Node.js 20, using a multi-stage Konflux build that compiles the React UI and packages both the MLflow server and the Kubernetes workspace provider. The server runs under Gunicorn/Uvicorn on port 5000 with configurable backends for tracking store (SQLAlchemy/PostgreSQL), artifact storage (S3/MinIO/GCS/Azure), and model registry.
