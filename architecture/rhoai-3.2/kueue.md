## Purpose
**Short**: Kueue is a Kubernetes-native job queueing system that manages when workloads should be admitted to start and when they should be stopped, based on quota and fair-sharing policies.

**Detailed**: Kueue provides a set of APIs and controllers for job-level resource management in Kubernetes clusters. It implements a quota management system that controls when jobs (pods) can be created by defining ClusterQueues with resource quotas, LocalQueues for namespace-level access, and ResourceFlavors to represent different types of compute resources (e.g., different GPU types or node pools).

The system integrates with multiple job frameworks including Kubernetes batch/Job, Kubeflow training operators (PyTorchJob, TFJob, MPIJob, etc.), Ray (RayJob, RayCluster), JobSet, LeaderWorkerSet, CodeFlare AppWrapper, and plain Pods. When a job is submitted, Kueue creates a corresponding Workload CR that represents its resource requirements. The scheduler then decides when to admit workloads based on available quota, priorities, fair-sharing policies, and admission checks (such as provisioning requests for cluster auto-scaling).

Key capabilities include partial admission, multi-cluster queueing (MultiKueue), topology-aware scheduling (TAS), hierarchical cohorts for resource sharing between queues, preemption based on priority and fair-share, and configurable resource transformations. The RHOAI distribution builds the controller with FIPS-compliant cryptography (`GOEXPERIMENT=strictfipsruntime`) and deploys into the `opendatahub` namespace.
