## Purpose
**Short**: Provides pre-built container images for Jupyter, Code Server, and RStudio workbench environments along with Elyra pipeline runtime images for the RHOAI platform.

**Detailed**: The notebooks repository is the source for all workbench (IDE) container images and data science pipeline runtime images shipped with Red Hat OpenShift AI. It does not contain an operator or controller — instead, it produces OpenShift ImageStream manifests and the container images those ImageStreams reference. The rhods-operator deploys these ImageStream resources via kustomize, making the workbench images available for users to select when creating notebook workbenches through the RHOAI dashboard.

The repository provides a layered image build hierarchy: base images (CPU, CUDA, ROCm) serve as foundations for specialized workbench images (Jupyter Minimal, Data Science, PyTorch, TensorFlow, TrustyAI, Code Server, RStudio) and pipeline runtime images. Each workbench type is offered across multiple Python versions (3.11, 3.12) and accelerator targets (CPU, NVIDIA CUDA 12.8, AMD ROCm 6.3). The manifests maintain a rolling window of image versions (N through N-6) to support upgrade and rollback scenarios.

All images are built by Konflux CI/CD using Tekton pipelines defined in the `.tekton/` directory. For RHOAI, the N-1 version images are sourced from `registry.redhat.io/rhoai/` while older versions reference `quay.io/modh/` repositories.
