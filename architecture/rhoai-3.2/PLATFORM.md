## Platform Purpose

Red Hat OpenShift AI (RHOAI) 3.2 is a comprehensive AI/ML platform deployed on OpenShift Container Platform (OCP) that provides end-to-end capabilities for model development, training, serving, evaluation, and governance. The platform is orchestrated by the **rhods-operator**, which manages the lifecycle of all components through two primary custom resources: `DSCInitialization` (platform-level initialization) and `DataScienceCluster` (component management).

RHOAI 3.2 introduces Gateway API-based ingress infrastructure (replacing Istio VirtualServices), with a centralized `data-science-gateway` Gateway in the `openshift-ingress` namespace and `kube-auth-proxy` for OAuth2/OIDC authentication. All components are built via the Konflux CI/CD system using UBI9 base images with FIPS-compliant cryptographic operations (`GOEXPERIMENT=strictfipsruntime` for Go, equivalent flags for other languages). The platform supports disconnected/air-gapped installations with all 78 container image references pinned by SHA256 digest in the operator bundle.

- **mTLS Backend**: DestinationRule configures mTLS on ports 8443 and 443

## Build & Release

### Konflux CI/CD

All 78 container images are built via Konflux (Tekton-based) CI/CD pipelines with:
- **Hermetic builds**: No external network access during build
- **Multi-architecture**: amd64, arm64, ppc64le, s390x (where applicable)
- **Base images**: UBI 9 (ubi9, ubi9-minimal, python-312, nodejs-22)
- **FIPS compliance**: Go binaries with `strictfipsruntime`, FIPS-validated crypto modules
- **Enterprise Contract**: Policy validation for signatures and compliance before release
