## Purpose
**Short**: A Kubernetes controller that manages the lifecycle of model serving infrastructure resources (Routes, Services, RBAC, monitoring, TLS certificates, and auth policies) for KServe InferenceServices and LLMInferenceServices on RHOAI/OpenShift.

**Detailed**: The odh-model-controller is a controller-runtime based Go operator that watches KServe InferenceService, LLMInferenceService, ServingRuntime, and NIM Account custom resources, and reconciles the supporting OpenShift/Kubernetes infrastructure needed for model serving. For KServe InferenceServices in RawDeployment mode, it creates OpenShift Routes for external exposure, ClusterRoleBindings, metrics Services, ServiceMonitors, PodMonitors, metrics dashboards (ConfigMaps), and KEDA TriggerAuthentications for autoscaling.

For LLMInferenceServices (the newer Gateway API-based serving path), it manages Kuadrant AuthPolicy CRs (for both Gateway-level and HTTPRoute-level auth), Istio EnvoyFilter CRs (for gateway TLS termination), and RBAC Roles/RoleBindings. This controller also manages NVIDIA NIM integration via the Account CRD, handles CA certificate bundle distribution for model storage, aggregates data connection secrets into KServe storage-config secrets, manages Ray TLS certificates for multi-node serving runtimes, and deploys out-of-the-box ServingRuntime Templates (vLLM CUDA/ROCm/Gaudi/CPU/Spyre, OVMS, guardrails).

The controller includes mutating and validating webhooks for Pods (injecting multi-node Ray TLS init containers), InferenceServices, InferenceGraphs, LLMInferenceServices, and NIM Accounts.

**InferenceService Route Creation (RHOAI 3.x, RawDeployment mode)**:
- Routes are created per InferenceService only when label `networking.kserve.io/visibility=exposed` is present
- Route targets the predictor or transformer Service
- TLS mode: Edge (default) or Reencrypt (when `security.opendatahub.io/enable-auth=true`)

**LLMInferenceService Auth (RHOAI 3.x, Gateway API pattern)**:
- AuthPolicies created targeting both Gateway (user-defined auth) and HTTPRoute (anonymous auth) resources
- EnvoyFilters created targeting the Gateway for TLS SSL configuration
- Default Gateway: `openshift-ai-inference` in namespace `openshift-ingress`
- AuthPolicies managed via Kuadrant operator
- Resources set Gateway as controller owner reference
