## Purpose
**Short**: Kubernetes operator that manages RayCluster lifecycle with OAuth proxy integration, mTLS certificate management, and embedded AppWrapper batch scheduling on OpenShift AI.

**Detailed**: The CodeFlare Operator extends Kubernetes/OpenShift to manage the lifecycle of RayCluster resources by adding security and networking infrastructure around Ray head nodes. On OpenShift, it automatically provisions OAuth proxy sidecars for dashboard authentication, creates OpenShift Routes for Ray dashboard and RayClient access, generates and rotates CA certificates for inter-node mTLS communication, and creates NetworkPolicies to secure Ray cluster traffic. On vanilla Kubernetes, it provisions Ingress resources instead of Routes.

The operator also embeds an optional AppWrapper controller (from the project-codeflare/appwrapper library) that provides batch job scheduling with quota management for workloads including RayClusters, PyTorchJobs, and JobSets. The AppWrapper controller wraps Kubernetes resources with scheduling policies and lifecycle management via the `AppWrapper` CRD. Both controllers expose mutating and validating webhooks to enforce immutability of security-critical configurations (OAuth proxy containers, mTLS certificates, environment variables).

## Security

### Sidecar Injection (via Mutating Webhook)

The RayCluster mutating webhook injects the following into head pod specs:

| Sidecar | Port | Image | Purpose |
|---------|------|-------|---------|
| oauth-proxy | 8443/TCP | registry.redhat.io/openshift4/ose-oauth-proxy | OpenShift OAuth authentication proxy fronting Ray Dashboard (upstream at localhost:8265) |
| create-cert (init container) | N/A | registry.redhat.io/ubi9 | Generates per-pod TLS certificates signed by the RayCluster CA for mTLS |

### mTLS Configuration

When mTLS is enabled (default: true), the webhook injects:
- `RAY_USE_TLS=1` environment variable on head and worker containers
- `RAY_TLS_SERVER_CERT`, `RAY_TLS_SERVER_KEY`, `RAY_TLS_CA_CERT` paths
- `create-cert` init container that generates per-node certificates from the cluster CA
- CA secret (`ca-secret-{cluster}`) containing `ca.key` and `ca.crt` (RSA 2048-bit, self-signed, 1-year validity)
