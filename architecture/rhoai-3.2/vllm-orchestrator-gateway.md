## Purpose
**Short**: An HTTP gateway that provides OpenAI-compatible chat completion endpoints with configurable guardrail detector pipelines, routing requests to the FMS Guardrails Orchestrator.

**Detailed**: The vLLM Orchestrator Gateway is a Rust-based HTTP proxy service that sits between clients and the FMS Guardrails Orchestrator. It exposes multiple route-specific `/v1/chat/completions` endpoints (e.g., `/{route_name}/v1/chat/completions`), each configured with a specific set of content detectors (e.g., PII detection via regex). This allows operators to create distinct guardrailed endpoints that enforce different content filtering policies while maintaining OpenAI API compatibility.

The gateway reads a YAML configuration file at startup that defines: (1) the upstream orchestrator service location, (2) available detector configurations with input/output classification and parameters, and (3) named routes that map to specific detector combinations with optional fallback messages. When a detection is found in the output, the gateway replaces the model's response with a configured fallback message. The gateway supports both streaming (SSE) and non-streaming chat completion modes, forwarding Authorization and X-Forwarded-* headers to the orchestrator. It supports mTLS for secure communication with the orchestrator when TLS certificates are present.
