## Purpose
**Short**: A build-time tool and data container that collects, enriches, and catalogs Red Hat AI validated model metadata from OCI container registries and HuggingFace.

**Detailed**: The model-metadata-collection component is a Go-based CLI tool and data container that extracts metadata from Red Hat AI model containers (modelcars) stored in OCI registries (primarily `registry.redhat.io/rhelai1/`), enriches that metadata with information from HuggingFace APIs (model details, README content, YAML frontmatter, tags), and produces structured YAML catalog files (`models-catalog.yaml`, `validated-models-catalog.yaml`).

The tool runs as part of a CI/CD pipeline (Konflux), not as a runtime service. It processes model container images by fetching their manifests and layers, extracting modelcard markdown files from annotated layers (`io.opendatahub.modelcar.layer.type: modelcard`), parsing metadata via YAML frontmatter and regex patterns, and enriching with HuggingFace collection data. The output catalog files are baked into a minimal container image (`ubi9-minimal`) that serves as a static data provider. This container runs `sleep infinity` and exposes its data via a volume mount at `/app/data`, making it suitable for use as an init container or sidecar that provides model catalog data to other RHOAI components (such as the dashboard).

The component supports two catalog tracks: a "Red Hat" catalog (`models-index.yaml` -> `models-catalog.yaml`) and a "validated models" catalog (`validated-models-index.yaml` -> `validated-models-catalog.yaml`), each with separate processing pipelines that share the same extraction and enrichment logic.
