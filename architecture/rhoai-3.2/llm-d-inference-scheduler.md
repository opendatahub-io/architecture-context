## Purpose
**Short**: An inference request scheduler that makes optimized routing decisions for LLM inference requests via Kubernetes Gateway API, extending the Gateway API Inference Extension (GIE).

**Detailed**: The llm-d-inference-scheduler provides the Endpoint Picker (EPP) component for the llm-d inference framework. It schedules incoming inference requests arriving through a Kubernetes Gateway to optimal model-serving pods using a pluggable system of filters, scorers, and scrapers. Routing decisions consider KV cache locality, session affinity, load balancing, and model metadata to optimize throughput and latency.

The scheduler extends the upstream Gateway API Inference Extension (GIE) project with llm-d-specific features, most notably disaggregated Prefill/Decode (P/D) execution. In P/D mode, the scheduler selects separate pods for prompt processing (prefill) and token generation (decode), with a routing sidecar on the decode node orchestrating communication between stages. The architecture supports multiple P/D connector protocols including NIXL v2, LMCache (deprecated), and SGLang.

The repository produces two container images: the EPP (inference scheduler) and the P/D routing sidecar. The EPP communicates with Envoy via gRPC ext-proc to make routing decisions, while the sidecar acts as a reverse proxy on decode worker pods to coordinate the prefill-decode lifecycle. Both components integrate with the InferencePool CRD from GIE to discover and manage backend model-serving pods.

### SSRF Protection

The P/D routing sidecar includes an optional SSRF protection mechanism (`--enable-ssrf-protection`). When enabled, it:
1. Watches the specified InferencePool resource via Kubernetes informers
2. Resolves the pool's label selector to discover matching pods
3. Builds a dynamic allowlist of pod IPs and names
4. Validates all prefill target host:port values against this allowlist before proxying
5. Returns HTTP 403 Forbidden for requests targeting hosts outside the allowlist
