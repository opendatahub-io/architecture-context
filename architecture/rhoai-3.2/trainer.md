## Purpose
**Short**: Kubeflow Trainer is a Kubernetes-native operator for managing distributed machine learning training jobs using the TrainJob, TrainingRuntime, and ClusterTrainingRuntime CRDs.

**Detailed**: Kubeflow Trainer (v2) provides a declarative API for orchestrating distributed training workloads on Kubernetes. Users define training jobs via `TrainJob` custom resources that reference reusable `TrainingRuntime` or `ClusterTrainingRuntime` templates. The controller reconciles these into underlying `JobSet` resources (from the Kubernetes JobSet API) that manage the actual training pods.

The operator supports multiple ML frameworks including PyTorch (torchrun), MPI (OpenMPI), DeepSpeed, MLX, and TorchTune. It provides gang-scheduling integration via both Kubernetes scheduler-plugins (coscheduling) and Volcano. The RHOAI distribution adds progression tracking capabilities (RHAI features), which poll training pods for real-time metrics (progress percentage, steps, epochs) and store them as TrainJob annotations.

For RHOAI, five pre-configured ClusterTrainingRuntime resources are shipped: `torch-distributed`, `torch-distributed-rocm`, `torch-distributed-th03-cuda128-torch28-py312`, `training-hub`, and `training-hub03-cuda128-torch28-py312`, providing out-of-the-box support for CUDA and ROCm GPU-accelerated PyTorch training.

## RHOAI-Specific Features

### Progression Tracking (RHAI)
- Enabled via `ENABLE_RHAI_FEATURES=true` environment variable on the controller
- Activated per-TrainJob via annotation: `trainer.opendatahub.io/progression-tracking: "true"`
- Controller polls primary training pod's HTTP `/metrics` endpoint (default port 28080)
- Configurable poll interval via annotation: `trainer.opendatahub.io/metrics-poll-interval` (default: 30s, min: 5s, max: 300s)
- Configurable metrics port via annotation: `trainer.opendatahub.io/metrics-port`
- Progress stored in TrainJob annotation: `trainer.opendatahub.io/trainerStatus` (JSON)
- Captures final metrics from pod termination messages on job completion/failure

### RBAC Progression Patch
- Adds `pods` get/list permissions to the controller ClusterRole (required for pod IP discovery)
