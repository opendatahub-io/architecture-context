## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of Llama Stack AI inference server deployments on OpenShift/Kubernetes clusters.

**Detailed**: The LlamaStack K8s Operator automates the deployment, configuration, and management of Llama Stack distribution servers within Kubernetes environments. It introduces a custom resource (`LlamaStackDistribution`) that allows users to declaratively specify their desired Llama Stack server configuration, including distribution type, container resources, storage, autoscaling, TLS CA bundles, and user configuration.

The operator watches for `LlamaStackDistribution` custom resources and reconciles the desired state by dynamically creating and managing Deployments, Services, PersistentVolumeClaims, ServiceAccounts, RoleBindings, NetworkPolicies, PodDisruptionBudgets, and HorizontalPodAutoscalers. It uses kustomize internally to render manifests from embedded base templates and applies runtime transformations (field mutations, namespace setting, name prefixing) before applying resources to the cluster.

The operator supports multiple Llama Stack distributions (e.g., `rh-dev` for RHOAI, `starter`, `remote-vllm`, `meta-reference-gpu`, `postgres-demo` for upstream), custom container images, user-provided ConfigMap-based configuration, CA bundle management (including auto-detection of ODH trusted CA bundles), and feature flags for optional capabilities like NetworkPolicy creation. It is deployed by the rhods-operator via kustomize overlays within the RHOAI platform.
