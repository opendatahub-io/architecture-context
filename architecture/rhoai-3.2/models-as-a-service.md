## Purpose
**Short**: MaaS provides a centralized API for managing, authenticating, and rate-limiting access to LLM inference services via Gateway API and Kuadrant policy engine.

**Detailed**: Models as a Service (MaaS) is a platform component that creates a comprehensive service layer for LLM model inference. It sits between users and KServe-hosted LLM models, providing an OpenAI-compatible model listing API, ephemeral and long-lived token management (via Kubernetes ServiceAccount tokens), and tier-based access control.

The platform deploys a Gateway API Gateway (`maas-default-gateway`) in the `openshift-ingress` namespace that serves as the single entry point for all model inference traffic. Kuadrant AuthPolicy and RateLimitPolicy resources are attached to this gateway to enforce authentication via Kubernetes TokenReview, tier-based authorization via SubjectAccessReview against `llminferenceservices` resources, and per-user rate limiting based on subscription tiers (free/premium/enterprise). A TokenRateLimitPolicy further enforces per-user token consumption limits.

The MaaS API server (a Go/Gin application) watches KServe `LLMInferenceService` and `InferenceService` CRDs across all namespaces and exposes them through an OpenAI-compatible `/v1/models` endpoint. It also manages token issuance by creating Kubernetes ServiceAccounts in tier-namespaced isolation and using the TokenRequest API to generate time-limited JWT tokens. API keys (long-lived tokens) are tracked in a pluggable metadata store (in-memory SQLite, disk SQLite, or external PostgreSQL).

The HTTPRoute `maas-api-route` references parent gateway `maas-default-gateway` in `openshift-ingress` namespace and routes:
- `/v1/models` -> `maas-api:8080` (direct path match)
- `/maas-api/*` -> `maas-api:8080` (with URL rewrite stripping `/maas-api` prefix)

LLMInferenceService models attach their own HTTPRoutes to the same `maas-default-gateway`, enabling direct model inference through the gateway with policy enforcement.

**Rate Limiting (per-user, enforced at gateway):**
- Free tier: 5 requests / 2 minutes, 100 tokens / 1 minute
- Premium tier: 20 requests / 2 minutes, 50,000 tokens / 1 minute
- Enterprise tier: 50 requests / 2 minutes, 100,000 tokens / 1 minute

**Token Management:**
- Ephemeral tokens: Kubernetes ServiceAccount tokens via TokenRequest API, configurable expiration (min 10 minutes, default 4 hours)
- API keys: Long-lived named tokens tracked in metadata store (SQLite or PostgreSQL)
- Token revocation: Deletes and recreates the user's ServiceAccount, invalidating all outstanding tokens
- Tier-based namespace isolation: Users get ServiceAccounts in `{instance}-tier-{tier}` namespaces
