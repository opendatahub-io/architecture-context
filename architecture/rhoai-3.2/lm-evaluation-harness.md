## Purpose
**Short**: A secure, batch-oriented container runtime for evaluating language models against standardized benchmarks using the EleutherAI lm-evaluation-harness framework.

**Detailed**: The lm-evaluation-harness (branded as LMEval Job in RHOAI) provides a unified framework for testing generative language models across a large number of evaluation tasks. It is the upstream project from EleutherAI that powers Hugging Face's Open LLM Leaderboard. The Red Hat Data Services fork packages this as a container image (`odh-ta-lmes-job-rhel9`) designed for one-shot evaluation workloads in OpenShift AI.

The container image is built using a multi-stage Dockerfile that supports all four target architectures (amd64, arm64, ppc64le, s390x). For ppc64le and s390x, PyTorch and Apache Arrow (PyArrow) are compiled from source because pre-built wheels are not available for those architectures. The image integrates the Unitxt evaluation framework, supports offline metrics, and includes S3/OCI artifact support for model download and result upload. It is designed to be launched as a Kubernetes Job by the TrustyAI LMEval operator, which manages the lifecycle of evaluation workloads via a custom `LMEvalJob` CRD (defined in the separate trustyai-service-operator repository).

The harness supports over 150 task categories containing hundreds of individual benchmarks, and can evaluate models loaded locally via HuggingFace Transformers, vLLM, SGLang, or remotely via API endpoints (OpenAI, Anthropic, IBM WatsonX, etc.).

## Build Architecture

### Multi-Architecture Container Build

The container image uses a multi-stage Dockerfile (`Dockerfile.konflux.lmes-job`) optimized for four CPU architectures:

| Architecture | PyTorch | PyArrow | OpenBLAS | Notes |
|-------------|---------|---------|----------|-------|
| amd64 | pip wheel | pip wheel | system | Standard pip install |
| arm64 | pip wheel | pip wheel | system | Standard pip install |
| ppc64le | Built from source (v2.6.0) | Built from source (v20.0.0) | Built from source (v0.3.30) | Requires gcc-toolset-13 |
| s390x | Built from source (v2.6.0) | Built from source (v20.0.0, with patch) | system (openblas-devel) | Requires gcc-toolset-13, custom Arrow patch |

### Build Stages (Dockerfile.konflux.lmes-job)

| Stage | Name | Purpose |
|-------|------|---------|
| 1 | builder | Base UBI9 Python 3.11 image with common build tooling |
| 2 | arrow-builder | Builds Apache Arrow/PyArrow from source for ppc64le and s390x |
| 3 | torch-builder | Builds PyTorch from source for ppc64le and s390x |
| 4 | openblas-builder | Builds OpenBLAS from source for ppc64le |
| 5 | final-build | Assembles final image with all dependencies and lm-eval package |
