## Purpose
**Short**: A reverse-proxy server that translates KServe V2 REST inference requests into gRPC calls for model serving backends.

**Detailed**: The KServe V2 REST Proxy leverages gRPC-Gateway to create a reverse-proxy server which translates RESTful HTTP API calls into gRPC. This enables clients to send inference requests using the KServe V2 REST Predict Protocol to model serving platforms (like ModelMesh) that natively expect the gRPC V2 Predict Protocol.

The proxy handles complex data transformations including multi-dimensional tensor flattening, base64-encoded byte content, and raw binary tensor data conversion between JSON and protobuf representations. It supports all KServe V2 tensor data types (BOOL, INT8-64, UINT8-64, FP32, FP64, BYTES) except FP16. The proxy is designed to run as a sidecar container alongside the model serving runtime within ModelMesh Serving deployments, listening on a configurable HTTP port (default 8008) and forwarding translated requests to the gRPC inference service on a configurable port (default 8033).
