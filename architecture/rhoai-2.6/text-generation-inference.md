## Purpose
**Short**: A high-performance text generation inference server forked from HuggingFace TGI, providing gRPC-based LLM serving with batching, streaming, and tensor parallelism.

**Detailed**: Text Generation Inference Server (TGIS) is an IBM-developed fork of HuggingFace's text-generation-inference, adapted for use in Red Hat OpenShift AI (RHOAI). It provides optimized serving of large language models (LLMs) including BLOOM, T5, GPT-NeoX, Falcon, LLaMA, and StarCoder families.

The architecture follows a three-tier design: a Rust-based **launcher** process that manages Python model shard processes and the Rust router; a Rust-based **router** that exposes both a gRPC API (primary, on port 8033) and an HTTP health/metrics endpoint (on port 3000), handles request validation, batching, and token decoding; and Python-based **model server shards** that perform the actual GPU inference using PyTorch with support for multiple inference engines (HuggingFace Transformers, Accelerate, custom tensor parallelism, ONNX Runtime, DeepSpeed).

Key differentiators from upstream HuggingFace TGI include: gRPC front-end interface (fmaas.GenerationService), batch input support in the API, tokenization API, configurable inference engine abstraction, UBI-based container images, dynamic batch sizing, and support for tuned prompts via PEFT.
