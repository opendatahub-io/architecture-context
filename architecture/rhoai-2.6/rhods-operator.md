## Purpose
**Short**: The RHOAI platform operator that manages the lifecycle of all Red Hat OpenShift AI components through DataScienceCluster and DSCInitialization custom resources.

**Detailed**: The rhods-operator (also known as opendatahub-operator v2) is the central Kubernetes operator for Red Hat OpenShift AI (RHOAI). It provides two primary cluster-scoped custom resources: `DSCInitialization` for platform-level initialization (namespace creation, monitoring setup, Service Mesh configuration, network policies) and `DataScienceCluster` for enabling/disabling individual AI/ML components.

The operator uses a component-based architecture where each AI/ML component (Dashboard, Workbenches, ModelMeshServing, KServe, DataSciencePipelines, CodeFlare, Ray, TrustyAI) is represented as a Go struct implementing the `ComponentInterface`. The operator deploys component manifests from `/opt/manifests` (baked into the container image at build time via `get_all_manifests.sh`) using Kustomize for rendering. It supports three platform modes: OpenDataHub (upstream), Self-Managed RHOAI, and Managed RHOAI (OSD/ROSA addon), with different behaviors for monitoring, networking, and component overlays in each mode.

A `SecretGenerator` controller also runs alongside the main controllers, automatically generating secrets with random data and creating corresponding `OAuthClient` resources for components that need OAuth-based authentication (e.g., the Dashboard). The operator manages Service Mesh (Istio) integration for KServe and configures Knative Serving for serverless model inference.
