## Purpose
**Short**: Kubeflow Pipelines with Tekton backend for building, deploying, and managing portable ML workflows on OpenShift.

**Detailed**: Data Science Pipelines (DSP) is the Red Hat distribution of Kubeflow Pipelines that uses Tekton as the workflow execution engine instead of Argo Workflows. It provides a platform for building and deploying portable, scalable machine learning (ML) workflows on OpenShift/Kubernetes. Users define pipelines using the KFP Python SDK, compile them to Tekton YAML, and upload them to the KFP API server for execution with full logging and artifact tracking.

The system consists of several components: an API server that exposes gRPC and REST APIs for pipeline/run/experiment management, a persistence agent that monitors Tekton PipelineRuns and persists their status to the database, a scheduled workflow controller that manages recurring pipeline runs via the ScheduledWorkflow CRD, a viewer controller for TensorBoard instances via the Viewer CRD, a frontend UI with oauth-proxy authentication, a visualization server, and ML Metadata (MLMD) services for artifact lineage tracking. The backend uses MySQL/MariaDB for metadata storage and S3-compatible object storage (MinIO) for pipeline artifacts.

This is the RHOAI 2.x pattern, using OpenShift Routes with oauth-proxy for ingress and authentication. The deployment manifests in `manifests/opendatahub/` provide kustomize-based deployment configurations with overlays for various components and database backends.
