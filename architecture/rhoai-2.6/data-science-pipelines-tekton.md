## Purpose
**Short**: Kubeflow Pipelines implementation using Tekton as the pipeline execution backend for building and running ML workflows on Kubernetes/OpenShift.

**Detailed**: Data Science Pipelines Tekton (KFP-Tekton) brings together Kubeflow Pipelines and Tekton Pipelines, enabling users to define, compile, upload, and execute machine learning workflows. Users create pipelines using the Kubeflow Pipelines DSL (Python SDK), compile them to Tekton YAML, and submit them via the KFP API server and UI. The system provides pipeline versioning, experiment tracking, run scheduling, artifact management, and metadata lineage tracking.

The platform consists of several microservices: an API server (Go) that provides REST and gRPC endpoints for managing pipelines, experiments, runs, and jobs; a persistence agent that synchronizes pipeline run status from Tekton PipelineRuns back to the API server; a scheduled workflow controller that manages recurring pipeline executions via the ScheduledWorkflow CRD; a pipeline UI (Node.js) with an OAuth proxy for OpenShift authentication; optional MLMD (ML Metadata) services for artifact lineage tracking via Envoy and a gRPC metadata server; a visualization server for rendering pipeline output visualizations; and a PipelineLoop custom Tekton controller for implementing loop constructs in pipelines. This is based on Kubeflow Pipelines 1.8.4 with Tekton >= 0.41.0.
