## Purpose
**Short**: TrustyAI is a Java library and Quarkus REST service providing AI fairness metrics, bias tracking, data drift detection, and model explainability for ML models served via ModelMesh/KServe.

**Detailed**: TrustyAI Explainability is the core library and service component of the TrustyAI platform within RHOAI. It provides a comprehensive suite of Explainable AI (XAI) tools including fairness metrics (Statistical Parity Difference, Disparate Impact Ratio), data drift detection (KS Test, Meanshift, Fourier MMD), and local/global model explainability algorithms (LIME, SHAP, Counterfactual, Partial Dependence Plots, TSSaliency).

The service operates as a Quarkus-based REST application that integrates with ModelMesh and KServe model serving infrastructure. It receives inference payloads (both full payloads and partial input/output payloads) from model serving platforms via HTTP REST endpoints and Knative CloudEvents, reconciles them into complete prediction records, and stores them for ongoing fairness and drift analysis. Computed metrics are exposed via Prometheus for monitoring and alerting, with configurable scheduling for continuous metric evaluation.

The repository is organized as a multi-module Maven project with four primary modules: `explainability-core` (the core XAI algorithms library), `explainability-connectors` (KServe/ModelMesh gRPC integration), `explainability-arrow` (Java-Python data interop via Apache Arrow), and `explainability-service` (the Quarkus REST service deployable on OpenShift/Kubernetes).
