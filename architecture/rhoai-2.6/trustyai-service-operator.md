## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of TrustyAI service instances for AI model explainability, fairness monitoring, and bias detection within OpenShift AI.

**Detailed**: The TrustyAI Service Operator is a controller-runtime based Go operator that watches `TrustyAIService` custom resources and reconciles the full deployment stack needed for AI/ML model monitoring. When a `TrustyAIService` CR is created, the operator provisions a PersistentVolumeClaim for data storage, a Deployment running the TrustyAI service container alongside an OAuth proxy sidecar, Kubernetes Services for both application and OAuth-authenticated traffic, OpenShift Routes for external access, ServiceMonitors for Prometheus metrics collection, and the necessary ServiceAccounts with RBAC bindings.

The operator integrates deeply with the KServe/ModelMesh serving infrastructure. It watches `InferenceService` and `ServingRuntime` resources, automatically configuring payload logging by patching ModelMesh deployments with `MM_PAYLOAD_PROCESSORS` environment variables pointing to the TrustyAI service endpoint, or by setting InferenceLogger specs on KServe InferenceService objects. This enables transparent data capture from model inference requests for downstream bias analysis and explainability computations. The operator uses finalizers to clean up these integrations when a TrustyAIService is deleted.
