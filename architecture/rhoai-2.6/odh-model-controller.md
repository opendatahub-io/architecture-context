## Purpose
**Short**: A Kubernetes controller that manages OpenShift-specific resources (Routes, NetworkPolicies, Service Mesh integration, monitoring) for KServe and ModelMesh InferenceService deployments on RHOAI.

**Detailed**: The ODH Model Controller is a Go-based Kubernetes operator that extends the functionality of KServe and ModelMesh model serving platforms on OpenShift. It watches `InferenceService` custom resources and automatically creates the supporting OpenShift and Istio Service Mesh infrastructure needed for model serving.

For KServe-based InferenceServices, the controller manages OpenShift Routes (pointing to the Istio ingress gateway in `istio-system` namespace), Istio ServiceMeshMemberRoll membership, PeerAuthentication policies (mTLS with port-level exceptions for metrics), Istio Telemetry objects, NetworkPolicies (allowing monitoring, ingress, and cross-namespace traffic), metrics Services, and ServiceMonitors/PodMonitors for Prometheus integration.

For ModelMesh-based InferenceServices (identified by the `serving.kserve.io/deploymentMode: ModelMesh` annotation), the controller manages OpenShift Routes pointing to the `modelmesh-serving` Service, ServiceAccounts (`modelmesh-serving-sa`), and ClusterRoleBindings for auth delegation. The controller also manages a `StorageSecret` reconciler that aggregates OpenDataHub-managed data connection secrets into a unified `storage-config` secret compatible with ModelMesh/KServe storage backends. A monitoring controller manages Prometheus RoleBindings for namespace-level metrics access.

**KServe Route Details**: Routes are created in the `istio-system` namespace, targeting the `istio-ingressgateway` Service. TLS termination is passthrough (when HTTPS URL scheme is configured) or plain HTTP. Route host is derived from the InferenceService `.status.url.Host`. Routes support internal-only visibility via `serving.knative.dev/visibility: cluster-local` label.

**ModelMesh Route Details**: Routes are created in the InferenceService namespace, targeting the `modelmesh-serving` Service. The path is `/v2/models/{isvc-name}`. When auth is enabled (via `enable-auth` annotation on ServingRuntime), the route uses TLS reencrypt to port 8443; otherwise TLS edge to port 8008.
