## Purpose
**Short**: Provides pre-built Jupyter notebook, code-server, RStudio, and Elyra runtime container images for RHOAI workbench environments.

**Detailed**: This repository contains the Dockerfiles, dependency specifications (Pipfile/Pipfile.lock), kustomize test manifests, and OpenShift ImageStream definitions for the complete set of workbench container images used in Red Hat OpenShift AI (RHOAI). These images form a layered build chain starting from Red Hat UBI (Universal Base Image) and CentOS Stream base images, progressing through base Python environments, minimal Jupyter notebooks, data science notebooks (with Elyra pipeline integration), and specialized GPU-accelerated variants for PyTorch, TensorFlow, and Habana Gaudi hardware.

The repository follows a downstream "z-test" pattern for RHOAI 2.6, where images are built from the upstream `red-hat-data-services/notebooks` repository and their digests are tracked via `params.env` and `commit.env` files. A GitHub Actions workflow (`notebook-digest-updater.yaml`) automates the process of fetching the latest image digests from quay.io/modh registries and updating the ImageStream manifests. The manifests folder deploys OpenShift ImageStream resources that the ODH Notebook Controller references when spawning user workbenches.

The images are not operators or services themselves; they are workbench environments launched on-demand by the ODH Notebook Controller. Each notebook image exposes a Jupyter, code-server, or RStudio web UI on port 8888 (notebooks) or via NGINX proxy (code-server), and integrates with the RHOAI platform through Elyra for Data Science Pipelines, the `oc` CLI for OpenShift interaction, and database client libraries for data connectivity.

## ImageStream Versions

Images are versioned with N/N-1/N-2 tags where:
- **N** (2023.2/2023b): Current recommended version
- **N-1** (2023.1/2023a): Previous version
- **N-2** (1.2): Oldest supported version (marked `image-tag-outdated: true`)

Image digests are pinned via `manifests/base/params.env` and build commits tracked in `manifests/base/commit.env`.
