## Purpose
**Short**: Web-based dashboard UI and API backend for managing Open Data Hub / Red Hat OpenShift AI platform resources.

**Detailed**: ODH Dashboard is a full-stack web application providing a user interface for the Open Data Hub (ODH) and Red Hat OpenShift AI (RHOAI) platforms. It consists of a Node.js/Fastify backend that serves as both a static file server for the React frontend and an API proxy to the Kubernetes API server, and a React/PatternFly frontend that provides the user experience.

The dashboard enables users to manage data science workbenches (Jupyter notebooks), explore and enable platform applications, manage model serving runtimes, configure accelerator profiles (GPU), manage project namespaces, configure cluster settings, and access learning resources (quickstarts, documentation). The backend authenticates users via an OpenShift OAuth proxy sidecar and proxies Kubernetes API calls using the user's forwarded access token, enforcing namespace-scoped security for resource access. It also watches key platform CRDs (OdhApplication, OdhDocument, OdhQuickStart, OdhDashboardConfig, AcceleratorProfile) and provides consolidated API endpoints for the frontend.

The application is deployed as a Kubernetes Deployment with two containers: the dashboard application (port 8080) and an oauth-proxy sidecar (port 8443) that handles TLS termination and OpenShift OAuth authentication. External access is provided via an OpenShift Route with TLS reencrypt termination.

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| odh-dashboard | Route (OpenShift) | Auto-generated | 443/TCP | HTTPS | TLS (reencrypt) | Reencrypt with HSTS | External |

**Authentication Flow**: External HTTPS traffic arrives at the OpenShift Route, which performs TLS reencrypt to the oauth-proxy sidecar on port 8443. The oauth-proxy handles OpenShift OAuth authentication (using `--provider=openshift`), enforces that the user has `get` access to the `odh-dashboard` service in the deployment namespace (`--openshift-delegate-urls`), sets a session cookie (23h expiry), and forwards the access token to the backend via the `x-forwarded-access-token` header. The backend on port 8080 receives plaintext HTTP from the oauth-proxy with the user's token available for impersonated K8s API calls.
