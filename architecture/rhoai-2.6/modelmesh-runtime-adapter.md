## Purpose
**Short**: Sidecar container that acts as an intermediary between ModelMesh and third-party model-serving runtimes, handling model storage retrieval and runtime-specific model layout adaptation.

**Detailed**: The modelmesh-runtime-adapter is a multi-purpose sidecar container image that runs within ModelMesh Serving model server Pods. It contains two main functional components: a **model-serving-puller** that downloads model artifacts from various cloud storage backends (S3, GCS, Azure Blob, HTTP, PVC) to local storage, and **runtime-specific adapters** that translate the ModelMesh internal gRPC `ModelRuntime` protocol into the native APIs of supported model serving runtimes.

The repository builds five separate Go binaries packaged into a single container image: the puller, and four runtime adapters for NVIDIA Triton Inference Server, Seldon MLServer, OpenVINO Model Server (OVMS), and TorchServe. Each adapter implements the `mmesh.ModelRuntime` gRPC service, intercepting model lifecycle requests (load, unload, runtime status) from ModelMesh. On model load, the adapter pulls model files from remote storage, adapts the model file layout to match the target runtime's expected directory structure, and then delegates to the runtime's native API for actual model loading.

The adapters also handle memory capacity management, reporting model sizes and runtime capabilities back to ModelMesh. The "pullman" library provides a pluggable storage provider architecture with built-in support for S3-compatible storage (via IBM COS SDK), Google Cloud Storage, Azure Blob Storage, HTTP(S) endpoints, and Kubernetes PersistentVolumeClaims.
