## Purpose
**Short**: ModelMesh is a distributed LRU cache and routing layer for high-scale, high-density model serving that acts as a sidecar alongside model runtime containers.

**Detailed**: ModelMesh is a mature, general-purpose model serving management and routing framework originally developed by IBM Watson and contributed to the KServe project. It provides intelligent model placement, caching, and load balancing across a cluster of model serving pods. The framework operates as a sidecar container alongside custom or existing model runtime containers, communicating with them via a gRPC interface (ModelRuntime) for model lifecycle management (load, unload, size) and forwarding inference requests to the appropriate pod where the target model is loaded.

The framework uses etcd as its distributed coordination store for model registry, instance tracking, leader election, and dynamic configuration. It implements a sophisticated distributed LRU cache that automatically loads, unloads, and migrates models across instances based on demand, available capacity, and type constraints. ModelMesh supports both gRPC and Thrift (via litelinks) for internal inter-pod communication, with optional TLS/mTLS for securing both external client-facing and internal cluster traffic.

In the RHOAI context, ModelMesh is deployed as part of the ModelMesh Serving stack (modelmesh-serving), where it provides the core model routing and caching layer for KServe/InferenceService workloads. The upstream version is v0.11.0 (kserve/modelmesh release-0.11.0 branch).
