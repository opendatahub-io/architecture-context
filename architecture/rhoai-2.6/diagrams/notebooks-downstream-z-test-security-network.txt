RBAC Summary:
- No RBAC resources defined in this repository
- RBAC for notebook workbenches is managed by the ODH Notebook Controller
  and the platform operator (rhods-operator)

Authentication Architecture:
- JupyterLab UI: OAuth via oauth-proxy sidecar (injected by notebook controller)
- Jupyter API: Token authentication disabled (--ServerApp.token='' --ServerApp.password='')
- Auth delegation: All authentication handled by oauth-proxy before reaching notebook
- OAuth Provider: OpenShift OAuth server

Secrets:
- No secrets defined in this repository
- Authentication secrets (oauth-proxy, TLS certificates) are managed by
  the ODH Notebook Controller when creating workbench pods

Services:
- {notebook-name}-notebook (ClusterIP)
  - Port: 8888/TCP → notebook-port (8888)
  - Protocol: HTTP
  - Encryption: None (platform-managed via oauth-proxy sidecar)
  - Exposure: Internal only

Network Policy Notes:
- Notebook images do not define NetworkPolicy resources
- Network isolation is managed at the platform/namespace level
- Egress to external databases and registries is user-configurable
