RBAC Summary:
- This component does not define ClusterRoles or RoleBindings
- Notebook Controller manages RBAC for spawned notebook pods
- User permissions: Users must own workbench or have namespace edit permissions
- ServiceAccount: {workbench-name}-sa created per workbench

Service Mesh Configuration:
- PeerAuthentication: Optional (not enforced by notebooks component)
- AuthorizationPolicy: Managed by Notebook Controller
- mTLS Mode: PERMISSIVE (allows both plaintext and mTLS)

Secrets:
- {workbench}-notebook-token (Opaque)
  - Content: Jupyter authentication token
  - Provisioned By: Notebook Controller
  - Rotation: Manual (no auto-rotate)
  - Mounted At: /opt/app-root/src/.jupyter/

- aws-connection-{name} (Opaque)
  - Content: S3 credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
  - Provisioned By: User or Dashboard
  - Rotation: Manual
  - Mounted At: /opt/app-root/src/.aws/

- generic-s3-secret (Opaque)
  - Content: Default S3 storage credentials
  - Provisioned By: User or Dashboard
  - Rotation: Manual
  - Mounted At: /opt/app-root/src/.aws/

Container Security:
- User: UID 1001, GID 0 (non-root)
- SELinux Context: container_t (OpenShift default)
- Read-only Root Filesystem: No (Jupyter requires writable /opt/app-root for pip)
- Capabilities: Default (no special capabilities required)
- Security Context Constraints (SCC): restricted-v2 (OpenShift default)

Authentication & Authorization:
1. External Access:
   - Mechanism: OAuth Bearer Token (JWT) or Session Cookie
   - Enforcement: OpenShift OAuth Proxy or Istio AuthorizationPolicy
   - Policy: User must own workbench or have namespace edit permissions

2. Jupyter API Access:
   - Mechanism: Token parameter or Authorization header
   - Enforcement: JupyterLab ServerApp
   - Policy: Token match or empty token (if auth disabled)
   - Default: Token authentication typically disabled in RHOAI (OAuth handles auth)

Image Distribution:
- Registry: quay.io/modh/* (RHOAI managed registry)
- ImageStream: Deployed by opendatahub-operator or RHOAI operator
- Versioning: Release-based tags (2023.2, 2023.1, 1.2 = N, N-1, N-2)
- Digest References: params.env uses SHA256 digests for immutable pulls

Environment Variables (Security-relevant):
- NOTEBOOK_PORT: 8888 (JupyterLab server port)
- NOTEBOOK_BASE_URL: / (base URL path for Jupyter)
- NOTEBOOK_ARGS: Additional Jupyter server arguments
- NVIDIA_VISIBLE_DEVICES: all (GPU device visibility - CUDA images)
- NVIDIA_DRIVER_CAPABILITIES: compute,utility (GPU capabilities - CUDA images)

Jupyter Server Security Configuration:
- ServerApp.token: Typically disabled (empty) in RHOAI (OAuth handles auth)
- ServerApp.password: Disabled (empty string)
- ServerApp.allow_origin: "*" (required for OAuth proxy compatibility)
- ServerApp.quit_button: False (prevents accidental shutdown)
- ServerApp.allow_remote_access: True
