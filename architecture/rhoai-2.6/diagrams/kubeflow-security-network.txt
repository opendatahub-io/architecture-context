Authentication & Authorization Flows:
──────────────────────────────────────

Webhook Endpoint (/mutate-notebook-v1):
  - Auth: mTLS (Kubernetes API server client certificate)
  - Enforcement: Kubernetes API server validates client cert
  - Policy: Only API server can call webhook

Notebook Access (with OAuth annotation):
  - Auth: OpenShift OAuth (redirects to OAuth server)
  - Authorization: SubjectAccessReview (user must have GET permission on Notebook CR)
  - Enforcement: OAuth Proxy Sidecar
  - Session: Cookie-based (secret in {notebook-name}-oauth-config)

Notebook Access (without OAuth annotation):
  - Auth: None
  - Exposure: Publicly accessible via Route (edge TLS termination)
  - Risk: No authentication - use for public demos only

Metrics Endpoint (/metrics):
  - Auth: None
  - Exposure: Internal only (ClusterIP service, no Route)
  - Access: Prometheus scraping from cluster network
