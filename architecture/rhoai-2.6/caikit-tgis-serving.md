## Purpose
**Short**: A container image that combines Caikit (AI toolkit) with TGIS (Text Generation Inference Server) to enable Large Language Model (LLM) inference on the RHOAI/ODH platform via KServe.

**Detailed**: Caikit-TGIS-Serving is a serving runtime container image designed for Large Language Model inference within the Red Hat OpenShift AI (RHOAI) platform. It is not an operator or controller; rather, it is a Python-based container image that gets deployed as a KServe `ServingRuntime` resource within a Knative + Istio service mesh environment.

The stack consists of two containers running as a pod: (1) TGIS (Text Generation Inference Server), which is the backend that loads and serves the actual model weights, and (2) Caikit, a Python wrapper that provides standardized HTTP (port 8080) and gRPC (port 8085) inference endpoints. Caikit communicates with the co-located TGIS process over localhost gRPC on port 8033. The entire stack is orchestrated by KServe `InferenceService` custom resources, with networking handled by Istio service mesh and Knative Serving for serverless scaling.

This repository primarily contains the Dockerfile for building the caikit-tgis-serving container image, configuration files, demo manifests for deploying the stack on OpenShift with KServe, and test utilities. It does not contain operator or controller code.
