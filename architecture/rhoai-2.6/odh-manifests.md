## Purpose
**Short**: A collection of Kustomize manifests that define the deployment configuration for all Open Data Hub / RHOAI platform components.

**Detailed**: The odh-manifests repository is a centralized collection of Kubernetes/OpenShift Kustomize manifests used to deploy the full suite of AI/ML platform components that make up Red Hat OpenShift AI (RHOAI). It does not contain application source code; instead, it packages CRD definitions, RBAC configurations, Deployment/StatefulSet specs, Service definitions, Route configurations, NetworkPolicies, and operator deployment manifests for each platform component.

The repository is consumed at deployment time by the KfDef operator (or rhods-operator), which reads KfDef custom resources that reference specific paths within this manifest collection. Each top-level directory corresponds to a platform component (e.g., `odh-dashboard`, `odh-notebook-controller`, `data-science-pipelines-operator`, `model-mesh`, `kserve`, `trustyai-service-operator`). Components use the Kustomize base/overlay pattern, with RHOAI-specific overlays in `overlays/rhods/` directories. This is a RHOAI 2.x era repository (branch `rhoai-2.6`), using the legacy Route + oauth-proxy ingress pattern for external access.

The repository is packaged into a container image (using `Dockerfile`) that bundles all manifests as a tarball at `/opt/odh-manifests.tar.gz`, which the operator extracts and applies during component reconciliation.
