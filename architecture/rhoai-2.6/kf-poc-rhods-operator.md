## Purpose
**Short**: The RHODS Operator is the central platform operator for Red Hat OpenShift AI (RHOAI), responsible for deploying and managing all data science components on OpenShift clusters.

**Detailed**: This operator serves as the orchestration layer for the RHOAI platform (version 2.x series). It provides two primary Custom Resources: `DSCInitialization` for platform-level initialization (namespace creation, monitoring stack, service mesh configuration, network policies) and `DataScienceCluster` for enabling/disabling individual data science components (Dashboard, Workbenches, KServe, ModelMesh, Data Science Pipelines, CodeFlare, Ray, TrustyAI).

The operator uses a component-based architecture where each data science component implements a `ComponentInterface`. During reconciliation, the `DataScienceCluster` controller iterates through all components, deploying manifests from `/opt/manifests/` using Kustomize. The operator supports three platform modes: OpenDataHub (community), Self-Managed RHOAI, and Managed RHOAI (addon), with platform-specific configurations and monitoring stacks. Service Mesh integration (Istio/Maistra) is managed via the Feature API for KServe serverless serving deployments.

The operator also includes a `SecretGenerator` controller that watches for annotated secrets and generates random secret values, optionally creating OpenShift OAuthClient resources for component authentication flows (e.g., dashboard OAuth).
