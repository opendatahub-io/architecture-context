## Purpose
**Short**: A Kubernetes operator that deploys and manages Data Science Pipelines (Kubeflow Pipelines) application stacks on OpenShift, including the API server, persistence agent, scheduled workflow controller, database, object storage, and ML metadata services.

**Detailed**: The Data Science Pipelines Operator (DSPO) provides a declarative, custom resource-driven approach to deploying and managing complete Kubeflow Pipelines (KFP) stacks on OpenShift. Users create a `DataSciencePipelinesApplication` (DSPA) custom resource specifying their desired pipeline configuration, and the operator reconciles the full set of sub-components needed: the KFP API server (using Tekton as the pipeline runtime), a persistence agent for recording pipeline run metadata, a scheduled workflow controller for cron-based pipeline runs, a database backend (MariaDB by default or external SQL), object storage (Minio by default or external S3-compatible), and optionally ML Metadata (MLMD) services with Envoy proxy.

The operator performs health checks against both the database and object storage before deploying pipeline components, and exposes Prometheus metrics for monitoring component readiness. It manages OpenShift Routes with OAuth proxy sidecars for secure external access to the API server, deploys NetworkPolicies to restrict inter-component traffic, and handles credential lifecycle management (auto-generating secrets when not user-provided). Each DSPA instance creates a fully isolated pipeline stack within its namespace, enabling multi-tenant pipeline deployments.
