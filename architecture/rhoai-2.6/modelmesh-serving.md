## Purpose
**Short**: Kubernetes controller that manages the lifecycle of ModelMesh model serving runtimes and predictors for multi-model serving.

**Detailed**: ModelMesh Serving is a Kubernetes operator that provides a scalable, intelligent model serving infrastructure built on top of the ModelMesh framework. It reconciles KServe-compatible Custom Resources (ServingRuntime, ClusterServingRuntime, Predictor, InferenceService) to dynamically create and manage Deployment workloads that host ML model inference servers with the ModelMesh sidecar for intelligent model routing and caching.

The controller manages the complete lifecycle of model serving deployments: it watches for ServingRuntime CRs, constructs multi-container Deployments (including the ModelMesh sidecar, optional REST proxy, optional storage helper/puller, and an oauth-proxy sidecar for authentication), and manages the Kubernetes Service resources for inference traffic routing. It communicates with ModelMesh via gRPC to register/unregister virtual models (VModels) and track model loading states. The controller supports both cluster-scoped and namespace-scoped operation, scale-to-zero when no predictors reference a runtime, HPA-based autoscaling, and TLS/mTLS for inference traffic.

The operator uses etcd (via a shared secret) as the distributed coordination backend for the ModelMesh framework, enabling model placement decisions, load balancing, and model state tracking across runtime pods. It also integrates with Prometheus for metrics collection and supports validating webhooks for ServingRuntime spec validation.
