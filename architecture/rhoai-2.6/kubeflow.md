## Purpose
**Short**: Manages the lifecycle of Jupyter Notebook servers on OpenShift/Kubernetes by reconciling Notebook CRs into StatefulSets, Services, Routes, and OAuth proxy sidecars.

**Detailed**: This repository contains two complementary Go controllers that together manage notebook workloads for RHOAI:

1. **KF Notebook Controller** (`components/notebook-controller`): The upstream Kubeflow notebook controller that watches `Notebook` CRDs (kubeflow.org) and reconciles them into StatefulSets, Services, and optionally Istio VirtualServices. It also includes a culling controller that monitors notebook kernel/terminal activity via the Jupyter API and stops idle notebooks by setting the `kubeflow-resource-stopped` annotation.

2. **ODH Notebook Controller** (`components/odh-notebook-controller`): An OpenShift-specific extension controller that adds RHOAI/ODH functionality on top of the KF controller. It creates OpenShift Routes (TLS edge or reencrypt), injects OAuth proxy sidecars for authentication, manages per-notebook NetworkPolicies, creates ServiceAccounts for OAuth, and handles trusted CA bundle injection for cluster-wide proxy support. It also provides a mutating webhook that injects the OAuth proxy sidecar container and reconciliation lock annotation on notebook creation.

In RHOAI 2.6, notebooks are exposed via **OpenShift Routes** with **oauth-proxy** sidecars for authentication (RHOAI 2.x pattern). The ODH controller creates a Route per notebook, an OAuth Service (port 443) pointing to the oauth-proxy sidecar (port 8443), and NetworkPolicies restricting ingress.

### Sidecar Containers (Injected by Webhook)

| Sidecar | Port | Protocol | Upstream | Auth Mechanism | Purpose |
|---------|------|----------|----------|----------------|---------|
| oauth-proxy | 8443/TCP | HTTPS | http://localhost:8888 | OpenShift OAuth + SAR check | Authenticates users via OpenShift OAuth, enforces RBAC via SubjectAccessReview on Notebook resource |

The oauth-proxy sidecar enforces authorization with an OpenShift SAR check: `{"verb":"get","resource":"notebooks","resourceAPIGroup":"kubeflow.org","resourceName":"{notebook-name}","namespace":"{namespace}"}`.
