## Purpose
**Short**: Provides pre-built container images for Jupyter notebooks, code-server, RStudio, and pipeline runtime environments used as workbenches in Red Hat OpenShift AI (RHOAI).

**Detailed**: This repository contains the Dockerfiles, dependency specifications (Pipfile/Pipfile.lock), build automation (Makefile), and deployment manifests for all notebook workbench images used by RHOAI. The images are organized in a layered build chain: base images (UBI8/UBI9/C9S with Python) serve as foundations, on which CUDA GPU support, Jupyter notebook environments (minimal, data science, PyTorch, TensorFlow, TrustyAI, HabanaAI), code-server (VS Code in browser), RStudio, and pipeline runtime images are built.

The repository also contains Kustomize manifests that define OpenShift ImageStream resources. These ImageStreams register the pre-built notebook images with the OpenShift cluster, making them available for selection by users through the ODH Dashboard / RHOAI workbench UI. The notebook controller (in a separate repository) uses these ImageStreams to launch notebook pods. Each ImageStream carries metadata annotations describing the software versions, Python dependencies, recommended accelerators, and build commit hashes for traceability.

This is NOT an operator or controller — it is a container image build repository whose outputs (container images) are consumed by the ODH Notebook Controller and referenced via OpenShift ImageStreams deployed by Kustomize manifests.

## Network Architecture

### Services

Notebook pods are created dynamically by the ODH Notebook Controller. Each notebook pod gets a Service:

| Service Name | Type | Port | Target Port | Protocol | Encryption | Auth | Exposure |
|--------------|------|------|-------------|----------|------------|------|----------|
| {notebook-name}-notebook | ClusterIP | 8888/TCP | 8888 | HTTP | plaintext | None (in-cluster) | Internal |

Note: For code-server workbenches, the container listens on port 8585/TCP (NGINX proxy to code-server). For RStudio workbenches, the container listens on port 8787/TCP (NGINX proxy to RStudio Server).

### Ingress

- **RHOAI 2.x pattern**: OpenShift Route + oauth-proxy sidecar for each notebook pod

| (managed by notebook controller) | Route (OpenShift) | {notebook}-{namespace}.apps.{cluster} | 443/TCP | HTTPS | TLS (edge termination) | Edge | External |

### Image Versioning

Images follow an N/N-1/N-2 versioning scheme:
- **N (2023.2)**: Current recommended version, marked with `opendatahub.io/workbench-image-recommended: 'true'`
- **N-1 (2023.1)**: Previous version, still available
- **N-2 (1.2)**: Oldest supported, marked with `opendatahub.io/image-tag-outdated: 'true'`

All images are pinned by SHA256 digest in `manifests/base/params.env` for reproducibility.
