Secrets:
- batch-gateway-secrets (Opaque)
  - redis-url: Redis connection URL
  - postgresql-url: PostgreSQL connection URL
  - inference-api-key: Bearer token for LLM inference gateway
  - s3-secret-access-key: AWS/S3 secret access key
  - Provisioned by: Manual (kubectl create secret)
  - Auto-Rotate: No

- {release}-apiserver-tls (kubernetes.io/tls)
  - tls.crt: TLS certificate for API server HTTPS
  - tls.key: TLS private key
  - Provisioned by: cert-manager (optional) or manual
  - Auto-Rotate: Yes (if cert-manager)

RBAC Summary:
  No Kubernetes RBAC roles defined. The batch-gateway does not interact with the
  Kubernetes API directly. Service accounts are created for each component but no
  ClusterRole/Role bindings are configured.

Authentication & Authorization:
  - /v1/batches/*: Tenant header (X-MaaS-Username) extracted by middleware
    - Last header value used (Envoy ext_authz compatible)
    - DB queries scoped by tenantID
    - Post-retrieval validation returns 404 on mismatch (prevents enumeration)
  - /v1/files/*: Same tenant isolation pattern
    - S3 paths use SHA256(tenantID) as folder prefix
  - /health, /ready, /metrics: No auth (separate ports 8081/9090)
  - Inference Gateway: Bearer token (Authorization header) + pass-through headers

TLS Configuration:
  - API Server: Optional TLS via ssl_cert_file/ssl_key_file or cert-manager
  - MinVersion: TLS 1.2 (crypto/tls)
  - Redis: TLS optional (connection URL scheme), mTLS supported
  - S3: TLS 1.2+ (standard HTTPS)
  - Inference Gateway: TLS 1.2+ (mTLS supported, InsecureSkipVerify configurable)
  - Custom CA bundle support (RootCAs), mTLS client certs (LoadX509KeyPair)

FIPS Compliance:
  - Build: CGO_ENABLED=0 (static linking, pure-Go crypto) — NOT FIPS compliant by default
  - CGO_ENABLED declared as ARG, can be overridden at build time
  - No GOEXPERIMENT=strictfipsruntime flag present
  - No OLM FIPS annotation
  - Crypto: stdlib crypto/tls only, no external crypto libraries
  - Risk: InsecureSkipVerify configurable (testing only)
