Secrets:
- feast-{name}-intra-communication (Opaque/ConfigMap) - HMAC-SHA256 JWT signing key, operator-generated, no auto-rotate
- TLS serving cert (kubernetes.io/tls) - Service TLS, OpenShift auto-provisioned, auto-rotated
- OIDC secret (Opaque) - OIDC client credentials, user-provisioned, no auto-rotate

Authentication & Authorization:
- Feature Server endpoints: Bearer Token (JWT via OIDC or K8s SA), enforced by FastAPI middleware (inject_user_details)
- Feast Permission RBAC: READ_ONLINE, WRITE_ONLINE, WRITE_OFFLINE, READ, DESCRIBE actions
- Intra-service auth: HMAC-SHA256 signed JWT with empty key, bypasses per-user checks
- /health endpoint: Open access (no auth)
- Fail-secure default: No permissions defined = deny all

FIPS Compliance:
- Operator: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, UBI9 base (OpenSSL)
- Feature Server (Python): Inherits FIPS from UBI9 Python base OpenSSL
- Go Feature Server: MinVersion: tls.VersionTLS12
- Risk: InsecureSkipVerify=true for operator→registry intra-cluster calls
