================================================================================
 NETWORK SUMMARY
================================================================================

INGRESS:
  - NONE. Neither the data container nor the kafka-processor expose any
    network services or accept inbound connections.

EGRESS:
  - kafka-processor ONLY (offline, not deployed to clusters):
    - Kafka broker: 9092/TCP (dev), 9096/TCP (prod)
    - Protocol: Kafka over TLS (SASL_SSL)
    - Auth: SCRAM-SHA-512
    - Optional mTLS: SSL_CERTIFICATE_LOCATION + SSL_KEY_LOCATION

  - Data container (runtime):
    - NONE. No outbound network connections.

================================================================================
 RBAC SUMMARY
================================================================================

  - No RBAC resources defined by this component.
  - Data container inherits the ServiceAccount of the model-catalog Deployment.
  - No ClusterRoles, Roles, or RoleBindings created.

================================================================================
 SECRETS
================================================================================

  Secret                                  Type        Purpose                           Provisioned By       Rotation
  ──────────────────────────────────────  ──────────  ────────────────────────────────  ───────────────────  ────────
  Kafka creds (SASL_USERNAME/PASSWORD)    Opaque      Auth to internal Kafka cluster    Manual               No
  SSL certs (CA, cert, key)              TLS files   mTLS auth to Kafka cluster        Manual               No

  NOTE: These secrets are used by kafka-processor ONLY (offline tool).
        The runtime data container has NO secrets.

================================================================================
 FIPS COMPLIANCE
================================================================================

  - Data container: No crypto operations. FIPS inherited from UBI9-minimal
    base image (system OpenSSL).
  - kafka-processor: Not shipped to clusters. TLS delegated to librdkafka
    which uses system OpenSSL. No non-FIPS crypto risks detected.

================================================================================
 CONFIDENTIALITY NOTE
================================================================================

  The benchmark data is CONFIDENTIAL. The repository must remain private.
  The container image should be distributed only through authorized registries.
  Data includes proprietary performance benchmarks for Red Hat-validated AI
  models tested against specific RHOAI/vLLM versions on specific hardware.
