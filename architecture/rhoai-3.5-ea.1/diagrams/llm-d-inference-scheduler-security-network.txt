SSRF Protection:
- Sidecar watches InferencePool resources
- Maintains allowlist of pod IPs from pool
- Validates prefiller/encoder targets from request headers against allowlist
- Prevents header-injection attacks redirecting traffic to arbitrary hosts
- Enabled via --enable-ssrf-protection flag

Security Risks / Notes:
- gRPC ext-proc (9002/TCP): No encryption, relies on cluster network isolation
- ZMQ connections (5555-5557/TCP): No encryption, no authentication
- OTLP exporter: Uses insecure gRPC by default
- InsecureSkipVerify: Configurable per P/D stage — potential MITM if enabled
- Gateway ingress: HTTP only (port 80), no TLS termination at gateway level
- SSRF protection is opt-in (--enable-ssrf-protection), not enabled by default
