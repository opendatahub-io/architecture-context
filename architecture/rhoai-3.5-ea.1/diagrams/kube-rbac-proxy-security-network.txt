RBAC Summary:
- Required ClusterRole permissions:
  - API Group: authentication.k8s.io
    - Resource: tokenreviews
    - Verbs: create
  - API Group: authorization.k8s.io
    - Resource: subjectaccessreviews
    - Verbs: create
- Provisioned by: rhods-operator (per-component RBAC)
- ServiceAccount: Inherits from parent component Pod

Hardcoded Authorization:
- SA: system:serviceaccount:openshift-monitoring:prometheus-k8s
  - Allowed: GET /metrics (unconditional, no SAR check)
  - Purpose: OpenShift Monitoring Prometheus metrics scraping
  - Note: Planned removal when Cluster Monitoring Operator adds static authorizer

Secrets:
- TLS cert/key (kubernetes.io/tls) - Server TLS, cert-manager, hot-reload 1min poll
- Client CA (Opaque, optional) - mTLS client verification, DynamicFileCAContent reload
- Upstream client cert (kubernetes.io/tls, optional) - Proxy-to-upstream mTLS, no auto-rotate
- OIDC CA (Opaque, optional) - OIDC issuer TLS verification, DynamicFileCAContent reload

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
- Runtime: stdlib crypto/tls via OpenSSL (UBI 9 base image)
- TLS: Configurable min version (default TLS 1.2), Go default cipher suites
- No non-FIPS crypto detected (no MD5, RC4, DES usage)

Authentication Methods:
1. Bearer Token → Kubernetes TokenReview API (delegation)
2. mTLS Client Certificate → Validated against --client-ca-file CA bundle
3. OIDC JWT → Validated against --oidc-issuer discovery/JWKS

Authorization Methods:
1. Kubernetes SubjectAccessReview → Format1 (top-level) or Format2 (path-scoped)
2. Hardcoded Metrics Authorizer → prometheus-k8s SA + GET /metrics
3. Path Bypass → --ignore-paths (no authn/authz, proxy directly)
