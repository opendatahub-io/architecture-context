FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, UBI9 base
- Runtime: OpenSSL via UBI9 ubi-minimal
- TLS: OpenShift serving certificates (no custom cipher suite config)
- No non-FIPS crypto detected (InsecureSkipVerify only in test files)

Authentication Flow:
1. Client sends Bearer token to OpenShift Router (443/TCP)
2. Router forwards to kube-rbac-proxy (8443/TCP, reencrypt TLS)
3. kube-rbac-proxy performs SubjectAccessReview against K8s API
   - Checks: user can "get" the registry/catalog Service resource
4. If authorized, proxies to rest-container (8080/TCP, localhost HTTP)
5. rest-container queries database (5432/TCP or 3306/TCP)
