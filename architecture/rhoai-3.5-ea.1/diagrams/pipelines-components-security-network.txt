RBAC Summary:
  No dedicated RBAC manifests. Components rely on:
  - KFP pipeline service account (injected at runtime)
  - Kubernetes bearer tokens from environment variables (not in-cluster SA mount)
  - Individual secret injection via kfp.kubernetes.use_secret_as_env()

Secrets Summary:
  - s3-secret (Opaque) - AWS S3 credentials, User/RHOAI Connections, No rotation
  - hf-token (Opaque) - HuggingFace Hub auth, User-provisioned, No rotation
  - kubernetes-credentials (Opaque) - K8s API access, Platform/SA, No rotation
  - oci-pull-secret-model-download (Opaque) - OCI registry auth, User, No rotation
  - ogx-secret (Opaque) - OGX API credentials, User/RHOAI Connections, No rotation
  - llm-credentials (Opaque) - LLM API key, User-provisioned, No rotation
  ⚠ No secrets have auto-rotation configured

Security Observations:
  1. Model Registry uses insecure HTTP (is_secure=False) - no TLS, no auth
  2. AutoRAG components implement SSL verify=False fallback for self-signed certs
  3. OGX_TLS_INSECURE flag available for dev environments (weakens TLS validation)
  4. Kubernetes auth via env var bearer tokens (not in-cluster SA mount)
  5. All secrets provisioned without auto-rotation
  6. Path traversal protection in init container (is_relative_to() validation)

Certificate Handling:
  - System trust store by default (UBI9/RHEL9 OpenSSL)
  - Custom CA: REQUESTS_CA_BUNDLE / SSL_CERT_FILE env vars
  - OGX_TLS_INSECURE=1 for dev environments
  - No FIPS-violating crypto, but SSL verify=False weakens validation
