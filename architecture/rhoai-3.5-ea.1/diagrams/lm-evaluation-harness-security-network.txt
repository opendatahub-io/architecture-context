RBAC Summary:
- No RBAC definitions in this repository
- RBAC is managed by the TrustyAI LMEval Operator that creates the Job pods
- Pod runs as non-root user 65532:65532

Secrets:
- Model auth secret (model.auth.secret_ref):
  - Type: Opaque
  - Keys: api_key, Authorization, hf-token (optional), ca_cert (optional)
  - Provisioned by: User / TrustyAI Operator
  - Auto-rotate: No
- Registry credentials:
  - Type: dockerconfigjson or username/password
  - Purpose: OCI registry auth for artifact push
  - Provisioned by: User / Platform
  - Auto-rotate: No

Security Controls:
- Credential sanitization: Bearer tokens, API keys, passwords, URL credentials
  redacted from error messages before EvalHub callbacks (_sanitize_error_message)
- Path traversal protection: Job spec reads confined to /meta/ directory
  (_resolve_job_spec_path_for_read) - CWE-22 mitigation
- Symlink rejection: CA bundle paths checked with raw_path.is_symlink()
  - CWE-61 mitigation
- Container: non-root user 65532:65532, no privilege escalation
- TLS: System OpenSSL via UBI9 base; ssl.create_default_context() with
  optional custom CA bundle; verify_certificate parameter support
- Crypto: Only FIPS-approved algorithms (SHA-256 for hashing, system TLS)
- No inbound network services (egress-only batch job)

FIPS Compliance:
- Base image: UBI9 (ubi9/python-311) with system OpenSSL
- Crypto: hashlib.sha256 (FIPS-approved), Python ssl module (system OpenSSL)
- No non-FIPS crypto detected (no MD5, RC4, DES, custom RNG)
- Build: hermetic: false (EA release), poetry.lock fully pinned
