RBAC Summary:
- No RBAC roles or bindings. This component is a model server container
  that does not access the Kubernetes API. RBAC is managed by KServe controller.

Secrets:
- (storage credentials) (Opaque) - S3/GCS/Azure/HF credentials for downloading
  model artifacts. Provisioned by user/platform admin. No auto-rotation.
- Mounted into pod by KServe controller based on InferenceService storageUri
  and ServiceAccount annotations.

Authentication & Authorization:
- No application-level authentication or authorization
- All auth enforced by RHOAI platform ingress stack:
  - Gateway API + kube-rbac-proxy for external access
  - Istio mTLS for service-to-service within mesh
- Endpoint auth: None at server level, platform-enforced at ingress

- FIPS compliance inherited from base image OpenSSL configuration
