RBAC Summary:
- RBAC is managed externally by rhods-operator (no RBAC resources in MLflow repo)
- MLflow ServiceAccount reads SA token from:
  /var/run/secrets/kubernetes.io/serviceaccount/token
- kubernetes-auth plugin provides permission-based authorization:
  - Permission levels: READ, EDIT, MANAGE (per resource type)
  - Resources: Experiments, Runs, Registered Models, Model Versions
  - Workspace scoping via ContextVar (per-request isolation)

Authentication Stack:
1. Platform Gateway: TLS termination (SIMPLE mode, TLS 1.2+)
2. kube-rbac-proxy sidecar: TokenReview validation against K8s API
3. MLflow kubernetes-auth plugin: Permission-based authorization
   - Reads SA token with 60-second TTL cache
   - Maps K8s namespace → MLflow workspace
4. CORSBlockingMiddleware: Blocks cross-origin state-changing methods
5. HostValidationMiddleware: DNS rebinding protection (private IPs + localhost)

Secrets:
- ServiceAccount token (projected volume) — auto-rotated by kubelet
- MLFLOW_CRYPTO_KEK_PASSPHRASE (env var from Secret) — KEK for envelope encryption, manual rotation via `mlflow crypto rotate-kek`
- MLFLOW_FLASK_SERVER_SECRET_KEY (env var from Secret) — Flask session signing, manual rotation
- Database connection URI (env var from Secret) — PostgreSQL credentials, managed by platform operator

FIPS Compliance:
- Python cryptography library delegates to system OpenSSL (UBI9)
- FIPS mode inherited from RHEL host when enabled
- MD5/SHA1 used with usedforsecurity=False (FIPS-permitted)
- AES-256-GCM for envelope encryption (FIPS-approved algorithm)

Build Hermeticity:
- ODH pipeline: hermetic=true
- RHOAI pipeline: hermetic=false (yarn/npm prefetch limitations)
- Cachi2/Hermeto prefetch for pip, yarn, RPM dependencies
- All Python deps use --require-hashes for integrity verification
- Lock files: rpms.lock.yaml, uv.lock, yarn.lock, konflux-*-requirements.txt
