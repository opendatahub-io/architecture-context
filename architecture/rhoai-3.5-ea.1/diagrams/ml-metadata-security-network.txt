INGRESS:
- No Ingress resources. MLMD is an internal-only service.
- Exposed via ClusterIP service on port 8080/TCP.
- Not accessible from outside the cluster.

RBAC Summary:
- No Kubernetes RBAC resources defined in this repository.
- RBAC is managed by the deploying operator (DSP Operator / rhods-operator).
- Service account and role bindings are provisioned externally.

Secrets:
- DB credentials (Opaque) - MySQL/PostgreSQL username and password
  Provisioned by: Platform operator or DSP operator
  Auto-Rotate: No

- SSL certificates (kubernetes.io/tls) - Optional mTLS between gRPC clients and server
  Provisioned by: Manual or cert-manager
  Auto-Rotate: Depends on deployment

- DB SSL certificates (Opaque) - TLS for database connections (sslcert, sslkey, sslrootcert)
  Provisioned by: Manual or platform operator
  Auto-Rotate: Depends on deployment

Authentication & Authorization:
- Default mode: NO authentication — insecure gRPC when SSLConfig is not configured
- Optional mTLS: When SSLConfig.client_verify=true, server requires and verifies client certs
- Enforcement: gRPC server (BoringSSL TLS stack)
- No application-level authorization — all authenticated clients have full API access

FIPS Compliance Notes:
- C++ binary links against BoringSSL (Bazel-bundled), NOT in FIPS mode
- System OpenSSL (UBI9-minimal) is FIPS-capable but not used by the binary
- FIPS gap: gRPC TLS uses non-FIPS BoringSSL regardless of host FIPS settings
- Workaround: Disable MLMD-level TLS, rely on Istio mTLS or TLS-terminating proxy
- Standard check-payload FIPS validation (Go binary) does not apply to C++ binaries

Build Hermeticity:
- Bazel WORKSPACE pins dependencies via SHA256 hashes (partial hermeticity)
- No rpms.lock.yaml for OS packages
- No Hermeto/cachi2 prefetch integration
- Tekton pipeline: hermetic=false
