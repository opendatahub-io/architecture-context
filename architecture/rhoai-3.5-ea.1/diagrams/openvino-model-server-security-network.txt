RBAC Summary:
- OVMS is a serving runtime container and does NOT define its own Kubernetes RBAC
- RBAC is managed by the deploying operator (KServe / rhods-operator)
- KServe InferenceService creates the pod with appropriate ServiceAccount
- kube-rbac-proxy sidecar enforces platform-level auth

Secrets:
- API key file (Opaque/file) - Bearer token for REST API auth, loaded at startup
  via --api_key_file CLI flag. Manual provisioning. No auto-rotation.
- AWS credentials (env vars) - AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY,
  AWS_SESSION_TOKEN. Manual / IRSA provisioning. No auto-rotation.
- Azure connection string (env var) - AZURE_STORAGE_CONNECTION_STRING.
  Manual provisioning. No auto-rotation.
- Google credentials (env var/file) - GOOGLE_APPLICATION_CREDENTIALS.
  Manual / Workload Identity. No auto-rotation.
- HuggingFace token (env var) - HF_TOKEN. Manual provisioning. No auto-rotation.

Authentication & Authorization:
- REST endpoints: Bearer Token (optional) checked by HttpRestApiHandler.isAuthorized()
  Only active if --api_key_file is configured at startup; otherwise ALL requests allowed
- gRPC endpoints: NO authentication (InsecureServerCredentials)
- Health endpoints (/v2/health/*): Always unauthenticated
- Metrics endpoint (/metrics): Always unauthenticated
- Platform-level: kube-rbac-proxy sidecar + Istio/Gateway API provide external auth

FIPS Compliance:
- C++ binary (not Go), no GOEXPERIMENT/strictfipsruntime flags
- Dynamically linked against UBI 9 OpenSSL (FIPS-validated module)
- No built-in TLS on inbound connections (relies on platform infrastructure)
- Uses OpenSSL MD5 for non-cryptographic model file checksums (acceptable)
- Outbound TLS via libcurl and libgit2 using system OpenSSL

Service Mesh Configuration:
- Istio sidecar injection: Optional, depends on platform namespace configuration
- When enabled: transparent mTLS between pods
- OVMS does not configure PeerAuthentication or AuthorizationPolicy directly

Crypto Analysis:
- Inbound: NO encryption (plaintext HTTP/gRPC). Relies on:
  * kube-rbac-proxy (TLS termination at pod level)
  * Istio sidecar (mTLS at mesh level)
  * Gateway API (TLS at ingress level)
- Outbound: TLS via system OpenSSL for cloud storage and HuggingFace
- MD5 usage: Non-cryptographic checksums for model files (openssl/md5.h)
