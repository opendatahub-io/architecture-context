Security Notes:
- InsecureSkipVerify available for both prefiller and decoder TLS connections
  (disabled by default; should remain disabled in production)
- SSRF protection is OPTIONAL; when disabled, any host in x-prefiller-host-port
  header is accepted without validation
- CGO_ENABLED=0 in upstream Dockerfile (fails check-payload FIPS gate)
- Downstream RHOAI build expected to use CGO_ENABLED=1 + GOEXPERIMENT=strictfipsruntime
- No authentication enforced by sidecar itself; relies on network/pod-level security
