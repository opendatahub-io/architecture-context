RBAC Summary:
- No Kubernetes RBAC resources — standalone service without Kubernetes API access

Secrets:
- TLS server cert (TLS_CERT_PATH) — PEM file, server-side TLS, provisioned externally
- TLS server key (TLS_KEY_PATH) — PEM file, server-side TLS private key, provisioned externally
- Client CA cert (TLS_CLIENT_CA_CERT_PATH) — PEM file, client cert verification for mTLS
- Per-service TLS certs (config tls section) — PEM files, client TLS/mTLS to downstream services
- API tokens (service.api_token config) — Bearer token auth to downstream HTTP services

Authentication & Authorization:
- /health, /info (8034) — No auth, open access
- /api/v1/*, /api/v2/* (8033) — Optional mTLS client certs (rustls WebPkiClientVerifier)
- Downstream detectors — Optional Bearer token (passthrough X-Forwarded-Access-Token or api_token)
- Downstream gRPC services — Optional mTLS client certs (tonic TLS config per service)

FIPS Compliance Notes:
- TLS: rustls with ring crypto provider (not OpenSSL)
- ring implements FIPS 140-2 algorithms but is NOT formally FIPS-validated
- OpenSSL 3.5.1-7.el9_7 is installed in container but NOT used by TLS stack
- ⚠ RISK: NoVerifier struct disables all cert verification when insecure=true
- No Go GOEXPERIMENT equivalent for Rust FIPS mode

- Cargo.lock: Present (Rust dependency pinning)

- Hermeto prefetch: Not present
