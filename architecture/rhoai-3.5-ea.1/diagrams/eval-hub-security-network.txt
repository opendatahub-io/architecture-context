Authentication & Authorization:
- All /api/v1/evaluations/* endpoints:
  - K8s TokenReview validates Bearer token identity
  - K8s SubjectAccessReview checks namespace-scoped RBAC
  - Tenant namespace extracted from X-Tenant HTTP header
  - Virtual RBAC resources (not actual CRDs):
    - trustyai.opendatahub.io/evaluations
    - trustyai.opendatahub.io/collections
    - trustyai.opendatahub.io/providers
    - trustyai.opendatahub.io/status-events
    - mlflow.kubeflow.org/experiments
- Public endpoints (no auth): /api/v1/health, /metrics, /openapi.yaml, /docs

Secrets:
- Database credentials (Opaque, file mount) — PostgreSQL auth, manual/operator provisioned, no auto-rotate
- ServiceAccount token (projected) — K8s API auth + sidecar auth, K8s-managed, auto-rotate
- MLflow token (projected SA token) — MLflow tracking auth, K8s-managed, auto-rotate
- S3 credentials (Opaque) — AWS_ACCESS_KEY_ID/AWS_SECRET_ACCESS_KEY, manual/operator, no auto-rotate
- OCI registry credentials (dockerconfigjson) — OCI registry auth, manual/operator, no auto-rotate
- Model auth secret (Opaque) — Model endpoint auth for adapters, user-provided, no auto-rotate
- TLS cert/key (kubernetes.io/tls) — HTTPS for eval-hub server, cert-manager/operator, configurable rotation

Pod Security:
- All containers run as non-root
- SeccompProfile: RuntimeDefault
- All capabilities dropped
- Sidecar uses KEP-753 pattern (restartPolicy: Always on init container)

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
- Base image: registry.access.redhat.com/ubi9/ubi-minimal (OpenSSL)
- TLS: Min 1.2, Max 1.3 enforced on all servers and clients
- Crypto: stdlib crypto/tls (FIPS via OpenSSL at runtime)
- No third-party crypto libraries
- InsecureSkipVerify available but defaults to false
