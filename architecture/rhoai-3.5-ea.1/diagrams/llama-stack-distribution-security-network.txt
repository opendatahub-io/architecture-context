RBAC Summary:
  No Kubernetes RBAC resources — this is a container image, not an operator.
  RBAC enforcement is handled by:
  - kube-rbac-proxy sidecar (Kubernetes SA-based authorization)
  - OGX built-in auth middleware (OAuth2 JWT, resource ownership policy)

Application-Level Access Policy:
  - System resources: All users can read (public)
  - User resources: Authenticated users can create
  - Owned resources: Only resource owners can read/update/delete

Secrets (injected via environment variables):
  - VLLM_API_TOKEN (Bearer) — vLLM inference auth
  - VLLM_EMBEDDING_API_TOKEN (Bearer) — vLLM embedding auth
  - POSTGRES_PASSWORD (Opaque) — PostgreSQL auth
  - AWS_BEARER_TOKEN_BEDROCK — AWS Bedrock (optional)
  - WATSONX_API_KEY — IBM watsonx (optional)
  - AZURE_API_KEY — Azure OpenAI (optional)
  - OPENAI_API_KEY — OpenAI API (optional)
  - MILVUS_TOKEN — Milvus vector DB (optional)
  - BRAVE_SEARCH_API_KEY — Brave Search (optional)
  - TAVILY_SEARCH_API_KEY — Tavily Search (optional)
