SDK → Ray Head Direct Connections (from client environment):
─────────────────────────────────────────────────────────────
  SDK ──── 10001/TCP ──── Ray Head (gRPC, mTLS, Client cert)  [ray.init()]
  SDK ──── 6379/TCP  ──── Ray Head (gRPC, mTLS, Client cert)  [GCS]

TLS Certificate Chain:
──────────────────────
  1. KubeRay Operator creates CA Secret ({cluster}-ca-secret-{uid})
  2. SDK reads CA cert+key from Kubernetes API
  3. SDK generates client cert (RSA-3072, SHA-256) signed by CA
  4. Client cert stored at ~/.local/share/codeflare/tls/ (0600 perms)
  5. SDK uses client cert for mTLS to Ray Head (ports 6379, 10001)

ConfigMap Mounts (into Ray Pods):
─────────────────────────────────
  - odh-trusted-ca-bundle → /etc/pki/tls/certs/odh-trusted-ca-bundle.crt
  - odh-trusted-ca-bundle → /etc/ssl/certs/ variants

FIPS Compliance:
────────────────
  - Algorithms: RSA-3072 + SHA-256 (FIPS-approved)
  - Library: cryptography 46.0.7 (wraps OpenSSL, FIPS-capable on RHEL/UBI)
  - No non-FIPS crypto detected (no MD5, RC4, DES)
  - skip_tls flag available but defaults to False
