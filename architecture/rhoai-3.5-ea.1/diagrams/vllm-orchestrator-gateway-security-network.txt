RBAC Summary:
  None — this is a stateless HTTP service with no Kubernetes API interactions.
  No ClusterRoles, Roles, or RoleBindings defined.

Authentication & Authorization:
  - Gateway does NOT enforce any authentication or authorization
  - Authorization header is forwarded as-is to the orchestrator
  - x-forwarded-* headers are forwarded for upstream identity propagation
  - Auth enforcement is expected at:
    * Network level (kube-rbac-proxy sidecar, service mesh, HTTPRoute)
    * Orchestrator level
    * Platform ingress level

TLS Configuration:
  - Inbound: Plaintext HTTP on 8090/TCP (no TLS termination at gateway)
  - Outbound: Optional mTLS to orchestrator
    * Enabled when TLS cert files exist at well-known paths
    * Falls back to plaintext HTTP when TLS files are absent
    * Uses openssl crate for PKCS#12 identity construction
    * Uses native-tls backend via reqwest
    * danger_accept_invalid_hostnames(true) when host is localhost only

FIPS Compliance Notes:
  - No FIPS-specific build flags
  - Dynamically links against system OpenSSL (UBI base image)
  - compat-openssl11 installed in release image
  - Relies on system OpenSSL FIPS configuration from UBI base
  - No explicit FIPS mode opt-in in application code
  - Cargo.lock provides deterministic Rust dependency resolution
  - Build is NOT hermetic (hermetic: false in Tekton pipeline)
