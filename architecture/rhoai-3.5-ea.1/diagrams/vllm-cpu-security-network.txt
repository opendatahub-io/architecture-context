RBAC Summary:
- No RBAC resources defined — this is a container image, not an operator
- RBAC is managed by KServe at the InferenceService level
- Container runs as non-root user 2000, group 0 (OpenShift arbitrary UID)

Secrets:
- HF_TOKEN (Opaque, env var) - HuggingFace Hub auth for model download, Manual/KServe provisioned, No auto-rotate
- SSL key/cert (kubernetes.io/tls) - TLS encryption for API server (optional), cert-manager/Manual provisioned, Yes auto-rotate (SSLCertRefresher hot-reload)
- VLLM_API_KEY (Opaque, env var) - API authentication token, Manual/KServe provisioned, No auto-rotate

Authentication & Authorization:
- /v1/* endpoints: Bearer Token (optional) - enforced only if --api-key or VLLM_API_KEY is set
- /health, /metrics, /version, /load: No auth required - always accessible
- gRPC 50051/TCP: No authentication mechanism

FIPS Compliance Notes:
- Python service, no Go binaries (N/A for check-payload GOEXPERIMENT=strictfipsruntime)
- GRPC_PYTHON_BUILD_SYSTEM_OPENSSL=1 ensures grpcio uses system OpenSSL from UBI9
- Python stdlib ssl module delegates to system OpenSSL
- UBI9 base image provides FIPS-validated OpenSSL
- blake3 used for hashing (not FIPS-validated, but not used for security purposes)
- Build is non-hermetic for EA release (hermetic: false in Tekton pipeline)

Build-Time Network Dependencies:
- PyPI (pypi.org) - Python package index
- IBM DevPI (wheels.developerfirst.ibm.com) - ppc64le-specific wheels
- Fedora EPEL repositories - additional RPM packages
- No lock files present (rpms.lock.yaml, uv.lock, poetry.lock all absent)
