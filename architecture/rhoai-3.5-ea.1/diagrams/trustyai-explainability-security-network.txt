RBAC Summary:
- RBAC is managed externally by trustyai-service-operator, not by this service
- Service relies on infrastructure-level auth enforcement:
  - kube-rbac-proxy sidecar (injected by operator)
  - Kubernetes NetworkPolicy
  - Cluster-internal access only (ClusterIP services)

Service Configuration:
- trustyai-service: ClusterIP, 8080/TCP HTTP (primary)
- trustyai-service: ClusterIP, 4443/TCP HTTPS (TLS-secured)
- Ingress: Managed by operator (HTTPRoute/Route, 443/TCP HTTPS, TLS SIMPLE)

Secrets:
- TLS certificates (kubernetes.io/tls) - HTTPS on port 4443, provisioned by operator/cert-manager
- Database credentials (Opaque) - MariaDB/MySQL auth (QUARKUS_DATASOURCE_USERNAME/PASSWORD), no auto-rotate
- MinIO credentials (Opaque) - S3 access-key/secret-key, user-provisioned, no auto-rotate

Authentication & Authorization:
- No application-level authentication on any REST endpoint
- All auth enforcement delegated to infrastructure:
  - kube-rbac-proxy sidecar (operator-injected)
  - Kubernetes NetworkPolicy
  - ClusterIP service exposure (cluster-internal only)
- /consumer/kserve/v2: Cluster-internal access only via Kubernetes networking

FIPS Compliance:
- Java on UBI 9 / OpenJDK 17: Inherits FIPS mode from host OS
- JVM uses FIPS-approved crypto providers (NSS/PKCS#11) on FIPS-enabled clusters
- TLS: Quarkus-managed with external cert files on port 4443
- ⚠ FIPS Gap: gRPC channels to KServe model servers use plaintext (usePlaintext())
  - Mitigated: Explainer endpoints disabled in RHOAI profile
- SecureRandom usage: FIPS-safe via JDK provider
- SunPKCS11 provider config modified in Dockerfile for build compatibility

Security Concerns:
1. ⚠ gRPC plaintext: KServe inference calls use usePlaintext() - no TLS
   - Risk: Inference data transmitted unencrypted within cluster
   - Mitigation: Explainers disabled in RHOAI profile
2. No application-level auth: All endpoints unprotected at app level
   - Mitigation: kube-rbac-proxy sidecar + NetworkPolicy
3. Database credentials: No auto-rotation configured
4. MinIO credentials: User-provisioned, no auto-rotation
