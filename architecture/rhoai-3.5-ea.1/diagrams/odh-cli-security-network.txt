LOCAL FILESYSTEM ACCESS:
- Backup command writes YAML manifests to --output-dir (file permissions: 0644)
- Reads kubeconfig from ~/.kube/config or $KUBECONFIG or --kubeconfig flag
- No other local filesystem writes

RBAC Summary:
- No ClusterRoles/ClusterRoleBindings installed by CLI
- Operates under authenticated user's existing RBAC permissions
- Minimum required: read-only (get, list, watch) across all ODH CRDs
- Write access: Only for deps install (create namespace, operatorgroup, subscription)
- Permission errors: Classified as exit code 4 (AUTH) with actionable suggestions

Supply Chain Security:
- Catalog source allowlist: redhat-operators, community-operators,
  certified-operators, redhat-marketplace
- Rejects installations targeting system namespaces (kube-*, openshift-*)
- Prevents privilege escalation via attacker-controlled catalogs/namespaces

FIPS Compliance:
- Build: CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime, -tags strictfipsruntime
- Linking: Dynamic (CGO_ENABLED=1 for OpenSSL linkage)
- TLS: Delegates to client-go default (system trust store, FIPS via OpenSSL)
- yq: Also built with FIPS flags
- No custom TLS config, no InsecureSkipVerify in source
- No non-FIPS crypto imports (MD5/RC4/DES)

Secrets:
- (kubeconfig) - Kubernetes API authentication credentials
  - Type: kubeconfig file or bearer token
  - Provisioned by: User / OpenShift login
  - No secrets created or managed by CLI

Build Hermeticity:
- OS packages: rpms.lock.yaml (hermetic)
- Go deps: go.sum (locked, NOT hermetic - network download at build time)
- Python deps: requirements.txt (pinned with hashes, NOT hermetic)
- No Hermeto/cachi2 prefetch integration
