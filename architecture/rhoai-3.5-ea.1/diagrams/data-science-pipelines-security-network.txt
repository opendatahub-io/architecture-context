Authentication Flow:
====================

  1. Multi-user mode:
     - HTTP Header (x-kubeflow-userid) extracted by API Server interceptor
     - User identity validated via Kubernetes SubjectAccessReview
     - Authorization checked against pipelines.kubeflow.org RBAC resources

  2. Bearer Token (fallback):
     - Token validated via Kubernetes TokenReview API
     - Audience: pipelines.kubeflow.org

  3. Internal service-to-service:
     - Projected ServiceAccount Token mounted at /var/run/secrets/kubeflow/tokens/
     - Audience: pipelines.kubeflow.org
     - Expiry: 1 hour with automatic refresh

  4. Webhook (PipelineVersion):
     - mTLS between Kubernetes API Server and API Server webhook endpoint (8443/TCP)

FIPS Compliance:
================

  All shippable components built with:
  - CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime
  - Dynamic linking with OpenSSL (UBI9 base image)
  - Launcher ships dual binary: standard (CGO_ENABLED=0) + FIPS (CGO_ENABLED=1)
  - API Server explicitly installs openssl-1:3.5.1-7.el9_7
  - No crypto/md5, crypto/rc4, or crypto/des usage detected
  - One InsecureSkipVerify=true in test-only code path (gated by flag)
