Authentication Chain:
====================
1. Browser → Gateway: OIDC Bearer Token (from OpenShift OAuth)
2. Gateway → kube-rbac-proxy: Token forwarded via HTTPS
3. kube-rbac-proxy: Validates token via TokenReview + SubjectAccessReview
4. kube-rbac-proxy → Backend: User identity injected as X-Auth headers
5. Backend → K8s API / Services: User's Bearer Token forwarded as-is
6. Backend → BFF: User's token forwarded via x-forwarded-access-token header

Secrets:
========
- dashboard-proxy-tls (kubernetes.io/tls)
  - Purpose: TLS cert for kube-rbac-proxy + all BFF modules
  - Provisioned By: OpenShift service-ca (via service annotation)
  - Auto-Rotate: Yes

- odh-dashboard-modules-token (kubernetes.io/service-account-token)
  - Purpose: Long-lived SA token for BFF module containers
  - Provisioned By: Kubernetes (manual Secret)
  - Auto-Rotate: No ⚠️

Security Notes:
===============
- ⚠️ Perses proxy uses HTTP (no encryption) on port 8080/TCP
- ⚠️ Llama Stack proxy uses HTTP (no encryption) on port 8321/TCP
- ⚠️ odh-dashboard-modules-token does not auto-rotate
- SA isolation: BFF sidecars use odh-dashboard-modules SA (limited perms)
  vs. main dashboard using odh-dashboard SA (broad perms)
- automountServiceAccountToken disabled; projected volumes mount correct SA tokens
- FIPS: BFF Go binaries built with CGO_ENABLED=1 + strictfipsruntime tag
- FIPS: Main dashboard removes esbuild Go binaries to prevent non-FIPS crypto
