## Purpose
**Short**: An HTTP gateway that provides OpenAI-compatible chat completion endpoints with configurable guardrail detector pipelines for content filtering.

**Detailed**: The vLLM Orchestrator Gateway is a lightweight Rust-based HTTP service that acts as a routing layer between clients and the FMS Guardrails Orchestrator. It provides multiple OpenAI-compatible `/v1/chat/completions` endpoints, each configured with specific detector pipelines for content filtering (e.g., PII detection, regex-based filtering). This allows drop-in replacement of unguarded chat completion models with guardrailed alternatives.

The gateway reads a YAML configuration file that defines the backend orchestrator connection, detector configurations (input/output classifiers with parameters), and named routes. Each route dynamically exposes a unique endpoint path (e.g., `/{route_name}/v1/chat/completions`) and injects the appropriate detector specifications into requests before forwarding them to the FMS Guardrails Orchestrator. When detections are found, the gateway substitutes the model's response with a configurable fallback message. The gateway supports mTLS for secure communication with the orchestrator backend and forwards authorization and `x-forwarded-*` headers from clients.
