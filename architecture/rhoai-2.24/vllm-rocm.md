## Purpose
**Short**: A high-throughput and memory-efficient inference and serving engine for Large Language Models (LLMs), built for AMD ROCm GPUs.

**Detailed**: vLLM is a high-performance LLM inference engine that provides OpenAI-compatible API serving with advanced memory management through PagedAttention. This repository is the Red Hat downstream fork specifically configured to build container images targeting AMD ROCm GPUs (gfx908, gfx90a, gfx942, gfx1100) for use in Red Hat OpenShift AI (RHOAI).

The container image ships two runtime targets: a primary `vllm-openai` image that serves an OpenAI-compatible REST API on port 8000, and a `vllm-grpc-adapter` image that adds a gRPC interface via `vllm-tgis-adapter` on port 8033 for integration with TGIS-compatible model serving infrastructure (e.g., ModelMesh/KServe in RHOAI). The build uses Red Hat UBI 9 as the base image and ROCm 6.3.4 for AMD GPU acceleration, with PyTorch 2.7.0 for the ROCm build.

Key features include continuous batching, speculative decoding, tensor/pipeline/data/expert parallelism for distributed inference, multi-LoRA support, prefix caching, and support for multimodal models (vision, audio). The engine integrates Flash Attention (ROCm fork) for optimized attention computation and supports quantization formats including GPTQ, AWQ, INT4, INT8, and FP8.
