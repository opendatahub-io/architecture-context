## Purpose
**Short**: A Kubeflow Pipelines v2 pipeline that orchestrates the InstructLab LLM fine-tuning workflow on Red Hat OpenShift AI.

**Detailed**: The ilab-on-ocp repository provides a complete KFP v2 pipeline definition and supporting container image for running the InstructLab large language model fine-tuning process on OpenShift clusters with Red Hat OpenShift AI (RHOAI). The pipeline automates the end-to-end workflow: Synthetic Data Generation (SDG) using a teacher model, multi-phase distributed training via PyTorchJob (Kubeflow Training Operator), model evaluation using MT-Bench and MMLU benchmarks with a judge model, and finally model output to OCI registries and/or Model Registry.

The pipeline is designed to run as a managed pipeline within the RHOAI Data Science Pipelines infrastructure. It is automatically provisioned into the pipeline server when enabled via the DataSciencePipelinesApplication (DSPA) resource. The repository produces a single container image (`odh-ml-pipelines-runtime-generic`) that embeds the compiled pipeline YAML and serves as the runtime image for generic pipeline tasks. The actual ML training workloads use the RHEL AI InstructLab image and are distributed across GPU-enabled nodes using the Kubeflow Training Operator's PyTorchJob CRD.

The repository also includes supplementary Kubernetes manifests for deploying the Teacher (Mixtral) and Judge (Prometheus) model servers via KServe InferenceService, as well as an optional NFS storage provisioner for development environments.
