## Purpose
**Short**: KubeRay operator manages the lifecycle of Ray clusters, jobs, and serving workloads on Kubernetes/OpenShift.

**Detailed**: The KubeRay operator is a Kubernetes controller that automates the deployment, scaling, and management of Ray clusters on OpenShift. It provides three Custom Resource Definitions (CRDs) -- RayCluster, RayJob, and RayService -- that allow users to declaratively manage Ray infrastructure.

The operator handles the complete lifecycle of Ray workloads: creating head and worker pods with proper networking, managing autoscaler RBAC, reconciling Kubernetes services for inter-node communication, creating OpenShift Routes (or Kubernetes Ingresses) for external dashboard access, and coordinating Ray Serve deployments for model serving. It integrates with batch schedulers (Volcano, Yunikorn, scheduler-plugins) for gang scheduling and supports GCS fault tolerance with automated Redis cleanup.

In RHOAI, this operator is deployed by the rhods-operator via kustomize overlays into the `opendatahub` namespace. It runs with a SecurityContextConstraints that enforces UID 1000 and drops all capabilities. The operator image is built with FIPS-compliant Go runtime (`strictfipsruntime` build tag) on UBI9.
