## Purpose
**Short**: REST API orchestration server that coordinates AI text generation with content safety guardrail detectors, chunkers, and LLM backends.

**Detailed**: The FMS Guardrails Orchestrator serves as middleware between clients and language models, providing a unified API for invoking content safety detectors on text generation input and output. It acts as an orchestration layer that manages the flow of requests between multiple services: text generation backends (TGIS, Caikit NLP, OpenAI-compatible), content safety detector services, and text chunker services.

The orchestrator supports both unary and streaming request patterns. For unary requests, it coordinates detection on input text, forwards to a generation backend, and optionally runs detection on output text before returning a combined response. For streaming, it processes text chunks as they arrive from generation backends, running detectors on partial output in real time. It also provides standalone detection endpoints for text content analysis, chat message analysis, context/document analysis, and generated text analysis without requiring a generation backend.

The server supports TLS and mTLS for both inbound connections (from clients) and outbound connections (to detector/generation/chunker services). It integrates with OpenTelemetry for distributed tracing and metrics export via gRPC or HTTP protocols.

### TLS Configuration

The orchestrator supports three TLS modes controlled by environment variables or CLI args:

1. **No TLS**: Default when `TLS_CERT_PATH` and `TLS_KEY_PATH` are not set. Guardrails API serves plaintext HTTP.
2. **TLS**: When `TLS_CERT_PATH` and `TLS_KEY_PATH` are set. Server presents certificate to clients.
3. **mTLS**: When additionally `TLS_CLIENT_CA_CERT_PATH` is set. Server requires and verifies client certificates against the provided CA.

For outbound connections, each service (generation, detector, chunker) can have independent TLS configuration with named TLS profiles defined in `config.yaml`. Client TLS supports cert/key pairs, CA certs, and an `insecure` flag to skip verification.
