Authentication Flow:
================================================================================

  1. User accesses dashboard via OpenShift Route (HTTPS/443)
  2. Route terminates external TLS, reencrypts to oauth-proxy (8443)
  3. oauth-proxy validates OpenShift OAuth cookie
     - If no valid cookie: redirects to OpenShift OAuth login
     - If valid: extracts access token from cookie
  4. oauth-proxy forwards request to backend (HTTP/8080)
     - Injects x-forwarded-access-token header
  5. Backend uses access token for Kubernetes API calls
     - User's RBAC permissions apply (no privilege escalation)
  6. /api/health and /metrics bypass authentication
