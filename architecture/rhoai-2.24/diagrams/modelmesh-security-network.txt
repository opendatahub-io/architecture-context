NETWORK POLICIES:
====================================================================
Policy: model-mesh
  Pod Selector: app=model-mesh, component=model-mesh
  Direction: Ingress
  Rules:
    - Allow 8033/TCP and 8080/TCP from pods with
      app=model-mesh, component=model-mesh (inter-pod)
    - Allow 8033/TCP from any (inference clients)
    - Allow 2112/TCP from any (Prometheus metrics scraping)

TLS CONFIGURATION:
====================================================================
  Server Certificate:  MM_TLS_KEY_CERT_PATH (PEM)
  Server Private Key:  MM_TLS_PRIVATE_KEY_PATH (PKCS8 PEM)
  Key Passphrase:      MM_TLS_PRIVATE_KEY_PASSPHRASE (optional)
  Client Auth Mode:    MM_TLS_CLIENT_AUTH (NONE / REQUIRE / OPTIONAL)
  Client Trust Certs:  MM_TLS_TRUST_CERT_PATH (comma-separated PEM paths)
  Internal Trust:      MM_INTERNAL_TRUST_CERTS (for cert rotation)
  FIPS Mode:           DISABLED (-Dcom.redhat.fips=false)

FIPS COMPLIANCE NOTE:
====================================================================
  Java FIPS mode is explicitly disabled (-Dcom.redhat.fips=false) in
  start.sh to avoid TLS breakage with Netty's native SSL support
  (BoringSSL/OpenSSL). This is a known limitation of the current
  TLS implementation.
