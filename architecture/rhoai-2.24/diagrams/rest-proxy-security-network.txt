RBAC Summary:
- No Kubernetes RBAC required
- rest-proxy is a stateless sidecar and does not interact with the Kubernetes API
- Authentication/authorization delegated to enclosing ModelMesh Serving infrastructure
  (ModelMesh Serving / Istio / kube-rbac-proxy)

TLS Configuration:
- Inbound (REST listener):
  - Enabled when MM_TLS_KEY_CERT_PATH and MM_TLS_PRIVATE_KEY_PATH are set
  - Uses http.ListenAndServeTLS
- Outbound (gRPC connection):
  - Enabled when REST_PROXY_USE_TLS=true
  - Optional cert verification skip via REST_PROXY_SKIP_VERIFY

FIPS Compliance:
- Built with GOEXPERIMENT=strictfipsruntime
- Uses -tags strictfipsruntime build flags
- CGO_ENABLED=1 (required for FIPS crypto)

Secrets:
- TLS certificate (path via MM_TLS_KEY_CERT_PATH) - TLS cert for HTTPS listener
  Provisioned by: ModelMesh Serving operator
  Auto-Rotate: Depends on cert provisioner
- TLS private key (path via MM_TLS_PRIVATE_KEY_PATH) - TLS key for HTTPS listener
  Provisioned by: ModelMesh Serving operator
  Auto-Rotate: Depends on cert provisioner

Container Security:
- Run User: 2000 (non-root)
- Base Image: registry.access.redhat.com/ubi9/ubi-minimal
- Build Image: registry.access.redhat.com/ubi9/go-toolset:1.23
- Build System: Konflux (Tekton pipelines)

Environment Variables (Security-Relevant):
- REST_PROXY_USE_TLS: Enable TLS for gRPC connection (default: false)
- REST_PROXY_SKIP_VERIFY: Skip TLS cert verification (default: false)
- MM_TLS_KEY_CERT_PATH: Path to TLS certificate for HTTPS listener
- MM_TLS_PRIVATE_KEY_PATH: Path to TLS private key for HTTPS listener
- REST_PROXY_GRPC_MAX_MSG_SIZE_BYTES: Max gRPC message size (default: 16 MB)
