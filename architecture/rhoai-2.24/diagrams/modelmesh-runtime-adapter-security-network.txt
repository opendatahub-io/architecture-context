RBAC Summary:
- No RBAC resources defined
- This component runs as a sidecar container and does not interact with the Kubernetes API
- Pod-level security is managed by modelmesh-serving

Secrets:
- storage-config (Opaque) - Storage credentials (S3 access keys, GCS service accounts,
  Azure credentials). Provisioned by user / modelmesh-serving controller. No auto-rotation.

Authentication & Authorization:
- Internal gRPC (8084, 8085): No authentication - localhost only, Pod network isolation
- S3 storage: Access Key ID + Secret Access Key (read-only access)
- GCS storage: OAuth2 / GCP Service Account (read-only access)
- Azure Blob: Azure DefaultAzureCredential (read-only access)

FIPS Compliance:
- Go binaries built with -tags strictfipsruntime and GOEXPERIMENT=strictfipsruntime
- FIPS-compliant cryptographic operations enforced at build time

Network Isolation:
- All internal communication is localhost-scoped within the Pod
- No Kubernetes Services created by this component
- No ingress resources
- Egress only to external storage backends (HTTPS/TLS 1.2+)
