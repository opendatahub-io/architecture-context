================================================================================
RBAC Summary
================================================================================

vLLM is a containerized service, NOT a Kubernetes operator.
RBAC is managed by the deploying platform (KServe/ModelMesh).
No ClusterRoles, ClusterRoleBindings, or ServiceAccount-specific
permissions are defined by vLLM itself.

================================================================================
Authentication & Authorization
================================================================================

Endpoint Pattern    | Auth Mechanism         | Enforcement Point
--------------------|------------------------|-----------------------------
/v1/*               | Bearer Token           | AuthenticationMiddleware (ASGI)
                    | (Authorization header) | Reject 401 if invalid
/health, /ping,     | None                   | N/A - publicly accessible
  /metrics, /load   |                        |
/tokenize,          | None                   | N/A - publicly accessible
  /detokenize       |                        |
All endpoints       | Optional mTLS          | SSL context (ssl_cert_reqs)
                    |                        | Client cert verification when
                    |                        | ssl_ca_certs configured

================================================================================
Secrets
================================================================================

Secret Name      | Type        | Purpose                        | Rotation
-----------------|-------------|--------------------------------|---------
ssl-keyfile      | File mount  | TLS private key for HTTPS      | Yes (SSLCertRefresher)
ssl-certfile     | File mount  | TLS certificate for HTTPS      | Yes (SSLCertRefresher)
ssl-ca-certs     | File mount  | CA certs for client verify     | No
api-key          | Env var     | Bearer token (VLLM_API_KEY)    | No
s3-credentials   | Env vars    | S3_ACCESS_KEY_ID,              | No
                 |             | S3_SECRET_ACCESS_KEY            |
hf-token         | Env var     | HuggingFace Hub auth (HF_TOKEN)| No

================================================================================
Container Security
================================================================================

Setting              | Value              | Purpose
---------------------|--------------------|---------------------------------
User                 | 2000 (vllm)        | Non-root for OpenShift
Group                | 0 (root group)     | OpenShift arbitrary UID support
HF_HUB_OFFLINE       | 1                  | Prevent runtime HF downloads
VLLM_NO_USAGE_STATS  | 1                  | Disable external telemetry
H11_MAX_INCOMPLETE   | 4194304 (4 MB)     | Mitigate HTTP header abuse
H11_MAX_HEADER_COUNT | 256                | Mitigate header abuse attacks
