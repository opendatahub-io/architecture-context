RBAC Summary:
- No RBAC resources defined by this component
- vLLM runs as non-root container (UID 2000, GID 0)
- Pod security managed by KServe/ModelMesh

Secrets:
- ssl-keyfile (kubernetes.io/tls) - TLS private key, User/Operator provisioned, hot-reload via watchfiles
- ssl-certfile (kubernetes.io/tls) - TLS certificate, User/Operator provisioned, hot-reload via watchfiles
- ssl-ca-certs (Opaque) - CA certs for client verification, User/Operator provisioned
- api-key (Opaque) - Bearer token for API auth (VLLM_API_KEY), User/Operator provisioned

Security Hardening:
- Non-root execution: UID 2000, GID 0
- HTTP header limits: Max 256 headers, 4MB max header size
- HuggingFace Hub offline: HF_HUB_OFFLINE=1 (no runtime downloads)
- Usage stats disabled: VLLM_NO_USAGE_STATS=1
- SSL cert hot-reload: --enable-ssl-refresh via watchfiles
- Optional mTLS: --ssl-ca-certs + --ssl-cert-reqs for client cert verification

- vllm-openai (ROCm): quay.io/modh/vllm:rhoai-2.24-rocm
  Base: UBI 9 Minimal, ROCm 6.3.4, PyTorch 2.7.0
- vllm-grpc-adapter (ROCm): Built on top of vllm-openai image
  Adds: vllm-tgis-adapter 0.8.0
