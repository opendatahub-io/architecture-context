RBAC Summary:
- No RBAC resources defined by vllm-gaudi
- Runs as non-root user (UID 2000, GID 0)
- RBAC for serving infrastructure managed by KServe and RHOAI operator
- Pod security managed by KServe/ModelMesh deployment

Secrets:
- SSL Key/Cert (TLS, optional) - TLS termination, User/Operator provisioned,
  auto-rotated via SSLCertRefresher file watching
- API Key (Opaque, optional) - Bearer token auth for /v1/* endpoints,
  User/Operator provisioned, no auto-rotation
- HuggingFace Token (Opaque, optional) - HF Hub model downloads,
  User provisioned, no auto-rotation

Authentication & Authorization:
- /v1/* endpoints: Optional Bearer Token via --api-key or VLLM_API_KEY env
  - 401 rejection on token mismatch; OPTIONS and non-/v1 paths skip auth
- /health, /load, /metrics, /version: No authentication required
- TLS: Optional via --ssl-keyfile/--ssl-certfile/--ssl-ca-certs
  - Optional client cert verification (--ssl-cert-reqs)
  - Certificate hot-reload via SSLCertRefresher

Container Security:
- Non-root execution: UID 2000, GID 0
- HF_HUB_OFFLINE=1: Prevents runtime model downloads
- VLLM_NO_USAGE_STATS=1: Disables usage telemetry
- OpenShift compatible: arbitrary UID support via GID 0
