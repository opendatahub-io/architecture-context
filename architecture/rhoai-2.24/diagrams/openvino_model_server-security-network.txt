RBAC Summary:
=============
  No RBAC resources defined by OVMS.
  OVMS is a container image, not a Kubernetes operator.
  RBAC is managed by the deploying platform (KServe/ModelMesh).

  Platform-level RBAC (managed externally):
  - KServe ServiceAccount for InferenceService management
  - Istio AuthorizationPolicy for request-level access control
  - kube-rbac-proxy for API gateway authentication

Service Mesh Configuration:
============================
  OVMS relies entirely on external service mesh for security:
  - PeerAuthentication: STRICT (expected, namespace-scoped via RHOAI)
  - mTLS: Enforced by Istio sidecar injection
  - AuthorizationPolicy: Managed by KServe/RHOAI platform
  - OVMS itself: Plaintext listeners, no TLS termination
