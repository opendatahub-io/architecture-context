## Purpose
**Short**: A lightweight Rust-based HTTP microservice that detects sensitive content (PII) in text using predefined and custom regular expressions, serving as a detector backend for the FMS Guardrails Orchestrator.

**Detailed**: The Guardrails Regex Detector is a high-performance text analysis service written in Rust using the Axum web framework and Tokio async runtime. It provides a REST API endpoint that accepts text content and a set of regex patterns (either built-in or custom), returning detection results with positional information (start/end offsets), matched text, detection type classification, and confidence scores.

The service includes three built-in PII detectors: email address, Social Security Number (SSN), and credit card number. Users can also supply arbitrary custom regex patterns in the request payload. It is designed to be integrated with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator) as a pluggable detector service, enabling content safety guardrails for AI/ML inference pipelines within the Red Hat OpenShift AI platform.

The container image is built on Red Hat UBI 9 Minimal for production use, ensuring compliance with Red Hat's security and support standards. The service is stateless and horizontally scalable with no external dependencies at runtime.

### Request/Response Schema

**Request body** (`POST /api/v1/text/contents`):
```json
{
  "contents": ["string array of text to analyze"],
  "detector_params": {
    "regex": ["ssn", "email", "credit-card", "^custom-regex$"]
  }
}
```

**Response body** (200 OK):
```json
[[{
  "start": 0,
  "end": 11,
  "text": "matched text",
  "detection_type": "pii|custom",
  "detection": "EmailAddress|SocialSecurity|CreditCard|custom",
  "score": 1.0
}]]
```
