## Purpose
**Short**: Kubeflow Pipelines platform with Tekton as the workflow execution backend, enabling ML pipeline authoring, scheduling, and execution on OpenShift.

**Detailed**: Data Science Pipelines (Tekton) is a fork of Kubeflow Pipelines that replaces the Argo Workflows execution engine with Tekton Pipelines. It provides the complete Kubeflow Pipelines platform including an API server for pipeline management, a web UI for pipeline visualization and management, a persistence agent for tracking pipeline run status, a scheduled workflow controller for cron/periodic pipeline execution, and ML Metadata (MLMD) services for artifact and lineage tracking.

The system compiles Kubeflow Pipelines DSL into Tekton PipelineRun YAML, stores pipeline definitions and run metadata in a MySQL/MariaDB database, manages pipeline artifacts in S3-compatible object storage (MinIO), and exposes both gRPC and REST APIs for pipeline operations. It integrates with Tekton CRDs (PipelineRun, TaskRun) for actual workflow execution and defines its own CRDs (ScheduledWorkflow, Viewer) for scheduling and visualization capabilities.

In the RHOAI distribution, the UI is fronted by an OAuth proxy for OpenShift authentication, and the component is deployed via kustomize manifests under the `manifests/opendatahub/` directory. The platform supports multi-user isolation when deployed with Istio service mesh.
