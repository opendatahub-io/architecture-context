## Purpose
**Short**: Provides pre-built workbench and runtime container images for data science development environments in Red Hat OpenShift AI.

**Detailed**: This repository contains Dockerfiles, dependency manifests (Pipfile.lock), deployment manifests, and CI/test infrastructure for building and shipping workbench images used by the RHOAI Notebook Controller. The workbench images include JupyterLab, Code Server (VS Code), and RStudio Server environments, each available in various configurations targeting different ML frameworks (PyTorch, TensorFlow, TrustyAI) and accelerators (CPU, NVIDIA CUDA, AMD ROCm).

The repository does not deploy a running service. Instead, it produces container images that are referenced by OpenShift ImageStream resources. The RHOAI operator deploys these ImageStreams into the cluster, and the ODH Notebook Controller uses them to launch user workbench pods. Each workbench image exposes a web-based IDE (JupyterLab on port 8888, Code Server on port 8787, RStudio Server) and is designed to run as a non-root user (UID 1001) on UBI9 or RHEL9 base images.

The repository supports Python 3.11 and 3.12, with Python 3.12 images provided as an additional overlay. Images are built by Konflux CI (preferred) and legacy Dockerfiles are maintained for compatibility. The manifests directory contains kustomize overlays that generate ConfigMaps and ImageStreams with pinned image digests for reproducible deployments.
