## Purpose
**Short**: A Kubernetes operator that manages the lifecycle of Llama Stack distribution server instances on OpenShift/Kubernetes clusters.

**Detailed**: The llama-stack-k8s-operator is a controller-runtime based Kubernetes operator that creates and manages Llama Stack server deployments. It watches `LlamaStackDistribution` custom resources (CRD) and reconciles them into Deployments, Services, PersistentVolumeClaims, ServiceAccounts, ClusterRoleBindings, and NetworkPolicies. The operator supports multiple Llama Stack distributions (e.g., starter, ollama, remote-vllm, tgi, bedrock, together, vllm-gpu) and can also accept direct container image references.

In the RHOAI context, the operator is deployed by the rhods-operator using kustomize overlays from `config/overlays/rhoai/`. The RHOAI overlay configures the operator image (`RELATED_IMAGE_ODH_LLAMASTACK_OPERATOR`) and the Red Hat distribution image (`RELATED_IMAGE_RH_DISTRIBUTION`) which maps to the `rh-dev` distribution name. The operator supports TLS CA bundle injection (both explicit and auto-detection of the `odh-trusted-ca-bundle` ConfigMap), user configuration via ConfigMaps, persistent storage via PVCs, and optional NetworkPolicy enforcement via feature flags. The controller renders embedded kustomize manifests at runtime, applies name-prefix and namespace transformations, and manages resources with proper owner references.

## Kustomize Deployment Structure

The operator is deployed via kustomize overlays. The RHOAI overlay at `config/overlays/rhoai/` references the default base at `config/default/` which includes CRDs, RBAC, and the manager Deployment.

**Overlay chain**: `config/overlays/rhoai/` -> `config/default/` -> `config/crd/` + `config/rbac/` + `config/manager/`

**RHOAI overlay configuration**:
- Namespace: `redhat-ods-applications`
- Injects `RELATED_IMAGE_ODH_LLAMASTACK_OPERATOR` as the operator image
- Injects `RELATED_IMAGE_RH_DISTRIBUTION` as the Red Hat Llama Stack distribution image
- Injects `OPERATOR_VERSION` as the operator version env var

**Runtime manifest rendering**: The operator embeds kustomize manifests at `controllers/manifests/base/` into the binary (via Dockerfile COPY). At runtime, it renders these manifests through kustomize with plugins that apply name-prefix, namespace, field mutations (storage size, service port, ClusterRoleBinding subjects).

**Embedded manifests** (rendered per LlamaStackDistribution CR):
- `pvc.yaml` - PersistentVolumeClaim (conditional, only if storage is configured)
- `serviceaccount.yaml` - ServiceAccount with `openshift.io/scc: anyuid` annotation
- `scc-binding.yaml` - ClusterRoleBinding to `system:openshift:scc:anyuid` ClusterRole
- `service.yaml` - ClusterIP Service on configurable port (default 8321/TCP)
