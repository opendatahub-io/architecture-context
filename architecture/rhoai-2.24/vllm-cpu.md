## Purpose
**Short**: A high-throughput, memory-efficient LLM inference and serving engine optimized for CPU execution, providing OpenAI-compatible API and gRPC (TGIS) endpoints.

**Detailed**: vLLM is an inference and serving engine for Large Language Models (LLMs) originally developed at UC Berkeley. This repository (`vllm-cpu`) is Red Hat's downstream fork specifically optimized for CPU-based inference on x86_64 (amd64), ppc64le, and s390x architectures within the Red Hat OpenShift AI (RHOAI) platform.

The component builds two container image targets: `vllm-openai` (an OpenAI-compatible HTTP API server) and `vllm-grpc-adapter` (a gRPC server using the vllm-tgis-adapter for TGIS protocol compatibility). Both targets are built on top of UBI 9 (Red Hat Universal Base Image) with architecture-specific Dockerfiles for ppc64le and s390x. The engine leverages PagedAttention for efficient memory management, continuous batching for throughput optimization, and supports a wide range of model architectures including Llama, Mixtral, Qwen, and many others.

The vllm-cpu variant uses PyTorch with CPU backend, Intel Extension for PyTorch (IPEX) on x86_64, and OpenBLAS on ppc64le. It serves as a model serving runtime that is typically deployed behind KServe or ModelMesh within the RHOAI platform, enabling model inference without requiring GPU hardware.
