## Purpose
**Short**: Kubernetes operator that manages TrustyAI explainability services, LM evaluation jobs, and guardrails orchestration for AI/ML model trustworthiness.

**Detailed**: The TrustyAI Service Operator is a multi-controller Go operator that provides three core services for the RHOAI platform. First, it manages TrustyAIService custom resources which deploy the TrustyAI explainability service alongside model serving deployments (ModelMesh and KServe), enabling fairness metrics, model monitoring, and bias detection through payload processing. Second, it manages LMEvalJob custom resources for running language model evaluation tasks using the lm-evaluation-harness framework, with Kueue integration for job queue management. Third, it manages GuardrailsOrchestrator custom resources for deploying content safety guardrails that protect LLM inference endpoints with configurable detectors and an optional sidecar gateway.

The operator is deployed by the rhods-operator using kustomize overlays from `config/overlays/rhoai/`. It runs as a single Deployment with `--enable-services TAS,LMES,GORCH` to activate all three controller subsystems. Each controller watches its respective CRD and dynamically creates Deployments, Services, Routes, ServiceAccounts, ServiceMonitors, and Istio networking resources (DestinationRules, VirtualServices) as needed.

| LMEvalJob Controller (LMES) | Go Controller | Reconciles LMEvalJob CRs; creates evaluation Pods with driver sidecar container, manages job lifecycle (New/Scheduled/Running/Complete/Cancelled/Suspended), handles PVC provisioning for outputs and offline storage |

| GuardrailsOrchestrator Controller (GORCH) | Go Controller | Reconciles GuardrailsOrchestrator CRs; creates Deployments (with optional built-in detector and gateway sidecar containers), Services, OpenShift Routes, ServiceAccounts |
| Controller Manager | Go Binary | Main entry point (`cmd/main.go`); initializes controller-runtime manager, registers all scheme types, starts health/readiness probes |
