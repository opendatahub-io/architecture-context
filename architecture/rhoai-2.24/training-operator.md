## Purpose
**Short**: Kubernetes operator for managing distributed machine learning training jobs across multiple frameworks (PyTorch, TensorFlow, XGBoost, MPI, PaddlePaddle, JAX).

**Detailed**: The Kubeflow Training Operator (KFTO) is a Kubernetes-native controller that orchestrates distributed training workloads for AI/ML models. It provides Custom Resource Definitions (CRDs) for six ML frameworks: PyTorchJob, TFJob, XGBoostJob, MPIJob, PaddleJob, and JAXJob.

Each CRD allows users to define distributed training topologies with multiple replica types (e.g., Master/Worker for PyTorch, PS/Worker for TensorFlow). The operator manages the full lifecycle of training jobs including pod creation, headless service creation for inter-worker communication, status tracking, restart policies, and cleanup. It supports gang scheduling via Volcano and scheduler-plugins for coordinated pod scheduling, elastic training with autoscaling for PyTorch, and NetworkPolicy creation for PyTorchJobs to isolate inter-pod training traffic.

The operator includes a validating webhook server for CREATE/UPDATE validation of training job resources, with self-managed TLS certificate rotation via the OPA cert-controller. In the RHOAI distribution, the operator is deployed into the `opendatahub` namespace with the `kubeflow-` name prefix and is built with FIPS-compliant Go runtime.
