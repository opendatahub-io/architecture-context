## Purpose
**Short**: A Kubernetes controller that manages the lifecycle of OpenShift Routes, Istio resources, network policies, monitoring, authentication, and certificate trust for KServe and ModelMesh InferenceServices on OpenDataHub/RHOAI.

**Detailed**: The odh-model-controller is a controller-runtime based Go operator that acts as an integration layer between KServe/ModelMesh model serving components and the OpenShift/RHOAI platform. It watches InferenceService, ServingRuntime, InferenceGraph, and related core Kubernetes resources, then dynamically creates and reconciles OpenShift Routes, Istio Gateways, VirtualServices, PeerAuthentications, Telemetry resources, NetworkPolicies, Authorino AuthConfigs, monitoring ServiceMonitors/PodMonitors, KEDA TriggerAuthentications, and storage configuration secrets.

The controller supports three InferenceService deployment modes: KServe Serverless (Knative + Istio service mesh), KServe RawDeployment (direct Kubernetes deployments with OpenShift Routes), and ModelMesh (multi-model serving). Each mode has its own set of sub-resource reconcilers that create the appropriate platform resources. Additionally, the controller manages NVIDIA NIM integration through its Account CRD, handling API key validation, model configuration, and pull secret management. The controller also manages Ray TLS certificates for multi-node serving runtimes and aggregates data connection secrets into a unified storage-config secret for model storage access.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| InferenceService Controller | Go Controller | Watches InferenceService CRs and delegates to mode-specific reconcilers (Serverless, RawDeployment, ModelMesh) to create Routes, Gateways, NetworkPolicies, AuthConfigs, ServiceMonitors, etc. |
| InferenceGraph Controller | Go Controller | Watches InferenceGraph CRs and reconciles Authorino AuthConfig resources for authorization |
| ServingRuntime Controller | Go Controller | Watches ServingRuntime CRs, manages monitoring RoleBindings, and reconciles Ray TLS certificates for multi-node inference |
| ConfigMap Controller | Go Controller | Watches CA bundle ConfigMaps (odh-trusted-ca-bundle, openshift-service-ca.crt) and creates/updates odh-kserve-custom-ca-bundle for KServe |
| Secret Controller | Go Controller | Watches ODH-managed secrets (data connections) and generates the aggregated storage-config secret for model storage access |
| Pod Controller | Go Controller | Watches predictor pods with Ray TLS enabled and manages per-pod TLS certificates for multi-node serving |
| NIM Account Controller | Go Controller | Reconciles NIM Account CRs for NVIDIA NIM integration: validates API keys, manages config, creates pull secrets and runtime templates |
| Webhook Server | Admission Webhooks | Mutating webhooks for Pod, InferenceService, InferenceGraph; Validating webhooks for NIM Account, InferenceService, InferenceGraph, Knative Service |
| KServe Serverless Reconciler | Sub-reconciler | Creates ServiceMeshMember, Routes (via Istio ingress), PeerAuthentication, Telemetry, ServiceMonitor, PodMonitor, NetworkPolicy, AuthConfig, Service certs, Gateway entries, metrics dashboards |
| KServe Raw Reconciler | Sub-reconciler | Creates ClusterRoleBindings, OpenShift Routes (direct to predictor service), metrics Services, ServiceMonitors, metrics dashboards, KEDA resources |
| ModelMesh Reconciler | Sub-reconciler | Creates OpenShift Routes (to modelmesh-serving service), ServiceAccounts, ClusterRoleBindings |
| Model Registry Reconciler | Sub-reconciler | Optional integration that syncs InferenceService state with Model Registry service |
