## Purpose
**Short**: A reverse proxy sidecar that orchestrates disaggregated prefill/decode (P/D) inference by routing requests between prefiller and decoder vLLM workers.

**Detailed**: The llm-d-routing-sidecar is a specialized HTTP/HTTPS reverse proxy designed to support disaggregated prefill/decode (P/D) inference architectures in LLM serving. It runs as a sidecar container alongside a vLLM decoder instance, intercepting OpenAI-compatible chat completion and completion requests.

When a request arrives with an `x-prefiller-host-port` header, the sidecar orchestrates a two-phase inference protocol: first forwarding the request to the designated prefiller worker for KV-cache population, then forwarding the modified request (with KV transfer parameters) to the local decoder for token generation. The sidecar supports multiple P/D connector protocols including NIXL v2 (current), NIXL v1 (deprecated), and LMCache (deprecated).

The sidecar includes SSRF (Server-Side Request Forgery) protection that watches Kubernetes InferencePool resources (from the Gateway API Inference Extension) to maintain a dynamic allowlist of valid prefill targets based on pod IPs and names matching the InferencePool's label selectors. It supports TLS for both proxy-facing and backend connections with configurable certificate paths.

### SSRF Protection Details

When `--enable-ssrf-protection=true`:
- Watches a specific InferencePool resource in the configured namespace
- Extracts pod label selectors from the InferencePool spec
- Creates pod informers to watch matching pods
- Builds an allowlist of pod IPs and pod names
- Validates the host portion of `x-prefiller-host-port` against the allowlist (port is not checked)
- Allowlist is automatically rebuilt on pod add/update/delete events
- Resync period: 30 seconds

## Data Flows

### Flow 1: Disaggregated Prefill/Decode (NIXL v2 Protocol)

| Step | Source | Destination | Port | Protocol | Encryption | Auth |
|------|--------|-------------|------|----------|------------|------|
| 1 | Client/Gateway | Routing Sidecar | 8000/TCP | HTTPS | TLS 1.2+ | None |
| 2 | Routing Sidecar | SSRF Validator | N/A (in-process) | N/A | N/A | InferencePool allowlist check |
| 3 | Routing Sidecar | Remote Prefiller vLLM | Variable | HTTP or HTTPS | TLS 1.2+ (configurable) | None |
| 4 | Remote Prefiller vLLM | Routing Sidecar | N/A (response) | HTTP or HTTPS | TLS 1.2+ | None |
| 5 | Routing Sidecar | Local Decoder vLLM | 8001/TCP | HTTP or HTTPS | TLS 1.2+ (configurable) | None |
| 6 | Local Decoder vLLM | Routing Sidecar | N/A (response) | HTTP or HTTPS | TLS 1.2+ | None |
| 7 | Routing Sidecar | Client/Gateway | N/A (response) | HTTPS | TLS 1.2+ | None |

**NIXL v2 Protocol Details**:
1. Sidecar receives request with `x-prefiller-host-port` header
2. Generates a UUID for request correlation
3. Sends modified request to prefiller with: `stream=false`, `max_tokens=1`, `kv_transfer_params={do_remote_decode: true, do_remote_prefill: false}`
4. Extracts `kv_transfer_params` from prefiller response (contains block IDs, engine ID, host, port)
5. Sends original request to local decoder with: original stream/max_tokens values restored, prefiller's `kv_transfer_params` attached
6. Streams decoder response back to client

### Flow 2: Passthrough (No Disaggregation)

| Step | Source | Destination | Port | Protocol | Encryption | Auth |
|------|--------|-------------|------|----------|------------|------|
| 1 | Client/Gateway | Routing Sidecar | 8000/TCP | HTTPS | TLS 1.2+ | None |
| 2 | Routing Sidecar | Local Decoder vLLM | 8001/TCP | HTTP or HTTPS | TLS 1.2+ (configurable) | None |
| 3 | Local Decoder vLLM | Client/Gateway | N/A (response) | HTTPS | TLS 1.2+ | None |

When no `x-prefiller-host-port` header is present, the request is directly proxied to the local decoder.

### Flow 3: SSRF Protection Allowlist Updates

| Step | Source | Destination | Port | Protocol | Encryption | Auth |
