## Purpose
**Short**: Provides pre-built container images for Jupyter, Code Server, and RStudio workbenches as well as Elyra pipeline runtime images for the RHOAI platform.

**Detailed**: The notebooks repository is the source for all workbench and pipeline runtime container images shipped with Red Hat OpenShift AI (RHOAI). It does not contain an operator or controller -- instead it produces container images that are registered as OpenShift ImageStream resources via kustomize manifests. The rhods-operator deploys these ImageStream manifests into the cluster, making the workbench images available to users through the RHOAI dashboard for creating interactive development environments (Jupyter, Code Server, RStudio) and Elyra pipeline runtimes.

The repository supports multiple Python versions (3.11 and 3.12), multiple hardware accelerators (NVIDIA CUDA, AMD ROCm), and multiple CPU architectures (amd64, ppc64le, s390x). Each workbench image is versioned using an N/N-1/N-2/N-3/N-4/N-5 scheme, allowing users to select from up to six historical versions of each image. Images are built by Konflux CI/CD pipelines defined in `.tekton/` and published to `quay.io/modh/` registries. The kustomize manifests use `params.env` files to inject pinned image digests into ImageStream tag references at deployment time.

## Kustomize Deployment Structure

### Overlay: `overlays/additional` (RHOAI Self-Managed)

The operator processes `manifests/overlays/additional/kustomization.yaml` which:
1. Includes `../../base` (all py311 ImageStreams + BuildConfigs + ConfigMaps)
2. Adds py312 ImageStream resources (jupyter, codeserver, runtime images)
3. Uses `replacements` to inject image references from `notebook-image-params` ConfigMap into ImageStream tags
4. Uses `replacements` to inject build commit hashes from `notebook-image-commithash` ConfigMap into ImageStream annotations
5. Labels all resources with `component.opendatahub.io/name: notebooks` and `opendatahub.io/component: "true"`
