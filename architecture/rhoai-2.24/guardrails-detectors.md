## Purpose
**Short**: A collection of text content detection microservices for the FMS Guardrails Orchestrator, providing PII detection, file-type validation, HuggingFace model-based content analysis, and LLM-as-a-Judge evaluation.

**Detailed**: The guardrails-detectors repository provides detector algorithms and microservices supported by the TrustyAI team for use with the FMS (Foundation Model Stack) Guardrails Orchestrator. These detectors analyze text content for various safety and compliance concerns, returning structured detection results with position information, confidence scores, and detection type classifications.

The repository produces three independent container images, each serving as a standalone microservice with a FastAPI-based REST API. The **built-in detector** provides lightweight, heuristic-based detections including regex pattern matching for PII (emails, credit cards, SSNs, phone numbers, IP addresses, UK post codes) and file-type validation (JSON, XML, YAML with optional schema validation). The **HuggingFace detector** wraps HuggingFace models (AutoModelForSequenceClassification and GraniteForCausalLM) for ML-based content analysis such as toxicity and prompt injection detection. The **LLM Judge detector** integrates the vLLM Judge library to use any OpenAI API-compatible LLM for evaluation against built-in metrics (safety, accuracy, helpfulness, etc.) or custom natural-language criteria.

All three detectors implement the IBM Detector API specification (`/api/v1/text/contents`) and are designed to be deployed as KServe InferenceServices within the RHOAI platform, integrated with Istio service mesh for traffic management and authentication.
