## Purpose
**Short**: A high-performance text generation inference server that serves large language models via gRPC and HTTP APIs with continuous batching.

**Detailed**: Text Generation Inference Server (TGIS) is an IBM-developed fork of HuggingFace's text-generation-inference, purpose-built for serving large language models (LLMs) in production. It provides a multi-component architecture consisting of a Rust-based launcher that orchestrates GPU model shards, a Rust-based router that handles request batching and gRPC/HTTP serving, and a Python-based model server that performs the actual GPU inference using PyTorch.

The system supports a wide range of models including causal LM (decoder-only) and sequence-to-sequence (encoder-decoder) architectures. It features continuous batching with memory-aware scheduling, flash attention v2 for efficient GPU memory usage, tensor parallelism for multi-GPU inference, GPTQ and bitsandbytes quantization, TLS/mTLS support, OpenTelemetry tracing, and Prometheus metrics. The server communicates internally via Unix domain sockets between the router and model shards.

TGIS is used within RHOAI as the backend inference runtime for model serving, typically deployed behind KServe/ModelMesh for production model serving workloads.
