## Purpose

**Short**: A secure, batch-oriented Python runtime for evaluating language models against standardized benchmarks and custom tasks using the EleutherAI lm-evaluation-harness framework.

**Detailed**: LM Evaluation Harness (lm-eval) is a framework originally developed by EleutherAI for evaluating autoregressive language models. This Red Hat downstream fork (`red-hat-data-services/lm-evaluation-harness`) packages the framework as a containerized job image (`odh-ta-lmes-job`) designed for one-shot evaluation workloads in RHOAI (Red Hat OpenShift AI).

The container provides a standardized runtime environment with pre-installed dependencies for model evaluation, including support for HuggingFace Transformers, vLLM, IBM watsonx.ai, and OpenAI-compatible API backends. It integrates the Unitxt framework for flexible, reusable data preparation and evaluation recipes. The image includes bundled offline metrics for disconnected/air-gapped environments and supports custom task definitions and metric catalogs.

This component is not an operator or long-running service — it is a job container image that is launched by the TrustyAI LMEval operator (`trustyai-service-operator`) as a Kubernetes Job/Pod to perform a single evaluation run, then exits. The operator creates the pod with the appropriate environment variables, model arguments, and task configurations.
