## Purpose
**Short**: Kubernetes operator that manages RayCluster lifecycle with OAuth proxy authentication and mTLS encryption, and optionally embeds the AppWrapper controller for batch workload management.

**Detailed**: The CodeFlare Operator is a controller-runtime based Kubernetes operator that watches RayCluster custom resources (from the KubeRay project) and augments them with security infrastructure for OpenShift deployments. When a RayCluster is created, the operator dynamically provisions OAuth proxy sidecars, OpenShift Routes for the Ray Dashboard and Ray Client, TLS certificates for inter-node mTLS communication, NetworkPolicies to restrict traffic, and ClusterRoleBindings for authentication delegation.

The operator also optionally embeds the AppWrapper controller (from the project-codeflare/appwrapper project), which provides batch workload management capabilities. AppWrappers allow users to submit multi-resource workloads that are managed as a single unit, with integration into Kueue for quota management. Both the RayCluster controller and AppWrapper controller expose mutating and validating admission webhooks to enforce configuration invariants.

The operator is deployed by the rhods-operator as part of RHOAI using kustomize overlays from the `config/default/` directory. It runs in the `opendatahub` namespace and manages resources across all namespaces where RayClusters are created.
