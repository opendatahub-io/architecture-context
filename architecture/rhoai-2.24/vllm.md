## Purpose
**Short**: A high-throughput and memory-efficient inference and serving engine for Large Language Models (LLMs).

**Detailed**: vLLM is an inference engine designed to serve LLMs with high throughput and low latency. It implements PagedAttention for efficient KV cache memory management, continuous batching of incoming requests, and optimized CUDA/HIP/CPU kernels. The engine supports a wide range of model architectures including transformer-based LLMs, Mixture-of-Expert models, embedding models, and multi-modal models (vision, audio, video).

In the RHOAI distribution, vLLM is packaged as a container image built on Red Hat UBI 9. It produces two container image variants: an OpenAI-compatible HTTP API server (`vllm-openai`) and a gRPC adapter image (`vllm-grpc-adapter`) that wraps vLLM with the vllm-tgis-adapter for compatibility with the Text Generation Inference Service (TGIS) protocol used by RHOAI model serving infrastructure (ModelMesh/KServe). The RHOAI builds support CUDA (amd64), ROCm (amd64), CPU on ppc64le, and CPU on s390x architectures.

vLLM supports distributed inference across multiple GPUs via tensor parallelism, pipeline parallelism, data parallelism, and expert parallelism. It can use either multiprocessing or Ray for distributed orchestration. Additional features include speculative decoding, prefix caching, multi-LoRA serving, quantization (GPTQ, AWQ, INT4, INT8, FP8), and OpenTelemetry-based tracing.
