## Purpose
**Short**: Sidecar container that acts as an intermediary between ModelMesh and third-party model-server runtimes, providing model storage pulling and runtime-specific model layout adaptation.

**Detailed**: The modelmesh-runtime-adapter is a multi-purpose container image deployed as a sidecar within ModelMesh Serving model server Pods. It contains five logical subcomponents built into a single image: a model-serving-puller and four runtime-specific adapters (Triton, MLServer, OpenVINO/OVMS, and TorchServe).

The **model-serving-puller** acts as a gRPC proxy that sits between ModelMesh and the model runtime. When ModelMesh requests a model to be loaded, the puller intercepts the request, downloads the model artifacts from remote storage (S3, GCS, Azure Blob, HTTP, or PVC), and then forwards the load request to the downstream model runtime with updated local filesystem paths. It implements the `ModelRuntime` gRPC service, the same interface that the runtime adapters implement.

The **runtime adapters** (Triton, MLServer, OVMS, TorchServe) also implement the `ModelRuntime` gRPC interface and translate ModelMesh's generic model lifecycle operations into runtime-specific API calls. Each adapter handles model directory layout adaptation (reorganizing downloaded files into the format expected by its target runtime), model loading/unloading via the runtime's native gRPC API, and reporting runtime status and model sizing back to ModelMesh. The adapters can optionally embed the puller functionality directly using the `USE_EMBEDDED_PULLER` configuration.

The **pullman** library provides a pluggable storage provider framework supporting S3 (via IBM COS SDK), Google Cloud Storage, Azure Blob Storage, HTTP(S) downloads, and PVC-based storage with client caching.
