## Purpose
**Short**: The Feast Operator manages the lifecycle of Feast (Feature Store) deployments on Kubernetes/OpenShift, automating the provisioning of online stores, offline stores, registries, and UI services.

**Detailed**: Feast is an open-source feature store for machine learning that helps teams manage, store, and serve ML features consistently across training and inference. The Feast Operator is a Kubernetes operator built with controller-runtime that watches `FeatureStore` custom resources and reconciles the desired state by creating and managing Deployments, Services, PersistentVolumeClaims, ConfigMaps, ServiceAccounts, RBAC Roles/RoleBindings, CronJobs, and (on OpenShift) Routes.

The operator supports multiple persistence backends for online stores (SQLite, Redis, PostgreSQL, Snowflake, Cassandra, DynamoDB, etc.), offline stores (Dask, DuckDB, Snowflake, PostgreSQL, Spark, etc.), and registries (file-based, SQL, Snowflake). It automatically configures TLS on OpenShift using service serving certificates and supports both Kubernetes RBAC and OIDC-based authorization. A CronJob is deployed by default (suspended) to execute `feast apply` and `feast materialize-incremental` commands against the feature store deployment.

When deployed on OpenShift, the operator detects the platform at startup via the `route.openshift.io` API group and enables OpenShift-specific features including automatic TLS via service serving certificates, CA bundle injection, and OpenShift Route creation for the UI service.

| UI Server | Container (feature-server) | Web UI for browsing the feature store (port 8888/TCP plain, 8443/TCP TLS) |
| CronJob | Kubernetes CronJob | Executes `feast apply` and `feast materialize-incremental` via kubectl exec against the deployment |
| Client ConfigMap | ConfigMap | Provides `feature_store.yaml` configuration for client applications |
| Authorization (Kubernetes) | RBAC Roles/RoleBindings | Creates Kubernetes RBAC roles for Feast permission model when Kubernetes authorization is configured |
| Authorization (OIDC) | Secret-based config | Supports OIDC-based authorization via secret references containing client credentials |
