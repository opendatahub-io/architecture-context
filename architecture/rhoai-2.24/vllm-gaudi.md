## Purpose
**Short**: A high-throughput, memory-efficient LLM inference and serving engine optimized for Intel Gaudi (HPU) AI accelerators, packaged for Red Hat OpenShift AI.

**Detailed**: vllm-gaudi is a fork of the vLLM project (originally from HabanaAI/vllm-fork) that provides optimized large language model (LLM) inference on Intel Gaudi 2 and Gaudi 3 AI accelerators. It implements PagedAttention for efficient key-value cache management, continuous batching of incoming requests, and supports tensor/pipeline parallelism for distributed inference across multiple Gaudi devices.

The container image ships two entrypoint modes: an OpenAI-compatible REST API server (the primary entrypoint in the `vllm-openai` build stage) and a gRPC adapter (`vllm-grpc-adapter` build stage) using the `vllm-tgis-adapter` package for compatibility with the Text Generation Inference Server (TGIS) protocol used by KServe/ModelMesh in RHOAI. The server supports a comprehensive set of OpenAI-compatible endpoints including chat completions, text completions, embeddings, pooling, scoring, reranking, and audio transcription.

The component is built using Konflux CI/CD (Tekton pipelines) from `Dockerfile.hpu.ubi` which uses a RHEL 9.4-based Habana PyTorch installer image. It is deployed as a serving runtime container within KServe InferenceService or ModelMesh ServingRuntime custom resources, not as a standalone operator.
