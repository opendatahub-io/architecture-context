## Purpose
**Short**: Manages the lifecycle of Data Science Pipelines (Kubeflow Pipelines v2) application components on OpenShift via the DataSciencePipelinesApplication (DSPA) custom resource.

**Detailed**: The Data Science Pipelines Operator (DSPO) is a Kubernetes operator built with controller-runtime that watches DataSciencePipelinesApplication (DSPA) custom resources and reconciles the full set of sub-components needed to run Kubeflow Pipelines v2 on OpenShift. When a user creates a DSPA CR in a namespace, DSPO deploys and configures: the KFP API Server, Persistence Agent, Scheduled Workflow controller, Argo Workflow Controller, MariaDB (or external DB), Minio (or external object storage), ML Metadata (MLMD) gRPC server with Envoy proxy, and optionally a KFP UI. Each sub-component is deployed from Go templates in `config/internal/`.

The operator performs health checks against the database and object store before deploying pipeline components, supports pod-to-pod TLS via OpenShift service-ca certificates, manages OAuth proxy sidecars for route-based authentication, provisions network policies to restrict inter-component traffic, and publishes Prometheus metrics for component readiness. It also supports a "BYOArgo" mode where the Argo Workflow Controller management can be toggled between Managed and Removed states. For RHOAI, the operator is deployed by rhods-operator via kustomize overlay at `config/overlays/rhoai/`.

| MLMD gRPC Server | Deployment (dynamically created) | Serves ML Metadata via gRPC for artifact and execution tracking |

| OAuth Proxy | Sidecar container | OpenShift oauth-proxy sidecar on API Server and MLMD Envoy for route authentication |
