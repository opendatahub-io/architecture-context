## Purpose
**Short**: A container image that provides the Caikit AI runtime for serving Large Language Models (LLMs) with optional TGIS (Text Generation Inference Server) backend, deployed as a KServe ServingRuntime on OpenShift.

**Detailed**: Caikit-TGIS-Serving is a Python-based container image that packages the Caikit AI toolkit runtime together with the caikit-nlp and caikit-tgis-backend libraries. It serves as the inference layer in the RHOAI model serving stack, providing developer-friendly APIs for LLM inference. The container runs the `caikit.runtime` module which exposes both HTTP (port 8080) and gRPC (port 8085) endpoints for model inference requests.

The serving architecture follows a multi-container pattern within KServe. In the "caikit-tgis" configuration, the Caikit runtime acts as a transformer container that receives inference requests and delegates the actual model execution to a co-located TGIS (Text Generation Inference Server) container via gRPC on localhost:8033. In the "caikit-standalone" configuration, the Caikit runtime directly loads and serves models without TGIS. Both configurations are deployed through KServe ServingRuntime and InferenceService custom resources, with Knative Serving providing serverless scaling and Istio Service Mesh providing mTLS security and traffic management.

This repository does not contain an operator or controller - it is a container image build repository. The container images are built by Konflux and published to `quay.io/modh/caikit-tgis-serving`. Deployment and lifecycle management is handled by KServe, which is managed by the RHOAI operator (rhods-operator) through the DataScienceCluster CR.
