## Purpose
**Short**: An Endpoint Picker (EPP) service that makes optimized routing decisions for LLM inference requests using Gateway API Inference Extension (GIE) and Envoy ext-proc.

**Detailed**: The llm-d-inference-scheduler is a core component of the llm-d inference framework that provides intelligent request routing for LLM inference workloads. It extends the upstream Kubernetes Gateway API Inference Extension (GIE) project with llm-d-specific scheduling plugins including KV-cache-aware routing, load-based scoring, session affinity, and disaggregated Prefill/Decode (P/D) support.

The EPP operates as a gRPC server that acts as an Envoy external processing (ext-proc) filter. When inference requests arrive at the Gateway (Envoy-based), they are forwarded to the EPP which applies a configurable pipeline of filters and scorers to select the optimal backend pod from an InferencePool. Filters narrow down candidate pods (e.g., by role label for P/D disaggregation), while scorers rank remaining candidates based on criteria like prefix cache hit rate, queue depth, and session affinity. The scheduler supports both standard single-profile routing and disaggregated Prefill/Decode mode where separate scheduling profiles select distinct prefill and decode worker pods.

The component integrates with the llm-d-kv-cache-manager for real-time KV cache state tracking via ZMQ event streams, enabling cache-locality-aware scheduling that can significantly reduce time-to-first-token for LLM inference requests.
