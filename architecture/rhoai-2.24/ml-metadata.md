## Purpose
**Short**: ML Metadata (MLMD) is a gRPC server for recording and retrieving metadata associated with ML pipeline workflows, tracking artifacts, executions, and their lineage.

**Detailed**: ML Metadata is a library and gRPC server originally developed by Google as part of TensorFlow Extended (TFX). It provides a metadata store for recording information about ML pipeline components, their executions (e.g., training runs), and resulting artifacts (e.g., trained models). In the RHOAI platform, the MLMD gRPC server is deployed as a container (`odh-mlmd-grpc-server`) and serves as the backend metadata store for Data Science Pipelines (DSP), enabling pipeline run tracking, artifact lineage, and experiment management.

The server supports multiple database backends including MySQL/MariaDB, PostgreSQL, and SQLite. It exposes a comprehensive gRPC API for managing metadata types (ArtifactType, ExecutionType, ContextType), instances (Artifact, Execution, Context), relationships (Events, Attributions, Associations), and lineage graphs. The server supports optional SSL/TLS for secure gRPC connections and configurable database connection SSL for secure backend communication.

### SSL/TLS Configuration

The server supports two levels of SSL/TLS:

1. **gRPC Server TLS** (`MetadataStoreServerConfig.SSLConfig`):
   - `server_key`: PEM-encoded private key
   - `server_cert`: PEM-encoded server certificate
   - `custom_ca`: Optional custom CA certificate
   - `client_verify`: When true, requires and verifies client certificates (mTLS)
   - If no `ssl_config` is provided, the server uses insecure (plaintext) gRPC

2. **Database Connection TLS**:
   - MySQL: `ssl_options` with `cert`, `key`, `ca`, `capath`, `cipher`, `verify_server_cert`
   - PostgreSQL: `ssloption` with `sslmode`, `sslcert`, `sslkey`, `sslpassword`, `sslrootcert`
