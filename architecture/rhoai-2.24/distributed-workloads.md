## Purpose
**Short**: Integration test suite and runtime container image definitions for the RHOAI Distributed Workloads stack (Kueue, KubeRay, Kubeflow Training Operator).

**Detailed**: The distributed-workloads repository is a supporting component in the RHOAI ecosystem that serves two primary functions: (1) it provides end-to-end integration tests that validate the distributed workloads stack including Kueue job queuing, KubeRay ray cluster orchestration, and the Kubeflow Training Operator (KFTO) for PyTorch distributed training, and (2) it contains Dockerfile definitions and Pipfile.lock dependency manifests for building GPU-accelerated runtime container images (Ray and Training images) used by data scientists in RHOAI.

This repository does not deploy any operators, controllers, or services itself. Instead, it exercises and validates the distributed workloads components that are deployed by other RHOAI operators (rhods-operator, codeflare-operator, kuberay-operator, training-operator, kueue). The test suite creates Kubernetes resources (RayClusters, RayJobs, PyTorchJobs, Kueue queues, Notebooks) to validate that the distributed workloads stack functions correctly on OpenShift. The container images built from this repository (Ray CUDA/ROCm, Training CUDA/ROCm) are shipped as part of RHOAI and used as base runtime images for distributed training and Ray workloads.
