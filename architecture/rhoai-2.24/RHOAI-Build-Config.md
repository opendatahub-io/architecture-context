## Purpose
**Short**: Central build configuration and OLM bundle/catalog repository for Red Hat OpenShift AI (RHOAI) 2.24, defining all shipped container images, operator metadata, and release artifacts for the Konflux build system.

**Detailed**: RHOAI-Build-Config is the authoritative source for the Red Hat OpenShift AI product build and release pipeline. It does not contain application source code or controllers itself; instead, it aggregates and defines the complete product surface: the OLM (Operator Lifecycle Manager) operator bundle manifests, the File-Based Catalog (FBC) fragments for operator discoverability on OpenShift, and the Konflux/Tekton CI/CD pipeline definitions that drive automated builds, testing, and releases.

The repository maintains the ClusterServiceVersion (CSV) for the `rhods-operator`, which declares all 73+ shipped container images as RELATED_IMAGE environment variables (critical for disconnected/air-gapped installations), defines 21 Custom Resource Definitions owned by the operator, configures 10 admission webhooks, and specifies the full RBAC permissions matrix. It also manages the FBC catalog entries across 6 supported OCP versions (v4.15 through v4.20), enabling the operator to appear in the OperatorHub catalog on each version.

The build pipeline is fully driven by Konflux (Red Hat's next-generation CI/CD platform). Tekton PipelineRun definitions handle both operator bundle builds and FBC fragment builds, with scheduled nightly builds and push-triggered builds. GitHub Actions workflows handle automated PR merging for dependency update PRs (Renovate-style component updates) and triggering nightly build schedules.
