## Purpose
**Short**: KServe is the model serving controller for RHOAI that manages the lifecycle of inference services, providing standardized APIs for deploying, scaling, and routing traffic to ML models on Kubernetes.

**Detailed**: KServe is a Kubernetes-native platform for serving machine learning models. It provides a custom resource-based API (InferenceService, LLMInferenceService, InferenceGraph, etc.) that abstracts the complexity of deploying ML models into production. The controller manages model deployment in multiple modes: Serverless (using Knative Serving), RawDeployment (using Kubernetes Deployments directly), and ModelMesh (for multi-model serving).

The RHOAI distribution of KServe includes significant enhancements for OpenShift, including OpenShift certificate injection (replacing upstream cert-manager), SecurityContextConstraints for LLM workloads, and integration with Istio service mesh for traffic management. The controller creates and manages VirtualServices, HTTPRoutes (Gateway API), Deployments, Services, and HorizontalPodAutoscalers based on the InferenceService specifications.

A key addition in this version is the LLMInferenceService controller, which provides specialized support for large language model deployment. It integrates with the Gateway API Inference Extension (InferencePool, InferenceModel), LeaderWorkerSet for multi-node inference, and the llm-d inference scheduler for intelligent request routing. The LLM controller manages complete deployment stacks including workloads, routing (HTTPRoutes + InferencePools), scheduler deployments, and monitoring resources.

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| {isvc-name} VirtualService | Istio VirtualService | {isvc-name}.{namespace}.{domain} | 80/TCP | HTTP | Via Istio Gateway TLS | SIMPLE | External (Serverless mode) |
| {isvc-name} HTTPRoute | HTTPRoute (Gateway API) | {predictor-name}-{namespace}.{domain} | 80/TCP | HTTP | Via Gateway TLS termination | SIMPLE | External (RawDeployment mode with enableGatewayApi=true) |
| {isvc-name}-predictor HTTPRoute | HTTPRoute (Gateway API) | {predictor-name}-{namespace}.{domain} | 80/TCP | HTTP | Via Gateway TLS termination | SIMPLE | External (RawDeployment per-component routes) |
| {llmisvc-name} HTTPRoute | HTTPRoute (Gateway API) | Generated from domain template | 80/TCP | HTTP | Via Gateway TLS termination | SIMPLE | External (LLMInferenceService) |

**Ingress Architecture Notes**:
- **Serverless mode** (default): Controller creates Istio VirtualServices referencing `knative-serving/knative-ingress-gateway` (ingress) and `istio-system/kserve-local-gateway` (local). The VirtualService routes traffic to Knative local gateway service.
- **RawDeployment mode with Gateway API**: Controller creates HTTPRoute CRs referencing parent Gateway `openshift-ingress/openshift-ai-inference`. Separate HTTPRoutes are created for predictor, transformer, explainer, and a top-level route.
- **RawDeployment mode without Gateway API**: Controller creates Kubernetes Ingress resources.
- **LLMInferenceService**: Uses HTTPRoutes with Gateway API and InferencePool/InferenceModel from Gateway API Inference Extension for intelligent routing.
- **ODH overlay configuration**: `disableIngressCreation: true` and `disableIstioVirtualHost: false` - ingress creation for RawDeployment is disabled by default; Istio virtual host is enabled.
