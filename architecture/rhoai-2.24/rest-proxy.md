## Purpose
**Short**: REST-to-gRPC reverse proxy that translates KServe V2 REST inference requests into gRPC V2 Predict Protocol calls for ModelMesh Serving.

**Detailed**: The rest-proxy is a lightweight Go service that leverages gRPC-Gateway to create a reverse-proxy server which translates RESTful HTTP API calls into gRPC calls. This enables clients to send inference requests using the KServe V2 REST Predict Protocol to model serving platforms (specifically ModelMesh) that natively speak the gRPC V2 Predict Protocol.

The proxy runs as a sidecar container alongside ModelMesh Serving inference containers. It listens for incoming HTTP/REST requests on a configurable port (default 8008/TCP), translates them into gRPC messages following the KServe V2 inference protocol, and forwards them to the local gRPC inference service endpoint (default localhost:8033). The proxy handles custom JSON marshaling/unmarshaling for tensor data, supporting various data types (BOOL, INT8-64, UINT8-64, FP16-64, BYTES) including multi-dimensional tensor shapes and base64-encoded binary content.

The service is built with FIPS compliance support (using `strictfipsruntime` build tags in the Konflux build) and supports optional TLS for both the incoming REST endpoint and the outgoing gRPC connection.

### TLS Configuration

The rest-proxy supports TLS in two directions:

1. **Inbound (REST listener)**: Enabled when `MM_TLS_KEY_CERT_PATH` and `MM_TLS_PRIVATE_KEY_PATH` environment variables are set. Uses `http.ListenAndServeTLS`.
2. **Outbound (gRPC connection)**: Enabled when `REST_PROXY_USE_TLS=true`. Supports optional certificate verification skip via `REST_PROXY_SKIP_VERIFY`.

### FIPS Compliance

The Konflux build (`Dockerfile.konflux`) compiles with `GOEXPERIMENT=strictfipsruntime` and `-tags strictfipsruntime`, ensuring FIPS-compliant cryptographic operations.
