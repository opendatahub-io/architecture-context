## Purpose
**Short**: Kubeflow Pipelines provides end-to-end ML workflow orchestration, enabling users to define, run, schedule, and monitor reusable machine learning pipelines on Kubernetes.

**Detailed**: Data Science Pipelines (based on upstream Kubeflow Pipelines) is the core ML workflow orchestration engine within Red Hat OpenShift AI. It provides a platform for building and deploying portable, scalable machine learning workflows based on containers. The system uses Argo Workflows as the underlying workflow execution engine to orchestrate Kubernetes resources for pipeline steps.

The component consists of several microservices: an API server that exposes both gRPC (v1beta1/v2beta1) and REST APIs for managing pipelines, experiments, runs, and recurring runs; a persistence agent that watches Argo Workflow execution and reports status back to the API server; a scheduled workflow controller that manages cron-based recurring pipeline runs; a frontend UI for pipeline visualization and management; a metadata gRPC server (ML Metadata) for artifact and execution lineage tracking; a cache server for pipeline step caching; and v2 driver/launcher binaries that execute within pipeline pods to manage I/O and artifact handling.

The RHOAI build uses Konflux with FIPS-compliant Go builds (`strictfipsruntime`), UBI 9 base images, and supports disconnected/air-gapped deployments. The system stores pipeline definitions in a relational database (MySQL or PostgreSQL) and pipeline artifacts in S3-compatible object storage (MinIO or external).

| metadata-envoy-deployment | Envoy Proxy | gRPC-Web proxy for ML Metadata; exposes metadata gRPC as HTTP |

| kfp-launcher | Go Binary (injected) | V2 pipeline launcher; runs inside Argo Workflow pods to manage artifact I/O with object storage |
