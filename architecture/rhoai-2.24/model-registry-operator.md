## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Model Registry instances for ML model metadata storage on RHOAI/ODH.

**Detailed**: The Model Registry Operator is a controller-runtime based Go operator that manages `ModelRegistry` custom resources. Each `ModelRegistry` CR represents a model registry instance backed by either PostgreSQL or MySQL for ML metadata storage using the ML Metadata (MLMD) gRPC server and a REST proxy service.

The operator reconciles `ModelRegistry` CRs by creating and managing Deployments (with REST proxy and optional oauth-proxy sidecar containers), Services, ServiceAccounts, RBAC resources, OpenShift Routes (for external access via oauth-proxy), and NetworkPolicies. On OpenShift clusters, it additionally creates user Groups and RoleBindings for access control. The operator supports optional Istio service mesh integration (for Authorino-based auth, though this is being deprecated in favor of oauth-proxy), and an optional Model Catalog feature that deploys a separate catalog service per registry instance.

The operator also handles CRD storage version migration from v1alpha1 to v1beta1 using either the StorageVersionMigration API or a manual fallback strategy. It is deployed by the rhods-operator via kustomize overlays, with container image references injected through params.env replacements at deploy time.
