## Purpose
**Short**: The ODH Dashboard is the web-based management console for Red Hat OpenShift AI, providing a unified UI for data scientists and administrators to manage notebooks, model serving, pipelines, and platform configuration.

**Detailed**: The ODH Dashboard is a full-stack web application consisting of a Node.js/Fastify backend and a React/PatternFly frontend. It serves as the primary user interface for Red Hat OpenShift AI (RHOAI), enabling data scientists to create and manage Jupyter notebooks (workbenches), deploy and monitor machine learning models through KServe and ModelMesh, manage data science pipelines, and configure model registries.

The backend acts as an API gateway, proxying authenticated requests to the Kubernetes API server on behalf of users. It uses the `@kubernetes/client-node` library to interact with cluster resources and forwards user access tokens (obtained via OpenShift OAuth) to the Kubernetes API for authorization. The backend also provides WebSocket proxying for real-time Kubernetes watch operations, a module federation system for integrating external UI components (e.g., Model Registry UI), and Prometheus metrics querying capabilities.

The dashboard is deployed as a Kubernetes Deployment with 2 replicas, fronted by an OpenShift oauth-proxy sidecar container that handles authentication via OpenShift OAuth. The deployment is managed by the rhods-operator using kustomize overlays. The dashboard defines and manages several Custom Resource Definitions (CRDs) including `OdhDashboardConfig`, `OdhApplication`, `OdhDocument`, `OdhQuickStart`, `AcceleratorProfile`, and `HardwareProfile`.

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| rhods-dashboard | Route (OpenShift) | https://rhods-dashboard-{namespace}.apps.{cluster} | 443/TCP | HTTPS | TLS (reencrypt) | reencrypt | External |

The dashboard uses the OpenShift Route with `reencrypt` TLS termination. The Route terminates external TLS and re-encrypts traffic to the oauth-proxy sidecar on port 8443 using the service serving certificate. HSTS is enforced with `max-age=31536000;includeSubDomains;preload`. HTTP to HTTPS redirect is configured via `insecureEdgeTerminationPolicy: Redirect`.

### Traffic Flow

```
External User
  → OpenShift Route (443/TCP, TLS termination + reencrypt)
    → oauth-proxy sidecar (8443/TCP, HTTPS, OpenShift OAuth authentication)
      → odh-dashboard backend (8080/TCP, HTTP, localhost)
        → Kubernetes API Server (HTTPS, user access token forwarded)
        → Thanos Querier (9092/TCP, HTTPS, for Prometheus metrics)
```

## Kustomize Deployment Structure

The dashboard is deployed via the rhods-operator using kustomize overlays:

- **RHOAI Self-Managed (onprem)**: `manifests/rhoai/onprem/kustomization.yaml`
- **RHOAI Managed (addon)**: `manifests/rhoai/addon/kustomization.yaml`
- **ODH**: `manifests/odh/`

The RHOAI onprem overlay includes:
- Common resources (CRDs, ISV apps, connection types)
- Shared RHOAI base (deployment, RBAC, service, route, OAuth secrets)
- RHOAI-specific ISV applications (Starburst Enterprise)
- OdhDashboardConfig with default notebook sizes and model server sizes
- ConsoleLink for OpenShift console integration

Key kustomize variables injected by the operator:
- `odh-dashboard-image`: Container image for the dashboard (RELATED_IMAGE_ODH_DASHBOARD_IMAGE)
- `section-title`: Console menu section name
- `dashboard-url`: Dashboard URL for ConsoleLink

All resource names are patched from `odh-dashboard` to `rhods-dashboard` in the RHOAI overlay.

## Container Image Build

The dashboard is built using Konflux (preferred) via `Dockerfile.konflux`:
- **Base image**: `registry.access.redhat.com/ubi9/nodejs-20`
- **Multi-stage build**: Builder stage compiles TypeScript, runtime stage runs the application
- **RHOAI branding**: Environment variables set Red Hat OpenShift AI logo, product name, doc links
- **Runtime user**: UID 1001 (non-root)
- **Entrypoint**: `npm run start` (runs `node ./dist/server.js`)
