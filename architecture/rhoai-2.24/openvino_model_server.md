## Purpose
**Short**: High-performance AI model inference server built on Intel OpenVINO, providing gRPC and REST APIs compatible with TensorFlow Serving, KServe v2, and OpenAI APIs.

**Detailed**: OpenVINO Model Server (OVMS) is a high-performance inference serving runtime implemented in C++ for scalability, optimized for deployment on Intel architectures. It serves AI models via multiple API frontends: TensorFlow Serving (TFS) API, KServe v2 Inference Protocol, and OpenAI-compatible generative AI APIs (v3). The server supports a wide range of model frameworks including OpenVINO IR, TensorFlow, PaddlePaddle, and ONNX formats.

OVMS provides advanced features including DAG (Directed Acyclic Graph) pipelines for chaining multiple models, MediaPipe graph execution, model versioning with configurable version policies, dynamic batching and reshaping, stateful model support for sequence processing, and metrics export compatible with Prometheus. Models can be loaded from local storage, AWS S3, Google Cloud Storage (GCS), Azure Blob Storage, or pulled from HuggingFace Hub.

Within the RHOAI platform, OVMS serves as the OpenVINO-based model serving runtime, typically deployed as an InferenceService backend through KServe/ModelMesh to provide optimized inference on Intel CPU and GPU hardware. It supports generative AI workloads including LLM text generation with continuous batching, text embeddings, reranking, and image generation through its MediaPipe-based graph execution pipeline.

| (REST listener) | In-container | configurable (default 0 = disabled)/TCP | N/A | HTTP/1.1 | plaintext | None | Internal (pod) |

**Notes:**
- gRPC port defaults to 0 (disabled). Must be explicitly set via `--port`.
- REST port defaults to 0 (disabled). Must be explicitly set via `--rest_port`.
- Bind addresses default to `0.0.0.0` for both gRPC (`--grpc_bind_address`) and REST (`--rest_bind_address`).
- gRPC max receive/send message size is 1GB.
- gRPC memory quota configurable via `--grpc_memory_quota` (default 2GB).
- TLS/auth is not handled at the application level; expected to be provided by the platform (KServe/ModelMesh/Service Mesh).
