## Purpose
**Short**: TrustyAI is a Java-based AI fairness, bias tracking, data drift detection, and model explainability service for ML models served via ModelMesh and KServe on OpenShift AI.

**Detailed**: TrustyAI Explainability provides a comprehensive suite of responsible AI capabilities for the Red Hat OpenShift AI (RHOAI) platform. It is deployed as a Quarkus-based microservice that integrates with model serving infrastructure (ModelMesh and KServe) to intercept inference payloads, store them for analysis, and compute fairness metrics (Statistical Parity Difference, Disparate Impact Ratio), data drift metrics (KS Test, Fourier MMD, Meanshift), and local/global explanations (LIME, SHAP, Counterfactual, Partial Dependence Plot).

The service operates as a sidecar-like component within the model serving namespace, consuming inference data via REST (for ModelMesh) or CloudEvents/Knative (for KServe). It stores inference data in configurable backends (PVC flat files, MinIO object storage, or MariaDB/MySQL databases) and exposes Prometheus-compatible metrics for scheduled fairness and drift computations. In RHOAI 2.24, the service is deployed via the TrustyAI Service Operator which manages its lifecycle.

The library portion (`explainability-core`) provides the algorithmic foundations for explainability, while `explainability-connectors` provides KServe v1/v2 protocol integration (both HTTP and gRPC), and `explainability-service` packages everything as a Quarkus application with REST endpoints.
