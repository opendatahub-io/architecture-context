## Purpose
**Short**: Kueue is a Kubernetes-native job queueing system that manages when workloads should be admitted to start based on quotas, priorities, and fair sharing policies.

**Detailed**: Kueue provides a set of APIs and a controller for job-level resource management in Kubernetes clusters. It acts as a job-level manager that decides when a job should be admitted to start (allowing pod creation) and when it should stop (deleting active pods), based on configured quotas, priorities, and fair sharing policies.

The controller watches for workloads submitted to LocalQueues, evaluates them against ClusterQueue resource quotas and admission checks, and admits workloads when resources are available. It supports multiple job frameworks including batch/Job, Kubeflow training jobs (PyTorchJob, TFJob, MPIJob, etc.), Ray jobs/clusters, JobSets, and CodeFlare AppWrappers. Kueue implements sophisticated scheduling features including partial admission, preemption, lending limits, cohort-based resource sharing, topology-aware scheduling, and multi-cluster job dispatch via MultiKueue.

In the RHOAI deployment, Kueue is deployed into the `opendatahub` namespace by the rhods-operator using kustomize overlays. The RHOAI configuration disables VisibilityOnDemand and MultiKueue features while enabling LocalQueueMetrics. Webhook namespace selectors are configured to exclude OpenShift system namespaces from job mutation/validation.
