## Purpose
**Short**: Llama Stack is a unified API server framework for building and deploying AI applications using Meta's Llama models and other LLM providers.

**Detailed**: Llama Stack provides a standardized set of REST APIs for inference, safety, agents, vector I/O, evaluation, scoring, telemetry, post-training, and tool execution. It acts as a distribution layer that abstracts over multiple backend providers (both inline/local and remote), enabling a pluggable architecture where different inference engines (vLLM, Ollama, TGI, etc.), vector stores (FAISS, Chroma, Qdrant, etc.), and safety mechanisms can be swapped in and out via YAML configuration.

The server is built on FastAPI/Starlette and supports both synchronous request-response and streaming (SSE) patterns. It includes middleware for authentication (OAuth2 JWT / custom endpoint / Kubernetes TokenReview), rate limiting/quota management, distributed tracing (OpenTelemetry), and client version negotiation. The distribution model supports building container images, conda environments, or virtual environments with the exact set of providers needed for a deployment.

In RHOAI, this component provides the Llama Stack server that enables users to interact with Llama models through a standardized API, with the `remote-vllm` template being the primary deployment pattern connecting to vLLM serving backends.

| Streamlit UI | Python Web App | Optional web UI for interacting with the Llama Stack server (port 8501) |

### Access Control (ABAC)

The system implements attribute-based access control with Cedar-like policy rules:
- **Actions**: create, read, update, delete
- **Principals**: User identifiers from auth provider
- **Resources**: Type-qualified identifiers (e.g., `model::my-model`, `vector_db::*`)
- **Conditions**: `when`/`unless` clauses supporting attribute matching, ownership checks
- Rules evaluated in order; first match wins; default deny if no match
