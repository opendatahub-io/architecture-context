## Purpose
**Short**: Manages the lifecycle of Jupyter Notebook server workloads on OpenShift via the `Notebook` CRD (kubeflow.org).

**Detailed**: This repository contains two complementary Go controllers that together manage Kubeflow Notebook workbenches in Red Hat OpenShift AI (RHOAI). The **kf-notebook-controller** (upstream Kubeflow controller) watches `Notebook` CRD instances and reconciles them into StatefulSets, Services, and optionally Istio VirtualServices. It also includes an idle-culling controller that scales notebooks to zero replicas after a configurable idle period.

The **odh-notebook-controller** (OpenShift-specific controller) extends the upstream behavior with OpenShift-native concerns: injecting an `oauth-proxy` sidecar for authentication via OpenShift OAuth, creating TLS-terminated Routes for external access, managing NetworkPolicies per notebook, handling trusted CA certificate bundles, syncing pipeline runtime image ConfigMaps from ImageStreams, and creating Elyra pipeline configuration secrets from DataSciencePipelinesApplication CRs. It also provides a mutating webhook that injects sidecar containers, volumes, and environment variables at Notebook creation/update time.

Both controllers are deployed as separate Deployments managed by the rhods-operator via kustomize overlays. The kf-notebook-controller uses the `overlays/openshift` overlay, and the odh-notebook-controller uses its own kustomize configuration. The notebook images are built using Konflux with FIPS-compliant Go builds.

|------|------|-------|------|----------|------------|----------|----------|

**Note**: This is an RHOAI 2.x component using the Route + oauth-proxy pattern for notebook ingress. Each Notebook CR gets an OpenShift Route created by the odh-notebook-controller. When OAuth injection is enabled (annotation `notebooks.opendatahub.io/inject-oauth: "true"`), the webhook injects an `oauth-proxy` sidecar on port 8443/TCP and the Route uses TLS reencrypt termination. When OAuth is disabled, a standard edge-terminated Route is created pointing to the notebook Service on port 80.
