## Purpose
**Short**: ModelMesh Serving Controller manages the lifecycle of ModelMesh multi-model serving deployments on Kubernetes/OpenShift, orchestrating model placement, runtime management, and inference service endpoints.

**Detailed**: ModelMesh Serving is a Kubernetes controller that manages the deployment and lifecycle of ModelMesh, an intelligent model serving layer that enables efficient multi-model serving. The controller watches custom resources (ServingRuntime, Predictor, InferenceService, ClusterServingRuntime) and reconciles the corresponding Kubernetes Deployments that host ModelMesh sidecar containers alongside model runtime containers (e.g., Triton, MLServer, etc.).

The controller dynamically constructs runtime Deployments by injecting a ModelMesh sidecar container, an optional REST proxy sidecar, an optional storage helper (puller) init/sidecar container, and an OAuth proxy sidecar for authentication. It manages the Service that fronts inference traffic, handles scale-to-zero functionality for idle runtimes, and coordinates with an etcd cluster for distributed model state management. The controller also manages HPA (Horizontal Pod Autoscaler) resources for automatic scaling of runtime deployments.

In the RHOAI distribution, the controller is deployed via kustomize overlays applied by the rhods-operator. It operates in cluster-scope mode, watching namespaces labeled with `modelmesh-enabled=true` and managing model serving infrastructure across those namespaces.
