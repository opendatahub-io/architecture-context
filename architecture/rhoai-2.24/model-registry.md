## Purpose
**Short**: A central repository for storing and managing ML model metadata, versions, artifacts, experiments, and inference service state.

**Detailed**: The Model Registry is a Go-based REST API server that provides a centralized metadata store for ML workflows. It replaces the CPP gRPC service from the ml-metadata project with a higher-level REST API, supporting operations on registered models, model versions, artifacts, serving environments, inference services, serve models, experiments, and experiment runs. The server uses an embedded metadata store (EmbedMD) backed by either MySQL or PostgreSQL databases.

The repository also includes an InferenceService controller that watches KServe InferenceService custom resources and synchronizes their metadata into the Model Registry via its REST API. This enables automatic tracking of deployed inference services, including their serving URLs and deployment state. Additionally, a Model Catalog service provides a read-only API for browsing models from external catalog sources (e.g., HuggingFace).

The component is built using Konflux with FIPS-compliant Go runtime (`strictfipsruntime` build tag). It supports Istio service mesh integration with mTLS via DestinationRule, VirtualService routing through Kubeflow gateway, and AuthorizationPolicy. A UI component is also deployable via kustomize overlays.
