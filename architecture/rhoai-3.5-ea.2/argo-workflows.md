## Purpose

**Short**: Argo Workflows is a container-native workflow engine for orchestrating parallel jobs on Kubernetes, used as the execution backbone for Data Science Pipelines in RHOAI.

**Detailed**: Argo Workflows implements a Kubernetes CRD-based workflow engine that defines and executes multi-step computational workflows where each step runs as a container. It supports DAG-based and step-based execution models, scheduled workflows via CronWorkflow, workflow templates (namespace-scoped and cluster-scoped), artifact management with multiple storage backends (S3, GCS, Azure Blob, HDFS, Git, HTTP, Artifactory, Alibaba OSS), and a REST/gRPC API server with a web UI.

In the RHOAI platform, Argo Workflows is deployed as part of the Data Science Pipelines component. The platform ships two container images from this repository: the **workflow-controller** (the Kubernetes controller that reconciles Workflow CRDs and orchestrates pod execution) and **argoexec** (the executor sidecar injected into workflow pods for artifact staging, output collection, and container lifecycle management). The Argo Server (CLI/API/UI) is not shipped as a separate Konflux image — only the controller and executor are part of the RHOAI build pipeline.

The workflow-controller manages the full lifecycle of workflows: scheduling, pod creation with init/wait sidecar containers, artifact handling, caching/memoization, synchronization via semaphores, garbage collection, and optional archival to SQL databases. It supports leader election for HA deployments and exposes Prometheus metrics for observability.

## Architectural Analysis

Argo Workflows in the RHOAI context serves as the execution engine underneath Data Science Pipelines. The two shipped components — workflow-controller and argoexec — form a clean separation of concerns: the controller manages orchestration logic (scheduling, dependency resolution, resource lifecycle) while the executor handles the data plane (artifact staging, container wrapping, output collection). The emissary executor pattern is particularly elegant — argoexec wraps the user's container command, managing inter-container dependencies within ContainerSet templates by polling exit code files on a shared `/var/run/argo` volume, eliminating the need for a Docker-in-Docker or privileged sidecar.

The FIPS compliance story is noteworthy. Recent commits (736b2245c, 5363dfa59) migrated from OpenSSL FIPS to native Go FIPS 140-3 modules, moving the FIPS configuration from Dockerfiles to `go.mod` via `godebug fips140=on`. The argoexec Dockerfile uniquely builds two binary variants — a FIPS-enabled `argoexec-fips` (CGO_ENABLED=1 with strictfipsruntime) and a standard `argoexec` (CGO_ENABLED=0, statically linked) — both shipped in the same image. This dual-binary approach allows runtime selection based on the cluster's FIPS requirements. The workflow-controller builds only the FIPS variant. However, there are application-level FIPS gaps: the agent executor's HTTP client defaults to `InsecureSkipVerify: true` (workflow/executor/agent.go:303), and the SSO configuration exposes an `InsecureSkipVerify` option (config/sso.go:23) — both of which bypass TLS verification entirely when enabled.

The controller's architecture follows established Kubernetes operator patterns — informer-based watches with workqueue decoupling, deterministic pod naming for idempotency, JSON Merge Patch for conflict-free updates, and keylock per-workflow to prevent concurrent reconciliation. The semaphore/synchronization system is particularly interesting, using ConfigMaps as distributed locks with a notification-based queueing delay to avoid thundering herd problems. The memoization cache, also backed by ConfigMaps, enables workflow step result caching with TTL-based expiration.

The `rhoai/` directory contains legacy Dockerfiles (UBI8, Go 1.21, no FIPS) that appear to be from an earlier build system and are now superseded by the Konflux Dockerfiles. The Konflux pipeline (`.tekton/`) produces the production images with SHA-pinned UBI9 base images and full FIPS enablement.
