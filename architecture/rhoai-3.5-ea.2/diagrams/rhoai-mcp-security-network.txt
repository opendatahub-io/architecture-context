Authentication Modes:
- Single-user: SA token identity for all K8s operations
- OIDC JWT: JWT validated via JWKS → user impersonation → SubjectAccessReview
- OIDC TokenReview: Opaque OAuth token → K8s TokenReview → user impersonation → SubjectAccessReview

Safety Controls:
- read_only_mode (default: true in RHOAI): Blocks create/update/delete/patch MCP tools
- enable_dangerous_operations (default: false): Gates delete operations
- Workflow tokens (HMAC-SHA256): Multi-step confirmation for destructive actions

Secrets:
- workflow-hmac-secret (Opaque) - Auto-generated 32-byte hex per process (ephemeral)
- hf-credentials (Opaque) - User-created HuggingFace token
- S3 data connections (Opaque) - User-created AWS S3 credentials

NetworkPolicy:
- Allows rhoai-mcp pod → model-catalog namespace on ports 8080/8443/443
- Namespace: deployment namespace (redhat-ods-applications or opendatahub)

Crypto Notes:
- TLS: Delegated to system OpenSSL (UBI9 base, FIPS-capable)
- JWT: PyJWT[crypto] for token validation
- HMAC: hashlib-based HMAC-SHA256 for workflow tokens
- Non-FIPS risk: hashlib.md5 used for cosmetic name suffixes only (not security-relevant)
- Configurable skip_tls_verify for in-cluster Model Registry connections
