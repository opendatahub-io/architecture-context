- Model Serving: Bearer Token (OPENAI_API_KEY) — enforced in main.py:488-494
- HuggingFace Hub: Bearer Token (HF_TOKEN) — optional, main.py:498-502
- EvalHub Callback: Bearer Token — via eval-hub-sdk DefaultCallbacks
- OCI Registry: Registry credentials — via skopeo / eval-hub-sdk
- S3 Storage: AWS IAM credentials — via boto3

SECURITY FEATURES:
- Error message sanitization: regex-based redaction of credentials in exception text
  (Authorization headers, key=value pairs, JSON credential fields, URL userinfo)
  Prevents credential leakage through EvalHub status reporting channel
- Path traversal protection: CWE-22 mitigation in _resolve_job_spec_path_for_read()
  Validates resolved paths remain under /meta
- TLS verification: Configurable via verify_certificate (True/False/CA cert path)
  Defaults to system trust store validation

FIPS COMPLIANCE:
- Python application on UBI 9 — inherits FIPS-compliant OpenSSL from base image
- CPython links against system OpenSSL for all TLS operations
- No non-FIPS crypto detected (no MD5, RC4, custom RNG for crypto purposes)
- FIPS-compliant when host is in FIPS mode (no explicit opt-in required)
