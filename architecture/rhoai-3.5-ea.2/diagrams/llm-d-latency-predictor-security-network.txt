SECURITY CONCERNS:
══════════════════
1. NO AUTHENTICATION on any endpoint (training or prediction)
2. NO TLS — all communication is plaintext HTTP
3. NO RBAC — component does not interact with Kubernetes API
4. NO SECRETS — all configuration via plaintext ConfigMap env vars
5. Training data ingestion (/add_training_data_bulk) is unauthenticated
   → Adversarial training data injection possible with network access
6. Model download path is plaintext HTTP with no integrity verification
   → MD5 used only for change detection, not cryptographic integrity
7. training-service-external LoadBalancer exposes training server externally
   → Reference manifests only; RHOAI production uses platform-managed ingress

CRYPTO USAGE:
═════════════
- hashlib.md5: Model file change detection (non-security, non-FIPS concern)
- No TLS configuration in application code
- OpenSSL available via UBI9 base image but unused by application

RBAC Summary:

- No ClusterRoles defined
- No RoleBindings defined
- No ServiceAccount RBAC configured
- Component does not access Kubernetes API

Secrets:
════════
- No secrets defined
- All configuration via environment variables in ConfigMaps

Network Policies:
═════════════════
- No NetworkPolicy resources defined in reference manifests
- RHOAI production: Expected to use platform-provided NetworkPolicy + service mesh mTLS

FIPS Compliance Notes:
══════════════════════

- MD5 usage is non-cryptographic (model change detection only)
- Not a Go binary (no CGO_ENABLED/goboring concerns)
