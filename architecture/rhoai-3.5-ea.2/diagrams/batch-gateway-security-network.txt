RBAC Summary:
- No Kubernetes RBAC required — component does not interact with K8s API server
- All state management via PostgreSQL and Redis
- Authentication/Authorization delegated to external gateway (Kuadrant/Authorino)
- Multi-tenancy enforced at data layer (tenant-scoped queries, SHA256 folder names)

Secrets:
- batch-gateway-secrets (Opaque)
  - redis-url: Redis/Valkey connection URL with credentials
  - postgresql-url: PostgreSQL connection URL with credentials
  - inference-api-key: Bearer token for downstream inference endpoints
  - s3-secret-access-key: AWS S3 secret access key
  - Provisioned: Manual (kubectl create secret)
  - Auto-Rotate: No
  - Mount: read-only at /etc/.secrets/ (accessed via os.OpenInRoot to prevent path traversal)

- {release}-batch-gateway-apiserver-tls (kubernetes.io/tls) [Optional]
  - TLS certificate for API server HTTPS
  - Provisioned: cert-manager or manual
  - Auto-Rotate: Yes (if cert-manager)

Pod Security Context (all three components):
- runAsNonRoot: true
- readOnlyRootFilesystem: true
- allowPrivilegeEscalation: false
- capabilities: drop ALL
- seccompProfile: RuntimeDefault

- Build flags: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
