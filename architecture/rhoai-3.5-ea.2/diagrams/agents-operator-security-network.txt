  - SCC: kagenti-authbridge (allows csi.spiffe.io CSI volumes for SPIRE)

Service Mesh Configuration:
- Istio ambient mesh enrollment via namespace label: istio.io/dataplane-mode=ambient
- mTLS via SPIFFE/SPIRE (not Istio PeerAuthentication)
- AuthBridge sidecar handles mTLS independently using go-spiffe X509Source

Network Policies:
- bundle-service: Only pods with label kagenti.dev/authbridge=true can access 8080/TCP
- webhook-service: API server can reach 9443/TCP
- Agent NetworkPolicies (dynamic): Verified agents get permissive policy;
  unverified agents get restrictive (K8s API only)

Secrets:
- webhook-server-cert (kubernetes.io/tls) - Webhook TLS, cert-manager, auto-rotate
- keycloak-initial-admin (Opaque) - Keycloak admin creds, pre-provisioned, no rotation
- keycloak-db-secret (Opaque) - PostgreSQL password, operator-generated, no rotation
- {ns}-{workload}-keycloak-creds (Opaque) - Per-agent OAuth2 client creds, operator-managed
- {workload}-tls-bridge-ca (kubernetes.io/tls) - Per-agent TLS bridge CA, cert-manager, 90d/15d
- cacerts (Opaque) - Istio intermediate CA certs, SharedTrustReconciler, auto on rotation

OpenShift SCC:
- kagenti-authbridge SCC: Allows CSI volumes (csi.spiffe.io), NET_ADMIN for iptables,
  RunAsUser/RunAsGroup 1337 (proxy UID), SELinux MustRunAs
