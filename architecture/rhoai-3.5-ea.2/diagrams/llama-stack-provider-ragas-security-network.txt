Authentication Summary:
  - Llama Stack Eval API: No built-in auth (relies on platform-level auth)
  - Kubeflow Pipelines: Bearer Token (OpenShift OAuth)
    Source: KUBEFLOW_PIPELINES_TOKEN env var OR kubeconfig api_key["authorization"]
  - S3 Storage: AWS IAM credentials from Kubernetes Secret
  - Kubernetes API: ServiceAccount token (in-cluster) or kubeconfig (external)

Security Considerations:
  - Llama Stack Server exposes HTTP (no TLS) on port 8321 - relies on service mesh
    or network policy for encryption in transit
  - KFP bearer token has no refresh mechanism (TODO in source code)
  - KFP healthz pre-flight uses default requests TLS (system trust store, no custom CA)
  - Python on UBI 9 inherits FIPS compliance from system OpenSSL
  - No non-FIPS crypto patterns detected (no MD5, RC4, DES, custom crypto)
  - Base image UBI 9 Python 3.12 (digest-pinned) but not AIPCC base image
