================================================================================
AUTHENTICATION FLOW SUMMARY
================================================================================

1. Dashboard Access (OAuth2/OIDC):
   Client → HTTPS :443 → Platform Gateway (Envoy)
     → kube-auth-proxy (OAuth2 session management)
     → Lua EnvoyFilter (cookie → Bearer token conversion)
     → kube-rbac-proxy :8443 (SubjectAccessReview)
     → odh-dashboard :8080

2. Model Inference (Kuadrant AuthPolicy):
   Client → HTTPS :443 → Platform Gateway (Envoy)
     → Kuadrant AuthPolicy (API Key via HTTP callback or TokenReview)
     → HTTPRoute → InferencePool
     → llm-d EPP (KV cache-aware pod selection)
     → llm-d Router (ext_proc reverse proxy)
     → vLLM pod :8080

3. MaaS External Provider (API Key + ext_proc):
   Client → HTTPS :443 → Platform Gateway (Envoy)
     → Kuadrant AuthPolicy (API Key SHA-256 validation)
     → ext_proc pre-auth (model name → HTTP header)
     → ext_proc post-auth (format translation + provider key injection)
     → ExternalName Service → External LLM Provider

4. Agent-to-Agent (SPIFFE/SPIRE + AuthBridge):
   Agent Pod → AuthBridge sidecar (JWT validation)
     → Token exchange (SPIFFE identity)
     → OPA policy evaluation
     → mTLS to target agent

5. Pipeline Access (kube-rbac-proxy):
   Client → kube-rbac-proxy :8443 (SubjectAccessReview)
     → DSP API Server :8888 (REST) / :8887 (gRPC)

================================================================================
SECURITY POSTURE NOTES
================================================================================

- mTLS Enforcement: MIXED
  - agents-operator uses SPIFFE/SPIRE for mTLS between agents
  - rhods-operator DestinationRule for kube-auth-proxy uses insecureSkipVerify: true
  - Most inter-component communication relies on kube-rbac-proxy TLS termination

- FIPS Compliance: PARTIAL
  - Go components: strictfipsruntime build tags with CGO_ENABLED=1
  - Python components: UBI9 system OpenSSL
  - Known gaps: KServe module controller (CGO_ENABLED=0), EPP (no strictfipsruntime)

- TLS Versions:
  - Ingress: TLS 1.3 (Platform Gateway)
  - kube-rbac-proxy: TLS 1.2+ (recently enforced MinVersion)
  - llm-d inference scheduler sidecar: TLS 1.2+ with curated cipher suites
  - EPP: defaults to TLS 1.0+ (gap noted)
  - Some metrics scraping: InsecureSkipVerify: true

- Container Security:
  - All containers: Non-root UIDs (1001 or 65532)
  - All containers: RuntimeDefault seccomp profiles
  - 96+ Dockerfiles: UBI9 base images
  - 28 Dockerfiles: Distroless/static final images
  - All capabilities dropped
