RBAC Summary:
- Training Hub does not define its own Kubernetes RBAC resources
- RBAC for training jobs is managed by the orchestrating RHOAI platform component
- The training job pod inherits RBAC from its ServiceAccount (set by orchestrator)

Secrets / Credentials:
- Training Hub does not define or manage Kubernetes Secrets
- All credentials are passed via environment variables or function parameters:
  * HF_TOKEN: HuggingFace Hub access (optional, for gated models)
  * WANDB_API_KEY: Weights & Biases experiment tracking
  * MLFLOW_TRACKING_TOKEN: MLflow server authentication
  * LLM API keys: Provider-specific keys passed via litellm configuration
  * vLLM API Key: Optional authentication for vLLM inference server

FIPS Compliance:
- Training Hub is a Python library, not a standalone container image
- No direct TLS configuration; delegates to underlying libraries
- No direct crypto imports
- FIPS compliance is inherited from the container image in which it runs
- OpenSSL/TLS behavior determined by host OS crypto policies

Supply Chain Security:
- litellm versions 1.82.7 and 1.82.8 explicitly excluded (!=1.82.7,!=1.82.8)
- These versions were compromised in the TeamPCP supply chain attack (2026-03-24)
- No dependency lock file in this repository (pyproject.toml specifies constraints only)

Build Hermeticity:
- No Dockerfile in this repository (pure Python library)
- No lock files (pip/setuptools, no poetry.lock or uv.lock)
- Version constraints in pyproject.toml but no exact pinning
- Lock files may exist in downstream container image repos that include training-hub
