- Data Connection secret (Opaque) - AWS S3 credentials
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_S3_ENDPOINT,
           AWS_S3_BUCKET, AWS_DEFAULT_REGION
  - Provisioned by: OpenShift AI Data Connection
  - Auto-rotate: No

Authentication & Authorization:
- Target LLM /v1/*: API Key (Bearer) via model-auth Secret
  - Enforcement: Application code (resolve_api_key)
  - Key read from volume mount at /mnt/model-auth or /var/run/secrets/model/

- KFP API: SA Token or KFP_AUTH_TOKEN
  - Enforcement: KFP Client (kfp.Client)
  - SA token from /var/run/secrets/kubernetes.io/serviceaccount/token

- S3 Storage: AWS IAM credentials
  - Enforcement: boto3 client
  - Credentials from Data Connection K8s Secret

Security Notes:
- TLS verification defaults to enabled (verify=True)
- utils.get_http_client_with_tls() supports disabling TLS via config
  ("false", "0", "no", "off") — for dev environments only
- KFP client supports verify_ssl=False and custom ssl_ca_cert for
  air-gapped OpenShift with non-standard PKI
- API keys are NEVER embedded in garak config sent to KFP
  - __FROM_ENV__ placeholders used, resolved at pod startup
  - config.json always redacted before upload to S3/OCI
- FIPS: OpenSSL inherited from AIPCC RHEL 9.6 UBI base image
- No direct MD5, RC4, DES, or custom cipher usage detected
