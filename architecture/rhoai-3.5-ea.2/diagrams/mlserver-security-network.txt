RUNTIME SECURITY MODEL:
  Production Mode (when /etc/mlserver/trusted-runtimes.json exists):
  - Only runtimes in allowlist can be loaded
  - Custom environment tarballs: BLOCKED
  - environment_path (sys.path injection): BLOCKED
  - Wildcard CORS: BLOCKED
  - Validation: at settings parse time AND property access time (defense-in-depth)
  - Import path validation: regex requires dotted paths with uppercase class names

  Development Mode (no allowlist file):
  - Any Python class can be loaded as model runtime
  - Dynamic imports from model directories permitted

RBAC Summary:
  - MLServer does not define RBAC resources
  - Runs as non-root user (UID 1000) inside KServe-managed pod
  - Pod RBAC managed by KServe's service account

Secrets:
  - None consumed directly by MLServer
  - Model storage credentials managed by KServe (storage-config secret)
  - TLS certificates managed by platform (ingress/sidecar layer)

Container Security:
  - User: UID 1000 (non-root)
  - Base image: quay.io/aipcc/base-images/cpu:3.5.0-ea.2 (RHEL 9.6)
  - OpenSSL: Inherited from base image (FIPS-capable)
  - No direct crypto library usage at application level
  - Hermetic build: pip deps prefetched with SHA256 verification

Kubernetes Health Probes:
  - Liveness: GET /v2/health/live (8080/TCP)
  - Readiness: GET /v2/health/ready (8080/TCP)
