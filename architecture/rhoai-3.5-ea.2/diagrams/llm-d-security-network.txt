RBAC Summary:
- No ClusterRoles defined in this repository
- Namespace Role: interactive-pod-editor
  - Binding: interactive-pod-editor-binding
  - ServiceAccount: interactive-pod
  - Purpose: Interactive benchmarking/debugging access
- Model server pods: automountServiceAccountToken: false (no API access)

Secrets:
- hf-token (Opaque) - Hugging Face API token for model downloads, manual provisioning, no auto-rotate
- epp-token-secret (Opaque) - EPP authentication token (kustomize patch), manual provisioning, no auto-rotate

Authentication & Authorization:
- Container images ship WITHOUT authentication — auth is added at platform level
- RHOAI deployments inject kube-rbac-proxy sidecars or OAuth proxy via rhods-operator
- Envoy ↔ EPP communication is unauthenticated (co-located, localhost)
- EPP → Kubernetes API uses ServiceAccount token authentication
- Model downloads use Bearer token (HF_TOKEN) to Hugging Face Hub

FIPS Compliance:
- CUDA images: UBI 9 base with FIPS-validated OpenSSL, Python ssl delegates to system OpenSSL
- CPU images: Ubuntu 22.04 base — NO FIPS-validated crypto (gap for RHOAI)
- ROCm/XPU images: Third-party bases — FIPS status depends on upstream
- No Go binaries — Python-based (vLLM), crypto handled by system OpenSSL

Network Encryption Gaps:
- Model server API (8000/TCP): Plaintext HTTP — relies on platform-level TLS termination
- nixl KV transfer (5600/TCP): No encryption on RDMA/TCP — pod-to-pod within cluster
- Envoy → EPP (9002/TCP): No encryption — localhost/co-located
- Gateway ingress (80/TCP): HTTP by default — TLS configurable per gateway variant
- OTEL export (4317/TCP): No encryption by default — configurable
