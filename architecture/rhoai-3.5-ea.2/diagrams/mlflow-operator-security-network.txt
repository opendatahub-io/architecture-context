    Provisioned by: OpenShift service-ca (via Service annotation)
    Auto-Rotate: Yes (service-ca auto-rotation)

  - mlflow-artifact-connection (Opaque, per-namespace)
    Purpose: S3 credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_S3_BUCKET, etc.)
    Provisioned by: User (per-workspace)
    Auto-Rotate: No

Authentication & Authorization:
  - MLflow server: Kubernetes auth plugin (SelfSubjectAccessReview)
    Every API request must carry a valid Kubernetes token
    Server checks RBAC permissions on mlflow.kubeflow.org pseudo-resources
  - Operator /metrics: Bearer Token via controller-runtime authn/authz filter
    Requires authentication and authorization via TokenReview/SubjectAccessReview

FIPS Compliance:
  - Operator: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, dynamic linking, UBI9 base
  - MLflow server: UVICORN_SSL_CIPHERS=PROFILE=SYSTEM (system FIPS-compliant cipher profile)
  - HTTP/2: Explicitly disabled (NextProtos: ["http/1.1"])
  - TLS: stdlib crypto/tls → OpenSSL at runtime via strictfipsruntime
  - No non-FIPS crypto risks detected
