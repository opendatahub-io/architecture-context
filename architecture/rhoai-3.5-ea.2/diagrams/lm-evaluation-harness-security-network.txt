Secrets:
- Model auth secret (model.auth.secret_ref)
  - Type: Opaque
  - Contents: API key for model endpoint + optional hf-token for gated datasets
  - Provisioned by: User / TrustyAI operator
  - Auto-rotate: No
- git-auth
  - Type: kubernetes.io/basic-auth
  - Contents: Git credentials for Tekton pipeline source checkout
  - Provisioned by: Konflux / PaC
  - Auto-rotate: No
- OCI auth (docker config)
  - Type: kubernetes.io/dockerconfigjson
  - Contents: Registry credentials for OCI artifact push
  - Provisioned by: User / Platform
  - Auto-rotate: No

Security Controls:
- Error message sanitization: Strips Authorization headers, Bearer tokens,
  key=value secrets, JSON credential fields, URL userinfo, query parameters,
  and fragments before sending to external callbacks
- CA certificate path validation: Rejects symlinks for verify_certificate paths
  (mitigates symlink-based path traversal against TLS trust store)
- Job spec confinement: _resolve_job_spec_path_for_read() confines reads to /meta/
  (prevents CWE-22 path traversal)
- TLS verification: ssl.create_default_context() with custom CA bundle support;
  requests library verify= parameter; aiohttp custom SSL context

FIPS Compliance:
- Python application (no Go binaries)
- Uses system OpenSSL via UBI 9 base image for all TLS operations
- No direct crypto library imports (no pycryptodome, no custom cipher suites)
- No non-FIPS crypto risks detected
