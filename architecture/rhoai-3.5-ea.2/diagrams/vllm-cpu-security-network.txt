PLATFORM INTEGRATION:
- KServe / ModelMesh manages this container as a ServingRuntime
- Ingress is NOT managed by vllm-cpu; platform's Gateway API / HTTPRoutes handle external access
- TLS termination for external traffic is handled by the platform ingress layer (Gateway API / Envoy)
- gRPC server uses add_insecure_port() — TLS termination must be handled by platform sidecar/gateway

RBAC Summary:
- No ClusterRoles defined
- No RoleBindings defined
- No ServiceAccount privileges required
- Container runs as non-root user (UID 2000, GID 0)
- No Kubernetes API access from the container

Secrets:
- No secrets defined by the component
- HF_TOKEN: Injected by platform at runtime (for HuggingFace model downloads)
- TLS certificates: Injected by platform at runtime (--ssl-keyfile, --ssl-certfile)
- API key: Injected via --api-key CLI arg or VLLM_API_KEY env var

Authentication & Authorization:
- /v1/* endpoints: Optional API key via Bearer token (AuthenticationMiddleware)
- /health, /metrics, /version, /load: Always accessible, no auth
- gRPC (50051/TCP): No authentication

FIPS Compliance:
- Python application inherits FIPS from UBI 9 base image OpenSSL
- hashlib.md5 uses usedforsecurity=False (FIPS-compliant)
- gRPC uses system OpenSSL
- No custom cipher suites configured

Crypto Notes:
- TLS optional on HTTP server (Uvicorn --ssl-keyfile/--ssl-certfile)
- SSL cert hot-reload supported (--enable-ssl-refresh)
- Outbound HTTPS uses system trust store
- gRPC server: INSECURE port only (no TLS)

Build-Time Dependencies (not runtime egress):
- PyTorch CPU package index (download.pytorch.org/whl/cpu) — 443/TCP HTTPS
- IBM DevPI (wheels.developerfirst.ibm.com) — 443/TCP HTTPS (ppc64le only)
