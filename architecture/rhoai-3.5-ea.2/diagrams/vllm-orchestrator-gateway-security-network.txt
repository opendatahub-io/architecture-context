TLS Configuration:
- Gateway listener: Plaintext HTTP (8090/TCP) — no TLS termination
- Upstream connection: Optional mTLS to orchestrator
  - Probes for certs at /etc/tls/private/tls.crt and /etc/tls/private/tls.key
  - If present: HTTPS with mTLS (client cert + CA verification)
  - If absent: plaintext HTTP (graceful degradation)
  - CA bundle: /etc/tls/ca/service-ca.crt (Kubernetes service CA)
  - PKCS#12 conversion: PEM cert+key → PKCS#12 in-memory (native-tls requirement)
- FIPS: Delegates to system OpenSSL (openssl-libs RPM); RHEL 9 FIPS crypto-policies
  apply automatically
- Risk: danger_accept_invalid_hostnames(true) when orchestrator host is "localhost"
  (development-only; not reachable when orchestrator addressed by K8s Service DNS)

RBAC Summary:
- No RBAC manifests — this is a stateless HTTP service, not a Kubernetes operator
- No ClusterRoles, Roles, or RoleBindings defined in the repository
- Access control is expected to be enforced externally:
  - kube-rbac-proxy sidecar
  - Ingress controller auth
  - Network policies

Secrets:
- TLS client cert (/etc/tls/private/tls.crt, tls.key)
  - Type: kubernetes.io/tls
  - Purpose: mTLS client identity for orchestrator communication
  - Provisioned by: Kubernetes service-serving-cert-signer or cert-manager
  - Rotation: Depends on provisioner
- CA certificate (/etc/tls/ca/service-ca.crt)
  - Type: Opaque (CA bundle)
  - Purpose: Verify orchestrator's server certificate
  - Provisioned by: Kubernetes service CA configmap injection
  - Rotation: Auto-rotated by platform

Authentication & Authorization:
- Endpoint: /{route_name}/v1/chat/completions (POST)
- Auth mechanism: Authorization header pass-through
- Enforcement: None at gateway level
- Policy: Gateway forwards Authorization header to orchestrator; does not validate
- Recommendation: Deploy with kube-rbac-proxy sidecar or behind authenticated ingress

Build Security:
- Hermetic build: Yes (Tekton PipelineRun, hermetic: true)
- Prefetch: cargo + rpm (Hermeto/cachi2)
- Lock files: Cargo.lock (Rust deps), rpms.lock.yaml (OS packages)
- Multi-arch: x86_64, aarch64, ppc64le, s390x
- OpenSSL: Dynamic linking against system openssl-libs (FIPS-capable)
