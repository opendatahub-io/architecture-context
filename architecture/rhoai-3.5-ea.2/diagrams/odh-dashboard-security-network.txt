Authentication Flow Summary:
  1. User → Platform Gateway (OAuth session / TLS)
  2. Gateway → kube-rbac-proxy (X-Forwarded headers)
  3. kube-rbac-proxy → K8s API (TokenReview + SubjectAccessReview)
  4. kube-rbac-proxy → Backend (X-Auth-Request-User/Groups)
  5. Backend → BFF Sidecar (x-forwarded-access-token)
  6. BFF → Upstream (user_token)
  7. BFF → K8s API (SubjectAccessReview for namespace authz)
