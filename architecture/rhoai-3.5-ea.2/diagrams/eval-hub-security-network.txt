RBAC Summary:
- No RBAC manifests in repository
- RBAC is configured by the platform operator (rhods-operator) when deploying eval-hub
- The eval-hub service itself performs NO token validation
- Authentication is fully delegated to kube-rbac-proxy sidecar

Authentication & Authorization Summary:
- /api/v1/evaluations/*: X-Tenant + X-User headers via kube-rbac-proxy (cluster mode)
- /api/v1/health, /openapi.yaml, /docs: No authentication (public)
- /metrics (8081): No authentication (network-level isolation)
- eval-hub → MLflow: Bearer token (file-based, 5-min atomic cache)
- sidecar → eval-hub: K8s ServiceAccount token (auto-rotated by kubelet)
- sidecar → OCI: Docker config.json → OCI v2 auth challenge flow
- evalhub-mcp (cluster): X-Tenant + X-User via kube-rbac-proxy
- evalhub-mcp (local): No authentication (development only, -local flag)

Secrets:
- database credentials (Opaque) - DB password via file-mounted secret, provisioned by platform operator, no auto-rotate
- MLflow token (Opaque) - MLflow API auth at /var/run/secrets/mlflow/token, provisioned by platform operator, no auto-rotate
- S3 credentials (Opaque) - AWS access key/secret at /var/run/secrets/test-data/, provisioned by platform operator, no auto-rotate
- OCI registry credentials (kubernetes.io/dockerconfigjson) - Docker config at /etc/evalhub/.docker/config.json, provisioned by platform operator, no auto-rotate
- K8s ServiceAccount token (kubernetes.io/service-account-token) - Sidecar → eval-hub API auth, auto-rotated by kubelet

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1 (all 4 binaries)
- Base image: UBI9 ubi-minimal (OpenSSL included)
- TLS: MinVersion=TLS 1.2, system-default cipher suites
- No non-FIPS crypto detected (no MD5/RC4/DES usage)
- InsecureSkipVerify: configurable but defaults to false
- WARNING: Non-Konflux Containerfile uses CGO_ENABLED=0 (NOT FIPS-compliant, local dev only)

Security Notes:
- kube-rbac-proxy is a CRITICAL security dependency — eval-hub has no built-in authentication
- Local mode (-local flag) bypasses all identity headers — MUST NOT be used in production
- Multi-tenancy enforced at data layer (DB query scoping), not at API gateway
- Token caching uses atomic pointers with 5-min TTL (avoids repeated file I/O)
- No ingress resources — platform operator creates HTTPRoute via Gateway API
