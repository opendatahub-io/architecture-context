OPERATIONAL NOTES
═══════════════════════════════════════════════════════════════════════════

Namespace Scoping:
  - Cluster-scope (default): Watches all namespaces
  - Namespace-scope: Set NAMESPACE_SCOPE=true to limit to single namespace

High Availability:
  - Controller: 1-3 replicas with leader election
  - Runtime pods: Configurable via podsPerRuntime (default: 2)
  - etcd: 3+ replicas recommended for production

Model Storage:
  - S3-compatible (primary): AWS S3, MinIO, Ceph
  - PersistentVolumeClaim (alternative)
  - Credentials stored in storage-config secret

Security Hardening:
  - Runtime pods: restricted SCC, capabilities dropped (ALL)
  - Network policies: Restrict ingress to required ports
  - Optional mTLS: Requires service mesh (Istio) integration
  - Webhook validation: Validates ServingRuntime specifications

Monitoring:
  - Runtime pods: Port 2112/TCP (HTTP, no auth)
  - Controller: Port 8443/TCP (HTTPS, mTLS via kube-rbac-proxy)
  - Metrics: model load time, inference latency, request count, cache hit ratio
