Service Mesh Configuration (when Istio enabled):
=================================================
- USE_ISTIO environment variable enables VirtualService creation
- PeerAuthentication: mTLS STRICT mode (when Istio enabled)
- AuthorizationPolicy: Configured at Istio Gateway level
- Istio sidecars inject mTLS for pod-to-pod communication

Secrets:
========
- No secrets directly managed by controller
- ServiceAccount token auto-mounted at: /var/run/secrets/kubernetes.io/serviceaccount/token
- TLS certificates managed by Istio control plane (when enabled)

Configuration:
==============
ConfigMap: config
- USE_ISTIO: Enable/disable Istio VirtualService creation
- ISTIO_GATEWAY: Gateway name for VirtualService (default: kubeflow/kubeflow-gateway)
- ADD_FSGROUP: Add fsGroup:100 to pod security context

ConfigMap: notebook-controller-culler-config
- ENABLE_CULLING: Enable/disable idle notebook culling (default: false)
- CULL_IDLE_TIME: Minutes before idle notebook is culled (default: 1440 = 1 day)
- IDLENESS_CHECK_PERIOD: Check interval in minutes (default: 1)

Network Flows:
==============

1. Notebook Creation (User → K8s API → Controller → Notebook Pod):
   User/Dashboard → K8s API (6443/TCP HTTPS TLS1.2+ Bearer) →
   NotebookReconciler (watch event) →
   K8s API (6443/TCP HTTPS TLS1.2+ ServiceAccount) →
   Kubelet (10250/TCP HTTPS mTLS) →
   Notebook Pod (container runtime)

2. Idle Notebook Culling (Controller → Notebook Pod):
   CullingReconciler → Notebook Service (80/TCP HTTP no-auth) →
   Notebook Pod (8888/TCP HTTP GET /api/kernels, /api/terminals) →
   CullingReconciler (delete via K8s API if idle)

3. External Access via Istio (User → Gateway → Service → Pod):
   External User → Istio Gateway (443/TCP HTTPS TLS1.3 OAuth2) →
   Notebook Service (80/TCP HTTP mTLS Istio Peer Auth) →
   Notebook Pod (8888/TCP HTTP mTLS Istio sidecar)

4. Metrics Collection:
   Prometheus → Controller (8080/TCP HTTP GET /metrics no-auth)
