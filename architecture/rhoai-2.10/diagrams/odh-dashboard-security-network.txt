TRUST BOUNDARIES
═══════════════════════════════════════════════════════════════════════════

1. External → Route (Public Internet to OpenShift Ingress)
   - Encryption: TLS 1.2+ (HTTPS)
   - Trust: Untrusted → DMZ
   - Auth: None (redirects to OAuth)

2. Route → OAuth Proxy (OpenShift Ingress to Application)
   - Encryption: TLS 1.2+ (Reencrypt mode)
   - Trust: DMZ → Trusted Application
   - Auth: OAuth Session Cookie

3. OAuth Proxy → Backend (Pod-local communication)
   - Encryption: None (localhost HTTP)
   - Trust: Same Pod (localhost only)
   - Auth: Bearer Token (injected by OAuth proxy)
   - Risk: ⚠️ Unencrypted but mitigated by localhost-only binding

4. Backend → Kubernetes API (Application to Cluster Services)
   - Encryption: TLS 1.2+ (HTTPS)
   - Trust: Trusted Application → Trusted Services
   - Auth: Service Account Token (auto-rotated)

5. Backend → Internal Services (Application to ODH Components)
   - Encryption: Variable (HTTP/HTTPS)
   - Trust: Trusted Application → Trusted Services
   - Auth: Bearer Token (user or service account)

SECURITY CONSIDERATIONS
═══════════════════════════════════════════════════════════════════════════

✅ Implemented:
  - TLS encryption for external access (HTTPS)
  - OAuth authentication enforcement
  - Authorization delegation to Kubernetes RBAC
  - Service account token auto-rotation
  - Automatic TLS certificate generation and rotation
  - HSTS headers enforced
  - Insecure traffic redirected to HTTPS
  - Authorization header redaction in logs
  - Principle of least privilege (granular RBAC)
  - Non-root container execution

⚠️ Risks & Mitigations:
  - Backend-to-OAuth-proxy communication is unencrypted
    → Mitigated: localhost-only binding (same pod)
  - OAuth client secret rotation requires manual intervention
    → Recommendation: Implement automated secret rotation
  - Cookie secret rotation requires pod restart
    → Impact: 23h session expiry limits exposure window
  - Broad RBAC permissions required for full functionality
    → Justification: Required for namespace/project management features
