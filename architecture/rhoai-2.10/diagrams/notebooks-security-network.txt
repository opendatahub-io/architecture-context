- Namespace: {user-namespace}
- Purpose: Minimal permissions for workbench pod operations
- Auto-mounted: Yes (/var/run/secrets/kubernetes.io/serviceaccount)

NOTE: This component does not define ClusterRoles or RoleBindings.
RBAC for notebook pods is managed by ODH Notebook Controller, which assigns
ServiceAccounts based on user permissions at pod creation time.

Users typically have permissions within their namespace to:
- create/delete/list pods
- create/delete/list services
- create/delete/list routes
- read/list configmaps and secrets (as needed for workbench)

NETWORK POLICIES (if configured)
═══════════════════════════════════════════════════════════════════════════

Default OpenShift SDN allows:
- Ingress: From OpenShift Router to OAuth Proxy (8443/TCP)
- Egress: To Kubernetes API (6443/TCP)
- Egress: To external HTTPS endpoints (443/TCP) for S3, PyPI, Git, etc.
- Pod-to-Pod: Within same namespace (default allow)

Recommended NetworkPolicy (if strict isolation required):
- Deny all ingress by default
- Allow ingress from openshift-ingress namespace (Router)
- Allow egress to kube-system (DNS)
- Allow egress to openshift-apiserver (K8s API)
- Allow egress to external HTTPS (443/TCP)
- Deny pod-to-pod communication (optional isolation)

COMPLIANCE NOTES
═══════════════════════════════════════════════════════════════════════════

Security Scanning:
- Base images: UBI9/UBI8 scanned by Red Hat
- Container vulnerability scan: Quay Security Scanner (on push)
- Python dependencies: Weekly pip-audit scans
- Update cadence: Weekly security patches (2024a-weekly tags)

Vulnerability Management:
- Major releases: Twice yearly (Spring/Fall - e.g., 2024a, 2024b)
- Patch releases: Weekly for CVE fixes
- Support window: 12 months minimum (two concurrent major releases)
- Critical CVE response: Hotfix within 48 hours

Data Protection:
- Data in transit: TLS 1.2+ for all external connections
- Data at rest: User responsible for S3 encryption (recommended: SSE-S3 or SSE-KMS)
- Secrets: Kubernetes Secrets (base64 encoded, not encrypted at rest unless etcd encryption enabled)
- Ephemeral storage: Notebook data lost on pod restart unless PVC configured
