SERVICE MESH CONFIGURATION
═══════════════════════════════════════════════════════════════════════════════════════

Namespace: istio-system
- ServiceMeshControlPlane: basic (RHOAI profile)
- Members: redhat-ods-applications, user namespaces (opt-in)

PeerAuthentication:
- Default mode: PERMISSIVE (allows plaintext and mTLS)
- Strict mode: STRICT (enforces mTLS for all traffic)
- Scope: Namespace-scoped for inference namespaces

AuthorizationPolicy (for InferenceServices):
- Allow: Specific ServiceAccounts from authorized namespaces
- Deny: Default deny if auth enabled
- Integration: Authorino for JWT validation (via RequestAuthentication)

VirtualService (per InferenceService):
- Routes: Traffic routing rules, canary deployments
- Gateways: knative-serving/knative-ingress-gateway
- Host: {isvc}.{namespace}.{cluster-domain}

Telemetry:
- Namespace-scoped telemetry configuration
- Metrics: Request count, duration, size
- Access logs: Conditional (not enabled by default)

NETWORK POLICIES
═══════════════════════════════════════════════════════════════════════════════════════

Operator Network Policies:
- odh-model-controller: Ingress restricted to 9443/TCP (webhook) from API server
- kueue: Ingress restricted to 9443/TCP (webhook) from API server
- modelmesh-serving: Ingress allowed on 8033 (gRPC), 8008 (REST), 2112 (metrics)
