TRUST BOUNDARIES
═════════════════════════════════════════════════════════════════════════════════════

1. EXTERNAL ↔ INGRESS
   ├─ Protocol: HTTPS (TLS 1.2+)
   ├─ Authentication: Bearer Token (JWT) - optional
   └─ Termination: Istio Ingress Gateway (TLS termination)

2. INGRESS ↔ SERVICE MESH
   ├─ Protocol: HTTP with mTLS (Istio STRICT)
   ├─ Authentication: Service Identity (X.509 certificates from Istio CA)
   └─ Encryption: mTLS enforced by Istio PeerAuthentication

3. SERVICE MESH (Internal)
   ├─ Protocol: HTTP with mTLS between all services
   ├─ Authentication: Mutual TLS (automatic certificate rotation)
   └─ Authorization: Istio AuthorizationPolicy (RBAC for service-to-service)

4. POD INTERNAL (localhost)
   ├─ Protocol: HTTP (plaintext, localhost only)
   ├─ Containers: kserve-agent ↔ Model Server
   └─ Encryption: None (localhost communication, no network exposure)

5. SERVICE MESH ↔ EXTERNAL SERVICES
   ├─ Protocol: HTTPS (TLS 1.2+)
   ├─ Destinations: S3, GCS, Kubernetes API
   └─ Authentication: IAM roles, service account keys, bearer tokens
