## Purpose

**Short**: A Kubernetes operator that manages the lifecycle of Llama Stack server distributions on OpenShift/Kubernetes clusters.

**Detailed**: The llama-stack-k8s-operator is a controller-runtime based Go operator that automates the deployment, configuration, and lifecycle management of Llama Stack AI inference server instances. Users create `LlamaStackDistribution` custom resources (CRDs) to declare desired server configurations, and the operator reconciles the necessary Kubernetes resources (Deployments, Services, NetworkPolicies, PVCs, Ingresses, HPAs, PDBs) to match.

The operator supports multiple distribution types including a Red Hat distribution (`rh-dev`) configured via the `RELATED_IMAGE_RH_DISTRIBUTION` environment variable, as well as upstream community distributions (starter, remote-vllm, meta-reference-gpu, postgres-demo). It provides features such as user-supplied ConfigMap-based server configuration, custom CA bundle injection (including auto-detection of ODH trusted CA bundles), horizontal pod autoscaling, pod disruption budgets, network policy-based access control, and optional Ingress-based external access. The operator uses kustomize internally to render and apply manifest templates, with Go-based transformer plugins for dynamic field mutation, namespace setting, and NetworkPolicy rule generation.

For RHOAI deployment, this operator is deployed by the rhods-operator using the kustomize overlay at `config/overlays/rhoai/`. The operator image and distribution image references are injected at deploy time via `RELATED_IMAGE_ODH_LLAMA_STACK_K8S_OPERATOR_IMAGE` and `RELATED_IMAGE_ODH_LLAMA_STACK_CORE_IMAGE` parameters.
