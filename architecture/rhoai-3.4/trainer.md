## Purpose
**Short**: Kubeflow Trainer is a Kubernetes operator that manages distributed training jobs and training runtimes for machine learning workloads on RHOAI.

**Detailed**: Kubeflow Trainer (formerly Kubeflow Training Operator) provides a Kubernetes-native API for defining and running distributed machine learning training jobs. It introduces three CRDs — `TrainJob`, `TrainingRuntime`, and `ClusterTrainingRuntime` — that allow users to define training workloads declaratively and reference pre-configured runtime templates for frameworks like PyTorch (including TorchTune for fine-tuning LLMs like Llama 3.2 and Qwen 2.5).

The operator translates `TrainJob` specifications into lower-level Kubernetes primitives, primarily `JobSet` resources, while handling distributed training configuration (rendezvous, node addressing, GPU allocation). It supports co-scheduling via Volcano and scheduler-plugins PodGroups. The RHOAI distribution adds Red Hat-specific enhancements including progression tracking (real-time training progress monitoring via HTTP metrics polling), NetworkPolicy-based pod isolation for training jobs, and pre-configured `ClusterTrainingRuntime` resources for CPU, CUDA, and ROCm accelerator stacks.

The controller ships with validating webhooks for all three CRDs and manages its own TLS certificates for webhook serving. It is deployed by the rhods-operator via kustomize overlays into the `opendatahub` namespace.

The controller dynamically creates a `NetworkPolicy` per `TrainJob` with the following rules:

| Rule | Direction | Source/Destination | Port | Purpose |
|------|-----------|-------------------|------|---------|
| Same-job pod communication | Ingress | Pods with `jobset.sigs.k8s.io/jobset-name: <trainjob-name>` | All ports | NCCL/MPI/gRPC inter-node communication for distributed training |
| Controller metrics access | Ingress | Pods in controller namespace with `app.kubernetes.io/name: trainer, app.kubernetes.io/component: controller` | Configurable (default 28080/TCP) | Allows controller to poll training progress metrics (only when progression tracking is enabled) |
