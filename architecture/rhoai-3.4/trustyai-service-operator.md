## Purpose
**Short**: The TrustyAI Service Operator manages AI/ML trustworthiness, evaluation, and guardrails infrastructure on OpenShift, providing model explainability, LLM evaluation, and content guardrails services.

**Detailed**: The TrustyAI Service Operator is a multi-controller Kubernetes operator built with the kubebuilder framework and controller-runtime. It manages five distinct services within the TrustyAI platform:

1. **TrustyAI Service (TAS)**: Deploys and manages TrustyAI explainability service instances that provide model bias detection, fairness metrics, and explainability for AI/ML inference services. It integrates with KServe InferenceServices and ModelMesh for payload processing and monitoring.

2. **LMEval (LMES)**: Manages LLM evaluation jobs that run language model evaluation harness benchmarks. Supports Kueue-based job scheduling for resource management, with configurable batch sizes and device detection. Jobs run as pods with a driver sidecar architecture.

3. **Guardrails Orchestrator (GORCH)**: Deploys guardrails orchestration infrastructure that intercepts and validates LLM inference requests against configurable content safety detectors. Supports auto-configuration from InferenceServices, built-in regex detectors, and a sidecar gateway pattern.

4. **NeMo Guardrails (NEMO_GUARDRAILS)**: Manages NVIDIA NeMo Guardrails server deployments providing programmable guardrail configurations for LLM applications with CA certificate management and kube-rbac-proxy authentication.

5. **EvalHub (EVALHUB)**: A centralized evaluation hub that orchestrates multiple evaluation providers (Garak, GuideLLM, LightEval, LM Evaluation Harness) with multi-tenant namespace support, provider/collection configuration management, and MLFlow integration for experiment tracking.
