## Purpose
**Short**: Central build configuration repository that defines all container images, OLM operator bundle manifests, and File-Based Catalog (FBC) fragments for Red Hat OpenShift AI (RHOAI) 3.4.

**Detailed**: RHOAI-Build-Config is the authoritative source-of-truth for the RHOAI product build and release pipeline. It does not contain operator or application source code itself; instead, it aggregates and coordinates the build outputs of approximately 86 container images across the entire RHOAI product portfolio. The repository manages three critical artifacts:

1. **Operator Bundle** (`bundle/`): Contains the OLM ClusterServiceVersion (CSV), CRD manifests, RBAC definitions, and metadata that define how the `rhods-operator` is installed via OLM. The CSV declares all 86+ RELATED_IMAGE environment variables (pinned by SHA256 digest) to support disconnected/air-gapped installations.

2. **File-Based Catalog (FBC) Fragments** (`catalog/`): Per-OCP-version catalog definitions (v4.19, v4.20, v4.21) that define the `rhods-operator` package with its `stable-3.4` channel in the OLM catalog. These are built into FBC fragment container images consumed by the Red Hat operator index.

3. **Build Automation** (`.tekton/`, `.github/workflows/`, `config/`): Konflux (Tekton) pipeline definitions and GitHub Actions workflows that automate the continuous rebuild and patching of the bundle and catalog whenever upstream component images are updated. The `config/build-config.yaml` defines registry mappings and supported OCP versions. The `bundle/bundle_build_args.map` and `catalog/catalog_build_args.map` files track git URLs and commit SHAs for every component, enabling full provenance tracing.

This repository is entirely built by Konflux. It is the central coordination point through which all RHOAI component image updates flow to become part of a shippable operator bundle.
