## Purpose
**Short**: MLflow is an open-source ML lifecycle platform providing experiment tracking, model registry, AI gateway, workspaces, and artifact management, extended with Kubernetes-native workspace and authentication support for RHOAI.

**Detailed**: MLflow is a comprehensive machine learning lifecycle management platform that provides experiment tracking, model versioning and registry, artifact storage, and an AI gateway for proxying requests to LLM providers (OpenAI, Anthropic, Gemini, Mistral, and others via LiteLLM). The RHOAI distribution extends upstream MLflow with a Kubernetes workspace provider that maps MLflow workspaces to Kubernetes namespaces, and a Kubernetes authorization plugin that enforces RBAC via `SelfSubjectAccessReview` against resources in the `mlflow.kubeflow.org` API group.

The server runs as a FastAPI application (wrapping a legacy Flask app via WSGIMiddleware) served by uvicorn on port 5000. It supports pluggable backend stores (SQLite, PostgreSQL, MySQL), artifact storage (S3, GCS, Azure, local filesystem), and workspace providers. The Kubernetes workspace provider uses watch loops to cache namespace state and supports per-namespace artifact storage overrides via an optional `MLflowConfig` CRD (`mlflowconfigs.mlflow.kubeflow.org`). The Kubernetes auth plugin validates Bearer tokens via the Kubernetes TokenReview API and authorizes operations via SubjectAccessReview.

The RHOAI build uses Konflux and ships the server as a UBI9-based container image (`odh-mlflow-rhel9`) with the React UI pre-built and bundled, PostgreSQL client libraries, and Prometheus metrics exporter support. Features disabled in the RHOAI build include the AI assistant and AI gateway UI (via environment variables `MLFLOW_ENABLE_ASSISTANT=false` and `MLFLOW_ENABLE_AI_GATEWAY=false`).
