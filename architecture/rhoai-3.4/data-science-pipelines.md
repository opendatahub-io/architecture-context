## Purpose
**Short**: Kubeflow Pipelines platform for orchestrating, scheduling, and managing end-to-end ML workflows on Kubernetes via Argo Workflows.

**Detailed**: Data Science Pipelines (DSP) is the Red Hat downstream distribution of Kubeflow Pipelines (KFP) v2, providing a complete platform for defining, running, scheduling, and monitoring machine learning pipelines on OpenShift. The platform includes an API server that exposes both gRPC and REST APIs (v1beta1 and v2beta1) for pipeline CRUD operations, experiment tracking, run management, and artifact storage. It uses Argo Workflows as the underlying workflow engine and stores pipeline metadata in MySQL/MariaDB or PostgreSQL databases, with pipeline artifacts stored in S3-compatible object storage (typically MinIO).

The system includes several controllers: the ScheduledWorkflow controller manages recurring pipeline runs via CronSchedule/PeriodicSchedule triggers, the Persistence Agent synchronizes workflow state from Argo back to the API server, the Viewer Controller manages TensorBoard viewer deployments, and the Cache Server provides a mutating webhook for step-level caching. The platform also supports Kubernetes-native pipeline storage via Pipeline and PipelineVersion CRDs (pipelines.kubeflow.org/v2beta1) with validating and mutating webhooks. All RHOAI builds use FIPS-compliant Go builds with `strictfipsruntime` and are based on UBI9 images, built by Konflux.

| ml-pipeline (API Server) | Go Service | Central API server exposing gRPC (8887/TCP) and HTTP/REST (8888/TCP) APIs for pipeline management, run execution, experiment tracking, and artifact handling. Also serves webhook endpoints (8443/TCP) for PipelineVersion validation/mutation. |
| ml-pipeline-scheduledworkflow | Go Controller | Watches ScheduledWorkflow CRs and creates Argo Workflow CRs based on cron or periodic schedules |

| ml-pipeline-viewer-crd | Go Controller | Watches Viewer CRs and creates Deployments/Services for TensorBoard instances |

| ml-pipeline-visualizationserver | Python Service | Generates pipeline visualizations, serves on port 8888 |
| cache-server | Go Webhook Server | Mutating admission webhook (port 8443/TCP over TLS) for caching pipeline step results |
| kfp-driver | Go Binary | V2 pipeline step driver, injected as init container during pipeline execution |
| kfp-launcher | Go Binary | V2 pipeline step launcher, injected as container during pipeline execution |
| metadata-grpc-server (ML Metadata) | gRPC Service | ML Metadata store serving on port 8080, stores artifact and execution metadata |
| metadata-envoy | Envoy Proxy | Envoy proxy for metadata gRPC service, serves on port 9090 |
| metadata-writer | Go Agent | Writes pipeline execution metadata to the ML Metadata store |
