## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Ray clusters, jobs, and services on OpenShift.

**Detailed**: KubeRay Operator is a controller-runtime-based Kubernetes operator that reconciles three custom resources: RayCluster, RayJob, and RayService. It automates the deployment, scaling, and management of Ray distributed computing clusters on Kubernetes/OpenShift. The operator creates and manages Ray head and worker pods, services, ingress resources, and supporting infrastructure.

In the RHOAI distribution, the operator includes significant OpenShift-specific enhancements: Gateway API integration (HTTPRoute-based ingress via the data-science-gateway), kube-rbac-proxy sidecar injection for authentication, mTLS certificate management via cert-manager, NetworkPolicy enforcement for cluster isolation, SecurityContextConstraints for pod security, and webhook-based validation/defaulting for RayCluster resources. The operator is deployed into the `opendatahub` namespace and manages Ray workloads across user namespaces.

The RHOAI build uses Konflux pipelines with FIPS-compliant Go compilation (`GOEXPERIMENT=strictfipsruntime`) and UBI9 base images, producing a single container image: `odh-kuberay-operator-controller`.

**Controller-managed ingress (RHOAI 3.x pattern)**:
- The Authentication Controller creates `HTTPRoute` CRs in the platform namespace (e.g., `opendatahub`) that reference the `data-science-gateway` Gateway in `openshift-ingress`
- `ReferenceGrant` CRs are created in user namespaces to allow cross-namespace Service references from platform namespace HTTPRoutes
- `kube-rbac-proxy` sidecars are injected into Ray head pods for RBAC-based authentication on port 8443/TCP
- The HTTPRoute path pattern is `/ray/{namespace}/{cluster}` with URL rewriting to the backend dashboard service on port 8265
- A redirect rule sends `/ray/{ns}/{cluster}` (exact) to `/ray/{ns}/{cluster}/#/` (302 redirect)

## Kustomize Deployment Structure

The RHOAI deployment uses the overlay at `ray-operator/config/openshift/`:

```
ray-operator/config/openshift/kustomization.yaml
  -> resources:
       ray_operator_scc.yaml (SecurityContextConstraints)
       ../default (-> ../crd, ../rbac, ../manager)
       webhook.yaml (Service + MutatingWebhookConfiguration)
  -> patches:
       webhook-deployment-patch.yaml (enables webhooks, mounts cert volume)
       kuberay-operator-image-patch.yaml (injects $(image) from params.env)
       gateway-env-patch.yaml (sets APPLICATION_NAMESPACE env var)
  -> configMapGenerator: ray-config (from params.env)
  -> namespace: opendatahub
  -> commonLabels: app.kubernetes.io/name=kuberay, app.kubernetes.io/component=kuberay-operator
```

Image injection at deploy time:
- `odh-kuberay-operator-controller-image` -> `RELATED_IMAGE_ODH_KUBERAY_OPERATOR_CONTROLLER_IMAGE`
- Default: `quay.io/opendatahub/kuberay-operator:v1.4.4`
