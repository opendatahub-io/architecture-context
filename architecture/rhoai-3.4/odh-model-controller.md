## Purpose
**Short**: Kubernetes operator that automates model serving infrastructure by managing routes, monitoring, autoscaling, authentication, and certificate management for KServe InferenceServices and LLMInferenceServices on OpenShift AI.

**Detailed**: The odh-model-controller is a controller-runtime based operator that extends KServe model serving capabilities for the Red Hat OpenShift AI (RHOAI) platform. It watches KServe InferenceService, ServingRuntime, LLMInferenceService, and NVIDIA NIM Account custom resources and reconciles supporting infrastructure including OpenShift Routes for model endpoint exposure, Prometheus ServiceMonitors/PodMonitors for observability, KEDA TriggerAuthentications for autoscaling on custom metrics, and Kuadrant AuthPolicies for authentication.

The controller provides two main reconciliation paths: (1) The InferenceService controller handles RawDeployment mode KServe workloads, creating Routes, ClusterRoleBindings, metrics dashboards, and KEDA autoscaling resources. (2) The LLMInferenceService controller handles the newer LLM inference pattern, managing Kuadrant AuthPolicies (both Gateway-level and HTTPRoute-level) and Istio EnvoyFilters for TLS between the gateway and Authorino auth service. Additionally, the controller manages CA certificate trust chains, storage-config Secrets from data connections, Ray TLS certificates for multi-node serving, and NVIDIA NIM account lifecycle including API key validation and template management.

A separate Gateway Discovery Server binary (built from `server/`) provides an HTTPS API endpoint (`/api/v1/gateways`) for discovering Gateway API resources, enabling model serving components to locate their ingress gateways dynamically.

**InferenceService Ingress (RawDeployment mode)**: The InferenceService controller creates OpenShift Route CRs for InferenceServices that have the label `networking.kserve.io/visibility=exposed`. Routes are created with TLS Edge termination (or Reencrypt when `security.opendatahub.io/enable-auth=true`), pointing to the predictor or transformer Service.

**LLMInferenceService Ingress (Gateway API pattern)**: The LLMInferenceService controller manages Kuadrant AuthPolicies targeting Gateway and HTTPRoute resources. It creates Gateway-level AuthPolicies for user-defined authentication and HTTPRoute-level AuthPolicies for anonymous access. It also creates Istio EnvoyFilters on Gateways to configure TLS between the Gateway and Authorino when `security.opendatahub.io/authorino-tls-bootstrap` annotation is present.
