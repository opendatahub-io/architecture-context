## Purpose

**Short**: Caikit-NLP is a Python library providing NLP capabilities (text generation, embeddings, reranking, classification, tokenization) built on the Caikit framework for use in RHOAI model serving.

**Detailed**: Caikit-NLP implements the Caikit framework's "task" and "module" patterns to deliver a comprehensive suite of Natural Language Processing capabilities. It provides modules for text generation (via local HuggingFace models or remote TGIS backends), text embeddings (via sentence-transformers), document reranking (via cross-encoders), sequence classification, token classification, and tokenization. The library supports parameter-efficient fine-tuning (PEFT) through prompt tuning and multi-task prompt tuning, allowing efficient model adaptation without modifying base model weights.

As a runtime library, caikit-nlp is packaged into a container image and deployed as part of the RHOAI model serving stack. It registers itself with the Caikit runtime framework, which exposes its modules over both gRPC (port 8085) and HTTP/REST (port 8080) protocols. The library integrates with the Text Generation Inference Server (TGIS) backend for distributed and optimized text generation inference, supporting both local model execution and remote TGIS-based serving. Models can be loaded from local storage or bootstrapped from HuggingFace Hub (when downloads are enabled).

## Architectural Analysis

Caikit-NLP follows a modular plugin architecture defined by the upstream Caikit framework. Each NLP capability is implemented as a "module" decorated with `@module` and annotated with task types via `@TaskType.taskmethod()`. This allows the Caikit runtime to dynamically discover and expose all modules over both gRPC and HTTP without caikit-nlp containing any server code itself. The library is purely a collection of task implementations — the serving infrastructure is entirely delegated to the Caikit runtime.

The text generation architecture has a deliberate split between local and remote execution paths. Local modules (`PeftPromptTuning`, `TextGeneration`) load HuggingFace models directly into GPU/CPU memory and execute inference using PyTorch. Remote modules (`PeftPromptTuningTGIS`, `TextGenerationTGIS`) delegate inference to an external TGIS server via gRPC, only managing prompt artifacts and verbalizer templates locally. This split is mediated by the `TGISAutoFinder`, a custom model finder that automatically detects whether a model should be served locally or via TGIS by testing connectivity to the TGIS backend. This finder is registered with the Caikit model management system via the `model_finder_factory`.

The resource hierarchy (`PretrainedModelBase` → `HFAutoCausalLM` / `HFAutoSeq2SeqLM` / `HFAutoSequenceClassifier`) provides a clean abstraction over HuggingFace model types. Each resource subclass declares its `MODEL_TYPE`, `TASK_TYPE`, and `SUPPORTED_MODEL_TYPES`, allowing the PEFT configuration and training code to work polymorphically across different model architectures. The tokenization pipeline is particularly interesting: causal LMs support two distinct approaches — a "seq2seq-style" approach (default) that masks source tokens in labels with `IGNORE_ID=-100`, and a "chunked" approach that concatenates and splits sequences into fixed-size windows.

Training is focused on parameter-efficient methods, specifically PEFT prompt tuning via the HuggingFace PEFT library (pinned at v0.6.0). The training loop uses HuggingFace Accelerate for mixed precision (auto-detecting fp16/bf16 support) and gradient accumulation. Distributed training across multiple GPUs is supported via PyTorch's elastic launch with configurable master address/port. Training outputs are compact prompt vectors (encoder.pt/decoder.pt files) rather than full model checkpoints, enabling efficient multi-tenant serving where many prompts share a single base model.

The embedding subsystem is architecturally separate from text generation — it uses sentence-transformers directly rather than the HuggingFace transformers pipeline. The `EmbeddingModule` supports configurable batch processing, PyTorch autocast for bfloat16 inference, optional PyTorch 2.0 compilation (`pt2_compile`), and Intel IPEX optimization. The `CrossEncoderModule` provides an alternative reranking implementation using cross-encoder architecture rather than bi-encoder embeddings.

Security-wise, model downloads are disabled by default (`allow_downloads: false`), and `trust_remote_code` defaults to `false` for sentence-transformers models. Path validation in the PEFT configuration prevents directory traversal attacks via regex checks. The TGIS connection supports optional mTLS with configurable CA, client cert, and client key files. However, caikit-nlp itself does not implement any authentication or authorization — these concerns are fully delegated to the Caikit runtime and the model serving platform (KServe/ModelMesh with kube-rbac-proxy).
