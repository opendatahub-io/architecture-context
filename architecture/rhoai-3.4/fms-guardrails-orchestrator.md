## Purpose
**Short**: REST API orchestrator that coordinates AI text generation with content safety guardrails by routing requests through configurable detector, chunker, and LLM backend services.

**Detailed**: The FMS Guardrails Orchestrator (fms-guardrails-orchestr8) acts as middleware between clients and language models to enforce content safety. It receives text generation or detection requests via HTTP/HTTPS REST APIs, then orchestrates calls to configured detector services (for content safety analysis), chunker services (for text tokenization/segmentation), and generation backends (TGIS, caikit-nlp, or OpenAI-compatible APIs) to produce guardrailed responses.

The orchestrator supports both unary and streaming request patterns across multiple API versions (v1 and v2). It provides standalone detection endpoints for content analysis, chat detection, context document detection, and generation-coupled detection where text is generated and simultaneously analyzed for harmful content. Detection results can trigger blocking or filtering of generated text based on configurable score thresholds.

The server is built with Rust using the Axum web framework and Tonic gRPC client library. It supports TLS and mTLS for both inbound connections and outbound client connections to backend services. OpenTelemetry integration provides distributed tracing and metrics export via gRPC or HTTP OTLP protocols.
