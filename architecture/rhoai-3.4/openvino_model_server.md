## Purpose
**Short**: High-performance AI model inference server implemented in C++ using OpenVINO for optimized serving of ML models via gRPC and REST APIs.

**Detailed**: OpenVINO Model Server (OVMS) is a high-performance model serving system developed by Intel, optimized for deployment on Intel architectures. It serves machine learning models using the OpenVINO inference engine and exposes them through multiple API protocols: TensorFlow Serving API (v1), KServe Inference Protocol (v2), and OpenAI-compatible generative API (v3). The server supports serving traditional vision/NLP models, large language models (LLMs) with continuous batching, embeddings, reranking, image generation, text-to-speech, and speech-to-text workloads.

OVMS is designed as a standalone containerized inference runtime used within the RHOAI platform. It runs as a single-process server with modular architecture consisting of a gRPC server module, HTTP/REST server module (using Drogon framework), servable manager module, metrics module, Python interpreter module, and a HuggingFace model pull module. Models can be loaded from local storage, Amazon S3, Google Cloud Storage, Azure Blob Storage, or pulled directly from HuggingFace Hub. The server supports MediaPipe graph pipelines, DAG-based inference pipelines, model versioning, dynamic model updates, GPU acceleration (via OpenCL), and Prometheus-compatible metrics.

In the RHOAI platform, OVMS is deployed as the inference runtime container image (`odh-openvino-model-server-rhel9`) that backs KServe InferenceService instances. It is not an operator itself but rather a serving runtime that is orchestrated by the KServe/ModelMesh control plane.
