## Purpose
**Short**: ML Metadata is a gRPC server that records and retrieves metadata associated with ML workflows, tracking artifacts, executions, contexts, and their relationships (lineage).

**Detailed**: ML Metadata (MLMD) is a library and service originally developed by Google for recording and retrieving metadata associated with ML developer and data scientist workflows. It provides a metadata store that tracks artifacts (datasets, models), executions (training runs, pipeline steps), contexts (experiments, pipelines), and the events/relationships between them, forming a complete ML lineage graph.

In the RHOAI platform, the MLMD gRPC server is shipped as the `odh-mlmd-grpc-server` container image, built from the `Dockerfile.konflux` using a multi-stage build on UBI9. The server binary (`metadata_store_server`) is compiled from C++ using Bazel, and exposes a gRPC API on port 8080 (configurable via `GRPC_PORT` environment variable). The server supports MySQL, PostgreSQL, and SQLite as backend metadata stores, with optional SSL/TLS for both gRPC transport and database connections.

The component is a critical dependency of the Data Science Pipelines subsystem (used by Kubeflow Pipelines / Tekton pipelines) to persist pipeline execution metadata, artifact lineage, and experiment tracking. It does not define any Kubernetes CRDs or controllers; it is a standalone stateless gRPC service that relies on an external database for persistence.
