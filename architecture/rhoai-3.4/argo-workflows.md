## Purpose
**Short**: Argo Workflows is the workflow execution engine used by Data Science Pipelines to orchestrate multi-step ML/AI pipeline runs on Kubernetes.

**Detailed**: Argo Workflows is an open-source, container-native workflow engine for orchestrating parallel jobs on Kubernetes. In the RHOAI (Red Hat OpenShift AI) distribution, it provides the core workflow execution capability for Data Science Pipelines, enabling users to define, schedule, and execute complex multi-step machine learning pipelines as Kubernetes-native workflows.

The component ships two primary container images built via Konflux: the **workflow-controller** (which watches for Workflow CRDs, creates pods to execute workflow steps, manages workflow lifecycle, and handles cron scheduling) and **argoexec** (the executor sidecar/init container that runs inside each workflow step pod to handle artifact collection, output parameter capture, and resource management). The workflow-controller also includes an optional Argo Server component that exposes a REST/gRPC API and web UI for workflow management, though in RHOAI this is typically fronted by the Data Science Pipelines API server.

Both images are built with FIPS compliance support (using `GOEXPERIMENT=strictfipsruntime` and `CGO_ENABLED=1`) for the RHOAI distribution. The argoexec image ships both FIPS and non-FIPS binaries. The workflow-controller is built exclusively with the FIPS-compliant binary for RHOAI.
