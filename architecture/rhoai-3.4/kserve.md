## Purpose
**Short**: KServe provides a Kubernetes-native platform for deploying, managing, and serving machine learning models and LLM inference services on OpenShift AI.

**Detailed**: KServe is the model serving controller for Red Hat OpenShift AI (RHOAI). It implements the KServe v1beta1 InferenceService API along with newer v1alpha1 InferenceGraph, TrainedModel, LocalModel, and v1alpha2 LLMInferenceService APIs. The controller watches these custom resources and reconciles the underlying Kubernetes resources needed to serve ML models - including Deployments, Services, HTTPRoutes (Gateway API), Istio VirtualServices, and autoscaling resources (HPA, KEDA ScaledObjects).

In RHOAI 3.x, KServe operates in "RawDeployment" mode by default, creating standard Kubernetes Deployments and Services rather than relying on Knative Serving. The controller creates HTTPRoute CRs that reference a parent Gateway (typically `openshift-ingress/openshift-ai-inference`) managed by the rhods-operator's gateway controller. For authentication, KServe injects kube-rbac-proxy sidecar containers into inference service deployments when the `security.opendatahub.io/enable-auth` annotation is set. The newer LLMInferenceService controller (v1alpha2) provides specialized support for LLM inference workloads, integrating with Gateway API Inference Extensions (InferencePool, InferenceModel), llm-d inference schedulers, and multi-node inference patterns.

**Notes on Ingress Patterns**:
- **InferenceService (RHOAI 3.x)**: Creates HTTPRoute CRs referencing parent Gateway `openshift-ingress/openshift-ai-inference`. The Gateway is deployed by rhods-operator. URL scheme defaults to `https`. Gateway API is the primary ingress pattern, with Istio VirtualService reconciliation as a fallback when VirtualService CRD is available.
- **InferenceGraph**: Uses OpenShift Route CRs for external access (routes directly to the inference graph router service). Creates ServiceAccount + ClusterRoleBinding for auth when needed.
- **LLMInferenceService**: Creates HTTPRoute CRs with support for Gateway API Inference Extensions (InferencePool, InferenceModel). Includes disaggregated prefill/decode routing and scheduler integration.
- **kube-rbac-proxy sidecar**: Injected when `security.opendatahub.io/enable-auth: "true"` annotation is set. Listens on port 8443/TCP with OpenShift serving certificate TLS. Enforces SubjectAccessReview authorization before proxying to the application container on port 8080/TCP.
