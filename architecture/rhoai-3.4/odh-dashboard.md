## Purpose
**Short**: The ODH Dashboard is the web-based management console for Red Hat OpenShift AI, providing a unified UI for managing AI/ML workloads, notebooks, model serving, pipelines, and related data science resources.

**Detailed**: The ODH Dashboard serves as the primary user interface for Red Hat OpenShift AI (RHOAI). It consists of a React/TypeScript frontend built with PatternFly 6 and a Fastify-based Node.js backend that proxies Kubernetes API requests on behalf of authenticated users. The backend communicates directly with the Kubernetes API server using the `@kubernetes/client-node` library, managing resources such as notebooks, model registries, inference services, and data science cluster configurations.

In RHOAI 3.x, the dashboard uses a modular architecture based on Webpack Module Federation, enabling multiple independently deployable UI modules (model-registry, gen-ai, maas, mlflow, eval-hub, automl, autorag) to run as sidecar containers within the same pod. Each module serves its own frontend assets and API proxy on a dedicated HTTPS port. The primary ingress pattern uses Gateway API HTTPRoute resources referencing the `data-science-gateway` Gateway, with kube-rbac-proxy providing authentication enforcement via SubjectAccessReview checks.

The dashboard is deployed by the rhods-operator using kustomize overlays. For RHOAI self-managed deployments, the operator applies the `rhoai/onprem` overlay, which layers RHOAI-specific branding, naming (rhods-dashboard), and configuration on top of the common base manifests. The operator injects container image references at deploy time via RELATED_IMAGE environment variables.

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| rhods-dashboard (HTTPRoute) | HTTPRoute (Gateway API) | Derived from data-science-gateway | 8443/TCP | HTTPS | TLS (Gateway-terminated) | Passthrough to backend TLS | External |
| odh-dashboard (Route - legacy) | Route (OpenShift) | Auto-generated | 8443/TCP | HTTPS | TLS (reencrypt) | reencrypt with HSTS | External |

**Gateway API Pattern (RHOAI 3.x)**:
- An HTTPRoute CR named `rhods-dashboard` is created referencing the `data-science-gateway` Gateway in the `openshift-ingress` namespace
- The HTTPRoute routes all traffic (path prefix `/`) to the `rhods-dashboard` Service on port 8443
- kube-rbac-proxy on port 8443 enforces authentication via SubjectAccessReview (checking `project.openshift.io/v1/projects` list permission)
- Authenticated requests are proxied to the main dashboard container on localhost:8080
- The `X-Auth-Request-User` and `X-Auth-Request-Groups` headers are injected by kube-rbac-proxy

**Legacy Route (also present)**:
- An OpenShift Route named `odh-dashboard` with TLS reencrypt termination also exists in the base manifests
- Uses HSTS header (`max-age=31536000;includeSubDomains;preload`)
