## Purpose
**Short**: Kubernetes operator that manages the full lifecycle of Data Science Pipelines (Kubeflow Pipelines v2) infrastructure on OpenShift.

**Detailed**: The Data Science Pipelines Operator (DSPO) is a controller-runtime based Go operator that watches `DataSciencePipelinesApplication` (DSPA) custom resources and reconciles a complete pipeline execution stack per DSPA instance. Each DSPA instance provisions: a DSP API Server (Kubeflow Pipelines REST/gRPC endpoint), a MariaDB or external database for metadata storage, an object storage backend (Minio or external S3-compatible), a Persistence Agent for recording pipeline run metadata, a Scheduled Workflow controller for cron-based pipeline scheduling, an Argo Workflow Controller for orchestrating pipeline DAG execution, and ML Metadata (MLMD) gRPC/Envoy services for artifact and execution lineage tracking.

The operator is deployed by the rhods-operator as a component of RHOAI. It uses kustomize overlays with image parameters injected at deploy time. The operator renders Go templates (`.yaml.tmpl` files) from `config/internal/` at runtime, using parameters extracted from the DSPA CR spec and environment-provided image references. It supports pod-to-pod TLS via OpenShift service-ca signed certificates, FIPS-compliant builds (via `GOEXPERIMENT=strictfipsruntime`), custom CA bundles for external storage/database connections, proxy configuration, and webhook-based pipeline version validation when using Kubernetes-native pipeline storage.
