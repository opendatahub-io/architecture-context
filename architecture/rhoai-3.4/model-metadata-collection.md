## Purpose
**Short**: A Go application that extracts, enriches, and catalogs metadata from Red Hat AI container images, producing static YAML catalogs shipped as a data-only container image.

**Detailed**: Model Metadata Collection is a build-time data pipeline that discovers AI models from HuggingFace collections, processes OCI container images from the Red Hat registry, and generates structured model catalogs. The tool extracts model cards from container image layers (using the `io.opendatahub.modelcar.layer.type` OCI annotation), enriches extracted metadata with HuggingFace API data (YAML frontmatter, tags, provider info), and produces aggregated YAML catalogs.

The component does NOT run as a service in the cluster. Instead, it runs during CI/CD to generate static catalog data files (`models-catalog.yaml`, `validated-models-catalog.yaml`, `other-models-catalog.yaml`), which are then packaged into a minimal UBI9-based data container image (`odh-model-metadata-collection`). This container image is consumed by other RHOAI components (such as the dashboard) as a sidecar or init container to access pre-built model catalog data at `/app/data`. The container runs `sleep infinity` and exposes data via volume mounts.

The pipeline supports concurrent processing of multiple models, model family-aware enrichment matching (preventing cross-family matches like Llama containers matching Granite HuggingFace entries), YAML frontmatter priority hierarchies, model type classification (generative/predictive/unknown), and metadata completeness reporting.
