## Purpose
**Short**: TrustyAI is a Java library and REST service providing AI fairness metrics, bias tracking, model drift detection, and explainability algorithms for ML models served by ModelMesh and KServe on OpenShift AI.

**Detailed**: TrustyAI Explainability is the core component of the TrustyAI platform, providing a comprehensive set of Explainable AI (XAI) capabilities. The project is structured as a multi-module Maven project consisting of: `explainability-core` (fairness metrics, drift detection, explainability algorithms), `explainability-connectors` (KServe/ModelMesh integration via gRPC and HTTP), `explainability-arrow` (Java-Python interop via Apache Arrow), and `explainability-service` (the Quarkus-based REST service that exposes these capabilities).

The service integrates with the model serving infrastructure by consuming inference payloads via two mechanisms: (1) a REST endpoint (`/consumer/kserve/v2`) that receives partial payloads from ModelMesh's payload processing pipeline, and (2) Knative CloudEvents for KServe inference logger integration. It stores observation data in PVC-backed flat files, MinIO object storage, or relational databases (MariaDB/MySQL) via Hibernate ORM, and exposes Prometheus metrics for fairness and drift monitoring.

The RHOAI distribution disables several explainer endpoints (SHAP, LIME, Counterfactual, TSSaliency, global explainers) while keeping fairness and drift metrics enabled. The service is deployed per-namespace alongside model serving components and is managed by the TrustyAI Service Operator (a separate repository).
