## Purpose
**Short**: Builds and ships container images for interactive data science workbenches (Jupyter, Code-Server, RStudio) and Elyra pipeline runtime images used in Red Hat OpenShift AI.

**Detailed**: The notebooks repository produces container images for interactive data science workbenches that run on OpenShift as part of Red Hat OpenShift AI (RHOAI). The images span three IDE types: JupyterLab notebooks (with variants for minimal, data science, PyTorch, TensorFlow, TrustyAI, and PyTorch+LLM Compressor), Code-Server (VS Code in the browser), and RStudio Server. Each workbench type has CPU, CUDA (NVIDIA GPU), and ROCm (AMD GPU) variants built on UBI9 with Python 3.12 (current) and Python 3.11 (legacy).

In addition to workbench images, the repository builds Elyra pipeline runtime images that execute notebook pipeline nodes in Data Science Pipelines. The repository does not deploy services itself — it produces OpenShift ImageStream manifests that the rhods-operator deploys to the cluster via kustomize. Each ImageStream carries multiple version tags (N through N-7) to support rolling upgrades and version compatibility. Image references are injected at deploy time through kustomize ConfigMap replacements, with the operator mapping `RELATED_IMAGE_*` environment variables to params.env keys.

## Network Architecture

### Services

This component does not deploy Kubernetes Services. The workbench pod networking is managed by the odh-notebook-controller, which creates per-notebook Services and HTTPRoutes.

**Within each workbench pod (RHOAI 3.x pattern)**:

| Container | Port | Target | Protocol | Purpose |
|-----------|------|--------|----------|---------|
| kube-rbac-proxy (sidecar) | 8443/TCP | → notebook container | HTTPS | Authentication enforcement before proxying to workbench |
| JupyterLab | 8888/TCP | — | HTTP | JupyterLab server (internal to pod) |
| Code-Server nginx | 80/TCP | → httpd 8080/TCP → code-server 8787/TCP | HTTP | Reverse proxy chain for idle culling emulation |
| RStudio nginx | 80/TCP | → httpd 8080/TCP → rstudio 8787/TCP | HTTP | Reverse proxy chain for idle culling emulation |

### Ingress

This component does not create Ingress/Route/HTTPRoute resources directly. The odh-notebook-controller creates HTTPRoute CRs (Gateway API, RHOAI 3.x) per notebook workbench, referencing the platform's `data-science-gateway` Gateway. Traffic flows:

| Step | Component | Type | Purpose |
|------|-----------|------|---------|
| 1 | data-science-gateway | Gateway (Gateway API) | Platform-level ingress managed by rhods-operator |
| 2 | Per-notebook HTTPRoute | HTTPRoute (Gateway API) | Created by odh-notebook-controller per Notebook CR |
| 3 | kube-rbac-proxy sidecar | Auth proxy | Authenticates user before proxying to workbench container |
| 4 | Workbench container | Application | JupyterLab (8888), Code-Server/RStudio (80 via nginx) |
