## Purpose
**Short**: A collection of detector algorithm microservices for the FMS Guardrails Orchestrator, providing content analysis for AI guardrailing.

**Detailed**: The Guardrails Detectors repository provides a suite of detector microservices that integrate with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator) to perform text content analysis and detection. These detectors evaluate input/output text from large language models (LLMs) to identify risks such as toxicity, PII (personally identifiable information), prompt injection, and format violations.

The repository ships three distinct detector types as separate container images: (1) a **HuggingFace detector** that performs ML-model-based content classification using AutoModelForSequenceClassification or GraniteForCausalLM models, (2) an **LLM Judge detector** that leverages vLLM Judge for LLM-as-a-judge based evaluation with built-in metrics like safety, accuracy, and helpfulness, and (3) a **Built-in detector** that provides lightweight heuristic detections including regex-based PII detection, file-type validation (JSON/XML/YAML), and user-defined custom detection functions.

All detectors expose a common REST API based on the IBM Detector API specification, built on FastAPI with Prometheus metrics instrumentation. The detectors are designed to be deployed as KServe inference services or standalone microservices within the RHOAI platform, and are built via Konflux for multi-architecture support (amd64, arm64, ppc64le, s390x).

| Built-in Detector | Python FastAPI Microservice | Lightweight heuristic detections: regex PII detection, file-type validation, custom user-defined detectors |
| Common Library | Python Library | Shared API schemas (Pydantic models), FastAPI base application, Prometheus instrumentation, and detector base classes |

| File Type Detector Registry | Python Module | JSON/XML/YAML validation with optional schema enforcement (JSON Schema, XSD) |
| Custom Detector Registry | Python Module | Dynamic loading of user-defined Python detection functions with static code analysis security checks, background execution support, and custom Prometheus metrics |
| InstrumentedDetector | Python Base Class | Base class providing Prometheus counter instrumentation for detections, requests, errors, and runtime tracking |
