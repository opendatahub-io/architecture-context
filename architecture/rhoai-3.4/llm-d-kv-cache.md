## Purpose
**Short**: A pluggable library and service for KV-Cache aware routing in LLM inference platforms, providing global tracking and scoring of KV-Cache block locality across vLLM pod fleets.

**Detailed**: The llm-d-kv-cache project is a core component of the llm-d distributed LLM inference platform. It provides a high-performance KV-Cache Indexer that maintains a global, near-real-time view of KV-Cache block locality across a fleet of vLLM pods. By tracking which KV-Cache blocks reside on which pods (and on which device tier, e.g., GPU or CPU), the indexer enables intelligent, cache-aware routing decisions that significantly improve Time To First Token (TTFT) and overall throughput.

The system has two primary data flows: a **Read Path** for scoring pods (given a prompt, which pod has the best KV-cache hit?) and a **Write Path** for ingesting cache events (ZMQ-streamed KVEvents from vLLM pods reporting block creation/eviction). The project is designed primarily as a Go library that integrates into the llm-d-inference-scheduler, but also ships as a standalone gRPC service with a companion UDS (Unix Domain Socket) tokenizer sidecar container built as a Python gRPC service.

The tokenization subsystem supports multiple backends: embedded Python/CGO tokenizers, local file-based tokenizers (for air-gapped environments), HuggingFace Hub tokenizers, and an external UDS-based tokenizer service. The UDS tokenizer service communicates via gRPC over Unix Domain Sockets for low-latency, high-throughput tokenization.
