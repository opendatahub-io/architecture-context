## Purpose

**Short**: MLServer is a Python-based inference server implementing the V2 Inference Protocol (KFServing/Open Inference Protocol) for serving machine learning models over REST and gRPC.

**Detailed**: MLServer is an open-source inference server originally developed by Seldon Technologies and adapted by Red Hat for inclusion in Red Hat OpenShift AI (RHOAI). It provides a standards-compliant model serving runtime that implements the V2 Inference Protocol (also known as the KFServing or Open Inference Protocol), enabling interoperability with other inference servers like Triton Inference Server.

MLServer exposes three network interfaces: an HTTP/REST server (FastAPI/Uvicorn on port 8080), a gRPC server (on port 8081), and a Prometheus metrics server (on port 8082). It supports pluggable inference runtimes for multiple ML frameworks, with the RHOAI distribution shipping LightGBM, ONNX, Scikit-Learn, and XGBoost runtimes. The server supports advanced features including adaptive batching, parallel inference workers, response caching, OpenTelemetry distributed tracing, and optional Kafka-based asynchronous inference. MLServer is deployed as a model serving container within the KServe/ModelMesh infrastructure of RHOAI, where it runs as a sidecar or standalone pod serving individual models loaded from a model repository path (typically `/mnt/models`).

MLServer does not manage its own ingress. In the RHOAI deployment:
- KServe/ModelMesh creates Kubernetes Services pointing to MLServer pods
- RHOAI 3.x uses HTTPRoute (Gateway API) + kube-rbac-proxy for external access to inference endpoints
- Istio sidecar injection provides mTLS between pods in the service mesh
