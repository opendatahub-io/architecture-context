## Purpose
**Short**: A central metadata repository for recording, versioning, and managing ML model metadata including registered models, model versions, artifacts, inference services, and serving environments.

**Detailed**: Model Registry provides a REST API service that stores and manages metadata associated with ML developer and data scientist workflows. It replaces the legacy CPP gRPC ml-metadata service with a Go-based server offering higher-level APIs, RBAC integration, and database flexibility (MySQL and PostgreSQL backends via GORM). The system supports the full lifecycle of ML models from registration through serving.

The component consists of several sub-systems: (1) the core REST API proxy server that exposes an OpenAPI v1alpha3 interface for CRUD operations on registered models, model versions, artifacts, inference services, serving environments, experiments, and experiment runs; (2) a model catalog server that manages curated model catalogs and MCP (Model Context Protocol) server catalogs backed by PostgreSQL with leader election; (3) a Kubernetes controller that watches KServe InferenceService CRs and synchronizes their state with the Model Registry API; (4) a UI consisting of a Go-based BFF (Backend-for-Frontend) and React frontend; (5) a CSI-compatible storage initializer for KServe that downloads model artifacts using `model-registry://` URIs; and (6) a Python-based async-upload job for model transfer operations.

The system integrates with Istio service mesh for mTLS, supports TLS database connections with configurable cipher suites, and is built for FIPS-compliant and disconnected environments. It is designed to work alongside KServe, the OpenShift AI platform operator, and the Dashboard UI.
