## Purpose
**Short**: An HTTP gateway that provides multiple OpenAI-compatible chat completion endpoints with configurable guardrails detector-based content filtering.

**Detailed**: The vLLM Orchestrator Gateway serves as a routing layer between clients and the FMS Guardrails Orchestrator. It exposes dynamically configured `/{route_name}/v1/chat/completions` endpoints that enforce specific detector pipelines on chat completion requests. Each configured route can associate one or more content detectors (e.g., PII regex detectors for email/SSN detection) and optionally define a fallback message returned when detections are found.

The gateway transforms standard OpenAI-compatible chat completion requests by injecting detector specifications into the payload before forwarding them to the backend FMS Guardrails Orchestrator's `/api/v2/chat/completions-detection` endpoint. It supports both non-streaming and streaming (SSE) responses. When detections are found in the orchestrator's response, the gateway can substitute the generated content with a configured fallback message. The gateway supports mTLS for secure communication with the orchestrator using OpenSSL-based certificate handling.

This enables organizations to deploy guardrailed chat completion endpoints as drop-in replacements for unguardrailed ones, with route-specific content filtering policies enforced transparently to the client.
