## Purpose
**Short**: A FIPS-compliant authentication and RBAC authorization proxy for OpenShift AI (RHOAI) environments, combining OAuth/OIDC authentication (kube-auth-proxy) and Kubernetes RBAC authorization (kube-rbac-proxy) into a single container image.

**Detailed**: kube-auth-proxy is a focused, FIPS-compliant authentication proxy derived from oauth2-proxy, streamlined exclusively for OIDC and OpenShift OAuth authentication providers. It serves as a drop-in replacement for existing oauth-proxy sidecars in ODH/RHOAI deployments, handling authentication (authn) through either external OIDC providers or OpenShift's internal OAuth service. Once authenticated, the proxy forwards requests to upstream applications with appropriate identity and authorization headers.

The container image bundles two distinct proxy binaries managed by a unified entrypoint: (1) `kube-auth-proxy` for OAuth2/OIDC authentication flows, and (2) `kube-rbac-proxy` for Kubernetes RBAC-based authorization via SubjectAccessReview. The entrypoint selects which proxy to run based on the `PROXY_MODE` environment variable or first CLI argument (`auth` or `rbac`), defaulting to `rbac` mode. This dual-mode design allows the same container image to serve either authentication or authorization needs across the RHOAI platform.

Additionally, kube-auth-proxy supports Kubernetes service account token validation via the TokenReview API, enabling service accounts to authenticate alongside human users. The proxy supports Envoy ext_authz compatibility, allowing it to integrate with Gateway API-based ingress infrastructure used in RHOAI 3.x.
