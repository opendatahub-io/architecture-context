## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Feast feature stores, deploying and reconciling online/offline store servers, registry services, UI, CronJobs, and client configurations.

**Detailed**: The Feast Operator is a controller-runtime based Kubernetes operator that reconciles `FeatureStore` custom resources to deploy and manage Feast feature store infrastructure on OpenShift and Kubernetes clusters. It provisions a complete feature store deployment consisting of configurable online store (feature serving), offline store (batch feature retrieval), registry (metadata management), and UI services — all within a single Deployment with multiple containers.

The operator handles TLS configuration automatically on OpenShift using service serving certificates, supports both file-based and database-backed persistence options (including PVC management), and provides Kubernetes RBAC-based or OIDC-based authorization for feature store access. It also manages CronJobs for scheduled feature materialization, creates client ConfigMaps for consumer applications, and maintains a namespace registry to track deployed feature store instances across the cluster. For RHOAI integration, it supports Kubeflow Notebook integration by watching Notebook CRs and creating per-notebook ConfigMaps with feast project configurations.

The operator is deployed by the RHOAI rhods-operator using kustomize overlays. Container images for the operator, feature server, and CronJob are injected via `RELATED_IMAGE_*` environment variables, enabling disconnected/air-gapped deployment. The operator is built with FIPS-compliant Go runtime (`strictfipsruntime` build tag).

| NotebookConfigMap Controller | Go Operator (controller-runtime) | Watches Kubeflow Notebook CRs; creates per-notebook ConfigMaps with feast project client configurations |
| FeastAuthorization | Go package | Deploys Kubernetes RBAC (Roles, RoleBindings, ClusterRoles, ClusterRoleBindings) for feature store authorization |
| Feature Server | Python (gunicorn) | Serves features online (HTTP), runs offline store server, registry server (gRPC + REST), and UI |
| CronJob Runner | ose-cli container | Executes scheduled `feast apply` and `feast materialize-incremental` commands via `kubectl exec` into the Feast deployment |
| Namespace Registry | ConfigMap-based registry | Tracks all deployed FeatureStore client configs across namespaces in a central ConfigMap |
