## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates NVIDIA Garak red-teaming framework for automated LLM vulnerability scanning, shield testing, and compliance assessment.

**Detailed**: This component implements a Llama Stack provider for Garak, enabling automated red-teaming and vulnerability assessment of Large Language Models. It provides two execution modes: an **inline provider** that runs Garak scans locally within the Llama Stack server process, and a **remote provider** that delegates scan execution to Kubeflow Pipelines (KFP) running on OpenShift AI / Data Science Pipelines Application (DSPA). The remote mode is the primary production deployment pattern.

The provider supports predefined benchmark profiles mapped to industry compliance frameworks including OWASP LLM Top 10, AVID (AI Vulnerability and Incident Database), CWE (Common Weakness Enumeration), and quality assessments. It also supports custom benchmarks, shield testing (measuring guardrail effectiveness with input/output shield orchestration), and an intents-based risk assessment workflow with Synthetic Data Generation (SDG). Scan results are uploaded to the Llama Stack Files API and include JSONL reports, AVID-format reports, HTML reports, and hit logs. The provider also integrates with the eval-hub evaluation platform as a framework adapter, supporting both simple (subprocess) and KFP execution modes within Kubernetes Jobs.

The container image (`odh-trustyai-garak-lls-provider-dsp-rhel9`) is built via Konflux and serves as the base image for KFP pipeline component pods. It includes all Garak dependencies pre-installed to avoid runtime downloads.
