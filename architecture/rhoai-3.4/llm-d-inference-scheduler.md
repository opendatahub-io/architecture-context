## Purpose
**Short**: An inference request routing scheduler that makes optimized endpoint selection decisions for LLM inference workloads within the llm-d framework via Gateway API Inference Extension (GIE).

**Detailed**: The llm-d Inference Scheduler provides two core binaries: the **Endpoint Picker (EPP)** and the **P/D Routing Sidecar**. The EPP extends the upstream Kubernetes Gateway API Inference Extension project, acting as an external processing (ext-proc) plugin for Envoy. It receives incoming inference requests via gRPC from Envoy and makes intelligent routing decisions based on pluggable filters and scorers — including KV-cache locality, load awareness, session affinity, and prefix cache matching.

The P/D (Prefill/Decode) Sidecar runs alongside vLLM inference engines as a reverse proxy that orchestrates disaggregated inference. When the EPP determines a request should use split prefill/decode, the sidecar coordinates the two-phase execution: sending the prefill stage to a designated prefill worker, collecting KV transfer parameters, and forwarding the decode stage to the local vLLM instance. The sidecar supports multiple P/D connector protocols including NIXL v2, shared storage, and SGLang, with built-in SSRF protection via InferencePool-based allowlisting.

Both components are built for Kubernetes-native deployment, leveraging Gateway API resources (Gateway, HTTPRoute, InferencePool) and controller-runtime for Kubernetes API interactions. The scheduler plugin architecture supports filters (by-label, by-label-selector, decode-filter, prefill-filter), scorers (precise-prefix-cache, load-aware, active-request, session-affinity, no-hit-lru), and profile handlers for disaggregated P/D scheduling.
