## Purpose
**Short**: vLLM hardware plugin that enables optimized large language model inference on Intel Gaudi AI accelerators within RHOAI.

**Detailed**: The vLLM Hardware Plugin for Intel Gaudi (vllm-gaudi) integrates Intel Gaudi AI accelerators (HPU) with the vLLM inference engine to provide optimized large language model serving. It implements the vLLM hardware plugin architecture, registering as a platform plugin (`hpu`), custom ops plugin, and custom models plugin via Python entry points.

The plugin provides Gaudi-specific implementations for attention backends (including unified attention and MLA variants), custom operators (fused MoE, rotary embeddings, LayerNorm, FP8/AWQ/GPTQ quantization), distributed communication (HCCL-based tensor/pipeline parallelism), KV cache transfer via NIXL, speculative decoding (EAGLE), and LoRA adapter support. It ships as a container image (`odh-vllm-gaudi-rhel9`) built on UBI 9.6 with Habana Synapse drivers, running the vLLM OpenAI-compatible API server as its entrypoint.

The container image is built by Konflux using a multi-stage Dockerfile that installs Habana system drivers (stage 1), PyTorch for Habana (stage 2), and vLLM + the Gaudi plugin (stage 3). It runs as a non-root user (UID 2000) for OpenShift compatibility.
