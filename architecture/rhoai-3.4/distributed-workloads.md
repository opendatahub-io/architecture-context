## Purpose
**Short**: Provides container runtime images for distributed ML training workloads and a comprehensive end-to-end test suite for validating distributed workload components on OpenShift AI.

**Detailed**: The distributed-workloads repository serves two primary functions within the Red Hat OpenShift AI (RHOAI) platform. First, it builds and ships container runtime images used for distributed machine learning training jobs. These images include various combinations of Python versions, GPU acceleration libraries (NVIDIA CUDA and AMD ROCm), and PyTorch versions, packaged on top of UBI9 (Red Hat Universal Base Image 9). The images support both traditional training runtimes and a new "universal" image format (Training Hub 0.6) that combines Jupyter workbench capabilities with ML runtime dependencies.

Second, the repository contains a comprehensive Go-based end-to-end test suite that validates the distributed workload stack components: Kubeflow Training Operator (PyTorchJob), Kubeflow Trainer v2 (TrainJob), KubeRay (RayCluster/RayJob), Kueue (job queuing and scheduling), JobSet, and the Kueue Operator. The test suite exercises these components on OpenShift clusters to verify correct behavior including GPU training, model fine-tuning (SFT, LoRA, OSFT), Kueue quota management, upgrade scenarios, and integration with DataScienceCluster CRDs. Tests interact with both ODH (Open Data Hub) and RHOAI product configurations, validating the distributed workloads stack across both distributions.
