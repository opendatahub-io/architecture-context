## Purpose
**Short**: A lightweight Rust-based HTTP service that detects sensitive content (PII) in text using regex pattern matching, serving as a detector backend for the FMS Guardrails Orchestrator.

**Detailed**: The Guardrails Regex Detector is a focused microservice written in Rust using the Axum web framework. It provides a REST API endpoint that accepts text content and applies regex-based pattern matching to detect sensitive information such as email addresses, Social Security numbers, and credit card numbers.

The service integrates with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator) as a pluggable detector backend. It supports both built-in detection patterns (email, SSN, credit card) and arbitrary custom regex patterns provided at request time. Each detection returns the matched text, position offsets, detection type classification, and a confidence score (always 1.0 for exact regex matches).

The service is designed to be stateless and horizontally scalable, with no external dependencies beyond the network connection from the orchestrator. It runs on a configurable HTTP port (default 8080) and exposes a health check endpoint for Kubernetes liveness/readiness probes.
