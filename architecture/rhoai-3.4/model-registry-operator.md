## Purpose
**Short**: Manages the lifecycle of Model Registry and Model Catalog instances on OpenShift/Kubernetes clusters.

**Detailed**: The Model Registry Operator is a controller-runtime based Kubernetes operator that reconciles `ModelRegistry` custom resources (CRD group: `modelregistry.opendatahub.io`). For each `ModelRegistry` CR, the operator provisions a complete model registry service stack including the REST API deployment, PostgreSQL database (optional auto-provisioned), kube-rbac-proxy sidecar for authentication, OpenShift Routes for external access, ServiceAccounts, RBAC resources, and NetworkPolicies.

The operator also includes a `ModelCatalogReconciler` that manages a singleton Model Catalog instance, providing a curated catalog of AI/ML models with its own PostgreSQL backend, kube-rbac-proxy authentication, and OpenShift Route exposure. The operator supports multiple platform modes: traditional OpenShift, BYOIDC-enabled OpenShift 4.20+ (where user.openshift.io APIs are absent), and plain Kubernetes.

The operator has migrated from OAuth proxy to kube-rbac-proxy for authentication (RHOAI 3.x pattern). It automatically migrates existing `OAuthProxy` configurations to `KubeRBACProxy` during reconciliation. It also handles CRD storage version migration between v1alpha1 and v1beta1 using both StorageVersionMigration API and manual fallback strategies.
