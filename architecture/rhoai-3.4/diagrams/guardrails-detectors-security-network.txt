RBAC Summary:
- No ClusterRoles defined in this repository
- Deployment relies on KServe and platform-level RBAC
- Example RoleBinding: user-one-view (ClusterRole: view) → SA: user-one (example only)

Secrets:
- aws-connection-minio-data-connection-guardrails-guardian (Opaque)
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_S3_ENDPOINT, AWS_S3_BUCKET
  - Purpose: S3 access for HF model download
  - Provisioned by: Manual / Data connection UI
  - Auto-rotate: No

Authentication & Authorization:
- /api/v1/text/contents (POST): Istio mTLS + KServe auth annotation
  - Enforcement: Istio sidecar + security.opendatahub.io/enable-auth: 'true'
- /api/v1/text/generation (POST): Istio mTLS + KServe auth annotation
- /health (GET): Open (no auth)
- /metrics (GET): Open (Prometheus scraping)

TLS Configuration:
- Application-level TLS: Optional (ssl.CERT_REQUIRED when ssl_ca_certs present)
- No explicit cipher suite restrictions
- Relies on UBI 9 system OpenSSL (3.5.1-5.el9_7 pinned in HF Dockerfile)
- No non-FIPS crypto usage detected

Security Notes:
- Custom detector subsystem performs AST-based static code analysis
  - Blocks imports: os, subprocess, sys, shutil
  - Blocks calls: eval, exec, open, compile, input
  - Best-effort inspection, NOT process isolation boundary
- No Konflux Dockerfile for LLM Judge detector on this branch
  (standard Dockerfile.judge only — different shipping maturity)
