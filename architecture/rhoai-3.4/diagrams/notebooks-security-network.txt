RBAC Summary:
- No RBAC resources defined by this repository
- RBAC for workbench pods is managed externally by:
  - odh-notebook-controller: Injects kube-rbac-proxy sidecar for AuthN/AuthZ
  - Platform operator: Manages namespace-level RBAC for workbench service accounts

Authentication & Authorization:
- JupyterLab UI: OAuth/OIDC via kube-rbac-proxy sidecar (user must have namespace access)
- Code-Server UI: OAuth/OIDC via kube-rbac-proxy sidecar (user must have namespace access)
- RStudio UI: OAuth/OIDC via kube-rbac-proxy sidecar (user must have namespace access)
- JupyterLab API (internal): No auth (ServerApp.token=''), pod-internal only, auth at sidecar

Secrets:
- (DSPA connection secret) (Opaque) - Data Science Pipelines endpoint and credentials, provisioned by odh-notebook-controller, No auto-rotate
- (CA bundle) (Opaque) - Custom CA certs at /etc/pki/tls/custom-certs/ca-bundle.crt, provisioned by odh-notebook-controller, No auto-rotate

FIPS Compliance:
- No Go binaries in shipped images (Python/R/Node.js runtimes)
- OpenSSL inherited from UBI9 base image
- compat-openssl11 installed for openshift-clients compatibility
- TLS via Python stdlib ssl module (delegates to system OpenSSL)
- Risk: User-installed packages may introduce non-FIPS crypto at runtime

Build Hermeticity:
- OS packages (RPM): rpms.lock.yaml (rpm-lockfile-prototype)
- Python deps: pylock.*.toml per image variant (uv, PEP 751)
- Go deps: go.sum (go mod) - for build tools only
- Artifacts: artifacts.lock.yaml (Hermeto, ODH only)
- Konflux Dockerfiles: fully air-gapped, all deps from /cachi2/output/deps/
