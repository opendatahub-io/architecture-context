RBAC Summary:
- This component does not define RBAC resources directly
- RBAC for workbench pods is managed by odh-notebook-controller and platform operator
- kube-rbac-proxy sidecar enforces access control based on user's OpenShift RBAC permissions
- User must have access to the Notebook CR namespace to use workbench

Authentication & Authorization:
- Workbench UI: OpenShift RBAC via kube-rbac-proxy (8443/TCP)
  - Methods: GET, POST, WebSocket
  - Policy: User must have access to the Notebook CR namespace
- /api/kernels/: OpenShift RBAC via kube-rbac-proxy (8443/TCP)
  - Purpose: Internal idle culler access
