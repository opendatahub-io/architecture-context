RBAC Summary:
  None — this component does not define Kubernetes RBAC resources.
  Runs as UID 1001 (non-root) without Kubernetes API access.

Service Mesh Configuration:
  No component-level service mesh configuration.
  Service mesh (if present) is managed by the RHOAI platform operator.

Secrets:
  - LLM API Key (environment variable)
    - Type: Environment variable
    - Purpose: Authentication to upstream LLM provider
    - Provisioned By: Manual / External
    - Auto-Rotate: No

  - OTEL configuration (environment variables)
    - Type: Environment variables (OTEL_EXPORTER_OTLP_ENDPOINT, OTEL_SERVICE_NAME)
    - Purpose: OpenTelemetry endpoint and service name configuration
    - Provisioned By: Platform operator
    - Auto-Rotate: No

Authentication & Authorization:
  - No built-in authentication on any endpoint
  - AuthN/AuthZ delegated entirely to platform ingress (kube-rbac-proxy / Gateway API)
  - /v1/models endpoint forwards Authorization header to upstream LLM provider
  - No service-to-service auth for internal actions server communication

FIPS Compliance:
  - Python application on UBI 9 — inherits FIPS-compliant OpenSSL from base image
  - cryptography==44.0.3 (transitive via Presidio) links against system OpenSSL
  - hashlib.md5 used for non-cryptographic cache key generation only
    - Fallback to SHA256 when MD5 unavailable in FIPS-restricted environments
  - No custom TLS configuration — relies on system trust store

Network Policy Recommendations:
  - INGRESS: Allow from RHOAI Gateway/kube-rbac-proxy on port 8000/TCP only
  - EGRESS: Allow to upstream LLM endpoints on port 443/TCP
  - EGRESS: Allow to OTel Collector on port 4317/TCP (when configured)
  - EGRESS: Allow to Redis on port 6379/TCP (when configured)
  - DENY: All other ingress/egress traffic
