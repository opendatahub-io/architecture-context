RBAC Summary:
- N/A — The autogluon server runs as a model serving container, not an
  operator. It has no direct Kubernetes API interaction. RBAC for the
  inference pods is managed by the KServe controller.

Authentication & Authorization:
- kube-rbac-proxy sidecar:
  - Listens on 8443/TCP (HTTPS, TLS 1.2+)
  - Forwards authenticated requests to 8080/TCP (HTTP, localhost)
  - Validates Bearer Tokens via SubjectAccessReview
  - Injected by RHOAI platform operator
- /metrics endpoint:
  - No authentication (internal scraping by Prometheus)
  - Access controlled via NetworkPolicy

Secrets:
- (storage credentials) [Opaque]
  - Purpose: S3/GCS/Azure credentials for model download
  - Provisioned by: User / platform admin
  - Auto-Rotate: No
  - Used by: Storage Initializer init container

FIPS Compliance:
- Python service — no Go binary compilation
- Uses UBI9 system Python with OpenSSL (FIPS-capable)
- cryptography==46.0.7 with OpenSSL backend
- No non-FIPS crypto detected in application code
- TLS termination handled by platform ingress, not by this component
