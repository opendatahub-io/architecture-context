RBAC Summary:
- No RBAC roles or role bindings defined by this component.
- The autogluon server runs as a model server container within KServe
  InferenceService pods; all RBAC is managed by the KServe controller.
- ServiceAccount is created per-InferenceService by KServe controller.

Authentication & Authorization:
- /v1/models/*:predict  — No auth at container level; enforced at KServe ingress
- /v2/models/*/infer    — No auth at container level; enforced at KServe ingress
- /metrics              — No auth; cluster-internal Prometheus scrape only

Secrets:
- (storage credentials) [Opaque] — S3/GCS/Azure credentials for model download
  - Provisioned by: User / platform admin
  - Auto-Rotate: No
  - Contains: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY (for S3),
              or equivalent credentials for GCS/Azure/HuggingFace

FIPS Compliance:
- Python application — no Go binary, no direct TLS configuration
- Uses system OpenSSL via cryptography==46.0.7 (FIPS-capable on UBI9)
- Base image: registry.redhat.io/rhai/base-image-cpu-rhel9 (provides OpenSSL)
- Certificate handling: system trust store from UBI9
- No non-FIPS crypto risks detected in production code paths
- hashlib.md5 usage found in test code only (not production)
