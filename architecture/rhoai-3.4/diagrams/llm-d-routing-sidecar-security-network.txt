Security Notes:
- SSRF protection validates host-only (any port on allowed pod IP is permitted)
- No authentication on proxy itself — relies on network-level isolation
- InsecureSkipVerify available via flags but disabled by default
- All cipher suites are FIPS-approved AEAD ciphers (no MD5/RC4/DES)
- CGO_ENABLED=0 in upstream Dockerfile — not FIPS compliant for check-payload
