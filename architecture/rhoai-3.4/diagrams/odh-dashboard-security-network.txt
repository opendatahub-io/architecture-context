================================================================================
 AUTHENTICATION & AUTHORIZATION SUMMARY
================================================================================

  Layer 1 - Gateway API:
    - External TLS termination (Gateway certificate)
    - Routes to pod port 8443

  Layer 2 - kube-rbac-proxy:
    - Validates OpenShift OAuth token
    - Check: can user "list" project.openshift.io/projects?
    - Injects identity headers: x-forwarded-access-token, x-auth-request-user

  Layer 3 - Application:
    - secureRoute(): namespace isolation (users restricted to own workbench ns)
    - secureAdminRoute(): admin-only endpoints (cluster-settings, model registries)
    - WebSocket: JWT via subprotocol (base64url.bearer.authorization.k8s.io.{token})
    - BFF modules: receive pre-validated token via x-forwarded-access-token header

================================================================================
 FIPS COMPLIANCE NOTES
================================================================================

  - Main dashboard (Node.js): inherits OpenSSL from UBI 9 base image
  - BFF modules (Go): CGO_ENABLED=1, -tags strictfipsruntime (FIPS-compliant)
  - esbuild Go binaries explicitly removed (non-FIPS, transitive dependency)
  - JWT decoded without verification in backend (kube-rbac-proxy already validated)
