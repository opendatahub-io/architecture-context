TLS Configuration:
- Auto-detection: Gateway checks for presence of TLS cert files at startup
  - If /etc/tls/private/tls.crt exists → HTTPS with mTLS to orchestrator
  - If no certs found → HTTP plaintext to orchestrator
- PKCS#12 identity constructed at runtime from PEM files using openssl crate
- CA trust: Custom CA from /etc/tls/ca/service-ca.crt
- danger_accept_invalid_hostnames(true) ONLY when host == "localhost" (dev only)
- Crypto: System OpenSSL via openssl + native-tls crates (FIPS-capable on UBI 9)

Secrets:
- /etc/tls/private/tls.crt (kubernetes.io/tls) - Client cert for mTLS
  - Provisioned by: cert-manager or service-serving-cert (external)
  - Auto-Rotate: Unknown
- /etc/tls/private/tls.key (kubernetes.io/tls) - Private key for mTLS
  - Provisioned by: cert-manager or service-serving-cert (external)
  - Auto-Rotate: Unknown
- /etc/tls/ca/service-ca.crt (Opaque) - CA certificate for orchestrator TLS verification
  - Provisioned by: service-ca-operator (external)
  - Auto-Rotate: Unknown

RBAC Summary:
- No RBAC manifests in this repository
- RBAC is expected to be defined by the consuming deployment
  (e.g., TrustyAI operator or platform operator)

Authentication & Authorization:
- Endpoint: /{route}/v1/chat/completions (POST)
- Auth Mechanism: Header forwarding (Authorization, X-Forwarded-*)
- Enforcement Point: None (gateway passes through to orchestrator)
- Policy: No local auth enforcement — all auth delegated to orchestrator

FIPS Compliance:
- Rust binary dynamically links against system OpenSSL on UBI 9
- openssl crate (0.10.73) + native-tls (0.2.12) delegate to system OpenSSL
- FIPS-validated crypto available when host is in FIPS mode
- No bundled crypto — all crypto operations use system libraries
- Risk: danger_accept_invalid_hostnames(true) for localhost only (dev)

Observability:
- Structured logging via tracing/tracing-subscriber (DEBUG level)
- No Prometheus metrics endpoint
- No health/readiness HTTP endpoints (TCP socket probe on 8090)
