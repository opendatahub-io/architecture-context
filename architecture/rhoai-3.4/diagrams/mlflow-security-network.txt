RBAC Summary:
- MLflow does not define Kubernetes RBAC resources directly
- RBAC is managed by the deploying operator (rhods-operator)
- Required permissions for MLflow ServiceAccount:
  - Read access to namespaces (for workspace discovery)
  - TokenReview access (for SA token validation)

Application-Level Authorization:
- Permission model: READ / USE / EDIT / MANAGE (per-resource)
- Workspace scoping: Three-tier hierarchy
  1. Resource-specific permissions
  2. Workspace-level permissions
  3. Default permissions
- Permission caching: 10,000-entry TTL cache
- Admin bypass: GraphQL field-level checks skipped for admins

Secrets:
- MLFLOW_CRYPTO_KEK_PASSPHRASE (Opaque) - KEK passphrase for envelope encryption, manual provisioning, no auto-rotate
- MLFLOW_FLASK_SERVER_SECRET_KEY (Opaque) - Flask CSRF protection key, manual provisioning, no auto-rotate
- Backend store credentials (Opaque) - PostgreSQL connection URI, provisioned by platform operator, no auto-rotate
- S3/Azure/GCS credentials (Opaque) - Artifact storage access credentials, provisioned by platform operator, no auto-rotate

Configuration Risks:
- MLFLOW_S3_IGNORE_TLS: Can disable S3 TLS verification (env var flag)
- MLFLOW_TRACKING_INSECURE_TLS: Can disable tracking server TLS verification (client-side)

FIPS Compliance:
- Python on UBI 9 inherits FIPS from system OpenSSL
- cryptography package uses OpenSSL backend (FIPS mode when system policy enabled)
- Algorithms: PBKDF2-HMAC-SHA256 (600k iterations), AES-256-GCM — both FIPS-approved
- hashlib.sha256 used for non-cryptographic job parameter hashing (acceptable)
- No non-FIPS crypto primitives detected in production code paths
