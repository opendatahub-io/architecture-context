RBAC Summary:
  No RBAC manifests in this repository. RBAC is managed externally by the
  KServe/RHOAI operator that deploys this container.

  Platform-level auth is enforced by the kube-rbac-proxy sidecar injected
  by RHOAI, which validates OAuth/OIDC tokens and performs RBAC checks
  before traffic reaches the vLLM process.

Secrets:
  - SSL key/cert (--ssl-keyfile, --ssl-certfile)
    Type: File mount
    Purpose: TLS termination at vLLM server
    Provisioned by: Manual / cert-manager
    Auto-rotate: Optional (--enable-ssl-refresh watches files via watchfiles)

  - SSL CA certs (--ssl-ca-certs)
    Type: File mount
    Purpose: Client certificate verification (mTLS)
    Provisioned by: Manual / cert-manager
    Auto-rotate: Optional (--enable-ssl-refresh)

  - API key (--api-key or VLLM_API_KEY)
    Type: Env var / CLI arg
    Purpose: Bearer token auth for /v1/* endpoints (SHA-256 validated)
    Provisioned by: Manual
    Auto-rotate: No

  - HuggingFace token (HF_TOKEN)
    Type: Env var
    Purpose: Authentication for gated model downloads
    Provisioned by: Manual
    Auto-rotate: No

  - S3 credentials (S3_ACCESS_KEY_ID, S3_SECRET_ACCESS_KEY)
    Type: Env var
    Purpose: S3-compatible model storage authentication
    Provisioned by: Manual
    Auto-rotate: No

Container Security:
  - Runs as non-root: UID 2000, GID 0
  - Base image: UBI 9 (ubi9/ubi-minimal)
  - FIPS: Inherited from system OpenSSL in UBI 9 base
  - Usage telemetry: Disabled (VLLM_NO_USAGE_STATS=1)
  - Build hermeticity: NOT hermetic (hermetic: false in Tekton pipeline)
  - No dependency lock files present (supply chain reproducibility gap)

FIPS Compliance Notes:
  - Python inherits FIPS from system OpenSSL in UBI 9
  - hashlib.md5 used with usedforsecurity=False (non-crypto, not a violation)
  - No custom cipher suite restrictions
  - TLS configurable via --ssl-keyfile/--ssl-certfile (delegates to system)
