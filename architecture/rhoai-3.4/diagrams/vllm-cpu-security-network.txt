RBAC Summary:
- No ClusterRoles or RoleBindings defined by vLLM CPU
- vLLM is a containerized service, not a Kubernetes operator
- RBAC is managed by KServe at the pod level
- ServiceAccount: assigned by KServe InferenceService

Secrets:
- SSL key/cert (--ssl-keyfile/--ssl-certfile)
  - Type: kubernetes.io/tls
  - Purpose: TLS termination for HTTPS serving
  - Provisioned By: Platform/cert-manager (optional)
  - Auto-Rotate: Yes (SSLCertRefresher file watcher)

- API key (--api-key / VLLM_API_KEY)
  - Type: Opaque
  - Purpose: API authentication for inference endpoints
  - Provisioned By: Manual
  - Auto-Rotate: No

- HF_TOKEN
  - Type: Opaque
  - Purpose: Hugging Face Hub authentication for gated model downloads
  - Provisioned By: Manual
  - Auto-Rotate: No

- S3 credentials (S3_ACCESS_KEY_ID / S3_SECRET_ACCESS_KEY)
  - Type: Opaque
  - Purpose: S3-compatible object storage authentication
  - Provisioned By: Manual
  - Auto-Rotate: No

Authentication & Authorization:
- /v1/* endpoints: Bearer API Key (optional), enforced by AuthenticationMiddleware
- /health, /metrics, /ping: No authentication (always accessible)
- External access: Platform RBAC via KServe InferenceService / Gateway

TLS Configuration:
- Server-side TLS: Optional via --ssl-keyfile, --ssl-certfile, --ssl-ca-certs
- SSL cipher configuration: --ssl-ciphers flag
- Certificate hot-reload: SSLCertRefresher watches cert files via watchfiles
- Default: Plaintext HTTP (relies on KServe gateway for TLS termination)

FIPS Compliance:
- Python application (no Go binaries, no GOEXPERIMENT=strictfipsruntime)
- Uses Python stdlib ssl module (delegates to system OpenSSL from UBI 9)
- blake3 used for content hashing (non-cryptographic, FIPS-irrelevant)
- No non-FIPS crypto risks detected in application code
