RBAC Summary:
- No ClusterRoles defined by this repository
- ServiceAccount: sa (automountServiceAccountToken: false)
- Minimal RBAC exposure by design
- WVA creates RoleBindings for metrics-reader access to Prometheus

Secrets:
- hf-token (Opaque) - HuggingFace API token for model downloads
  Provisioned: Manual (user-provided), Auto-Rotate: No
- prometheus-web-tls (kubernetes.io/tls) - TLS certs for Prometheus HTTPS
  Provisioned: Script-generated (self-signed), Auto-Rotate: No
- controller-manager-token (Opaque) - WVA controller auth token
  Provisioned: Manual, Auto-Rotate: No

Pod Security (OpenShift):
- SecurityContext: runAsNonRoot, readOnlyRootFilesystem
- Capabilities: drop ALL
- Pod-level security contexts defined in OpenShift-specific gateway patches

Network Security Notes:
- ENTIRE inference data path is unencrypted HTTP within cluster
- No mTLS, no network policies defined in upstream manifests
- Security relies on Kubernetes network isolation (namespace, NetworkPolicy)
- Platform operator expected to add:
  - Network policies for pod-to-pod restrictions
  - Gateway-level authentication (Istio RequestAuthentication, etc.)
  - kube-rbac-proxy sidecars for metrics endpoints (RHOAI integration)
  - Service mesh mTLS if required by deployment environment
