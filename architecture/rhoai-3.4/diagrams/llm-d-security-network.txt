  - API Groups: ""
  - Resources: pods

Secrets:
- hf-secret (Opaque) - Hugging Face API token for model downloads
  - Provisioned by: Manual (user-provisioned)
  - Auto-Rotate: No
  - Mounted as: env var HF_TOKEN in model server pods

Authentication & Authorization Summary:
- /v1/completions (POST): No auth (open within cluster network)
- /v1/chat/completions (POST): No auth (open within cluster network)
- /health (GET): No auth (probe endpoint)
- /metrics (GET): No auth (metrics scraping)
- Gateway (external): Configurable per provider (AgentGateway/Istio/GKE)

Security Notes:
- All in-cluster communication is PLAINTEXT (no TLS, no mTLS)
- TLS termination is delegated to the Gateway provider
- NIXL KV-cache transfer (5600/TCP) is unauthenticated and unencrypted
- NVSHMEM/UCX GPU communication uses dynamic ports with no encryption
- Security relies on Kubernetes network policies for pod-to-pod isolation
- In multi-tenant environments, NetworkPolicies are critical for NIXL/NVSHMEM isolation

FIPS Compliance:
- No Go binaries (Python-based runtime) - N/A for check-payload
- No TLS at application level - crypto delegated to infrastructure
- Base images: UBI9 (CUDA) / Ubuntu 22.04 (CPU) provide system OpenSSL
- Downstream RHOAI enforces FIPS via Konflux pipeline on release branches
