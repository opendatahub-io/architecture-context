    Auth: None (application-level)
    Enforcement: Relies on:
      1. Kubernetes namespace isolation
      2. NetworkPolicies
      3. kube-rbac-proxy sidecar (injected by TrustyAI Operator)
    WARNING: No application-level authentication on data ingestion or metric endpoints

FIPS Compliance:
  Java-based component on UBI9 (openjdk-17-runtime)
  FIPS compliance delegated to OS-level OpenSSL and JVM FIPS provider
  No application-level non-FIPS crypto detected
  JVM auto-uses system FIPS provider when host is in FIPS mode

Security Observations:
  1. OpenShift Route has tls: null — service exposed over plaintext HTTP externally
  2. ClusterRole grants configmap CRUD cluster-wide (namespace Role would be more restrictive)
  3. No application-level auth on REST endpoints — relies entirely on network controls
  4. Database connection (when used) is plaintext JDBC with no TLS
  5. KServe model server connections are plaintext (gRPC and HTTP)
  6. TLS available on port 4443 but not enforced by default ingress
