INGRESS: None
  This component is a batch job. It does not accept inbound connections.
  No Kubernetes Services, Ingress, or Routes are created.

RBAC Summary:
  No RBAC resources defined in this repository.
  RBAC is managed by the parent operator (TrustyAI/EvalHub Operator)
  that creates and manages the Kubernetes Job.

Secrets:
  - Model auth secret (model.auth.secret_ref)
    Type: Opaque
    Contents: OPENAI_API_KEY, optional hf-token key
    Provisioned by: User/Operator
    Auto-Rotate: No

  - OCI registry auth (/tmp/.docker/config.json)
    Type: Docker config
    Contents: Registry authentication credentials
    Provisioned by: EvalHub/Operator
    Auto-Rotate: No

  - CA certificate bundle (verify_certificate)
    Type: Opaque
    Contents: Custom CA certificate for model endpoint TLS
    Provisioned by: User/Operator
    Auto-Rotate: No
    Note: Validated to not be symlink (defense against K8s
          secret volume mount symlink attacks)

TLS/Certificate Handling:
  - Uses ssl.create_default_context() with system trust store
  - Supports custom CA bundle via verify_certificate parameter
  - Custom CA paths validated: not symlink, absolute path, regular file
  - FIPS-compliant via UBI9 system OpenSSL when FIPS mode enabled
  - No non-FIPS crypto: hashlib.sha256 used only for cache keys (non-security)

Authentication Mechanisms:
  - Model Endpoint: Bearer Token via OPENAI_API_KEY env var
  - EvalHub Callbacks: Bearer Token from job spec
  - HuggingFace Hub: HF_TOKEN env var (optional)
  - OCI Registry: Docker config or OCI_USERNAME/OCI_PASSWORD
  - S3 Storage: AWS IAM / IBM COS credentials
  - MLflow: Bearer Token
