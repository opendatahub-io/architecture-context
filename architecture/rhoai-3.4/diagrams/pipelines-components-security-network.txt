Security Notes:
⚠ Kubeflow Model Registry uses HTTP (port 8080) with no TLS and no authentication
  - Risk: Model metadata in transit is unencrypted and unauthenticated
  - Recommendation: Enable mTLS via service mesh for in-cluster communication

⚠ SSL verification bypass (verify=False) used as fallback in AutoRAG components
  - Affected: documents_discovery, documents_indexing
  - Risk: MITM attacks on S3 connections when SSL errors occur
  - Recommendation: Fix certificate chain rather than bypassing verification

⚠ LlamaStack TLS is optional (LLAMA_STACK_TLS_INSECURE flag)
  - Risk: API key transmitted in plaintext if TLS disabled
  - Recommendation: Enforce TLS in production deployments

⚠ No secrets have auto-rotation configured
  - Risk: Long-lived credentials increase exposure window
  - Recommendation: Implement secret rotation policies

FIPS Compliance:
- Python application — no Go binaries, no GOEXPERIMENT=strictfipsruntime needed
- Crypto via cryptography==46.0.7 (PyCA wrapping OpenSSL — FIPS-capable on UBI9)
- Base images: UBI9 (FIPS-validated OpenSSL)
- Non-FIPS risk: SSL verify=False bypasses FIPS-validated TLS path
