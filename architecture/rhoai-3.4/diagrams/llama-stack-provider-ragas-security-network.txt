RBAC Summary:
  No RBAC resources (ClusterRoles, Roles, RoleBindings) are defined by this
  component. It runs within the Llama Stack server pod and relies on:
  - Llama Stack Operator's ServiceAccount for Kubernetes API access
  - Explicit Bearer tokens for KFP API access (via Secret)
  - AWS credentials for S3 access (via Kubernetes Secret)

Secrets:
  - kubeflow-pipelines-token (Opaque) - Bearer token for KFP API auth
    Provisioned: Manual (oc whoami -t)
    Auto-Rotate: No (user token, expires)

  - llama-stack-inference-model-secret (Opaque) - vLLM endpoint URL, API token, TLS settings
    Provisioned: Manual
    Auto-Rotate: No

  - aws-credentials (Opaque) - AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_DEFAULT_REGION
    Provisioned: Manual
    Auto-Rotate: No

  - S3 credentials (for KFP steps) (Opaque) - AWS credentials mounted via kubernetes.use_secret_as_env
    Provisioned: Manual
    Auto-Rotate: No

Security Concerns:
  ⚠️  KFP pipeline pods communicate with Llama Stack Server over plaintext HTTP
     (no TLS, no authentication between KFP pods and LLS)
  ⚠️  Demo deployment includes VLLM_TLS_VERIFY="false" which bypasses TLS
     certificate verification
  ⚠️  Secrets are manually provisioned with no auto-rotation
  ✅ pipelines_api_token handled as Pydantic SecretStr (prevents log leakage)
  ✅ Python cryptography library uses system OpenSSL (FIPS-compliant on UBI9)

FIPS Status:
  - Base image: registry.redhat.io/ubi9/python-312 (FIPS-validated OpenSSL)
  - Crypto: cryptography==46.0.2 (uses system OpenSSL backend)
  - No explicit FIPS opt-in in source code
  - FIPS compliance relies on OS-level FIPS mode being enabled
