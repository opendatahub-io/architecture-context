FIPS Compliance Notes:
- Application uses rustls with ring crypto backend (BoringSSL-derived, NOT FIPS-certified)
- OpenSSL 3.5.1-7.el9_7 installed in image for check-payload compliance only
- Application TLS does NOT use system OpenSSL — potential FIPS gap
- NoVerifier struct (src/utils/tls.rs) bypasses cert validation when insecure=true
  (development/testing use only)

Container Security:
- Runs as non-root user: orchestr8 (UID 1001, GID 0)
- Base image: UBI 9 minimal
- Multi-stage build with cargo test in builder stage
- No capabilities or privilege escalation

Header Security:
- passthrough_headers config controls header forwarding to backends
- rewrite_forwarded_access_header: converts X-Forwarded-Access-Token → Authorization: Bearer
  (for KServe/InferenceService integration)
