RBAC Summary:
- MLServer is a Python application (not a Kubernetes operator)
- Does NOT define RBAC roles, ClusterRoles, or ServiceAccounts
- RBAC is managed entirely by KServe/ModelMesh infrastructure

Secrets:
- MLServer does NOT directly reference Kubernetes secrets
- Configuration is environment-variable-based via MLSERVER_ prefix
- Model storage credentials managed by KServe/ModelMesh (e.g., storage-config secret)

Service Mesh Configuration:
- MLServer relies on external service mesh (Istio) for mTLS
- Istio sidecar injection provides pod-to-pod encryption
- Gateway API + kube-rbac-proxy provides external access authentication (RHOAI 3.x)
- MLServer binds to plaintext ports; TLS termination is external

Authentication & Authorization:
- All REST endpoints (8080/TCP): No application-level auth
- All gRPC endpoints (8081/TCP): No application-level auth
- Metrics endpoint (8082/TCP): No auth (restricted by NetworkPolicy)
- Auth enforcement: External (KServe/Istio/Gateway API/kube-rbac-proxy)

Network Binding:
- Host: 0.0.0.0 (all interfaces)
- All three servers bind to plaintext (no TLS)
- Container runs as UID 1000 (non-root)
- Compatible with OpenShift random UID assignment
