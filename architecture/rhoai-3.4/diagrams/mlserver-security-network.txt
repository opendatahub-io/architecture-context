RBAC Summary:
- No ClusterRoles defined (MLServer does not interact with Kubernetes API)
- No RoleBindings defined
- No ServiceAccount RBAC permissions required
- All Kubernetes interactions are filesystem-based (mounted volumes, SA token)

Secrets:
- No Kubernetes secrets defined or required by MLServer
- TLS certificates managed by platform gateway infrastructure
- Model storage credentials managed by KServe storage initializer

Runtime Security:
- trusted-runtimes.json: Baked allowlist of permitted model implementation import paths
- Production mode enforced when /etc/mlserver/trusted-runtimes.json exists
- Prevents arbitrary code execution via model-settings.json injection
- Custom Python environments rejected in production mode
- Wildcard CORS origins rejected in production mode
- Build-time validation: is_valid_runtime_import_path() check

FIPS Compliance Notes:
- No application-level TLS (gRPC uses add_insecure_port(), REST uses plaintext HTTP)
- No direct crypto library imports
- Relies on system OpenSSL from UBI base image for FIPS compliance
- OpenTelemetry exporter uses insecure=True (no TLS to collector)
- TLS termination expected at platform gateway layer
- batch_processing.py: ssl_context=False skips verification, ssl_context=None uses system default

Container Image:
- Base: quay.io/aipcc/base-images/cpu:3.4.0 (RHOAI UBI base)
- Dependencies installed from Red Hat-hosted PyPI index
- Hash-pinned via requirements-cpu.txt
- No Hermeto/cachi2 prefetch integration
