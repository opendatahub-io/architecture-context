Authentication Flow:
- External Client → Istio Gateway: User Bearer token (TLS 1.2+)
- Istio Gateway → Payload Processing: None (pod-local gRPC, plaintext)
- Payload Processing → K8s API: ServiceAccount token (TLS 1.2+)
- Payload Processing → NeMo: None (relies on network/mTLS)
- Envoy → External Providers: Injected API key per provider format
  - Original user Authorization header STRIPPED before forwarding
