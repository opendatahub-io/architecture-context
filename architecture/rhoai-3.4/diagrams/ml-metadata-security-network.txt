Secrets:
- Database credentials (Opaque) - MySQL/PostgreSQL user and password
  - Provisioned by: DSP operator / administrator
  - Auto-rotate: No

- SSL/TLS certificates (kubernetes.io/tls) - Server key/cert for gRPC TLS, optional client CA for mTLS
  - Provisioned by: Administrator / cert-manager
  - Auto-rotate: No

- MySQL SSL certificates (Opaque) - Client key/cert/CA for MySQL TLS connections
  - Provisioned by: Administrator
  - Auto-rotate: No

- PostgreSQL SSL certificates (Opaque) - Client key/cert/CA/password for PostgreSQL TLS connections
  - Provisioned by: Administrator
  - Auto-rotate: No

Authentication:
- Endpoint: MetadataStoreService (all RPCs)
- Method: gRPC
- Auth Mechanism: Optional mTLS (SSLConfig.client_verify)
- Enforcement Point: gRPC server (BoringSSL)
- Policy: If client_verify=true, client must present valid certificate

FIPS Compliance Notes:
- C++ binary (not Go) — Go-specific FIPS mechanisms do not apply
- Crypto library: BoringSSL (compiled from source via Bazel)
- BoringSSL has FIPS 140-2 validation (Google) but NOT Red Hat-validated
- Binary is statically linked against BoringSSL, NOT system OpenSSL
- POTENTIAL FIPS COMPLIANCE GAP for Red Hat-certified crypto requirements

Build Security:
- Build is NOT hermetic (hermetic: false in Tekton pipeline)
- No RPM lock file (rpms.lock.yaml absent)
- No Hermeto/cachi2 prefetch integration
- Bazel deps pinned by sha256 but downloaded at build time
- Container runs as non-root (65534:65534)
- Multi-arch: amd64, arm64, ppc64le
