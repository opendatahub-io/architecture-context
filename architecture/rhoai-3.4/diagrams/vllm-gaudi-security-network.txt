RBAC Summary:
  RBAC is fully delegated to KServe — this container defines no ClusterRoles,
  Roles, or RoleBindings. The KServe controller manages all RBAC for the
  InferenceService pod.

Authentication & Authorization:
  - /v1/* endpoints: No auth at container level
  - Enforcement point: KServe / Platform ingress (kube-rbac-proxy or Gateway API)
  - Policy: Fully delegated to platform

Secrets:
  - No secrets used directly by the container
  - Model access credentials are mounted by the platform (KServe storage initializer)

Security Observations:
  ⚠ openssl-fips-provider-so explicitly removed from UBI9 base image
    (Dockerfile.konflux.gaudi line 49) — container runs with standard
    (non-FIPS) OpenSSL. Required for Habana software stack compatibility.

  ⚠ HCCL and NIXL inter-device/inter-node communication is unencrypted
    with no authentication. Data in transit between HPU cards and nodes
    (including model activations and KV cache) is transmitted in plaintext
    over RDMA/TCP.

  ⚠ Build is non-hermetic (hermetic: false in Tekton PipelineRun manifests).
    No pip lock files, RPM lock files, or Hermeto prefetch integration.
    Build downloads dependencies from PyPI, GitHub, and Habana artifactory
    at build time — supply chain integrity not guaranteed.

  ⚠ Container listens on HTTP (plaintext) port 8000. TLS is terminated
    at the platform ingress layer, not at the container.

Container Security Context:
  - Non-root: UID 2000, GID 0
  - OpenShift arbitrary UID: Supported (GID 0 membership)
  - Group-writable dirs: /home/vllm, /usr/src, /workspace, /var/log/habana_logs
