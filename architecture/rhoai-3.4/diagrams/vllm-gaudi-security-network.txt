RBAC Summary:
  No RBAC resources defined by this component.
  RBAC is managed by the deploying platform (KServe/RHOAI operator).

Secrets:
  - (model-credentials) (Opaque) - S3/storage credentials for model download
    Provisioned by: User / RHOAI, Auto-Rotate: No
  - (huggingface-token) (Opaque) - HuggingFace Hub access token
    Provisioned by: User, Auto-Rotate: No
  Note: Secret names are user-configured at InferenceService creation time.

Authentication & Authorization:
  - /v1/* endpoints: Bearer Token / None, enforced at RHOAI Gateway (kube-rbac-proxy)
  - /health endpoint: Unauthenticated health probe, no enforcement

Container Security:
  - Base Image: registry.access.redhat.com/ubi9/ubi:9.6
  - Run As User: 2000 (vllm), Group: 0 (root, OpenShift compat)
  - FIPS: openssl-fips-provider removed (incompatible with Habana drivers)
  - Capabilities: Default (no privileged capabilities)
  - License: Apache 2.0

Security Concerns - Plaintext Communication:
  - Ray cluster coordination (6379/TCP) uses plaintext TCP
  - HCCL inter-HPU communication uses plaintext TCP/RDMA
  - NIXL KV transfer uses plaintext TCP/RDMA
  - GLOO process group uses plaintext TCP
  These are internal to the pod/node and rely on network isolation for security.
