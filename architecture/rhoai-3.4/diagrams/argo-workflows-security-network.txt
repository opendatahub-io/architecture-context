- argo-binding (cluster): argo-cluster-role -> SA: argo
- argo-binding (namespace): argo-role -> SA: argo (in argo namespace)

  - Provisioned by: DSP operator / admin | Auto-rotate: No
- argo-workflows-agent-ca-certificates (Opaque) - CA certs for agent pod TLS
  - Provisioned by: Admin / cert-manager | Auto-rotate: No
- (database credentials) (Opaque) - PostgreSQL/MySQL credentials
  - Provisioned by: Admin | Auto-rotate: No
- (artifact store credentials) (Opaque) - S3/GCS/Azure access credentials
  - Provisioned by: Admin / cloud provider | Auto-rotate: Varies
- (SSO client secret) (Opaque) - OAuth2 client secret for SSO
  - Provisioned by: Admin / IdP | Auto-rotate: No

TLS Notes:
- Metrics endpoint (9090/TCP): Self-signed ECDSA P-256, InsecureSkipVerify=true
- Min TLS version: TLS 1.2 (configurable via TLS_MIN_VERSION env)
- FIPS: workflow-controller built with strictfipsruntime; argoexec has dual binary
- SSO config supports InsecureSkipVerify for OIDC (should be disabled in production)
