Service Mesh Configuration:
===========================
- Gateway API: Primary ingress pattern (RHOAI 3.x)
  - GatewayClass: data-science-gateway
  - EnvoyFilter: ext_authz → kube-auth-proxy
  - Components: Dashboard, KServe, MLflow, Model Registry, MaaS

- Istio (Legacy / Multi-user):
  - PeerAuthentication: Namespace-wide mTLS (DSP multi-user namespaces)
  - DestinationRule: ISTIO_MUTUAL TLS (DSP ml-pipeline, ml-pipeline-ui)
  - AuthorizationPolicy: Service-level access control (DSP multi-user)

Authentication Mechanisms:
==========================
- kube-rbac-proxy (SubjectAccessReview): Dashboard, KServe, DSPO, KubeRay, MLMD, Model Registry
- OAuth2/OIDC (kube-auth-proxy): All HTTPRoute-based services via data-science-gateway
- Bearer Token (K8s SA JWT): DSP API, EvalHub, MLflow, Argo Server
- Kubernetes TokenReview: EvalHub, Dashboard backend, kube-auth-proxy
- Istio mTLS: DSP (multi-user), Model Registry (when mesh enabled)
- X-Tenant Header: Batch Gateway, EvalHub (multi-tenancy)
- S3 IAM Credentials: KServe, DSP, Argo, MLflow

FIPS Compliance:
================
- All Go-based components built with GOEXPERIMENT=strictfipsruntime
- Applies to: rhods-operator, KServe, DSPO, all Go operators, kube-auth-proxy, kube-rbac-proxy
- Non-Go components (Python/C++ runtimes): FIPS compliance varies
