RBAC Summary:
- No Kubernetes RBAC defined in this repository
- RBAC is managed by the platform operator (rhods-operator)
- Container runs with default service account unless overridden by operator

Authentication & Authorization:
- OAuth2 JWT middleware (built-in to Llama Stack server)
- Enabled only when AUTH_ISSUER environment variable is set
- JWKS-based token validation (AUTH_JWKS_URI)
- Token audience: AUTH_AUDIENCE (default: "llama-stack")
- Policy:
  - Read unowned resources: permit all (no auth required)
  - Create resources: permit authenticated users
  - Read/update/delete owned resources: permit resource owner only
  - /v1/health: always accessible (no auth)

Secrets:
- rhel-ai-private-index-auth (Opaque, build secret) - GitLab PAT for AIPCC wheel index, manual rotation
- git_auth_secret (Opaque) - Git auth for Tekton PipelineRun, Pipelines-as-Code managed

FIPS Compliance:
- Base image: AIPCC (UBI9 with FIPS-validated OpenSSL)
- Python ssl module delegates to system OpenSSL
- TLS bypass risks: VLLM_TLS_VERIFY=false, AUTH_VERIFY_TLS=false
- No explicit non-FIPS crypto imports detected
- tiktoken uses hashlib internally (FIPS-safe on UBI9)

Build Hermeticity:
- hermetic: false (Dockerfile.konflux / .tekton config)
- No lock files (no uv.lock, requirements.txt with hashes, rpms.lock.yaml)
- Dependencies pinned by version in Containerfile but not hash-locked
- Private wheel index used for RHOAI builds (GitLab PAT auth)

Security Concerns:
- Server runs on plaintext HTTP (8321/TCP) - relies on platform ingress for TLS
- OAuth2 auth is OPTIONAL (disabled when AUTH_ISSUER is not set)
- Without AUTH_ISSUER, all APIs are completely open
- PostgreSQL connection is plaintext by default (no TLS)
- VLLM_TLS_VERIFY=false option can disable TLS verification
