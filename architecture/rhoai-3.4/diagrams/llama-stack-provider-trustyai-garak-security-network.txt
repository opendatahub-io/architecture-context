================================================================================
  AUTHENTICATION & AUTHORIZATION
================================================================================

API Key Resolution (Multi-Level):
  1. Environment variable: MODEL_API_KEY, SDG_API_KEY (role-specific)
  2. Mounted Secret: Read from well-known file path (K8s Secret volume)
  3. Fallback: "DUMMY" (allows execution without Secret infrastructure)

KFP Authentication:
  1. Config-provided Bearer token
  2. Kubeconfig ServiceAccount token (fallback)

S3 Authentication (EvalHub mode):
  - AWS access key + secret key
  - Injected via kubernetes.use_secret_as_env() in KFP components

Shield Safety API:
  - API key authentication
  - Via SimpleShieldOrchestrator middleware

================================================================================
  RBAC SUMMARY
================================================================================

  No RBAC manifests defined in this repository.
  When running in KFP mode, relies on:
  - KFP pipeline runner's ServiceAccount for K8s API access
  - ConfigMap read: trustyai-service-operator-config
    (namespaces: redhat-ods-applications, opendatahub)

================================================================================
  SECRETS
================================================================================

  | Secret                | Type   | Purpose                              | Rotate |
  |-----------------------|--------|--------------------------------------|--------|
  | Model API keys        | Opaque | Auth to LLM inference endpoints      | No     |
  | KFP API token         | Opaque | Auth to Kubeflow Pipelines API       | No     |
  | S3 Data Connection    | Opaque | AWS creds for S3 (eval-hub mode)     | No     |
  | HuggingFace token     | Opaque | HF Hub model downloads               | No     |

  Provisioned By: Manual / K8s Secret mount / DSPA (S3)
  Auto-Rotate: None

================================================================================
  TLS CONFIGURATION
================================================================================

  - Configurable via `tls_verify` parameter
  - Uses httpx.Client(verify=...) supporting:
    - Boolean (True/False)
    - CA certificate path (string)
  - System trust store by default
  - Custom CA bundle path supported
  - WARNING: tls_verify can be set to False (disables verification)
  - Crypto delegated to system OpenSSL via Python ssl module (UBI9)
  - No direct use of MD5, RC4, DES, or custom RNG

================================================================================
  FIPS COMPLIANCE
================================================================================

  - Base image: UBI9 (FIPS-validated OpenSSL)
  - No Go/Rust compilation (Python project)
  - Dummy Rust project exists for Hermeto/cachi2 compliance only
  - All TLS via system OpenSSL through Python ssl module + httpx
  - No OLM FIPS annotation (not an operator)
  - No non-FIPS crypto risks detected
