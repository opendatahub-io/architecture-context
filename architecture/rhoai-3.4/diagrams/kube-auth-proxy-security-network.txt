RBAC Summary:
- (pod service account) → authentication.k8s.io/tokenreviews: create
- (pod service account) → authorization.k8s.io/subjectaccessreviews: create
- (pod service account, OpenShift) → oauth.openshift.io/oauthaccesstokens: delete
- Note: RBAC roles not defined in this repo; created by rhods-operator

Secrets:
- TLS cert/key (rbac mode) (kubernetes.io/tls) - Server TLS certificate, provisioned by platform operator, no auto-rotate (file-watched)
- Cookie secret (auth mode) (Opaque) - Session cookie encryption key, provisioned by platform operator, no auto-rotate
- Client ID/Secret (auth mode) (Opaque) - OAuth2 client credentials, provisioned by platform operator, no auto-rotate
- Redis credentials (auth mode, optional) (Opaque) - Redis session store auth, provisioned by platform operator, no auto-rotate
- Service account token (kubernetes.io/service-account-token) - K8s API auth, provisioned by Kubernetes, auto-rotate: yes

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1, -tags strictfipsruntime
- Runtime: Go crypto/tls delegates to OpenSSL for FIPS-approved algorithms
- Risk: SHA1 used in htpasswd validation (#nosec G401/G505 acknowledged)
- Risk: InsecureSkipVerify available for upstream TLS and Redis TLS

Authentication Chain (auth mode, priority order):
1. K8s Service Account Token (TokenReview API)
2. JWT Bearer Token (OIDC verification)
3. OpenShift OAuth Bearer Token (sha256~ format)
4. HTTP Basic Auth (htpasswd file)
5. Stored Cookie Session (with refresh + Redis distributed locking)

Authorization Chain (rbac mode, priority order):
1. Hardcoded metrics exemption (prometheus-k8s SA)
2. Static rules from config file
3. Kubernetes SubjectAccessReview (allow cache 5min, deny cache 30s)
