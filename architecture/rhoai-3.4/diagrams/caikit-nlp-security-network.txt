RBAC Summary:
- No Kubernetes RBAC defined by caikit-nlp
- RBAC is managed by the model serving platform (KServe/ModelMesh)
- Pod runs under the ServiceAccount provided by the deployment platform

Secrets:
- TGIS TLS certs (TLS certificates) - mTLS with TGIS backend
  - ca_cert_file: CA certificate for TGIS server verification
  - client_cert_file: Client certificate for mTLS
  - client_key_file: Client private key for mTLS
  - Provisioned by: Manual / platform config
  - Auto-Rotate: No

Authentication & Authorization:
- HTTP API (8080): No auth implemented by caikit-nlp
- gRPC API (8085): No auth implemented by caikit-nlp
- Auth enforcement delegated to model serving platform:
  - KServe: kube-rbac-proxy sidecar
  - ModelMesh: Platform-level auth
- TGIS Backend: Optional mTLS (configurable per connection)

Security Controls:
- Model downloads disabled by default (allow_downloads: false)
- trust_remote_code defaults to false for sentence-transformers
- Path validation in PEFT config prevents directory traversal
- No direct crypto imports - relies on system OpenSSL (UBI 9)
- FIPS compliance inherited from UBI 9 base image

- WARNING: No lock files found (no rpms.lock.yaml, no pip lock file)
- pip install without --require-hashes
- No Hermeto/cachi2 prefetch integration
- Dependencies resolved at build time from PyPI (non-hermetic)
