Authentication Mechanisms:

  1. External API Access:
     - HTTP Header (user ID) — when behind kube-rbac-proxy
     - Bearer Token — validated via Kubernetes TokenReview API
     - SubjectAccessReview — enforces namespace isolation in multi-user mode

  2. Internal Service-to-Service:
     - Projected ServiceAccount Tokens
       - Audience: pipelines.kubeflow.org
       - Expiry: 1 hour (auto-rotated by kubelet)
       - Used by: Persistence Agent, Scheduled Workflow Controller

  3. Webhook:
     - TLS certificate (kubernetes.io/tls)
     - Provisioned by: cert-manager or manual
     - Port: 8443/TCP (HTTPS required)
