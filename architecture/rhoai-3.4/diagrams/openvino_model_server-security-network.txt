================================================================================
  RBAC SUMMARY
================================================================================

OVMS does not define Kubernetes RBAC resources. It runs as a user-space
process within a pod. RBAC for the pod is managed by KServe and the RHOAI
operator.

================================================================================
  FIPS / CRYPTO NOTES
================================================================================

- OVMS is a C++ binary (not Go) — GOEXPERIMENT=strictfipsruntime does not apply
- Dynamically links against system OpenSSL from UBI9 base image
- OpenSSL MD5 used for non-security config change detection (not crypto ops)
- No application-level TLS — gRPC: InsecureServerCredentials(), REST: plaintext
- Certificate handling: system CA bundle (/etc/ssl/certs/ca-bundle.crt)
- GIT_SSL_CERT_LOCATIONS configurable for libgit2 SSL cert location
- Build is NOT hermetic (hermetic: false in Tekton PipelineRun)

================================================================================
  NETWORK POLICY RECOMMENDATIONS
================================================================================

Ingress:
  - Allow: kube-rbac-proxy → OVMS REST port (localhost within pod)
  - Allow: Prometheus → OVMS REST port (metrics scraping)
  - Deny: All direct external access to OVMS ports

Egress:
  - Allow: OVMS → S3 (443/TCP) if S3 storage backend used
  - Allow: OVMS → GCS (443/TCP) if GCS storage backend used
  - Allow: OVMS → Azure Blob (443/TCP) if Azure storage backend used
  - Allow: OVMS → HuggingFace Hub (443/TCP) if HF model pull used
  - Deny: All other egress (OVMS needs no other outbound connections)
