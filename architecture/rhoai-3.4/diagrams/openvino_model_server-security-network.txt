RBAC Summary:
- No RBAC resources defined
- OVMS runs as non-privileged container (UID 5000, GID 5000)
- No Kubernetes API interactions from the container
- Container does not require elevated privileges

Authentication Summary:
- /v3/* endpoints: Optional API Key (Bearer Token via api_key_file or API_KEY env)
  - Enforcement: OVMS HttpRestApiHandler
  - Policy: Rejects request if key is configured but not provided
- /v1/*, /v2/* endpoints: Unauthenticated (open access)
- /metrics endpoint: Unauthenticated (open access)
- All gRPC services: Unauthenticated (InsecureServerCredentials)
  - No TLS support on gRPC by default

Secrets:
- API_KEY (env var) - API key for v3 endpoints, User/Operator provisioned, No auto-rotation
- HF_TOKEN (env var) - HuggingFace Hub auth, User/Operator provisioned, No auto-rotation
- AWS credentials (env vars) - S3 storage auth, User/Operator provisioned, No auto-rotation
- Azure credentials (env vars) - Azure Blob auth, User/Operator provisioned, No auto-rotation
- GCS credentials (file) - GCS auth (service account JSON), User/Operator provisioned, No auto-rotation

Security Notes:
- OVMS does NOT support TLS natively on its listener ports
- TLS termination relies on KServe Gateway / Service Mesh sidecar
- gRPC uses InsecureServerCredentials (no encryption, no auth)
- API Key authentication only available on v3 (generative) endpoints
- Container runs as non-root (UID/GID 5000)
