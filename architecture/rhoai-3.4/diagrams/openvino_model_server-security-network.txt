SECURITY NOTES:
══════════════

No Built-in TLS:
- gRPC server uses InsecureServerCredentials() (src/grpcservermodule.cpp:138)
- REST server uses plaintext HTTP (Drogon)
- TLS termination is ENTIRELY delegated to orchestration layer:
  * Istio sidecar (RHOAI default)
  * Optional NGINX mTLS proxy (extras/nginx-mtls-auth/)
  * KServe gateway

API Authentication:
- REST endpoints: Optional API key via --api_key_file or API_KEY env var
- gRPC endpoints: No built-in authentication
- Health endpoints (/v2/health/*): Always accessible, no auth
- Metrics endpoint (/metrics): Always accessible, no auth

Container Security:
- Runs as non-root user: ovms (UID 5000)
- Base image: ubi9-minimal
- Build flags: Stack protector, FORTIFY_SOURCE, format security
- LTO optimization with -march=haswell

RBAC Summary:
═════════════
N/A - OVMS is a service binary, not a Kubernetes operator.
RBAC is configured by the deploying operator (KServe, rhods-operator).

Secrets:
════════
- API key file (Opaque, file mount) - REST API auth via --api_key_file - No auto-rotate
- AWS credentials (env vars) - S3 access (ACCESS_KEY_ID, SECRET_ACCESS_KEY, SESSION_TOKEN) - No auto-rotate
- Azure credentials (env vars) - Azure Blob Storage access - No auto-rotate
- GCS credentials (env vars) - Google Cloud Storage access - No auto-rotate
- HuggingFace token (env var) - HF Hub model download - No auto-rotate

Crypto Notes:
═════════════
- BoringSSL linked via Bazel (filesystem checksums only)
- MD5 used for file integrity checks (deprecated, pragma-suppressed)
- No FIPS-validated crypto in-process (delegated to platform TLS)
- Optional NGINX mTLS proxy uses OpenSSL with SHA512 for cert generation

Build Hermeticity:
══════════════════
- Tekton pipeline: hermetic: false (full network access during build)
- ~50+ external dependencies downloaded at build time
- No RPM lock files, no Hermeto prefetch
- Only yarn.lock (minor JS deps) and go.sum (client library) present
