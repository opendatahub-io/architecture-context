RBAC Summary:
- ClusterRole: eval-hub-service-full-access (example)
  - API Group: trustyai.opendatahub.io
  - Resources: evaluations, collections, providers
  - Verbs: get, list, create, update, patch, delete
  - API Group: trustyai.opendatahub.io
  - Resources: status-events
  - Verbs: create

Authorization Model:
- Authentication: Bearer Token → Kubernetes TokenReview API
- Authorization: SubjectAccessReview with:
  - Resource: mapped from REST endpoint (evaluations, collections, providers, status-events)
  - Verb: mapped from HTTP method (GET→get, POST→create, PUT→update, DELETE→delete, PATCH→patch)
  - Namespace: from X-Tenant request header
  - API Group: trustyai.opendatahub.io (evaluations/collections/providers)
                mlflow.kubeflow.org (experiments - for job creation)
- Caching: Allow decisions cached 5 minutes, deny cached 30 seconds

Secrets:
- Per-job OCI credentials (kubernetes.io/dockerconfigjson) - OCI registry auth - User/Operator provisioned - No auto-rotate
- Per-job model auth (Opaque) - Model endpoint auth - User provisioned - No auto-rotate
- Per-job test-data S3 (Opaque) - AWS IAM for S3 download - User/Operator provisioned - No auto-rotate
