RBAC Summary:
- RBAC is provisioned by TrustyAI Service Operator (not bundled in eval-hub repo)
- Required ServiceAccount permissions:
  - API Group: batch, Resources: jobs, Verbs: create, delete, list, watch
  - API Group: core, Resources: configmaps, Verbs: create, delete, get
  - API Group: authentication.k8s.io, Resources: tokenreviews, Verbs: create
  - API Group: authorization.k8s.io, Resources: subjectaccessreviews, Verbs: create

Authorization Model:
- HTTP endpoints mapped to K8s resource attributes via config/auth.yaml
- Custom API groups: trustyai.opendatahub.io, mlflow.kubeflow.org
- X-Tenant header determines namespace for SAR checks
- Standard K8s Role/RoleBinding resources for RBAC policies

Secrets:
- evalhub-service-ca (ConfigMap, service-ca.crt) - OpenShift Service CA, auto-rotate
- mlflow projected SA token (ProjectedServiceAccountToken) - MLflow auth, 3600s expiry, auto-rotate
- test-data S3 credentials (Opaque) - AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, no auto-rotate
- OCI registry credentials (kubernetes.io/dockerconfigjson) - OCI registry auth, no auto-rotate
- model auth credentials (Opaque) - Model endpoint auth, no auto-rotate
- database password (Secret file mapping) - PostgreSQL password, no auto-rotate

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1 (Dockerfile.konflux only)
- TLS: MinVersion=1.2, MaxVersion=1.3 (all HTTP clients and server)
- Runtime: Reads /proc/sys/crypto/fips_enabled; enforces InsecureSkipVerify=false when FIPS=1
- No non-FIPS crypto detected (no crypto/md5, crypto/rc4, crypto/des)
