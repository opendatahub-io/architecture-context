TLS Configuration:
  - EPP ext-proc (9002/TCP): TLS enabled by default (--secure-serving=true)
    - Self-signed RSA 4096-bit certs (default) or user-provided certs
    - NextProtos: ["h2"] (HTTP/2 for gRPC)
    - fsnotify-based certificate hot reload (--enable-cert-reload)
  - BBR ext-proc (9004/TCP): TLS enabled by default (--secure-serving=true)
    - Same TLS configuration as EPP
  - Health ports (9003, 9005): No TLS (plaintext gRPC)
  - Metrics ports (9090): No TLS (plaintext HTTP)
  - Model server scraping: InsecureSkipVerify configurable (default true)

Secrets:
  - TLS cert (optional): kubernetes.io/tls
    - Purpose: gRPC server TLS certificate for ext-proc endpoints
    - Provisioned by: Manual or cert-manager
    - Auto-rotate: Yes (via fsnotify cert reload)

FIPS Compliance:
  - Status: NOT FIPS COMPLIANT (upstream)
  - CGO_ENABLED=0 (static linking, no FIPS crypto)
  - Base image: distroless/static:nonroot (no OpenSSL)
  - No GOEXPERIMENT=strictfipsruntime
  - RHOAI downstream would need Konflux Dockerfiles with UBI base + CGO_ENABLED=1

Security Risks:
  - InsecureSkipVerify=true default for model server metrics scraping
  - No client certificate validation on ext-proc endpoints (server TLS only)
  - Health and metrics ports exposed without encryption
  - Latency predictor communication over plaintext HTTP
  - No explicit cipher suite restrictions in TLS configuration
