Authentication & Authorization:
- Online/Registry serving: Kubernetes RBAC (SubjectAccessReview) OR OIDC (Bearer token JWT validation)
- Operator → Registry: Intra-communication JWT (HS256) with ":::" prefix to bypass per-user auth
- Health probes: Unauthenticated (8081/TCP)
- Metrics: Unauthenticated (8000/TCP, Prometheus scrape)

TLS Configuration:
- Service serving certificates: Auto-provisioned by OpenShift Service CA
  - Annotation: service.beta.openshift.io/serving-cert-secret-name
  - Secret type: kubernetes.io/tls
  - Auto-rotated by OpenShift
- Custom CA bundles: Mounted from ConfigMaps at /etc/pki/tls/custom-certs/
- OIDC CA certs: Mounted at /etc/pki/tls/oidc-ca/
- ODH CA bundle: Auto-detected from odh-trusted-ca-bundle ConfigMap

Secrets:
- feast-{name}-{service}-tls (kubernetes.io/tls) - Service TLS cert/key, OpenShift Service CA, auto-rotated
- (user-provided, Opaque) - Database credentials for persistence backends, manual management
- (user-provided, Opaque) - OIDC client credentials (client_id, client_secret, auth_discovery_url), manual
- (user-provided, Opaque) - Git authentication tokens for project directory cloning, manual

FIPS Compliance:
- Operator (Go): CGO_ENABLED=1 GOEXPERIMENT=strictfipsruntime -tags strictfipsruntime
- Feature Server (Python): Relies on UBI9 base image system OpenSSL
- Risk: InsecureSkipVerify=true used for intra-cluster registry REST calls
- Risk: Go Redis client uses default tls.Config{} without explicit cipher suites
