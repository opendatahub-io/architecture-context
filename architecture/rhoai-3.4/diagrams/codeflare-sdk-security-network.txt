Secrets:
- {cluster-name}-ca-secret-{suffix} (Opaque) - CA cert+key for mTLS, provisioned by KubeRay Operator, no auto-rotate
- {job-name}-files (Opaque) - Local files for RayJob submission, provisioned by CodeFlare SDK, no auto-rotate

Authentication Methods:
- Kubernetes API: Bearer Token (JWT) / kubeconfig / in-cluster SA token via kube-authkit
- Ray Dashboard (RHOAI 3.x): Bearer Token forwarded via kube-rbac-proxy / HTTPRoute
- Ray Dashboard (legacy): OAuth proxy sidecar (OpenShift OAuth)
- Ray Client (mTLS): Client TLS certificate (RSA 3072-bit, SHA-256, signed by cluster CA)

FIPS Notes:
- SDK is a Python library, not a compiled binary
- Uses cryptography 46.0.7 with OpenSSL backend (FIPS-compatible when OpenSSL FIPS-enabled)
- RSA 3072-bit keys and SHA-256 signatures are FIPS-approved algorithms
- verify_tls flag can disable TLS verification (user opt-in only)

ODH Trusted CA Bundle:
- ConfigMap: odh-trusted-ca-bundle (mounted as optional volume)
- Paths: /etc/pki/tls/certs/, /etc/ssl/certs/ (in Ray pods)
- Custom CA: CF_SDK_CA_CERT_PATH env or /etc/pki/tls/custom-certs/ca-bundle.crt
