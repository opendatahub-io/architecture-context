  - MLflow server: Kubernetes SelfSubjectAccessReview against pseudo-resources
    (experiments, datasets, registeredmodels) in mlflow.kubeflow.org API group.
    Enforced by the kubernetes-auth app plugin.
  - Operator metrics: Bearer token validated via TokenReview + SubjectAccessReview.
    Only accounts with metrics-reader ClusterRole can access.
  - MLflow /health endpoint: Unrestricted (health check only).
