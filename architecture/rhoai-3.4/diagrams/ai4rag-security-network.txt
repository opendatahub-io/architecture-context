Secrets / Credentials:
- API_KEY (Environment variable) — Llama Stack server authentication
  - Provisioned by: User (.env file)
  - Auto-Rotate: No
  - Used by: LlamaStackClient (Bearer token in HTTP Authorization header)
- BASE_URL (Environment variable) — Llama Stack server endpoint
  - Provisioned by: User (.env file)
  - Auto-Rotate: No

RBAC Summary:
- None — ai4rag is a Python library with no Kubernetes RBAC requirements.
- No ClusterRoles, Roles, or RoleBindings.
- No ServiceAccounts.

Service Mesh / Network Policies:
- N/A — ai4rag is a library, not a deployed service.
- No PeerAuthentication, AuthorizationPolicy, or NetworkPolicy resources.
- When deployed within a pod (e.g., as part of a KFP pipeline step),
  it inherits the pod's network policy and service mesh configuration.

TLS / Crypto Notes:
- No custom TLS configuration — relies on httpx/urllib3 defaults.
- TLS version determined by Python runtime's OpenSSL configuration.
- Certificate verification uses system trust store (default httpx behavior).
- hashlib.sha256 used for document deduplication (not security-critical).
- No MD5, RC4, DES, or other non-FIPS crypto detected.
- FIPS compliance inherited from host Python/OpenSSL runtime.

Threading:
- ThreadPoolExecutor used for parallel RAG queries (max_threads=10).
- No explicit rate limiting or backoff for external API calls.
- Failed iterations return None score without retry.
