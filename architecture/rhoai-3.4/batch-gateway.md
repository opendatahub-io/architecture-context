## Purpose
**Short**: OpenAI-compatible batch inference API gateway for processing bulk LLM inference requests via the `/v1/files` and `/v1/batches` endpoints.

**Detailed**: The batch-gateway is an llm-d incubation project that provides an OpenAI-compatible implementation of the Batch API (`/v1/batches`) and Files API (`/v1/files`). It enables users to submit large batches of inference requests (up to 50,000 per file, 200 MB max) as JSONL files, which are then asynchronously processed by a pool of worker goroutines against a downstream inference gateway (llm-d or other OpenAI-compatible endpoint).

The system consists of two independently deployable components: an **API Server** that exposes the REST API for managing files and batch jobs, and a **Batch Processor** that polls a priority queue for pending jobs, dispatches them to a configurable worker pool, and forwards individual requests to a downstream inference endpoint. Both components share a common database layer (with Redis and PostgreSQL backends) and file storage layer (with S3 and local filesystem backends). Multi-tenancy is supported via a configurable HTTP header (`X-MaaS-User` by default) for tenant isolation.

The component is in active early-stage development with several TODO items indicating incomplete functionality (e.g., file validation in the processor, response writing to output files, event handling for cancel/pause/resume). The API Server currently uses mock database and file store implementations in its handler wiring.
