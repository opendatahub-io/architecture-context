## Purpose
**Short**: CPU-optimized build of vLLM providing high-throughput LLM inference and serving without GPU hardware requirements.

**Detailed**: vLLM CPU is a Red Hat downstream fork of the open-source vLLM project, specifically built and optimized for CPU-only inference workloads. It provides an OpenAI-compatible REST API server (and a gRPC server) for serving large language models on CPU hardware across multiple architectures including x86_64, ARM (aarch64), IBM Power (ppc64le), and IBM Z (s390x).

The component is packaged as a container image (`odh-vllm-cpu-rhel9`) built on UBI 9 via Konflux pipelines. It supports key vLLM features including continuous batching, PagedAttention for efficient memory management, multi-LoRA support, structured output generation, streaming responses, and embedding/pooling tasks. The image is designed to run as a serving runtime within the RHOAI platform, typically managed by KServe as an InferenceService.

The build process uses architecture-specific build scripts (`build_vllm_s390x.sh`, `build_vllm_ppc64le.sh`) that compile platform-specific dependencies from source, including OpenBLAS, numactl, and LLVM, to ensure optimal performance on each target CPU architecture.
