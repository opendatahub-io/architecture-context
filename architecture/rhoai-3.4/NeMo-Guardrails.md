## Purpose
**Short**: NeMo Guardrails is an open-source toolkit for adding programmable guardrails to LLM-based conversational systems, providing input/output content safety, jailbreak detection, topic safety, and sensitive data detection.

**Detailed**: NeMo Guardrails enables developers building LLM-based applications to add programmable guardrails between application code and LLMs. It provides an OpenAI-compatible API server that intercepts chat completion requests, applies configurable safety rails (input rails, output rails, dialog rails), and enforces content policies before responses reach users. The toolkit supports multiple guardrail types including content safety checks, jailbreak detection (both model-based and heuristic), topic safety enforcement, sensitive data detection (PII), hallucination detection, and integration with third-party safety services.

The RHOAI distribution builds this as a container image (`odh-nemo-guardrails-rhel9`) based on UBI9/Python 3.12 using the Konflux build system. The build uses the "opensource" profile which filters out closed-source guardrail providers (ActiveFence, Cleanlab, Clavata, PrivateAI, Fiddler, PatronusAI, Prompt Security, GCP Moderate Text) and pre-downloads required NLP models (spaCy, sentence-transformers, FastEmbed, NLTK) into the container image for disconnected/air-gapped operation. The container exposes an HTTP API on port 8000 that is OpenAI-compatible and can proxy to upstream LLM inference endpoints.
