## Purpose
**Short**: A centralized library of reusable Kubeflow Pipelines (KFP) components and pipelines for AI/ML workflows including LLM fine-tuning, AutoML tabular training, evaluation, and model registration.

**Detailed**: The pipelines-components repository provides a structured collection of reusable Kubeflow Pipelines SDK v2 components and pre-built pipelines for Red Hat OpenShift AI (RHOAI). Components are organized into four categories: data processing, training, evaluation, and deployment. Each component is defined as a Python function decorated with `@dsl.component`, specifying a base container image and runtime dependencies.

The repository's primary value is in providing end-to-end ML pipelines for LLM fine-tuning workflows. It supports three fine-tuning algorithms (OSFT, SFT, LoRA) orchestrated through 4-stage pipelines: dataset download, distributed training via Kubeflow Trainer, model evaluation with lm-eval harness, and model registration to Kubeflow Model Registry. Additionally, it includes an AutoML pipeline for tabular data using AutoGluon. Components run as containerized tasks within Kubeflow Pipelines on Kubernetes, using PVC-based shared storage, Kubernetes Secrets for credentials, and GPU scheduling for training and evaluation workloads.

The Konflux build system produces a single shipped container image (`odh-automl-rhel9`) containing the AutoML runtime with AutoGluon, PyTorch (CPU), and related ML libraries. Other components use pre-built base images from the ODH training hub or Red Hat registries at pipeline runtime.
