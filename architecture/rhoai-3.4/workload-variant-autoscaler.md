## Purpose
**Short**: A Kubernetes-based global autoscaler for inference model servers serving LLMs, using saturation-based scaling with KV cache utilization and queue depth metrics.

**Detailed**: The Workload Variant Autoscaler (WVA) is a Kubernetes controller that performs intelligent autoscaling for LLM inference workloads. It works alongside standard Kubernetes HPA and external autoscalers like KEDA to optimize replica counts for inference model servers by analyzing saturation metrics (KV cache utilization and queue depth) from Prometheus.

A "variant" in WVA represents a specific way of serving a given model: a scale target (Deployment, StatefulSet, or LeaderWorkerSet) with a particular combination of hardware (e.g., A100, H100, L4 accelerators), runtimes, and serving approach. When multiple variants serve the same model, WVA intelligently chooses which variant to scale, for example adding capacity on the cheapest variant and removing it from the most expensive.

WVA consists of a reconciler (Kubernetes controller managing VariantAutoscaling CRs), a collector (gathering metrics from Prometheus), a saturation engine (determining optimal replicas based on KV cache and queue depth thresholds), a scale-from-zero engine, and an actuator (emitting optimization metrics back to Prometheus for HPA/KEDA consumption). The controller also watches InferencePool resources from the Gateway API Inference Extension for endpoint-aware scaling.

| Metrics Emitter | Go Library | Emits custom Prometheus metrics (desired/current replicas, scaling events, desired ratio) |
| Datastore | Go In-Memory Store | Tracks namespace-to-VA mappings and InferencePool endpoint data for controllers and engines |
