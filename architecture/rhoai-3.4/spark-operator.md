## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Apache Spark applications on OpenShift/Kubernetes clusters.

**Detailed**: The Spark Operator (originally from the Kubeflow project) automates the submission, scheduling, monitoring, and lifecycle management of Apache Spark applications on Kubernetes. It provides three Custom Resource Definitions (CRDs): `SparkApplication` for one-time Spark jobs, `ScheduledSparkApplication` for cron-scheduled recurring Spark jobs, and `SparkConnect` for persistent Spark Connect server sessions.

The operator consists of two deployment components: a **controller** that watches SparkApplication CRDs and manages driver/executor pods, services, and ingresses; and a **webhook** that validates and mutates SparkApplication resources and Spark pods before they are created. The webhook injects Spark configuration, volumes, and environment variables into driver and executor pods. Both components expose Prometheus metrics for monitoring.

In the RHOAI distribution, the operator is deployed into the `redhat-ods-applications` namespace by the rhods-operator using kustomize overlays. The operator manages Spark workloads across namespaces, creating driver pods that in turn spawn executor pods for distributed data processing.
