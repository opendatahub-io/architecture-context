## Purpose
**Short**: Kubernetes operator that manages the lifecycle of MLflow tracking server instances on OpenShift, providing experiment tracking, model registry, and AI Gateway capabilities.

**Detailed**: The MLflow Operator is a controller-runtime based Go operator that reconciles cluster-scoped `MLflow` Custom Resources to deploy and manage MLflow tracking server instances. It uses an embedded Helm chart (`charts/mlflow`) as a templating engine to render Kubernetes resources (Deployment, Service, NetworkPolicy, PVC, ServiceAccount, ClusterRole/ClusterRoleBinding, ServiceMonitor) based on the MLflow CR spec, then applies them via Server-Side Apply.

The operator dynamically creates Gateway API `HTTPRoute` resources to expose MLflow through the RHOAI data-science-gateway (RHOAI 3.x ingress pattern), and OpenShift `ConsoleLink` resources for integration with the OpenShift web console. It supports multi-instance deployments with unique resource naming, workspace-aware artifact storage via namespace-scoped `MLflowConfig` CRs, and comprehensive CA bundle management through init containers and sidecar watchers.

MLflow itself serves over HTTPS (TLS terminated directly by uvicorn) on port 8443, uses Kubernetes-native RBAC authorization via `SelfSubjectAccessReview`, and supports configurable backend stores (PostgreSQL, MySQL, SQLite) and artifact destinations (S3, GCS, local file storage).

### Ingress

| Name | Type | Hosts | Port | Protocol | Encryption | TLS Mode | Exposure |
|------|------|-------|------|----------|------------|----------|----------|
| mlflow | HTTPRoute (Gateway API) | Via data-science-gateway | 8443/TCP | HTTPS | TLS (end-to-end) | Passthrough to backend | External (via gateway) |

The operator dynamically creates `HTTPRoute` resources (Gateway API v1) when the HTTPRoute CRD is detected in the cluster. The HTTPRoute references the `data-science-gateway` Gateway in the `openshift-ingress` namespace as its parent. Three route rules are created per MLflow instance:
1. `/mlflow/api/*` - rewrites to `/api/*` on the backend service (port 8443)
2. `/mlflow/v1/*` - rewrites to `/v1/*` on the backend service (port 8443)
3. `/mlflow/*` - passes through to the backend service (port 8443)
