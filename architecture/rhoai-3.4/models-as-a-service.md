## Purpose
**Short**: A platform for providing Models as a Service with API key management, subscription-based access control, and token rate limiting for LLM inference endpoints on OpenShift.

**Detailed**: Models as a Service (MaaS) provides a comprehensive platform for exposing LLM inference endpoints as managed services with policy-driven access control. The system consists of two main components: the **maas-controller** (a Kubernetes operator using controller-runtime) that reconciles custom resources (MaaSModelRef, MaaSAuthPolicy, MaaSSubscription) into Kuadrant AuthPolicy and TokenRateLimitPolicy resources targeting Gateway API HTTPRoutes, and the **maas-api** (a Go REST API built with Gin) that handles API key lifecycle management (create, validate, revoke, search) backed by PostgreSQL, model discovery via MaaSModelRef CRs, and subscription selection for the Authorino auth policy evaluation chain.

The system integrates deeply with the Kuadrant/Authorino ecosystem for authentication and authorization. When a user creates a MaaSAuthPolicy, the controller generates a Kuadrant AuthPolicy targeting the model's HTTPRoute. This AuthPolicy configures Authorino to validate API keys via an HTTP callback to maas-api, check group membership, and inject identity headers. MaaSSubscription resources are similarly translated into Kuadrant TokenRateLimitPolicy resources that enforce per-user, per-model token rate limits with subscription tier selection logic.

**Gateway API Architecture (RHOAI 3.x)**:
- The `maas-default-gateway` Gateway resource (gatewayClassName: `openshift-default`) is deployed in `openshift-ingress` namespace
- Model HTTPRoutes (created by KServe for LLMInferenceService) reference this gateway as parent
- The `maas-api-route` HTTPRoute exposes maas-api endpoints at `/v1/models` and `/maas-api/*` through the same gateway
- Kuadrant AuthPolicy and TokenRateLimitPolicy resources target individual model HTTPRoutes for per-model access control
- Gateway-level default policies (gateway-default-auth, gateway-default-deny) act as catch-all deny for unconfigured models
- Authorino calls back to maas-api at `https://maas-api.<namespace>.svc.cluster.local:8443/internal/v1/api-keys/validate` for API key validation
