## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Model Registry instances for ML model metadata storage and retrieval in RHOAI.

**Detailed**: The Model Registry Operator is a controller-runtime based Go operator that manages `ModelRegistry` custom resources. When a user creates a ModelRegistry CR, the operator provisions a complete model registry stack including a REST API service (model-registry), a gRPC metadata service (ml-metadata/MLMD), an optional OAuth proxy sidecar for authentication, OpenShift Routes for external access, RBAC resources for user access control, and NetworkPolicies for network isolation.

The operator also manages a Model Catalog feature through a dedicated `ModelCatalogReconciler` controller. The catalog provides a centralized collection of model metadata sourced from configurable catalog sources, served via a separate REST API with its own OAuth proxy and Route infrastructure.

The operator supports both PostgreSQL and MySQL as backend databases for ML Metadata storage, with configurable SSL/TLS connections. It handles CRD API version migration (v1alpha1 to v1beta1) via storage version migration strategies. The operator is deployed by the rhods-operator using kustomize overlays, with image references and configuration injected at deploy time via params.env.
