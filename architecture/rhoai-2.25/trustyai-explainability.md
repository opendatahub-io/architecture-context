## Purpose
**Short**: TrustyAI is a Java library and Quarkus-based REST service providing fairness metrics, model explainability (XAI), and data drift detection for ML models served by ModelMesh and KServe on OpenShift AI.

**Detailed**: TrustyAI Explainability is the core engine behind the TrustyAI service in Red Hat OpenShift AI. It provides a comprehensive set of AI fairness and transparency capabilities including group fairness metrics (Statistical Parity Difference, Disparate Impact Ratio), data drift detection (KS Test, Approximate KS Test, Fourier MMD, Meanshift), and local/global explainability algorithms (LIME, SHAP, Counterfactual, Partial Dependence Plots, Aggregated LIME, Time-Series Saliency).

The service integrates with ML model serving platforms (ModelMesh and KServe) to consume inference payloads, store them persistently (via PVC flat files, MinIO/S3, or MariaDB/MySQL databases via Hibernate ORM), and compute scheduled fairness/drift metrics that are exposed as Prometheus metrics for monitoring. It supports both direct REST API consumption of inference payloads (for ModelMesh) and CloudEvent-based consumption (for KServe via Knative eventing). The explainability module connects to model serving endpoints via gRPC (KServe v2 protocol) to generate local and global explanations of model predictions.

In RHOAI, the service runs as a per-namespace Quarkus deployment managed by the TrustyAI Service Operator. Certain endpoints are feature-flagged by profile: RHOAI disables local explainers (SHAP, LIME, Counterfactual, TSSaliency) and global explainers, while enabling fairness and drift metrics.
