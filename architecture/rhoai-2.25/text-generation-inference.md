## Purpose
**Short**: A high-performance text generation inference server that provides gRPC and HTTP APIs for serving large language models (LLMs) with continuous batching, tensor parallelism, and flash attention support.

**Detailed**: Text Generation Inference Server (TGIS) is an early fork of Hugging Face's text-generation-inference, developed and maintained by IBM. It provides an optimized serving runtime for large language models (LLMs) including both causal (decoder-only) and sequence-to-sequence (encoder-decoder) architectures. The server supports features such as flash attention v2, tensor parallelism via sharding across multiple GPUs, GPTQ quantization, bitsandbytes quantization, ONNX runtime, and PyTorch 2.x compilation.

The architecture follows a multi-process design: a Rust **launcher** process manages one or more Python **model shard** processes (connected via Unix domain sockets) and a Rust **router** process. The router exposes a gRPC API (port 8033) implementing the `fmaas.GenerationService` interface for text generation, streaming generation, tokenization, and model info. It also exposes an HTTP API (port 3000) for health checks and Prometheus metrics. The router handles request validation, batching, and routing to model shards for inference.

Within the RHOAI ecosystem, TGIS is deployed as a model serving runtime, typically managed by KServe or ModelMesh. It serves as the inference backend for deployed LLM models, consuming GPU resources and providing gRPC-based inference to upstream components like caikit-tgis-serving.

## Supported Models

TGIS includes deployment examples for the following model families:

- BLOOM (bloom, bloomz, bloomchat-v1)
- FLAN-T5 (flan-t5, flan-t5-3b)
- FLAN-UL2 (flan-ul2, flan-ul2-tp)
- GPT-NeoX
- MPT-7B (mpt-7b, ibm-mpt-7b-instruct)
- MT0
- UL2

The server supports any HuggingFace-compatible causal LM or seq2seq LM via AutoModel, with specialized support for flash attention and custom kernels.
