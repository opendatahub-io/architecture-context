## Purpose
**Short**: Argo Workflows is a workflow execution engine for Kubernetes that orchestrates parallel jobs as DAGs, used by Data Science Pipelines to execute ML pipeline steps.

**Detailed**: Argo Workflows provides the workflow orchestration layer for Red Hat OpenShift AI Data Science Pipelines (DSP). It defines and executes complex multi-step workflows on Kubernetes by representing each step as a Pod. The Workflow Controller watches Workflow custom resources, creates and manages Pods for each step, handles DAG/step dependencies, manages artifact passing between steps, and tracks workflow state. The repository produces two container images for RHOAI: the Workflow Controller (which reconciles Workflow CRs and manages Pod lifecycle) and the Argo Executor (argoexec, which runs as an init/sidecar container in workflow pods to handle artifact collection, script execution, and resource management). In the RHOAI context, this component is not deployed directly by users but is managed by the Data Science Pipelines Operator (DSPO) which deploys and configures both the workflow controller and executor as part of the DSP stack.

Note: The Argo Server (API server with UI) is also defined in this codebase but is **not** shipped as a separate RHOAI container image. In RHOAI, the Data Science Pipelines API server handles API interactions instead.
