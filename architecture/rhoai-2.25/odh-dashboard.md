## Purpose
**Short**: Red Hat OpenShift AI web dashboard providing a unified management interface for data science projects, notebooks, model serving, pipelines, and platform configuration.

**Detailed**: The odh-dashboard is the primary web user interface for Red Hat OpenShift AI (RHOAI). It provides a React-based single-page application frontend with a Fastify-based Node.js backend that acts as a Backend-for-Frontend (BFF) proxy to the Kubernetes API server. The dashboard allows data scientists and administrators to manage data science projects, Jupyter notebooks/workbenches, model serving configurations (KServe and ModelMesh), data science pipelines, model registries, accelerator/hardware profiles, and platform-wide settings.

The application is deployed as a multi-container pod consisting of three containers: the main dashboard application (Node.js serving both the React SPA and API routes on port 8080), an OAuth proxy sidecar (OpenShift oauth-proxy on port 8443 providing authentication via OpenShift OAuth), and a Model Registry UI sidecar (Go-based BFF serving the model registry microfrontend on port 8043 via Module Federation). The backend proxies authenticated user requests to the Kubernetes API using the user's forwarded access token, ensuring that all operations respect the user's RBAC permissions. In RHOAI deployments, the dashboard is renamed to `rhods-dashboard` and deployed in the `redhat-ods-applications` namespace.
