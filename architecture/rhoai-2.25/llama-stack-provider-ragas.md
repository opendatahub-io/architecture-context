## Purpose
**Short**: An out-of-tree Llama Stack evaluation provider that integrates the Ragas framework for LLM evaluation, supporting both inline (in-process) and remote (Kubeflow Pipelines) execution modes.

**Detailed**: This component implements the [Ragas](https://github.com/explodinggradients/ragas) evaluation framework as a Llama Stack provider, enabling standardized LLM evaluation through the Llama Stack Eval API. It provides metrics such as answer relevancy, context precision, faithfulness, and context recall for Retrieval-Augmented Generation (RAG) pipeline quality assessment.

The provider operates in two modes: an **inline provider** that runs Ragas evaluation within the Llama Stack server process, using the server's own inference and embedding APIs directly; and a **remote provider** that offloads evaluation to Kubeflow Pipelines (Data Science Pipelines), creating multi-step pipeline runs that retrieve datasets from Llama Stack, execute Ragas evaluation, and store results in S3-compatible storage. The remote mode uses the `llama-stack-client` SDK to communicate with the Llama Stack server from within KFP pipeline pods.

The container image produced by Konflux (`Dockerfile.konflux`) packages the provider with the `[remote]` optional dependencies (KFP SDK, Kubernetes client, S3 filesystem) for use as the base image in Kubeflow Pipeline component pods. This is a TrustyAI sub-project, as reflected in the component label `odh-trustyai-ragas-lls-provider-dsp`.
