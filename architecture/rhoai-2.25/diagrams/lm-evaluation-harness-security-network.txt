RBAC Summary:
- This component defines NO RBAC resources.
- All RBAC for LMEval Jobs is managed by the trustyai-service-operator.
- The lmes-job Pod runs under a ServiceAccount created by the operator.

Secrets (User-Provisioned, Environment Variables):
- HF_TOKEN (Opaque)          - HuggingFace Hub auth for gated models/datasets. No auto-rotate.
- WATSONX_API_KEY (Opaque)   - WatsonX AI API authentication. No auto-rotate.
- WATSONX_URL (Opaque)       - WatsonX AI API endpoint URL. No auto-rotate.
- WATSONX_PROJECT_ID (Opaque)- WatsonX project scope. No auto-rotate.
- TEXTSYNTH_API_SECRET_KEY   - TextSynth API authentication. No auto-rotate.
- WANDB_API_KEY (Opaque)     - Weights & Biases experiment logging. No auto-rotate.

Container Security:
- Runs as non-root user (UID 65532, GID 65532)
- Base image: UBI 9 (Python 3.11) - FIPS-capable via system OpenSSL
- TLS inherits from Python ssl module using system OpenSSL on UBI 9
- All crypto operations use FIPS-approved algorithms (SHA-256 for caching)
- No MD5/RC4/DES usage detected
- confirm_run_unsafe_code flag required for code-eval benchmarks

Network Profile:
- INGRESS: None (batch job, not a service)
- EGRESS: HTTPS/443 to external model/dataset APIs (all TLS 1.2+)
- No service mesh sidecar required (ephemeral batch job)
- Air-gapped mode supported via HF_EVALUATE_OFFLINE=1 (zero egress)

FIPS Compliance:
- UBI 9 base image enforces FIPS at OS level when host policy is enabled
- Python ssl module uses system OpenSSL in FIPS mode automatically
- hashlib.sha256 (cache hashing) is FIPS-approved
- No explicit FIPS opt-in required
