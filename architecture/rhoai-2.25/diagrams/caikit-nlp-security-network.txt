RBAC Summary:
- No ClusterRoles or RoleBindings — caikit-nlp is a library, not a
  Kubernetes operator. RBAC is managed by the model serving
  infrastructure (KServe / ModelMesh).

Secrets:
- RUNTIME_TLS_SERVER_KEY (TLS key file) - gRPC/HTTP server TLS private key
  Provisioned by: Manual / cert-manager | Auto-Rotate: No
- RUNTIME_TLS_SERVER_CERT (TLS cert file) - gRPC/HTTP server TLS certificate
  Provisioned by: Manual / cert-manager | Auto-Rotate: No
- RUNTIME_TLS_CLIENT_CERT (TLS cert file) - mTLS client certificate verification
  Provisioned by: Manual / cert-manager | Auto-Rotate: No
- TGIS ca_cert_file (CA cert file) - TGIS backend TLS CA certificate
  Provisioned by: Manual | Auto-Rotate: No
- TGIS client_cert_file (TLS cert file) - TGIS backend mTLS client certificate
  Provisioned by: Manual | Auto-Rotate: No
- TGIS client_key_file (TLS key file) - TGIS backend mTLS client key
  Provisioned by: Manual | Auto-Rotate: No

Authentication & Authorization:
- gRPC services: mTLS optional (RUNTIME_TLS_CLIENT_CERT)
  Enforcement: Caikit runtime gRPC server
- HTTP endpoints: Configurable
  Enforcement: Caikit runtime HTTP server
- TGIS backend: mTLS optional (ca_cert_file/client_cert_file/client_key_file)
  Enforcement: gRPC channel
- PyTorch Distributed: No auth, no encryption (port 29550/TCP)
  Risk: Training traffic is unencrypted within cluster

FIPS Compliance:
- Python library on UBI 9 — inherits FIPS from base image OpenSSL
- No direct crypto library usage (relies on system ssl module and grpcio)
- grpcio links against system OpenSSL
- No FIPS opt-in or opt-out in source code
