RBAC Summary:
- Not applicable. vllm-cpu runs as a container image, not a Kubernetes operator.
- RBAC is managed by the deploying operator (KServe / ModelMesh).
- No ClusterRoles or ClusterRoleBindings defined by this component.

Secrets:
- ssl-keyfile (File mount) - TLS private key for HTTPS, provisioned by User/Operator, auto-rotate via SSLCertRefresher
- ssl-certfile (File mount) - TLS certificate for HTTPS, provisioned by User/Operator, auto-rotate via SSLCertRefresher
- ssl-ca-certs (File mount) - CA certificate chain, provisioned by User/Operator, auto-rotate via SSLCertRefresher
- hf-token (Env var HF_TOKEN) - Hugging Face Hub authentication, provisioned by User, no auto-rotate
- api-key (Env var VLLM_API_KEY or --api-key) - API key for request auth, provisioned by User, no auto-rotate

Authentication & Authorization:
- All API endpoints: Optional API Key (Bearer header), enforced by FastAPI middleware when --api-key set
- /metrics: Never authenticated, always accessible for Prometheus scraping
- /health, /ping: Never authenticated, always accessible for health probes
- TLS: Optional client cert validation via --ssl-cert-reqs (Uvicorn SSL)
- gRPC (TGIS): No authentication at application level

Network Exposure:
- Service Type: ClusterIP (internal only)
- No LoadBalancer or NodePort services defined
- External access managed by KServe (Istio Gateway) or ModelMesh
- Multi-arch support: x86_64, ppc64le, s390x
