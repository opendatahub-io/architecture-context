RBAC Summary:
- No RBAC resources — this is a runtime service, not a Kubernetes operator.
- RBAC is managed by the KServe ServingRuntime that deploys this container.
- Pod runs as UID 2000 (non-root) on UBI 9 minimal base.

Secrets:
- HF_TOKEN (env, Opaque) — HuggingFace Hub authentication, provisioned by user/operator, no auto-rotate
- S3_ACCESS_KEY_ID / S3_SECRET_ACCESS_KEY (env, Opaque) — S3 storage auth, provisioned by user, no auto-rotate
- VLLM_API_KEY (env, Opaque) — Client request authentication, provisioned by user, no auto-rotate
- SSL key/cert (file mount, kubernetes.io/tls) — TLS encryption, provisioned by cert-manager/manual, optional auto-rotate via --enable-ssl-refresh

FIPS Compliance:
- Base image: UBI 9 minimal (FIPS-validated OpenSSL)
- Python ssl module: Uses system OpenSSL (FIPS-compliant when OS FIPS mode enabled)
- hashlib: Uses hashlib.md5(usedforsecurity=False) for cache keys — compliant (Python 3.9+ FIPS-aware)
- hashlib.sha256: Used for API key validation (constant-time comparison)
- No Go binaries (Python application)
- No statically linked crypto libraries

Build Hermeticity:
- WARNING: No lock files present (rpms.lock.yaml, uv.lock, artifacts.lock.yaml)
- Uses uv pip install with version-pinned requirements/*.txt
- No Hermeto/cachi2 prefetch
- Supply chain risk at all three layers (OS, language deps, artifacts)

Authentication Flow:
- Client → KServe Route (TLS termination, 443/TCP)
- KServe → vllm-openai (8000/TCP, optional API key)
- API key validation: FastAPI AuthenticationMiddleware
  - Mechanism: SHA-256 hash comparison (constant-time via secrets.compare_digest)
  - Source: VLLM_API_KEY env var or --api-key CLI flag
  - Scope: All /v1/* endpoints; health/metrics endpoints excluded
