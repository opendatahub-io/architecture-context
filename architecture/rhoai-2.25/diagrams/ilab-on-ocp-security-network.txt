Security Notes:
===============

1. PyTorchJob worker pods run with Istio sidecar DISABLED for NCCL/Gloo
   inter-node communication (plaintext). This is a known security trade-off
   for distributed training performance.

2. Local vLLM server in Final Eval Pod runs on localhost with no auth
   (dynamic port, HTTP, plaintext). This is acceptable as it is pod-local.

3. All external egress connections use TLS 1.2+.

4. Model serving endpoints are protected by Authorino (Red Hat) authentication.

5. All secrets are user-provisioned (no auto-rotation).
