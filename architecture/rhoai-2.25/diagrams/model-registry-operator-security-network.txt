Authentication & Authorization Flows:
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│ OAuth Mode (External Access):                                                           │
│ 1. User sends Bearer token in Authorization header                                      │
│ 2. OAuth proxy validates token with OpenShift OAuth Server                              │
│ 3. OAuth proxy checks RBAC: User must have GET permission on service/{name}             │
│ 4. If authorized, OAuth proxy forwards request to REST container on localhost:8080      │
│                                                                                          │
│ Non-OAuth Mode (Internal Access):                                                       │
│ 1. Internal cluster pods access service directly on port 8080/TCP                       │
│ 2. No authentication (cluster-internal only)                                            │
│ 3. Use NetworkPolicies to restrict access if needed                                     │
│                                                                                          │
│ Operator Metrics:                                                                       │
│ 1. Prometheus sends Bearer token (ServiceAccount)                                       │
│ 2. kube-rbac-proxy validates token (tokenreviews, subjectaccessreviews)                 │
│ 3. Serves metrics on /metrics endpoint                                                  │
│                                                                                          │
│ Webhook Validation:                                                                     │
│ 1. API server sends request with webhook client certificate                             │
│ 2. Webhook server validates certificate against CA bundle                               │
│ 3. Returns validation/mutation result                                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

Security Best Practices:
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│ 1. TLS Encryption:                                                                      │
│    - All external traffic encrypted with TLS 1.2+                                       │
│    - Database connections support optional TLS (configure sslMode)                      │
│    - Webhook traffic uses TLS with certificate validation                               │
│                                                                                          │
│ 2. Authentication:                                                                      │
│    - OAuth mode: OpenShift OAuth with Bearer tokens                                     │
│    - Database: Username/Password stored in secrets                                      │
│    - Operator: ServiceAccount tokens for API server access                              │
│                                                                                          │
│ 3. Authorization:                                                                       │
│    - OAuth proxy enforces RBAC (GET permission on service/{name})                       │
│    - Operator uses ClusterRole with least-privilege permissions                         │
│    - NetworkPolicies restrict ingress to OAuth proxy port                               │
│                                                                                          │
│ 4. Secrets Management:                                                                  │
│    - Database passwords stored in Kubernetes secrets                                    │
│    - OAuth cookie secret generated by operator                                          │
│    - TLS certificates auto-rotated by Service CA                                        │
│    - Never store secrets in CR specs (use secret references)                            │
│                                                                                          │
│ 5. Pod Security:                                                                        │
│    - runAsNonRoot: true                                                                 │
│    - restricted-v2 SCC (OpenShift)                                                      │
│    - seccompProfile: RuntimeDefault                                                     │
│    - No privileged containers                                                           │
│                                                                                          │
│ 6. Network Segmentation:                                                                │
│    - NetworkPolicies limit ingress to operator-managed resources                        │
│    - OAuth proxy acts as authentication/authorization gateway                           │
│    - Internal cluster access uses separate service port (8080)                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘
