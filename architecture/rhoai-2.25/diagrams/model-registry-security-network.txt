Service Mesh Configuration:
================================================================================

Istio Integration:
  - VirtualService: Routes traffic from kubeflow-gateway to model-registry-service:8080
  - DestinationRule: ISTIO_MUTUAL TLS mode for model-registry-service
  - AuthorizationPolicy: ALLOW all requests to model-registry-server pods
  - PeerAuthentication: Sidecar injection for mTLS encryption

Authentication & Authorization:
  - REST API (/api/model_registry/*): Istio mTLS when mesh enabled, AuthorizationPolicy
  - Metrics (/metrics): Bearer Token with TokenReview + SubjectAccessReview
  - Database: Username/Password + optional TLS client certificates
