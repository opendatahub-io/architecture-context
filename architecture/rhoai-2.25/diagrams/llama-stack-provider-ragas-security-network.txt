================================================================================
NOTES
================================================================================

1. The inline provider has NO network exposure - it runs entirely in-process
   within the Llama Stack server.

2. The remote provider requires network access to:
   - Kubeflow Pipelines API (443/TCP, HTTPS, Bearer Token)
   - S3 Storage (443/TCP, HTTPS, AWS IAM)

3. KFP pipeline pods require network access back to:
   - Llama Stack Server (configurable port, HTTP/HTTPS)
   - S3 Storage (443/TCP, HTTPS, AWS IAM)

4. The sample distribution server (8321/TCP) is intended for localhost/dev use
   only. Production deployments use the Llama Stack K8s Operator.

5. The Ollama backend (11434/TCP) is only used in the sample distribution
   and communicates over unencrypted HTTP.
