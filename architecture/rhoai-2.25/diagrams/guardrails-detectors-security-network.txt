TLS Support:
- All detectors run plaintext by default
- Optional TLS support via ssl_ca_certs in config.yaml
- When enabled: ssl.CERT_REQUIRED (mutual TLS)
- Platform-level mTLS available via KServe / Istio Service Mesh

RBAC Summary:
- No RBAC defined in this repository
- RBAC managed by deploying operator (rhods-operator) and KServe
- No ServiceAccounts defined in detector code

Authentication & Authorization:
- No authentication at microservice level
- Security enforced at platform level:
  - KServe InferenceService configuration
  - NetworkPolicy (cluster-level)
  - Optional Istio service mesh mTLS (STRICT mode)

Secrets:
- MODEL_DIR (volume mount) - Path to HuggingFace model files, provisioned by KServe/Operator, no auto-rotation
- VLLM_BASE_URL (environment variable) - URL of external vLLM server, provisioned by Operator/ConfigMap, no auto-rotation

Container Images:
- odh-built-in-detector-rhel9 (Dockerfile.konflux.builtIn) - Base: ubi9/ubi-minimal, Port: 8080/TCP
- odh-guardrails-detector-huggingface-runtime-rhel9 (Dockerfile.konflux.hf) - Base: ubi9/ubi-minimal, Port: 8000/TCP
- LLM Judge detector (Dockerfile.judge) - Not shipped via Konflux in rhoai-2.25 branch
