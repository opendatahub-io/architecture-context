RBAC Summary:
- No RBAC required. rest-proxy is a sidecar container that does not
  interact with the Kubernetes API.

TLS Configuration:
- REST Listener TLS:
  - Enabled when: MM_TLS_KEY_CERT_PATH and MM_TLS_PRIVATE_KEY_PATH are set
  - Method: http.ListenAndServeTLS
  - FIPS: Yes (strictfipsruntime)

- gRPC Upstream TLS:
  - Enabled when: REST_PROXY_USE_TLS=true
  - Method: credentials.NewTLS
  - Verify: Configurable (REST_PROXY_SKIP_VERIFY)
  - Disabled: insecure.NewCredentials() (plaintext gRPC)

Authentication & Authorization:
- Endpoint: /v2/models/* (GET, POST)
- Auth at proxy: None
- Enforcement: Delegated to ModelMesh Serving infrastructure
- The proxy itself performs no authentication or authorization

Secrets:
- TLS cert/key (kubernetes.io/tls) - TLS termination for HTTPS listener
  and/or gRPC upstream TLS. Provisioned by ModelMesh Serving operator /
  cert-manager. Rotation depends on provisioner.

FIPS Compliance:
- Built with GOEXPERIMENT=strictfipsruntime -tags strictfipsruntime
- Uses FIPS-validated cryptographic modules on RHEL/UBI with FIPS mode
- Base image: registry.access.redhat.com/ubi9/ubi-minimal

Supported Architectures: amd64, arm64, ppc64le, s390x
