TLS Configuration:
  REST Endpoint (client-facing):
  - Enabled when: MM_TLS_KEY_CERT_PATH and MM_TLS_PRIVATE_KEY_PATH are set
  - Certificate source: Provided by ModelMesh Serving / cert-manager
  - Protocol: HTTPS (TLS)
  - When disabled: Plain HTTP on port 8008

  gRPC Backend (pod-internal):
  - Enabled when: REST_PROXY_USE_TLS=true
  - InsecureSkipVerify: Configurable via REST_PROXY_SKIP_VERIFY (default: false)
  - Scope: localhost:8033 only
  - When disabled: Plain gRPC on port 8033

FIPS Compliance:
  - Build: Konflux Dockerfile uses CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime
  - Runtime base: ubi9/ubi-minimal (includes OpenSSL for FIPS)
  - Crypto: stdlib crypto/tls (FIPS via OpenSSL with strictfipsruntime)
  - Note: Upstream Dockerfile uses CGO_ENABLED=0 (non-FIPS, statically linked)

RBAC Summary:
  - No ClusterRoles defined
  - No RoleBindings defined
  - Component does not interact with Kubernetes API
  - Runs as sidecar container within ModelMesh Serving pod

Secrets:
  - TLS cert/key (via env vars MM_TLS_KEY_CERT_PATH / MM_TLS_PRIVATE_KEY_PATH)
    - Type: TLS certificate
    - Provisioned by: ModelMesh Serving / cert-manager
    - Auto-Rotate: Unknown

Authentication & Authorization:
  - No authentication at proxy level
  - Authentication delegated to enclosing ModelMesh Serving infrastructure
  - No authorization policies enforced by rest-proxy

Environment Variables:
  - MM_TLS_KEY_CERT_PATH: Path to TLS certificate for HTTPS listener
  - MM_TLS_PRIVATE_KEY_PATH: Path to TLS private key for HTTPS listener
  - REST_PROXY_USE_TLS: Enable TLS for gRPC backend connection
  - REST_PROXY_SKIP_VERIFY: Skip TLS certificate verification for gRPC (default: false)
  - REST_PROXY_LISTEN_PORT: REST listen port (default: 8008)
  - REST_PROXY_GRPC_PORT: gRPC backend port (default: 8033)
