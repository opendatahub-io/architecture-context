Secrets:
================================================================================
- argo-workflows-agent-ca-certificates (Opaque)
  Purpose: CA certificates for agent containers
  Provisioned by: Administrator
  Auto-rotate: No

- argo-server-tls (kubernetes.io/tls, optional)
  Purpose: TLS certificates for Argo Server
  Provisioned by: Administrator or self-signed
  Auto-rotate: No

- workflow-controller-configmap (ConfigMap)
  Purpose: Controller configuration (artifact repos, persistence, SSO)
  Provisioned by: Administrator
  Auto-rotate: No

Container Security:
================================================================================
- workflow-controller: UID 8737, Read-Only Root FS, No Privilege Escalation, Drop ALL caps
- argoexec: UID 2000, Writable Root FS (needed for artifacts), No Privilege Escalation
- argo-server: Non-root, Read-Only Root FS, No Privilege Escalation, Drop ALL caps

Authentication:
================================================================================
- Controller Metrics (9090/TCP): No auth (internal only)
- Controller Healthz (6060/TCP): No auth (Pod-only)
- Argo Server API (2746/TCP, not deployed in RHOAI):
  Bearer Token (K8s SA token), SSO (OIDC), or Server mode
  Enforced by gRPC interceptor (gatekeeper) with K8s RBAC impersonation
