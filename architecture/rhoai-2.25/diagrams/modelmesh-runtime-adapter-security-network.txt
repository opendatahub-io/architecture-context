RBAC Summary:
  None. This component runs as a sidecar and does not directly interact with
  the Kubernetes API. The model-serving pod's service account provides any
  required permissions (managed by modelmesh-controller).

Service Mesh Configuration:
  N/A. All adapter communication is pod-internal (localhost). No service mesh
  sidecar is needed for adapter ↔ runtime communication. External network
  policies for egress to cloud storage are managed at the namespace level
  by modelmesh-controller or cluster network policies.
