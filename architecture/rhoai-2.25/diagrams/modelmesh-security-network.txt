NETWORK POLICIES:
================================================================================
Policy: model-mesh
  Pod Selector: app=model-mesh, component=model-mesh
  Ingress Rules:
  - Intra-deployment pods → 8033/TCP, 8080/TCP (internal mesh communication)
  - Any → 8033/TCP (inference requests from clients)
  - Any → 2112/TCP (Prometheus metrics scraping)

TLS CONFIGURATION:
================================================================================
Environment Variables:
  - MM_TLS_PRIVATE_KEY_PATH         → TLS private key file path
  - MM_TLS_KEY_CERT_PATH            → TLS certificate chain file path
  - MM_TLS_PRIVATE_KEY_PASSPHRASE   → Optional passphrase for encrypted key
  - MM_TLS_CLIENT_AUTH              → NONE | REQUIRE | OPTIONAL
  - MM_TLS_TRUST_CERT_PATH          → Trusted CA certificate path(s)

SECRETS:
================================================================================
  - model-mesh-tls-secret (kubernetes.io/tls)
    Purpose: TLS key and certificate for gRPC server and inter-pod communication
    Provisioned by: Cluster admin / cert-manager
    Auto-Rotate: No (manual)

  - model-mesh-icd-etcd (Opaque)
    Purpose: etcd connection configuration (JSON: endpoints, credentials)
    Provisioned by: Cluster admin
    Auto-Rotate: No

SSRF PROTECTION:
================================================================================
  RemotePayloadProcessor validates URIs:
  - Only HTTP and HTTPS schemes allowed
  - Rejects localhost/loopback addresses
  - Rejects private IP ranges (RFC 1918)
  - Prevents Server-Side Request Forgery attacks

AUTHENTICATION & AUTHORIZATION:
================================================================================
  - gRPC API (8033): mTLS client certs (optional, configurable per deployment)
  - Inference RPCs: mm-model-id / mm-vmodel-id metadata header required
  - Prometheus /metrics (2112): Network-level only (no auth)
  - Health probes (8089): Kubelet access only (no auth)
