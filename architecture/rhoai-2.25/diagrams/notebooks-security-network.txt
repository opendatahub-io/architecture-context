================================================================================
CONTAINER SECURITY SUMMARY
================================================================================

Base Images:
- Red Hat UBI9 (ubi9/python-312) - certified, FIPS-compliant
- CUDA 12.6/12.8 base for GPU variants
- ROCm 6.2/6.4 base for AMD GPU variants

Container Hardening:
- Non-root execution: UID 1001, GID 0 (OpenShift compatible)
- Package verification: uv pip install --verify-hashes (pylock.toml)
- Vulnerability mitigation: dnf -y upgrade --refresh in every build
- Image pinning: All images referenced by SHA256 digest in params.env
- Build traceability: commit.env tracks exact source commit per image

Secrets:
- rhel-subscription-secret (Opaque) - RHEL subscription for RStudio BuildConfig
  - Provisioned by: Manual/Cluster Admin
  - Auto-Rotate: No

Authentication & Authorization:
- Notebook containers do NOT implement authentication
- Auth handled by sidecar injection from ODH Notebook Controller:
  - RHOAI 3.x: kube-rbac-proxy sidecar (SubjectAccessReview)
  - RHOAI 2.x: oauth-proxy sidecar (OAuth)
- Policy: User must have access to notebook namespace

Ports Summary:
- 443/TCP  - HTTPS (ingress, K8s API, S3, registry) - TLS 1.2+
- 8443/TCP - kube-rbac-proxy sidecar - TLS
- 8888/TCP - JupyterLab (HTTP, pod-local only)
- 8080/TCP - Code Server via NGINX (HTTP, pod-local only)
- 8787/TCP - RStudio / code-server internal (HTTP, pod-local only)
