RBAC Summary:
- No RBAC resources defined by this repository
- Workbench pods inherit ServiceAccount RBAC configured by odh-notebook-controller
- kube-rbac-proxy sidecar enforces OAuth + RBAC at pod ingress

Authentication & Authorization:
- JupyterLab (:8888) → All methods → kube-rbac-proxy (RBAC + OAuth) → Enforcement: sidecar :8443
- code-server (:8787) → All methods → kube-rbac-proxy (RBAC + OAuth) → Enforcement: sidecar :8443
- RStudio (:8787)     → All methods → kube-rbac-proxy (RBAC + OAuth) → Enforcement: sidecar :8443
- Policy: Only authenticated users with notebook access can reach the workbench

FIPS Compliance:
- mongocli: CGO_ENABLED=1, -tags strictfipsruntime (FIPS-compliant Go crypto)
- Python: Inherits OpenSSL from UBI9 base image (system FIPS mode)
- TLS: Terminated by kube-rbac-proxy sidecar (not application)
- No custom crypto in workbench application code
- CVE-2026-2492: TensorFlow HDF5 plugin disabled (HDF5_PLUGIN_PATH=disable)

Secrets:
- No secrets defined by this repository
- At runtime, workbench pods may mount user-configured secrets:
  - S3 credentials (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
  - Database connections (PostgreSQL, MySQL)
  - Managed by RHOAI Dashboard and notebook controller

Build Supply Chain:
- Python deps: pylock.toml (PEP 751) with --verify-hashes --compile-bytecode
- Go deps: go.sum (mongocli build stage only)
- RPMs: No lock file (versions float with UBI9 base)
- No Hermeto/cachi2 prefetch (builds download from PyPI directly)
