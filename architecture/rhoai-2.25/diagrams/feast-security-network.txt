TLS Configuration:
- Operator: OpenShift service serving cert annotations on Services
- Feature Server: TLS ports (6567, 6571, 6573, 8816, 8443) when on OpenShift
- Routes: re-encrypt TLS mode (external TLS → internal TLS)
- Custom CA: odh-trusted-ca-bundle ConfigMap (label-driven injection)
- Go Server Redis: Optional tls.Config{} (empty/default when ssl=true)

Secrets:
- feast-{name}-{type}-tls (kubernetes.io/tls) - Service TLS certs, OpenShift auto-provisioned, auto-rotated
- User-provided OIDC secret (Opaque) - client_id, auth_discovery_url, client_secret
- User-provided DB credentials (Opaque) - Database connection credentials

Authentication Modes:
- No auth (default) - All endpoints open
- Kubernetes RBAC - Empty Roles populated by admins; feast SA checks SubjectAccessReview
- OIDC - Bearer token validation via configured OIDC provider

Security Notes:
- ⚠ CronJob uses kubectl exec (pods/exec) - privileged operation
- ⚠ Transformation Service uses insecure gRPC (no TLS, no auth)
- ⚠ Go embedded server does NOT enforce authorization (unlike Python server)
- ⚠ Python feature-server requirements.txt has no lock file (hermeticity gap)
- ✓ FIPS: Operator uses CGO_ENABLED=1, GOEXPERIMENT=strictfipsruntime
- ✓ FIPS: Feature server inherits OpenSSL from UBI9 base
- ⚠ FIPS risk: hashlib.md5 in bigtable online store (non-default backend)
