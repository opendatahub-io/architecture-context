Secrets Management:
===================

  - Purpose: S3/storage credentials for model retrieval
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, endpoint, region
  - Provisioned By: User/Admin
  - Auto-Rotate: No (manual management)
  - Mounted By: Storage puller containers
  - Exposure: Environment variables in puller container

  - Purpose: ETCD connection configuration and credentials
  - Keys: etcd_connection (JSON with endpoints, credentials)
  - Provisioned By: User/Admin
  - Auto-Rotate: No (manual management)
  - Mounted By: ModelMesh containers
  - Exposure: Environment variable ETCD_CONFIG

  - Methods: POST
  - Auth: Optional (application-level, can integrate with Service Mesh)
  - Enforcement: Application code or Service Mesh
  - Policy: User-defined (extensible via Istio AuthorizationPolicy)
  - Purpose: Model inference requests

Endpoint: ETCD (2379/TCP)
  - Methods: ALL (gRPC/HTTP)
  - Auth: Optional mTLS
  - Enforcement: ETCD server
  - Policy: ETCD authentication (if enabled)
  - Purpose: Model metadata storage and retrieval


Service Mesh Integration (Optional):
=====================================

When integrated with OpenShift Service Mesh / Istio:
  - PeerAuthentication: Can enforce mTLS STRICT mode for pod-to-pod communication
  - AuthorizationPolicy: Can restrict access to inference endpoints based on service identity
  - DestinationRule: Can configure TLS settings for ETCD egress
  - VirtualService: Can expose inference endpoints via Istio Gateway
  - Telemetry: Can collect distributed traces and advanced metrics

Note: By default, ModelMesh Serving does NOT enforce mTLS or Service Mesh integration.
This is an optional enhancement for production deployments requiring higher security.
