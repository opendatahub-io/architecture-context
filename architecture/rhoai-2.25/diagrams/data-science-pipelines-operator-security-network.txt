NETWORK POLICIES:
- mariadb-{name}: Restricts ingress to MariaDB from API Server and MLMD only
- metadata-grpc-{name}: Restricts ingress to MLMD gRPC from pipeline pods
- ds-pipeline-{name}: Common network policy for pipeline namespace
- mlmd-envoy-dashboard-access: Allows dashboard access to MLMD Envoy

RBAC Summary:
- ClusterRole: manager-role
  - API Groups: datasciencepipelinesapplications.opendatahub.io, "", apps, rbac, route, argoproj.io, kubeflow.org, pipelines.kubeflow.org, monitoring.coreos.com, networking.k8s.io, batch, serving.kserve.io, ray.io, machinelearning.seldon.io, workload.codeflare.dev
  - Verbs: Full CRUD on most resources
  - Scope: Cluster-wide

- ClusterRole: manager-argo-role
  - API Groups: argoproj.io, "", coordination.k8s.io, policy
  - Resources: workflows, workflowtasksets, cronworkflows, pods, configmaps, secrets, leases
  - Scope: Cluster-wide

- Aggregate ClusterRoles:
  - aggregate-dspa-admin-edit: Grants DSPA + Pipeline CRUD to admin/edit roles
  - aggregate-dspa-admin-view: Grants DSPA + Pipeline read to view roles

Per-DSPA Namespace Roles:
- ds-pipeline-{name}: API server service account permissions
- pipeline-runner-{name}: Pipeline pod execution permissions (includes KServe, Ray, Seldon, CodeFlare)
- ds-pipeline-user-access-{name}: User access to pipeline resources
- ds-pipeline-persistenceagent-{name}: Persistence agent permissions
- ds-pipeline-scheduledworkflow-{name}: Scheduled workflow permissions
- ds-pipeline-workflow-controller-{name}: Argo workflow controller permissions

Secrets:
- ds-pipelines-proxy-tls-{name} (kubernetes.io/tls) - OAuth proxy TLS, service-ca, auto-rotate
- ds-pipelines-envoy-proxy-tls-{name} (kubernetes.io/tls) - Envoy OAuth proxy TLS, service-ca, auto-rotate
- ds-pipelines-ui-proxy-tls-{name} (kubernetes.io/tls) - UI OAuth proxy TLS, service-ca, auto-rotate
- ds-pipelines-mariadb-tls-{name} (kubernetes.io/tls) - MariaDB TLS, service-ca, auto-rotate
- ds-pipeline-metadata-grpc-tls-certs-{name} (kubernetes.io/tls) - MLMD gRPC TLS, service-ca, auto-rotate
- ds-pipeline-metadata-grpc-tls-config-secret-{name} (Opaque) - MLMD gRPC TLS config, operator, no auto-rotate
- ds-pipeline-db-{name} (Opaque) - MariaDB password, auto-generated if not user-provided
- ds-pipeline-s3-{name} (Opaque) - MinIO credentials, auto-generated if not user-provided
- {webhook-name}-tls (kubernetes.io/tls) - Webhook TLS, service-ca, auto-rotate

CA Bundle Aggregation:
- Sources: odh-trusted-ca-bundle ConfigMap + user CABundle + service-ca certs
- Merged into: dsp-trusted-ca-{name} ConfigMap
- Mounted into: All components via SSL_CERT_DIR
- Purpose: Custom/proxy CA trust for external DB/S3 connections

FIPS Compliance:
- Build: GOEXPERIMENT=strictfipsruntime, CGO_ENABLED=1
- Runtime: FIPS launcher binaries (launcher-v2-fips, argoexec-fips) when DSPO.FIPSEnabled=true
- TLS: stdlib crypto/tls with FIPS-compliant OpenSSL backend
- Risk: Database TLS supports "skip-verify" (InsecureSkipVerify) which weakens TLS but does not break FIPS
