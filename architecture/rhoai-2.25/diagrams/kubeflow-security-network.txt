    datasciencepipelinesapplications (get,list,watch)

- Aggregated Roles:
  - kubeflow-notebooks-admin: aggregation rule for notebooks
  - kubeflow-notebooks-edit: notebooks (full CRUD)
  - kubeflow-notebooks-view: notebooks (get,list,watch)

Secrets:
- {notebook-name}-oauth-config (Opaque) - OAuth proxy cookie secret, 16 random bytes base64
- {notebook-name}-oauth-client (Opaque) - OAuth client secret, 16 random alphanumeric chars
- {notebook-name}-tls (kubernetes.io/tls) - TLS cert for OAuth service, OpenShift Service CA, auto-rotate
- odh-notebook-controller-webhook-cert (kubernetes.io/tls) - Webhook TLS cert, OpenShift Service CA, auto-rotate
- ds-pipeline-config (Opaque) - Elyra runtime config JSON (DSPA endpoint, S3 creds)

Authentication Architecture:
- Notebook access: OpenShift OAuth with oauth-proxy sidecar
- SAR authorization: verb=get, resource=notebooks, group=kubeflow.org
- OAuthClient: cluster-scoped, managed with finalizer (notebook-oauth-client-finalizer.opendatahub.io)
- Webhook: fail-closed (failurePolicy: Fail), only API server can call

Network Isolation:
- Per-notebook NetworkPolicies:
  1. Allow controller → notebook:8888/TCP (culling)
  2. Allow ingress → notebook:8443/TCP (OAuth proxy)
- All other ingress denied by default
