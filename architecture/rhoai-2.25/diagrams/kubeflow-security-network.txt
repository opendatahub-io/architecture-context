Known Security Considerations:
═══════════════════════════════════════════════════════════════════════════
1. Metrics endpoint (/metrics:8080) exposed over HTTP without auth
   → Mitigation: ClusterIP service, cluster-internal access only

2. Culling feature requires unauthenticated access to /api/kernels:8888
   → Mitigation: Pod-to-pod communication within cluster only

3. Health probes (/healthz, /readyz:8081) exposed over HTTP
   → Mitigation: ClusterIP service, used by Kubernetes only

4. Notebook pods may run with HTTP (not HTTPS)
   → Mitigation: Use Istio VirtualService for TLS termination at gateway
