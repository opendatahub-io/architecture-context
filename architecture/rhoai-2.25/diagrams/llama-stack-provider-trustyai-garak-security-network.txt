RBAC Summary:
- No Kubernetes RBAC resources defined by this component
- RBAC is managed by llama-stack-k8s-operator when deployed via operator
- KFP access controlled via kubeconfig service account token

Authentication & Authorization:
- Llama Stack API (8321/TCP): No application-level auth (delegated to platform)
- vLLM endpoint: Bearer Token (VLLM_API_TOKEN env var)
- KFP Pipelines API: Bearer Token (kubeconfig service account)
- S3/MinIO: AWS IAM (AccessKey/SecretKey)

TLS Configuration:
- GARAK_TLS_VERIFY: Controls TLS for shield orchestrator HTTP calls
  (boolean or CA cert path)
- VLLM_TLS_VERIFY: Controls TLS for vLLM endpoint connections
- Provider-level tls_verify: Pydantic-validated boolean or cert file path
- Remote mode: verify_ssl passed to KFP client and boto3 S3 client

Secrets:
- VLLM_API_TOKEN (env var) - vLLM auth, User/Platform provisioned, No auto-rotate
- AWS_ACCESS_KEY_ID (env var / K8s Secret) - S3 auth, User/Platform, No auto-rotate
- AWS_SECRET_ACCESS_KEY (env var / K8s Secret) - S3 auth, User/Platform, No auto-rotate
- aws-connection-pipeline-artifacts (K8s Secret) - KFP pod S3 access, DSP provisioned
- OPENAICOMPATIBLE_API_KEY (env var) - Generator API key, defaults to "DUMMY"
- Kubeconfig token (in-cluster SA) - KFP API auth, OpenShift managed, Auto-rotate: Yes
