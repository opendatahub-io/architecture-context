  - Provisioned by: Operator/platform
  - Auto-Rotate: No
- TLS client CA cert (File-based PEM) - mTLS client verification
  - Provisioned by: Operator/platform
  - Auto-Rotate: No
- TLS client certs per service (File-based PEM) - mTLS to backend services
  - Provisioned by: Operator/platform
  - Auto-Rotate: No

TLS Configuration:
- Server-side: rustls with ring crypto provider, TLS 1.2 and TLS 1.3
  - Configured via: --tls-cert-path, --tls-key-path CLI args
  - mTLS: --tls-client-ca-cert-path (optional, WebPKI verification)
- Client-side: Per-service named TLS configs in config.yaml
  - Supports: cert/key/CA per backend
  - gRPC clients: tonic TLS
  - HTTP clients: hyper-rustls
  - Option: insecure: true to skip server cert verification

Authentication & Authorization:
- /health, /info: No authentication (open endpoints)
- /api/v1/*, /api/v2/*: mTLS (optional, server-side rustls)
- Backend services: mTLS or Bearer token passthrough (per-service config)
- Header passthrough: Configurable via passthrough_headers
- X-Forwarded-Access-Token rewrite to Bearer: configurable
