NAMESPACE SUMMARY:
- redhat-ods-operator: OLM-managed operator deployment (1 pod)
- redhat-ods-applications: Platform control plane (~14 controller pods + dashboard)
- redhat-ods-monitoring: Observability stack (MonitoringStack, Tempo, OTEL)
- istio-system (optional): Service mesh control plane + Authorino
- User namespaces: Per-project workloads (notebooks, ISVCs, pipelines, etc.)

SERVICE MESH CONFIGURATION (when enabled):
- Mode: STRICT mTLS for RHOAI namespaces
- PeerAuthentication: Namespace-scoped, enforced at sidecar
- AuthorizationPolicy: Per-InferenceService via Authorino
- VirtualService: KServe serverless traffic routing
- DestinationRule: TrustyAI, odh-model-controller
- EnvoyFilter: KServe activator filter, inference graph filter
- Ray clusters: Independent mTLS via self-signed CA (not Istio)
