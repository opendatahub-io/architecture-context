RBAC Summary:
- ServiceAccount: odh-dashboard (namespace: redhat-ods-applications)
- ClusterRole: odh-dashboard
  - Broad read: nodes, machinesets, clusterversions, CSVs, users, groups, pods, services, events
  - Write: configmaps, secrets, PVCs, notebooks, model registries, RBAC bindings
  - Cluster-scoped: storageclasses (update, patch), nodes (get, list)
- ClusterRole: dsg-cluster-roles
  - Read: imagestreams, imagestreams/layers, builds, buildconfigs
- ClusterRoleBinding: odh-dashboard-monitoring → cluster-monitoring-view
- ClusterRoleBinding: odh-dashboard-auth-delegator → system:auth-delegator
  - Purpose: Token review for user validation

Authentication Chain:
1. OpenShift OAuth (oauth-proxy sidecar, port 8443)
   - Validates OAuth session cookie
   - Requires: user can `list` `projects` resource
   - Extracts: Bearer token → x-forwarded-access-token header
2. Backend Route Security (secureRoute / secureAdminRoute)
   - Validates: user token exists, namespace ownership
   - Admin routes: checks adminGroups membership
3. BFF SAR/SSAR Authorization (model-registry-ui BFF)
   - Per-request SubjectAccessReview against K8s API
   - Validates: user has access to specific model registry service
4. K8s Native RBAC (/api/k8s/* proxy)
   - Direct passthrough: K8s enforces RBAC on the proxied API call

Secrets:
- dashboard-oauth-config-generated (Opaque) - OAuth cookie secret, 16-char random, Kustomize secretGenerator, no auto-rotation
- dashboard-oauth-client-generated (Opaque) - OAuth client secret, 32-char random, Kustomize secretGenerator, no auto-rotation
- dashboard-proxy-tls (kubernetes.io/tls) - TLS cert for oauth-proxy, OpenShift service serving cert controller, auto-rotated

NetworkPolicy:
- RHOAI shared overlay adds NetworkPolicy restricting ingress to dashboard pod
- Modular architecture overlay adds NetworkPolicy for model-registry-ui sidecar communication

FIPS Compliance:
- Main dashboard (Node.js): Inherits FIPS from UBI 9 base image OpenSSL
- Model Registry BFF (Go): CGO_ENABLED=1, -tags strictfipsruntime, TLS 1.3 minimum
- Gen AI BFF (Go): CGO_ENABLED=1, -tags strictfipsruntime
  *** RISK: InsecureSkipVerify=true hardcoded on outbound HTTP transport ***

Security Observations:
- LlamaStack communication is plaintext HTTP (port 8321) - no TLS
- Gen AI BFF has InsecureSkipVerify=true bypassing TLS cert validation on all outbound calls
- OAuth secrets (cookie + client) are not auto-rotated
- WebSocket auth uses base64url-encoded token in subprotocol header (non-standard but functional)
