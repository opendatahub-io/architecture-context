Authentication Flow:
  1. User accesses dashboard Route (HTTPS/443)
  2. OpenShift Router terminates external TLS, re-encrypts to oauth-proxy (8443)
  3. oauth-proxy checks for valid OAuth session cookie
     - If no session: redirects to OpenShift OAuth login
     - If session valid: validates projects:list permission via delegate-urls
  4. oauth-proxy forwards request with x-forwarded-access-token header
  5. Backend uses user's token for all K8s API calls (pass-through)
  6. K8s API enforces user's own RBAC permissions on all operations
  7. /metrics endpoint bypasses auth (skip-auth-regex)
