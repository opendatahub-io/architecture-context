================================================================================
DEPLOYMENT PLATFORMS
================================================================================

OVMS does not manage its own ingress, services, or RBAC. It relies on:

  [KServe]
  - Deploys OVMS as InferenceService container
  - Creates Kubernetes Services, Istio VirtualServices, or Gateway API HTTPRoutes
  - Manages external traffic routing and TLS termination

  [ModelMesh]
  - Deploys OVMS as a ServingRuntime backend
  - Exposes unified inference endpoint
  - Routes requests to OVMS pods

================================================================================
RBAC SUMMARY
================================================================================

No RBAC resources defined by OVMS.
OVMS runs as a non-root application (UID 5000) within a pod.
All RBAC is managed by the deploying platform (KServe/ModelMesh).

================================================================================
AUTHENTICATION & AUTHORIZATION
================================================================================

OVMS has NO built-in authentication or authorization.
All API endpoints are accessible without credentials.

Security is enforced externally by:
  - KServe Istio sidecar (mTLS, AuthorizationPolicy)
  - kube-rbac-proxy sidecar (RHOAI 3.x pattern)
  - Network Policies applied by the platform operator
