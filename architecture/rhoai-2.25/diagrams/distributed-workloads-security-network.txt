RBAC Summary:
- ClusterRole: rhods-operator-scc
  - API Groups: security.openshift.io
  - Resources: securitycontextconstraints
  - Verbs: get, watch, list, create, update, patch, delete
  - Bound to: rhods-operator SA (redhat-ods-operator namespace)

- Dynamic Test Roles (created per test):
  - API Groups: ray.io, kubeflow.org, kueue.x-k8s.io
  - Resources: rayclusters, pytorchjobs, localqueues
  - Verbs: get, list, watch, create, delete
  - Scope: Test namespace, cleaned up after test

Admission Control:
- ValidatingAdmissionPolicy: kueue-validating-admission-policy-binding
  - Enforces: kueue.x-k8s.io/queue-name label on RayClusters
  - Action: Deny creation of RayCluster without queue-name label

Authentication & Authorization:
- Kubernetes API: Bearer Token (kubeconfig) + RBAC
- Prometheus API: Bearer Token via oauth-proxy
- Ray Dashboard: No authentication (test-only, not production exposure)
- S3 Storage: AWS IAM credentials (env vars: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
- HuggingFace: HF_TOKEN environment variable for gated model access

Secrets:
- AWS credentials (Environment Variables) - S3 access, manual provisioning, no auto-rotation
- HF_TOKEN (Environment Variable) - HuggingFace gated model access, manual provisioning, no auto-rotation

Security Notes:
- Ray inter-pod communication (6379, 10001, 29500) uses plaintext with NO authentication
- Ray Dashboard (8265) has NO authentication - should only be exposed during testing
- NCCL rendezvous (29500) uses plaintext TCP - relies on network isolation
- No Service Mesh / mTLS configured for workload pods
- No NetworkPolicies defined in this repository
