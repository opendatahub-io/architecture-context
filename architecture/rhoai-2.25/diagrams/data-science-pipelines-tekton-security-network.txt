RBAC Summary:
- ClusterRole: ds-pipeline-persistenceagent-clusterrole
  - API Groups: argoproj.io, kubeflow.org, tekton.dev
  - Resources: workflows, scheduledworkflows, pipelineruns, taskruns, conditions
  - Verbs: get, list, watch (argoproj/kubeflow); create, get, list, watch, update, patch, delete (tekton)

- Role: ds-pipeline (namespace-scoped)
  - SA: ds-pipeline
  - Resources: pods, pods/log, workflows, pipelineruns, taskruns, scheduledworkflows, pipelineloops, subjectaccessreviews, tokenreviews
  - Verbs: varies per resource (full CRUD on tekton/kubeflow; create on SAR/TR)

- Role: ds-pipeline-scheduledworkflow-role (namespace-scoped)
  - SA: ds-pipeline-scheduledworkflow
  - Resources: workflows, scheduledworkflows, pipelineruns, taskruns, events, pipelineloops
  - Verbs: create, get, list, watch, update, patch, delete

- Role: pipeline-runner (namespace-scoped) ⚠ PRIVILEGED
  - SA: pipeline-runner
  - Resources: secrets, configmaps, PVs, PVCs, pods/exec, services, deployments, replicasets, jobs, seldondeployments, ALL kubeflow.org resources
  - Verbs: *(wildcard) on most resources

Secrets:
- mlpipeline-minio-artifact (Opaque) - MinIO/S3 credentials (accesskey, secretkey, host, port, secure) - DSPO/Manual, No auto-rotate
- mysql-secret (Opaque) - MySQL credentials (username, password) - DSPO/Manual, No auto-rotate
- tekton-pipelineloop-webhook-certs (kubernetes.io/tls) - TLS certs for PipelineLoop webhook - Knative cert controller, Auto-rotate
- (cache server TLS) (kubernetes.io/tls) - TLS cert/key at /etc/webhook/certs - Cache deployer, Auto-rotate

Authentication Chain (API Server - multi-user mode):
1. Extract user identity from HTTP header (x-goog-authenticated-user-email)
2. Validate Bearer token via Kubernetes TokenReview (audience: pipelines.kubeflow.org)
3. Authorize write operations via SubjectAccessReview on pipelines.kubeflow.org resources

Webhook Authentication:
- Cache Server (/mutate): TLS client cert from Kubernetes API (MutatingWebhookConfiguration)
- PipelineLoop Webhook (/defaulting, /resource-validation): TLS client cert (ValidatingWebhookConfiguration)

Ingress:
- No ingress defined in component manifests
- Ingress is managed externally by the Data Science Pipelines Operator or platform operator

FIPS Notes:
- API Server, Persistence Agent, Scheduled Workflow: UBI base (inherits OpenSSL)
- Cache Server, Viewer Controller: Alpine base (no OpenSSL, non-FIPS)
- Tekton Catalog components: CGO_ENABLED=0 (static linking, would fail check-payload)
- No Konflux Dockerfiles on this branch; downstream builds likely differ
