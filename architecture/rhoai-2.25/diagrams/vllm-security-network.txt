RBAC Summary:
- No RBAC resources defined in this component
- RBAC is managed by KServe and the platform operator
- Pod ServiceAccount determined by KServe InferenceService configuration

Secrets:
- No secrets defined in this repository
- Model storage credentials provided by KServe at runtime (storage-config Secret)
- TLS certificates managed by platform ingress / cert-manager

Authentication & Authorization:
- vLLM container ports (8000, 8033) are pod-internal and NOT directly exposed
- Authentication enforced by sidecar proxy:
  - RHOAI 2.x: oauth-proxy (8443/TCP)
  - RHOAI 3.x: kube-rbac-proxy (8443/TCP)
- External access via platform ingress (HTTPRoute/Route) with TLS 1.2+

FIPS Compliance:
- Build: No Go binaries compiled in this repo (Python runtime only)
- Crypto: Inherited from RHAIIS base image (UBI9 with FIPS-validated OpenSSL)
- Assessment: Full FIPS evaluation requires RHAIIS base image source analysis
- Base image: registry.redhat.io/rhaiis/vllm-cuda-rhel9 (UBI9-based)
