RBAC Summary:
- No Kubernetes RBAC resources defined
- This is a standalone gRPC server without Kubernetes API interactions at runtime
- Access control is managed via optional mTLS on the gRPC channel

Secrets:
- mysql-config (Opaque) - MySQL connection credentials (host, port, database, user,
  password). Provisioned by: data-science-pipelines-operator. Auto-rotate: No
- metadata-grpc-tls (kubernetes.io/tls, optional) - Server TLS cert, key, CA for
  gRPC SSL. Provisioned by: Cluster admin / cert-manager. Auto-rotate: Depends
- mysql-ssl-certs (Opaque, optional) - MySQL client SSL cert, key, CA, cipher config.
  Provisioned by: Cluster admin. Auto-rotate: No
- postgres-ssl-certs (Opaque, optional) - PostgreSQL client SSL cert, key, password,
  root cert. Provisioned by: Cluster admin. Auto-rotate: No

Authentication & Authorization:
- MetadataStoreService gRPC (all 40+ RPCs):
  - Optional mTLS via SSLConfig.client_verify
    If client_verify=true: requires valid client certificate signed by custom_ca
  - Default: InsecureServerCredentials (no TLS, no auth)
  - No built-in RBAC or fine-grained authorization on individual RPCs

Container Security:
- Run as user: 65534 (nobody)
- Run as group: 65534 (nobody)
- Base image: registry.access.redhat.com/ubi9/ubi-minimal
- Build system: Konflux (Dockerfile.konflux)

Network Exposure:
- NO external ingress (internal-only service)
- ClusterIP service on port 8080/TCP
- Egress to database backends (MySQL 3306/TCP, PostgreSQL 5432/TCP)
