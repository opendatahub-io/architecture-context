NETWORK EXPOSURE:
- Ingress: NONE - Internal ClusterIP service only, no external access
- Egress: MySQL/MariaDB (3306/TCP), PostgreSQL (5432/TCP)
- The MLMD gRPC server has NO external ingress. It is accessed only by
  other components within the cluster (primarily Data Science Pipelines).

RBAC Summary:
- No RBAC resources defined in this repository
- RBAC is managed externally by the data-science-pipelines-operator
- The MLMD server itself has no built-in authorization mechanism

Secrets:
- Database credentials (Opaque) - MySQL/PostgreSQL username and password
  Provisioned by: DSP operator / cluster admin | Auto-Rotate: No
- TLS certificates (kubernetes.io/tls) - Server key, cert, CA for gRPC
  Provisioned by: Manual / cert-manager | Auto-Rotate: Varies
- MySQL SSL certificates (Opaque) - Client cert, key, CA, cipher config
  Provisioned by: Manual | Auto-Rotate: No
- PostgreSQL SSL certificates (Opaque) - sslcert, sslkey, sslpassword, sslrootcert
  Provisioned by: Manual | Auto-Rotate: No

Authentication & Authorization:
- MetadataStoreService (all RPCs):
  - Optional mTLS via SSLConfig with client_verify=true
  - If enabled, client must present certificate signed by custom CA
  - No application-level token/RBAC auth
  - Access control relies on network-level isolation or mTLS

FIPS Compliance Notes:
- C++ binary built with Bazel on UBI9 (not Go - no GOEXPERIMENT flags)
- BoringSSL bundled via Bazel for gRPC TLS (FIPS module NOT explicitly enabled)
- System OpenSSL (FIPS-validated) linked at build time on UBI9
- Actual FIPS compliance depends on which SSL library is used at link time
- Konflux build is NOT hermetic (hermetic: false)

Container Security:
- Runtime image: UBI9 ubi-minimal
- Runs as non-root (UID 65534)
- Dynamically linked C++ binary
