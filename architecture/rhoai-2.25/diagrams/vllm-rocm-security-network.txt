RBAC Summary:
- No RBAC resources defined in this component (container image, not an operator)
- Pod runs under ServiceAccount assigned by KServe ServingRuntime
- kube-rbac-proxy sidecar performs SubjectAccessReview for incoming requests

Authentication & Authorization:
- All HTTP/gRPC endpoints: Platform-level auth
  - Enforcement Point: kube-rbac-proxy sidecar or Istio mesh
  - AuthN: Bearer Token validation (kube-rbac-proxy) or mTLS (Istio)
  - AuthZ: Kubernetes SubjectAccessReview (kube-rbac-proxy) or AuthorizationPolicy (Istio)
  - vLLM process itself does NOT implement authentication

Secrets:
- No secrets defined in this repository
- Model serving may require secrets for model storage credentials (provisioned by platform/user)
- S3 credentials: Typically provided via environment variables or mounted secrets
- Hugging Face token: Optional, provided via HF_TOKEN environment variable

FIPS Compliance:
- FIPS compliance inherited entirely from RHAIIS base image (registry.redhat.io/rhaiis/vllm-rocm-rhel9)
- Base image uses UBI9 with OpenSSL and system FIPS mode integration
- This repo adds no binaries or crypto libraries

Network Ports Summary:
- 8443/TCP: kube-rbac-proxy (HTTPS, TLS 1.2+, external-facing)
- 8000/TCP: vLLM HTTP server (HTTP, plaintext, localhost/in-cluster only)
- 8033/TCP: TGIS gRPC adapter (gRPC, platform-level encryption)
- 443/TCP:  Egress to S3 storage and Hugging Face Hub (HTTPS, TLS 1.2+)
