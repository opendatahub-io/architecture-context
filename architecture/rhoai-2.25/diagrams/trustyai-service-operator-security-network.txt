Authentication & Authorization Flows:
┌─────────────────────────────────────────────────────────────────────────┐
│ TrustyAI Service Access (via Route)                                     │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. Client → OpenShift Route (HTTPS/443)                                │
│ 2. Route → OAuth Proxy Sidecar (HTTPS/8443, TLS reencrypt)             │
│ 3. OAuth Proxy checks Bearer Token:                                     │
│    - If missing: Redirect to OpenShift OAuth server                     │
│    - If present: Validate via OpenShift API                             │
│ 4. OAuth Proxy performs SubjectAccessReview:                            │
│    - Resource: pods                                                      │
│    - Verb: get                                                           │
│    - Namespace: {user-namespace}                                         │
│ 5. If authorized: Forward to TrustyAI Service (HTTP/8080 or HTTPS/4443)│
│ 6. TrustyAI Service processes request (no additional auth)              │
├─────────────────────────────────────────────────────────────────────────┤
│ Guardrails Orchestrator Access                                          │
├─────────────────────────────────────────────────────────────────────────┤
│ - Health endpoint (/apis/v1beta1/healthz): Bypass OAuth (skip-auth)    │
│ - All other endpoints: Same OAuth flow as TrustyAI Service              │
├─────────────────────────────────────────────────────────────────────────┤
│ Operator Metrics Access                                                 │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. Prometheus → kube-rbac-proxy (HTTPS/8443)                           │
│ 2. kube-rbac-proxy validates Kubernetes Bearer Token                    │
│ 3. kube-rbac-proxy performs RBAC check:                                 │
│    - Subject: ServiceAccount (prometheus)                               │
│    - Verb: get                                                           │
│    - Resource: /metrics endpoint                                        │
│ 4. If authorized: Forward to operator metrics (HTTP/8080 internal)     │
├─────────────────────────────────────────────────────────────────────────┤
│ LMEvalJob Creation (Non-Admin Users)                                    │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. User → Kubernetes API (kubectl apply -f lmevaljob.yaml)             │
│ 2. API Server checks RBAC:                                              │
│    - ClusterRole: non-admin-lmeval-role                                 │
│    - ClusterRoleBinding: default-lmeval-user-rolebinding               │
│    - Subjects: system:authenticated (all authenticated users)           │
│ 3. If authorized: Create LMEvalJob CR                                   │
│ 4. Operator watches CR and creates Job resources                        │
└─────────────────────────────────────────────────────────────────────────┘
