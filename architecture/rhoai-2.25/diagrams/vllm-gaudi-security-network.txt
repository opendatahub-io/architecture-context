  - Only available when dev mode is explicitly enabled

FIPS Compliance:
- CRITICAL: openssl-fips-provider-so is explicitly REMOVED from image
  (Dockerfile.konflux.gaudi line 62: rpm -e --nodeps openssl-fips-provider-so || true)
- Reason: Habana SynapseAI runtime incompatible with FIPS-mode OpenSSL
- Impact: Container CANNOT operate in FIPS-compliant mode
- Python uses system OpenSSL from RHEL UBI but without FIPS provider
