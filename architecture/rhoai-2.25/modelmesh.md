## Purpose
**Short**: ModelMesh is a distributed LRU cache and routing layer for high-scale, high-density model serving on Kubernetes.

**Detailed**: ModelMesh is a mature, general-purpose model serving management and routing framework designed for high-scale, high-density, and frequently-changing model use cases. It acts as a sidecar to external model runtime containers, coupling with them via an internal gRPC service interface (ModelRuntime) to manage model loading, unloading, and inference routing.

The framework implements a distributed LRU cache that intelligently manages which models are loaded across a cluster of pods. It handles model registration, placement optimization, and transparent routing of inference requests to the correct pod where a model is currently loaded. Models are treated as immutable entities, with VModels providing a layer of indirection for model versioning and blue-green transitions.

ModelMesh uses etcd (or ZooKeeper) as a distributed key-value store for coordination, model registry state, leader election, and instance tracking. It exposes both a management gRPC API (for model lifecycle operations) and transparently proxies arbitrary inference gRPC methods to the colocated model runtime containers. Within the RHOAI platform, ModelMesh is deployed as part of the modelmesh-serving stack and is managed by the ModelMesh Serving controller and KServe.

### SSRF Protection

The `RemotePayloadProcessor` validates URIs to prevent Server-Side Request Forgery (SSRF) attacks by rejecting requests to private/internal network addresses. Only HTTP and HTTPS schemes are allowed, and localhost/loopback/private IP ranges are blocked.
