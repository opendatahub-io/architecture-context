## Purpose
**Short**: KServe is a Kubernetes-native platform for serving machine learning models with support for serverless inference, raw deployment, and LLM-optimized serving via the Gateway API Inference Extension.

**Detailed**: KServe provides a standardized, cloud-native way to deploy, manage, and scale machine learning models on Kubernetes. It defines Custom Resource Definitions (CRDs) such as InferenceService, ServingRuntime, InferenceGraph, and LLMInferenceService that abstract the complexity of model serving infrastructure. The KServe controller manager watches these CRDs and reconciles the desired state by creating and managing Kubernetes Deployments, Services, Knative Services (for serverless mode), Istio VirtualServices, Gateway API HTTPRoutes, and other supporting resources.

In the RHOAI distribution, KServe is deployed by the rhods-operator using kustomize overlays. The ODH overlay configures OpenShift-specific features including OpenShift certificate injection for webhooks, SecurityContextConstraints for LLM workloads, and integration with Istio service mesh for ingress routing. The controller supports multiple deployment modes: Serverless (via Knative Serving), RawDeployment (via standard Kubernetes Deployments with HTTPRoute or Ingress), and ModelMesh (delegating to ModelMesh controllers). A newer LLMInferenceService controller provides advanced LLM-specific capabilities including inference scheduling via Gateway API Inference Extension (InferencePool, InferenceModel), multi-node serving via LeaderWorkerSet, and prefill/decode disaggregation patterns.

**Ingress Mode Selection**: The controller selects ingress based on `inferenceservice-config` ConfigMap:
- **Serverless mode** (default in RHOAI): Creates Istio VirtualService + ExternalName Service pointing to Knative local gateway
- **RawDeployment + Gateway API** (`enableGatewayApi: true`): Creates HTTPRoute CRs referencing `kserveIngressGateway` (default: `openshift-ingress/openshift-ai-inference`)
- **RawDeployment + Ingress**: Creates Kubernetes Ingress resources
- **RHOAI ODH overlay default**: `disableIngressCreation: true` with `enableGatewayApi: false` (odh-model-controller manages Routes)
