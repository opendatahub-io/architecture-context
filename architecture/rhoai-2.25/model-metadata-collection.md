## Purpose
**Short**: A Go-based build-time tool that extracts, enriches, and catalogs metadata from Red Hat AI container images (ModelCars) and HuggingFace, producing a static YAML catalog shipped in a minimal container image.

**Detailed**: The model-metadata-collection component is a data pipeline tool, not a runtime service. It runs during CI/CD (Konflux build) to discover AI models from HuggingFace collections, extract model cards from OCI container image layers (ModelCar format), enrich metadata with data from the HuggingFace API, and generate an aggregated `models-catalog.yaml` file. This catalog describes all Red Hat AI validated models with their names, providers, descriptions, licenses, languages, tasks, tags, and OCI artifact references.

The final container image (`odh-model-metadata-collection-rhel9`) is a minimal UBI9-micro image containing only the pre-generated `models-catalog.yaml` and `models-index.yaml` files. It exposes a `/app/data` volume mount point and runs `sleep infinity` so other RHOAI components (such as the ODH Dashboard) can mount or copy the catalog data. The image contains no running services, no exposed ports, and no network endpoints at runtime. It is purely a data-bearing container image.

The tool supports concurrent processing of multiple models, intelligent model family matching to prevent cross-family enrichment errors, tag merging with deduplication, metadata format migration from legacy schemas, and comprehensive metadata completeness reporting.

## Data Processing Pipeline

### Priority Hierarchy (highest to lowest)
1. **Modelcard.md YAML frontmatter** — structured data embedded in container image model cards
2. **HuggingFace YAML frontmatter** — from model README on HuggingFace
3. **Modelcard.md regex extraction** — text parsing from model card markdown
4. **HuggingFace API** — model details endpoint
5. **HuggingFace tags** — parsed from repository tags
6. **Registry metadata** — OCI manifest timestamps and annotations
7. **Generated defaults** — fallback descriptions and license URLs

### Model Index
The `data/models-index.yaml` contains 42 OCI model references from `registry.redhat.io/rhelai1/modelcar-*` covering:
- Llama 4 (Scout, Maverick)
- Llama 3.3 / 3.1
- Mistral Small 3.1 / 24B
- Granite 3.1
- Phi 4
- Qwen 2.5 / Qwen 3
- Gemma 2 / 3n
- Mixtral 8x7B
- DeepSeek R1
- Kimi K2

Models include full-precision, FP8 dynamic, w4a16 quantized, and w8a8 quantized variants, all labeled as "validated" with some additionally marked "featured".
