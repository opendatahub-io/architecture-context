## Purpose
**Short**: Central build configuration repository that defines the RHOAI operator bundle, OLM catalog (FBC), image registries, and release artifacts for Red Hat OpenShift AI 2.25.

**Detailed**: RHOAI-Build-Config is not a runtime component; it is the meta-repository that orchestrates the build, packaging, and release of the entire Red Hat OpenShift AI (RHOAI) product. It defines the OLM (Operator Lifecycle Manager) operator bundle for the `rhods-operator`, including the ClusterServiceVersion (CSV) that specifies all 76+ container images shipped as RELATED_IMAGE environment variables, CRD manifests for all RHOAI-managed custom resources, and RBAC permissions needed by the operator.

The repository also maintains the File-Based Catalog (FBC) fragments for each supported OpenShift Container Platform version (v4.16 through v4.20), enabling the operator to be discovered and installed via the OperatorHub. It contains Tekton pipeline definitions (via Konflux/App Studio) for building the operator bundle and FBC fragment images, GitHub Actions workflows for processing bundle patches and catalog updates, and release configuration files that track production and staging releases with CVE and issue references. Additionally, it includes PNC (Project Newcastle) build configurations for Java-based sub-components (ModelMesh, TrustyAI) and ImageDigestMirrorSet definitions for disconnected/mirror registry support.
