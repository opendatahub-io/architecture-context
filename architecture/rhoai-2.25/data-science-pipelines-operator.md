## Purpose
**Short**: Manages the lifecycle of Data Science Pipelines (Kubeflow Pipelines v2) infrastructure on OpenShift via the DataSciencePipelinesApplication (DSPA) custom resource.

**Detailed**: The Data Science Pipelines Operator (DSPO) is a Kubernetes operator built with controller-runtime that watches `DataSciencePipelinesApplication` (DSPA) custom resources and reconciles the complete infrastructure required to run Kubeflow Pipelines v2 on OpenShift. For each DSPA CR, the operator dynamically deploys and manages a full pipeline stack including: an API server for pipeline management, a persistence agent for recording pipeline run metadata, a scheduled workflow controller for cron-based pipelines, an Argo Workflow controller for pipeline execution orchestration, an ML Metadata (MLMD) gRPC server with Envoy proxy for artifact lineage tracking, and backing storage infrastructure (MariaDB for metadata, Minio for artifact storage).

The operator uses Go templates (located in `config/internal/`) to render Kubernetes manifests at reconciliation time, injecting DSPA-specific parameters such as image references, database credentials, and object storage configuration. It supports both internal (MariaDB/Minio) and external (any MySQL-compatible DB / S3-compatible storage) backing services. The operator performs health checks against the database and object store before deploying pipeline components, and publishes Prometheus metrics for each component's readiness state. It also supports pod-to-pod TLS using OpenShift service-ca certificates, custom CA bundles for connecting to external services, FIPS-compliant builds, and Kubernetes-native pipeline storage via validating/mutating webhooks.

| DS Pipelines API Server | Deployment (created per DSPA) | Kubeflow Pipelines v2 API server for pipeline CRUD, run management, and artifact URLs (ports 8888/HTTP, 8887/gRPC) |
| OAuth Proxy Sidecar | Container (in API Server pod) | OpenShift oauth-proxy for authentication on API server Route (port 8443/HTTPS) |
| Persistence Agent | Deployment (created per DSPA) | Records pipeline run status and metadata to the database |
| Scheduled Workflow | Deployment (created per DSPA) | Manages cron-scheduled pipeline runs |
| Workflow Controller | Deployment (created per DSPA, conditionally) | Argo Workflows controller for pipeline step orchestration |
| MLMD gRPC Server | Deployment (created per DSPA) | ML Metadata gRPC server for artifact/execution lineage tracking |
| MLMD Envoy Proxy | Deployment (created per DSPA) | Envoy proxy fronting MLMD gRPC with OAuth proxy sidecar for external access (ports 9090, 8443) |
| MariaDB | Deployment (created per DSPA, optional) | Default database backend for pipeline metadata storage (port 3306) |
| Minio | Deployment (created per DSPA, optional) | Default S3-compatible object storage for pipeline artifacts (port 9000) |
| DS Pipelines Webhook | Deployment (in DSPO namespace, conditional) | Validates/mutates Pipeline and PipelineVersion CRs when pipelineStore=kubernetes (port 8443) |
