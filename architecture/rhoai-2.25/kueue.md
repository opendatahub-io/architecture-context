## Purpose
**Short**: Kueue is a Kubernetes-native job queueing system that manages when workloads should be admitted to start based on quota and resource availability.

**Detailed**: Kueue provides a set of APIs and controllers for job-level resource management in Kubernetes clusters. It acts as a job-level manager that decides when a job should be admitted to start (allowing pod creation) and when it should stop (deleting active pods). The system introduces concepts like ClusterQueues (cluster-scoped resource pools with quotas), LocalQueues (namespace-scoped entry points for workloads), ResourceFlavors (different types of resources like GPU types or node pools), and Workloads (abstract representations of jobs requesting resources).

In the RHOAI distribution, Kueue is deployed as a component managed by the rhods-operator via kustomize overlays. It integrates with multiple job frameworks including Kubernetes batch Jobs, Kubeflow training jobs (PyTorchJob, TFJob, MPIJob, PaddleJob, XGBoostJob), Ray jobs/clusters, JobSets, AppWrappers (CodeFlare), and more. The controller watches these job types through admission webhooks and creates corresponding Workload resources to manage their lifecycle through the queueing system. It provides fair sharing, preemption, lending limits, and topology-aware scheduling capabilities. The RHOAI overlay explicitly disables VisibilityOnDemand and MultiKueue features, and enables LocalQueueMetrics.
