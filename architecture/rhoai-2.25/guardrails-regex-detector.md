## Purpose
**Short**: A lightweight regex-based text detection service for identifying PII (Personally Identifiable Information) patterns in text content, used as a guardrails detector within the RHOAI AI safety pipeline.

**Detailed**: The guardrails-regex-detector is a Rust-based HTTP microservice that scans text content for sensitive information using regular expression pattern matching. It supports both built-in detection patterns (email addresses, social security numbers, credit card numbers) and user-supplied custom regex patterns. The service returns structured detection results including match positions, matched text, detection type classification, and confidence scores.

This detector is designed to integrate with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator), which coordinates multiple guardrails detectors to ensure AI model outputs comply with safety and privacy policies. The regex detector serves as one of potentially many detector backends that the orchestrator invokes to scan LLM input/output for prohibited or sensitive content. By providing fast, regex-based PII detection, it enables real-time content filtering as part of the RHOAI guardrails infrastructure.

The service is built on the Axum web framework with Tokio async runtime, providing a high-performance, low-overhead detection endpoint. It runs as a standalone container image based on Red Hat UBI 9 minimal.
