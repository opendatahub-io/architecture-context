## Purpose
**Short**: Kubernetes operator that manages the lifecycle of Feast feature store deployments on OpenShift.

**Detailed**: The Feast Operator automates the deployment and management of Feast feature store instances on Kubernetes and OpenShift clusters. Feast is an open-source feature store for machine learning that helps teams manage, serve, and discover features for ML models.

The operator watches `FeatureStore` custom resources and reconciles the desired state by dynamically creating and managing Deployments, Services, PersistentVolumeClaims, ConfigMaps, ServiceAccounts, RBAC resources, CronJobs, and OpenShift Routes. It supports multiple service types within a single FeatureStore deployment: an online store (feature server), offline store, registry (gRPC and/or REST), and UI server -- all deployed as containers within a single Deployment.

The operator provides built-in support for OpenShift service serving certificates (automatic TLS), custom CA bundles (for disconnected environments), Kubernetes RBAC-based authorization, OIDC-based authorization, persistent storage via PVCs or database backends, a namespace registry ConfigMap for cross-namespace feature store discovery, and CronJob-based materialization workflows. It is built with FIPS-compliant Go runtime when deployed in RHOAI.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| FeatureStore Controller | Go Operator (controller-runtime) | Reconciles FeatureStore CRs; dynamically creates Deployments, Services, ConfigMaps, PVCs, ServiceAccounts, Roles, RoleBindings, CronJobs, and OpenShift Routes |
| Feature Server (Online Store) | Python Application Container | Serves online features via HTTP (port 6566/TCP) or HTTPS (port 6567/TCP) |
| Offline Store Server | Python Application Container | Serves offline features via gRPC (port 8815/TCP) or gRPC+TLS (port 8816/TCP) |
| Registry Server | Python Application Container | Feature registry via gRPC (port 6570/TCP or 6571/TCP with TLS) and/or REST API (port 6572/TCP or 6573/TCP with TLS) |
| UI Server | Python Application Container | Web UI for browsing feature store (port 8888/TCP or 8443/TCP with TLS) |
| CronJob | Kubernetes CronJob | Executes feast apply and materialization commands against the FeatureStore deployment |
| Authorization Module | Go Reconciler Logic | Deploys Kubernetes RBAC Roles/RoleBindings for Feast permission model; supports OIDC auth via secrets |
| Namespace Registry | ConfigMap-based Registry | Cross-namespace ConfigMap (`feast-configs-registry`) tracking all deployed FeatureStore instances |
| Client ConfigMap | Kubernetes ConfigMap | Contains `feature_store.yaml` client configuration for consumers of the feature store |
