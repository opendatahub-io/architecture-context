## Purpose
**Short**: Caikit-TGIS-Serving is a model serving runtime container that enables Large Language Model (LLM) inference using the Caikit AI toolkit with the Text Generation Inference Server (TGIS) backend.

**Detailed**: Caikit-TGIS-Serving provides a container image that packages the Caikit AI framework together with the caikit-nlp library and the caikit-tgis-backend connector. It acts as a "transformer" container in a multi-container KServe ServingRuntime pod, where the TGIS container loads and runs the actual model while the Caikit container provides standardized HTTP and gRPC API interfaces for inference requests.

The container is designed to be deployed via KServe InferenceService custom resources on OpenShift, running alongside the TGIS inference engine container. The Caikit runtime connects to the local TGIS instance on port 8033 via gRPC to delegate actual model inference, while exposing user-facing HTTP (port 8080) and gRPC (port 8085) endpoints. The stack integrates with OpenShift Service Mesh (Istio) for mTLS and traffic management, and OpenShift Serverless (Knative) for autoscaling and routing.

This repository does not contain operator code or controllers — it is purely a container image build repository. The deployment is orchestrated by KServe, which manages the ServingRuntime and InferenceService lifecycle.
