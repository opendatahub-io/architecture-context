## Purpose
**Short**: ODH Notebook Controller extends the upstream Kubeflow Notebook Controller with OpenShift-specific features including OAuth proxy injection, TLS routes, network policies, certificate management, and Elyra pipeline integration.

**Detailed**: The ODH Notebook Controller is a Go-based Kubernetes controller and mutating admission webhook that watches `Notebook` custom resources (CRD defined by the upstream Kubeflow project at `kubeflow.org/v1`) and reconciles OpenShift-specific infrastructure for each notebook workbench instance.

On each Notebook reconciliation, the controller dynamically creates: OpenShift Routes (with TLS edge termination or re-encrypt for OAuth), Services for OAuth proxy traffic, ServiceAccounts with OAuth redirect references, Secrets for OAuth cookie and client credentials, OAuthClient cluster-scoped resources for OpenShift OAuth integration, NetworkPolicies to secure notebook pod traffic, ConfigMaps for trusted CA certificate bundles and pipeline runtime images, and RoleBindings for Data Science Pipeline access.

The mutating webhook intercepts Notebook create/update operations and injects: an OAuth proxy sidecar container (listening on port 8443/TCP with TLS), CA certificate bundle volume mounts, pipeline runtime image ConfigMap mounts, Elyra DSP configuration secret mounts, cluster-wide proxy environment variables, reconciliation lock annotations, and ImageStream-based container image resolution. The webhook also implements a "pending updates" mechanism that blocks webhook-initiated pod template changes on running notebooks to prevent unintended restarts.

|------|------|-------|------|----------|------------|----------|----------|

**Controller-managed ingress (RHOAI 2.x pattern)**:
- For notebooks with `notebooks.opendatahub.io/inject-oauth: "true"` and service mesh disabled:
  - Creates OpenShift Route with TLS re-encrypt termination pointing to `{notebook-name}-tls` Service (port 443 -> oauth-proxy port 8443)
  - OAuth proxy sidecar (8443/TCP HTTPS) authenticates via OpenShift OAuth, proxies to notebook container (localhost:8888 HTTP)
  - Creates OAuthClient (cluster-scoped) with auto-generated client secret and route-based redirect URI
- For notebooks without OAuth injection:
  - Creates OpenShift Route with TLS edge termination pointing to `{notebook-name}` Service (port http-{notebook-name} -> 8888)
- Route subdomain max length is 63 characters; uses `generateName: "nb-"` if notebook name + namespace exceeds limit
