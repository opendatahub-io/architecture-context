## Purpose
**Short**: Kubernetes controller that automates model serving infrastructure management for KServe and ModelMesh InferenceServices in OpenShift AI.

**Detailed**: The odh-model-controller is a Kubernetes operator built with controller-runtime that bridges the gap between user-created model serving resources (InferenceService, ServingRuntime, InferenceGraph, LLMInferenceService) and the underlying platform infrastructure required to expose and secure them. It automatically manages OpenShift Routes, Istio Gateway configurations, service mesh membership, network policies, monitoring resources (ServiceMonitors, PodMonitors), AuthConfig/AuthPolicy for authorization, and certificate management for TLS.

The controller supports three deployment modes for InferenceServices: Serverless (KServe with Knative + Istio service mesh), RawDeployment (KServe without Knative), and ModelMesh. For each mode, different sub-resource reconcilers handle the creation of Routes, ClusterRoleBindings, ServiceAccounts, metrics services, KEDA autoscaling resources, and network policies. Additionally, the controller manages NVIDIA NIM integration through the Account CRD, handling API key validation, model catalog ConfigMap generation, pull secret management, and OpenShift Template creation for NIM ServingRuntimes.

The controller also handles cross-cutting concerns: storage-config Secret generation from data connections, CA certificate bundle management for KServe, Ray TLS certificate management for multi-node serving, and Model Registry integration for InferenceService metadata synchronization.

### Dynamically Created Network Resources

The controller creates per-InferenceService resources depending on deployment mode:

**KServe Serverless mode** creates:
- ServiceMeshMember (enrolls ISVC namespace in service mesh)
- Istio Gateway server entry (per ISVC, port 8445/HTTPS with serving cert)
- OpenShift Route (points to istio-ingressgateway Service)
- Istio PeerAuthentication (mTLS enforcement)
- Istio Telemetry (metrics configuration)
- AuthConfig (Authorino authorization, when mesh auth enabled)
- NetworkPolicy (per namespace)
- ServiceMonitor and PodMonitor (metrics collection)
- Metrics Service and ServiceMonitor (dedicated metrics endpoint)

**KServe RawDeployment mode** creates:
- OpenShift Route (points to predictor/transformer Service)
- ClusterRoleBinding (for service account)
- Metrics Service and ServiceMonitor
- KEDA TriggerAuthentication, Roles, RoleBindings (autoscaling)
- Metrics Dashboard ConfigMap

**ModelMesh mode** creates:
- OpenShift Route (points to modelmesh-serving Service)
- ServiceAccount
- ClusterRoleBinding
