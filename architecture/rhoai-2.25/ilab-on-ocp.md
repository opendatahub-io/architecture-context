## Purpose
**Short**: Provides the InstructLab LLM fine-tuning pipeline for Red Hat OpenShift AI, orchestrating synthetic data generation, distributed training, and model evaluation via Data Science Pipelines.

**Detailed**: The `ilab-on-ocp` repository is the central location for running the [InstructLab](https://instructlab.ai/) large language model (LLM) fine-tuning workflow on OpenShift with Red Hat OpenShift AI (RHOAI). It packages a multi-stage Kubeflow Pipelines v2 (KFP) pipeline that orchestrates the complete InstructLab workflow: prerequisites validation, synthetic data generation (SDG), two-phase distributed training via PyTorchJob, model evaluation (MT-Bench and MMLU), and model output/registration.

The repository produces a single container image (`odh-ml-pipelines-runtime-generic`) that serves as the generic runtime for pipeline tasks and also embeds the compiled pipeline definition. The pipeline is automatically provisioned by the DataSciencePipelinesApplication (DSPA) resource when InstructLab is enabled. At runtime, pipeline tasks execute as pods within a user's Data Science project namespace, leveraging GPU-enabled nodes for SDG, training, and evaluation. Training is performed using Kubeflow Training Operator's PyTorchJob CRs for multi-node, multi-GPU distributed training with FSDP. Supplementary manifests are provided for deploying Teacher (Mixtral) and Judge (Prometheus) model servers via KServe InferenceService resources.

The pipeline supports disconnected cluster environments, OCI model push, and Model Registry integration for tracking trained model artifacts.
