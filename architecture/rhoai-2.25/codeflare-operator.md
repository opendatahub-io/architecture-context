## Purpose
**Short**: Manages RayCluster lifecycle with OAuth proxy injection, mTLS certificate management, and optional embedded AppWrapper controller for batch workload scheduling.

**Detailed**: The CodeFlare Operator is a Kubernetes operator built with controller-runtime that watches `RayCluster` custom resources (from the KubeRay project) and automatically provisions supporting infrastructure for secure access to Ray dashboards and Ray client endpoints. On OpenShift, it creates OAuth proxy sidecars, OpenShift Routes, ServiceAccounts, and ClusterRoleBindings to enable authenticated access to the Ray dashboard. On vanilla Kubernetes, it creates Ingress resources instead. The operator also manages mTLS certificates for inter-node communication within Ray clusters, generating CA certificates and injecting init containers that create per-node TLS certificates.

Additionally, the operator can optionally embed the AppWrapper controller (from the `project-codeflare/appwrapper` project), which provides a Kueue-integrated workload management layer. When enabled, AppWrappers can wrap Kubernetes resources (RayClusters, PyTorchJobs, Jobs, etc.) to provide gang scheduling and quota management. The operator deploys both mutating and validating webhooks for RayCluster and AppWrapper resources.

The operator is deployed by the rhods-operator using kustomize overlays from the `config/default/` directory, with the container image reference injected at deploy time via the `RELATED_IMAGE_ODH_CODEFLARE_OPERATOR_IMAGE` environment variable.
