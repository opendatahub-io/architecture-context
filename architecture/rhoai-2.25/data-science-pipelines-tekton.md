## Purpose
**Short**: Kubeflow Pipelines backend using Tekton as the workflow execution engine for ML pipeline orchestration on OpenShift.

**Detailed**: Data Science Pipelines Tekton (DSP-Tekton) is a fork of Kubeflow Pipelines that replaces the Argo Workflows backend with Tekton Pipelines as the workflow execution engine. It provides a complete ML pipeline platform including an API server for pipeline management (CRUD operations on pipelines, runs, experiments, jobs), a frontend UI for pipeline visualization and management, scheduled workflow controllers for recurring pipeline runs, a persistence agent for syncing pipeline run status to the backend database, and integration with ML Metadata (MLMD) for artifact and execution tracking.

The component is deployed as part of Red Hat OpenShift AI (RHOAI) and provides the data science pipelines capability. It translates Kubeflow Pipelines DSL-compiled pipeline definitions into Tekton PipelineRun resources, manages their lifecycle, tracks artifacts in object storage (MinIO/S3-compatible), and stores metadata in a relational database (MariaDB/MySQL/PostgreSQL). The UI overlay uses oauth-proxy for OpenShift authentication, exposed via an OpenShift Route with TLS re-encryption.

| metadata-envoy | Envoy Proxy | Envoy sidecar proxy fronting the MLMD gRPC server; serves on port 9090 |
| metadata-writer | Python Service | Writes pipeline execution metadata to MLMD from Tekton PipelineRun annotations |
| cache-server | Go Webhook | Mutating admission webhook for pipeline step caching; serves on port 8443 with TLS |
