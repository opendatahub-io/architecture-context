## Purpose
**Short**: OpenVINO Model Server is a high-performance C++ inference server that serves AI/ML models via gRPC and REST APIs, compatible with TensorFlow Serving, KServe v2, and OpenAI v3 protocols.

**Detailed**: OpenVINO Model Server (OVMS) is a high-performance model serving system implemented in C++ and optimized for deployment on Intel architectures. It provides inference services through multiple API protocols: TensorFlow Serving (TFS) API for backward compatibility, KServe v2 Inference Protocol for standardized model serving, and OpenAI-compatible v3 API for generative AI workloads including chat completions, embeddings, reranking, and image generation.

OVMS supports loading models from local filesystem, Amazon S3, Google Cloud Storage (GCS), Azure Blob Storage, and HuggingFace Hub. It leverages Intel's OpenVINO toolkit as the inference backend, supporting CPU, GPU, and other accelerators. The server features a modular architecture with pluggable frontend/backend components, MediaPipe graph execution for complex inference pipelines (DAGs), Python node support for custom processing, and Prometheus-compatible metrics. It supports model versioning, dynamic model updates, batch processing, and streaming inference via gRPC.

Within RHOAI, OVMS serves as one of the available model serving runtimes, deployed as a container alongside model serving infrastructure managed by KServe or ModelMesh. It provides the actual inference execution layer that processes prediction requests from clients.
