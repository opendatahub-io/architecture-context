## Purpose
**Short**: CPU-optimized build of vLLM providing high-throughput LLM inference serving with OpenAI-compatible and gRPC APIs, without requiring GPU hardware.

**Detailed**: vLLM-CPU is a Red Hat downstream fork of the vLLM project, specifically built for CPU-only inference workloads. It provides a high-throughput, memory-efficient inference and serving engine for Large Language Models (LLMs) using PagedAttention for efficient key-value cache management, continuous batching of incoming requests, and speculative decoding.

The container image ships two entrypoints: an OpenAI-compatible REST API server (default, port 8000) and a gRPC adapter via vllm-tgis-adapter (port 8033) for integration with the TGIS (Text Generation Inference Server) protocol used in RHOAI's model serving stack. The image is built for multiple CPU architectures including x86_64, ppc64le, and s390x, with architecture-specific optimizations (Intel Extension for PyTorch on x86_64, OpenBLAS on ppc64le/s390x).

Within RHOAI, this image is deployed as a model serving runtime by KServe or ModelMesh, allowing users to serve LLMs on CPU nodes. It supports features such as multi-LoRA serving, streaming outputs, quantized models (INT4, INT8), multimodal inputs, and OpenTelemetry tracing.
