## Purpose
**Short**: AMD ROCm-accelerated vLLM inference runtime container image for serving large language models on AMD GPUs within RHOAI.

**Detailed**: This repository is a thin Konflux build wrapper that produces the `odh-vllm-rocm-rhel9` container image for Red Hat OpenShift AI (RHOAI). It does not contain application source code itself; instead, it layers a custom entrypoint and environment configuration on top of the upstream Red Hat AI Inference Server (RHAIIS) vLLM ROCm base image (`registry.redhat.io/rhaiis/vllm-rocm-rhel9`).

The resulting container image provides a GPU-accelerated LLM inference server using AMD ROCm (HIP), exposing both an OpenAI-compatible HTTP API (port 8000) and a gRPC endpoint via the vllm-tgis-adapter (port 8033). It is designed to be deployed as a serving runtime within KServe or ModelMesh on OpenShift, enabling high-performance inference with features such as PagedAttention, continuous batching, speculative decoding, tensor/pipeline parallelism, and quantization (GPTQ, AWQ, INT4, INT8, FP8).

The build is managed entirely through Konflux CI/CD pipelines defined in the `.tekton/` directory, which are synchronized from the central `konflux-central` repository. The image is published to `quay.io/rhoai/odh-vllm-rocm-rhel9`.
