## Purpose
**Short**: vLLM hardware plugin for Intel Gaudi (Habana) accelerators, enabling optimized LLM inference on HPU devices within RHOAI.

**Detailed**: The vllm-gaudi repository provides a hardware plugin that integrates Intel Gaudi (Habana) AI accelerators with the vLLM inference engine. It implements the vLLM plugin architecture ([RFC: Hardware pluggable](https://github.com/vllm-project/vllm/issues/11162)) to provide HPU-specific attention backends, custom operators, model implementations, distributed communication, and KV cache management optimized for Intel Gaudi 2 and Gaudi 3 hardware.

The plugin registers itself via Python entry points (`vllm.platform_plugins` and `vllm.general_plugins`) and overrides vLLM's default GPU-oriented codepaths with Habana-optimized implementations. It supports quantization methods (FP8, INC, compressed-tensors, AWQ, GPTQ, ModelOpt), speculative decoding (EAGLE), LoRA adapters, multimodal models (Gemma3, Qwen2.5-VL, Qwen3-VL, Pixtral, Ovis), and distributed inference via tensor parallelism with HCCL. The container image ships as a standalone OpenAI-compatible API server built on UBI9 with Habana SynapseAI 1.23.0 and PyTorch 2.9.0.
