## Purpose
**Short**: ML Metadata (MLMD) is a gRPC server and library for recording and retrieving metadata associated with ML workflows, tracking artifacts, executions, and their lineage.

**Detailed**: ML Metadata is a Google-originated library and server that provides a metadata store for machine learning pipelines. It records information about artifacts (datasets, models, metrics), executions (training runs, preprocessing steps), and contexts (experiments, pipelines) along with the events and relationships between them. This enables full lineage tracking of ML artifacts through their lifecycle.

In the RHOAI platform, MLMD serves as the metadata backbone for Data Science Pipelines (based on Kubeflow Pipelines). The MLMD gRPC server is deployed as a sidecar or companion service alongside the pipeline infrastructure, storing metadata in either MySQL or PostgreSQL backends. It exposes a gRPC API (via protobuf service definitions) that pipeline components use to record and query metadata about pipeline runs, artifacts produced, and execution lineage.

The Red Hat distribution builds the C++ gRPC server binary using Bazel and packages it into a UBI9-based container image (`odh-mlmd-grpc-server`). The server listens on port 8080 by default and supports configurable database backends (MySQL, PostgreSQL, SQLite) with optional SSL/TLS encryption for both the gRPC channel and database connections.
