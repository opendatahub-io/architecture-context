## Purpose
**Short**: ModelMesh Serving is a Kubernetes controller that manages the lifecycle of multi-model serving runtimes, enabling intelligent model placement, loading, and inference routing across a cluster using the ModelMesh framework.

**Detailed**: ModelMesh Serving acts as the control plane for the ModelMesh inference serving system within Red Hat OpenShift AI. It reconciles ServingRuntime, ClusterServingRuntime, Predictor, and InferenceService custom resources to dynamically create and manage Deployments for model serving runtimes. Each runtime Deployment includes a ModelMesh sidecar container (Java-based) that coordinates model placement and routing across replicas using etcd for state coordination.

The controller supports scale-to-zero for idle runtimes, horizontal pod autoscaling (HPA), multi-model serving where many models share a single runtime pod, and REST proxy integration for HTTP-based inference. It manages the lifecycle of etcd secrets for cross-namespace model-mesh coordination, creates Kubernetes Services for inference traffic routing, and provisions ServiceMonitor resources for Prometheus metrics collection. The system supports both namespace-scoped and cluster-scoped operation modes.

The controller also includes a validating webhook for ServingRuntime resources, ensuring that user-defined runtime specifications don't conflict with internal ModelMesh container names, ports, or volume mounts. An oauth-proxy sidecar is injected into runtime Deployments to provide authentication via OpenShift OAuth for REST API access.
