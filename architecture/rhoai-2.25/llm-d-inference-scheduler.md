## Purpose
**Short**: An inference request scheduler (Endpoint Picker / EPP) that makes optimized routing decisions for LLM inference requests in the llm-d framework using Gateway API and Envoy ext-proc.

**Detailed**: The llm-d-inference-scheduler provides an "Endpoint Picker (EPP)" component for the llm-d inference framework. It schedules incoming inference requests arriving through a Kubernetes Gateway API gateway to optimal model-serving pods based on pluggable filters and scorers. The EPP extends the upstream Gateway API Inference Extension (GIE) project with llm-d-specific features.

The scheduler operates as an Envoy external processing (ext-proc) filter. When an inference request arrives at the Envoy-based gateway, Envoy calls the EPP via gRPC to determine which backend pod should handle the request. The EPP evaluates candidate pods in an InferencePool through a pipeline of filters (e.g., by-label, decode-filter, prefill-filter) and scorers (e.g., load-aware, prefix-cache, session-affinity) to select the optimal endpoint. Key llm-d-specific features include disaggregated Prefill/Decode (P/D) scheduling, KV-cache-aware routing with real-time cache tracking via ZMQ events, and session affinity.
