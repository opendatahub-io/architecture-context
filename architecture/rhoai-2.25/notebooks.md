## Purpose
**Short**: Provides pre-built container images for data science workbenches (Jupyter, Code Server, RStudio) and Elyra pipeline runtimes used in Red Hat OpenShift AI.

**Detailed**: The notebooks repository builds and maintains a collection of container images that serve as interactive development environments (workbenches) for data scientists and ML engineers on OpenShift AI. These images are organized into two primary categories: workbench images (Jupyter notebooks, VS Code Server, RStudio) and pipeline runtime images (for Elyra pipeline execution).

Each workbench image comes in multiple variants optimized for different hardware accelerators (CPU-only, NVIDIA CUDA GPU, AMD ROCm GPU) and includes pre-installed data science libraries such as PyTorch, TensorFlow, scikit-learn, and TrustyAI. The repository maintains a versioned image lifecycle with up to 7 tagged versions per image stream (N through N-6), enabling users to pin to specific releases while receiving security updates.

The deployment mechanism uses OpenShift ImageStream resources managed via Kustomize. The rhods-operator applies the `overlays/additional` kustomization, which includes both the base ImageStreams (py3.11 versions with full version history) and additional py3.12 ImageStreams. Image references are parameterized through ConfigMap replacements, allowing the operator to inject pinned image digests from `params.env` at deployment time.

### Services

This component does not deploy Kubernetes Services directly. The notebook containers expose HTTP ports that are fronted by Services created by the ODH Notebook Controller when a Notebook CR is instantiated.

### Ingress

Ingress is managed by the ODH Notebook Controller, not by this component. When a Notebook CR is created:
- **RHOAI 3.x pattern**: The notebook controller creates an HTTPRoute (Gateway API) per notebook referencing the platform data-science-gateway, with a kube-rbac-proxy sidecar for authentication.
- **RHOAI 2.x pattern**: The notebook controller creates an OpenShift Route per notebook with an oauth-proxy sidecar for authentication.

### Authentication & Authorization

Authentication and authorization for notebook workbenches are handled externally by the ODH Notebook Controller, which injects either a kube-rbac-proxy sidecar (RHOAI 3.x) or oauth-proxy sidecar (RHOAI 2.x) in front of each notebook Pod. The notebook containers themselves do not implement authentication.

| Endpoint | Methods | Auth Mechanism | Enforcement Point | Policy |

## Kustomize Deployment Structure

### Overlay Hierarchy

```

  base/
    kustomization.yaml          # Base: all py3.11 ImageStreams + runtime ImageStreams + RStudio BuildConfigs
    params.env                  # Pinned image digests (py3.11 N-1 through N-6 versions)

    *-imagestream.yaml          # 11 notebook + 7 runtime + 2 BuildConfig definitions
  overlays/

      *-py312-ubi9-imagestream.yaml  # 9 notebook + 5 runtime py3.12-specific definitions
```

### Image Versioning Scheme

The base ImageStreams maintain up to 7 tagged versions per image:
- **N (2025.2)**: Current version, Python 3.12, `opendatahub.io/workbench-image-recommended: true`
- **N-1 (2025.1)**: Previous version, Python 3.11, `recommended: false`
- **N-2 (2024.2)**: Older, `opendatahub.io/image-tag-outdated: true`
- **N-3 through N-6**: Legacy versions (Python 3.11, 3.9, 3.8), all marked outdated

The overlay py3.12 ImageStreams have a single tag (2025.1) and are initially labeled `opendatahub.io/notebook-image: false` (not yet visible in dashboard).

### Label Convention

All deployed resources carry:
- `component.opendatahub.io/name: notebooks`
- `opendatahub.io/component: "true"`

ImageStreams are annotated with:
- `opendatahub.io/notebook-image: "true"` or `opendatahub.io/runtime-image: "true"` to indicate type
- `opendatahub.io/notebook-image-name`: Display name with format "IDE | Framework | Accelerator | Python Version"
- `opendatahub.io/notebook-image-desc`: Human-readable description
- `opendatahub.io/notebook-image-order`: Integer ordering for dashboard display
- `opendatahub.io/recommended-accelerators`: JSON array of accelerator resource names (e.g., `["nvidia.com/gpu"]`)
- `opendatahub.io/notebook-software`: JSON array of included software with versions
- `opendatahub.io/notebook-python-dependencies`: JSON array of Python libraries with versions
