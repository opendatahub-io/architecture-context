## Purpose
**Short**: Kubernetes operator that creates and manages Llama Stack server deployments for running Meta Llama AI model inference distributions.

**Detailed**: The llama-stack-k8s-operator is a Kubernetes operator built with controller-runtime and the Operator SDK framework that manages the lifecycle of Llama Stack distributions on OpenShift/Kubernetes clusters. It watches `LlamaStackDistribution` custom resources and reconciles them into fully functional Llama Stack server deployments, including Deployments, Services, NetworkPolicies, PersistentVolumeClaims, ServiceAccounts, and optional Ingress resources for external access.

The operator supports multiple Llama Stack distribution types (e.g., starter, ollama, vllm-gpu, remote-vllm) and resolves container images either from a named distribution mapping or a direct image reference. In RHOAI mode, the `RELATED_IMAGE_RH_DISTRIBUTION` environment variable overrides the embedded distributions, providing a single curated "rh-dev" distribution image. The operator also handles TLS CA bundle configuration (including auto-detection of ODH trusted CA bundle ConfigMaps), user-provided server configuration via ConfigMaps, persistent storage with PVC permission management via init containers, and OpenShift SCC (Security Context Constraints) integration via the `anyuid` SCC.

The operator uses a kustomize-based manifest rendering pipeline with custom Go plugins for name prefixing, namespace setting, field mutation, and NetworkPolicy rule generation. It watches ConfigMaps referenced by CRs and automatically triggers rolling updates when configuration changes are detected by injecting content-based hash annotations into pod templates.

## Kustomize Deployment

### RHOAI Overlay Structure

The operator is deployed by rhods-operator using the kustomize overlay at `config/overlays/rhoai/kustomization.yaml`. This overlay:

1. References `../../default` which includes `../crd`, `../rbac`, and `../manager`
2. Sets namespace to `redhat-ods-applications`
3. Applies `namePrefix: llama-stack-k8s-operator-` to all resources
4. Uses `configMapGenerator` with `params.env` to inject:
   - `RELATED_IMAGE_ODH_LLAMASTACK_OPERATOR` -> operator container image
   - `OPERATOR_VERSION` -> "0.3.0"
   - `RELATED_IMAGE_RH_DISTRIBUTION` -> Llama Stack core distribution image
5. Image parameters injected by rhods-operator at deploy time:
   - `RELATED_IMAGE_ODH_LLAMASTACK_OPERATOR` -> `RELATED_IMAGE_ODH_LLAMA_STACK_K8S_OPERATOR_IMAGE`
   - `RELATED_IMAGE_RH_DISTRIBUTION` -> `RELATED_IMAGE_ODH_LLAMA_STACK_CORE_IMAGE`

### Internal Manifest Pipeline (Controller-managed resources)

The controller embeds a kustomize manifest base at `controllers/manifests/base/` which is copied into the container image at build time. For each LlamaStackDistribution CR, the controller:

1. Runs kustomize on the embedded base manifests
2. Applies Go-based transformer plugins:
   - **NamePrefixPlugin**: Prefixes resource names with the CR instance name (excludes Deployment)
   - **NamespacePlugin**: Sets namespace to match the CR namespace
   - **FieldMutatorPlugin**: Injects instance-specific values (name, port, storage size, replicas, service account)
   - **NetworkPolicyTransformerPlugin**: Configures NetworkPolicy ingress rules based on `spec.network.allowedFrom`
3. Applies the rendered resources to the cluster via server-side apply
