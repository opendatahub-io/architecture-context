## Purpose
**Short**: Kubeflow Pipelines (KFP) provides end-to-end ML workflow orchestration on Kubernetes, enabling pipeline authoring, scheduling, execution, and artifact tracking.

**Detailed**: Data Science Pipelines is the Red Hat distribution of Kubeflow Pipelines v2, providing a platform for building, deploying, and managing machine learning workflows on OpenShift. It enables data scientists and ML engineers to define reusable ML pipelines as directed acyclic graphs (DAGs) of containerized tasks, with built-in support for experiment tracking, artifact management, caching, and scheduled recurring runs.

The system consists of several microservices: an API server (Go) that provides both gRPC (port 8887) and REST/HTTP (port 8888) interfaces for pipeline CRUD and run management; a persistence agent that synchronizes Argo Workflow execution state back to the API server; a scheduled workflow controller that manages recurring pipeline runs via CRDs; a cache server that intercepts pod creation via mutating webhooks to enable step-level caching; a metadata/MLMD gRPC service for artifact lineage tracking; a frontend UI (React/TypeScript) served on port 3000; and v2 driver/launcher binaries injected into pipeline task pods for orchestrating individual pipeline steps. Pipelines are compiled from the KFP Python SDK and executed as Argo Workflows on the cluster.

The RHOAI distribution builds all Go binaries with FIPS-compliant runtime (`GOEXPERIMENT=strictfipsruntime`) on UBI9 base images, and produces 5 container images from this repository: API server, persistence agent, scheduled workflow controller, driver, and launcher.

|-----------|------|---------|
| ml-pipeline (API Server) | Go Service | Central API server providing gRPC (8887/TCP) and REST (8888/TCP) endpoints for pipeline/run/experiment management; also serves webhook endpoints (8443/TCP) for PipelineVersion validation/mutation |
| Persistence Agent | Go Agent | Watches Argo Workflow and ScheduledWorkflow CRs, reports execution status back to the API server via gRPC/HTTP |
| Scheduled Workflow Controller | Go Controller | Reconciles ScheduledWorkflow CRDs, creates Argo Workflow CRs on cron/periodic schedules |
| Viewer Controller | Go Controller (controller-runtime) | Reconciles Viewer CRDs, creates Deployment and Service resources for TensorBoard instances |
| Cache Server | Go Webhook Server | Mutating admission webhook (8443/TCP, TLS) that intercepts pod creation to enable pipeline step caching |
| ML Pipeline UI | Node.js/React Frontend | Web UI for viewing pipelines, runs, experiments; proxies to API server, served on port 3000 |
| Visualization Server | Python Service | Generates visualizations for pipeline artifacts on port 8888 |
| Metadata Writer | Python Agent | Watches Argo Workflows and writes execution metadata to ML Metadata (MLMD) gRPC service |
| Metadata gRPC Server | gRPC Service (MLMD) | ML Metadata store for artifact lineage tracking on port 8080 |
| Metadata Envoy Proxy | Envoy Proxy | Fronts the MLMD gRPC server on port 9090, provides HTTP/gRPC transcoding |
| KFP Driver (v2) | Go Binary (injected) | Injected into pipeline task pods; orchestrates DAG/container task execution, handles input resolution |
| KFP Launcher (v2) | Go Binary (injected) | Injected into pipeline task pods; handles artifact upload/download, metadata logging. Ships both standard and FIPS-compliant binaries |
