## Purpose

**Short**: The RHOAI operator (rhods-operator) is the central lifecycle management operator for Red Hat OpenShift AI, orchestrating the deployment, configuration, and management of all RHOAI platform components on OpenShift clusters.

**Detailed**: The rhods-operator is the top-level operator for the Red Hat OpenShift AI (RHOAI) platform. It manages the full lifecycle of data science and AI/ML components on OpenShift through two primary Custom Resources: `DSCInitialization` (DSCI) for platform-wide initialization (namespace setup, monitoring, service mesh, trusted CA bundles, authentication) and `DataScienceCluster` (DSC) for enabling/disabling individual AI/ML components such as Dashboard, Workbenches, KServe, ModelMesh, Data Science Pipelines, Model Registry, Ray, CodeFlare, Kueue, TrustyAI, Training Operator, Feast Operator, and LlamaStack Operator.

The operator uses a component-based architecture where each AI/ML component has its own sub-controller that manages the deployment of component-specific manifests via kustomize overlays stored in `prefetched-manifests/`. The operator also manages platform services including authentication (RBAC group management via Authorino), monitoring (Prometheus/OpenTelemetry integration), and service mesh configuration (Istio/Maistra integration for KServe). It includes admission webhooks for validating and mutating DataScienceCluster, DSCInitialization, HardwareProfile, InferenceService, Notebook, and Kueue-related resources.

The operator is built using controller-runtime and integrates deeply with OpenShift APIs (Routes, OAuth, SecurityContextConstraints, ImageStreams) as well as Kubernetes ecosystem projects (Knative Serving, Istio, Prometheus Operator, cert-manager, KEDA, Authorino). It deploys with 3 replicas for high availability and uses leader election for coordinated reconciliation.
