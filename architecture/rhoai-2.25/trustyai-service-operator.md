## Purpose
**Short**: A multi-controller Kubernetes operator that manages AI/ML trustworthiness, model evaluation, and guardrails orchestration services on OpenShift.

**Detailed**: The TrustyAI Service Operator manages three distinct but related services for AI/ML model trustworthiness on OpenShift:

1. **TrustyAI Service (TAS)**: Deploys and manages TrustyAI service instances that provide AI explainability and fairness monitoring. The controller watches InferenceService resources (KServe) and patches them with payload processor endpoints, enabling inference data capture for bias detection, drift monitoring, and model explainability. It integrates with both ModelMesh and KServe (raw and serverless) deployment modes, and supports PVC or database (Hibernate/JDBC) backends for data storage.

2. **LM Evaluation Service (LMES)**: Manages LMEvalJob custom resources that orchestrate language model evaluation tasks using the lm-evaluation-harness framework. The controller creates evaluation pods with configurable models (HuggingFace, OpenAI, WatsonX, etc.), task recipes (via Unitxt), and resource specifications. It integrates with Kueue for workload queuing and supports offline evaluation modes with S3 or PVC storage.

3. **Guardrails Orchestrator (GORCH)**: Deploys and manages guardrails orchestration infrastructure that intercepts and validates LLM inference traffic. The controller auto-detects KServe InferenceServices via label matching, auto-generates orchestrator and gateway configuration, and deploys the FMS Guardrails Orchestrator with optional built-in regex detectors and a sidecar gateway for streaming guardrails. It supports TLS-secured communication between components and OAuth proxy authentication.

## Architecture Components

| Component | Type | Purpose |
|-----------|------|---------|
| TrustyAIServiceReconciler (TAS) | Go Controller | Manages TrustyAI service deployments, oauth-proxy sidecars, OpenShift Routes, Services, ServiceMonitors, PVCs, and patches KServe InferenceServices with payload processors |
| LMEvalJobReconciler (LMES) | Go Controller | Creates evaluation pods for LM evaluation jobs, manages job lifecycle (New/Scheduled/Running/Complete/Cancelled/Suspended), communicates with driver sidecar via exec |
| GuardrailsOrchestratorReconciler (GORCH) | Go Controller | Deploys guardrails orchestrator with optional built-in detectors and gateway sidecar containers, auto-generates ConfigMaps from InferenceService discovery, creates OpenShift Routes and Services |
| LMEvalJob WorkloadManager (JOB_MGR) | Kueue Integration | Integrates LMEvalJob with Kueue workload scheduling framework for resource-aware job queuing |
| LMES Driver | Go Binary (cmd/lmes_driver) | Sidecar process running inside evaluation pods that manages evaluation lifecycle and reports results back to the controller |
