## Purpose
**Short**: Provides container runtime images and integration tests for distributed ML workloads (Ray, PyTorch/Kubeflow Training Operator, Kueue) in Red Hat OpenShift AI.

**Detailed**: The distributed-workloads repository serves two primary functions within the RHOAI ecosystem. First, it builds and ships GPU-accelerated container runtime images for distributed machine learning training. These images include Ray cluster runtimes (with CUDA and ROCm GPU support across multiple Python/PyTorch versions) and Kubeflow Training Operator (KFTO) runtime images for PyTorch-based distributed training jobs. The images are built via Konflux CI/CD pipelines defined in Tekton PipelineRun manifests.

Second, the repository contains a comprehensive Go-based end-to-end integration test suite that validates the distributed workloads stack on OpenShift. The tests exercise core components including KubeRay (RayCluster, RayJob), Kubeflow Training Operator (PyTorchJob), Kueue (ResourceFlavor, ClusterQueue, LocalQueue, Workload), and AppWrapper resources. Tests cover scenarios from basic smoke tests and MNIST training to LLM fine-tuning with DeepSpeed, hyperparameter optimization with Ray Tune, and Validating Admission Policy enforcement for queue-based workload management.

This repository does **not** contain an operator or controller — it is a companion repository that produces runtime container images consumed by other RHOAI components (CodeFlare Operator, KubeRay Operator, Kubeflow Training Operator, Kueue) and validates their integration on OpenShift clusters.
