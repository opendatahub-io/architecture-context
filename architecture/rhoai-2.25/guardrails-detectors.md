## Purpose
**Short**: A collection of detector algorithm microservices for the FMS Guardrails Orchestrator, providing content safety analysis via regex/heuristic, HuggingFace model, and LLM-as-Judge based detection.

**Detailed**: The guardrails-detectors repository provides three independent detector microservices that integrate with the [FMS Guardrails Orchestrator](https://github.com/foundation-model-stack/fms-guardrails-orchestrator). Each detector is packaged as its own container image and exposes a REST API conforming to the IBM Detector API specification.

The **built-in detector** provides lightweight, zero-dependency detection functions including regex-based PII detection (email, credit card, SSN, phone numbers, IP addresses, UK post codes) and file type validation (JSON, XML, YAML with optional schema validation). The **HuggingFace detector** is a generic ML-based detector compatible with `AutoModelForSequenceClassification` and `GraniteForCausalLM` models, supporting GPU/CUDA acceleration for content safety classification. The **LLM Judge detector** leverages the [vLLM Judge](https://github.com/trustyai-explainability/vllm_judge) library to use an LLM-as-a-judge architecture, connecting to an external OpenAI-compatible LLM server for content evaluation across multiple built-in metrics (safety, toxicity, accuracy, helpfulness, etc.).

All three detectors are built on FastAPI with Uvicorn, expose Prometheus metrics via `prometheus-fastapi-instrumentator`, and implement a common API schema for content analysis responses. They are deployed as KServe inference services within the RHOAI platform and are orchestrated by the FMS Guardrails Orchestrator.
