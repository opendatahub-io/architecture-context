## Purpose
**Short**: A reverse-proxy sidecar that translates KServe V2 REST inference requests into gRPC calls for ModelMesh Serving.

**Detailed**: The rest-proxy is a lightweight Go service that acts as a protocol translation layer between REST/HTTP clients and gRPC-based model inference servers. It leverages gRPC-Gateway to automatically generate a reverse-proxy server from the KServe V2 Predict Protocol protobuf definitions, enabling clients to send inference requests using the KServe V2 REST Predict Protocol to platforms that only support the gRPC V2 Predict Protocol.

The proxy is deployed as a sidecar container within ModelMesh Serving pods. It listens for incoming HTTP/HTTPS REST requests on a configurable port (default 8008), translates them into gRPC calls, and forwards them to the co-located ModelMesh gRPC inference service on localhost (default port 8033). The proxy handles custom JSON marshaling/unmarshaling for tensor data including support for multiple data types (BOOL, INT8-64, UINT8-64, FP16, FP32, FP64, BYTES), nested array flattening, and base64-encoded binary content. It supports optional TLS for both the REST listener and the upstream gRPC connection.

### TLS Configuration

The rest-proxy supports two independent TLS configurations:

1. **REST Listener TLS** (HTTPS): Enabled when `MM_TLS_KEY_CERT_PATH` and `MM_TLS_PRIVATE_KEY_PATH` environment variables are set. The proxy serves HTTPS using `http.ListenAndServeTLS`.

2. **gRPC Upstream TLS**: Enabled when `REST_PROXY_USE_TLS=true`. Uses `credentials.NewTLS` with configurable certificate verification (`REST_PROXY_SKIP_VERIFY`). When disabled, uses `insecure.NewCredentials()` for plaintext gRPC.

### FIPS Compliance

The Konflux build uses `GOEXPERIMENT=strictfipsruntime` and `-tags strictfipsruntime` build flags, ensuring the binary uses FIPS-validated cryptographic modules when running on RHEL/UBI with FIPS mode enabled.
