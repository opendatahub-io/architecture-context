## Purpose
**Short**: A framework for evaluating generative language models on standardized academic benchmarks and custom evaluation tasks.

**Detailed**: LM Evaluation Harness (lm-eval) provides a unified framework to test generative language models on a large number of different evaluation tasks. It supports over 60 standard academic benchmarks with hundreds of subtasks and variants, including GSM8K, MMLU, HellaSwag, ARC, TruthfulQA, and many more. The framework supports models loaded via HuggingFace Transformers, vLLM, OpenAI-compatible APIs, IBM watsonx.ai, and other backends.

In the RHOAI context, lm-eval is packaged as the `odh-ta-lmes-job-rhel9` container image, designed for secure, batch-oriented, one-shot evaluation workloads. It runs as a Kubernetes Job managed by the TrustyAI LMEval operator (from the `trustyai-service-operator` repository). The container provides a standardized runtime with Unitxt framework integration, offline metric evaluation support, custom task and metric injection directories, and output management for evaluation results. It builds natively on UBI9 Python 3.11 with multi-architecture support (amd64, arm64, ppc64le, s390x), including from-source builds of PyTorch, Arrow, and OpenBLAS for non-x86 architectures.
