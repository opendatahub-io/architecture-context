## Purpose
**Short**: A REST API server that stores and manages ML model metadata, including registered models, model versions, artifacts, inference services, serving environments, and experiments.

**Detailed**: The Model Registry provides a centralized metadata repository for machine learning workflows. It exposes an OpenAPI v1alpha3 REST API for creating, reading, updating, and deleting ML metadata entities such as Registered Models, Model Versions, Model Artifacts, Inference Services, Serving Environments, Experiments, and Experiment Runs. It replaces the legacy ml-metadata (MLMD) C++ gRPC service with a Go-based implementation that uses an embedded metadata store backed by MySQL or PostgreSQL databases.

The repository ships three main binaries: (1) the `proxy` server which runs the OpenAPI REST API on port 8080, (2) a `controller` binary which runs an InferenceService reconciler watching KServe InferenceService CRs and synchronizing their state with Model Registry entries, and (3) a `catalog` server that provides a read-only model catalog API sourcing from HuggingFace or YAML-based catalogs. Additionally, a CSI storage initializer binary (`mr-storage-initializer`) enables KServe to fetch model artifacts via `model-registry://` URIs.

The component is built using FIPS-compliant Go compilation (`strictfipsruntime`) for RHOAI, uses Konflux for CI/CD builds, and is deployed via Kustomize manifests with optional Istio service mesh integration for mTLS and traffic routing.
