NETWORK SECURITY NOTES
═══════════════════════════════════════════════════════════════════════════

⚠️  SECURITY CONSIDERATIONS:

1. Ray worker-to-head communication is UNENCRYPTED by default
   - GCS (port 6379): TCP plaintext, no authentication
   - Dashboard API (port 8265): HTTP plaintext, no authentication
   - Recommendation: Configure TLS for production deployments

2. No Service Mesh integration
   - Ray pods do NOT have Istio/service mesh sidecars
   - No automatic mTLS for pod-to-pod communication
   - Network policies recommended for isolation

3. Optional authentication
   - Ray Dashboard: Can enable Bearer token authentication
   - Ray Serve: Can enable authentication at application level
   - Default: No authentication required

4. Operator metrics exposed without authentication
   - Port 8080/metrics: Open for Prometheus scraping
   - Internal access only (ClusterIP service)

5. External Routes (optional, user-configured)
   - TLS termination at route level (TLS 1.2+)
   - Backend communication to Ray pods is HTTP (no encryption)
   - Users should enable Ray-level auth for external exposure

6. FIPS mode enabled in RHOAI builds
   - Built with -tags strictfipsruntime
   - Operator runs in FIPS-compliant mode
   - Ray pods: FIPS compliance depends on user-provided images
