Authentication & Authorization Flow:
1. User → Route (443/TCP HTTPS, no auth, redirect to OAuth)
2. OAuth Proxy → OpenShift OAuth Server (OAuth 2.0 flow)
3. OAuth Server validates user, issues JWT token
4. OAuth Proxy validates token, requires projects:list verb
5. OAuth Proxy forwards request to Backend with Bearer token
6. Backend uses ServiceAccount token for K8s API calls
7. Kubernetes API enforces ClusterRole/Role RBAC permissions
