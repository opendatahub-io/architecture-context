TRUST BOUNDARIES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. EXTERNAL → KUBERNETES API (Boundary: Cluster Ingress)
   - TLS 1.2+ termination at API server
   - Bearer token authentication (user credentials)
   - RBAC enforcement

2. KUBERNETES API → CONTROLLER (Boundary: Internal Control Plane)
   - ServiceAccount token authentication
   - Watch API for resource events
   - No direct network connection (event-driven)

3. KUBERNETES API → WEBHOOKS (Boundary: Admission Control)
   - mTLS authentication (K8s API Server client certs)
   - TLS 1.2+ encryption
   - NetworkPolicy restricts access to port 9443

4. CONTROLLER → KUBERNETES API (Boundary: Control Plane Access)
   - ServiceAccount token authentication
   - RBAC enforcement (odh-model-controller-role)
   - TLS 1.2+ encryption

5. CONTROLLER → EXTERNAL SERVICES (Boundary: Cluster Egress)
   - NGC API: HTTPS with API key authentication
   - S3 (via InferenceServices): HTTPS with IAM/access keys
   - No direct egress from controller to S3 (only InferenceServices)

6. PROMETHEUS → CONTROLLER (Boundary: Monitoring)
   - Bearer token authentication
   - HTTP (no encryption, internal network)
   - RBAC via prometheus-ns-access role

7. SERVICE MESH (Boundary: mTLS Enforcement)
   - KServe Serverless mode uses Istio mTLS STRICT
   - PeerAuthentication enforces mTLS between services
   - AuthorizationPolicies control access

8. INGRESS → INFERENCESERVICES (Boundary: External Access)
   - OpenShift Routes: Edge TLS termination (ModelMesh/Raw)
   - Istio Gateway: TLS 1.3 SIMPLE/MUTUAL (Serverless)
   - Authorino: Bearer token validation (optional)
