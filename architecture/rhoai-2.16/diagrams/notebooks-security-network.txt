RBAC Summary:
- No ClusterRoles/RoleBindings defined in Notebooks repository
- RBAC managed by Notebook Controller (creates per-notebook ServiceAccounts)
- ServiceAccount: {notebook-name}-sa (created in user namespace)
- RoleBindings: Created by Notebook Controller to grant namespace resource access
- User authorization: OAuth Proxy enforces that user is notebook owner or has namespace access

Service Mesh Configuration:
- OAuth Proxy sidecar: Provides TLS termination and OAuth2 authentication
- JupyterLab container: Listens on localhost only (8888/TCP), no direct external access
- Service mesh (optional): When enabled, provides mTLS for egress to RHOAI services
- Network policies: Applied by platform to restrict pod-to-pod communication

Secrets (Mounted at Runtime):
- aws-connection-* (Opaque)
  - Keys: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, endpoint, region
  - Purpose: S3 credentials for data access
  - Auto-rotate: No (user-managed)

- git-credentials (kubernetes.io/ssh-auth)
  - Keys: ssh-privatekey, known_hosts
  - Purpose: Git repository authentication
  - Auto-rotate: No (user-managed)

- database-credentials (Opaque)
  - Keys: username, password, host, port, database
  - Purpose: External database connections
  - Auto-rotate: No (user-managed)

- rhel-subscription-secret (Opaque) [Build-time only]
  - Purpose: RHEL subscription for RStudio BuildConfig
  - Provisioned by: Platform Admin
  - Auto-rotate: No

Security Hardening:
- Container user: non-root (UID 1001)
- SELinux: Enforced (OpenShift default)
- Seccomp: runtime/default profile
- Capabilities: Dropped all unnecessary capabilities
- Read-only root filesystem: No (writable for package installation)
- Persistent storage: /opt/app-root/src mounted from PVC (user data persistence)
- Resource limits: CPU and memory limits enforced by StatefulSet
- Network egress: Controlled by NetworkPolicies (if configured)

ImageStream Security:
- Images signed: Yes (Red Hat GPG signature)
- Image scanning: Yes (Clair, Quay Security Scanner)
- Base images: RHEL UBI9, RHEL9 (official Red Hat sources)
- Vulnerability management: Regular updates via Konflux CI/CD
- Image digests: SHA256 hashes in params.env for reproducibility

Workbench Isolation:
- Namespace isolation: Each user has dedicated namespace
- StatefulSet: One replica per workbench (no sharing)
- PVC: Dedicated persistent volume per workbench
- ServiceAccount: Unique per workbench instance
- OAuth enforcement: Only notebook owner can access (or authorized namespace users)
