ATTACK SURFACE ANALYSIS
════════════════════════════════════════════════════════════════════════════

Exposed Endpoints:
  1. /metrics (8080/TCP HTTP)
     - Risk: LOW (internal only, operational metrics)
     - Mitigation: ClusterIP service, PodMonitor-based access

  2. /healthz, /readyz (8081/TCP HTTP)
     - Risk: LOW (health checks only, no sensitive data)
     - Mitigation: Only accessed by kubelet probes

  3. Training pod-to-pod communication (23456/TCP, 2222/TCP)
     - Risk: MEDIUM (plaintext by default, namespace-scoped)
     - Mitigation: User-implemented application-level encryption
     - Note: Istio disabled to prevent interference

Privileged Operations:
  1. pods/exec permission
     - Risk: MEDIUM (allows exec into training pods)
     - Justification: Required for MPIJob launcher functionality
     - Mitigation: ClusterRole scoped, operator SA only

  2. Cross-namespace resource creation
     - Risk: MEDIUM (operator creates pods in user namespaces)
     - Mitigation: RBAC-controlled, only creates from validated CRDs

  3. No admission webhooks
     - Risk: MEDIUM (no validation/mutation of job specs)
     - Note: Webhook server disabled in RHOAI deployment
     - Mitigation: Relies on Kubernetes API validation of CRDs

Supply Chain:
  1. Container image: Built via Konflux with FIPS compliance
  2. Base image: registry.redhat.io/ubi8/ubi-minimal (Red Hat signed)
  3. Build: CGO_ENABLED=1, strictfipsruntime tags
